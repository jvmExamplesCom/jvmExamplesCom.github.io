Swapping Numbers Without Using Third Variable in Java

Swapping numbers without the need for a third variable can be a useful programming trick. In Java, this is achieved through the use of bitwise operators such as "^" and "&". This article will explain how to swap two numbers without a third variable in Java, and provide examples of how this can be implemented.

What is Bitwise Operators?

Bitwise operators are used to perform operations on binary representations of numbers, such as AND and OR. These operators are represented by symbols such as ‘&’, ‘|’, ‘^’, ‘~’, ‘<<’, and ‘>>’. The bitwise operators allow us to manipulate the 1s and 0s that represent the numbers in binary form, thus allowing us to perform arithmetic operations without using arithmetic operators. We can use the bitwise operators to manipulate the bits of a number to achieve our desired result. For example, using bitwise operators we can swap two numbers without a third variable.

Java Code Example: Swap Without Using a Third Variable

In the following Java code example, two numbers are swapped without the need for a third variable: 

          
           
int a = 5, b = 10;
 
a = a ^ b; // a becomes 15 (1111)
b = a ^ b; // b becomes 5 (0101)
a = a ^ b; // a becomes 10 (1010)

System.out.println("a = " + a);
System.out.println("b = " + b);
The output of the above program will be: 

          
           
a = 10
b = 5

Explanation of the Code

In the code, two variables ‘a’ and ‘b’ are declared and initialized with values 5 and 10 respectively. Then, using bitwise operators ‘^’ and ‘~’, the values of ‘a’ and ‘b’ are swapped without the need of a third variable. First, the bitwise XOR operator ‘^’ is applied on the two variables. This operator performs an exclusive OR operation on each bit of the numbers and the result is stored in ‘a’. In the above program, 'a' becomes 15 (1111 in binary). Now, the bitwise XOR operation is performed again, this time between ‘a’ and ‘b’. This results in ‘b’ becoming 5 (0101 in binary). Finally, the bitwise XOR operation is performed one last time, this time between ‘a’ and ‘b’. This results in ‘a’ becoming 10 (1010 in binary), thus achieving the purpose of swapping the two numbers without using a third variable.
