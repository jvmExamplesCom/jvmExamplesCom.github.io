Converting Byte to String in Java

Java provides multiple methods to convert a byte array into a string, depending on the character encoding used to encode the byte data. The two most popular character encodings are UTF-8 and ISO-8859-1.

Using UTF-8 Encoding

When encoding with UTF-8, you can use either the 

          
           
String
constructor or the 

          
           
getBytes
method of the 

          
           
String
class: 

          
           
// Using the String constructor
String s = new String(byteArray, StandardCharsets.UTF_8);

// Using the getBytes() method
String s = new String(byteArray).getBytes(StandardCharsets.UTF_8);
The 

          
           
StandardCharsets
class is part of the JDK and contains constants for common character encodings.

Using ISO-8859-1 Encoding

When encoding with ISO-8859-1, you can use the same 

          
           
String
constructor as with UTF-8, but the 

          
           
getBytes
method should be avoided: 

          
           
// Using the String constructor
String s = new String(byteArray, StandardCharsets.ISO_8859_1);
It is important to note that if you want to use the 

          
           
getBytes
method with ISO-8859-1 encoding, you must specify it explicitly, as the default encoding is UTF-8: 

          
           
// Not specifying the correct encoding results in an exception
String s = new String(byteArray).getBytes(); // Wrong!

// Specifying the correct encoding
String s = new String(byteArray).getBytes(StandardCharsets.ISO_8859_1);

Choosing the Right Encoding

When choosing which encoding to use, it is important to consider the purpose of your application. If the data is intended to be displayed in a web browser, it is best to use UTF-8 encoding, as it is the most widely supported encoding. On the other hand, if the data is intended for a local application or file, it is best to use ISO-8859-1, as it is more widely supported on these platforms.
