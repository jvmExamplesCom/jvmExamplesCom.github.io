Parsing a CSV File using StringTokenizer

Java provides a number of utility classes for parsing strings into their component parts. One such class is the 

          
           
StringTokenizer
. When presented with a string, the 

          
           
StringTokenizer
class can be used to break the string into its component parts, or tokens. This is particularly useful for parsing strings such as comma-separated values (CSV) files.

Creating a StringTokenizer Object

The 

          
           
StringTokenizer
class is part of the Java 

          
           
java.util
package and so must be imported into any class that wishes to use it. 

          
           
import java.util.StringTokenizer;
A 

          
           
StringTokenizer
object can then be created by passing in two arguments; the string to be tokenized, and the delimiter character that separates each token. 

          
           
StringTokenizer st = new StringTokenizer ("this,is,a,test",",");
In the above example, the string “this,is,a,test” will be tokenized using the comma character (‘,’) as the delimiter.

Parsing the String

Once the 

          
           
StringTokenizer
object has been created, the tokens can be extracted using the 

          
           
nextToken()
method. This method will return the next token as a string. It is important to note that the 

          
           
StringTokenizer
object progresses through the string from left to right. 

          
           
String s1 = st.nextToken();
String s2 = st.nextToken();
String s3 = st.nextToken();
String s4 = st.nextToken();
In the example above, the first call to 

          
           
nextToken()
will return the string “this”, the second “is”, the third “a”, and the fourth “test”.

Checking for More Tokens

It is important to check whether there are any more tokens before attempting to retrieve them. This can be done using the 

          
           
hasMoreTokens()
method. 

          
           
while(st.hasMoreTokens()) {
    String token = st.nextToken();
    // Do something with token
}
In the above example, the 

          
           
hasMoreTokens()
method is used to check whether there are any more tokens to be retrieved. If there are, the next token is retrieved using the 

          
           
nextToken()
method and stored in the 

          
           
token
variable.

Example

The following example shows how the 

          
           
StringTokenizer
class can be used to parse a CSV file. In this example, the CSV file contains three columns; “name”, “email”, and “phone”. 

          
           
import java.util.StringTokenizer;

String csv = "John,john@example.com,123456789";
StringTokenizer st = new StringTokenizer(csv,",");

String name = st.nextToken();
String email = st.nextToken();
String phone = st.nextToken();

System.out.println("Name: " + name);
System.out.println("Email: " + email);
System.out.println("Phone: " + phone);
When the above code is executed, the output will be: 

          
           
Name: John
Email: john@example.com
Phone: 123456789
