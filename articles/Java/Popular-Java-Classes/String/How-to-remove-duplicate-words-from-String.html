How to Remove Duplicate Words from a String in Java

Removing duplicate words from a string is a common task in programming. In this article, we will look at two ways to remove duplicate words from a given string in Java.

Method 1: Using Regular Expressions

The most straightforward way to solve this problem is by using the 

          
           
replaceAll()
method in Java with the help of regular expression (regex). We can use the following regex pattern to search for and remove every occurrence of duplicate words: 

          
           
\b(\w+)(?:(?:\s+\1\b)+)\s+
Here is the code snippet which uses this regex pattern to remove duplicate words from a given string: 

          
           
String input = "This is This the the output output";
String output = input.replaceAll("\\b(\\w+)(?:\\s+\\1\\b)+\\s+", "");
System.out.println("Input: " + input + "  Output: " + output);

Method 2: Using a HashSet

We can also use a 

          
           
HashSet
data structure to store the words encountered in the string and output it as the desired result. The idea is to iterate through all the words in the string and store them in the 

          
           
HashSet
. If a word is encountered again, we ignore it and do not store it. At the end of iteration, we return the contents of the 

          
           
HashSet
. Here is the Java code that implements this approach: 

          
           
String input = "This is This the the output output";
String[] words = input.split("\\s+");
Set<string> set = new HashSet<>();

for (String word : words) {
  set.add(word);
}

System.out.println("Input: " + input + "  Output: " + String.join(" ", set));

Conclusion

In this article, we looked at two ways of removing duplicate words from a given string in Java. The first approach used regular expressions combined with the 

          
           
replaceAll()
method and the second approach used a 

          
           
HashSet
data structure to store the words. We saw that both approaches produce the same output. Which approach you choose depends on your specific requirements and the context you are working in. If you need more control over the process, regular expressions might be the best choice, but if a simpler and faster solution is enough, then using a 

          
           
HashSet
could be the better option.
