Applet Directory URL or Code Base Example

The Applet Directory URL or Code Base is a way to represent the path to the directory or files, which an applet needs in order to run correctly. It generally contains the application code and class files for the applet.

Basic Syntax and Usage

The basic syntax for setting up the Applet Directory URL is the following:


          
           
<applet code="MyApplet" codebase="directoryURL">
 </applet>
Where the 
code
attribute defines the name of the applet class and the 
codebase
attribute represents the directory URL that contains the applet related files and classes.

Example Code

Here is a simple example of an applet that uses the directory URL to access the required resources:


          
           
<applet code="MyApplet" codebase="http://localhost/appletDir/" height="200" width="400">
	<param name="parameter1" value="value param 1" />
	<param name="parameter2" value="value param 2" />
</applet>
In this example, we are using the 
MyApplet
class, which is located inside the 

          
           
http://localhost/appletDir/
directory. We are also passing two parameters with 
parameter1
and 
parameter2
assigned to 
value param 1
and 
value param 2
respectively.

Output as HTML

Here is the HTML code that is generated when the example code is executed:


          
           
<applet code="MyApplet" codebase="http://localhost/appletDir/" height="200" width="400">
	<param name="parameter1" value="value param 1" />
	<param name="parameter2" value="value param 2" />
</applet>

Conclusion

The Applet Directory URL or Code Base is an essential concept for running applets correctly. It allows applications to reference the necessary files and classes without having to manually type out the full paths. By understanding how to use the code base attribute, you can easily create and deploy apps that can take advantage of applets.
