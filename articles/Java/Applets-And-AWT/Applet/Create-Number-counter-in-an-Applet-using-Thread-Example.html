Create Number Counter in an Applet Using Thread Example

Java applets provide a great way to create dynamic and interactive webpages. In this tutorial, we will learn how to create a number counter in an applet using thread example. This tutorial will cover the basic concepts of Java threads, and their various implementations. In order to use threads in a Java applet, there is some setup that needs to be done. First, the applet must subclass the Thread class. Then, the main() method must be called with the thread's name as an argument. For example: 

          
           
public class NumberCounter extends Thread { 
  public void run() { 
    for (int i = 0; i < 1000; i++) { 
      System.out.println(i); 
    } 
  } 
  public static void main(String[] args) { 
    new NumberCounter().start(); 
  } 
}
The above code snippet creates a new thread called NumberCounter. Inside the run() method, a loop is used to print out the numbers from 0 to 1000. Finally, the main() method is called with the thread's name as an argument. Once the thread is created, it can be started with the start() method. This will cause the thread to start running. The code inside the run() method will then be executed. In this case, the loop will print out the numbers from 0 to 1000. It is important to note that when a thread is started, the main thread will continue to execute any code after the call to start(). This means that it is possible to do other tasks while the number counter is running. For example, the main thread could check if the user has clicked a button, or display a message on the screen. Another important concept of threads is synchronization. Synchronization allows multiple threads to communicate with each other and coordinate their actions. It allows threads to safely access shared data without interfering with each other. To demonstrate synchronization, let's assume that we want to have two threads counting numbers from 0 to 10. Both threads should stop counting when one of them reaches 10. Here is the code for this example: 

          
           
public class SyncedNumberCounter extends Thread { 
  private int count = 0; 
  public void run() { 
    while (true) { 
      synchronized(this) { 
        if (count == 10) break; 
        System.out.println(count); 
        count++; 
      } 
    } 
  } 
  public static void main(String[] args) { 
    SyncedNumberCounter t1 = new SyncedNumberCounter(); 
    SyncedNumberCounter t2 = new SyncedNumberCounter(); 
    t1.start(); 
    t2.start(); 
  } 
}
In the above code, both threads attempt to access the same variable (count). Before any thread can access the variable, it must first obtain a lock on the object. This is done by synchronizing on the object (in this case, "this"). Once a thread obtains the lock, it can safely access the shared data. When the count is 10, the loop is broken and the threads stop counting. By using synchronization, we were able to ensure that both threads stopped counting once one of them reached 10. Let's create an applet that displays a number counter. The applet will have a button that starts the number counter and a text field that displays the numbers. Here is the code for this example: 

          
           
import java.awt.*; 
import javax.swing.*; 
public class NumberCounterApplet extends JApplet { 
  JButton startButton; 
  JTextField numberDisplay; 
  public void init() { 
    startButton = new JButton("Start"); 
    startButton.addActionListener(new NumberCounter()); 
    numberDisplay = new JTextField(20); 
    Container cp = getContentPane(); 
    cp.setLayout(new FlowLayout()); 
    cp.add(startButton); 
    cp.add(numberDisplay); 
  } 
} 
class NumberCounter implements ActionListener { 
  int count = 0; 
  public void actionPerformed(ActionEvent evt) { 
    new Thread() { 
      public void run() { 
        for (int i = 0; i < 1000; i++) { 
          numberDisplay.setText(String.valueOf(i)); 
        } 
      } 
    }.start(); 
  } 
}
The above code creates an applet with a button and a text field. The button's action listener is set to the NumberCounter class. The NumberCounter class creates a new thread, which counts from 0 to 1000 and updates the number display.
