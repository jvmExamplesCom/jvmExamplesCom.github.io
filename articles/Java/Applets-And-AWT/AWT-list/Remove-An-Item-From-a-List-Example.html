Remove An Item From a List Example in Java

Removing an item from a List is a common programming task, whether it's a LinkedList, ArrayList, Vector, or any other type of List. In this article, we will learn how to remove an item from a List example in Java.

Using the Iterator Class

The best way to remove an item from a List is to use the Iterator class. This approach allows us to safely iterate over the List and remove any items we don't want. The following example demonstrates how to remove an item from a List by using the Iterator class. 

          
           
List<String> list = new ArrayList<String>();
list.add("apple");
list.add("orange");
list.add("banana");

Iterator<String> itr = list.iterator();
while (itr.hasNext()) {
  String item = itr.next();
  if (item.equals("banana")) {
    itr.remove();
  }
}

// Output: [apple, orange]
In this example, we first create an ArrayList and add three items - apple, orange and banana. Then, we create an Iterator and loop through the List, checking for the item we want to remove. If the item equals banana, we use the Iterator's remove() method to remove the item from the List. After iterating through the List, we print out the remaining items.

Using the List Interface

We can also use the List interface to remove an item from a List. The following example shows how to remove an item from a List using the remove() method of the List interface. 

          
           
List<String> list = new ArrayList<String>();
list.add("apple");
list.add("orange");
list.add("banana");

list.remove("banana");

// Output: [apple, orange]
In this example, we first create an ArrayList and add three items - apple, orange and banana. Then, we use the remove() method of the List interface to remove the item banana from the List. Finally, we print out the remaining items.

Conclusion

In this article, we learned how to remove an item from a List example in Java. We saw two different ways to remove an item from a List - using the Iterator class and using the List interface. Both approaches are valid, but it is important to understand which one is more suitable for your particular use case.
