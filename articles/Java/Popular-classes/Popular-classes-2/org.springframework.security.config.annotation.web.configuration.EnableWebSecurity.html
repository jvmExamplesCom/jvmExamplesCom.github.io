org.springframework.security.config.annotation.web.configuration.EnableWebSecurity

The 
EnableWebSecurity
annotation is an annotation that enables Spring Security web security support. It adds an additional layer of security to existing applications by applying a number of security constraints and measures in order to protect against potential threats or malicious attacks.

This annotation is placed on the configuration class where you want to add web security processing. It is typically used within the root configuration class of your application. For example:


          
           
@EnableWebSecurity
public class AppConfig extends WebSecurityConfigurerAdapter{
}

Once this annotation is present, the framework will start looking for any beans of type 


          
           
AuthenticationManager
to be configured. If one is not found, it will create a default one. The same goes for 
UserDetailsService
and other related beans. The 
EnableWebSecurity
annotation also triggers auto-configuration of other important components of the security framework, such as 
HttpSecurity
. This auto-configuration process is driven by the presence of the 

          
           
WebSecurityConfigurerAdapter
class. If present, the framework will look for any methods annotated with the 
@Configuration
annotation and use those to configure Spring Security.

The auto-configuration process can be extended and modified by overriding the 


          
           
configure(HttpSecurity)
method. This method takes an 
HttpSecurity
object which can then be used to define custom security rules and constraints. For example: 

          
           
@Override
protected void configure(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .antMatchers("/admin/**").hasRole("ADMIN")
            .anyRequest().authenticated()
        .and()
        .formLogin()
        .and()
        .logout();
}

This example defines a rule that only users with the 

ROLE_ADMIN
can access URLs starting with 
/admin/
. All other requests must be authenticated in order to proceed. The 
formLogin()
and 
logout()
methods are added to enable basic login and logout support. The 
configure()
method can also be used to configure additional features such as session management, multitenancy and more. Furthermore, if you need more fine-grained control over how requests are mapped and handled, you can create multiple 

          
           
WebSecurityConfigurerAdapter
classes and register them with the framework. The 
EnableWebSecurity
annotation provides a simple and powerful way to enable web security in Spring applications. And by leveraging the auto-configuration process, it is easy to configure common security features without having to write a lot of code.
