javax.servlet.http.Cookie

Cookies are small pieces of data that is stored on the client-side web browser when the user visits a website. They can be used to store information such as login credentials and preferences, or be used to track the userâ€™s browsing behavior. The javax.servlet.http.Cookie class provides methods to access these cookies on the server-side.

Creating a Cookie

We can create a new Cookie by using the Cookie constructor, specifying its name and value. Both the name and value need to be URL encoded:


          
           
String name = "user_name";
String value = URLEncoder.encode( "John Smith", "UTF-8" );
Cookie cookie = new Cookie( name, value );

We can set additional attributes for the cookie, such as its lifetime or path. By default, cookies have a lifetime of -1 (which means it lasts until the browser is closed). We can change this by setting the 

setMaxAge()
method: 

          
           
cookie.setMaxAge( 60 ); // lifetime of 60 seconds
We can also specify the cookie path: 

          
           
cookie.setPath( "/" ); // available in all pages

Adding Cookies to the Response

Once we have created the cookie, we can add it to the response:


          
           
response.addCookie( cookie );

Reading Cookies from the Request

We can access the cookies sent by the client by using the 


          
           
request.getCookies()
method. This returns an array of 
Cookie
objects, which we can loop through: 

          
           
Cookie[] cookies = request.getCookies();
for ( Cookie cookie : cookies ) {
    String name = cookie.getName();
    String value = cookie.getValue();

    if ( name.equals( "user_name" ) ) {
        value = URLDecoder.decode( value, "UTF-8" );
        // do something with the value
    }
}

Deleting a Cookie

To delete a cookie, we can set its max age to 0: 

          
           
cookie.setMaxAge( 0 );

This will instruct the browser to delete the cookie. Note that this only works if the cookie path and domain matches the current request.
