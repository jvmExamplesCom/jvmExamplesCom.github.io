androidx.fragment.app.Fragment

A 

          
           
Fragment
is a piece of an application's user interface or behavior that can be placed in an 

          
           
Activity
. Fragments represent a behaviour or a portion of user interface in an 

          
           
Activity
. You can combine multiple fragments in a single 

          
           
Activity
to build a multi-pane UI and reuse a fragment in multiple activities. Fragments were introduced in Android 3.0 (API level 11) to support more dynamic and flexible UI designs on large screens, such as tablets. A fragment must always be embedded in an 

          
           
Activity
and the 

          
           
Fragment
class takes care of creating and managing its views.

Creating a Fragment

The simplest way to create a 

          
           
Fragment
is to derive from the 

          
           
androidx.fragment.app.Fragment
class and implement the 

          
           
onCreateView()
callback method to return a View hierarchy to display in the 

          
           
Activity
or in another 

          
           
Fragment
. The following code demonstrates how to create a simple 

          
           
Fragment
: 

          
           
public class MyFragment extends androidx.fragment.app.Fragment {

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container, Bundle savedInstanceState) {
        // Inflate the layout for this fragment
        return inflater.inflate(R.layout.my_fragment, container, false);
    }
}
In this example, the 

          
           
onCreateView()
callback method inflates a layout resource (located in the file 

          
           
res/layout/my_fragment.xml
) into the container 

          
           
ViewGroup
specified. The 

          
           
fragment
's 

          
           
ViewGroup
(the one returned by the 

          
           
onCreateView()
method) is the parent of any views contained within the 

          
           
Fragment
.

Adding a Fragment to an Activity

Once you've created a 

          
           
Fragment
, you can add it to an 

          
           
Activity
using an 

          
           
FragmentTransaction
. An 

          
           
FragmentTransaction
is used to perform any number of operations, such as add, remove, replace, or animate 

          
           
Fragment
s. Each 

          
           
FragmentTransaction
is added to a back stack of transactions, and the back stack allows users to navigate backward through the 

          
           
Fragment
transactions (just like the activity back stack). The following code demonstrates how to add a 

          
           
Fragment
to an 

          
           
Activity
: 

          
           
MyFragment fragment = new MyFragment();
FragmentManager fragmentManager = getSupportFragmentManager();
FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();
fragmentTransaction.add(R.id.container, fragment);
fragmentTransaction.commit();
In this example, a new instance of 

          
           
MyFragment
is created and the 

          
           
FragmentTransaction
is used to add it to the 

          
           
Activity
. The 

          
           
R.id.container
represents the 

          
           
ViewGroup
in the 

          
           
Activity

Fragment
.

Removing Fragments

Just as you can add a 

          
           
Fragment
to an 

          
           
Activity
, you can also remove a 

          
           
Fragment
. To do so, you use the 

          
           
remove()
method of the 

          
           
FragmentTransaction
. The following code demonstrates how to remove a 

          
           
Fragment
from an 

          
           
Activity
: 

          
           
MyFragment fragment = (MyFragment)getSupportFragmentManager().findFragmentById(R.id.my_fragment);
FragmentManager fragmentManager = getSupportFragmentManager();
FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();
fragmentTransaction.remove(fragment);
fragmentTransaction.commit();
In this example, the instance of 

          
           
MyFragment
is retrieved from the 

          
           
FragmentManager
and then removed using the 

          
           
FragmentTransaction
.

Replacing Fragments

You can also replace one 

          
           
Fragment
with another using the 

          
           
replace()
method of the 

          
           
FragmentTransaction
. This is useful when your application requires different views of data at different times. The following code demonstrates how to replace a 

          
           
Fragment
in an 

          
           
Activity
: 

          
           
MyNewFragment newFragment = new MyNewFragment();
FragmentManager fragmentManager = getSupportFragmentManager();
FragmentTransaction fragmentTransaction = fragmentManager.beginTransaction();
fragmentTransaction.replace(R.id.container, newFragment);
fragmentTransaction.commit();
In this example, a new instance of 

          
           
MyNewFragment
is created and the 

          
           
FragmentTransaction
is used to replace the existing 

          
           
Fragment

Conclusion

The 

          
           
androidx.fragment.app.Fragment
class is a powerful way to create re-usable chunks of user interface and behavior in an
