org.springframework.security.core.userdetails.UserDetailsService

The org.springframework.security.core.userdetails.UserDetailsService interface is an important interface used in authentication and authorization, especially in Spring Security. It provides the methods needed to retrieve user information from a configured user repository or store.

Implementation

A typical implementation of the UserDetailsService interface would look something like this:


          
           
public class MyUserDetailsService implements UserDetailsService {

    private UserRepository userRepository;

    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        Optional<User> user = userRepository.findByUsername(username);

        if (!user.isPresent()) {
            throw new UsernameNotFoundException("Username not found.");
        }

        return user.get();
    }
}

In the example above, we have a UserRepository class which contains a findByUsername method. This method will be used to locate the user in question, and if it's not found then a UsernameNotFoundException will be thrown.

UserDetails Object

Once the user details are retrieved, they are encapsulated in a UserDetails object. This is an important part of the authentication process as it contains all the data needed to validate the user. The information stored in the UserDetails object includes the user's username, password, granted authorities, and other relevant data.


          
           
UserDetails userDetails = new User(
    user.getUsername(),
    user.getPassword(),
    user.getRoles());

The UserDetails object is then passed to the authentication provider for validation. Once the authentication provider has verified the credentials, it calls the AuthenticationManager to log the user into the application.

Using

Once the UserDetailsService is implemented, it can be used in an AuthenticationProvider. For example, the following code snippet shows how to use the UserDetailsService in an LDAPAuthenticationProvider:


          
           
LDAPAuthenticationProvider ldapAuthenticationProvider = new LDAPAuthenticationProvider(
    userDetailsService,
    ldapBindAuthenticator);

Here, the userDetailsService is being injected into the LDAPAuthenticationProvider. The LDAPAuthenticationProvider will then use the UserDetailsService to lookup the user in question, and verify the credentials.
