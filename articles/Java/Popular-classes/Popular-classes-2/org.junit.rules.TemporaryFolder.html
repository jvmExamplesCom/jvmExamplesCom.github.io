org.junit.rules.TemporaryFolder

The org.junit.rules.TemporaryFolder rule allows creation of files and folders that are guaranteed to be deleted when the test method finishes (whether it passes or fails). It also provides support for listing certain files and folders.

Creating Temporary Folders and Files

You can create temporary folders and files within your JUnit tests by using the 

create()
methods provided by the TemporaryFolder class. The following example demonstrates how to use the 
create()
methods: 

          
           
@Rule
public TemporaryFolder tempFolder = new TemporaryFolder();

@Test
public void testCreateTempFolderWithFile() throws IOException {
    File dir = tempFolder.create("test");
    File file = tempFolder.newFile("test.txt");
    Assert.assertTrue(dir.isDirectory());
    Assert.assertTrue(file.exists());
}

In the above example, the TemporaryFolder class is used to create a directory named "test" and a file named "test.txt". The 

create()
methods returns a File object which can be used to further manipulate the created directory and file.

Listing Folders and Files

The TemporaryFolder class also provides methods for listing existing folders and files. The following example demonstrates how to use the

@Test

listFiles()
