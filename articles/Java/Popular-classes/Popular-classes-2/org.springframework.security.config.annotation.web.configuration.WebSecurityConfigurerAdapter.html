org.springframework.security.config.annotation.web.configuration.WebSecurityConfigurerAdapter

The WebSecurityConfigurerAdapter class is a helpful framework which can be used to configure web security for an application. This adapter class provides base implementation of WebSecurityConfigurer which allows fine-grained control over web security. It integrates with Spring Security's authentication and access control features, providing an easy-to-use way to apply basic security within an application.

Configuring Security with WebSecurityConfigurerAdapter

The adapter has two methods for configuring security:

  • configure(HttpSecurity) - This is the primary method used for configuring security rules for individual requests.
  • configure(AuthenticationManagerBuilder) - This is used to configure global security settings, such as user authentication.

The following example shows a sample configuration using WebSecurityConfigurerAdapter.


          
           
@Configuration
public class SecurityConfiguration extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/users/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            .and()
            .formLogin()
                .loginPage("/login")
                .permitAll();
    }

    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        auth.inMemoryAuthentication()
            .withUser("user1")
            .password("{noop}password")
            .roles("USER");
    }
}

The above configuration will require users to authenticate before they can access any URL which matches the 

/users/**
pattern. If they have the 
ADMIN
role, they will be allowed access. Otherwise, they will be redirected to a login page. The 

          
           
configure(AuthenticationManagerBuilder)
method is used to define an in-memory user and role configuration.
