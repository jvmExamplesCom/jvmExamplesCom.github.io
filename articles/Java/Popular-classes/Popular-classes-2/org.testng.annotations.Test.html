org.testng.annotations.Test

The org.testng.annotations.Test annotation is used to mark a method as a test method in TestNG. This annotation is used to define the number of times a particular test should be repeated, specify parameters for the test method, specify dependencies for the test method, and much more. This annotation helps developers to create automated tests for their application easily and effectively.

Usage of the @Test Annotation

The @Test annotation can be used to mark any method as a test method. If the method is marked as a @Test annotation, it will be executed when the TestNG suite runs. The following code demonstrates how to use the @Test annotation to mark a method as a test method: 

          
           
@Test
public void myTestMethod(){
   // Code for your test
}

Parameters of the @Test Annotation

The @Test annotation has several parameters that can be used to configure the test methods. Here are a few of the most commonly used parameters:
  • enabled: The enabled parameter specifies whether or not the test method should be executed. The default value is true. To disable the test, set this parameter to false.
  • timeOut: The timeOut parameter specifies the timeout in milliseconds after which the test method should be considered failed if it has not completed.
  • dependsOnMethods: The dependsOnMethods parameter specifies the list of test methods on which the test method depends.
  • description: The description parameter specifies a brief description about the test method.

Example Code for the @Test Annotation

Here is an example of how to use the @Test annotation with multiple parameters: 

          
           
@Test (enabled = true, timeOut = 5000, dependsOnMethods = {"testMethodA", "testMethodB"}, description = "This test will execute after testMethodA and testMethodB")
public void myTestMethod(){
   // Code for your test
}
In this example, the test method myTestMethod() will be executed after both testMethodA and testMethodB have been executed and it will timeout after 5000 milliseconds. The description parameter provides a brief description about the test.

The org.testng.annotations.Test annotation is used to mark a method as a test method in TestNG. This annotation has several parameters which can be used to configure the test method. The example above demonstrates how to use the @Test annotation with multiple parameters.
