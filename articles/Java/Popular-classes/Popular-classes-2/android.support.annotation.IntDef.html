android.support.annotation.IntDef

android.support.annotation.IntDef is an annotation type that can be used to restrict the integer constants used in your program. This annotation can be used to specify a restricted set of integral constants that can be assigned to a given variable or parameter. When using this annotation, the compiler will only accept values that are in the defined set and fails with an error message if a value outside of this set is used.

This annotation is useful for type safety and can be used to reduce the risk of development mistakes by ensuring that only valid values are used. It is also useful for documentation purposes, as it provides an explicit list of accepted values. This annotation was introduced in Android API level 11 (Android 3.0) and is part of the Android Support Annotations library.

Syntax

The syntax for android.support.annotation.IntDef annotation is as follows:


          
           
@IntDef({value1, value2, ...})
public @interface AnnotatedType {
}

where value1, value2, … are integral constant values.

Example

Consider a 
Car
class with an attribute for its color. We want to define a set of accepted color values and use 
@IntDef
annotation to enforce that only valid values are used. 

          
           
public class Car {
    public static final int COLOR_RED = 0;
    public static final int COLOR_BLUE = 1;
    public static final int COLOR_GREEN = 2;

    @IntDef({COLOR_RED, COLOR_BLUE, COLOR_GREEN})
    public @interface Colors {}

    private Colors color;

    public void setColor(@Colors int newColor){
        this.color = newColor;
    }
}
Now, when creating an instance of 
Car
, we can only use the predefined color values: 

          
           
//This is valid
Car car1 = new Car();
car1.setColor(Car.COLOR_RED);

//This will cause a compilation error
Car car2 = new Car();
car2.setColor(4);
