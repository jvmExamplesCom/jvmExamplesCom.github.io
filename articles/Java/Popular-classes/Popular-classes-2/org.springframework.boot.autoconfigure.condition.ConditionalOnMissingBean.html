org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean

The 

          
           
org.springframework.boot.autoconfigure.condition.ConditionalOnMissingBean
annotation is used to determine if a bean should be configured when no bean is already defined for the injection point. The annotation is typically applied to fields, constructor parameters and methods in configuration classes to conditionally create beans depending on the presence or absence of other beans.

Code Example 1


          
           
public class MyService {

    @Autowired
    private MyRepository myRepository;

    @Autowired 
    @ConditionalOnMissingBean
    private MyExternalService myExternalService;

    public void doSomething() {
        // Do something with the Repository
    }
}

In the example above, a missing bean (MyExternalService) will not cause an issue. If the MyExternalService bean is found during the Spring Boot context initialization, it will be injected into the service instance.

Code Example 2


          
           
@Configuration
@AutoConfigureAfter(MyOtherConfiguration.class)
public class MyConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public MyService myService(MyRepository myRepository) {
       return new MyService(myRepository);
    }
}

In this example, the MyService bean is created only if there is no other bean with the same type already defined in the application context. This allows for customizing the service implementation if needed.

Code Example 3


          
           
@Configuration
public class MyConfiguration {

   @Autowired
   private MyRepository myRepository;
   
   @Bean
   @ConditionalOnMissingBean({MyService.class})
   public MyService myService() {
      return new MyService(myRepository);
   }
}

In this example, the MyService bean is created if there is no other bean of type MyService already defined in the application context.

Code Example 4


          
           
@Configuration
public class MyConfiguration {

    @Autowired
    private MyRepository myRepository;

    @Bean
    @ConditionalOnMissingBean(name="myService")
    public MyService myService() {
        return new MyService(myRepository);
    }
}

In this example, the MyService bean is created if a bean named "myService" is not already defined in the application context.
