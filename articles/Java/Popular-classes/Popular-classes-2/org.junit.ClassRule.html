org.junit.ClassRule

The 

          
           
org.junit.ClassRule
is a powerful API within the JUnit testing framework used to define rules that can be applied to test classes at runtime. This API is a great tool for organizing tests consistently and ensuring repeatability between tests. Overview of org.junit.ClassRule

The 

          
           
org.junit.ClassRule
API allows developers to define a rule that is applied to a test class before each test begins. This means that any changes made by a rule will be reverted before the next test begins so that the tests remain isolated and repeatable. For example, let's say you have a set of tests which all require a specific object setup. You could create a SetupRule that sets up the objects before each test runs and a TeardownRule that resets the objects afterwards. Any changes made by these rules would then be reverted before the next test begins.

Using org.junit.ClassRule
