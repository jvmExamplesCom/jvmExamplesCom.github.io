org.springframework.transaction.annotation.EnableTransactionManagement

org.springframework.transaction.annotation.EnableTransactionManagement enables transaction management using annotation-driven configuration. This allows for annotations such as the @Transactional annotation to be used to automatically manage the transactions. This annotation can be placed on classes or methods and it will activate transaction management for all methods in the class, limiting their ability to perform any modifications that aren't part of the transaction.

In order to use this annotation, a transaction manager must be configured. The transaction manager is responsible for managing the transactions and ensuring that all modifications are committed or rolled back if there is an error.

Example code:


          
           
@Configuration
@EnableTransactionManagement
public class TransactionManagementConfiguration {
}

The above code shows a simple configuration class that enables transaction management using the @EnableTransactionManagement annotation. The important thing to note here is that no other configuration is required; the annotation alone is sufficient to enable transaction management.

Transaction Example:


          
           
@Transactional
public void doSomething() {
   // Data modification code
   doSomething();
   doSomethingElse();
   // More data modification code
   persistData();
}

In the above code example, the @Transactional annotation is used to mark the method as transactional. Any changes performed within the method will be managed by a transaction, and if there is an exception, the changes will be rolled back automatically.
