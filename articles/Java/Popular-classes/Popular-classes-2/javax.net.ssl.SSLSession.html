javax.net.ssl.SSLSession

The javax.net.ssl.SSLSession is an interface representing an SSL session. It enables access to session-specific state information and provides a secure communication between two peers over the Secure Socket Layer (SSL) protocol.

This interface enables applications to access session-specific security attributes such as cipher suite and peer certificates. This interface also enables applications to create a new SSLSession object for an underlying SSL connection.

Creating a New SSLSession

To create a new SSLSession, an application has to define the protocol, whether it is TLS or SSL, and then select the desired cipher suite. Once the protocol and cipher suite is selected, the application can call the 

createSSLSession()
method on the 
SSLContext
class to create a new 
SSLSession
instance. 

          
           
SSLContext sslCtx = SSLContext.getInstance("TLS");
sslCtx.init(keyManagers, trustManagers, null);

String[] cipherSuites = { "SSL_DH_anon_WITH_RC4_128_MD5" };
SSLSession session = sslCtx.createSSLSession(cipherSuites);

Once an 
SSLSession
instance is created, the application can access its security attributes using the 
getCipherSuite()
method and the 

          
           
getPeerCertificates()
method. The 
getCipherSuite()
method returns the name of the cipher suite used in this session and the 

          
           
getPeerCertificates()
method returns the peer's certificate chain. 

          
           
System.out.println("Cipher Suite: " + session.getCipherSuite());

X509Certificate[] certs = session.getPeerCertificates();

}
