javax.xml.xpath.XPathFactory

Java provides an XPath API with several implementations that are available for use. The javax.xml.xpath.XPathFactory class is used to create an XPath implementation and it serves as the entry point of the XPath API.

Creating an XPath object

The static newInstance() factory method of the XPathFactory takes a string parameter specifying the version of the API that we want to use and returns an XPath object. 

          
           
XPath xpath = XPathFactory.newInstance("XPath 1.0").newXPath();

Using the XPath Expression


          
           
String expression = "/root/node";
NodeList nodes = (NodeList) xpath.evaluate(expression, sourceDoc, XPathConstants.NODESET);
for (int i = 0; i < nodes.getLength(); i++) {
    Node node = nodes.item(i);
    // process the node
}
The XPath expressions used on an XML document can be very powerful and concise. It is also possible to evaluate expressions using namespaces, using a prefix-to-namespace mapping. The namespaces can be declared on the XPath object itself or on the source document as attributes of the root element. 

          
           
XPath xpath = XPathFactory.newInstance("XPath 1.0").newXPath();
Map<String, String> prefixes = new HashMap<String, String>();
prefixes.put("prefix", "urn:myns:v1");
xpath.setNamespaceContext(new SimpleNamespaceContext(prefixes));
String expression = "/prefix:root/prefix:node";
NodeList nodes = (NodeList) xpath.evaluate(expression, sourceDoc, XPathConstants.NODESET);
for (int i = 0; i < nodes.getLength(); i++) {
    Node node = nodes.item(i);
    // process the node
}

Evaluating Expressions

The evaluate() method of the XPath object is used to evaluate an expression and return a result object that corresponds to the type argument passed to the method. We can use one of the constants from the XPathConstants class to specify the type of the result object to be returned. 

          
           
Node node = (Node) xpath.evaluate("/root/node[@id='1']", sourceDoc, XPathConstants.NODE);
// process the node
We can also evaluate XPath expressions that return a Boolean result: 

          
           
boolean result = (Boolean) xpath.evaluate("count(//node[@status='active']) > 0", sourceDoc, XPathConstants.BOOLEAN);

Compiling Expressions

The compile() method of the XPath object can be used to compile an expression into an XPathExpression object, which can be used to efficiently evaluate the same expression multiple times. 

          
           
XPathExpression expr = xpath.compile("/root/node[@id='1']");
Node node = (Node) expr.evaluate(sourceDoc, XPathConstants.NODE);
// process the node

Conclusion

The javax.xml.xpath.XPathFactory class is the entry point of the Java XPath API. Using the XPathFactory we can create an XPath object, compile expressions and evaluate them against an XML document. The evaluate() method can return different types of objects depending on the type argument passed to it, such as a NodeList of matching nodes, a single Node object or a Boolean value.
