Introduction to org.hamcrest.Matchers

org.hamcrest.Matchers is a library of static methods for creating matcher objects. These matcher objects are used to verify the content of an object. Matchers allow for improved readability of tests and greater flexibility in defining conditions for test success.

Matcher Creation

To create a matcher, use one of the static methods provided by the org.hamcrest.Matchers class. The method should be provided with one or more arguments that define the condition to be tested.


          
           
import static org.hamcrest.Matchers.*;

public class Test {

    public void test() {
        // Create a matcher that checks if "foo" is equal to "foo"
        Matcher matcher = equalTo("foo");
    }
}

Matcher Evaluation

Once a matcher has been created, it can be evaluated against a given value. For example, to check if the given value is true:


          
           
boolean result = matcher.matches(true);

The result will be true if the condition defined by the matcher is satisfied, and false otherwise.

Examples of Matchers

Here are some examples of commonly used matchers with the corresponding condition they test for:


          
           
equalTo(x):         x == expectedValue
    not(x):               x != expectedValue
    anything():           x is any value
    is(x):                x instanceof expectedClass
    isA(x):               x.getClass() == expectedClass
    notNullValue():       x != null
    nullValue():          x == null
