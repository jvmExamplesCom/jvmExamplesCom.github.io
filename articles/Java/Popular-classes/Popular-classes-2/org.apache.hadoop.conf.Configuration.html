org.apache.hadoop.conf.Configuration

The org.apache.hadoop.conf.Configuration class provides a way to manage configuration of Hadoop components. It is used by clients, admin tools and daemons to read the configuration information for the entire Hadoop system.

Creating a Configuration Object

To create a Configuration object, the following code can be used:


          
           
Configuration conf = new Configuration();

Loading Configuration from a File

It is possible to load configuration from a file. The following code shows how this can be done:


          
           
Configuration conf = new Configuration();
conf.addResource("config.xml");

Setting Configuration Properties

It is also possible to set configuration properties directly. The following code shows how this can be done:


          
           
Configuration conf = new Configuration();
conf.set("name", "value");

Accessing Configuration Properties

To access configuration properties, the following code can be used:


          
           
Configuration conf = new Configuration();
String value = conf.get("name"); // retrieves the value for the property "name"

Configuration Resources

Configuration instances can contain multiple resources. A resource is a file used for configuration. The following code shows how a resource can be added:


          
           
Configuration conf = new Configuration();
conf.addResource("config.xml"); // adds the file "config.xml" as a resource

The following code shows how resources can be removed:


          
           
Configuration conf = new Configuration();
conf.removeResource("config.xml"); // removes the file "config.xml" as a resource

Updating Configuration Resources

Configuration resources can be updated using the reloadConfiguration() method. The following code shows how this can be done:


          
           
Configuration conf = new Configuration();
conf.reloadConfiguration(); // reloads all configuration resources
