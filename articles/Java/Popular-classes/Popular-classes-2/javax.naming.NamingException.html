javax.naming.NamingException

javax.naming.NamingException is an exception that is thrown when a program tries to access the information stored on a remote server. It is a type of checked exception, which means that the code needs to deal with it before continuing execution. It is thrown by the JNDI (Java Naming and Directory Interface) API.

The NamingException class defines a standard set of exceptions that can be thrown when something goes wrong while accessing a Java naming service. These include NameNotFoundException, InvalidNameException, and CompoundNameException. All of these subclasses are descended from NamingException.

The most common situation in which this exception may be thrown is when an application attempts to look up an object in the directory service, but the object does not exist. In addition, NamingException may be thrown when an application tries to bind a name to an object, but the name is already bound to another object.

The following code example demonstrates how to catch a NameNotFoundException:
