Introduction to MockitoJUnitRunner

MockitoJUnitRunner is an extension of JUnit's BlockJUnit4ClassRunner which provides support for using Mockito in unit tests. It takes care of initializing mocks and validating the framework's state throughout the test execution. It is often used when the test subject depends on external collaborators, allowing the test author to isolate the test subject from its environment.

Creating a Unit Test with MockitoJUnitRunner

Creating a unit test with MockitoJUnitRunner is straightforward. The following code example demonstrates how to initialize a test class with the Runner: 

          
           
@RunWith(MockitoJUnitRunner.class)
public class MyTest {
    // test methods go here
}
The @RunWith annotation is used to specify which runner should be used for executing the unit test. In this case, MockitoJUnitRunner is specified as the Runner for executing the MyTest class.

Mocking Collaborators

In a unit test, it is frequently necessary to isolate the test subject from its environment by replacing collaborators with mocks. The following code example demonstrates how to create a mock object with MockitoJunitRunner: 

          
           
@Mock
private Collaborator collaborator;
The @Mock annotation is used to create a mock object instance of the specified type. This allows the mock object to be referenced within the test class.

Validating the Test State

MockitoJUnitRunner allows us to validate the test state after every test execution. We can use the @Verify annotation to validate the state of our test, as illustrated below: 

          
           
@Verify
public void verify() {
    verify(collaborator).someMethod(); 
}
The @Verify annotation is used to validate the state of a unit test. In this example, the verify() method is executed after each test execution, verifying that the collaborator’s someMethod() was called.

Conclusion

MockitoJUnitRunner is a powerful tool for unit testing that simplifies the process of initializing mocks, isolating test subjects from their environment, and validating the test state. With MockitoJUnitRunner, we can quickly set up unit tests with all the features we need to run efficient and effective tests.
