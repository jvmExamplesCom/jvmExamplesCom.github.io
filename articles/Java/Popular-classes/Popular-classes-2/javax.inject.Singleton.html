javax.inject.Singleton

Introduction

javax.inject.Singleton is an interface that can be annotated to a class, making it a singleton. A singleton is a type of class for which only one instance exists throughout the life of the application. It is useful for classes, such as factories and utility classes, that do not need to create multiple instances.

Example

Let's look at an example of using javax.inject.Singleton to make a class a singleton:


          
           
@Singleton
public class MyFactory {

    public Object getObject() {
        return new Object();
    }
}

Now, we can access this singleton by simply injecting it into another class:


          
           
public class MyClass {

    @Inject
    MyFactory myFactory;

    public Object getObject() {
        return myFactory.getObject();
    }
}

The myFactory object will now always be the same instance, no matter how many times it is requested.

Benefits

Using javax.inject.Singleton comes with several benefits:

  • Performance benefits due to avoiding costly object creation
  • Easy global access to shared resource
  • Simpler code without needing to keep track of object lifetime

Conclusion

javax.inject.Singleton is a powerful annotation that can be used to make a class a singleton. This can be useful in situations where you want a global access to a shared resource or performance benefits from avoiding costly object creation.
