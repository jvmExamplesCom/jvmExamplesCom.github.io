javax.annotation.PreDestroy

javax.annotation.PreDestroy is a Java annotation used for annotating methods that should be invoked when the component that's hosting them is about to be destroyed. PreDestroy is part of the javax.annotation package and it was introduced with the release of J2EE 5.0 in 2006. It provides an easy way to specify callback methods that are part of the lifecycle of a component. These methods may be used to clean up resources or perform any other tasks necessary before the component is destroyed.

Usage

The @PreDestroy annotation is typically used with components such as servlets, sessions beans, message-driven beans, and other components that have a well-defined lifecycle. The annotation can be used to define a method that will be invoked just before the component is destroyed. The method must not have any parameters and must not throw any exceptions.

Example

The following example demonstrates how to use the @PreDestroy annotation to create a callback method that will be invoked when the component is destroyed:


          
           
@PreDestroy
public void cleanUp() {
    // clean up resources here
}

Output

When the component is destroyed, the cleanUp() method will be invoked and the resources will be cleaned up.
