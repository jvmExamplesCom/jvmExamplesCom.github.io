Overview of org.aspectj.lang.annotation.Around

Within the java language, a special annotation exists, the 

@Around
annotation. This annotation is used to specify code that should be executed around a certain method call. This annotation is part of the AspectJ library, and thus should be imported as: 

          
           
import org.aspectj.lang.annotation.Around;
, before being used within your codebase.

Usage of @Around Annotation

The main usage of the @Around annotation is to perform various operations before or after a call to a specific method. Using this annotation, you can modify the input or output of a function, as well as any other side effects it might produce. The @Around annotation is often used to implement aspects or cross-cutting concerns, such as logging or security.

Structure of @Around Annotation

The structure of an @Around annotation must always adhere to the following:


          
           
@Around("join point expression") 
public Object advise(ProceedingJoinPoint pjp) throws Throwable {
  // do something  

  // call proceed() to continue with the joinpoint execution
  return pjp.proceed();
}

Wherein, the "join point expression" defined within the annotation is a pointcut - a definition of which methods should be intercepted. The 

advice
function is called with a joinpoint object which contains the parameters passed to the intercepted method, as well as a 
proceed()
function that triggers the execution of the joinpoint.

Examples of @Around Annotation

The following is an example of a simple use of the @Around annotation. In this case, the annotation is used to log the parameters passed to a method and the result generated by the method: 


          
           
@Around("execution(* mypackage.MyClass.*(..))")
public Object aroundLogging(ProceedingJoinPoint pjp) throws Throwable {
    LOGGER.info("Parameters: " + Arrays.toString(pjp.getArgs()));  
    Object result = pjp.proceed();
    LOGGER.info("Result: " + result);

    return result;
}

In this example, a log message is printed both before and after the execution of any method within the mypackage.MyClass class.

Conclusion

The @Around annotation is a powerful tool for applying cross-cutting logic around method calls. It helps us to cleanly separate our code into orthogonal concerns and makes our applications easier to maintain and extend.
