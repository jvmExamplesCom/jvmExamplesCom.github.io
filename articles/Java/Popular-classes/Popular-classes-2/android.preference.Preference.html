The android.preference.Preference Library

The android.preference.Preference library provides access to an application’s UI for setting user preferences. This library features a series of APIs and components which allow developers to customize appearance and behavior of the user interface, as well as allowing users to control certain aspects of the user experience. In this article we will explore how to use android.preference.Preference within your application.

Getting Started

The first step when using android.preference.Preference is to add the library to your project. This can be done in the following way: 

          
           
dependencies {
    implementation 'androidx.preference:preference:1.1.0'
}
Once you have added the library to your project, you then need to declare the PreferenceFragmentCompat within your XML layout file, so that the android.preference.Preference library can be used in your application. 

          
           
<fragment
    android:name="androidx.preference.PreferenceFragmentCompat"
    android:layout_width="match_parent"
    android:layout_height="match_parent"/>

Creating a PreferenceScreen

The next step when using android.preference.Preference is to create a PreferenceScreen, which will contain all of the preferences that should be displayed within the application. This is done using the PreferenceScreen class, and it should be added to the XML layout file as follows: 

          
           
<PreferenceScreen xmlns:android="http://schemas.android.com/apk/res/android">
Within this PreferenceScreen, you can then add different types of preferences, including CheckBoxPreference, ListPreference, and EditTextPreference. An example of this is shown below: 

          
           
<CheckBoxPreference
    android:key="checkbox_preference"
    android:title="Checkbox Preference" />

<ListPreference
    android:key="list_preference"
    android:title="List Preference"
    android:entries="@array/list_preference_entries"
    android:entryValues="@array/list_preference_values"
/>

<EditTextPreference
    android:key="edit_text_preference"
    android:title="Edit Text Preference" />

</PreferenceScreen>
The above code creates three separate preferences, which are a checkbox, a list, and an edit text. You must also provide entries and entry values for the list preference. This is done in the following way: 

          
           
<string-array name="list_preference_entries">
    <item>List Option 1</item>
    <item>List Option 2</item>
    <item>List Option 3</item>
</string-array>

<integer-array name="list_preference_values">
    <item>1</item>
    <item>2</item>
    <item>3</item>
</integer-array>

Retrieving User Preferences

Once the user has set their preferences, the developer can retrieve them using the PreferenceManager class. An example of how to retrieve the values of the preferences is shown below: 

          
           
SharedPreferences sharedPreferences = PreferenceManager.getDefaultSharedPreferences(context);

boolean checkBoxPreference = sharedPreferences.getBoolean("checkbox_preference", false);
String listPreference = sharedPreferences.getString("list_preference", "");
String editTextPreference = sharedPreferences.getString("edit_text_preference", "");

Conclusion

In this article we looked at how to use the android.preference.Preference library to add user preferences to your application. We saw how to add the library to our project, create a PreferenceScreen, and retrieve the user's preferences. We hope you found this article helpful!
