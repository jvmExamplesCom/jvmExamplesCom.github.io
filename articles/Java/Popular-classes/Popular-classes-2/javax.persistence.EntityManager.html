javax.persistence.EntityManager Overview

The javax.persistence.EntityManager is an interface that provides a set of methods that allow developers to query and manipulate entities in a database. It acts as an intermediary between the database and the Java application, allowing the application to make changes to entities stored in the database. EntityManager is part of the Java Persistence API (JPA), which is a specification for object-relational mapping (ORM). It allows developers to access and modify data in a database without having to write lengthy SQL statements.

Creating and Accessing EntityManager Instances

An instance of an EntityManager can be created by using the following code:


          
           
EntityManagerFactory emf =
    Persistence.createEntityManagerFactory("MyPU");
EntityManager em = emf.createEntityManager();

In the above example, "MyPU" is the name of the persistence unit. This is an XML configuration file which defines the database connection details, the name of the entities, and other related settings. For more information on configuring persistence units, please refer to the JPA documentation.

Once you have an instance of EntityManager, you can use it to create, read, update, and delete entities stored in the database. For example, the following code snippet shows how to create an entity and persist it into the database:


          
           
Employee employee = new Employee();
employee.setName("John Doe");
employee.setAge(20);

em.getTransaction().begin();
em.persist(employee);
em.getTransaction().commit();

Here, the entity is first created, and then persisted into the database using the EntityManager's persist() method. All changes made to entities are done within the context of transactions, so it is important to call the getTransaction() method before making any changes to the entity.

Querying Entities with JPQL

The EntityManager also provides methods that allow you to query entities, using the Java Persistence Query Language (JPQL). JPQL is a SQL-like language that allows developers to query entities instead of writing SQL statements. For example, the following JPQL statement returns all employees with the name "John Doe":


          
           
SELECT e FROM Employee e WHERE e.name = 'John Doe'

This statement can be used with the EntityManager's createQuery() method, as shown below:


          
           
TypedQuery<Employee> query = 
    em.createQuery("SELECT e FROM Employee e WHERE e.name = 'John Doe'", Employee.class);
List<Employee> employees = query.getResultList();

Here, the createQuery() method takes two parameters, the JPQL statement and the type of the entity that is being queried. The result of the query is returned in a list of entities.

Conclusion

The javax.persistence.EntityManager is an important part of the JPA specification. It allows developers to manipulate entities stored in a database by providing a set of methods for creating, reading, updating, and deleting entities. It also provides query methods, allowing developers to query entities using the Java Persistence Query Language (JPQL).
