java.nio.file.StandardOpenOption

Java.nio.file.StandardOpenOption is an enum type used for specifying various options for read and write operations on files. It helps in determining the behavior of a file system when it comes to opening a file for manipulation. StandardOpenOptions can be used to open a file for read/write/create/delete operations with various other properties. This allows developers to control the exact behavior of a file system when opening a file.

In this article, we will be looking at how the options in java.nio.file.StandardOpenOption work, how they are used and the output generated by them. We will also look at some simple code examples showing the usage of these options.

How does it work?

The StandardOpenOption enum type provides a set of constants that allow us to specify various properties for the file manipulation operations. The constants provided include READ, WRITE, CREATE, TRUNCATE_EXISTING, DELETE_ON_CLOSE, SPARSE and SYNC.

By default, the READ option is enabled. When a file is opened with the WRITE option, its existing content is cleared out and replaced with the new content. With the CREATE option, the specified file is created if it does not already exist. The TRUNCATE_EXISTING option will truncate an existing file. The DELETE_ON_CLOSE flag ensures that the file is deleted when the stream is closed by the application. The SPARSE option requests that any newly created files should be marked as sparse files. Finally, the SYNC option forces all changes made to the file to be written to persistent storage immediately.

Output

The output generated by these various options is as follows:

  • StandardOpenOption.READ will open the file for reading.
  • StandardOpenOption.WRITE will open the file for writing, clearing out any existing content.
  • StandardOpenOption.CREATE will create the file if it does not already exist.
  • StandardOpenOption.TRUNCATE_EXISTING will truncate the file if it already exists.
  • StandardOpenOption.DELETE_ON_CLOSE will delete the file when the stream is closed.
  • StandardOpenOption.SPARSE will mark any newly created files as sparse files.
  • StandardOpenOption.SYNC will force all changes made to the file to be written to persistent storage immediately.
