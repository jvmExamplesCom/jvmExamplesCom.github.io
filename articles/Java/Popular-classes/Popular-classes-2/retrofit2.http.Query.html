Retrofit2 HTTP Query

Retrofit2 provides an easy way to retrieve data from a server using HTTP requests. It also provides various annotations to enable an HTTP request to be constructed correctly. The Query annotation is one such annotation that can be used for GET and POST requests to specify the query parameters.

GET Request Examples

For example, if we have a request endpoint with the following formatting: ?name= &address=

, then we can use Query annotation to specify the query parameters as shown below: 

           
            
@GET("/get user") 
Call<User> getUser(@Query("name") String name, @Query("address") String address);

The complete URL would be formatted as: /get user?name= &address=

.

POST Request Examples

For POST requests, it is possible to specify the query parameters using QueryMap annotation as shown below:


            
             
@POST("/save user")
Call<Void> saveUser(@QueryMap Map<String, Object> params);

The map should include the query parameters with keys denoting the parameter name and value being the respective parameter value. The complete URL would be formatted as: /save user?name= &address=

.

Example Output

When the query parameters are specified correctly, the output of the request will be the expected result from the server. For example, if the server returns the JSON object { "name": "John Smith", "address": "123 Main Street" } then this will be the result of the request:


             
              
{
  "name": "John Smith",
  "address": "123 Main Street"
}
