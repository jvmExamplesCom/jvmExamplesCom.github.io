javax.ws.rs.QueryParam

The javax.ws.rs.QueryParam annotation provides a way to associate the query parameters of a request with the Java method parameters. This annotation is used on the method parameters and it is considered part of the JAX-RS API. It allows the JAX-RS runtime to inject the query parameter values into the method parameters of the annotated methods.

Overview of javax.ws.rs.QueryParam

JAX-RS uses annotations to mark Java method parameters as query parameters. The annotation is called QueryParam and it can be used to specify the name of the query parameter in the request. The values of the query parameters are then injected into the method parameters corresponding to the names specified in the QueryParam annotation. This annotation can also be used to specify if the query parameter is optional or required.

Example Using javax.ws.rs.QueryParam

Let's consider an example to understand how to use the QueryParam annotation. In this example, we will define two query parameters using the QueryParam annotation - 'name' and 'age'. 

          
           
@GET
@Path("/query")
public String getQueryParam(@QueryParam("name") String name, 
                           @QueryParam("age") Integer age){
   return "Name = "+name+" , Age = "+age;
}
In the above example, we used the QueryParam annotation to mark the 'name' and 'age' parameters of the getQueryParam() method as query parameters. The values of these query parameters will be injected into the corresponding method parameters when the request is received.

Using Default Values for Query Parameters

The QueryParam annotation also supports the use of default values for query parameters. If the query parameter is not found in the request, then its default value is used instead. This can be useful in situations where the query parameter is not required or when the value of the query parameter is not known. 

          
           
@GET
@Path("/query")
public String getQueryParam(@QueryParam("name") String name, 
                           @QueryParam("age") Integer age
                           @DefaultValue("10") @QueryParam("pageSize") Integer pageSize){
   return "Name = "+name+" , Age = "+age+" , PageSize = "+pageSize;
}
In the above example, we have used the DefaultValue annotation to set the default value of the pageSize query parameter to 10. If the pageSize query parameter is not specified in the request, then its value will be set to 10 by default.

Handling Required Query Parameters

If the value of a query parameter is required, then we can use the @Required annotation to specify that it is required. This will ensure that an error is thrown if the query parameter is not specified in the request. 

          
           
@GET
@Path("/query")
public String getQueryParam(@QueryParam("name") String name, 
                           @Required @QueryParam("age") Integer age){
   return "Name = "+name+" , Age = "+age;
}
In the above example, we have used the required annotation to mark the 'age' query parameter as required. If the age query parameter is not specified in the request, then an error will be thrown.

Conclusion

In this article, we discussed the use of the javax.ws.rs.QueryParam annotation and saw how it can be used to bind the query parameters of a request to the method parameters of a Java method. We also looked at how to use the DefaultValue and Required annotations to set default values andvalidate required query parameters.
