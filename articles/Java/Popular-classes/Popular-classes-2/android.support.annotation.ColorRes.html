android.support.annotation.ColorRes

Android development often involves the usage of colors in various ways. Android provides the android.support.annotation.ColorRes annotation to help developers better handle colors in their apps. ColorRes is an annotation that provides compile-time type safety when working with colors inside of your application. It is used to ensure that colors are given as resource identifiers rather than as constant values.

Java Usage

When using this annotation, you must declare a color resource for each color that is used in the application. The Color Resource is declared inside of an XML file (usually colors.xml). This xml file should be placed inside of the res/values/ folder. For example, if you want to declare a color named “primary”, you could do the following: 

          
           
<?xml version="1.0" encoding="utf-8"?>
<resources>
    <color name="primary">#009900</color>
</resources>
The ColorRes annotation should then be used to signify that this color has been declared as a resource: 

          
           
@ColorRes
int primaryColor = R.color.primary;

Usage in Layout Files

This annotation can also be used in layout files. To use it, you need to provide the Resource ID for the color using the @ symbol. For example, if you wanted to create a Text View with the color “primary”, you could do the following: 

          
           
<TextView
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Hello World!"
    android:textColor="@color/primary" />

Usage in Styles

You can also use the ColorRes annotation when defining styles. This allows you to easily reference colors in your styles. For example, if you wanted to create a style called “primaryTheme” with the color “primary”, you could do the following: 

          
           
<style name="primaryTheme">
    <item name="android:textColor">@color/primary</item>
</style>
