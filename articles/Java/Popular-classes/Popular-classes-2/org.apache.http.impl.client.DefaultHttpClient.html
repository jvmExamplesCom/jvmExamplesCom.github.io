Overview of org.apache.http.impl.client.DefaultHttpClient

Connection Pool Management Strategy

The DefaultHttpClient also supports multiple connection pool management strategies, allowing developers to tailor their application's connections to the specific environment. The default connection pool management strategy is based on the concept of a “per-route” pool, which maintains separate pools for each host and port combination that it makes requests to. This allows for a more efficient use of resources as all requests from that host and port can be reused. Other strategies are also supported, such as a global pool which maintains a single connection pool for all requests and an isolated pool, where a separate pool is maintained for each host.
