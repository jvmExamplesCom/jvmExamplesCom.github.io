java.util.concurrent.SynchronousQueue

The java.util.concurrent.SynchronousQueue class is a queue that can store a single element at a time. It is an implementation of the BlockingQueue interface and allows one thread to wait for another thread to put an element in the queue. It is a bounded queue, but with its capacity fixed at zero. The java.util.concurrent.SynchronousQueue class does not store elements. Each insert operation must wait for a corresponding remove operation by another thread, which makes the SynchronousQueue class different from other BlockingQueue implementations.

Example

Let's look at a simple SynchronousQueue example to understand it better. The following is a simple Java program that uses a SynchronousQueue to print a message.


          
           
import java.util.concurrent.SynchronousQueue;

public class SynchronousQueueExample {

  public static void main(String[] args) {
    SynchronousQueue<String> queue = new SynchronousQueue<>();

    Thread t1 = new Thread(() -> {
      try {
        System.out.println("Putting a message in the queue...");
        queue.put("Hello World!");
      } catch (InterruptedException e) {
        e.printStackTrace();
      }
    });

    Thread t2 = new Thread(() -> {
      try {
        String msg = queue.take();
        System.out.println("Message received: " + msg);
      } catch (InterruptedException e) {
        e.printStackTrace();
      }
    });

    t1.start();
    t2.start();
  }

}

When we run the above program, we get the following output:


          
           
Putting a message in the queue...
Message received: Hello World!
