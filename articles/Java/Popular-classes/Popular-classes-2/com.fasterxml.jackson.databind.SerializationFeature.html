SerializationFeature in Java

The 

          
           
com.fasterxml.jackson.databind.SerializationFeature
class is a Jackson Databind component used to help control low-level serialization features: how output is written, whether type information (for possible polymorphic handling) is included and if so, in what form etc. The class defines all available standard SerializationFeatures.

Example 1: Using SerializationFeature


          
           
// create an ObjectMapper instance
ObjectMapper mapper = new ObjectMapper();

// enable/disable SerializationFeature
mapper.configure(SerializationFeature.WRITE_NULL_MAP_VALUES, false);

Example 2: Configuring SerializationFeature

We can also configure different aspects of the serialization feature programmatically. For example:


          
           
mapper.configure(SerializationFeature.WRITE_ENUMS_USING_TO_STRING, true);

This code snippet configures the ObjectMapper to serialize Enums using the String representation of their values rather than the default numerical representation.

Example 3: Serializing Collections with SerializationFeature

The SerializationFeature class can also be used to control how collections are serialized. For example:


          
           
mapper.configure(SerializationFeature.WRITE_EMPTY_JSON_ARRAYS, false);

This code snippet configures the ObjectMapper to omit empty JSON arrays when it encounters them while serializing a Java collection.

Example 4: Serializing Dates with SerializationFeature

The SerializationFeature class can also be used to control how dates are serialized. For example:


          
           
mapper.configure(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS, false);

This code snippet configures the ObjectMapper to serialize dates as ISO8601 strings rather than as timestamps.
