org.springframework.boot.autoconfigure.condition.ConditionalOnProperty

The org.springframework.boot.autoconfigure.condition.ConditionalOnProperty annotation is used to conditionally enable or disable a bean depending upon the value of a property defined in the application configuration. It provides the flexibility to easily control the loading of beans based on properties that can be set via application configuration files or environment variables.

It is very useful in situations where you need to configure the behavior of your application at startup. For example, if you have different databases that need to be used depending on the environment that the application is running in, you can use this annotation to enable or disable specific beans or configurations.

Let's look at an example of using this annotation.


          
           
@Configuration
public class MyConfiguration {
  @Bean  
  @ConditionalOnProperty(name = "environment", havingValue = "dev")
  public DataSource devDataSource() {
    System.out.println("Creating Dev Data Source");
    return new DataSource();
  }
}

In this example, the devDataSource bean will be enabled only if the environment property has the value of "dev". If the environment property is set to something other than "dev" then the bean will not be enabled.

This annotation can also be used to conditionally choose between two different beans. Let's look at an example of this.


          
           
@Configuration
public class MyConfiguration {
  @Bean  
  @ConditionalOnProperty(name = "database", havingValue = "mysql")
  public DataSource mysqlDataSource() {
    System.out.println("Creating Mysql Data Source");
    return new DataSource();
  }

  @Bean
  @ConditionalOnProperty(name = "database", havingValue = "postgresql")
  public DataSource postgresqlDataSource() {
    System.out.println("Creating Postgresql Data Source");
    return new DataSource();
  }
}

In this example, the mysqlDataSource bean will be created if the database property is set to "mysql", and the postgresqlDataSource bean will be created if the database property is set to "postgresql". If the database property is set to something else then neither bean will be created.

Other Uses

The annotation can also be used with arguments that accept a list of values. If a property is set to one of the specified values then the bean will be enabled. For example:


          
           
@Configuration
public class MyConfiguration {
  @Bean  
  @ConditionalOnProperty(name = "database", havingValue = {"mysql", "postgresql"})
  public DataSource relationalDataSource() {
    System.out.println("Creating Relational Data Source");
    return new DataSource();
  }
}

In this example, the relationalDataSource bean will be enabled if the database property is either set to "mysql" or "postgresql". It will be disabled if the database property is set to something else.

Conclusion

The org.springframework.boot.autoconfigure.condition.ConditionalOnProperty annotation is a powerful tool that allows you to easily control the loading of beans based on properties that can be set via application configuration files or environment variables. It can be used to conditionally enable or disable beans as well as to choose between multiple beans.
