android.widget.AdapterView.OnItemClickListener

Android's 

          
           
AdapterView.OnItemClickListener
is an interface used to define the behavior of a list item or grid item when it is clicked. It can be implemented in any Activity, Fragment, or View. When the user clicks on a particular item, the onItemClick method is invoked and passes the view, position, and id of the item that was clicked.

Implementation Example

There are two main parts to implementing this interface. First, you need to create an instance of OnItemClickListener and register it with the AdapterView. For example, if you have a ListView named listView in your layout, you could register the OnItemClickListener like this:


          
           
listView.setOnItemClickListener(new AdapterView.OnItemClickListener() {
    @Override
    public void onItemClick(AdapterView<!--?--> parent, View view, int position, long id) {
        // Handle item click here
    }
});
The second part is to implement the 
onItemClick
method inside the listener. Here is an example of how to handle the click based on the position of the item that was clicked: 

          
           
public void onItemClick(AdapterView<!--?--> parent, View view, int position, long id) {
    switch (position) {
        case 0:        
            // Handle click on first item
            break;
        case 1:        
            // Handle click on second item
            break;
        // etc
    }
}

Putting It All Together

Now that we have our OnItemClickListener set up, let's put it to use by adding some code inside the 

onItemClick
method. In this example, we will start an Activity based on the item that was clicked: 

          
           
public void onItemClick(AdapterView<!--?--> parent, View view, int position, long id) {
    switch (position) {
        case 0:        
            Intent intent = new Intent(this, FirstActivity.class);

    }
}
