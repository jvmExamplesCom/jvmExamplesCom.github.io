javax.crypto.BadPaddingException

The javax.crypto.BadPaddingException is a subclass of the Java Cryptography API's IllegalBlockSizeException, which indicates that a block-based cipher's plaintext input is not properly padded. This exception is thrown in cases where the plaintext input is expected to have a certain length, and either not enough padding or too much padding has been provided.

Code Examples

Below are some code examples that demonstrate how a BadPaddingException can be thrown.


          
           
Cipher cipher = Cipher.getInstance("AES/ECB/PKCS5Padding");
byte[] plainText= "Hello World!".getBytes();

//not enough padding
cipher.init(Cipher.ENCRYPT_MODE, key);
byte[] cipherText = cipher.doFinal(plainText);

//throws BadPaddingException

In this example, the input plaintext is not padded to the expected length. This causes the encryption process to throw a BadPaddingException.


          
           
Cipher cipher = Cipher.getInstance("AES/ECB/PKCS5Padding");
byte[] plainText= "Hello World!!!!".getBytes();

//too much padding
cipher.init(Cipher.ENCRYPT_MODE, key);
byte[] cipherText = cipher.doFinal(plainText);

//throws BadPaddingException

Similarly, in this example, the input plaintext contains too much padding and the encryption process will throw a BadPaddingException.


          
           
Cipher cipher = Cipher.getInstance("AES/ECB/NoPadding");
byte[] plainText= "Hello World!".getBytes();

//no padding
cipher.init(Cipher.ENCRYPT_MODE, key);
byte[] cipherText = cipher.doFinal(plainText);

//throws BadPaddingException

Finally, in this example, the plaintext is not padded at all, since the NoPadding cipher mode is used. This will again cause the encryption process to throw a BadPaddingException.
