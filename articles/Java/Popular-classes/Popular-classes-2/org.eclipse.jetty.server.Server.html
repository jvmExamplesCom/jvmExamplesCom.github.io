The 

          
           
org.eclipse.jetty.server.Server
is a Java HTTP server and servlet container that provides an environment for web applications to run. It allows users to deploy and manage web applications from within the server. Jetty is used in a variety of applications, such as web applications, embedded servlet containers, and enterprise applications.

Creating a Server Instance

To create a new instance of the Jetty server, you must call the Server constructor:


          
           
Server server = new Server();

Configuring the Server

You can configure the server using setter methods on the Server instance. For example, to set the port the server should listen on, use the 

setPort()
method: 

          
           
server.setPort(8080);

Adding Handlers

In order for the server to handle requests, it must have one or more handlers configured. A handler is an object that handles HTTP requests and produces responses. The Jetty server provides several built-in handlers, including:

  • ContextHandler
    – handles requests for web applications
  • ResourceHandler
    – handles requests for static resources
  • ServletHandler
    – handles requests for servlets
To add a handler to the server, use the 
addHandler()
method: 

          
           
server.addHandler(handler);

Starting and Stopping the Server

Once the server is configured and the necessary handlers are added, it is ready to be started. To start the server, simply call the 

start()
method on the Server instance: 

          
           
server.start();

When the server is no longer needed, it can be stopped by calling the 

stop()
method: 

          
           
server.stop();
