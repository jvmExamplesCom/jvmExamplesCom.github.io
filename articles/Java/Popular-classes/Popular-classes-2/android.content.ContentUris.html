ContentUris in Java and Android

ContentUris is an important part of the Java platform, used in multiple Android applications. This article will cover the basics of ContentUris and provide examples of how they are implemented in Android. The 

          
           
android.content.ContentUris
class provides access to a content provider's data set as a set of URI references, or "URIs", which are URLs with a specific form. A content URI is a URI that identifies data in a specific way. It can be used to query for and update data in a content provider.

Content URIs Basics

Content URIs have a specific structure. They typically have the scheme 

          
           
content://
followed by the authority of the content provider, and then the type of data followed by the ID of the data item. For example: 

          
           
content://com.example.myprovider/mydata/42
The authority, which begins after the 

          
           
content://
portion of the URI, is the qualified name of the content provider, such as 

          
           
com.example.myprovider
. The third part of the URI, the type of data following the authority, is the path of the data in the content provider. Lastly, the ID of the data item is the value that follows the data path.

Content Uris Methods

The 

          
           
android.content.ContentUris
class provides some basic methods for working with content URIs. The 

          
           
withAppendedId()
method can be used to append an ID to a content URI. For example, this code snippet appends the ID 5 to the content://myprovider/ path: 

          
           
ContentUris.withAppendedId(Uri.parse("content://myprovider/"), 5);
// Returns content://myprovider/5
The 

          
           
parseId()
method can be used to parse the ID out of a content URI. For example, this code snippet parses out the ID from the content://myprovider/5 URI: 

          
           
ContentUris.parseId(Uri.parse("content://myprovider/5"));
// Returns 5

ContentResolver resolver = getContentResolver();
Uri myUri = ContentUris.withAppendedId("content://myprovider/", 5);
Cursor cursor = resolver.query(myUri, null, null, null, null);

android.content.ContentResolver

ContentValues

ContentResolver resolver = getContentResolver();
Uri myUri = Uri.parse("content://myprovider/");
ContentValues myValues = new ContentValues();

update()

android.content.ContentResolver

ContentValues

delete()

android.content.ContentResolver
class accepts a content URI and an optional selection string. The selection string is used to specify which record should be deleted. If no selection string is provided, then all records matching the URI will be deleted.
