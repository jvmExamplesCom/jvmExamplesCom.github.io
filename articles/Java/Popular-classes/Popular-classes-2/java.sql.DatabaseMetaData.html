Java.sql.DatabaseMetaData

The Java Database Connectivity (JDBC) API uses the 


          
           
java.sql.DatabaseMetaData
to provide information about the database, such as its data types, stored functions, and existing tables. It also allows developers to use SQL statements to access and manipulate data in the database.

Overview of DatabaseMetaData Interface

The DatabaseMetaData interface provides a number of methods for retrieving information about the database. The most commonly used methods are listed below:

  • getTables()
    - returns the names of all the tables in the database.
  • getColumns()
    - returns the columns of all the tables in the database.
  • getPrimaryKeys()
    - returns the primary keys of the tables in the database.
  • getProcedures()
    - returns the stored procedures of the database.
  • getIndexInfo()
    - returns the indexes of the tables in the database.

Example Usage

In this section, we will look at examples of how to use the DatabaseMetaData interface to retrieve information about the database.

Retrieving Table Names


          
           
try {
    ArrayList<String> tables = new ArrayList<String>();
    DatabaseMetaData metaData = connection.getMetaData();
    ResultSet resultSet = metaData.getTables(null, null, null, new String[] { "TABLE" });
    while (resultSet.next()) {
        String tableName = resultSet.getString("TABLE_NAME");
        tables.add(tableName);
    }
    resultSet.close();
    System.out.println(tables);
}

Retrieving Column Information

    DatabaseMetaData metaData = connection.getMetaData();
    ResultSet resultSet = metaData.getColumns(null, null, "STUDENTS", null);
