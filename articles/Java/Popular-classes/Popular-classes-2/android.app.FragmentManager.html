Android.App.FragmentManager

FragmentManager is a powerful class used in Android development to manager Fragments and add, replace, or remove fragments in an activity. It also keeps track of all active fragments, provides transaction management, and allows fragments to communicate with one another. In this article, we’ll discuss the FragmentManager and its various features, as well as how to use it in your applications.

Creating an FragmentManager Instance

The first step to using the FragmentManager is creating an instance of it. The simplest way of doing this is by calling the getSupportFragmentManager() method from an Activity class, like this:


          
           
FragmentManager fragmentManager = getSupportFragmentManager();

Adding and Removing Fragments

Once you have an instance of the FragmentManager, you can use it to add, replace, and remove Fragments from the activity. The primary methods used to do this are add(), replace(), and remove(). Let’s look at each of these methods in more detail.

Add Method

The add() method adds a Fragment to the activity. It takes two parameters: a ViewGroup where the Fragment will be added, and the Fragment to add. The code looks like this:


          
           
fragmentManager.add(viewGroup, fragment);

Replace Method

The replace() method replaces an existing Fragment with a new one. It takes three parameters: the ViewGroup where the Fragment will be added, an id for the ViewGroup where the Fragment will be placed, and the Fragment to add. The code looks like this:


          
           
fragmentManager.replace(viewGroup, id, fragment);

Remove Method

The remove() method removes an existing Fragment from the activity. It takes the Fragment to remove as a parameter. The code looks like this:


          
           
fragmentManager.remove(fragment);

Transaction Management

FragmentManager also offers transaction management. Transactions are used to wrap a set of related changes to the activity. This makes it easier to manage and roll back the changes if something goes wrong. To start a transaction, you use the beginTransaction() method, like this:


          
           
fragmentManager.beginTransaction();

You can then add, remove, and replace Fragments as part of the transaction. Once you’ve made all the changes you want, you can commit the transaction with the commit() method, like this:


          
           
fragmentManager.commit();

Fragment Communication

FragmentManager can also be used to allow Fragments to communicate with each other. This is done using the findFragmentByTag() method, which takes a String tag as a parameter. The method returns the Fragment with the matching tag, or null if no such Fragment exists. The code looks like this:


          
           
Fragment fragment = fragmentManager.findFragmentByTag(tag);

Once you have a reference to the Fragment, you can call its public methods to communicate with it. This is a useful way to pass information between Fragments, as well as trigger events in other Fragments.

Conclusion

In this article, we discussed Android’s FragmentManager and how to use it in your applications. We looked at its various features, such as adding, removing, and replacing Fragments, transaction management, and Fragment communication. With this knowledge, you should now be able to use the FragmentManager to create powerful and dynamic user interfaces.
