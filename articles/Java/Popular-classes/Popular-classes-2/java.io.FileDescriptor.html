Introduction to Java.io.FileDescriptor

Java.io.FileDescriptor is an abstract class used to represent an operating system dependent device handle. It can be used to identify open files and network connections.

Creating a FileDescriptor Instance

FileDescriptor instances cannot be created directly. They must be obtained from another source, such as a FileInputStream or FileOutputStream.


          
           
FileInputStream fis = new FileInputStream("foo.txt");
FileDescriptor fd = fis.getFD();

Methods

The FileDescriptor class has several methods for managing file descriptors. Some of the more commonly used ones are:


          
           
sync() - Forces any data buffered in the descriptor to be written out to the file system.
valid() - Checks whether the descriptor is valid (i.e. it points to an open file).

Example: Syncing Data To The File System

The following code example shows how to use FileDescriptor.sync() to write data buffered in a file descriptor to the file system.


          
           
FileInputStream fis = new FileInputStream("foo.txt");
FileDescriptor fd = fis.getFD();
fd.sync();
