Android Hardware SensorManager

Android's SensorManager is a package that provides access to device's sensors. It is part of the Android core library and is used to detect various physical properties of the device such as its orientation, acceleration, etc. The SensorManager allows apps to access information from different types of sensors in the device such as accelerometers, gyroscopes, light sensors and other types of sensors. It also allows applications to register for notification of sensor changes.

SensorManager Basics

In order to use the SensorManager API, an application needs to obtain a reference to the SensorManager instance. This instance can be obtained by calling the getSystemService() method from an Activity, Service or Application class. The following code snippet shows how to obtain a reference to the SensorManager from an Activity class: 

          
           
SensorManager sensorManager = (SensorManager)getSystemService(Context.SENSOR_SERVICE);
This SensorManager instance can then be used to access available sensors on the device. Each type of sensor can be accessed by calling the getDefaultSensor() method and passing in the type of sensor to access. For example, to access the accelerometer sensor, call the getDefaultSensor() method with the Sensor.TYPE_ACCELEROMETER constant as the parameter: 

          
           
Sensor accelerometer = sensorManager.getDefaultSensor(Sensor.TYPE_ACCELEROMETER);
The Sensor object returned can then be used to register and start monitoring a particular sensor.

Registering Listeners

The SensorManager also provides methods for registering and unregistering listeners to monitor sensor events. When registering a listener, you must provide an object that implements the SensorEventListener interface. This listener will be called when a new sensor event is detected. To register a SensorEventListener, call either the registerListener() or registerListener() method on the SensorManager instance. Both methods take in a SensorEventListener object and the Sensor object representing the sensor that is to be monitored. The following code snippet shows how to register a SensorEventListener for the accelerometer sensor: 

          
           
sensorManager.registerListener(sensorEventListener, accelerometer, SensorManager.SENSOR_DELAY_NORMAL);
The last parameter passed to the registerListener() method is the rate at which the listener should be called. This parameter must be one of the constants defined on the SensorManager class.

Handling Events

When a new sensor event is detected, the registered SensorEventListener’s onSensorChanged() method is called. This method takes in a SensorEvent object containing data about the sensor event. The following code snippet shows how to handle a sensor change event:

   float x = event.values[0];

   float z = event.values[2];

}
The SensorEvent object contains an array of floats representing the sensor values. The number of values in the array and what they represent depends on the type of the sensor.

Unregistering Listeners

When a SensorEventListener is no longer needed, it is important to unregister it in order to stop the listener from receiving any more sensor events. This can be done by calling the unregisterListener() method on the SensorManager instance. The following code snippet shows how to unregister a SensorEventListener: 

          
           
sensorManager.unregisterListener(sensorEventListener);
It is important to note that all listeners must be unregistered when the application is paused or stopped in order to free up resources.

Conclusion

The Android SensorManager API provides developers with a powerful set of tools to access and monitor device sensors. With this API, developers can create apps that can detect various physical properties of the device such as its orientation, acceleration and other sensor readings.
