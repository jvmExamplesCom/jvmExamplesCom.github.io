android.os.Binder

The android.os.Binder class of the Android operating system is an abstract class that acts as a bridge between two applications and can be used to allow remote communication. It provides a way for one application to call methods on another application and send data across process boundaries. Binder is a powerful tool for connecting applications, services and processes in Android.

Basics of Binder

Binder is a general-purpose inter-process communication (IPC) mechanism that allows code running in one process to access code and data in other processes.

It provides a generic interface for transferring data between two applications. It works by marshalling the data from one application (the caller) into a binary format that can be read by the other application (the callee). This marshalling process happens via a Binder object, which is responsible for taking the data from the caller, encoding it in a defined format and sending it over to the callee.

When the callee receives the data, it then uses the reverse process (known as unmarshalling) to read the data and understand what it is. Once this is done, the callee can then perform the appropriate action.
