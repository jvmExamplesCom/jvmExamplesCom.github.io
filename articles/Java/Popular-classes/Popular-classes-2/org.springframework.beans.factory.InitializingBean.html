org.springframework.beans.factory.InitializingBean

The 

          
           
org.springframework.beans.factory.InitializingBean
interface is a Spring-specific interface which provides an initializing hook for beans during the post-registration phase of their lifecycle. Implementing this interface allows beans to have their properties populated directly prior to being used, thus avoiding the typical issue of leaking components and resources from the container.

Usage

To use the 
InitializingBean
interface, simply implement the 

          
           
afterPropertiesSet()
method. This method will be called by the container once all properties have been set and is a good place to perform any initialization logic on the bean, such as validating the required properties or registering resources with the application context. 

          
           
public class MyBean implements InitializingBean {

  private String property;

  public void setProperty(String property) {
    this.property = property;
  }

  @Override
  public void afterPropertiesSet() throws Exception {
    if (property == null) {
      throw new IllegalArgumentException("property must not be null");
    }
  }
}
In addition to the 

          
           
afterPropertiesSet()
method, it is also possible to define init-methods in the bean definition. These methods will be called by the container, following the same pattern as the traditional JavaBeans spec. This can be useful if we want to perform initialization logic on the bean, but don't want to couple the bean to the Spring-specific 
InitializingBean
interface. 

          
           
<bean id="myBean" class="com.example.MyBean" init-method="init"/>

Summary

The 

          
           
org.springframework.beans.factory.InitializingBean
interface provides a hook for beans during the post-registration phase of their lifecycle, allowing them to have their properties populated prior to being used. It is recommended to use this interface whenever possible if we need to perform initialization logic on a bean. In addition, it is also possible to define traditional JavaBean-style init-methods in the bean definition.
