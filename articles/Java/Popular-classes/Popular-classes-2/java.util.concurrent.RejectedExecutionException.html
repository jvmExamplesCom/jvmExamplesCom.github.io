java.util.concurrent.RejectedExecutionException

The 

          
           
java.util.concurrent.RejectedExecutionException
is an unchecked exception that can be thrown by the application, or by any of the Executors in the java.util.concurrent package. It indicates that the particular task cannot be accepted for execution by the ExecutorService.

Example 1: Using a RejectedExecutionException

This example uses a 

          
           
RejectedExecutionHandler
to handle a thrown 

          
           
RejectedExecutionException
. The 

          
           
RejectedExecutionHandler
is registered with an 
ExecutorService
and is used to handle any exceptions thrown when tasks are submitted to the executor. 

          
           
import java.util.concurrent.*;

public class RejectedExecutionExample {

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(5);

        //register the RejectedExecutionHandler
        executor.setRejectedExecutionHandler(new RejectedExecutionHandler() {
            @Override
            public void rejectedExecution(Runnable r, ThreadPoolExecutor executor) {
                System.out.println("Task " + r.toString() + 
                        " rejected from " + executor.toString());
            }
        });

        //submit some tasks which exceeds pool size
        for(int i=0;i<10;i++) {
            Runnable runnableTask = new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    System.out.println("Executing Task");
                }
            };

            executor.submit(runnableTask);
        }
    }
}

Example 2: Handling RejectedExecutionException

In this example, we will handle the 

          
           
RejectedExecutionException
using the 
try-catch
block. We will submit a task to an 
ExecutorService
and handle any exception that might be thrown while submitting the task. 

          
           
import java.util.concurrent.*;

public class RejectedExecutionExample2 {

    public static void main(String[] args) {
        ExecutorService executor = Executors.newFixedThreadPool(5);

        Runnable runnableTask = new Runnable() {
            @Override
            public void run() {
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("Executing Task");
            }
        };

        try {
            for(int i=0;i<10;i++) {
                executor.submit(runnableTask);
            }
        } catch (RejectedExecutionException e) {
            System.err.println("Task rejected from "+executor.toString());
        }
    }
}

When the above program is executed, it produces the following output:


          
           
Task rejected from [Running, pool size = 5, active threads = 5, queued tasks = 0, completed tasks = 0]

Conclusion

The 

          
           
RejectedExecutionException
is an unchecked exception that is thrown when a task cannot be accepted for execution by the ExecutorService. It can be handled using a 

          
           
RejectedExecutionHandler
or using a 
try-catch
block. By handling this exception, the application is able to gracefully handle the situation when tasks exceed the capacity of the executor.
