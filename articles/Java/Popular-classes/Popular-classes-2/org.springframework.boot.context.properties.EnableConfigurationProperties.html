Configure EnableConfigurationProperties

The org.springframework.boot.context.properties.EnableConfigurationProperties annotation is used to enable configuration of properties in the application context. This annotation can be specified on any @Configuration class, and it will enable configuration of externalized application configuration in a specific namespace, through the @ConfigurationProperties annotation.

Adding EnableConfigurationProperties in your Application

To use the EnableConfigurationProperties annotation, you must add it to your application's main class. For example:


          
           
@SpringBootApplication
@EnableConfigurationProperties
public class MyApplication {
  public static void main(String[] args) {
    SpringApplication.run(MyApplication.class, args);
  }
}

Configuration Using @ConfigurationProperties

Now that you have enabled configuration with EnableConfigurationProperties, you can use @ConfigurationProperties to easily configure your application using external properties.

For example, if you have a properties file in the same package as your application (or in your classpath), you can use it to configure the application:


          
           
@ConfigurationProperties("myapp")
public class MyAppConfiguration {
  private String property;

  public String getProperty() {
    return property;
  }

  public void setProperty(String property) {
    this.property = property;
  }
}

And then you can access the value in your application:


          
           
System.out.println(myAppConfiguration.getProperty());

Conclusion

The @EnableConfigurationProperties annotation enables easy configuration of an application using externalized properties. You can use this annotation to specify a namespace for configuration, and then configure your application using the @ConfigurationProperties annotation.
