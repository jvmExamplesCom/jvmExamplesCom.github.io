java.util.concurrent.ScheduledThreadPoolExecutor

The ScheduledThreadPoolExecutor class is an implementation of the Java ExecutorService that supports scheduled tasks for both single-shot and repeated execution. This is useful for creating a task that runs at a specific time, or at intervals. It is part of the java.util.concurrent package.

Example

The following example shows how to use a ScheduledThreadPoolExecutor to schedule a task to be executed every 10 seconds.


          
           
ScheduledExecutorService executor = new ScheduledThreadPoolExecutor(1);
Runnable task = () -> System.out.println("Task executed.");
executor.scheduleWithFixedDelay(task, 0, 10, TimeUnit.SECONDS);

Using the Attribute Tasks

The ScheduledThreadPoolExecutor also supports scheduled tasks with attributes such as delay, period and fixed-delay. The following example shows how to use the attribute tasks to schedule a task with a delay of 10 seconds and a period of 20 seconds.


          
           
ScheduledExecutorService executor = new ScheduledThreadPoolExecutor(1);
Runnable task = () -> System.out.println("Task executed.");
executor.scheduleAtFixedRate(task, 10, 20, TimeUnit.SECONDS);

Using the execute() Method

The ScheduledThreadPoolExecutor also provides an execute() method which executes a Runnable task immediately. The following example shows how to use the execute() method to execute a Runnable task.


          
           
ScheduledExecutorService executor = new ScheduledThreadPoolExecutor(1);
Runnable task = () -> System.out.println("Task executed.");
executor.execute(task);

Using the submit() Method

The ScheduledThreadPoolExecutor also provides a submit() method which submits a Runnable task for asynchronous execution. The following example shows how to use the submit() method to submit a Runnable task.

executor.submit(task);
