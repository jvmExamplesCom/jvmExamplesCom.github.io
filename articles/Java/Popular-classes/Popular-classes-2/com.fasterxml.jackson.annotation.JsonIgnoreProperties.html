JsonIgnoreProperties

The Jackson annotation 

          
           
com.fasterxml.jackson.annotation.JsonIgnoreProperties
provides a convenient way to ignore incoming JSON properties. This annotation is used as part of the Jackson Data Processor to help process data received from external sources.

This annotation is useful when you want to specify properties that should not be bound to your application’s domain objects. It allows you to ignore any number of properties in the JSON payload and only map the fields you wish to consume. Additionally, this annotation can help keep your code clean and maintainable by preventing unwanted properties from polluting your program's Domain Objects.

How to Use JsonIgnoreProperties

JsonIgnoreProperties can be applied either to entire classes or individual fields. To ignore properties for binding on a class-level, simply annotate the class with the 


          
           
@JsonIgnoreProperties
annotation and specify the properties you would like to ignore. 

          
           
// Ignore 'description' field from JSON payload 
@JsonIgnoreProperties({"description"})
public class Car { ... }

To ignore properties on an individual field level, use the 


          
           
@JsonIgnoreProperties
annotation and specify the field name in the annotation parameter. 

          
           
// Ignore 'speed' field from JSON payload 
    @JsonIgnoreProperties("speed") 
    public int gear;

It's important to note that both class-level and field-level annotations will override one another. In other words, if the same field is specified in both a class- and field-level annotation, the field-level annotation will take precedence and the property will be ignored.

Using Wildcards with JsonIgnoreProperties

In addition to explicitly specifying properties to be ignored, you can also use wildcards when using the 


          
           
@JsonIgnoreProperties
annotation. Wildcards allow you to specify a pattern that matches multiple properties. For example, the following example uses a wildcard to ignore all properties starting with “xyz”: 

          
           
// Ignore all properties starting with 'xyz' 
@JsonIgnoreProperties("xyz*") 
public class Person { ... }

You can use both the “*” and “?” characters when specifying wildcard patterns with the 


          
           
@JsonIgnoreProperties
annotation. The “*” character is used to match zero or more characters, while the “?” character is used to match a single character.
