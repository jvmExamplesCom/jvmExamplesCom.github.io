android.view.SurfaceView

The android.view.SurfaceView class is a view that provides a dedicated drawing surface embedded inside of a view hierarchy. This is useful for drawing graphics that are not possible with the standard View system, such as animations, game graphics, or video playback. The surface can be associated with a SurfaceHolder object which provides access and control over the underlying surface.

Creating a Surface View

To create a SurfaceView, you will usually extend the SurfaceView class and override at least the methods in the SurfaceHolder.Callback. These methods will be executed when the surface is created or destroyed, and when it is changed. You can use them to handle these events. Here is an example of a custom SurfaceView implementation:


          
           
public class MySurfaceView extends SurfaceView implements SurfaceHolder.Callback {
    public MySurfaceView(Context context) {
        super(context);
        // Add callback
        getHolder().addCallback(this);
    }

    @Override
    public void surfaceCreated(SurfaceHolder holder) {
        // create drawing surface here
    }

    @Override
    public void surfaceChanged(SurfaceHolder holder, int format, int width, int height) {
        // update size of drawing surface here
    }

    @Override
    public void surfaceDestroyed(SurfaceHolder holder) {
        // destroy drawing surface here
    }
}

Drawing on the Surface

Once you have a valid SurfaceHolder, you can obtain a Canvas object and draw onto it. To do this, you will need to create a BigPicture object, which handles the drawing operations. Here is an example:


          
           
BigPicture bigPicture = new BigPicture();

The BigPicture object will contain all the information necessary to render the picture. For example, it might contain an array of pixels, an array of colors, or an array of textures. It is up to you to decide what kind of data your BigPicture object contains.

Animations

To do this, you can use the same techniques as above, but instead of drawing a single image, you will need to draw a series of images, each one slightly different from the last. To achieve a smooth animation, it is important that each frame of the animation is drawn as quickly as possible. One way to do this is to use a separate thread for rendering.
