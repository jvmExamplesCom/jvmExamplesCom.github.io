android.view.accessibility.AccessibilityEvent

The android.view.accessibility.AccessibilityEvent class is used for sending and receiving accessibility events in Android applications. It is part of the Android accessibility framework and provides a way for applications to report events that can be used by Assistive Technologies (ATs) to provide the user with better access to their application. This guide will review what an AccessibilityEvent is, how to create and send one, and provide code examples showing how to do so.

What is an AccessibilityEvent?

An AccessibilityEvent is an event object sent by an Android application when something changes within the view hierarchy. These events can include changes in focus, selection, or content changes, among others. When these events are sent, they are broadcast to all the Assistive Technologies that are registered to receive them. The registered ATs can then make use of this information to improve the user experience for the application.

How to Create and Send an AccessibilityEvent

Creating and sending an AccessibilityEvent consists of three steps: 1. Create an AccessibilityEvent object. 2. Set the values of the object. 3. Send the event using View.sendAccessibilityEvent().

Step 1: Create an AccessibilityEvent Object

The first step is to create an AccessibilityEvent object that can be used to send an event. 

          
           
AccessibilityEvent event = AccessibilityEvent.obtain();

Step 2: Set the Values of the Object

The next step is to set the values of the object so that it contains information about the type of event, the source of the event, the content of the event, and the action that was performed. All of these values can be set by calling the various setters on the AccessibilityEvent object.

Set Event Type

The type of event can be set with the setEventType() method. The most commonly used types are TYPES_TEXT_CHANGED, TYPES_VIEW_SELECTED, and TYPES_VIEW_FOCUSED. 

          
           
event.setEventType(AccessibilityEvent.TYPES_TEXT_CHANGED);

Set Event Source

The source of the event can be set with the setSource() method. This should be set to the view the event is associated with. 

          
           
event.setSource(myTextView);

Set Event Content

The event content can be set with the setContentDescription() method. This should be set to the text that has been changed. 

          
           
event.setContentDescription("New Text: " + myTextView.getText());

Set Event Action

The action that was performed can be set with the setAction() method. This should be set to ACTION_TEXT_CHANGED if the text was changed, ACTION_VIEW_SELECTED if a view was selected, etc. 

          
           
event.setAction(AccessibilityEvent.ACTION_TEXT_CHANGED);

Step 3: Send the Event Using View.sendAccessibilityEvent()

Finally, once the values of the AccessibilityEvent have been set, the event can be sent by calling the View.sendAccessibilityEvent() method. This should be called from the view the event is associated with, so that the source of the event can be set correctly. 

          
           
myTextView.sendAccessibilityEvent(event);

Conclusion

This guide has reviewed the basics of creating and sending AccessibilityEvents in Android and provided code examples showing how to do so. By making use of AccessibilityEvents, Android applications can provide better support for Assistive Technologies, which can lead to improved accessibility and a better user experience.
