android.support.v4.view.GravityCompat

Android SDK's GravityCompat class provides a compatibility library to support the Gravity class of the Android framework in all versions of the platform back to API level 4. It is located under 


          
           
android.support.v4.view
. By using GravityCompat as a substitute for Gravity, an application can ensure that its behavior is consistent across all platforms.

Overview

The GravityCompat class provides a set of convenience methods which are meant to be used as substitutes for Gravity methods. All of these methods are static and take in two parameters: the Gravity constant and a set of flags.

Example

Here is an example of how to use GravityCompat when defining the layout for a TextView:


          
           
<TextView
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:gravity="@android:id/gravityCompat.start|center_vertical"/>

In this example, the GravityCompat class is being used with the “start” and “center_vertical” constants. This will cause the TextView to be centered vertically on the screen, while being aligned left (or start) horizontally.

API Usage Details

Most of the methods provided in GravityCompat are simple wrappers of Gravity methods. For example, 


          
           
GravityCompat.getAbsoluteGravity(int, int)
is an alternative version of 

          
           
Gravity.getAbsoluteGravity(int, int)
. The main difference is that in GravityCompat, the flags parameter will accept either pre-defined constants from the Gravity class or constants from the GravityCompat class.

Some of the methods provided by GravityCompat also have new functionality. For instance, 


          
           
GravityCompat.apply(int, int, int, Rect, int, int, Rect)
will apply a partial gravity, or a subset of gravity constants. Partial gravity is useful for situations where one wants to align only 1 or 2 edges of a view instead of all four.

Summary

In summary, the GravityCompat class provides an easy way to ensure an app's behavior is consistent across different Android versions. Using GravityCompat instead of Gravity can help make an app more future-proof, while also providing some additional functionality.
