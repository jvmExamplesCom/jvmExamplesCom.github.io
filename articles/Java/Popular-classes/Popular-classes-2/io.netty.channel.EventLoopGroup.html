EventLoopGroup in Java

In Java, EventLoopGroup is a type of class from 
io.netty.channel
package which is used to handle and process I/O operations. It manages a pool of Threads with different event loops for handling various channels operations such as connection making, data transferring, etc. In general, the Networking application is built upon multiple EventLoopGroups; one for the main thread and the other for worker threads.

Creating an EventLoopGroup Object

The simplest way of creating an EventLoopGroup object is using its static method 

newEventLoopGroup()
. This takes two parameters: the number of threads in the group, and the thread factory. 

          
           
// construct EventLoopGroup object
EventLoopGroup group = newEventLoopGroup(2, new DefaultThreadFactory());

The EventLoopGroup can be specified by a number of parameters including the type of the group, the number of threads, and the thread factory.


          
           
// specify the EventLoopGroup with type, number of threads and thread factory

Before using the EventLoopGroup, we need to initialize it. The general initialization procedure includes setting up the threads, allocating resources and registering handlers.


          
           
// initialize the EventLoopGroup
group.initialize();

Shutting down the EventLoopGroup

When the EventLoopGroup is not needed anymore, we need to shut it down. This releases all the resources allocated to this group.
