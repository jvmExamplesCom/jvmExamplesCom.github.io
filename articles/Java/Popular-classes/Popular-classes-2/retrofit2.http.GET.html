Retrofit2.Http.GET

Retrofit is an easy to use library that allows developers to make HTTP requests easier to manage. It has the ability to easily add functionality such as request headers and body payloads, while also providing a simple way of making asynchronous requests. The Retrofit2.Http.GET method is a convenient way to make GET requests to a remote endpoint.

Code Examples


          
           
@GET("/endpoint")
Call<ResponseBody> getEndpoint();

          

          
           
@GET("/endpoint?param={param}")
Call<ResponseBody> getEndpoint(@Path("param") int param);

          

          
           
@GET("/endpoint")
Call<ResponseBody> getEndpoint(@Query("param") String param);

          

          
           
@GET("/endpoints")
Call<List<ResponseBody>> getEndpoints();

          

          
           
@GET("/endpoint/{id}")
Call<ResponseBody> getEndpointById(@Path("id") int id);

          

          
           
@GET("/endpoint")
Call<List<ResponseBody>> getEndpoints(@QueryMap Map<String, Object> queryParams);

          

          
           
@GET("/endpoint/{id}")
Call<ResponseBody> getEndpointById(@Path("id") int id, @QueryMap Map<String, Object> queryParams);

Summary

Retrofit2.Http.GET is a great way to make GET requests from a remote endpoint. It makes it easier to add custom functionality such as request headers and body payloads. There are multiple ways of making GET requests, such as with query parameters, path parameters and query map parameters. With these code examples, you can quickly and easily make GET requests with Retrofit2.Http.GET.
