javax.ws.rs.Consumes

The javax.ws.rs.Consumes annotation is used to specify a list of media types that the Java method can consume, such as XML or JSON. The media type will be specified in the HTTP request by the caller. Placing the annotation on a class or method can limit the media type of the content that can be processed by that class or method.

Examples of javax.ws.rs.Consumes

Below are some examples of using the javax.ws.rs.Consumes annotation:


          
           
@Consumes(MediaType.APPLICATION_XML)
public Response postXML(String xmlString) {
  // Process the XML string and return a response
}

@Consumes(MediaType.APPLICATION_JSON)
public Response postJson(String jsonString) {
  // Process the JSON string and return a response
}
The 
@Consumes
annotation can also be used to limit the media type the method can consume. For example, if the method should only accept XML, it can be defined as: 

          
           
@Consumes({MediaType.APPLICATION_XML})
public Response postXML(String xmlString) {
  // Process the XML string and return a response
}

javax.ws.rs.Consumes Sample Output

When the above methods are invoked with the appropriate media type, they will return a response based on the data passed. If a call is made with an incorrect media type, an error code (415 Unsupported Media Type) will be returned instead.
