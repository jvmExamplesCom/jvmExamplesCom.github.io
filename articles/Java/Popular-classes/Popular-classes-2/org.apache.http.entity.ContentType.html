org.apache.http.entity.ContentType

The org.apache.http.entity.ContentType class represents a content type as described in RFC 2045 and RFC 2616. It consists of a content-type, media type, and subtype. Content types are used to describe the content of a message entity. It is an abstract class based on which objects of specific content type are created.

Creating a ContentType Object

A content type object can be created from either the toString() method or from the contentType() constructor. The toString() method takes either a String or an InputStream. The contentType() constructor takes three parameters. These are the content-type, media type, and subtype.


          
           
String contentTypeStr = "text/xml; charset=utf-8";
ContentType contentType = ContentType.create(contentTypeStr);

String mediaType = "text";
String subType = "xml";
String charset = "utf-8";
ContentType contentType = ContentType.create(mediaType, subType, charset);

The above code creates two content type objects using the two methods discussed above. Note that when using the contentType() constructor, the charset field is optional.

Getting the Content Type

Once a content type object has been created, its various components can be accessed. The getCharset() method returns the charset value and the getMimeType() method returns the MIME type value.


          
           
ContentType contentType = ContentType.create(mediaType, subType, charset);
String mimeType = contentType.getMimeType();
String charset = contentType.getCharset();
// mimeType is "text/xml"
// charset is "utf-8"

Comparing Content Types

The contentTypeEquals() method can be used to compare two content type objects. This method will return true if both content types have the same media type, subtype and charset values.


          
           
ContentType contentType1 = ContentType.create("text", "xml", "utf-8");
ContentType contentType2 = ContentType.create("text", "html", "utf-8");
boolean equals = contentType1.contentTypeEquals(contentType2);
// equals is false

Content Type Parameters

Content type parameters are additional information included in a content type header. They are specified using the format of "name=value". A content type object can have multiple parameters associated with it. Parameter names are case-insensitive.


          
           
ContentType contentType = ContentType.create("text", "xml", "utf-8");
contentType.setParameter("charset", "utf-16");
String parameterValue = contentType.getParameter("charset");
// parameterValue is "utf-16"

The ContentType class also provides methods for searching, adding or removing parameters. The removeParameter(), containsParameter() and getParameter() methods provide the ability to search for and manipulate parameters.

Converting to a String

Once a content type has been created, it can be converted back to a string value. This is done using the toString() method. The toString() method will return a string corresponding to the given content type object.


          
           
ContentType contentType = ContentType.create("text", "xml", "utf-8");
contentType.setParameter("charset", "utf-16");
String contentTypeStr = contentType.toString();
// contentTypeStr is "text/xml; charset=utf-16"
