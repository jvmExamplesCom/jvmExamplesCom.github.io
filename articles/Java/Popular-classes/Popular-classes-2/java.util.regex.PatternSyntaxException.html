java.util.regex.PatternSyntaxException

A java.util.regex.PatternSyntaxException is a type of exception thrown when an error is encountered in a regular expression pattern. It can happen if the pattern is not well formed, if special characters don't have their expected meanings, or if capturing groups are nested too deeply. This exception can also be thrown when using the replaceAll method of the String class.

When this exception is thrown, it will include information about where in the pattern the problem occurred, and a brief description of the syntax error.

Example 1

This example uses incorrect syntax for specifying a group in a regular expression pattern. It results in a PatternSyntaxException being thrown:


          
           
String pattern = "\\(\\d+\\)";  
try {  
  Pattern.compile(pattern);  
} catch (PatternSyntaxException pse) {  
  System.out.println("Error - " + pse.getDescription());  
}
In this example, the output would be: 

          
           
Error - Unclosed group near index 2
