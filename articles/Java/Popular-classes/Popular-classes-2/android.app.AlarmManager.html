Android.app.AlarmManager

The Android.app.AlarmManager class is designed for scheduling repeating alarms and one-time alarms. It provides access to the system alarm services and allows you to schedule alarms with a trigger time and an action to be performed when the alarm is triggered. Alarms scheduled using AlarmManager persist across device reboots.

Creating an Alarm

An AlarmManager instance can be obtained by calling Context.getSystemService(Context.ALARM_SERVICE). Once you have an AlarmManager instance, you can set up a repeating alarm using the setRepeating() method or a one-time alarm using the set() method. For example, the following code creates a one-time alarm that triggers 10 seconds from now: 

          
           
// Create an AlarmManager instance 
    AlarmManager alarmManager = (AlarmManager)context.getSystemService (Context.ALARM_SERVICE); 
    
   // Create a PendingIntent to be triggered when the alarm fires 
    Intent intent = new Intent("example.ALARM_ACTION"); 
    PendingIntent pendingIntent = PendingIntent.getBroadcast(context, 0, intent, 0); 
    
    // Set an alarm to fire in 10 seconds 
    long triggerTime = System.currentTimeMillis() + 10000; 
    alarmManager.set(AlarmManager.RTC_WAKEUP, triggerTime, pendingIntent);
In this example, we create a PendingIntent with an action of "example.ALARM_ACTION". When the alarm is triggered, the intent is broadcasted to any receivers that are registered for this action. You can also use a broadcast receiver as the target for the PendingIntent.

Cancelling an Alarm

Alarms can be cancelled using the cancel() method. This will remove any pending alarms with a matching PendingIntent. For example, the following code cancels the alarm created above: 

          
           
alarmManager.cancel(pendingIntent);

Working with Elapsed Real Time

Alarms can be triggered based on the time since device boot (elapsed real time). The set() and setRepeating() methods both take a parameter of type "long" that specifies the time since device boot at which the alarm should be triggered. The getElapsedRealtime() method on the SystemClock class can be used to obtain the current time since device boot in milliseconds. For example, the following code creates a repeating alarm that triggers every 10 seconds after device reboot. 

          
           
// Create an AlarmManager instance 
    AlarmManager alarmManager = (AlarmManager)context.getSystemService (Context.ALARM_SERVICE); 
    
   // Create a PendingIntent to be triggered when the alarm fires 
    Intent intent = new Intent("example.ALARM_ACTION"); 
    PendingIntent pendingIntent = PendingIntent.getBroadcast(context, 0, intent, 0); 
    
    // Set an alarm to fire in 10 seconds 
    long triggerTime = SystemClock.elapsedRealtime() + 10000; 
    alarmManager.setRepeating(AlarmManager.ELAPSED_REALTIME_WAKEUP, triggerTime, 10000, pendingIntent);

Working with Wall Clock Time

Alarms can also be triggered based on the current wall clock time. The set() and setRepeating() methods both take a parameter of type "Calendar" that specifies the date/time at which the alarm should be triggered. The getTimeInMillis() method on the Calendar class can be used to obtain the time in milliseconds. For example, the following code creates a repeating alarm that triggers every 10 seconds. 

          
           
// Create an AlarmManager instance 
    AlarmManager alarmManager = (AlarmManager)context.getSystemService (Context.ALARM_SERVICE); 
    
   // Create a PendingIntent to be triggered when the alarm fires 
    Intent intent = new Intent("example.ALARM_ACTION"); 
    PendingIntent pendingIntent = PendingIntent.getBroadcast(context, 0, intent, 0); 
    
    // Set an alarm to fire in 10 seconds 
    Calendar calendar = Calendar.getInstance(); 
    long triggerTime = calendar.getTimeInMillis() + 10000; 
    alarmManager.setRepeating(AlarmManager.RTC_WAKEUP, triggerTime, 10000, pendingIntent);

Conclusion

The Android.app.AlarmManager class provides a flexible way to schedule repeating alarms and one-time alarms. Alarms scheduled using AlarmManager can be based either on elapsed real time or wall clock time and will persist across device reboots.
