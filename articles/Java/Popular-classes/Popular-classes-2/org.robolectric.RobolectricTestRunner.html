org.robolectric.RobolectricTestRunner

RobolectricTestRunner is a test runner for the JUnit4 testing framework designed specifically for the Robolectric framework. It provides a convenient way to execute tests against a virtual Android environment.

Getting Started

To get started with RobolectricTestRunner, you will need to create a new test class that extends from org.robolectric.RobolectricTestRunner:


          
           
public class MyTest extends RobolectricTestRunner {

    @Test
    public void testSomething() {
        // your test code here
    }
}

With the test class created, you can now run it using the standard JUnit test runner (JUnitCore) or any other JUnit-compatible runner:


          
           
JUnitCore.runClasses(MyTest.class);

RobolectricTestRunner will automatically set up a virtual Android environment in which tests can be executed safely and without any real devices or emulators.

Using Configuration Options

RobolectricTestRunner supports a number of configuration options that can be used to control aspects of the virtual Android environment:

  • testSdk - the version of the Android SDK to use for the virtual environment. Defaults to the latest available version.
  • testApiLevel - the API level of the virtual environment. This can be used to emulate older Android versions.
  • testManifest - path to an Android manifest XML file, which can be used to define specific settings for the virtual environment.

These configuration options can be specified when creating the test class by passing an instance of org.robolectric.Config to the test constructor:


          
           
public class MyTest extends RobolectricTestRunner {

    public MyTest(Config config) {
        super(config);
    }

    @Test
    public void testSomething() {
        // your test code here
    }
}

The Config arguments can then be passed when running the test class:


          
           
JUnitCore.runClasses(MyTest.class, Config.of(testSdk: 28));

Conclusion

RobolectricTestRunner provides a convenient way to execute tests against a virtual Android environment. With its configuration options, it makes it easy to customize the environment for specific testing scenarios.
