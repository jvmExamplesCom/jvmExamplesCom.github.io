org.aspectj.lang.annotation.Pointcut

The org.aspectj.lang.annotation.Pointcut annotation is used to define a pointcut expression in the AspectJ language. It is mainly used to identify join points that can be advised using AspectJ-style advice. A Pointcut expression can be used to identify certain methods, classes, and execution paths in a program. With it, advice can be applied at the join point identified by the expression. This is the main tool for applying Aspect-Oriented Programming (AOP) in Java.

Example

The following code is an example of a Pointcut expression which identifies all methods in the package com.test:


          
           
@Pointcut("execution(* com.test.*(..))")
public void pointcutName(){ }

The expression can then be applied in an aspect, as follows:


          
           
@Before("pointcutName()")
public void adviseMethod() {}

Syntax

The syntax of a Pointcut expression is similar to that of a method signature in Java. It consists of four components: the access type, the return type, the method name, and the argument signature.

The access type of a Pointcut expression is represented as “execution” in AspectJ. It specifies the access type of the advised join point. The return type is the return type of the method matched by the Pointcut expression. The method name is the name of the method matched by the expression. Lastly, the argument signature is the parameter types of the method.


          
           
@Pointcut("execution([access type] [return type] [method name]([argument signature]))")

Wildcards

Wildcards can be used to define Pointcut expressions that match any return type, method name, or argument signature. For example, “*” can be used to match any return type, while “..” can be used to match any argument signature.


          
           
@Pointcut("execution(* *(..))")
public void pointcutName(){ }

This Pointcut expression matches any method with any return type, regardless of the argument signature.

Argument Wildcards

Argument wildcards can also be used to match certain arguments of a method. For example, the “+” wildcard matches any subtype of a specified type. For instance, “java.lang.Number+” matches any subtype of java.lang.Number, such as Integer, Double, etc.


          
           
@Pointcut("execution(* *(java.lang.Number+))")
public void pointcutName(){ }

This Pointcut expression matches any method with one argument of a subtype of java.lang.Number.
