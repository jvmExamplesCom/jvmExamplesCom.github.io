java.nio.file.SimpleFileVisitor

The java.nio.file.SimpleFileVisitor class is part of the Java NIO package and provides a straightforward way for developers to implement file tree traversal algorithms. The class provides four different callback methods which can be used to perform custom logic during the file tree walk process.

Methods

  • preVisitDirectory(final Path dir, final BasicFileAttributes attrs) - This method is invoked before entering any directory. It is provided with the directory's path and basic attributes information.
  • postVisitDirectory(final Path dir, final IOException exc) -This method is invoked after leaving a directory. It is provided with the directory's path and an 
    IOException
    object indicating the reason why it failed to visit the directory. If the visit was successful, then the 
    IOException
    parameter is null.
  • visitFile(final Path file, final BasicFileAttributes attrs) - This method is invoked when encountering a file in the tree. It is provided with the file's path and basic attributes information.
  • visitFileFailed(final Path file, final IOException exc) - This method is invoked when recognizing a file in the tree but failing to visit the file. It is provided with the file's path and an 
    IOException
    object indicating the reason why it failed to visit the file.

Examples

Here is an example demonstrating how to implement a simple file tree walker using the 

SimpleFileVisitor
class. 

          
           
public class SimpleFileVisitorDemo {

    public static void main(String[] args) throws IOException {
        Path startingDir = Paths.get("/path/to/root/directory");
        
        Files.walkFileTree(startingDir, new SimpleFileVisitor<Path>() {
            public FileVisitResult preVisitDirectory(Path dir, BasicFileAttributes attrs) {
                // Perform custom logic here
                System.out.println("Visiting directory: " + dir.toString());
                return FileVisitResult.CONTINUE;
            }

            public FileVisitResult visitFile(Path file, BasicFileAttributes attrs) {
                // Perform custom logic here
                System.out.println("Visiting file: " + file.toString());
                return FileVisitResult.CONTINUE;
            }

            public FileVisitResult visitFileFailed(Path file, IOException exc) {
                // Perform custom logic here
                System.out.println("Failed to visit file: " + file.toString());
                return FileVisitResult.CONTINUE;
            }

            public FileVisitResult postVisitDirectory(Path dir, IOException exc) {
                // Perform custom logic here
                System.out.println("Finished visiting directory: " + dir.toString());
                return FileVisitResult.CONTINUE;
            }
        });
    }
}

The 
preVisitDirectory()
method can also be used to check if a directory should even be visited. In this example, all directories beginning with the letter 'a' will be skipped: 

          
           
public class SimpleFileVisitorDemo {

    public static void main(String[] args) throws IOException {
        Path startingDir = Paths.get("/path/to/root/directory");
        
        Files.walkFileTree(startingDir, new SimpleFileVisitor<Path>() {
            public FileVisitResult preVisitDirectory(Path dir, BasicFileAttributes attrs) {
                // Skip directory if it begins with letter "a"
                if (dir.getFileName().toString().startsWith("a")) {
                    System.out.println("Skipping directory: " + dir.toString());
                    return FileVisitResult.SKIP_SUBTREE;
                } else {
                    System.out.println("Visiting directory: " + dir.toString());
                    return FileVisitResult.CONTINUE;
                }
            }
        });
    }
}
