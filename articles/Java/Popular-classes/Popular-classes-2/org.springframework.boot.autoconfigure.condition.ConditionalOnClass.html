org.springframework.boot.autoconfigure.condition.ConditionalOnClass

The class org.springframework.boot.autoconfigure.condition.ConditionalOnClass is a convenient way to conditionally enable auto-configuration depending on whether certain classes or packages are present on the classpath. This annotation is used together with @Configuration to indicate that the configuration should be registered if the specified class or package is available.

Example Usage

Let's consider an example where we have an application that needs to use a separate library for some functionality.


          
           
@Configuration
@ConditionalOnClass(LibraryClass.class)
public class MyConfiguration {

    @Bean
    public SomeBean someBean() {
        // create and return SomeBean instance
    }
}

In this case, the @Configuration will only be registered if the LibraryClass class is available. If the class is not present, then the @Configuration will not be registered and no beans will be created.

Attributes

  • value: The classes or packages to check for.
  • name: An optional name for the configuration.

Annotations

This annotation has to be used in conjunction with a @Configuration annotation. It can also be combined with other auto-configuration annotations such as @ConditionalOnMissingBean, @ConditionalOnProperty or @ConditionalOnWebApplication.

Conclusion

The org.springframework.boot.autoconfigure.condition.ConditionalOnClass annotation is a convenient way to conditionally enable auto-configuration depending on whether certain classes or packages are present on the classpath.
