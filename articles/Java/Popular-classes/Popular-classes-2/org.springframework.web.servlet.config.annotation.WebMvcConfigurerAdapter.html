org.springframework.web.servlet.config.annotation.WebMvcConfigurerAdapter

The 

          
           
org.springframework.web.servlet.config.annotation.WebMvcConfigurerAdapter
class is a sub-class of the 

          
           
org.springframework.web.servlet.config.annotation.WebMvcConfigurer
interface, providing empty implementations of the necessary methods. It is primarily used to enable Java-based configuration of the Spring MVC framework by extending the adapter and overriding the necessary methods.

Example 1: Basic Configuration with WebMvcConfigurerAdapter

In this example we use the 

          
           
WebMvcConfigurerAdapter
class to configure basic Spring MVC features such as view controllers and message converters. 

          
           
@Configuration
public class WebConfiguration extends WebMvcConfigurerAdapter {

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/").setViewName("index");
    }

    @Override
    public void configureMessageConverters(List<HttpMessageConverter<?>> converters) {
        HttpMessageConverter<String> stringConverter = new StringHttpMessageConverter();
        converters.add(stringConverter);
    }
}

Example 2: Configuring RequestMappings with WebMvcConfigurerAdapter

In this example, we use the 

          
           
WebMvcConfigurerAdapter
class to configure the request mapping for the application. We can add multiple mappings for specific URL paths and also specify the HTTP method type for those paths. 

          
           
@Configuration
public class WebConfiguration extends WebMvcConfigurerAdapter {

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/").setViewName("index");
    }

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/static/**")
                .addResourceLocations("classpath:/static/");
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new LoggingInterceptor())
                .addPathPatterns("/**");
    }

    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
                .allowedOrigins("http://domain.com")
                .allowedMethods("PUT", "DELETE")
                .allowedHeaders("header1", "header2", "header3")
                .exposedHeaders("header1", "header2")
                .allowCredentials(false).maxAge(3600);
    }

    @Override
    public void addRequestMappings(RequestMappingHandlerRegistry registry) {
        registry.addMapping("/user/{id}").method(RequestMethod.GET);
        registry.addMapping("/user/update").method(RequestMethod.PUT);
        registry.addMapping("/user/delete/{id}").method(RequestMethod.DELETE);
    }
}

Example 3: Configure ContentNegotiatingViewResolver with WebMvcConfigurerAdapter

In this example, we use the 

          
           
WebMvcConfigurerAdapter
class to configure the ContentNegotiatingViewResolver view resolver. We can specify a view name for a particular request format. 

          
           
@Configuration
public class WebConfiguration extends WebMvcConfigurerAdapter {

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/").setViewName("index");
    }

    @Override
    public void configureContentNegotiation(ContentNegotiationConfigurer configurer) {
        configurer.favorParameter(true)
                .ignoreAcceptHeader(true)
                .defaultContentType(MediaType.APPLICATION_JSON)
                .mediaType("xml", MediaType.APPLICATION_XML)
                .mediaType("json", MediaType.APPLICATION_JSON);
    }

    @Bean
    public ViewResolver contentNegotiatingViewResolver(
            ContentNegotiationManager manager) {
        ContentNegotiatingViewResolver resolver =
                new ContentNegotiatingViewResolver();
        resolver.setContentNegotiationManager(manager);
        List<View> views = new ArrayList<>();
        views.add(new MappingJackson2JsonView());
        views.add(new MarshallingView());
        resolver.setDefaultViews(views);
        return resolver;
    }
}
