Java NIO File Visit Result Overview

The Java NIO File Visit Result class is a utility interface available in the java.nio.file package which provides a convenience for the returning of results from a file tree traversal.

It is used as part of the 
FileVisitor
interface which is used to visit the elements of a file tree in a recursive manner. The 
FileVisitor
interface provides four different type of method calls on encountering a file or directory:
  • Pre-visit a directory.
  • Visit a file.
  • Post-visit a directory.
  • Visit a directory.
Each of these callback methods requires a 
FileVisitResult
object to be returned. This allows the caller to control the behaviour of the file visitor, allowing it to continue, terminate or skip a directory.

The 
FileVisitResult
Enum Values

The 
FileVisitResult
enum defines four different values that can be returned from a file visitor callback: 
  • CONTINUE
    - Tells the file visitor to continue visiting the next element in the tree.
  • TERMINATE
    - Tells the file visitor to cease visiting any further elements in the tree.
  • SKIP_SUBTREE
    - Tells the file visitor to skip the current subtree and move to the next element.
  • SKIP_SIBLINGS
    - Tells the file visitor to skip any siblings of this element and move to the next element.

Using 
FileVisitResult

Below is an example of how the 
FileVisitResult
can be used with a file visitor. In this example, a file visitor is created which prints out all of the files in the directory specified: 

          
           
public class MyFileVisitor implements FileVisitor<Path> {
    @Override
    public FileVisitResult preVisitDirectory(Path dir, BasicFileAttributes attrs) throws IOException {
        System.out.println("Entering directory: " + dir.toString());
        return FileVisitResult.CONTINUE;
    }
 
    @Override
    public FileVisitResult visitFile(Path file, BasicFileAttributes attrs) throws IOException {
        System.out.println("Printing file: " + file.toString());
        return FileVisitResult.CONTINUE;
    }
 
    @Override
    public FileVisitResult visitFileFailed(Path file, IOException exc) throws IOException {
        return FileVisitResult.CONTINUE;
    }
 
    @Override
    public FileVisitResult postVisitDirectory(Path dir, IOException exc) throws IOException {
        return FileVisitResult.CONTINUE;
    }
}

Path start = Paths.get("/path/to/start/dir");
Files.walkFileTree(start, new MyFileVisitor());

value is returned by each of the methods. This tells the file visitor to proceed to the next item in the tree. Alternatively, a different 
FileVisitResult
value could be returned which would change the behaviour of the file visitor.

Conclusion

The 
FileVisitResult
class provides a convenient way to control the behaviour of a file visitor when travelling through a file tree. It allows the behaviour of the file visitor to be dynamically changed dependent on the result returned by the callback methods.
