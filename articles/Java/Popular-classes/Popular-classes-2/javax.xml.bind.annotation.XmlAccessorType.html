javax.xml.bind.annotation.XmlAccessorType

javax.xml.bind.annotation.XmlAccessorType is a class and an annotation type used for controlling the mapping of fields and properties in Java objects to XML constructs and vice versa.

Overview

The XmlAccessorType annotation defines how fields and properties of a class are mapped to XML elements.

By default, all non-static fields and properties in a class are mapped to XML elements. This behaviour can be changed by using the XmlAccessorType annotation. The annotation has two possible values: XmlAccessType.FIELD and XmlAccessType.PROPERTY.

If XmlAccessType.FIELD is specified, only fields will be mapped to XML elements. If XmlAccessType.PROPERTY is specified, only properties will be mapped to XML elements.

Example

Consider the following class: 

          
           
public class Foo {
  private int x;
  private int y;

  public int getX() {
  	return x;
  }

  public void setX(int x) {
  	this.x = x;
  }

  public int getY() {
  	return y;
  }

  public void setY(int y) {
  	this.y = y;
  }
}

By default, all non-static fields and properties in the class Foo will be mapped to XML elements.

To change this behaviour, we can use the XmlAccessorType annotation:


          
           
@XmlAccessorType(XmlAccessType.FIELD)
public class Foo {...}

This annotation specifies that only fields should be mapped to XML elements.

For example, the field x will be mapped to the XML element , and the property y will not be mapped to an XML element.

Similarly, if we specify XmlAccessType.PROPERTY, only properties will be mapped to XML elements.
