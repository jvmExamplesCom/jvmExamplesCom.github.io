Using java.nio.file.StandardCopyOption

The java.nio.file.StandardCopyOption class provides enumeration constants suitable for use with the Files.copy(Path, Path, CopyOption...) method.

It is used to indicate how symbolic links are handled by the copy operation. Symbolic links are special files which contain a reference to another file.

CopyOption Constants

The following constants are defined: 
  • COPY_ATTRIBUTES
    - Indicates that the attributes of the source file should be copied to the target file.
  • REPLACE_EXISTING
    - Indicates that if the target file already exists it should be replaced.
  • ATOMIC_MOVE
    - Indicates that the target file should be moved in an atomic operation.

Example Usage

The following example shows how to use the 
Files.copy()
method to copy a file from one directory to another, preserving the attributes of the original file: 

          
           
Path src = Paths.get("srcDir");
Path dest = Paths.get("destDir");
Files.copy(src, dest, StandardCopyOption.COPY_ATTRIBUTES);
The following example shows how to use the 
Files.copy()
method to replace an existing file, but preserve its attributes: 

          
           
Path src = Paths.get("srcFile");
Path dest = Paths.get("destFile");
Files.copy(src, dest, StandardCopyOption.REPLACE_EXISTING, StandardCopyOption.COPY_ATTRIBUTES);

Atomic Move

The 
ATOMIC_MOVE
constant is used to indicate that the target file should be moved in an atomic operation. This means that the move will either succeed or fail without leaving any partially completed files. This constant should be used if the source and destination files are on the same filesystem, as in this example: 

          
           
Path src = Paths.get("srcFile");
Path dest = Paths.get("destFile");
Files.move(src, dest, StandardCopyOption.ATOMIC_MOVE);

Conclusion

In summary, the java.nio.file.StandardCopyOption class provides enumeration constants suitable for use with the Files.copy(Path, Path, CopyOption...) method. The constants can be used to specify how symbolic links should be handled by the copy operation, and to specify whether the attributes of the source file should be preserved when copying or replacing the target file.
