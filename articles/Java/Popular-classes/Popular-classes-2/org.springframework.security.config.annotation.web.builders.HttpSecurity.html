org.springframework.security.config.annotation.web.builders.HttpSecurity

The org.springframework.security.config.annotation.web.builders.HttpSecurity class is a part of the Spring Security framework and provides configuration for securing web applications. It allows users to specify which roles are authorized to access certain URLs and also enables them to configure authentication and authorization policies.

Example 1: Configuring Authorization using HttpSecurity

In this example a Spring MVC application sets up an authorization system using the HttpSecurity class.


          
           
@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/admin/**").hasRole("ADMIN")
                .antMatchers("/user/**").hasAnyRole("ADMIN", "USER")
                .antMatchers("/**").permitAll()
            .and()
            .formLogin();
    }
}

In this configuration, all URLs are accessible by default but URLs prefixed with “/admin/” can only be accessed by users with the role “ADMIN” and those prefixed with “/user/” can be accessed by users with either the “ADMIN” or “USER” roles. Lastly, form login is enabled which allows users to authenticate using a web form.

Example 2: Securing Paths using HttpSecurity

This example shows how the HttpSecurity class can be used to secure individual paths in a Spring MVC application.


          
           
@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests()
            .antMatchers("/admin/**").hasRole("ADMIN")
            .antMatchers("/user/**").hasAnyRole("ADMIN", "USER")
            .antMatchers("/secure/**").authenticated()
            .anyRequest().permitAll();
    }
}

In this configuration, all URLs are accessible by default but URLs prefixed with “/secure/” require authentication regardless of the user’s role, while URLs prefixed with “/user/” can be accessed by users with either the “ADMIN” or “USER” roles and those prefixed with “/admin/” can only be accessed by users with the role “ADMIN”.

Example 3: Configuring Authentication using HttpSecurity

This example shows how to configure authentication using the HttpSecurity class.


          
           
@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http
            .authorizeRequests()
                .antMatchers("/public/**").permitAll()
            .and()
            .formLogin()
                .loginPage("/login")
                .defaultSuccessUrl("/user/home", true);
    }
}

In this configuration, all URLs prefixed with “/public/” are accessible by default while all other URLs require authentication. Login is handled using a web form on the page “/login”, and upon successful authentication the user is redirected to the page “/user/home”.
