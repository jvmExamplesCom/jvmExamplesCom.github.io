org.springframework.boot.autoconfigure.EnableAutoConfiguration

The org.springframework.boot.autoconfigure.EnableAutoConfiguration annotation is used to enable auto-configuration of the Spring ApplicationContext. It can be applied to any Java class, and it will trigger the Spring Boot auto-configuration process.

Example

The following code example shows how to use the EnableAutoConfiguration annotation:


          
           
@SpringBootApplication
@EnableAutoConfiguration
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }

}

In this example, we have annotated the MyApplication class with @EnableAutoConfiguration. This annotation triggers the auto-configuration process in Spring Boot.

How does auto-configuration work?

When auto-configuration is enabled, Spring Boot looks for specific classes in the classpath of your application. These classes are known as auto-configuration classes. If any of these auto-configuration classes are found, then Spring Boot will automatically configure your application based on their configuration.

For example, if your application contains a class that is annotated with @Configuration, then Spring Boot will automatically use it to configure your application. Similarly, if your application contains a class that is annotated with @Component, then Spring Boot will automatically add it to the Spring application context.

Auto-configuration is a powerful feature of Spring Boot, as it allows you to quickly and easily configure your application without having to manually define every bean or configuration property.

Customize Auto-Configuration

You can also customize the auto-configuration process in Spring Boot by using various annotations and classes. For example, you can use the @EnableConfigurationProperties annotation to enable configuration properties in your application. You can also use the @ConfigurationProperties annotation to specify custom configuration properties.

You can also use the @ConditionalOnBean annotation to determine whether a particular bean should be configured or not. This allows you to fine-tune the auto-configuration process based on the beans that are present in your application.
