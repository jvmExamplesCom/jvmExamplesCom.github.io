Java.util.MissingResourceException

Java.util.MissingResourceException is thrown by the ResourceBundle class when a requested resource bundle can not be located. The class MissingResourceException extends the RuntimeException class and provides accessors for the following variables:

  • resourceBundleName - which is the name of the resource bundle that was not found
  • key - which is the key of the resource that could not be located

The following example shows how to handle a MissingResourceException if it were to occur when attempting to retrieve a resource bundle called 

MyResources
: 

          
           
try {  
  ResourceBundle myResources = ResourceBundle.getBundle("MyResources");
  // ...
} catch(MissingResourceException mre) {
  System.err.println("Cannot find MyResources");
}
The resourceBundleName 
MyResources
was not found, so the catch block is executed, printing the error message 

          
           
Cannot find MyResources
. The same exception is thrown if the requested resource bundle exists, but the requested key 
key
cannot be located within the resource bundle. The following example checks whether a key 
greeting
exists in a ResourceBundle named 
MyResources
, and prints a default message if the key is missing: 

          
           
try {  
  ResourceBundle myResources = ResourceBundle.getBundle("MyResources");
  String greeting = myResources.getString("greeting");
  System.out.println(greeting);
} catch(MissingResourceException mre) {
  System.out.println("Hello!");
}
If the key 
greeting
exists in the resource bundle, its value will be printed to the console. Otherwise, the catch block will execute, printing 
Hello!
. Finally, we can add our own custom message to the MissingResourceException using its constructor: 

          
           
try {
  // ...
} catch(MissingResourceException mre) {
  throw new MissingResourceException(
    "Error retrieving resource bundle MyResources",
    "MyResources",
    "greeting"
  );
}
