Introduction to org.elasticsearch.search.aggregations.AggregationBuilders

The org.elasticsearch.search.aggregations.AggregationBuilders class is a powerful API designed to provide support for aggregations in the Elasticsearch Query DSL. This API allows developers to easily create, manage, and manipulate aggregation types such as terms, filters, histograms, and spatial coordinates.

Creating Aggregations with AggregationBuilders

The simplest way to create an aggregation using the AggregationBuilders class is to instantiate a new aggregation instance and then attach it to a query builder (or another aggregation). For example, the following code snippet creates a simple histogram aggregation:


          
           
AggregationBuilder aggregation = AggregationBuilders.histogram("salesPerHour")
        .field("sales")
        .interval(1);
QueryBuilder queryBuilder = QueryBuilders.boolQuery();
queryBuilder.addAggregation(aggregation);

The above code example creates an aggregation instance called "salesPerHour" which is then added to the query builder. The histogram aggregation has two parameters: field, which specifies the field to be used in the aggregation, and interval, which specifies the size of each histogram bucket.

It is also possible to create more complex aggregations with the AggregationBuilders class. For example, the following code snippet creates an aggregation that groups documents by a user-defined field, then creates a sub-aggregation which counts the number of documents in each group:


          
           
AggregationBuilder aggregation = AggregationBuilders.terms("countByGroup")
        .field("group")
        .subAggregation(AggregationBuilders.count("count"));
QueryBuilder queryBuilder = QueryBuilders.boolQuery();
queryBuilder.addAggregation(aggregation);

Using AggregationBuilders with Queries

QueryBuilder queryBuilder = QueryBuilders.boolQuery();
