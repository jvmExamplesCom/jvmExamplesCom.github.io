org.springframework.boot.context.embedded.EmbeddedServletContainerCustomizer

The EmbeddedServletContainerCustomizer class is a collection of callbacks used to customize the embedded servlet container created by Spring Boot. It allows for injecting configuration into an embedded servlet container such as Tomcat, Jetty and Undertow before it gets started. The interface provides two methods that can be used to customize the instance, namely configure and customize.

Implementation

In order to use the EmbeddedServletContainerCustomizer, you need to implement the customize method in a custom class. This is a good place to configure additional filters, add servlet mappings and customize the embedded servlet container settings. 


          
           
// Custom class
public class MyServletContainerCustomizer implements EmbeddedServletContainerCustomizer {

    @Override
    public void customize(ConfigurableEmbeddedServletContainer container) {
        // Configure container here
    }
}

Once the custom class has been created, the bean can be registered in the application context.


          
           
@Configuration
public class AppConfig {

    @Bean
    public EmbeddedServletContainerCustomizer servletContainerCustomizer() {
        return new MyServletContainerCustomizer();
    }

}

Now the MyServletContainerCustomizer will be used to customize the embedded servlet container instance.

Example

Here's an example of how to customize the embedded servlet container settings.


          
           
public class MyServletContainerCustomizer implements EmbeddedServletContainerCustomizer {

    @Override
    public void customize(ConfigurableEmbeddedServletContainer container) {
        container.setPort(8080);
        container.addErrorPages(new ErrorPage(HttpStatus.NOT_FOUND, "/error/404.html"));
        container.setSessionTimeout(30, TimeUnit.MINUTES);
    }

}

This example sets the listening port to 8080, adds a custom 404 error page and sets the session timeout to 30 minutes.
