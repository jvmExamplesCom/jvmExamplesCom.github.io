CommonDataKeys in Java with Intellij openapi

The IntelliJ open API provides an interface for accessing the data associated with a project. In particular, the CommonDataKeys class serves to give access to data associated with the project’s files, including keys that denote the current project. This article will discuss the various methods available to access and manipulate the data associated with a project’s files using the Intellij openapi and the CommonDataKeys class.

CommonDataKeys Methods

The CommonDataKeys class provides several methods for accessing the data associated with a project’s files. These methods can be used to retrieve and manipulate the relevant file data in a consistent manner. The following are some of the most commonly used methods from the CommonDataKeys class:
  • getFilePath(): This method allows you to retrieve the absolute path to the file that is currently being opened or edited in the editor.
  • getSelectionLength(): This method returns the number of characters selected by the user in the editor.
  • getSelectionStart(): This method returns the index of the first character of the selection made by the user in the editor.
  • getDataContext(): This method returns the instance of the DataContext object which can be used to get the values of the various data associated with the project's files.
  • getProjectName(): This method returns the name of the project whose files are currently being edited.
  • isVirtual(): This method returns a boolean value indicating whether the current file is a virtual file.
  • getEditor(): This method returns the Editor instance associated with the file currently being edited.
  • getCurrentLine(): This method returns the line number of the cursor in the editor.
  • getDocument(): This method returns the document object associated with the file currently being edited.
