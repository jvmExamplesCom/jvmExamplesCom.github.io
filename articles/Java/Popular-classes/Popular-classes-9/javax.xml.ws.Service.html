Introduction to 

           
            
javax.xml.ws.Service

The 

          
           
javax.xml.ws.Service
class is a base class for JAX-WS service access points. This class provides the client view of a web service. It provides methods for accessing and dispatching operations to a target service endpoint.

This class is the starting point for client programs that need to access a web service. It offers access to several methods for accessing web services, such as creating proxy objects or access the Dispatch interface.

Creating a Service Object

In order to use a web service, a client must create an instance of the Service class. The following code example shows how to create a Service object using a WSDL document (input stream) and a QName object:


          
           
InputStream wsdlStream = new FileInputStream("foo.wsdl");
QName serviceName = new QName("http://example.com/service", "FooService");
Service service = Service.create(wsdlStream, serviceName);

Creating a Proxy Object

Once the Service object is created, a client can use it to instantiate proxy objects. A proxy object represents a web service endpoint. The following code example shows how to create a proxy object from a Service object:


          
           
FooService port = service.getPort(FooService.class);

Accessing the Dispatch Interface

The Service class also provides access to the Dispatch interface. The Dispatch interface allows interaction with web services that don't follow the standard Java API for XML Web Services (JAX-WS). The following code example shows how to create a Dispatch object from a Service object:


          
           
Dispatch<Source> dispatch = service.createDispatch(portName, Source.class, Mode.PAYLOAD);

Conclusion

The javax.xml.ws.Service class provides a client view of a web service. It offers methods for creating proxy objects and accessing the Dispatch interface, allowing more flexibility when working with web services.
