android.view.View.OnKeyListener

The android.view.View.OnKeyListener interface is used by the clients of View to intercept key events before they are passed on to the View or its descendants. All callbacks will be made on the main thread.

Constructors

android.view.View.OnKeyListener has no constructors.

Methods

  • onKey()
    This method is called when a key event is dispatched to a view. This allows listeners to get a chance to respond before the target view.

Example - Implementing OnKeyListener

In this example, we create a custom view class that implements the OnKeyListener interface. 

          
           
public class MyView extends View implements OnKeyListener {
    public MyView(Context context) {
        super(context);
        setFocusable(true);
        setFocusableInTouchMode(true);
        setOnKeyListener(this);
    }

    @Override
    public boolean onKey(View v, int keyCode, KeyEvent event) {
        if (event.getAction() == KeyEvent.ACTION_DOWN) {
            switch (keyCode) {
            case KeyEvent.KEYCODE_DPAD_CENTER:
                Log.v("MyView", "Center button pressed");
                return true;
            case KeyEvent.KEYCODE_DPAD_DOWN:
                Log.v("MyView", "Down button pressed");
                return true;
            case KeyEvent.KEYCODE_DPAD_UP:
                Log.v("MyView", "Up button pressed");
                return true;
            }
        }
        return false;
    }
}

Example - Using OnKeyListener in an Activity

In this example, we show how to use the OnKeyListener interface in an Activity. 

          
           
public class MyActivity extends Activity {

    private MyView mMyView;

    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        mMyView = new MyView(this);

    }
}

Example - Intercepting Key Events with a View

In this example, we show how to intercept key events with a View. 

          
           
public class MyView extends View {
    public MyView(Context context) {
        super(context);
        setFocusable(true);
        setFocusableInTouchMode(true);
    }

    @Override
    public boolean onKeyDown(int keyCode, KeyEvent event) {
        switch(keyCode) {
            case KeyEvent.KEYCODE_DPAD_CENTER:
                Log.v("MyView", "Center button pressed");
                return true;
            case KeyEvent.KEYCODE_DPAD_DOWN:
                Log.v("MyView", "Down button pressed");
                return true;
            case KeyEvent.KEYCODE_DPAD_UP:
                Log.v("MyView", "Up button pressed");
                return true;
        }
        return super.onKeyDown(keyCode, event);
    }
}

Conclusion

android.view.View.OnKeyListener is an interface used by clients of a View to intercept key events before they are passed on to the View or its descendants. It has two methods, onKey(), which is called when a key event is dispatched to a view, and onKeyDown(), which can be overridden to intercept key events with a View.
