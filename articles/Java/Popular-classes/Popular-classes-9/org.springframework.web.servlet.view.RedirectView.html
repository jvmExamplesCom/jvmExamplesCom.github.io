Introduction to org.springframework.web.servlet.view.RedirectView

The org.springframework.web.servlet.view.RedirectView is a Spring MVC class used for sending HTTP redirect responses to the client. It is used for redirecting requests from one's server to another URL. This class provides support for both temporary and permanent redirects.

How it works

RedirectView is used in the same way as other classes in the Spring MVC View hierarchy, such as InternalResourceView, JstlView and so on. When it is added as a bean to the RequestMappingHandlerAdapter, it is used to process requests with an @ResponseBody annotation. When the RedirectView is configured, the request is handled by the RedirectView and the response is sent directly to the client. The RedirectView handles the HTTP redirect response by sending an HTTP status code, the new location, and the appropriate content-type headers.

Example Code


          
           
@Controller
public class HomeController {
	
    @GetMapping("/index")
    public RedirectView handleIndexRequest() {
        return new RedirectView("http://example.com/newLocation");
    }

}

          

          
           
In this example, when the URL http://example.com/index is requested, the RedirectView will create an HTTP redirect response that points the user to http://example.com/newLocation.

Temporary vs. Permanent Redirects

RedirectView supports two types of redirects: temporary (HTTP 302) and permanent (HTTP 301). Temporary redirects are used when the original URL should remain available for use for some time. Conversely, a permanent redirect should be used when the previous URL should no longer be used. 

          
           
@Controller
public class HomeController {
	
    @GetMapping("/index")
    public RedirectView handleIndexRequest() {
        RedirectView redirectView = new RedirectView("http://example.com/newLocation");
        redirectView.setStatusCode(HttpStatus.MOVED_PERMANENTLY);
        return redirectView;
    }

}

          

          
           
In this example, the RedirectView is configured to send a permanent (HTTP 301) redirect response.

Conclusion

The org.springframework.web.servlet.view.RedirectView is a powerful tool for handling requests in Spring MVC. It can be used to send both temporary and permanent redirect responses to the client. By configuring the view in the RequestMappingHandlerAdapter, requests with an @ResponseBody annotation can be properly handled.
