org.springframework.web.context.ServletContextAware

ServletContextAware is an interface implemented by application objects such as beans in order to be notified of the ServletContext that it runs in. This can be used to initialize resources or perform certain operations on startup, especially when used in combination with the WebApplicationContext.

Example Usage

Let's look at an example of how this might be used in a Spring Framework application. Here, we have a bean which implements ServletContextAware. When the bean is instantiated, the setServletContext() method will be called and passed the servlet context from the web application.


          
           
public class ExampleBean implements ServletContextAware {
    private ServletContext servletContext;

    @Override
    public void setServletContext(ServletContext servletContext) {
        this.servletContext = servletContext;
    }

    public void init() {
        // use servlet context to do something
    }
}

Output

Once the bean is initialized, you can use the servlet context to do something. For example, you could use it to get a list of attributes associated with the application, or to get the name of the application:


          
           
// Get application attributes
Map<String, Object> attributes = servletContext.getAttributes();

// Get application name
String applicationName = servletContext.getInitParameter("applicationName");
