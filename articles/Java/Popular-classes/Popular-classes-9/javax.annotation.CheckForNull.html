javax.annotation.CheckForNull

javax.annotation.CheckForNull is an annotation type that can be used to indicate that a parameter, return value or field may be null. This annotation can help document the programmer's intent when using a nullable type.

Usage:

The @CheckForNull annotation can be applied to a parameter, return value or field to indicate that a method might return or receive a null value. For example, 


          
           
public class MyClass {

    @CheckForNull
    public String getValue(){
        //...
    }

}

In this example, the getValue() method may return a null value. Thus, whenever the method is called, the caller should check for a null return value.

Example 1: Check for Null in Parameter

This example shows how to use the @CheckForNull annotation to indicate that a method might accept a null value as a parameter.


          
           
public class MyClass {

    public void doSomething(@CheckForNull String arg){
        if (arg==null) {
            // handle null argument
        }else{
           // process argument
        }
    }

}

This example shows how to use the @CheckForNull annotation to indicate that a method might return a null value.


          
           
public class MyClass {

    @CheckForNull
    public String getValue(){
        if (someCondition) {
           // return a string
        }else{
           // return null
        }

This example shows how to use the @CheckForNull annotation to indicate that a field might contain a null value.


          
           
public class MyClass {
