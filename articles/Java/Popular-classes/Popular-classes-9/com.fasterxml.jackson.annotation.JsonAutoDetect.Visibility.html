Introduction to com.fasterxml.jackson.annotation.JsonAutoDetect.Visibility

com.fasterxml.jackson.annotation.JsonAutoDetect.Visibility is an enum that defines how visible a field or method must be in order to be included in the serialization/deserialization process. This can be used to control access to fields and methods during serialization and deserialization, as well as to control which JavaBean properties are used when mapping JSON.

The Four Available Visibilities

The four available visibilities are: PUBLIC_ONLY, NONE, PRIVATE_AND_PUBLIC, and PROTECTED_AND_PUBLIC. Each visibility has different behavior when it comes to serializing and deserializing.

PUBLIC_ONLY

This is the default visibility, meaning that only public fields and methods will be considered for serialization and deserialization. Any fields or methods with package, protected, or private visibility will be ignored.

public class MyClass {
    private String myPrivateField;
 
    public String getMyPrivateField() {
        return myPrivateField;
    }
}

none
