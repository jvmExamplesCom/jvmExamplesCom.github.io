Introduction to com.fasterxml.jackson.annotation.JsonAutoDetect.Visibility

com.fasterxml.jackson.annotation.JsonAutoDetect.Visibility is an enum that defines how visible a field or method must be in order to be included in the serialization/deserialization process. This can be used to control access to fields and methods during serialization and deserialization, as well as to control which JavaBean properties are used when mapping JSON.

The Four Available Visibilities

The four available visibilities are: PUBLIC_ONLY, NONE, PRIVATE_AND_PUBLIC, and PROTECTED_AND_PUBLIC. Each visibility has different behavior when it comes to serializing and deserializing.

PUBLIC_ONLY

This is the default visibility, meaning that only public fields and methods will be considered for serialization and deserialization. Any fields or methods with package, protected, or private visibility will be ignored.


          
           
@JsonAutoDetect(fieldVisibility = JsonAutoDetect.Visibility.PUBLIC_ONLY)
public class MyClass {
    private String myPrivateField;
 
    public String getMyPrivateField() {
        return myPrivateField;
    }
}

getMyPrivateField()
method will be considered for serialization and deserialization, but the 
myPrivateField
field will not.

NONE

This visibility indicates that nothing should be included in the serialization/deserialization process. It is equivalent to setting all field and method visibilities to 

none
. 

          
           
@JsonAutoDetect(fieldVisibility = Visibility.NONE)

This example will result in neither fields nor methods being included in the serialization/deserialization process.

PRIVATE_AND_PUBLIC

This visibility indicates that both public and private fields and methods should be considered for serialization/deserialization. It is equivalent to setting all field and method visibilities to

@JsonAutoDetect(fieldVisibility = Visibility.PRIVATE_AND_PUBLIC)

Conclusion

com.fasterxml.jackson.annotation.JsonAutoDetect.Visibility provides an easy way to control what fields and methods are included in the serialization/deserialization process.
