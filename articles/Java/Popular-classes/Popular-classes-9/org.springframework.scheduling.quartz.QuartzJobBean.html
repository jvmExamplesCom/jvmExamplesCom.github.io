org.springframework.scheduling.quartz.QuartzJobBean

The 

          
           
org.springframework.scheduling.quartz.QuartzJobBean
is a convenience class that makes it easy to use Quartz with the Spring Framework. It provides an implementation of the Quartz Job interface, as well as convenience methods for accessing the job data map.

Example

In this example, we will create a simple QuartzJobBean which will print out “Hello World!” every 10 seconds.


          
           
@PersistJobDataAfterExecution
@DisallowConcurrentExecution
public class HelloWorldJob extends QuartzJobBean {
    @Override
    protected void executeInternal(JobExecutionContext context) {
        System.out.println("Hello World!");
    }
}

The job is configured in the Spring configuration file with the following bean definition:


          
           
<bean id="helloWorldJob" class="package.HelloWorldJob"/>

Finally, we must add the job to the Quartz scheduler by specifying the job's details in a 


          
           
JobDetailFactoryBean
bean definition: 

          
           
<bean id="jobDetail" class="org.springframework.scheduling.quartz.JobDetailFactoryBean">
    <property name="jobClass" value="package.HelloWorldJob"/>
    <property name="durability" value="true"/>
</bean>
We can then create a 

          
           
CronTriggerFactoryBean
to schedule the job to be executed every 10 seconds: 

          
           
<bean id="cronTrigger" class="org.springframework.scheduling.quartz.CronTriggerFactoryBean">
    <property name="jobDetail" ref="jobDetail"/>
    <property name="cronExpression" value="0/10 * * ? * * *"/>
</bean>

Conclusion

In conclusion, the QuartzJobBean provides a convenient way to integrate Quartz with the Spring Framework. By using the convenience methods provided by the QuartzJobBean, we can easily access the job data map. Additionally, we are able to add JobDetails and CronTriggers to the Quartz Scheduler in our Spring configuration file with ease.
