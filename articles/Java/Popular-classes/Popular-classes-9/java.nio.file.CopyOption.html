java.nio.file.CopyOption

Java.nio.file.CopyOption is an interface used to represent options that can be specified when copying files and directories with the Files.copy(Path, Path, CopyOption...) method. The options are open for interpretation by the copy implementation and may be ignored by some implementations.

Standard CopyOptions

There are three standard CopyOptions included in the Java.nio.file.CopyOption interface, which are REPLACE_EXISTING, COPY_ATTRIBUTES and ATOMIC_MOVE. The REPLACE_EXISTING option overwrites existing files or directories, while the COPY_ATTRIBUTES option copies the file attributes associated with the source file to the destination file. Finally, the ATOMIC_MOVE option moves a file as an atomic file system operation.

Example Usage

The following code example demonstrates how to use the Files.copy(Path, Path, CopyOption...) method to copy a file from one directory to another, while also setting the REPLACE_EXISTING option to overwrite any existing files.


          
           
Path sourceFile = Paths.get("src/file.txt");
Path destinationFile = Paths.get("dest/file.txt");
Files.copy(sourceFile, destinationFile, StandardCopyOption.REPLACE_EXISTING);
