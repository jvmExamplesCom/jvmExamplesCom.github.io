com.squareup.okhttp.MediaType

The MediaType class from the com.squareup.okhttp package provides a way to represent media types such as images, audio, video, and so on and provides convenience methods for easily creating and matching them. It is an important part of the OkHttp library which is used for sending and receiving HTTP requests in Java.

Creating MediaTypes

A MediaType can be created using the static 
parse(String)
method, passing in a string representation of the desired media type. This string must follow the structure of / ; charset= as defined in RFC 6838: 

             
              
MediaType textHtmlUtf8 = MediaType.parse("text/html; charset=utf-8");

Alternatively, if the charset is not needed, the media type can be created using 


             
              
create(<type>, <subtype>)
method. For example : 

             
              
MediaType applicationJson = MediaType.create("application", "json");

Matching MediaTypes

The match(String) method allows a user to test whether two MediaTypes are the same, based on their type and subtype. For example:


             
              
boolean isTextHtml = textHtmlUtf8.match("text/html");

The match method also supports wildcards such as * and + for matching any part of the MediaType. For example:


             
              
boolean isAnythingXml = textHtmlUtf8.match("*/*+xml");
