Introduction to org.apache.shiro.authc.IncorrectCredentialsException

org.apache.shiro.authc.IncorrectCredentialsException is an exception that is thrown by the Apache Shiro authentication system when an incorrect password or other credentials are provided. This exception is part of the Apache Shiro authentication framework, which provides an integrated security solution for Java-based applications.

Example: Throwing an IncorrectCredentialsException

The following code example shows how to throw an IncorrectCredentialsException in an application:


          
           
try {
    String username = getUserNameFromRequest();
    String enteredPassword = getPasswordFromRequest();
    String storedPassword = retrieveStoredPasswordForUser(username);
    if (enteredPassword.equals(storedPassword)) {
        // correct credentials!
    } else {
        throw new org.apache.shiro.authc.IncorrectCredentialsException("Incorrect password!");
    }
} catch (org.apache.shiro.authc.IncorrectCredentialsException e) {
    log.error("Incorrect credentials", e);
}

Handling a IncorrectCredentialsException

The following code example shows how to handle an IncorrectCredentialsException:


          
           
try {
    // authentication logic here
} catch (org.apache.shiro.authc.IncorrectCredentialsException e) {
    // handle the exception
    log.error("Incorrect credentials", e);
    redirectToLoginPage();
}

In the example above, we are catching the IncorrectCredentialsException and then logging the exception and redirecting the user to the login page.
