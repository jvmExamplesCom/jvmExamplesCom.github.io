Exploring org.eclipse.aether.RepositorySystemSession

A RepositorySystemSession is an instance of a session-scoped repository system, a component that is able to manage remote and/or local software repositories as defined by the org.eclipse.aether.repository.Repository interface.

In this article, we'll take a look at the various aspects of the RepositorySystemSession including its methods and properties and explore code examples for each of them.

Creating a RepositorySystemSession

The first step in using a RepositorySystemSession is to create one. It is typically done by providing a RepositorySystem object along with a RepositorySystemSessionConfiguration to the method createSession().


          
           
RepositorySystem system = Aether.newRepositorySystem();
RepositorySystemSessionConfigurator configurator = new DefaultRepositorySystemSessionConfigurator();
RepositorySystemSession session = system.createSession(configurator);

getLocalRepositoryManager()

The getLocalRepositoryManager() method returns a LocalRepositoryManager instance associated with the RepositorySystemSession. This is used for managing local repositories which are used for caching artifacts and metadata. 


          
           
LocalRepositoryManager localRepoManager = session.getLocalRepositoryManager();

getRemoteRepositoryManager()

The getRemoteRepositoryManager() method returns a RemoteRepositorymanager instance associated with the RepositorySystemSession. This is used for managing remote repositories which are used for downloading artifacts and metadata. 


          
           
RemoteRepositoryManager repoManager = session.getRemoteRepositoryManager();

getTransferListener()

The getTransferListener() method returns a TransferListener associated with the RepositorySystemSession. A TransferListener is a component that is used to monitor the progress of a repository transfer. 


          
           
TransferListener listener = session.getTransferListener();

getNotFoundCache()

The getNotFoundCache() method returns a NotFoundCache associated with the RepositorySystemSession. A NotFoundCache is used for tracking missing artifacts and metadata. 


          
           
NotFoundCache notFoundCache = session.getNotFoundCache();

getChecksumPolicy()

The getChecksumPolicy() method returns the ChecksumPolicy associated with the RepositorySystemSession. The ChecksumPolicy is used to specify how to handle checksums when downloading artifacts.


          
           
ChecksumPolicy checksumPolicy = session.getChecksumPolicy();

The getArtifactTypeRegistry() method returns an ArtifactTypeRegistry associated with the RepositorySystemSession. The ArtifactTypeRegistry is used to define how artifacts should be handled.


          
           
ArtifactTypeRegistry artifactTypeRegistry = session.getArtifactTypeRegistry();

getData()

The getData() method returns a Map of additional properties associated with the RepositorySystemSession.
