android.support.annotation.MainThread

The android.support.annotation.MainThread annotation indicates that the annotated method must be called on the UI thread. It should be used only in methods that must be executed on the main thread, as specified by the Android documentation.

Code Examples and Output

Here is an example of a MainThread-annotated method in a class:


          
           
@MainThread
public void showAlertDialog() {
    //Show alert dialog
}

In this example, the showAlertDialog method must be called on the main UI thread.

Another example of a MainThread-annotated method looks like this:


          
           
@MainThread
public void setText(String text) {
    //Set Text
}

The setText method must also be called on the main UI thread.

In order to ensure that these methods are called on the main UI thread, we can use the handler class from the Android framework:


          
           
Handler mainHandler = new Handler(Looper.getMainLooper());
Runnable myRunnable = new Runnable() {
   @Override
   public void run() {
      //call showAlertDialog() or setText()
   }
};
mainHandler.post(myRunnable);

In this example, we use the Handler class to post a Runnable to the main UI thread, thus ensuring that the showAlertDialog and setText methods are called on the main UI thread.

Conclusion

The android.support.annotation.MainThread annotation is used to indicate that a method must be called on the UI thread. We can use the Handler class from the Android framework to ensure that such methods are executed on the main UI thread.
