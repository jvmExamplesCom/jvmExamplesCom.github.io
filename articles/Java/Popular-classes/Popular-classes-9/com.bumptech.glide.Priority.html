com.bumptech.glide.Priority

The com.bumptech.glide.Priority class is a priority for Glide requests. It is used to prioritize individual loads and is used to determine the order in which images are processed among multiple requests. The higher the priority, the earlier it will be processed.

The possible values are: 
  • LOW
    - A low priority. This is usually used for background requests that are lower priority than usual.
  • NORMAL
    - A normal priority. This is usually used for requests that have no particular preference.
  • HIGH
    - A high priority. This is usually used for requests that should be processed as soon as possible.

Using the com.bumptech.glide.Priority class is straightforward. When making a request, simply specify the priority desired as one of the parameters:


          
           
Glide.with(context)
  .load(url)
  .priority(Priority.HIGH)
  .into(imageview);

Priorities.valueOf

int customPriority = 10;

When multiple requests are made, the priorities will be evaluated to determine the order in which they are processed. For example, if two requests are made with different priorities, the one with the higher priority will be processed first. The same applies when a single request has multiple parts that need to be processed.

It is important to keep in mind that while the Priority class can be used to prioritize requests, it does not guarantee that all requests with the same priority will be processed in the same order. The com.bumptech.glide.Priority class provides a convenient way to control the order in which requests are processed and can be used to ensure that higher priority requests are processed first.
