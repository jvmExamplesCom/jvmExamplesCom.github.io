javax.swing.plaf.ComponentUI

The 

          
           
javax.swing.plaf.ComponentUI
class is a UI delegate for providing the look and feel for a particular component. It defines the methods that enable the look and feel implementation to set and query the state of the object as well as define its visual representation. This is an abstract class that should be subclassed. Subclasses must provide implementations for all of the abstract methods.

Example Code Usage


          
           
import javax.swing.plaf.basic.BasicButtonUI;
public class MyButtonUI extends BasicButtonUI {
   ....
}

The above code shows how to create a custom UI delegate for a button. All we need to do is make a subclass of BasicButtonUI and override the necessary methods. Then we can apply the new UI delegate to the button.


          
           
MyButton.setUI(new MyButtonUI());

Methods

The ComponentUI class is abstract and thus does not have any concrete methods. However, it does define three abstract methods that all subclasses must implement. These are:


          
           
update(Graphics g, JComponent c) 
installUI(JComponent c) 
uninstallUI(JComponent c)

The update method is used to paint the component based on its current state. The install and uninstall methods are used to set up and tear down the look and feel of the component respectively.
