androidx.annotation.WorkerThread

androidx.annotation.WorkerThread is an annotation in the Android Support Library that indicates that a method should only be called from a worker thread. This annotation is a way to ensure that long-running tasks are performed off the main thread, which can often lead to improved performance. Worker threads are also known as background threads, and are used for tasks such as communication or large calculation requests.

Example Usage

Let's look at a simple example of using this annotation. We will define a method that takes a large data set and runs some calculations on it. We want to ensure that this computation is not done on the main thread, so we add the @WorkerThread annotation.


          
           
// Compute values based on input data
@WorkerThread
public static void computeValues(List<Double> data) {
    // do computations
}

We can now call this method from any thread, however, when calling it from the main thread a warning will be logged in the console, since this method should only be called from a worker thread.

Reference

The annotation's reference definition can be found here. Note the @IntDef parameter which has the constants, MAIN_THREAD and WORKER_THREAD. This annotation is only called within the Android API and thus is not visible to user code. Only API methods will call this annotation, and the resulting warnings will be logged in the console.
