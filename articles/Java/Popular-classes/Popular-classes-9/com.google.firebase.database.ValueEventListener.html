com.google.firebase.database.ValueEventListener

Introduction

Firebase Realtime Database is a cloud-hosted NoSQL database that allows developers to store and sync data between the cloud and their applications in real time. It provides an easy way to read and listen to data stored in Firebase Realtime Database. The com.google.firebase.database.ValueEventListener interface is an event listener for reading a static snapshot of the data at a given path. The ValueEventListener interface gives you a single callback method, onDataChange() where you can access the data snapshot for the data that matches your query. This method is triggered once when the listener is attached and again anytime the data changes as defined by your query.

Adding a Value Event Listener

To add a ValueEventListener to your Firebase Realtime Database, you need to use the .addValueEventListener() method within the reference. This adds your listener and returns a Registration object you can use to remove the listener using the .remove() method. Here is an example of adding a ValueEventListener: 

          
           
DatabaseReference ref = FirebaseDatabase.getInstance().getReference().child("my_data");

// Attach a listener to read the data at our posts reference
ref.addValueEventListener(new ValueEventListener() {
    @Override
    public void onDataChange(DataSnapshot dataSnapshot) {
        String data = dataSnapshot.getValue(String.class);
        System.out.println(data);
    }

    @Override
    public void onCancelled(DatabaseError databaseError) {
        System.out.println("The read failed: " + databaseError.getCode());
    }
});

onDataChange()
