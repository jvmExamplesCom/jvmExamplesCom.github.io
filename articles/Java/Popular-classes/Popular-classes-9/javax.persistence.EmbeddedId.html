javax.persistence.EmbeddedId

The javax.persistence.EmbeddedId is a special type of persistent field used to map composite values in a JPA entity. It is used when an entity has a composite primary key and is typically used when the primary key is composed of two or more fields. The EmbeddedId annotation allows a composite primary key class to be treated as a single logical key value.

Example

The following example shows an entity with a composite primary key consisting of two fields:


          
           
@Entity
public class BankAccount {

    @EmbeddedId
    private BankAccountPK primaryKey;

    // Attributes and getters/setters, etc...

}

@Embeddable
public class BankAccountPK implements Serializable {

    private String accountNumber;
    private int bankCode;

    // Getters/Setters

}
In this example, the 
BankAccount
entity has a composite primary key consisting of the 
accountNumber
and the 
bankCode
. The 
BankAccountPK
class is annotated with the 
@Embeddable
annotation and is used to represent the composite primary key.

Using the EmbeddedId Annotation

To use the 
EmbeddedId
annotation, annotate the persistent field with the 
@EmbeddedId
annotation: 

          
           
@Entity
public class BankAccount {

    @EmbeddedId
    private BankAccountPK primaryKey;

}

The embedded primary key class must also be annotated with the 

@Embeddable

    private String accountNumber;
    private int bankCode;

    // Getters/Setters

}

Note that the embedded primary key class must implement the 


          
           
java.io.Serializable

Mapping the EmbeddedId

The 
EmbeddedId
annotation can be used to map the composite primary key by specifying the 
name
attribute. The 
name
attribute specifies the name of the mapped column in the database. For example:

    @EmbeddedId(name = "bank_acct")

}
In this example, the 
BankAccountPK
