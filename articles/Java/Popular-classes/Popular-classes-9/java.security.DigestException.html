What is java.security.DigestException?

java.security.DigestException is a subclass of the general SecurityException which is thrown when an error occurs while processing a message digest or a keyed-hash message authentication code (HMAC). This exception can also be thrown when a particular cryptographic algorithm is requested but is not available in the environment.

Examples of java.security.DigestException


          
           
try {
    //some code that may throw java.security.DigestException
} catch (DigestException e) {
    System.err.println("Error occurred while computing message digest: " + e);
}
Here's another example of java.security.DigestException being thrown when a particular cryptographic algorithm is not available: 

          
           
try {
    MessageDigest md = MessageDigest.getInstance("SHA-1");
} catch (NoSuchAlgorithmException e) {
    System.err.println("The requested algorithm isn't available: " + e);
    throw new DigestException("The requested algorithm isn't available: " + e);
}
We can also assign a detailed message to this exception, such as when the length of the provided data is wrong: 

          
           
try {

    md.update("data".getBytes("utf-8"));
} catch (NoSuchAlgorithmException e) {
    System.err.println("The requested algorithm isn't available: " + e);
    throw new DigestException("The requested algorithm isn't available: " + e);

    System.err.println("Wrong length of data provided: " + e);
    throw new DigestException("Wrong length of data provided: " + e);
