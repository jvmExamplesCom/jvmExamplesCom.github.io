android.support.design.widget.BottomSheetBehavior

BottomSheetBehavior allows the bottom sheets to be created on Android apps. It is part of the Design Support library and is used to create consistent and interactive bottom sheets on Android.

Creating a BottomSheet

To use BottomSheetBehavior, we first need to create the layout for our bottom sheet. This can be done by creating a layout file with the following code:

<android.support.design.widget.CoordinatorLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/bottom_sheet"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:behavior_peekHeight="80dp">

    <!--Content goes here-->

</android.support.design.widget.CoordinatorLayout>
This layout should be placed in the res/layout folder. The essential attribute for creating a BottomSheet is the 

          
           
android:id="@+id/bottom_sheet"
. This id should be unique and also referenced when creating the BottomSheetBehavior.

Creating the BottomSheetBehavior

CoordinatorLayout coordinatorLayout = findViewById(R.id.bottom_sheet);
BottomSheetBehavior bottomSheetBehavior = BottomSheetBehavior.from(coordinatorLayout);

Setting the Peek Height

The peek height is one of the most important settings for the bottom sheet. This setting determines how much of the bottom sheet is shown when it is collapsed. To set the peek height, we can use the 

          
           
setPeekHeight()
method on the BottomSheetBehavior object. 

          
           
bottomSheetBehavior.setPeekHeight(100);
The parameter for this method should be an integer value or a dimension resource (e.g. 

          
           
R.dimen.peek_height
).

bottomSheetBehavior.show(BottomSheetBehavior.STATE_ANIMATED);
To hide the bottom sheet without animation, the following code can be used:
