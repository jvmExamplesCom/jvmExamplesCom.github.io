javax.websocket.CloseReason

javax.websocket.CloseReason is a Java API that provides a way to communicate the reason for closure of a WebSocket connection between two endpoints. This is done by sending a Close message from one side to the other, containing an appropriate CloseReason object.

To create a CloseReason object, its constructor can be used, as seen in the example below:


          
           
CloseReason closeReason = new CloseReason(CloseReason.CloseCodes.NORMAL_CLOSURE,
                                                    "Connection closed due to normal operation");

The constructor takes two parameters: a CloseCode (an enum of type CloseCodes) and a String containing the reason for closing the connection. The CloseCode is a standard HTTP status code, and the string should explain what happened in more detail.

Once the CloseReason has been created, it can then be sent in a Close message from the endpoint initiating the closure. This is done using the 


          
           
Session.close()
method, like so: 

          
           
session.close(closeReason);

On the other side of the connection, the receiving endpoint will receive a Close message containing the CloseReason created by the initiator. This CloseReason can then be accessed using the following method:


          
           
CloseReason receivedCloseReason = session.getCloseReason();

This will give the receiving endpoint the same CloseReason object that was sent by the initiator. It can then be used to determine the cause of closure, and take any appropriate action. For example, if the CloseCode is NORMAL_CLOSURE, no further action needs to be taken other than logging the closure. If the reason is something else, such as INTERNAL_ERROR or CONTAINER_SHUTDOWN, then more detailed logging or clean up may be required.
