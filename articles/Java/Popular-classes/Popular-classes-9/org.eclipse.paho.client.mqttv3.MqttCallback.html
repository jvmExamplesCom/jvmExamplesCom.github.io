org.eclipse.paho.client.mqttv3.MqttCallback

The org.eclipse.paho.client.mqttv3.MqttCallback is an interface used to provide notifications from the MQTT client to the application. It enables applications to receive a message from a remote MQTT server and process it. This interface must be implemented so that the client application can receive messages from the broker.

Methods of org.eclipse.paho.client.mqttv3.MqttCallback

  • connectionLost(Throwable cause) – This method is called when the connection between the client and the broker is lost. The connection might be lost due to the network being disconnected or other technical issues.

  • messageArrived(String topic, MqttMessage message) – This method is called when a message arrives from the broker. The client application should process the message with this method. The parameters passed to this method are the topic the message is received on, and the message payload.

  • deliveryComplete(IMqttDeliveryToken token) – This method is called when a message has been successfully delivered to the broker. The parameter passed is the delivery token associated with the message.

Example Codes

The following code shows a basic example of how to use the org.eclipse.paho.client.mqttv3.MqttCallback interface.


          
           
MqttCallback mc = new MqttCallback() {

@Override
public void connectionLost(Throwable cause) {
    System.out.println("Connection lost " + cause);
}

@Override
public void messageArrived(String topic, MqttMessage message) throws Exception {
    System.out.println("Received Message: "+ new String(message.getPayload()));
}

@Override
public void deliveryComplete(IMqttDeliveryToken token) {
    System.out.println("Delivery complete");
}};

In the above code example, we have implemented the MqttCallback interface and have defined methods for connectionLost(), messageArrived(), and deliveryComplete(). All methods must be overridden in order for the callback to be successful.

The following code snippet shows an example of how to register the callback for a given MQTT client.


          
           
client.setCallback(mc);

Once the callback is registered, the client will call the appropriate methods when a particular event occurs. For example, if the connection with the broker is lost, the connectionLost() method will be called and the message will be printed in the console.
