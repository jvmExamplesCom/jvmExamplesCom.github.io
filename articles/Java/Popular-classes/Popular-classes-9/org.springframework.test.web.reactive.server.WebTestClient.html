What is WebTestClient?

The WebTestClient is an API that allows the testing of the web layer of Spring applications. It provides fluent and concise APIs to create, perform and assert operations on web applications or web services.

WebTestClient Basics

Using the org.springframework.test.web.reactive.server.WebTestClient class, developers can create a test environment based on an existing application context and then perform requests directly against it. The WebTestClient object will build an engine for a given request and response context and will then execute the request within the created environment.

The request and response context varies for different scenarios, but typically includes a set of request headers, parameters and a body. The response context may also include a status code and an optional set of response headers and body.

WebTestClient objects also come with additional features and functionalities such as support for HTTP methods, cookie handling, HTTP response validation, content type negotiation, and basic authentication.

@RunWith(SpringRunner.class)
@SpringBootTest
public class WebTestClientExample {

    @Autowired
    private WebTestClient webClient;

    @Test
    public void getToDoList() {
        this.webClient.get().uri("/todo")
            .accept(MediaType.APPLICATION_JSON)
            .exchange()
            .expectStatus().isOk()
            .expectBodyList(ToDo.class);
    }
}
