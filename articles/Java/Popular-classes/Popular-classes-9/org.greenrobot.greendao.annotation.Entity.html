Introduction to org.greenrobot.greendao.annotation.Entity

The org.greenrobot.greendao.annotation.Entity annotation is used to mark a Java class as an entity that Green DAO can understand and map to the database. This annotation is applied on an entity class, and must be followed by at least one property or field definition.

Before applying the Entity annotation, the entity class must be public and have a constructor with no-arguments.

Properties of the Entity Annotation

The Entity annotation has the following properties:

  • schema
    , which allows you to define a custom schema name for the associated table;
  • nameInDb
    , which is used to specify the name of the associated table for the entity;
  • active
    , which is used to enable/disable the entity;
  • createInDb
    , which is used to ask GreenDAO to create/update the table in the database.

Example of org.greenrobot.greendao.annotation.Entity Annotation

In the following example, we create a simple entity class called “User” with two fields – firstName and lastName. The Entity annotation is applied at the class level, and the fields are annotated using the Property annotations.


          
           
@Entity
public class User {

  @Id
  private Long id;

  @Property(nameInDb = "FIRST_NAME")
  private String firstName;

  @Property(nameInDb = "LAST_NAME")
  private String lastName;

  // constructors, getters, and setters
}

The above example will result in the creation of a table named “USER”, with two columns – “FIRST_NAME” and “LAST_NAME”.

Conclusion

The org.greenrobot.greendao.annotation.Entity annotation is used to identify a class as an entity that GreenDAO can work with. This annotation also allows us to specify certain properties such as the schema name, table name, and so on. It is essential to remember that a non-argument constructor must be present for the entity class for GreenDAO to recognize it.
