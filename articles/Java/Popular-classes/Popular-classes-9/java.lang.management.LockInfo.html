Java’s LockInfo class is part of the java.lang.management package, and provides an object representing a lock in the virtual machine. It is available since Java 5.0.

A LockInfo object contains a Class object which describes the lock, as well as its identity hashcode. This information can be used to identify the object, and help diagnose potential locking problems.

LockInfo objects can be used to help debug locking problems in the virtual machine. For example, you can use them to determine which threads are waiting on which locks, or what objects are locked by which threads.
