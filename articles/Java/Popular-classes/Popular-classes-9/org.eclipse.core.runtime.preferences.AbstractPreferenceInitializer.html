AbstractPreferenceInitializer

AbstractPreferenceInitializer is a class in the org.eclipse.core.runtime.preferences package which allows developers to set default values for their application's preferences. This class is written to be used with the Eclipse platform but can theoretically be used anywhere.

Code Example: Setting Default Preference Values

The following is a basic example of using AbstractPreferenceInitializer to set some default preference values.


          
           
public class MyPreferenceInitializer extends AbstractPreferenceInitializer {

public void initializeDefaultPreferences() {
  IPreferenceStore store = Activator.getDefault().getPreferenceStore();
  store.setDefault(MyPreferencePage.PREF_STRING, "My String");
  store.setDefault(MyPreferencePage.PREF_BOOLEAN, true);
  store.setDefault(MyPreferencePage.PREF_INT, 2000);
 }
}

The code example initializes the MyPreferencePage constants for three different types of preference values. It sets the initial value for a string to "My String", the boolean to true and the integer to 2000.

Adding the Initializer Class to Eclipse

Once you have written your initializer class, you need to add it to Eclipse so that it can be used within your plug-in. This is done by editing the plug-in's manifest file (plugin.xml) and adding an extension of type "org.eclipse.core.runtime.preferences". The following code example shows how this could be done.


          
           
<extension
     point="org.eclipse.core.runtime.preferences">
  <initializer
        class="my.package.MyPreferenceInitializer"/>
</extension>

Now when the plug-in is started, the initializeDefaultPreferences() method of the MyPreferenceInitializer class will be called and the default preference values will be set.

Conclusion

AbstractPreferenceInitializer is an important class to know when working with preferences in Eclipse. It allows developers to easily set default preference values for their applications and makes the process of managing preferences much easier.
