Introduction to org.eclipse.paho.client.mqttv3.persist.MemoryPersistence

The org.eclipse.paho.client.mqttv3.persist.MemoryPersistence class is an implementation of the MQTT Client's persistence interface. It supports storing client data in memory, providing basic data persistence and allowing clients to reconnect automatically when a connection is broken. This class can be used with any MQTT client to provide basic persistence for messages, subscriptions and other data.

Why Use MemoryPersistence?

The MemoryPersistence class is useful for MQTT clients that need to maintain data for a short period of time and don’t require fault-tolerant data stores. This could include applications that send simple messages, such as chat rooms, or applications that collect small amounts of data from sensors that is reported regularly. MemoryPersistence can be used in any MQTT client, even if it is not connected to a server or is only connected intermittently.

The MemoryPersistence API

public void open(String clientId, Properties props)

public void close()
This method closes the MemoryPersistence store and clears all stored data. 

          
           
public byte[] get(String key)
This method retrieves a value from the MemoryPersistence store for the specified key.

This method stores a value in the MemoryPersistence store for the specified key. 

          
           
public void remove(String key)
This method removes a value from the MemoryPersistence store for the specified key.

Example Usage

Here is an example of using MemoryPersistence to store a message: 

          
           
String clientId = "example-client";
Properties props = new Properties();
MemoryPersistence persistence = new MemoryPersistence();
