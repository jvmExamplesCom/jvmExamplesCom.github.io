org.openqa.selenium.support.ui.ExpectedCondition

The org.openqa.selenium.support.ui.ExpectedCondition class provides methods for handling expectations in a Selenium test. This is particularly useful when waiting for an element to appear or become available on the page. The ExpectedCondition class contains several predefined conditions that can be used when waiting for an element to appear, such as visibilityOfElementLocated() and textToBePresentInElement().

Implementing Expected Conditions

To implement an ExpectedCondition, start by extending the org.openqa.selenium.support.ui.ExpectedCondition class:


          
           
public abstract class MyExpectedCondition extends ExpectedCondition {
    // public abstract Method(WebDriver driver);
}

MyExpectedCondition myCond = new MyExpectedCondition();

wait.until(myCond);
