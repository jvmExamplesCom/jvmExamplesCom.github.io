javax.persistence.Basic

Java Persistence API (JPA) is a framework for persisting data from Java objects to a relational database. The javax.persistence.Basic class is an important part of the JPA specification and facilitates basic operations like accessing and persisting data from the database.

Syntax

The syntax of the javax.persistence.Basic class is as follows: 

          
           
@Basic(
    fetch = FetchType.LAZY,
    optional = false
)
private String name;

Parameters

The javax.persistence.Basic class takes in two parameters:
  • fetch: This parameter defines whether the associated data should be fetched eagerly or lazily. The default value is lazy.
  • optional: This parameter defines whether the associated data is mandatory or not. The default value is false.

Example

Let's look at an example of how to use the javax.persistence.Basic class. In this example, we will create an entity class called User with a name attribute. We will also set the fetch parameter to LAZY and the optional parameter to false. 

          
           
@Entity
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Basic(
        fetch = FetchType.LAZY,
        optional = false
    )
    private String name;

    // Getters and setters

}
In this example, the name attribute has been annotated with the javax.persistence.Basic annotation. The fetch parameter has been set to LAZY and the optional parameter has been set to false. This means that the associated data for the name attribute will be fetched lazily and it is mandatory for the entity.
