org.apache.hadoop.conf.Configurable

The 

          
           
org.apache.hadoop.conf.Configurable
interface is used to define objects that can be configured using a 
Configuration
object. This enables different implementations of the same framework to configure themselves based on their own needs. Configurations can be read from an xml file, from the command line, from system properties, and from a Configuration object. This interface allows for robust configuration of applications by providing a set of common configuration keys.

Implementation

The core elements of the 
Configurable
interface are: 
  • 
            
             
void setConf(Configuration conf)
    - Sets the Configuration object to be used by the Configurable object.
  • 
            
             
Configuration getConf()
    - Gets the Configuration object used by the Configurable object.
To implement the 
Configurable
interface in a class, it must implement the two methods above. The following example shows how a class might be implemented using the 
Configurable
interface. 

          
           
public class MyClass implements Configurable {
    private Configuration conf;

    @Override
    public void setConf(Configuration conf) {
        this.conf = conf;
    }

    @Override
    public Configuration getConf() {
        return conf;
    }
}
When an instance of 
MyClass
is created, the 
setConf
method must be called with a valid 
Configuration
object in order for the class to be usable. The configuration object can then be used to read and write configuration data through the 
getConf()
method.

MyClass
class might be used to read a value from a configuration object. 

          
           
Configuration conf = new Configuration();

myClass.setConf(conf);

Conclusion

The 

          
           
org.apache.hadoop.conf.Configurable
interface provides an easy way to configure objects based on their configuration needs. It allows for application-level configuration without having to hard-code values into the source code.
