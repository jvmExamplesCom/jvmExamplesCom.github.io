What is io.vertx.ext.unit.TestContext?

io.vertx.ext.unit.TestContext is a class used for executing tests in Vert.x which allows to manage the asynchronous execution of a unit test. It provides methods for running assertions, creating timers and spawning tasks. TestContext also provides methods to control synchronization of test execution.

Creating a TestContext


          
           
TestContext context = new TestContext();

Assertions

The assert methods of TestContext allow for verifying a condition is true or false.


          
           
context.assertTrue(result);
context.assertFalse(result);
context.assertNotNull(obj);
context.assertNull(obj);
context.assertEquals(value1, value2);

Timers

The timer methods of TestContext can be used to delay the execution of a statement until a certain time has passed.


          
           
context.setTimer(delay, handler);
context.setPeriodic(period, handler);
context.cancelTimer(timerId);

Tasks

The task methods of TestContext allow creating tasks which execute concurrently.


          
           
context.executeBlocking(handler, resultHandler);
context.executeAsync(handler);

Synchronization

The sync methods of TestContext allow to control the amount of concurrent threads a test can spawn.


          
           
context.awaitCompletion();
context.completeNow();
context.asyncAssertSuccess();
context.asyncAssertFailure();
