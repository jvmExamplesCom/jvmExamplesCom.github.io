SettableFuture

SettableFuture is part of the Google Guava library, and is a specialized form of a Future. It allows one to manually set its value, either a successful result or an exception.

Example Usage

A simple usage might be to set a result of some computation:


          
           
SettableFuture<String> future = SettableFuture.create();
future.set(doSomeExpensiveOperation());
We can also set an exception if we encounter one: 

          
           
SettableFuture<String> future = SettableFuture.create();
try {
  future.set(doSomeExpensiveOperation());
} catch (Exception e) {
  future.setException(e);
}

The SettableFuture works in synchronization with the main thread, making sure that both threads are aware of the changes. In this way it's possible to have multiple threads running concurrently but still keeping them synchronized.

Another feature provided by the SettableFuture is that it allows us to check for completion. We can use the isDone() method to check if the Futures have been completed. This can be useful in coordination between threads.


          
           
SettableFuture<String> future = SettableFuture.create();
if (future.isDone()) {
  // do something after work done
}

Advantages of SettableFuture

The use of SettableFuture has certain advantages compared to a regular Future. One of the most important is that it allows us to explicitly set values, which helps ensure that our code remains deterministic. This can be important in situations where you don't want to rely on a third-party library or platform for setting values.

It also provides an easy way of coordinating the execution of multiple threads. Since the status of the Futures can be checked, it's possible to know when all of them are completed. This makes it easy to use the Future framework to implement synchronization points between threads.

Finally, having explicit control over the state of the Futures makes it easier to reason about and debug code. If there is an issue occurring, it can be easier to identify what's causing it, as we can look at the state of the Futures and determine which one is causing the issue.

Conclusion

SettableFuture is a useful tool from the Google Guava library for managing asynchronous tasks. It allows us to explicitly set values, allowing for easier debugging and ensuring deterministic behavior. It also provides an easy way of coordinating the execution of multiple threads. For these reasons, it can be a useful tool for implementing concurrent applications.
