BenchmarkMode in Java

The 

          
           
org.openjdk.jmh.annotations.BenchmarkMode
annotation is one of the core annotations used in the Java Microbenchmark Harness (JMH), which allow developers to compare, measure and profile different parts of their Java code. This annotation describes the type of benchmark measurements that should be performed.

Syntax

The syntax of this annotation is as follows: 

          
           
@BenchmarkMode(mode)
Where 

          
           
mode
specifies the attribute that determines the type of benchmark measurement that should be performed.

Supported Modes

The BenchmarkMode annotation supports a range of modes that can be used in JMH. These are: 

          
           
Throughput
AverageTime
SampleTime
SingleShotTime
All

Throughput Mode

The 

          
           
Throughput
mode specifies that JMH should measure the operations per unit time. This mode can be used with all benchmark tests, but it is most applicable to those tests that involve running some process on a loop. For example, the following code shows how to use the 

          
           
Throughput
mode when measuring the performance of an array sorting algorithm: 

          
           
@BenchmarkMode(Mode.Throughput)
public void sortArray(MyState state) {
   state.myArray.sort();
}

AverageTime Mode

The 

          
           
AverageTime
mode specifies that JMH should measure the average execution time of the benchmarked code. This mode is useful when trying to determine the performance of individual operations or tasks. For example, the following code shows how to use the 

          
           
AverageTime
mode when measuring the performance of a string concatenation operation: 

          
           
@BenchmarkMode(Mode.AverageTime)
public void myStringConcat(MyState state) {
   state.myString = state.myString +"test";
}

SampleTime Mode

The 

          
           
SampleTime
mode is similar to the 

          
           
AverageTime
mode, but instead of measuring the average execution time of the benchmarked code, it measures the distribution of the benchmarked execution times. This mode is useful for determining whether there are any outliers in the performance of the code that could impact the overall results. For example, the following code shows how to use the 

          
           
SampleTime
mode when measuring the performance of an object initialization operation: 

          
           
@BenchmarkMode(Mode.SampleTime)
public void myObjectInit(MyState state) {
   state.myObject = new Object();
}

SingleShotTime Mode

The 

          
           
SingleShotTime
mode specifies that JMH should measure the total execution time of the benchmarked code within a single iteration. This mode is useful when measuring the performance of operations that take a relatively short amount of time to complete. For example, the following code shows how to use the 

          
           
SingleShotTime
mode when measuring the performance of a method invocation operation: 

          
           
@BenchmarkMode(Mode.SingleShotTime)
public void myMethodInvoke(MyState state) {
   state.myObject.myMethod();
}

All Mode

The 

          
           
All
mode specifies that all of the available measurement modes will be used to measure the performance of the benchmarked code. This mode is useful for situations where you want to get a comprehensive overview of the performance of the code. For example, the following code shows how to use the 

          
           
All
mode when measuring the performance of a file writing operation: 

          
           
@BenchmarkMode(Mode.All)
public void myFileWrite(MyState state) {
   state.myFile.write(state.myBytes);
}

Conclusion

The 

          
           
org.openjdk.jmh.annotations.BenchmarkMode
