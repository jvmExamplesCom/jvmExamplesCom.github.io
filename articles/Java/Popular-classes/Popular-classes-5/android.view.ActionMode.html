Introduction to Android ActionMode

Android ActionMode is a built-in contextual action mode which enables users to select multiple items in an activity and perform the same action on all of them. This mode can be used to create a variety of activities like deleting, copying and pasting, or simply performing any action that can be performed on a single item. This tutorial will cover the basics of using Android ActionMode including how to create and customize it, as well as some tips on using it effectively.

Creating an ActionMode

Creating an ActionMode requires implementing an interface from the android.view.ActionMode package. This interface is OnActionModeStartedListener. The following code example shows how to create an ActionMode and set its callback - 

          
           
ActionMode actionMode = startActionMode(new OnActionModeStartedListener() {
    @Override
    public boolean onCreateActionMode (ActionMode mode, Menu menu) {
        //Do something when ActionMode is created
        return true;
    }

    @Override
    public boolean onPrepareActionMode (ActionMode mode, Menu menu) {
        //Do something when ActionMode is started
        return true;
    }

    @Override
    public boolean onActionItemClicked (ActionMode mode, MenuItem item) {
        //Do something when user clicks a menu item
        return true;
    }

    @Override
    public void onDestroyActionMode (ActionMode mode) {
        //Do something when ActionMode is finished
    }
});

Customizing ActionMode

ActionModes can be customized to change the look, feel, and behaviour of the ActionMode’s UI. For example, the following code snippet shows how to customize the ActionMode’s title: 

          
           
actionMode.setTitle("My Custom ActionMode");

actionMode.setCustomView(R.drawable.my_custom_icon);

Using ActionMode Effectively

When using ActionMode there are several best practices to keep in mind for effective usage. First, it is important to remember to always include a “Cancel” option in the ActionMode’s menu. This gives users the ability to easily exit the mode without having to take any action. Second, it is best practice to display the current selection count in the ActionMode’s title. This allows users to quickly see how many items they have selected and it can be updated whenever the selection changes. Finally, it is important to use the setTag() and getTag() methods on the ActionMode object to store and retrieve state data that may be needed for processing the selected items in the future.
