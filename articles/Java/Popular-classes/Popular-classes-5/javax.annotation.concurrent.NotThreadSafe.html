javax.annotation.concurrent.NotThreadSafe

The javax.annotation.concurrent.NotThreadSafe annotation indicates that a class is not thread-safe and must be externally synchronized when accessed from multiple threads. This annotation can be used to denote classes that must be protected by some form of synchronization to maintain their thread-safety. Use this annotation for any class or interface in which the non-atomic behavior of one thread affects the behavior of another thread.

Code Examples

Example 1: This code shows an example of a class that is not thread-safe and therefore must be externally synchronized:


          
           
@NotThreadSafe
public class NotThreadSafeClass {
    private List<String> list = new ArrayList<>();

    public void add(String str) {
        list.add(str);
    }
}

Example 2: The following code shows how to use a synchronized block to synchronize access to the NotThreadSafeClass instance:


          
           
NotThreadSafeClass notThreadSafeClass = new NotThreadSafeClass();
synchronized (notThreadSafeClass) {
    notThreadSafeClass.add("string");
}

Example 3: This code shows an example of a class that is thread-safe and does not need external synchronization:


          
           
@ThreadSafe
public class ThreadSafeClass {
    private List<String> list = Collections.synchronizedList(new ArrayList<>());

    public void add(String str) {
        list.add(str);
    }
}

Example 4: This code shows how to use the thread-safe version of the NotThreadSafeClass class:


          
           
ThreadSafeClass threadSafeClass = new ThreadSafeClass();
threadSafeClass.add("string");
