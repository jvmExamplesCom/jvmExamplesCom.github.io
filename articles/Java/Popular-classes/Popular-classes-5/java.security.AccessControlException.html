The 

          
           
java.security.AccessControlException
is an exception that is thrown when access to a resource is denied due to the security policy of the Java Virtual Machine (JVM). It can also be thrown when attempting to modify or create a security-protected resource, or when trying to invoke a protected method.

When an AccessControlException is thrown, the application should handle it in a meaningful way, as it indicates that something illegal has been attempted.

Example 1

Consider the following code example, which attempts to delete a file named 

test.txt
from the current working directory: 

          
           
File file = new File("test.txt");
file.delete();

AccessControlException
being thrown with a message similar to the following: 

          
           
access denied ("java.io.FilePermission" "test.txt" "delete")
