org.testng.annotations.BeforeTest

The BeforeTest annotation can be used when writing testNG tests to execute code before the tests start running. This annotation can be used in two ways: to name a method as the BeforeTest or to specify a group of methods to be run as the BeforeTest. In either case, the BeforeTest will be executed before any other test methods.

Example 1: Specifying a Method as the BeforeTest

The following example shows how to mark a method with the @BeforeTest annotation. This method will be executed before any other test methods. 


          
           
@Test
public class BeforeTestExample{
    @BeforeTest
    public void setUp(){
        System.out.println("Executing setUp method");
    }

    @Test
    public void testMethod(){
        System.out.println("Executing testMethod");
    }
}
The output from this code would be: 

          
           
Executing setUp method
Executing testMethod

Example 2: Specifying a Group of Methods as the BeforeTest

The following example shows how to use the @BeforeTest annotation to specify a group of methods, which will all be executed before any other test methods. This example uses the @Test annotation along with the attribute groups, which is used to indicate which group the test belongs to.


          
           
@Test(groups="groupA")
public class BeforeTestGroupExample{
    @BeforeTest(groups="groupA")
    public void setUpGroupA(){
        System.out.println("Executing setUpGroupA method");
    }

    @Test
    public void testMethodA(){
        System.out.println("Executing testMethodA");
    }

    @BeforeTest(groups="groupB")
    public void setUpGroupB(){
        System.out.println("Executing setUpGroupB method");
    }

    @Test(groups="groupB")
    public void testMethodB(){
        System.out.println("Executing testMethodB");
    }
}
The output from this code would be: 

          
           
Executing setUpGroupA method
Executing testMethodA
Executing setUpGroupB method
Executing testMethodB
