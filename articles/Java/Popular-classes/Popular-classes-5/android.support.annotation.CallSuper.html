android.support.annotation.CallSuper

The android.support.annotation.CallSuper is an annotation which indicates that any overridden method should call the super implementation. It's commonly used to help reduce bugs in projects, by providing compiler time checks for methods that have been annotated as requiring a super call. In particular, if a method annotated with @CallSuper is overridden and the overriding method does not make a call to its superclass corresponding method, then compilation will fail. This helps to ensure that overridden methods are following the appropriate guidelines and don't accidentally miss something important in their implementation.

Examples

Let's look at some examples of how @CallSuper might be used in Android applications.

Example 1: Overriding onResume()

Suppose we have a custom Activity subclass with the following onResume() method: 

          
           
@CallSuper
protected void onResume() {
   // Do something
}
We can see that this method is annotated with @CallSuper. This means that any method which overrides it must call the super implementation, or else compilation will fail with an error like "Call to 'super' implementation missing". For example, if we have another custom Activity subclass which overrides this method, it might look something like this: 

          
           
protected void onResume() {
    // Do something else

    super.onResume(); // Note the call to super.onResume()
}
Here, we can see that the overriding method is making a call to its superclass implementation, thereby preventing any potential issues which may arise from omitting the call.
