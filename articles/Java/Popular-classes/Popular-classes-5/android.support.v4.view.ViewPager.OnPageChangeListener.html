android.support.v4.view.ViewPager.OnPageChangeListener

The android.support.v4.view.ViewPager.OnPageChangeListener interface is used to listen for changes in the current page being viewed by a ViewPager object. This includes changes that are due to the user swiping, flinging and/or selecting a new page. It also includes page changes triggered by the setCurrentItem() method.

Implementation

To use a OnPageChangeListener, it must be registered with a ViewPager object via its addOnPageChangeListener() method. Once this is done, the OnPageChangeListener's methods will be called when appropriate. The code snippet below shows how a OnPageChangeListener is implemented:


          
           
ViewPager viewPager = new ViewPager(this);
viewPager.addOnPageChangeListener(new ViewPager.OnPageChangeListener() {
    @Override
    public void onPageScrolled(int position, float positionOffset, int positionOffsetPixels) {
        // Do something when the page is scrolled
    }

    @Override
    public void onPageSelected(int position) {
        // Do something when the page has been selected
    }

    @Override
    public void onPageScrollStateChanged(int state) {
         // Do something when the scroll state changes
    }
});

The onPageScrolled() method is called when the user scrolls or flicks the pages in the ViewPager. The position argument indicates which page is currently being viewed, information about the page's position offset is passed in through the positionOffset and positionOffsetPixels arguments, in the form of a fraction and pixel offset respectively.

The onPageSelected() method is called when the user selects a page; the current page being viewed is passed in as an argument. Finally, the onPageScrollStateChanged() method is called when the scroll state of the ViewPager changes. The possible states are either SCROLL_STATE_IDLE, SCROLL_STATE_DRAGGING and SCROLL_STATE_SETTLING.

Example Code

The following code snippet shows how a simple OnPageChangeListener can be used in an activity to log some information to the console when the ViewPager is scrolled, selected and scroll state changes:


          
           
public class MyActivity extends Activity {

    private ViewPager viewPager;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);

        viewPager = (ViewPager) findViewById(R.id.viewPager);
        viewPager.addOnPageChangeListener(new ViewPager.OnPageChangeListener() {
            @Override
            public void onPageScrolled(int position, float positionOffset, int positionOffsetPixels) {
                Log.d("ViewPager", "onPageScrolled: position=" + position
                        + ", positionOffset=" + positionOffset
                        + ", positionOffsetPixels=" + positionOffsetPixels);
            }

            @Override
            public void onPageSelected(int position) {
                Log.d("ViewPager", "onPageSelected: position=" + position);
            }

            @Override
            public void onPageScrollStateChanged(int state) {
                Log.d("ViewPager", "onPageScrollStateChanged: state=" + state);
            }
        });
    }
}

When the above activity is run and the ViewPager is scrolled, the following logs will appear on the console:


          
           
ViewPager: onPageScrolled: position=0, positionOffset=0.7, positionOffsetPixels=56
ViewPager: onPageSelected: position=1
ViewPager: onPageScrollStateChanged: state=1
