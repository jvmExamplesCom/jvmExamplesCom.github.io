Introduction to org.elasticsearch.client.transport.TransportClient

The org.elasticsearch.client.transport.TransportClient class provides a Java API for making requests to an Elasticsearch cluster using the Elasticsearch Transport Protocol. This can be used to perform operations such as creating, deleting, and updating documents as well as interacting with the index, mapping, and search APIs. Additionally, TransportClient also supports retrieval of cluster state information, managing index templates, and executing searches.

Creating a TransportClient Instance

Creating a new TransportClient instance is done by passing the configuration settings for the desired target Elasticsearch cluster. These settings may include connection information such as the URL and port, authentication credentials, and other options.


          
           
TransportClient client =
    new PreBuiltTransportClient(Settings.EMPTY)
        .addTransportAddresses(
            new InetSocketAddress(hostName, portNumber));

Sending Requests

To send a request to the target cluster, use the TransportClient.performRequest() method passing the API endpoint, request type (GET, POST, PUT, etc.), and any necessary parameters. The following example uses the GET API to retrieve all documents in the twitter index:


          
           
Response response = client.performRequest("GET", "/twitter/_search");

The response object contains the status code, headers, and response body, which is typically either in JSON or text format. To access the response body as a String, use Response.getEntity().getContentLength() as shown in the following example:


          
           
InputStream stream = response.getEntity().getContent();
String content = IOUtils.toString(stream);

Closing the Client

When the client is no longer being used, it should be closed using the TransportClient.close() method. This will ensure that all connections to the cluster are gracefully closed.


          
           
client.close();
