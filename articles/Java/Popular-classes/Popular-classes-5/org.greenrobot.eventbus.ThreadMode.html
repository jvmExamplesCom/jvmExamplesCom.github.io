org.greenrobot.eventbus.ThreadMode

The org.greenrobot.eventbus.ThreadMode class is an enumeration that specifies the thread in which an event bus listener should be called on a subscribed event.

ThreadMode Enums

  • POSTING - The event will be delivered directly to the subscribed listener in the same thread of the sender.
  • MAIN - The event will be delivered to the subscribed listener in the main thread.
  • BACKGROUND - The event will be delivered to the subscribed listener in the background thread.
  • ASYNC - The event will be delivered to the subscribed listener in a new thread.

Example of Using ThreadMode with @Subscribe Annotation


          
           
@Subscribe(threadMode = ThreadMode.MAIN)
public void onMessageEvent(MessageEvent event){
  // Do something
}

Example of Using PostSticky Method

The postSticky method can be used for posting events to all subscribers, regardless of when they subscribed. A sticky event is also retained in the event bus between event bus restarts makes it particularly useful for maintaining state between sessions.


          
           
EventBus.getDefault().postSticky(myEvent);
