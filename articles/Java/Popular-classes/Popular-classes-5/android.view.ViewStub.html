What Is Android ViewStub?

Android ViewStub is an invisible, zero-sized View that can be used to lazily inflate layout resources at runtime. When a ViewStub is made visible, or when 


          
           
inflate()
is invoked, the layout resource is inflated. The ViewStub then replaces itself in its parent with the inflated View or ViewGroup.

Implementation of ViewStub

ViewStub can be implemented in the following steps:

  1. Create an XML layout resource file.
  2. In your Activity's onCreate() method, call 
    setContentView()
    to inflate the layout resource you created in step 1.
  3. Find the ViewStub in the inflated layout and call 
    findViewById()
    .
  4. Set an 
    OnClickListener()
    to the ViewStub.
  5. When the ViewStub is clicked, call 
    inflate()
    .
  6. The inflated view will replace the ViewStub in the view hierarchy.

Example

Let's look at a simple example of using ViewStub to inflate a layout. We first need to create a layout resource file with the ViewStub. We will call it 


          
           
stub_layout.xml
. 

          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical"
    android:layout_width="match_parent"
    android:layout_height="match_parent">
    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        android:padding="16dp"/>
    <ViewStub
        android:id="@+id/view_stub"
        android:layout="@layout/stub_inflated"
        android:layout_width="match_parent"
        android:layout_height="match_parent"/>
</LinearLayout>

Next, we create the layout resource that will be inflated by the ViewStub. We will call it 


          
           
stub_inflated.xml
. 

          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical"
    android:layout_width="match_parent"
    android:layout_height="match_parent">
    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="This is inflated!"
        android:padding="16dp"/>
</LinearLayout>
Now, in the activity's 

          
           
onCreate()
method, we set the content view to our layout: 

          
           
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.stub_layout);
}
Now, we find the ViewStub and set an 

          
           
OnClickListener()
for it: 

          
           
ViewStub viewStub = (ViewStub) findViewById(R.id.view_stub);
viewStub.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        // inflating the layout resource
    }
});

When the ViewStub is clicked, we inflate the layout resource:


          
           
viewStub.inflate();

The inflated view will replace the ViewStub in the view hierarchy. Now, the layout should look like this:


          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical"
    android:layout_width="match_parent"
    android:layout_height="match_parent">
    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        android:padding="16dp"/>
    <LinearLayout
        android:orientation="vertical"
        android:layout_width="match_parent"
        android:layout_height="match_parent">
        <TextView
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="This is inflated!"
            android:padding="16dp"/>
    </LinearLayout>
</LinearLayout>

Conclusion

ViewStub is a convenient way to load complex layouts at runtime. It is lightweight and has low memory footprint. ViewStub can also be used to conditionally show or hide views in a layout. It is a great alternative to creating multiple layout files.
