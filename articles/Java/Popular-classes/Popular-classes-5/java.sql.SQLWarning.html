Java.sql.SQLWarning - Overview

The Java.sql.SQLWarning class is a subclass of the java.sql.SQLException class and provides specific warning information about database access issues. This class is available in the Java SE platform since version 1.6.

Key Features

  • Provides a warning message that can be used to diagnose and correct an application's database related problems.
  • The warning object returned by the getNextWarning() method has a cause attribute. This attribute can point to another SQLException object that caused this warning to occur.
  • Provides support for printing the warning message in a user-friendly format via the toString() method.

Code Example - Get Warnings

In the following code example, we are accessing the ‘test’ database and retrieving the results from a table called ‘test_data’ using the executeQuery() method. We then check for any warnings using the getWarnings() method and output the warning message to the console. 

          
           
try{
    // Create a connection
    Connection conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/test");

    // Execute the query
    ResultSet rs = conn.createStatement().executeQuery("SELECT * FROM test_data");

    // Check for any warnings
    SQLWarning warn = conn.getWarnings();
    if(warn != null){
        System.out.println("Warning Message: " + warn.getMessage());
    }
}catch(SQLException e){
    e.printStackTrace();
}

Example Output

The output of the above code example would be a warning message like the one below if there were any warnings from the query: 

          
           
Warning Message: The query did not return any rows. Some possible causes could be invalid criteria or columns.
