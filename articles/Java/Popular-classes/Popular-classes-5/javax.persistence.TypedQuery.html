javax.persistence.TypedQuery

The javax.persistence.TypedQuery interface is part of Java Persistence API in the Java language for accessing database resources. It is used to create and execute typed queries, which are defined by a query language such as SQL. It provides methods for setting query parameters, executing the query, and retrieving the results.

Creating a TypedQuery Object

To create an instance of javax.persistence.TypedQuery, use the EntityManager's createQuery() method, passing in the query string, followed by the desired entity type:


          
           
EntityManager em = EntityManagerFactory.createEntityManager(); 
 String queryString  = "SELECT e FROM Employee e";  
 TypedQuery<Employee> query = em.createQuery(queryString, Employee.class);

Setting Query Parameters

We can set query parameters using the setParameter() method. This takes two parameters: a parameter name and the value we want to set it to. For example, to set the firstName parameter to "John":


          
           
query.setParameter("firstName", "John");

Executing the Query

Once our query is ready, we can execute it using one of the following methods:

  • 
            
             
TypedQuery.getResultList()
    - returns a List of the results
  • 
            
             
TypedQuery.getSingleResult()
    - returns a single result
  • 
            
             
TypedQuery.executeUpdate()
    - returns the number of records affected by the query

Retrieving the Results

The TypedQuery.getResultList() and TypedQuery.getSingleResult() methods return the query results. We can then access these results with standard Java collections operations, such as iterating over the list or accessing individual elements. For example:


          
           
List<Employee> employees = query.getResultList(); 
 for (Employee employee : employees) { 
     System.out.println("EmployeeFirstName: " + employee.getFirstName()); 
 }
