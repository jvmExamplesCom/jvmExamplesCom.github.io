Introduction to MockHttpServletResponse

The 

          
           
org.springframework.mock.web.MockHttpServletResponse
class is a tool provided by the Spring framework for testing servlets and web applications. It provides a convenient way to simulate HTTP requests and responses without having to actually hit an HTTP server. This is useful when unit testing code that relies on an HTTP response, such as AJAX calls. In this article, we will explore how to use the MockHttpServletResponse class to test these types of applications.

Creating a Basic MockHttpServletResponse

Creating a MockHttpServletResponse is easy. All you have to do is call the constructor, passing in the request object that you wish to mock.


          
           
MockHttpServletResponse response = new MockHttpServletResponse(request);

Setting Response Headers

You can set response headers using the 
setHeader()
method. For example, to set the response code to 200, you can do this: 

          
           
response.setHeader("HTTP/1.1", "200 OK");

You can also set other response headers like content type, content length, etc. using the same method.

Setting the Response Body

response.setContent("<html><head><title>My HTML Document</title></head><body>This is my HTML document</body></html>");
