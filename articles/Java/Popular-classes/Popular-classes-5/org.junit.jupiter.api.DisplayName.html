Exploring org.junit.jupiter.api.DisplayName

JUnit 5 introduced the org.junit.jupiter.api.DisplayName annotation which allows developers to dynamically assign a test name. This can be particularly useful when tests names become unwieldy in length due to the structure of the naming convention chosen. By using the DisplayName annotation, you can override those long-form names with a more descriptive, user-friendly version. In this article, we’ll explore how to use the DisplayName annotation and some of the best practices when doing so.

Setting the Test Name with a String

The simplest way to use the DisplayName annotation is to pass it a String argument that will become the test name. This String can contain spaces and other characters that normally wouldn’t be used in a test method’s name. For example, let’s imagine we have a class called 

          
           
UserManager
and a test method called 

          
           
testCreateUser
. This can be annotated with the DisplayName annotation: 

          
           
@Test
@DisplayName("Create a User")
void testCreateUser() {
    // ...
}
This is particularly helpful when you want to make the test name more descriptive and easier to remember.

Using Placeholders with Formatted Strings

The DisplayName annotation also supports the use of placeholders in formatted strings. These placeholders are used to insert variable pieces of data into the string when the test runs. This is especially useful for parameterizing tests. For example, let’s imagine we have the following method: 

          
           
void testAddUser(String name, int age)
The DisplayName annotation can be used to add more information to the test name by using a formatted string and the parameters as placeholders: 

          
           
@Test
@DisplayName("Add user \"%s\" (age %d)")
void testAddUser(String name, int age) {
    // ...
}
This will output the name of the user and their age in the test name when the test is run.

Conclusion

The DisplayName annotation is a powerful tool that can make tests easier to read and understand. It allows developers to assign dynamic names to their tests, making them easier to find and interpret. Furthermore, the annotation supports formatted strings that allow developers to parameterize their tests and add more information to the test name. Overall, the DisplayName annotation is a great way to keep your test code organized and readable.
