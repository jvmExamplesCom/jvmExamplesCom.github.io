java.lang.instrument.Instrumentation

java.lang.instrument.Instrumentation is an interface that enables tools to instrument Java programming language code. It can be used by external agents, such as profilers and code coverage tools, to gain access to information about the running application.

Instrumentation Basics

Instrumentation provides a way to alter the bytecodes of an application after it has been loaded into the Java Virtual Machine (JVM). This allows for reactive and proactive monitoring, as well as on-the-fly modification of an application.

Instrumentation works by adding a subclass of java.lang.instrument.Instrumentation to the JVM, which then receives events when classes are loaded and unloaded. This includes the loading of classes into the JVM as part of application startup as well as the unloading of classes after application shutdown.

When a class is loaded, Instrumentation can make modifications to the bytecode, such as inserting hooks into method invocations or disabling certain operations. It can also modify class data, such as adding fields or methods to a class.

Example Usage

Instrumentation can be used for a variety of tasks, including performance profiling, debugging, security audits, code coverage analysis, and more. One common use of Instrumentation is for implementing application-level caching. Caching can improve performance in applications by providing quick access to frequently accessed objects without having to re-instantiate them each time.

public class CacheInstrumentation implements ClassFileTransformer {
    private final Map cache;
 
    public CacheInstrumentation() {
        this.cache = new HashMap();
    }
 
    public byte[] transform(ClassLoader loader, String className, Class<?> classBeingRedefined, ProtectionDomain protectionDomain, byte[] classfileBuffer) {
        // if the object is in the cache, return it
        if (cache.containsKey(className)) {
            return (byte[]) cache.get(className);
        }
 
        // otherwise, add it to the cache
        cache.put(className, classfileBuffer);
        return classfileBuffer;
    }
}

Instrumentation provides a powerful way to instrument Java applications at runtime.
