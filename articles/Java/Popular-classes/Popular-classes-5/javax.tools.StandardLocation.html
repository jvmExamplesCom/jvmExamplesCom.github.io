javax.tools.StandardLocation

javax.tools.StandardLocation is a file-system independent way of identifying the location of compiled files. It is used in various Java tools such as the Java Compiler API and other tools like javac, apt, and javadoc.

What is the StandardLocation class?

The StandardLocation class is an abstraction from the underlying file-system that allows client code to work with an organized view of the available locations for compiled files and their associated metadata. The classes in this package provide basic ways to identify these locations and to retrieve the files and metadata that can be found within them.

Example of usage

Below is an example of how to retrieve the source path of a project using the StandardLocation API: 

          
           
StandardLocation sourcePath = StandardLocation.SOURCE_PATH;

List<File> files = compiler.getLocation(sourcePath);
The above code will return a list of all paths that contain source files for the project.

Other Examples

The StandardLocation class also allows you to access other types of files such as class files, module descriptor files, and more. Here is an example of how to get the module descriptor of a project:


          
           
StandardLocation moduleDescriptorPath = StandardLocation.MODULE_PATH;

File moduleDescriptor = compiler.getLocation(moduleDescriptorPath);

The above code will return the module descriptor file for the project.

Conclusion

The StandardLocation class provides an abstract way of working with files and their associated metadata. It is used in a variety of Java tools and can be used to retrieve source paths, class files, module descriptors, and more. This makes it an invaluable tool when dealing with Java projects.
