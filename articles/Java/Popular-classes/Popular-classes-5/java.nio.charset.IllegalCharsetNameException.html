java.nio.charset.IllegalCharsetNameException

Introduction

Java.nio.charset.IllegalCharsetNameException is an exception class in Java that indicates that a given charset name is illegal or unsupported. It is a subclass of IllegalArgumentException which itself is unchecked exception, meaning it extends RuntimeException. This exception is thrown when invalid charset names are used as arguments to constructors.

Example Usage

In the following example, we have an invalid charSetName used to call the charsetForName method of the Charset class with two possible outcomes. 

          
           
String charSetName= "UTF-8§";
try {
    Charset.forName(charSetName);
} catch (IllegalArgumentException e) {
    System.out.println("Invalid charsetName: "+charSetName);
    e.printStackTrace();
}
The above code will throw an 

          
           
IllegalCharsetNameException
telling us that the charset name provided was invalid. The output would be something like this: 

          
           
Invalid charsetName: UTF-8§
java.lang.IllegalCharsetNameException: UTF-8§
	at java.nio.charset.Charset.checkName(Charset.java:540)
	at java.nio.charset.Charset.forName(Charset.java:571)
        ...

Sample Solution

The solution to this problem is to ensure that any charset name passed to the constructor is a valid one. This can be achieved by checking the list of supported charset names from the Charset class. 

          
           
// Get the list of supported charset names
Set<String> charsetNames = Charset.availableCharsets().keySet();

// Check if the provided charsetName is present in the set
if(charsetNames.contains(charSetName)) {
    Charset.forName(charSetName);
} else {
    System.out.println("Invalid charsetName: "+charSetName);
}
The above code will check if the charsetName provided is a valid one and only proceed if it is. Otherwise, an error message will be printed out.

Conclusion

In this article, we discussed the Java.nio.charset.IllegalCharsetNameException and how to prevent it. We saw how to check for valid charset names before passing them to the charsetForName() method. Although this exception is unchecked and can be ignored, it is important to properly handle exceptions in order to ensure our programs are robust and reliable.
