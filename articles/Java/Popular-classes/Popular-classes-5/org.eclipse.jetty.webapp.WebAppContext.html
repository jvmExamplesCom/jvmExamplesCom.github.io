Understanding the org.eclipse.jetty.webapp.WebAppContext

The WebAppContext is an extensible servlet context handler for Jetty. It provides a configured environment and access to the server's request, response and session attributes, making it possible to create servlet-based applications using the Jetty web server.

Creating a Simple Hello World WebApp

First things first, we'll need to create a simple webapp to demonstrate how to use the WebAppContext. We can do this by creating files in our project directory:

  • index.html - serves as the entry point of our application
  • HelloWorldServlet.java - creates a simple "hello world" response

Once these files are created, we can configure the WebAppContext. This can be done in two ways: through XML or Java.

XML Configuration

We'll start with the XML configuration, which requires us to create a web.xml file in the project directory. This file should contain the following code:


          
           
<!DOCTYPE web-app PUBLIC
 "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN"
 "http://java.sun.com/dtd/web-app_2_3.dtd">

<web-app>

	<servlet>
		<servlet-name>HelloWorldServlet</servlet-name>
		<servlet-class>HelloWorldServlet</servlet-class>
	</servlet>

	<servlet-mapping>
		<servlet-name>HelloWorldServlet</servlet-name>
		<url-pattern>/*</url-pattern>
	</servlet-mapping>

</web-app>

This file tells the WebAppContext to use our HelloWorldServlet for all requests. Once this is done, we can create a WebAppContext instance and pass it the web.xml file. For example:


          
           
WebAppContext context = new WebAppContext();
context.setDescriptor("web.xml");
context.setResourceBase(".");
context.setContextPath("/");

Finally, we can start the webapp by passing our WebAppContext instance to the Jetty server:


          
           
Server server = new Server(8080);
server.setHandler(context);
server.start();
server.join();

At this point, we should have a working webapp that responds to the path / with our "Hello World" response.

Java Configuration

The Java configuration is a bit more involved, but still relatively simple. We'll need to add a few extra lines to our WebAppContext setup, as follows:


          
           
WebAppContext context = new WebAppContext();
context.setResourceBase(".");
context.setContextPath("/");
context.addServlet(HelloWorldServlet.class, "/*");

Here we're instructing the WebAppContext to use our HelloWorldServlet for all requests, just like we did with the XML configuration. As before, we can start the webapp by passing our context instance to the Jetty server, and we should have a working webapp.

Conclusion

We've seen how to use the WebAppContext to create a simple "hello world" webapp with both XML and Java configuration. The WebAppContext provides a convenient way to create servlet-based applications using the Jetty web server, making it an essential part of any Jetty-based application.
