javax.management.remote.JMXConnector

javax.management.remote.JMXConnector is an interface that forms the base of the JMX Remote API. This API allows applications to connect and interact with a remote instance of a Java Virtual Machine (JVM) through the use of Management Beans, or MBeans, allowing for greater control over remote JVMs.

Connecting to a Remote JVM

In order to connect to a remote JVM, an instance of the 

JMXConnector
interface can be created using the following code: 

          
           
JMXServiceURL url = new JMXServiceURL("service:jmx:rmi:///jndi/rmi://localhost:1099/jmxrmi");
JMXConnector jmxc = JMXConnectorFactory.connect(url);
This code creates an instance of 
JMXServiceURL
which defines the address of the remote JVM instance, then creates a 
JMXConnector
from the URL using the static 

          
           
JMXConnectorFactory#connect
method.

Using the Connector

Once the connector has been created, the MBeans stored on the remote JVM can be accessed using the 


          
           
MBeanServerConnection
instance returned by the 
JMXConnector
: 

          
           
MBeanServerConnection connection = jmxc.getMBeanServerConnection();

Using this connection instance, MBeans can be accessed and manipulated in the same way as if they were local, e.g. invoking methods, setting and getting attributes etc.


          
           
ObjectName mbeanName = new ObjectName("example:name=ExampleMBean");
ExampleMBean mbeanProxy = JMX.newMBeanProxy(connection, mbeanName, ExampleMBean.class);
int result = mbeanProxy.someMethod();

Closing the Connection

When the connection is no longer needed, it should be closed using the 

JMXConnector#close
method: 

          
           
jmxc.close();
