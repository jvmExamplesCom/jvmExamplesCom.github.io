Introduction to android.os.PersistableBundle

Android is an open source mobile operating system developed by Google and the Open Handset Alliance. The Android platform includes a powerful set of application programming interfaces (APIs) for creating applications. One of those APIs is the android.os.PersistableBundle API, which enables the developer to persist state data across different activities in their application. In this article, we will examine the basics of android.os.PersistableBundle and look at some examples of how it can be used.

What is android.os.PersistableBundle?

android.os.PersistableBundle is an API that provides a persistent store for data across different activities within an Android application. With the help of android.os.PersistableBundle, you can save and restore data associated with a particular activity using lifecycle methods. It is important to understand that data stored in android.os.PersistableBundle is persistent across multiple application states, such as when your application launches or when the user navigates away from the current activity.

Using android.os.PersistableBundle

android.os.PersistableBundle can be used in an Android application in two ways: by saving data with its put*() methods or by restoring data with its get*() methods.

Saving Data with android.os.PersistableBundle

You can save data with android.os.PersistableBundle by calling the put*() methods that correspond to the type of data you are saving. For example, if you have a string variable called “myString”, you would call the putString() method as follows: 

          
           
myPersistableBundle.putString("myString", myString);
Other types of data that can be saved in android.os.PersistableBundle include booleans, integers, floats, and arrays.

Restoring Data with android.os.PersistableBundle

Restoring data from android.os.PersistableBundle is just as easy as saving it. To restore data from this bundle, simply call the get* method corresponding to the type of data you want to retrieve. For example, if you have a string variable called “myString”, you would call the getString() method as follows: 

          
           
myString = myPersistableBundle.getString("myString");

Example Application

Now let's look at a simple application that uses android.os.PersistableBundle to store and retrieve data. This example application will have a text field and a button. When the button is pressed, the text in the text field will be saved to android.os.PersistableBundle and when the screen is re-visited, the text will be restored from the bundle. 

          
           
public class MainActivity extends AppCompatActivity {

    EditText editText;
    Button saveButton;
    PersistableBundle persistableBundle;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        
        editText = findViewById(R.id.editText);
        saveButton = findViewById(R.id.saveButton);

        // Restore data if it has been saved
        if (savedInstanceState != null) {
            persistableBundle = savedInstanceState.getPersistableBundle("myBundle");
            if (persistableBundle != null) {
                String text = persistableBundle.getString("myText");
                editText.setText(text);
            }
        }

        saveButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                String text = editText.getText().toString();
                if (persistableBundle == null) {
                    persistableBundle = new PersistableBundle();
                }
                persistableBundle.putString("myText", text);
            }
        });
    }

    @Override
    protected void onSaveInstanceState(Bundle outState) {
        super.onSaveInstanceState(outState);
        outState.putPersistableBundle("myBundle", persistableBundle);
    }
}
In this example, we save the text from the text field when the button is pressed and restore it when the Activity is re-visited. To do this, we use the onSaveInstanceState() and onCreate() methods, which are called when the Activity's state is being saved and restored, respectively.
