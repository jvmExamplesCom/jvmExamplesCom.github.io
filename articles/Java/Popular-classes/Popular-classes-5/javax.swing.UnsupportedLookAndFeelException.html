javax.swing.UnsupportedLookAndFeelException

javax.swing.UnsupportedLookAndFeelException is an exception thrown when a program attempts to set the look and feel (L&F) of a Swing component but the L&F is not supported on the current platform. It is a subclass of java.lang.Exception and is thrown whenever a UIManager.setLookAndFeel call can’t find the specified L&F class, or when a LookAndFeel class obtained via UIManager.getLookAndFeel returns false when asked if supported on this platform.

Example usage

In the following example, an UnsupportedLookAndFeelException is thrown as the Nimbus L&F is not supported on the current platform: 

          
           
try {
    UIManager.setLookAndFeel("javax.swing.plaf.nimbus.NimbusLookAndFeel");
} catch (ClassNotFoundException | InstantiationException | IllegalAccessException | UnsupportedLookAndFeelException e) {
    throw new RuntimeException(e);
}
This will throw an UnsupportedLookAndFeelException with a message similar to: 

          
           
javax.swing.UnsupportedLookAndFeelException: Nimbus [....] not supported on this platform

Handling

When handling an UnsupportedLookAndFeelException, it is generally best practice to simply log the exception before re-throwing it - any action to reset the look and feel must be done manually, since the look and feel that was attempted to be set was not supported on the current platform anyway. In the example code above, the exception is handled by wrapping it in a runtime exception and re-throwing it.

Convenience methods

The UIManager class provides several convenience methods for obtaining the current look and feel, for instance UIManager.getSystemLookAndFeelClassName(), which returns the name of the current system look and feel class. This can be used to avoid needing manual error handling for any potential UnsupportedLookAndFeelException thrown when attempting to set a specific look and feel. For example, the following code sets the look and feel to the current system look and feel, avoiding any manual error handling: 

          
           
UIManager.setLookAndFeel(UIManager.getSystemLookAndFeelClassName());

Conclusion

javax.swing.UnsupportedLookAndFeelException is an exception that is thrown whenever attempting to set a Swing look and feel class that is not supported on the current platform. When handling this exception it is usually best to log it rather than try to manually reset the look and feel - instead, use convenience methods provided by the UIManager class to obtain the current system look and feel instead.
