org.junit.runners.Parameterized.Parameter

The 
Parameterized
class from the 
org.junit.runners
package is a type of JUnit runner which allows users to run the same test over and over again with different values. It is a convenience feature, allowing for concise and readable test cases with minimal repetition. The 
Parameter
class is a helper class needed to specify the parameters given to the Parameterized runner.

The @Parameters annotation

The 
@Parameters
annotation is used to specify the data that should be passed to the test method. For example: 

          
           
@RunWith(Parameterized.class)
public class MyClassTest {

    // Populated by the @Parameters annotation
    @Parameters
    public static Collection<Object[]> data() {
        return Arrays.asList(new Object[][] { { 33 }, { 34 } });
    }  
}

This annotation specifies that two sets of data should be passed to the test, with the values 33 and 34.

The Parameter class

To pass data to a test method, the 
Parameter
class must be used: 

          
           
@RunWith(Parameterized.class)
public class MyClassTest {

    // Populated by the @Parameters annotation
    @Parameters
    public static Collection<Object[]> data() {
        return Arrays.asList(new Object[][] { { 33 }, { 34 } });
    }

    @Parameter
    public int num;

    @Test
    public void testMyClass() {
        // do something with "num"
    }
}

In this example, two parameters are specified in the 

@Parameters
annotation. When the test is run, the value of each parameter will be assigned to the 
num
variable in turn. This means that when the test is run twice, it will execute once with the value 
num = 33

num = 34
.

Using the Parameter class in test methods
