Introduction To org.apache.shiro.realm.AuthorizingRealm

org.apache.shiro.realm.AuthorizingRealm is an abstract class which provides a convenient base class for implementing Shiro Realms that support authorization operations. It encapsulates the most common authorization-related behavior and leaves only the actual authorization (access control) logic to subclasses.

What Is Authorization?

Authorization is the process of granting access rights to user roles or users. In Shiro, this usually means granting access to certain protected resources/actions associated with a user’s identity.

Basic Flow

When a request is made for protected resources, the application must first verify that the current user has the correct permissions to access those resources. This is typically done by obtaining the current Subject’s authority list and comparing it against the privileges required to access the target resource.

Using AuthorizingRealm

To implement authorization in a Shiro realm, we can extend from the AuthorizingRealm class and implement the methods defined in the class. The AuthorizingRealm interface defines two methods that have to be implemented:

  • 
            
             
doGetAuthorizationInfo(PrincipalCollection principals)
    : This method should return a list of authorization information of the specified principals.
  • 
            
             
doGetAuthenticationInfo(AuthenticationToken token)
    : This method should return authentication information of the specified token.

Authentication And Authorization Example

Step 2: Override doGetAuthenticationInfo() Method

We then override the doGetAuthenticationInfo() method to perform authentication. In this method, we get the authentication token, which contains the user’s credentials. We then use these credentials to lookup the user in the database (or other store), and if found, return an AuthenticationInfo object containing the authenticated user’s identity. 


          
           
@Override
protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken token) throws AuthenticationException {
  // get user credentials from token
  String username = (String) token.getPrincipal();
  String password = new String((char[]) token.getCredentials());

  // lookup user in database
  User user = userDao.findByUsername(username);

  // verify password
  if (!password.equals(user.getPassword())) {
    throw new AuthenticationException("Invalid username or password");
  }

  // return an AuthenticationInfo object
  return new SimpleAuthenticationInfo(username, password, getName());
}

Step 3: Override doGetAuthorizationInfo() Method

Next, we override the doGetAuthorizationInfo() method to perform authorization. In this method, we obtain the authenticated user’s identity from the PrincipalCollection object, and then use this identity to lookup the user’s roles and/or permissions in the database. We then return a list of AuthorizationInfo objects containing the user’s roles and/or permissions. 


          
           
@Override
protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principals) {
  // get username from PrincipalCollection
  String username = (String) principals.getPrimaryPrincipal();

  // lookup roles/permissions in database
  Set<String> roles = userDao.findRolesByUsername(username);
  Set<String> permissions = userDao.findPermissionsByUsername(username);

  // return list of AuthorizationInfo objects
  SimpleAuthorizationInfo info = new SimpleAuthorizationInfo(roles);
  info.setStringPermissions(permissions);
  return info;
}

Finally, we register our custom realm with the SecurityManager. This allows Shiro to use our custom realm when authenticating and authorizing users.

SecurityUtils.setSecurityManager(securityManager);
