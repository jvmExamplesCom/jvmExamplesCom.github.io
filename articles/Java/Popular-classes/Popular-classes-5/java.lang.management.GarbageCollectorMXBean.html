Java.Lang.Management.GarbageCollectorMXBean

Garbage Collectors help to manage memory and resources in Java applications. The GarbageCollectorMXBean interface is used to provide access to the instrumentation and management of a Java Virtual Machine's garbage collection. It wraps a Management Interface for the Hotspot Garbage Collector.

Overview of GarbageCollectorMXBean

The GarbageCollectorMXBean class provides methods related to data about a garbage collector's memory management system, such as its name, the names of the memory pools it manages, and the type of garbage collector. It also provides access to statistics about the garbage collector's performance, such as the total amount of time spent collecting garbage, the number of times garbage has been collected, and the number of objects that were discarded during garbage collection.

Code Example


          
           
List<GarbageCollectorMXBean> gcBeans = 
    ManagementFactory.getGarbageCollectorMXBeans();

  System.out.println("Available Garbage Collectors:");
  for (GarbageCollectorMXBean gcBean : gcBeans) {
    System.out.println(gcBean.getName());
  }

This code will print out a list of available garbage collectors. This can be useful if you want to know which garbage collectors are enabled in your application.

Methods

The GarbageCollectorMXBean class provides several methods that can be used to get information about a garbage collector. These methods include:

  • getName()
    - Returns the name of the garbage collector.
  • 
            
             
getMemoryPoolNames()
    – Returns an array of strings representing the names of the memory pools managed by this garbage collector.
  • getObjectName()
    – Returns the ObjectName of this GarbageCollectorMXBean.
  • getType()
    – Returns the type of this garbage collector.
  • 
            
             
getCollectionCount()
    – Returns the number of collections that have occurred.
  • getCollectionTime()
    – Returns the approximate accumulated collection elapsed time in milliseconds.
  • resetPeakUsage()
    – Resets the peak usage statistics.
  • 
            
             
getMemoryManagerNames()
    – Returns the names of all the memory managers managed by this garbage collector.
  • 
            
             
getObjectPendingFinalizationCount()
    – Returns the number of objects for which finalization is pending.

Example for a Garbage Collector


          
           
GarbageCollectorMXBean gcBean = 
  ManagementFactory.getGarbageCollectorMXBean(0);

System.out.println("Name: " + gcBean.getName());
System.out.println("Type: " + gcBean.getType());
System.out.println("Memory Pool Names: " + 
  Arrays.toString(gcBean.getMemoryPoolNames()));
System.out.println("Collection Count: " + 
  gcBean.getCollectionCount());
System.out.println("Collection Time: " + 
  gcBean.getCollectionTime());
System.out.println("Objects Pending Finalization: " + 
  gcBean.getObjectPendingFinalizationCount());

This code example will print out the name, type, memory pool names, number of collections, collection time and the number of objects that are pending finalization of the first garbage collector.
