java.io.NotSerializableException

The java.io.NotSerializableException is a type of exception that is raised when an object cannot be serialized in Java. Serialization is the process of taking an object and converting it into a stream of bytes for storage or transmission over a network. When this process fails, the NotSerializableException will be thrown. This exception extends the java.io.ObjectStreamException class and is commonly seen in Java programs.

Overview

When an object does not implement the Serializable interface, it cannot be serialized. If an attempt is made to serialize an object that does not implement a Serializable interface, a NotSerializableException will be thrown. The NotSerializableException is a checked exception, meaning that the compiler will check that any code that could potentially throw this exception is handled properly. This means that any code that attempts to serialize a non-Serializable object must be surrounded in a try-catch block.

Example Code

The following example demonstrates how a NotSerializableException will be thrown if an object does not implement the Serializable interface.


          
           
class Fruit {
    public void eat() {
        System.out.println("Yum!");
    }
}

public class Test {
    public static void main(String[] args) {
        Fruit apple = new Fruit();

        try {
            ByteArrayOutputStream out = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(out);
            oos.writeObject(apple);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
This code results in the following error: 

          
           
Exception in thread "main" java.io.NotSerializableException: Fruit
	at java.io.ObjectOutputStream.writeObject0(ObjectOutputStream.java:1184)
	at java.io.ObjectOutputStream.writeObject(ObjectOutputStream.java:348)
	at Test.main(Test.java:13)

Solution

To solve this issue, the class needs to implement the Serializable interface. This interface allows the object to be serialized and deserialized. Once this has been done, the NotSerializableException will no longer be thrown.


          
           
import java.io.Serializable;

class Fruit implements Serializable {
    public void eat() {
        System.out.println("Yum!");
    }
}

public class Test {
    public static void main(String[] args) {
        Fruit apple = new Fruit();

        try {
            ByteArrayOutputStream out = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(out);
            oos.writeObject(apple);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

This code will no longer throw a NotSerializableException, and the object can be serialized and deserialized as expected.

Conclusion

The java.io.NotSerializableException is a type of exception raised when an object does not implement the Serializable interface. In order to fix this error, the class must implement the Serializable interface. Once this has been done, the object can be serialized and deserialized as expected.
