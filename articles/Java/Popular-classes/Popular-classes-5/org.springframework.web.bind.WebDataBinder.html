Introduction to org.springframework.web.bind.WebDataBinder

The org.springframework.web.bind.WebDataBinder is a part of the Spring framework for the Java platform. It provides a convenient way to bind request parameter values to an object or a collection of objects, as well as to register custom editors for a given type. The WebDataBinder can be used in both web-based and non web-based applications.

Example

Let’s look at a simple example where we bind a request parameter to an object. We'll be using the Person class below:


          
           
public class Person {
    
    private Long id;
    private String name;
    private Integer age;
    
    // Getters and setters omitted
}

We can then use the WebDataBinder to bind the request parameters to a new instance of the Person class like so:


          
           
WebDataBinder binder = new WebDataBinder(new Person());
binder.bind(request);
Person p = (Person) binder.getTarget();

Once this is done, the properties of the Person class will be populated with the values of the request parameters. For example, if the request contains the parameters "id = 1", "name = John" and "age = 15", the properties of the Person class would be populated as follows:


          
           
p.getId() -> 1
p.getName() -> "John"
p.getAge() -> 15

Customized binding behavior

The WebDataBinder can also be used to customize the binding behavior. For example, we can use the setAllowedFields method to specify which request parameters should be bound to the target object. This can be used to prevent malicious users from manipulating the binding behavior to their own ends.
