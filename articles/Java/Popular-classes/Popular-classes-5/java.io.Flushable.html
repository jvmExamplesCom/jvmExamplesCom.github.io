Java.io.Flushable

Java.io.Flushable is an interface in the java.io package used for indicating whether a class can be flushed - that is, have its buffered output data written to the underlying stream without closing it. It is an abstract class with one method, flush(), which when called flushes any buffered data from its output stream.

Examples of Flushable Usage

Here are some examples of how you might use the Java.io.Flushable interface in your code: 

          
           
PrintWriter out = new PrintWriter(new FileOutputStream("myfile.txt"));
out.println("Hello World!");
out.flush(); // Flush the data to the output stream
In the example above, we are creating a PrintWriter object and writing "Hello World!" to the output stream. We then call the flush() method to ensure that the buffered data is written to the output stream without closing it. Another example could be using the flush() method to ensure data is written to the output stream before a long-running task is executed: 

          
           
FileWriter writer = new FileWriter("myfile.txt");
writer.write(someData);
writer.flush(); // Flush the data to the output stream
// execute long running task
writer.close();
Using the flush() method here makes sure the data written to the output stream is not lost due to the long-running task.

Flushable Class Methods

The Java.io.Flushable class has only one method: 

          
           
flush()
This method is used to flush any buffered data from its output stream, so it is written to the underlying stream without closing it. The method takes no parameters and returns void.

Conclusion

Java.io.Flushable is an interface used for indicating whether a class can be flushed - that is, have its buffered output data written to the underlying stream without closing it. It has only one method - flush() - which can be used to ensure data is written to the output stream before long-running tasks or when closing the output stream.
