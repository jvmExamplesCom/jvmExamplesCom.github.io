JUnit 4 Categories

JUnit 4 provides a powerful mechanism for categorizing tests. It allows you to annotate each test method with a category and run all tests in a certain category separately. This has multiple advantages: it helps you organize tests and reduces the build time. The main interface for working with categories is a JUnit 4 annotation - org.junit.experimental.categories.Category. It's a marker annotation that can be used on classes and methods.

@Category

Let's see how to use this annotation. First, let's define some categories: 

          
           
public interface DataLayerTests { }
public interface LogicTests { }
public interface UITests { }
Now, we can assign test methods to these categories by adding the @Category annotation: 

          
           
@Category(DataLayerTests.class)
@Test
public void testDataLayer() {
    // ...
}

@Category(LogicTests.class)
@Test
public void testLogic() {
    // ...
}

@Category(UITests.class)
@Test
public void testUI() {
    // ...
}
There are a few rules that you should follow when using categories:
  • Any test class or method can have only one category.
  • You can create sub-categories by extending existing categories.
  • You can assign any test to multiple categories at once.

Running Tests in Specific Categories

Now that our tests are categorized, let's see how to run them. We can use the 

          
           
org.junit.experimental.categories.Categories.ExcludeCategory
class to exclude certain categories when running tests. For example, if we want to run all tests except tests in the 

          
           
UITests
category, we can do the following: 

          
           
Categories.ExcludeCategory(UITests.class)
We can also use the 

          
           
org.junit.experimental.categories.Categories.IncludeCategory
class to include only certain categories when running tests. For example, if we want to run only tests in the 

          
           
UITests
category, we can do the following: 

          
           
Categories.IncludeCategory(UITests.class)
The same approach can be used when running tests from the command line. In addition, you can specify multiple categories to be included or excluded. For example, if we want to run all tests except those in the 

          
           
DataLayerTests
and 

          
           
LogicTests
categories, we can do the following: 

          
           
Categories.ExcludeCategory(DataLayerTests.class, LogicTests.class)

Conclusion

In this article, we've seen how to use the JUnit 4 Categories feature to organize tests and reduce build time. We've also seen how to run tests in specific categories from the command line. With this powerful mechanism, you can easily manage and run your tests in an efficient way.
