Introduction to Spring Cloud Config Server 

The Spring Cloud Config Server provides an externalized configuration service to applications in cloud environments. It allows an application to automatically pull its configuration data from a centralized source, such as a Git repository. This helps ensure that applications are always using the most up-to-date configuration and simplifies the process of making changes to configurations across an entire system. In this article, we will explore how to use the Config Server with the @EnableConfigServer annotation.

Creating a Spring Boot App with the @EnableConfigServer Annotation

The quickest way to set up a Spring Boot app with the @EnableConfigServer annotation is to use the Spring Initializr web tool. On the Initializr’s web page, select “Spring Boot” as the project type, and include the “Config Server” dependency. Once the project is created, open it with your favorite IDE. The first step is to add the @EnableConfigServer annotation to the main class, which is responsible for bootstrapping the application. This annotation enables the Config Server and sets up the necessary infrastructure to pull configuration from a Git repository. Here’s an example: 

          
           
@SpringBootApplication
@EnableConfigServer
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
Next, set up the application configuration in the application.yml file. This configuration will tell the Config Server what Git repository to pull configuration data from and how often it should look for updates. Here’s an example: 

          
           
spring:
  cloud:
    config:
      server:
        git:
          uri: https://github.com/myorg/myrepo
          searchPaths: configs/**
          refreshRate: 10s
The searchPaths parameter will define what paths should be searched for configuration files. In this example, any file in the configs folder of the repository will be loaded when the Config Server is started. The refreshRate parameter tells the server how often to check for updates from the Git repository.

Retrieving Configuration Data in a Client App

Once the Config Server is set up and running, you can create a client app to retrieve configuration information. Create a new Spring Boot app using the Initializr, and make sure to include the “Config Client” dependency. Open the client app and add the necessary application configuration in the application.yml file. This configuration will tell the client app where to find the Config Server and what configuration data to retrieve. Here’s an example: 

          
           
spring:
  application:
    name: my-client-app
  cloud:
    config:
      uri: http://localhost:8888
      failFast: true
      name: ${spring.application.name}
In this example, the uri parameter tells the client app where the Config Server can be found. The name parameter specifies what configuration data to retrieve from the Config Server. In this case, we’re asking for the configuration data associated with the “my-client-app” application. Once the application configuration is in place, you can use the @ConfigurationProperties annotation to access the configuration data. Here’s an example of a configuration class that binds the configuration data to a POJO: 

          
           
@ConfigurationProperties("my-client-app")
public class ClientAppConfig {

    private String someProperty;

    public String getSomeProperty() {
        return someProperty;
    }

    public void setSomeProperty(String someProperty) {
        this.someProperty = someProperty;
    }
}
Now the configuration data from the Config Server can be accessed from anywhere in the application by autowiring the ClientAppConfig class.

Conclusion

The Spring Cloud Config Server provides an easy way to manage externalized configuration in cloud environments. By using the @EnableConfigServer annotation, you can easily set up a Config Server to pull configuration data from a Git repository and make it available to application clients. With the help of the @ConfigurationProperties annotation, you can also bind the configuration data to a POJO and access it from anywhere in your application.
