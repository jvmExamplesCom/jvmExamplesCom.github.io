org.springframework.beans.factory.BeanFactoryAware

The 
BeanFactoryAware
interface is a part of the Spring Framework which allows beans to be aware of their bean factories, and therefore have access to mechanisms such as lifecycle events. This interface is typically implemented by application contexts, in order to expose their respective BeanFactories. By implementing this interface, beans are granted access to their parent factory and can therefore use events such as the 
BeanPostProcessor
, which is triggered before any initialization method on the bean.

Example Usage of BeanFactoryAware

Let's take a look at an example of how BeanFactoryAware can be used. In this example, we will create a counter bean which will be aware of the total number of beans that are created in its factory.


          
           
public class CounterBean implements BeanFactoryAware {
    private BeanFactory beanFactory;

    @Override
    public void setBeanFactory(BeanFactory beanFactory) {
        this.beanFactory = beanFactory;
    }

    public int getNumberOfBeans() {
        return beanFactory.getBeanDefinitionCount();
    }
}
The above implementation of the 
BeanFactoryAware
interface ensures that our 
CounterBean
has access to the 
BeanFactory
. We can then use this factory to retrieve the total number of beans that are available in the factory. In order for the 
CounterBean
to be able to use the 
BeanFactory
, it must be registered in the application context. This can be done using XML configuration as follows: 

          
           
<bean id="counterBean" class="example.CounterBean"/>
Or, if you are using the JavaConfig API: 

          
           
@Configuration
public class MyConfiguration {

    @Bean
    public CounterBean counterBean() {
        return new CounterBean();
    }
}

Conclusion

The BeanFactoryAware interface is an incredibly useful tool in the Spring Framework which enables beans to access their factories and consequently make use of features like the BeanPostProcessor. The interface is easy to implement and use, so there is no excuse not to take advantage of this powerful tool.
