Using ResourceServerConfigurerAdapter for Authentication and Authorization

ResourceServerConfigurerAdapter is used to provide authentication and authorization for OAuth-protected resources. It provides two methods for authenticating users: 1. `configure(AuthenticationManagerBuilder)` – this method configures an authentication manager with the credentials provided in the application.yml file. 2. `configure(HttpSecurity)` – this method configures an http security filter chain to control access to the OAuth-protected resource. Both methods call a `authenticate()` method which uses the following steps to authenticate the user:
  • Get the user’s credentials
  • Verify the credentials with a trusted identity provider
  • Create a security context with the user’s roles and permissions
Once the user has been authenticated, the ResourceServerConfigurerAdapter will use the configured authorities to authorize the user for the requested resource. This authorization is done by checking if the user has the required authority to access the resource. If the user does not have the required authority, an exception is thrown.

@Configuration

  }
}
