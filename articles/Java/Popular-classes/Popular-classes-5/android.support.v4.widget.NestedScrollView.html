Understanding android.support.v4.widget.NestedScrollView

The android.support.v4.widget.NestedScrollView class is a type of FrameLayout which allows one to fill its content with a scrolling view that has multiple child views. It is the same as a ViewGroup but with an extra layer of functionality to allow nested scrolling within it. The NestedScrollView class helps to optimize performance when having multiple views and widgets within a single layout. It also helps to ensure smooth scrolling, even when there are more than a few views in the layout.

Using NestedScrollView in Code

We can use the NestedScrollView by adding the following code to our layout file:


          
           
<android.support.v4.widget.NestedScrollView 
    android:layout_width="match_parent" 
    android:layout_height="match_parent">
 
    <LinearLayout 
        android:orientation="vertical"
        android:layout_width="match_parent"
        android:layout_height="match_parent">
 
    <!-- All the child views here-->
 
    </LinearLayout>
</android.support.v4.widget.NestedScrollView>

In the above code, the NestedScrollView is the parent view which encapsulates the LinearLayout. All the child views that need to be scrolled inside the NestedScrollView must be direct children of the LinearLayout.

The NestedScrollView also has some useful attributes that can be configured like:


          
           
scrollbars="none" // To hide the ScrollBars
fillViewport="true" // To make the NestedScrollView occupy the entire screen
nestedScrollingEnabled="true" // To enable scrolling within the NestedScrollView
