Introduction to ApiImplicitParam

The io.swagger.annotations.ApiImplicitParam annotation is used in the Swagger framework to define a single parameter in an API operation. It can be used to describe parameters that are sent in the request body or as part of a query string, or those that are accepted as part of a response. The annotation describes the type, format and other meta-data about the parameter. This can help developers understand how the API works and how to use it effectively.

Anatomy of an ApiImplicitParam Annotation

An ApiImplicitParam annotation is made up of several parts:
  • name: the name of the parameter. It must match the name of the parameter in the method signature.
  • value: a brief description of the parameter.
  • dataType: the data type of the parameter, e.g. String, Integer.
  • required: a boolean value that indicates whether the parameter is required or optional.
  • paramType: the type of parameter, e.g. query, path, body.
  • allowableValues: a set of allowable values for the parameter.
