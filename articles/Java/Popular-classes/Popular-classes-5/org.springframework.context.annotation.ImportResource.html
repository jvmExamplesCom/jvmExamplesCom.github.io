org.springframework.context.annotation.ImportResource

In Spring Framework, org.springframework.context.annotation.ImportResource is an annotation that allows specifying an XML config file to be imported into a Java configuration class. The XML config file may contain
elements, or elements from any valid namespace. The ImportResource annotation is commonly used to import a Spring XML file into a configuration class.

Examples

The following example shows a Spring config file containing two beans, with the beans' definition being imported into an annotation-based configuration class.


          
           
<beans xmlns="http://www.springframework.org/schema/beans">
    <bean id="beanA" class="com.example.MyBeanA" />
    <bean id="beanB" class="com.example.MyBeanB" />
</beans>

The following example shows a configuration class that imports and uses the beans defined in the previous example.


          
           
@Configuration
@ImportResource("classpath:/my-config.xml")
public class MyConfig {

    @Autowired
    private MyBeanA beanA;

    @Autowired
    private MyBeanB beanB;

    //...
}

In this example, the XML config file is imported using the 

@ImportResource
annotation. The beans defined in the config file can then be injected into the configuration class using the 
@Autowired
annotation. The 
@ImportResource
annotation supports wildcards for importing multiple XML config files. The following example shows a configuration class importing all XML config files in the classpath. 

          
           
@Configuration
@ImportResource("classpath*:/*.xml")
public class MyConfig {
    //...
}
The 
@ImportResource
annotation also supports importing resources from the file system, as shown in the following example. 

          
           
@Configuration
@ImportResource("file:/configs/*.xml")
public class MyConfig {
    //...
}

Conclusion

The 
@ImportResource
annotation is a powerful way to import XML configurations into an annotation-based configuration class. It supports importing from the classpath, from a file system, and supports wildcards for importing multiple resources. This makes it very convenient for working with XML configurations in Spring.
