org.springframework.boot.test.autoconfigure.web.servlet.AutoConfigureMockMvc

The org.springframework.boot.test.autoconfigure.web.servlet.AutoConfigureMockMvc annotation enables the use of MockMvc from the convenience of an application context. It can be used to perform integration tests for applications that use the Spring MVC framework. This annotation creates a mock web environment in which requests can be sent and responses verified.

Example Usage

The following example shows how the AutoConfigureMockMvc annotation is used in an application test.


          
           
@RunWith(SpringRunner.class)
@SpringBootTest
@AutoConfigureMockMvc
public class MySpringMvcAppTest {

    @Autowired
    private MockMvc mockMvc;

    @Test
    public void getHomePage() throws Exception {
        mockMvc.perform(get("/"))
            .andExpect(status().isOk())
            .andExpect(view().name("home"));
    }

}

The AutoConfigureMockMvc annotation will configure a MockMvc instance and inject it into the test. This can then be used to send requests and verify responses.

The annotation can be used with either the JUnit @RunWith or the SpringJUnit4ClassRunner. The annotation also requires that the @SpringBootTest annotation is used, so that the Spring Boot test context can be configured.
