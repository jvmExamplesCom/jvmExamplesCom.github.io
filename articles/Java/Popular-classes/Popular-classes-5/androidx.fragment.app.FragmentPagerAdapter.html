androidx.fragment.app.FragmentPagerAdapter

androidx.fragment.app.FragmentPagerAdapter is a subclass of PagerAdapter that is used to create and manage fragments that are associated with a ViewPager. It is a convenient way to provide swiping between pages of tabs, without having to manually manage the Fragments attached to the ViewPager.

The FragmentPagerAdapter class simplifies how you interact with the ViewPager; it takes care of adding, removing and updating the Fragments automatically when you make changes to the ViewPager. The FragmentPagerAdapter requires you to implement a getItem() method, which returns the fragment at the current position.

Example Code

Below is an example of a FragmentPagerAdapter implementation:


          
           
public class SampleFragmentPagerAdapter extends FragmentPagerAdapter {

  public SampleFragmentPagerAdapter(FragmentManager fm) {
    super(fm);
  }

  @Override
  public Fragment getItem(int position) {
    // Return a new fragment depending on the position
    switch (position) {
      case 0:
      return new SampleFragmentOne();

      case 1:
      return new SampleFragmentTwo();

      case 2:
      return new SampleFragmentThree();

      default:
      return null;
    }
  }

  @Override
  public int getCount() {
    return 3;
  }
}

Once you've implemented your FragmentPagerAdapter, you need to set it up in your Activity or Fragment:


          
           
mViewPager.setAdapter(new SampleFragmentPagerAdapter(getSupportFragmentManager()));

And that's it! Now your activity will be able to swipe between the three Fragments you've added. This is a simple example of how to use the FragmentPagerAdapter, but there are more advanced features it can provide, such as customizing the transition animations between pages.
