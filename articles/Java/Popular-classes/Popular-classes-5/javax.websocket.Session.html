javax.websocket.session

A javax.websocket.Session object is the basis for a WebSocket connection between two endpoints, like a client and a server, or two peers. It provides access to messages that are exchanged, as well as methods to send and receive data. This article will provide an overview of the Session class and how to use it with code examples.

Creating a Session

Creating a session is relatively simple, for example:


          
           
Session session = new Session();

This can be done with a configuration file which will define various aspects, such as security settings, or with a set of parameters passed in as arguments.

Using the Session Object

The Session class provides several methods for sending and receiving data. To send data, you can use 

sendTextMessage()
, which takes as an argument the text message you wish to send. Similarly, 
sendBinaryMessage()
can be used to send binary messages, and 
sendObjectMessage()
can be used to send serializable objects. 

          
           
// Sending a text message 
String textMessage = "Hello World!";
session.sendTextMessage(textMessage);

// Sending a binary message
byte[] binaryMessage = new byte[1024];
session.sendBinaryMessage(binaryMessage);

// Sending an object message
MySerializableObject obj = new MySerializableObject(); 
session.sendObjectMessage(obj);
To receive messages, the 
getMessage()
method is used. It will return the message which corresponds to the message type that is specified, which can be text, binary, or an object. 

          
           
Message message = session.getMessage();

Closing a Session

When a Session is no longer needed, it should be closed. The 

close()
method can be used to close a Session: 

          
           
session.close();

The Session will be closed gracefully, with any pending data sent before the connection is terminated.

Conclusion

The javax.websocket.Session class provides the basis for a WebSocket connection, and can be used for sending and receiving data. It is relatively simple to use, and by following the examples provided in this article, you should be able to get a connection up and running quickly.
