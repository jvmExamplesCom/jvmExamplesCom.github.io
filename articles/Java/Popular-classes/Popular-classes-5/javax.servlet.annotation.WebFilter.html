javax.servlet.annotation.WebFilter

Java Servlet technology provides web components that can respond to client requests and generate dynamic content. WebFilters are special kinds of servlets that are used to intercept requests from clients before they get processed by the resources (e.g. servlets, JSP pages) of the application. They can be used for logging, authentication, request encryption, etc. The javax.servlet.annotation.WebFilter annotation is one of the convenient method to configure a WebFilter in a web application.

Example

The following example shows how to define a WebFilter using the javax.servlet.annotation.WebFilter annotation: 

          
           
@WebFilter(urlPatterns = "/*", initParams = { @WebInitParam(name="foo", value="bar") })
public class ExampleFilter implements Filter {
    // implementation goes here
}
This annotation declares a filter named ExampleFilter which will be invoked for all requests in the application (specified by /* URL pattern). The initParams attribute enables you to initialize the filter with input values. In this example, we are passing an init param with the name foo and value bar.

Life Cycle of WebFilter

A filter has a life cycle similar to that of a servlet. It follows the following steps:
  • The web container loads and instantiates the filter object.
  • The filter object is initialized with init() method.
  • For each incoming request, the doFilter() method of the filter is invoked. This method does the actual work of filtering the data.
  • Finally, the destroy() method is called to clean up the resources acquired by the filter.

Methods of WebFilter

The javax.servlet.annotation.WebFilter annotation contains the following methods:
  • dispatcherTypes: Specifies the dispatcher types of the filter. Supported dispatcher types are Request, Forward, Include, and Error.
  • filterName: The filter name. If not specified, the filter class name (with the first letter in lowercase) is used as the filter name.
  • initParams: Used to specify initialization parameters that will be passed to the filter object.
  • urlPatterns: Specifies the URL patterns that the filter should be applied to.

Examples of Using Filters with Annotations

In the following example, we defined a filter with the name LoggingFilter to log request information and filter it based on the URL pattern. 

          
           
@WebFilter(filterName = "LoggingFilter", urlPatterns = "/*")
public class LoggingFilter implements Filter {

    public void doFilter(ServletRequest request, 
            ServletResponse response, 
            FilterChain chain) {
        // Log the request
        System.out.println("LoggingFilter => " + 
                request.getRemoteAddr());
        // Process request
        chain.doFilter(request, response);
    }

    public void init(FilterConfig filterConfig) {}
    public void destroy() {}
}
In the following example, we defined a filter with the name AuthenticationFilter to authenticate user requests. 

          
           
@WebFilter(filterName = "AuthenticationFilter", 
        urlPatterns = "/*",
        initParams = { @WebInitParam(name="username", value="admin"), 
                        @WebInitParam(name="password", value="secret")})
public class AuthenticationFilter implements Filter {

    public void doFilter(ServletRequest request, 
            ServletResponse response, 
            FilterChain chain) {
        String username = 
                ((HttpServletRequest)request).getParameter("username");
        String password = 
                ((HttpServletRequest)request).getParameter("password");
        if (username.equals("admin") &amp;&amp; password.equals("secret")) {
            // Process request
            chain.doFilter(request, response);
        }
        else {
            HttpServletResponse httpResponse = 
                    (HttpServletResponse)response;
            httpResponse.sendError(403);
        }
    }

    public void init(FilterConfig filterConfig) {}
    public void destroy() {}
}

Conclusion

The javax.servlet.annotation.WebFilter annotation is a convenient way to configure a WebFilter in a web application. It can be used to add logging, authentication and other features to your application. A filter has a life cycle similar to that of a servlet and contains several methods to specify initialization parameters, filter name and URL patterns.
