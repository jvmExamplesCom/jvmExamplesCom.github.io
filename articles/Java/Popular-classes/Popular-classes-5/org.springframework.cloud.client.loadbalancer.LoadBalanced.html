org.springframework.cloud.client.loadbalancer.LoadBalanced

The org.springframework.cloud.client.loadbalancer.LoadBalanced is an interface used to annotate a RestTemplate object for use in application that relies on spring cloud load balancing.

Usage

Annotating a RestTemplate with LoadBalanced allows the RestTemplate to use Ribbon with the default configuration settings which includes round robin load balancing.


          
           
@LoadBalanced 
@Bean
public RestTemplate restTemplate() {
    return new RestTemplate();
}

Configuration Settings

By default, the round robin load balancing strategy is used. This can be overridden, for example, to use a random strategy like this:


          
           
@LoadBalanced 
@Bean
public RestTemplate restTemplate() {
    String loadBalancingStrategy = "Random";
    return new RestTemplate(new RibbonLoadBalancingStrategy(loadBalancingStrategy));
}

In addition to the strategy, other configuration settings such as hostname verification can be customized and specified in the RibbonLoadBalancerClient properties as attributes in the @Bean annotation:


          
           
@LoadBalanced 
@Bean
public RestTemplate restTemplate() {
    String loadBalancingStrategy = "Random";
    Map<String, Object> ribbonProperties = new HashMap<>();
    // Configure properties here
    ribbonProperties.put("ribbon.hostnameVerificationEnabled", false);
    return new RestTemplate(new RibbonLoadBalancingStrategy(loadBalancingStrategy, ribbonProperties));
}

Example

The following example is a sample application that uses the LoadBalanced annotation:


          
           
@RestController
@Slf4j
public class MyController {
    @Autowired
    private RestTemplate restTemplate;
  
    @RequestMapping(value = “/”)
    public String example() {
        String response = restTemplate.getForObject(“http://example-service/foo”, String.class);
        log.info(“Response: {}”, response);
        return response;
    }
}
