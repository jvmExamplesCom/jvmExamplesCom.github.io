org.elasticsearch.client.Client

org.elasticsearch.client.Client is a Java client for connecting to an Elasticsearch cluster. The Client class provides a high-level interface for interacting with the cluster and performing operations like indexing, searching, and updating mappings. It also provides more powerful features like scrolling, scanning, and paging.

Creating a Client Instance

To get started, you need to create a Client instance. Here's an example of how to do that:


          
           
Settings settings = Settings.builder()
    .put("cluster.name", "mycluster")
    .build();

TransportClient client = new PreBuiltTransportClient(settings)
    .addTransportAddress(new InetSocketTransportAddress(InetAddress.getByName("localhost"), 9300));

The first line sets up a Settings object to configure the connection. The second line creates an instance of the TransportClient and adds the localhost address as an address to search for nodes in the cluster.

Indexing Data

Once you have a Client instance, you can start to index documents in Elasticsearch. Here's an example of how to do that:


          
           
IndexResponse response = client.prepareIndex("index_name", "type_name")
    .setSource(jsonBuilder()
        .startObject()
            .field("user", "John Doe")
            .field("postDate", new Date())
            .field("message", "Hello world!")
        .endObject())
    .execute()
    .actionGet();

This code creates an IndexRequest to index a JSON document in the index_name index and type_name type. The setSource() method sets the document content to be indexed. You can also use the bulk API to index multiple documents at once.

Searching Data

Once documents have been indexed, you can search them using the Search API. Here's an example of searching for documents that contain the term "hello":


          
           
SearchResponse response = client.prepareSearch("index_name")
    .setTypes("type_name")
    .setQuery(QueryBuilders.termQuery("message", "hello"))  
    .execute()
    .actionGet();

This code creates a SearchRequest to search for documents in the index_name index and type_name type. The setQuery() method sets the query to search for documents that contain the term "hello" in the message field. You can also use the multi-search API to execute multiple searches at once.

Updating Mappings

Elasticsearch supports dynamic mappings, which allows developers to define field mappings without having to manually update the mappings. Here's an example of how to update mappings for the type_name type in the index_name index:


          
           
PutMappingResponse response = client.admin().indices()
    .preparePutMapping("index_name")
    .setType("type_name")
    .setSource(jsonBuilder()
        .startObject()
            .startObject("properties")
                .startObject("user")
                    .field("type", "text")
                    .field("analyzer", "standard")
                .endObject()
            .endObject()
        .endObject())
    .execute()
    .actionGet();

This code creates a PutMappingRequest to update mappings for the type_name type in the index_name index. The setSource() method sets the mapping for the user field to be of type "text" and to use the "standard" analyzer. You can also use the bulk API to update multiple mappings at once.

Conclusion

In this article, we have seen how to use the org.elasticsearch.client.Client class to interact with an Elasticsearch cluster. We have looked at how to create a Client instance, how to index documents, search for documents, and update mappings. By using the Client class, developers can quickly and easily build applications that connect to and use Elasticsearch.
