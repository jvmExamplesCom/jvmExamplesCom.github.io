Java.awt.HeadlessException

Java.awt.HeadlessException is an exception that’s thrown by AWT when a graphical environment isn't accessible on the current platform. It is thrown by AWT methods when it encounters a platform that doesn’t support the operation in order to provide a way for program execution to continue.

Java.awt.HeadlessException is derived from java.lang.RuntimeException and is an unchecked exception, which means that it can be propogated out of a method when it’s thrown. It is used to allow applications and applets that rely on AWT to be executed in environments where a display isn’t available.

When is Java.awt.HeadlessException thrown?

Java.awt.HeadlessException is thrown when a method in the AWT package is invoked in a Headless environment. Some examples of when this exception is thrown include:

  • When a method such as createImage() is called from inside an Applet or Application in a headless environment.
  • When a method such as contains() is called from a MenuItem in an application that is running in a headless environment.

In these cases, AWT will throw a HeadlessException to allow program execution to continue without crashing due to a lack of display or input devices.

Example of Java.awt.HeadlessException

The following example demonstrates when a Java.awt.HeadlessException is thrown. In this example, we are calling createImage() from inside an Applet in a headless environment.


          
           
import java.awt.*;

public class HeadlessApplet extends Applet {
  public void init() {
    try {
      Image img = createImage(200, 200);
    }
    catch (AWTError e) {
      System.out.println("Caught HeadlessException: " + e.getMessage());
    }
  }
}

When we run this code in a headless environment, we’ll get the following output:


          
           
Caught HeadlessException: Can't connect to X11 window server using ':0.0' as the value of the DISPLAY variable.

This shows that the createImage() method threw a HeadlessException since it cannot be used in a headless environment.

How to handle Java.awt.HeadlessException

The best way to handle a Java.awt.HeadlessException is to check whether the code is running in a headless environment before invoking any AWT methods. This can be done using the following code:


          
           
if (GraphicsEnvironment.isHeadless()) {
  System.out.println("Running in a headless environment");
}

If GraphicsEnvironment.isHeadless() returns true, then the code is running in a headless environment and any AWT methods should not be invoked. If it returns false, then the code can use any AWT methods without worrying about throwing a HeadlessException.

It is also possible to wrap calls to methods that may throw a HeadlessException in try-catch blocks. However, it should be noted that this approach does not prevent the exception from being thrown if the code is running in a headless environment. The only sure way to avoid the exception is to check for a headless environment before invoking any AWT methods.
