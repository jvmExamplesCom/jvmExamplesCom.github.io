org.openqa.selenium.support.ui.WebDriverWait

Introduction

The org.openqa.selenium.support.ui.WebDriverWait class provides a convenient way to wait for a condition to be fulfilled before proceeding further with the test case execution. It is an implementation of the Wait interface and can be used to wait until a certain condition is met. By default, WebDriverWait will poll the DOM every 500 ms to check if the condition has been met.

Creating WebDriverWait Object

WebDriverWait object is created using the new keyword passing two arguments: the web driver instance, and the timeout in seconds. 

          
           
WebDriverWait wait = new WebDriverWait(driver, 10);
The default polling interval is set to 500 milliseconds. If you want to set a different polling interval you can use the withTimeout() method. 

          
           
WebDriverWait wait = new WebDriverWait(driver, 10).withTimeout(5, TimeUnit.SECONDS);

Using Conditions

The WebDriverWait class provides several methods that allow you as a tester to wait for certain conditions to be fulfilled. These methods return a true or false value based on the condition being checked. The most commonly used methods are:
  • presenceOfElementLocated
  • visibilityOfElementLocated
  • elementToBeClickable
  • textToBePresentInElement

Example

Let's see how we can use the WebDriverWait class to wait for an element to be visible on the page. 

          
           
WebDriverWait wait = new WebDriverWait(driver, 10);
WebElement element = wait.until(ExpectedConditions.visibilityOfElementLocated(By.xpath("//input[@name='test']")));
This code will create a WebDriverWait object, and wait up to 10 seconds for the element located by the provided xpath to be visible on the page. If the element is not visible within 10 seconds, an exception will be thrown and the test will fail.

Conclusion

The WebDriverWait class provides a convenient way to wait for certain conditions to be met before continuing with the next step of your test. It is an implementation of the Wait interface, and allows you to set a specific polling interval as well as using a variety of conditions to wait for.
