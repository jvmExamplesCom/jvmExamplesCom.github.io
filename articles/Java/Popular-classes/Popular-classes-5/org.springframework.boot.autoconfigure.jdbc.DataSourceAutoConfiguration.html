org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration

The org.springframework.boot.autoconfigure.jdbc.DataSourceAutoConfiguration class provides auto-configuration for a data source in the Spring Boot application. This auto-configuration class activates when the application context includes a DataSource bean and at least one other bean marked with @ConfigurationProperties annotation.

Code Examples

Let's look at an example of how to configure a DataSource in the Spring Boot application.


          
           
@Configuration
public class DataSourceConfiguration {  
    
    @Bean
    public DataSource dataSource() {
      return new EmbeddedDatabaseBuilder()
        .setType(EmbeddedDatabaseType.H2)
        .build();
    }
}

The above code creates an embedded database using H2. We can also use other types of databases by changing the type.

We can also use the DataSourceAutoConfiguration class to configure a data source. The following example shows how to do that:


          
           
@SpringBootApplication
@EnableAutoConfiguration(
exclude = { DataSourceAutoConfiguration.class })
public class MyApplication {  

  //Spring boot application main() method

}

By default, the DataSourceAutoConfiguration class enables auto-configuration of the data source. To disable it, we need to add the above annotation to the main application class.

We can also use different configuration properties to customize the auto-configured data source. For example, the following code configures the data source URL, username, and password.


          
           
spring.datasource.url=jdbc:h2:~/test
spring.datasource.username=sa
spring.datasource.password=

The DataSourceAutoConfiguration class also supports customizing the connection pooling parameters. For example, the following configuration properties can be used to configure the minimum, maximum and initial size of the connection pool.


          
           
spring.datasource.hikari.min-pool-size=10
spring.datasource.hikari.max-pool-size=20
spring.datasource.hikari.initial-pool-size=15

Finally, the DataSourceAutoConfiguration class also provides support for setting up the validation query and test-on-borrow configuration properties.


          
           
spring.datasource.validation-query=SELECT 1
spring.datasource.test-on-borrow=true
