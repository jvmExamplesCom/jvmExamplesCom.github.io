io.netty.handler.codec.http.HttpMethod

The io.netty.handler.codec.http.HttpMethod class provides an enum that defines all the different types of HTTP request methods which can be used while communicating over the Hypertext Transfer Protocol (HTTP). The different types of requests are GET, POST, HEAD, OPTIONS, PUT, DELETE, TRACE, and PATCH.

Methods

There are a number of HTTP methods that are available for use:

  • GET
    - Used to retrieve information from the server
  • POST
    - Used to send new information to the server
  • HEAD
    - Used to retrieve the header information of a resource without the response body
  • OPTIONS
    - Used to determine the capabilities of the server
  • PUT
    - Used to store a new resource on the server
  • DELETE
    - Used to delete a resource from the server
  • TRACE
    - Used to trace the path of a request from one end to the other
  • PATCH
    - Used to modify an existing resource on the server

Example

The following example illustrates how to use the io.netty.handler.codec.http.HttpMethod class to set the method of a request:


          
           
DefaultFullHttpRequest request = new DefaultFullHttpRequest(
    HttpVersion.HTTP_1_1,
    HttpMethod.POST,
    "/foo");

HttpMethod.POST
is used to set the method to
