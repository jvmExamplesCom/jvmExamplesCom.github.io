Rx.Subjects.PublishSubject

Rx.Subjects.PublishSubject is an implementation of the Rx.Subjects interface in the ReactiveX library. It is a type of subject that only publishes the last item it received before its subscription. In this way, it acts as a one-time observer that waits for the next item in the sequence and publishes it.

A PublishSubject can be used in any scenario where you'd want to cache the last item in a sequence for quick access. This could include monitoring changes in a file or data structure, or updating a UI in reaction to a user action.

Let’s take a look at how we can use PublishSubject with an example.

Example Usage

In this example, we’ll open a PublishSubject and use it to write the last item of an array to the console whenever it is updated.


          
           
let subject = new Rx.Subjects.PublishSubject();

let stream = Rx.Observable.from([1, 2, 3])
  .subscribe(subject);

subject.subscribe(val => {
   console.log(val);
});

// Output: 3

Here we create a PublishSubject and subscribe it to a stream of numbers. Whenever the stream is updated, the PublishSubject will publish the last number to its subscribers. In this case, we’ve subscribed the subject to the console and will print out the last item of the array (3) when it is updated.

This example is quite simple but PublishSubject can be used in a number of interesting ways. We can use it to turn a stream of data into a “push” mechanism, where items are pushed to downstream subscribers whenever the source or underlying data is updated.


          
           
let subject = new Rx.Subjects.PublishSubject();

let stream = Rx.Observable.from([1, 2, 3])
  .subscribe(subject);

subject.subscribe(val => {
   doSomething(val) 
})

// doSomething is called each time the subject publishes a new value

In this example, we’ve created a PublishSubject and subscribed it to a stream of numbers. Whenever the stream is updated, the PublishSubject will publish the last number to its subscribers. We’ve subscribed a function to the subject and so it will be called each time the subject publishes a new value.

If we subscribe multiple times to the subject, the Promise will only be called once and the data will be cached. In this way, we can limit the amount of network requests our application is making.
