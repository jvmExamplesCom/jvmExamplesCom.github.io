org.apache.log4j.PatternLayout

The org.apache.log4j.PatternLayout class is a key component of the Log4j logging framework that enables developers to write their log messages in a customizable and flexible format. The PatternLayout allows developers to specify the pattern of each log statement, including the order, type, and length of fields. The PatternLayout offers two different approaches for formatting: one for single-line log statements and another for multiline statements.

Simple Formatting

The simplest form of PatternLayout consists of a single line of text with placeholders for the fields that will be output. Each placeholder is represented by the % symbol followed by a number or letter (from 0 to 9 or from A to Z) that denotes the field that should be printed in that position. For example, the following PatternLayout: 

          
           
%d %p %l - %m%n
Will print out log messages in the following format: 

          
           
12/07/2017 12:00:00pm INFO com.example.MyClass - This is a log message
In this example, the placeholder %d will be replaced by the date, the %p will be substituted with the log level, %l will be replaced by the location of the log statement, and %m will be replaced by the actual log message. The %n at the end indicates that a line break should be added after the log statement.

Complex Formatting

%d{dd/MM/yyyy HH:mm:ssa} %p %l - %m%n
will produce the same result as the simple format above. Notice that the date placeholder %d was changed to include a specific date format. This allows developers to fully customize the format of the log messages.
