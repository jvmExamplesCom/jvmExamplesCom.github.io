javax.servlet.http.Part

The javax.servlet.http.Part interface is part of the Java Servlet API, and provides access to a file contained within a 

multipart/form-data
request body. It is commonly used as part of a form submission, and can be retrieved from within the server-side code handling the form submission. In this article, we will look at various ways of handling form data when using the javax.servlet.http.Part interface.

Retrieving the Part

A Part object can be retrieved from a request using the method 

getPart()
. The request must include a header of 
multipart/form-data
. For example: 

          
           
String name = request.getParameter("name");
Part part = request.getPart("file");
The 
name
parameter must match the 'name' attribute of the 

          
           
<input type='file'>
element of the HTML form. The Part object returned will contain the contents of the file specified in the form.

Retrieving Attributes

The Part interface contains several methods for retrieving attributes associated with the file. These include the file name (


          
           
getSubmittedFileName()
), content type ( 
getContentType()
), and size ( 
getSize()
). 

          
           
String fileName = part.getSubmittedFileName();
String contentType = part.getContentType();
long fileSize = part.getSize();

Writing to File

The contents of a Part can be written to a file using the method 


          
           
write(String fileName)
. This will write the contents of the Part to the specified file. For example: 

          
           
String savePath = "/path/to/save/to";
String fileName = part.getSubmittedFileName();

In addition to the 
write()
method, the Part also includes the methods 
getInputStream()
and 
getReader()
, which can be used to read the contents of the Part as a stream or reader, respectively.
