An Introduction to com.nineoldandroids.animation.Animator

The com.nineoldandroids.animation.Animator class allows developers to compose animations that can be easily applied to their applications. This section of article will discuss the various capabilities and usage of this powerful animation class.

Creating an Animation

Using the Animator class, developers can create an animation by defining a set of values for a given property. For example, the following code creates an animation that sets a View's alpha value from 0.0 to 1.0 over a 1000 millisecond period: 

          
           
Animator animator = ObjectAnimator.ofFloat(view, "alpha", 0.0f, 1.0f);
animator.setDuration(1000);
animator.start();
The animation created by this code will cause the given view to slowly become visible.

Animator Listeners

Often, developers may wish to respond to events occuring during an animation. This can be accomplished through the use of Animator Listeners. Listeners can be registered on an Animator object and will be called when certain events occur. For example, the following code registers a listener that logs a message when the animation starts: 

          
           
animator.addListener(new AnimatorListenerAdapter() {
    @Override
    public void onAnimationStart(Animator animation) {
        Log.d(TAG, "Animation started!");
    }
});

Value Animators

In addition to Animator objects, developers can also make use of Value Animators. These are objects which can be used to animate properties of arbitrary types. They are more flexible than Animator objects, as they can be used to animate any kind of value. For example, the following code creates a Value Animator which animates between two arbitrary color values: 

          
           
ValueAnimator colorAnimator = ValueAnimator.ofObject(new ArgbEvaluator(),
        Color.RED, Color.BLUE);
colorAnimator.setDuration(1000);
colorAnimator.addUpdateListener(new ValueAnimator.AnimatorUpdateListener() {
    @Override
    public void onAnimationUpdate(ValueAnimator animation) {
        view.setBackgroundColor((int) animation.getAnimatedValue());
    }
});
colorAnimator.start();

Interpolators

Animator and Value Animator objects can also have interpolators applied to them. Interpolators modify the rate at which an animation progresses over time. For example, the following code creates an animation with an AccelerateDecelerateInterpolator, which causes the animation to accelerate and decelerate as it progresses: 

          
           
ObjectAnimator animator = ObjectAnimator.ofFloat(view, "alpha", 0.0f, 1.0f);
animator.setDuration(1000);
animator.setInterpolator(new AccelerateDecelerateInterpolator());
animator.start();

Conclusion

The com.nineoldandroids.animation.Animator class provides a powerful set of tools for creating flexible and rich animations in Android applications. Its capabilities range from simple Animators to complex Value Animators and Interpolators. With these tools, developers can easily create animations that bring life to their applications.
