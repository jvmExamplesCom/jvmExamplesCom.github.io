javax.ws.rs.core.SecurityContext

The javax.ws.rs.core.SecurityContext class provides access to security related information that is relevant to request/response message exchanges in the scope of a single JAX-RS request invocation. This is an abstract class and should not be used directly. Instead, an implementation of this class must be provided by an application or an application server.

The SecurityContext interface provides methods to retrieve information about the user associated with the current request, whether the user is authenticated, the authentication scheme in use and the associated principal, and to indicate if the user is authorized to access particular resources.

Example

The following example shows how to implement the SecurityContext interface.


          
           
public class MySecurityContext implements SecurityContext 
{
  private final String username;
  private final boolean secure;

  public MySecurityContext(String username, boolean secure) 
  {
    this.username = username;
    this.secure = secure;
  }

  @Override
  public Principal getUserPrincipal() 
  {
    return new Principal() {
      @Override
      public String getName() 
      {
        return username;
      }
    };
  }

  @Override
  public boolean isSecure() 
  {
    return secure;
  }

  @Override
  public boolean isUserInRole(String role) 
  {
    return false;
  }

  @Override
  public String getAuthenticationScheme() 
  {
    return null;
  }
}

Methods

The javax.ws.rs.core.SecurityContext class provides the following methods:


          
           
getUserPrincipal(): Return a java.security.Principal object containing the name of the current authenticated user.
isSecure(): Return true if the request was made using a secure connection; otherwise, return false.
isUserInRole(String role): Return true if the user making the request belongs to the specified role; otherwise, return false.
getAuthenticationScheme(): Return the authentication scheme used to authenticate the user, such as "BASIC" or "FORM".

The SecurityContext interface also defines the set of methods a SecurityContext implementation must support. These include the isUserInRole() and the getAuthenticationScheme() methods.
