org.elasticsearch.action.index.IndexRequest

The org.elasticsearch.action.index.IndexRequest class is an ActionRequest class used to store data in an Elasticsearch index. It encapsulates the data itself as well as the metadata associated with it, such as the type of data being stored, the index name, and various other settings. This class is typically used when adding new documents to an Elasticsearch index.

Constructing an Index Request

To construct an IndexRequest, you must provide two elements: the JSON document to be indexed, and the index name. The index name can either be the name of an existing Elasticsearch index, or a new one. The document should be serialized in JSON format.


          
           
IndexRequest request = new IndexRequest("my_index")
    .source(myJsonDocument);

Setting Metadata

In addition to the required elements, you can also set various metadata-related options on the IndexRequest. These are commonly used to customize the behavior of the index operation.

Type and Id

You can specify the type and id of the document you are indexing, if known.


          
           
request.type("document_type")
        .id("document_id");

Timeouts

You can specify timeout values for the operation to ensure it is not blocked for too long.


          
           
request.timeout(TimeValue.timeValueMinutes(2))
        .timeout("2m");

Operation Type

It's possible to specify the type of operation that should be performed when indexing the document. By default, this is "index". Valid values are "index", "update", and "create".


          
           
request.opType("create");

Executing the Request

Once the request is constructed, it can be executed using an appropriate ActionRequestExecutor. Typically, this will be an instance of IndexAction class.
