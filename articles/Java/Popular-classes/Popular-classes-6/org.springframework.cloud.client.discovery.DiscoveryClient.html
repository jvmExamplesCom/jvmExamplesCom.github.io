Introduction to org.springframework.cloud.client.discovery.DiscoveryClient

The DiscoveryClient is an interface part of the Spring Cloud project designed to provide a consistent abstraction over a number of different communication mechanisms to allow service discovery in a cloud environment. This way, services can communicate with each other without having to be aware of their individual locations.

Using DiscoveryClient to Lookup Services

The DiscoveryClient interface provides the lookup method which allows for looking up services using their service ID. This can be used in combination with the getInstances method to return information about individual services that have been discovered.


          
           
DiscoveryClient client = ...
List<ServiceInstance> instances = client.getInstances("my-service");

The ServiceInstance object contains methods to access things like the host name and port of the discovered service.


          
           
String hostname = instance.getHost();

Registering Services With DiscoveryClient

DiscoveryClient also provides the registerServiceInstance method which can be used to register a service with the DiscoveryClient. This needs to be called whenever a service wants to make itself discoverable.


          
           
DiscoveryClient client = ...
ServiceInstance instance = new DefaultServiceInstance("service-instance-123", "my-service", "localhost", 8081, false);
client.registerServiceInstance(instance);

The parameters for the constructor represent the service instance ID, service ID, hostname, port and secure indicator.

Unregistering Services From DiscoveryClient

It is also possible to unregister a service with the DiscoveryClient. This should be done when a service no longer wishes to be discoverable by other services. The unregisterServiceInstance method can be used for this, passing in the same ServiceInstance object that was used for registering.


          
           
DiscoveryClient client = ...
ServiceInstance instance = new DefaultServiceInstance("service-instance-123", "my-service", "localhost", 8081, false);
client.unregisterServiceInstance(instance);
