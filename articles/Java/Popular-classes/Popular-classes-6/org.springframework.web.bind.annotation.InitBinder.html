org.springframework.web.bind.annotation.InitBinder

The 
@InitBinder
annotation is used to bind fields on a controller class to custom WebDataBinder instances. It enables us to use special types of data binding, such as type conversion and other custom data binding.

Example

Let's say we have a Spring MVC form with a field called 'name' which is an input text box. 


          
           
<form:input path="name" />
We can use the 
@InitBinder
annotation on a method in our controller to configure a 
WebDataBinder
for this field. 

          
           
@InitBinder
public void initBinder(WebDataBinder binder) {
    binder.registerCustomEditor(String.class, new StringTrimmerEditor(true));
}

This will register a custom editor that trims whitespace from the value in the 

name
field before it is bound to the controller.

Using Custom Binders

The 
@InitBinder
annotation also allows us to use custom binders. For example, let's say we have a domain object, 
Person
, and we want to use a custom binder that binds the 
name
field of the 
Person
object using a custom 
NameBinder
. 

          
           
public class Person {
    private String name;
    // Getters and setters omitted
}

          

          
           
public class NameBinder extends PropertyEditorSupport {

    @Override

We can now register this custom binder in our controller like this:


          
           
@InitBinder
public void initBinder(WebDataBinder binder) {
    binder.registerCustomEditor(Person.class, new NameBinder());

Conclusion

The 
@InitBinder
annotation allows us to do custom data binding and type conversion on fields in our Spring MVC controllers. We can register custom editors to do type conversion and custom binders to bind complex objects. This makes it easier to work with complicated forms and data models in our web applications.
