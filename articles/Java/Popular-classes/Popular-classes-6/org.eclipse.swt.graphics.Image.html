Introduction to org.eclipse.swt.graphics.Image

The org.eclipse.swt.graphics.Image class is part of the Standard Widget Toolkit (SWT) of the Eclipse software platform and provides a high-level graphics programming interface for developers creating graphical user interfaces (GUIs). In this article, we will look closer at the Image class and examine some of the most commonly used methods and features that can be leveraged when using the class.

Creating an Image Object

The most basic way of creating an Image object is done by using the Image constructor: 

          
           
Image image = new Image(display, "myimage.gif");
This creates an Image object with the width and height of the image defined by the myimage.gif file. This can be used to simply load a GIF file into the program, however, the following code samples show some more advanced ways of creating an Image object. 

          
           
int[] data = new int[64*64]; // Create array of 64x64 pixels
ImageData imageData = new ImageData(64,64,1,new PaletteData(0xFF, 0xFF00, 0xFF0000),32,data);
The above code defines an ImageData object and assigns it the attributes of a 64x64 pixel image, with a color depth of one bit and a palette composed of red, green and blue values. The final parameter of 32 specifies the number of bits of each pixel. Once this is created, the resulting ImageData object can be used as an argument for the Image constructor: 

          
           
Image image = new Image(display, imageData);
This creates an Image object from the ImageData object which can then be used further in the program. Another way to create an Image object is by using two parameters of type InputStream and Display. An InputStream object contains all the data required to define an image, while the Display object serves as the host device for the Image: 

          
           
InputStream inputStream = new FileInputStream("myimage.gif");
Image image = new Image(display, inputStream);
Once again, this creates an Image object which can be used further in the program.

Disposing an Image Object

Once an Image object has been created, it must eventually be disposed using the dispose method: 

          
           
image.dispose();
The dispose method also releases any underlying system resources associated with the Image. Failing to call the dispose method when finished with the Image can lead to memory leaks, so it is important to ensure that dispose is called when no longer needed.

Drawing an Image

Scaling an Image

It is also possible to resize an image using the SWT scale method: 

          
           
Image scaledImage = new Image(display, image.getImageData().scaledTo(width, height));
The above code uses the getImageData() method to return the original ImageData object, before calling the scaledTo() method with the desired width and height parameters. This then returns a new ImageData object, which can then be used to create a new scaled Image.

Rotating an Image
