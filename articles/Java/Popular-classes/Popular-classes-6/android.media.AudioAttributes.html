Introduction to android.media.AudioAttributes

android.media.AudioAttributes is a class which allows application developers to control various audio playback characteristics for audio output that is provided via the Android system. These characteristics include the volume, source, sample rate, latency among others. It was introduced in API Level 21 (Lollipop) and opened up avenues to create richer and more dynamic audio effects and experiences.

Understanding AudioAttributes

audioAttributes are used to describe a specific type of audio setting. It allows us to set the volume, content type, flags and usage of an audio output. The following values are used to categorize audio settings:
  • Content type – music, alarm, notification, ringtone and more.
  • Flags – defines the behavior of audio when it reaches a certain volume level.
  • Usage – defines the purpose of audio.
  • Volume – the degree or intensity of the sound.

Using AudioAttributes in an App

To use audioAttributes in an app, first create an instance of android.media.AudioAttributes as shown below: 

          
           
// Create an instance of AudioAttributes        
AudioAttributes audioAttributes = new AudioAttributes.Builder()
  .setContentType(AudioAttributes.CONTENT_TYPE_MUSIC)
  .setFlags(AudioAttributes.FLAG_AUDIBILITY_ENFORCED)
  .setUsage(AudioAttributes.USAGE_MEDIA)
  .build();
In the code above, we set the ContentType to Music, the Flags to be enforced and the Usage to be Media. Once the AudioAttributes is created, we need to apply it to a MediaPlayer instance to play music as shown below: 

          
           
// Create a media player with given attributes
MediaPlayer mp = MediaPlayer.create(this, R.raw.my_music, audioAttributes, null);

// Start playing the media player
mp.start();
With this setup, the music is played with its original volume and the audioAttributes specific volume.

Conclusion

android.media.AudioAttributes is a powerful class which allows us to control various audio playback characteristics like volume, source, sample rate, etc. It helps us create richer and more dynamic audio effects and experiences in our apps.
