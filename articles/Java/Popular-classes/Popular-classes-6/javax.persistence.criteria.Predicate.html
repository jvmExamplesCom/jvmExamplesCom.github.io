javax.persistence.criteria.Predicate

The 

          
           
javax.persistence.criteria.Predicate
class is a key part of the Java Persistence API that provides a means for forming dynamic queries in an object-oriented manner. It represents a single criterion that is applied to a query and can be used to constrain the results returned from a database query. This article will discuss the use of this class, its purpose and its various methods.

Purpose

The primary purpose of the Predicate class is to provide a way to create dynamic queries in an object-oriented fashion. The class allows a programmer to specify conditions with which to filter the query results and determine which records will be included in the result set. The criteria specified by the Predicate class can be combined with other elements of the query such as sorting, aggregating, and grouping.

Methods

The Predicate class contains several methods to define criteria for a query. Below are brief summaries of each.

and()

This method combines two predicates together with a logical 

AND
operation. It takes an array of two predicates and returns a single predicate that evaluates to true when both of the predicates evaluated individually are true.

//Combine them using the and() method

These methods compare the value of a field to the value passed as a parameter and return a predicate accordingly. The

This method returns true if the field value matches one of the values in the parameter list. The parameters can either be a single array or multiple variables.
