org.kohsuke.stapler.DataBoundConstructor

The 

          
           
org.kohsuke.stapler.DataBoundConstructor
annotation is used in Jenkins plugin development to indicate a constructor that should be bound to a web request. This annotation is an alternative to using the standard JavaBeans conventions, and it provides an easier way of mapping web request parameters to the constructor parameters without coding any tedious logic.

Example

Let's look at an example of a class with a 

          
           
DataBoundConstructor
annotation: 

          
           
public class MyClass {

  private final String name;
  private final int age;

  @DataBoundConstructor
  public MyClass(String name, int age) {
    this.name = name;
    this.age = age;
  }
}
The 

          
           
@DataBoundConstructor
annotation tells Jenkins that when the user creates an instance of 
MyClass
, it should map the web request parameters to the constructor arguments. In this case, there would be two parameters: 
name
and 
age
.

Usage

The 

          
           
org.kohsuke.stapler.DataBoundConstructor
annotation should be used when the class you are creating needs to receive data from the user via a web request. This is often the case when creating plugins. The annotation makes it easier to bind web request parameters to the constructor arguments without having to write any extra code.

Limitations

The 

          
           
org.kohsuke.stapler.DataBoundConstructor
annotation can only be used for constructors. It cannot be used for methods. In addition, the annotation can only be used for simple types like strings, integers, and booleans. It cannot be used for more complex types like collections.
