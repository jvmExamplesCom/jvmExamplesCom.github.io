com.google.common.cache.RemovalListener

The com.google.common.cache.RemovalListener interface is a Java library for the Google Guava project, which provides a framework for working with caches. The interface provides a way to detect and respond to cache removals when using Google Guava caching. Removal listeners are notified when a cached entry is removed from a cache due to eviction or explicit removal.

Creating a RemovalListener

A removal listener can be created by implementing the RemovalListener interface. The implementing class must provide an implementation for the onRemoval() method, which is invoked when a cached entry is removed. The method takes three parameters: a notification object, a key, and a value. The notification object provides information about the removal, including the cause of the removal and a boolean indicating whether the removal was done explicitly or by eviction.


          
           
public class MyRemovalListener implements RemovalListener<String, Object> {

    @Override
    public void onRemoval(RemovalNotification<String, Object> notification) {
        // Handle removed entry as needed
    }
}

Using the RemovalListener

Once the removal listener has been created, it can be used when constructing a cache. This is done by specifying the removal listener when building the cache:


          
           
Cache<String, Object> cache = CacheBuilder.newBuilder()
    .removalListener(new MyRemovalListener())
    .build();

Now, every time an entry is removed from the cache (both through explicit removal or eviction), the removal listener's onRemoval() method will be invoked.

Handling Notification

When the onRemoval() method is called, it will be passed the removal notification object. This object contains information about the removal, including the removal cause and a boolean indicating whether the removal was done explicitly or by eviction. The removal cause will be one of the following values:


          
           
EXPLICIT - Removed explicitly
REPLACED - Replaced by a new entry
COLLECTED - Collected by garbage collector
EXPIRED - Expired due to ttl
SIZE - Removed to make room

It is up to the user to decide how to handle the notification object. For example, the notification object can be used to log information about the removal or perform additional cleanup tasks.

    @Override
    public void onRemoval(RemovalNotification<String, Object> notification) {
        // Log info
        System.out.println("Cache entry removed! Cause: " +
            notification.getCause());
        
        // Perform additional cleanup as needed
    }
}

With the removal listener in place, the cache is now able to detect when entries are removed, providing a convenient way to respond to removal events.
