javax.swing.ToolTipManager

The javax.swing.ToolTipManager is a class used in the Java Swing API to manage the tooltips for registered components. Tooltips are small pop-up windows that appear when a component is hovered by the mouse pointer, providing additional information about the component. The ToolTipManager class is used to specify the duration of the display and the initial delay of the component's tooltip.

The ToolTipManager class contains three methods which are used to get and set the displayed duration of the tooltip and the initial delay. These three methods are:

  • 
            
             
setDismissDelay(int)

            
  • 
            
             
setInitialDelay(int)

            
  • getDismissDelay()
The 
setDismissDelay
method takes an integer as its parameter that defines the number of milliseconds after which the tooltip will be dismissed. By default, this value is set to 4000ms. The 
setInitialDelay
method takes an integer as its parameter that defines the number of milliseconds after which the tooltip will appear. By default, this value is set to 500ms. The 
getDismissDelay
method returns the current dismiss delay in milliseconds.

Example

In this example we'll create a simple button which will display a tooltip after a 800ms delay. We'll change the dismiss delay to 1000ms.


          
           
ToolTipManager ttm = ToolTipManager.sharedInstance();
ttm.setInitialDelay(800);
ttm.setDismissDelay(1000);

JButton b1 = new JButton("Button");
b1.setToolTipText("This is a tooltip example.");

To enable the tooltip we need to register the component on which the tooltip will be displayed by calling the 

registerComponent()
method of the ToolTipManager class. 

          
           
ttm.registerComponent(b1);

We can also unregister the component from the ToolTipManager if necessary, by calling the 


          
           
unregisterComponent()
method. 

          
           
ttm.unregisterComponent(b1);
