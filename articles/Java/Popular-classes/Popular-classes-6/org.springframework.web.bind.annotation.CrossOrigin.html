org.springframework.web.bind.annotation.CrossOrigin

The CrossOrigin annotation is part of the Spring Framework's web module and allows for defining cross-origin resource sharing (CORS) rules when making web requests.

What is CORS?

Cross-origin resource sharing (CORS) is a mechanism that allows restricted resources on a web page to be requested from another domain outside the domain from which the resource originated. For example, if a web page from a domain www.example.com makes a request for an image from www.foo.com, the request will fail unless the response includes the proper 


          
           
Access-Control-Allow-Origin: *
header, indicating that the resource can be requested by any origin.

How do you use CrossOrigin?

The CrossOrigin annotation is used to define specific rules for enabling CORS requests. First, the annotation should be added to the class or method declaration:


          
           
@CrossOrigin(origins = "*")
public class MyController { 
  // ... 
}

In this code example, a CrossOrigin annotation is added to the MyController class so that a CORS policy is applied for all requests. By setting the 

origins
to 
*
, the CORS policy allows for requests from any origin. If more specific origins need to be allowed, the 
origins
attribute can be set to a list of origins that are allowed to make requests. For example: 

          
           
@CrossOrigin(origins = { "http://www.example.com", "http://www.foo.com" })
public class MyController { 
  // ... 
}
In this example, only requests from the domains 
example.com
and 
foo.com
are allowed.

How do you disable CrossOrigin?

CrossOrigin can be disabled by using the 

          
           
@CrossOrigin(enabled=false)
annotation. This will prevent CORS requests from being allowed on the controller. 

          
           
@CrossOrigin(enabled=false)
public class MyController { 
  // ... 
}

Conclusion

The CrossOrigin annotation is an important part of the Spring Framework's web module and provides a way to enable and disable CORS on specific controllers. By providing a simple, yet powerful annotation, the Spring Framework allows developers to easily enable CORS on their web applications.
