org.springframework.transaction.TransactionStatus: Introduction

The 

          
           
org.springframework.transaction.TransactionStatus
interface represents the transaction status for a transaction managed by the Spring Framework. It provides access to the underlying transaction implementation object, such as the Hibernate Transaction object. It provides methods to query the current state of the transaction and to mark it for rollback. It also provides a notion of nested transactions via savepoints.

Example: Obtaining the TransactionStatus object

TransactionStatus objects can be obtained from from the PlatformTransactionManager implementation. For example:


          
           
PlatformTransactionManager txManager = new DataSourceTransactionManager(dataSource);
TransactionDefinition txDef = new DefaultTransactionDefinition();
TransactionStatus status = txManager.getTransaction(txDef);

The above code creates a transaction manager and then obtains the 

TransactionStatus
object for the transaction.

Example: Querying the TransactionStatus object

Once you have the 
TransactionStatus
object, there are various methods available to query it. For example: 

          
           
boolean isRollbackOnly = status.isRollbackOnly();
boolean isCompleted = status.isCompleted();
boolean isNewTransaction = status.isNewTransaction();

The above code queries the state of the transaction, including whether it has been marked for rollback, whether it is completed, and whether it is a new transaction.

Example: Marking the TransactionStatus object for rollback

The 
TransactionStatus
object can also be marked for rollback. This can be done with the following code: 

          
           
status.setRollbackOnly();

The above code marks the transaction for rollback.

Example: Nested Transactions

Nested transactions can also be performed using the 

TransactionStatus
object. This is done by creating a savepoint and rolling back to it if necessary. For example: 

          
           
TransactionStatus savepoint = status.createSavepoint();
// do stuff in the transaction
if (someCondition) {
    status.rollbackToSavepoint(savepoint);
}

The above code creates a savepoint in the transaction and rolls back to it if the condition is met. This can be used to perform nested transactions.

The 

          
           
org.springframework.transaction.TransactionStatus
