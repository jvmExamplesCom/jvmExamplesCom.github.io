javax.management.InstanceAlreadyExistsException

The javax.management.InstanceAlreadyExistsException is an exception thrown when a new instance of an MBean already exists. It is a subclass of the 


          
           
javax.management.MBeanException
class.

This exception is thrown when a call is made to an MBean server’s 

createMBean()
method and an instance already exists with the same name. This prevents duplicate instances from being created which could lead to errors in JMX applications.

Example Code

The following code example shows how the javax.management.InstanceAlreadyExistsException is thrown when an MBean has already been created with the same name:


          
           
MBeanServer mbeanServer = ManagementFactory.getPlatformMBeanServer();

//Create Simple Standard MBean
ObjectName name = new ObjectName("simpleBeans:name=SimpleStandard");

// Create MBean
mbeanServer.createMBean(SimpleStandard.class.getName(), name);

// Try to re-create the same MBean
mbeanServer.createMBean(SimpleStandard.class.getName(), name);
// Throws InstanceAlreadyExistsException
In this example, the second call to 
createMBean()
will throw a javax.management.InstanceAlreadyExistsException because an instance of the SimpleStandard MBean has already been created.

Output

When the javax.management.InstanceAlreadyExistsException is thrown, the output looks like this:


          
           
javax.management.InstanceAlreadyExistsException: simpleBeans:name=SimpleStandard

This output tells us that a new instance cannot be created for the MBean named "simpleBeans:name=SimpleStandard" because this instance already exists.
