Introduction to android.hardware.camera2.CameraCaptureSession

android.hardware.camera2.CameraCaptureSession is a class in the Android framework that provides access to an existing camera capture session. The class encapsulates all the parameters and state associated with a single capture session, such as orientation, format, features, and more. It also provides methods to begin and end this session, as well as query its state.

Overview

The CameraCaptureSession class is a crucial part of the Android camera API. It contains all the information necessary to configure a camera session object, including its camera device, parameters, and other settings. Once created, it can be used for both still images and video-recording functions. During the CameraCaptureSession’s lifecycle, the user may configure it with different parameters such as resolution, picture format, and orientation. A CameraCaptureSession instance may be used to start and stop capturing images and video, query the session’s state, and issue commands to the camera device.

Example Usage

Here is an example of creating and using a CameraCaptureSession object: 

          
           
CameraManager manager = (CameraManager) getSystemService(Context.CAMERA_SERVICE);
String[] camList = manager.getCameraIdList();

// Create a session with the desired camera and parameters.
CameraCaptureSession session = manager.createCaptureSession(
                camList[0], // Camera ID
                new OutputConfiguration[] { // Output configuration
                    new OutputConfiguration(Surface.ROTATION_90),
                    new OutputConfiguration(Surface.ROTATION_180)
                },
                null, // StateCallback
                null // Handler
            );

// Start the session.
session.start();

// Do something with session…

// Stop the session.
session.stop();
In the above code snippet, the CameraManager instance is used to retrieve the list of available cameras, then the createCaptureSession() method is used to create and populate a CameraCaptureSession object. The OutputConfiguration array is used to specify the output configurations, such as the surface rotation settings. Finally, the start and stop methods are used to control the session’s state.

Conclusion

The android.hardware.camera2.CameraCaptureSession is an important part of the Android camera API that provides access to an existing camera capture session. The CameraCaptureSession class encapsulates all parameters and settings related to the capture session, and it can be used to start and stop capturing images and videos. Its methods offer a way to query the session’s state and issue commands to the camera devices.
