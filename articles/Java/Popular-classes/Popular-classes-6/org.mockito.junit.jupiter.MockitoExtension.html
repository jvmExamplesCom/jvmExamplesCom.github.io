MockitoExtension JUnit 5

The Mockito Extension JUnit 5 enables us to use the popular library Mockito with JUnit 5. This extension allows us to use the annotation-based mocking capabilities that Mockito provides, allowing us to write unit tests with less boilerplate, making them more readable and easier to manage.

Maven Dependencies

In order to use Mockito with JUnit 5, we will need to include two Maven dependencies in our project:


          
           
<dependency>
    <groupId>org.mockito</groupId>
    <artifactId>mockito-junit-jupiter</artifactId>
    <version>3.3.3</version>
    <scope>test</scope>
</dependency>

<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.6.2</version>
    <scope>test</scope>
</dependency>

Annotations

Now that we have the necessary dependencies, we are ready to start writing our tests. To enable Mockito's annotation-based features, we will make use of the 

@ExtendWith
annotation on our test class, which is provided by JUnit 5. 

          
           
@ExtendWith(MockitoExtension.class)
public class MyTest {
    // ...
}
We will also be using the 
@Mock
and 
@InjectMocks
annotations provided by Mockito. The 
@Mock
annotation is used to create mock objects that can be used in our tests. The 
@InjectMocks
annotation is used to create objects that will have the mock objects injected into them. These annotations should be placed on the fields of the test class that we want to use them with. 

          
           
@ExtendWith(MockitoExtension.class)
public class MyTest {

    @Mock
    private MyService service;

    @InjectMocks
    private MyClass myClass;

    // ...
}

Typically, these tests will involve verifying that our code is correct by verifying that certain methods were called and that the results of our code were as expected.
