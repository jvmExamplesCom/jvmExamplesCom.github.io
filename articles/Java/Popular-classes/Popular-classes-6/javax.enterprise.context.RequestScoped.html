javax.enterprise.context.RequestScoped

The javax.enterprise.context.RequestScoped is a Java EE web application scope in which an application instance exists for each client request.

Understanding the Request Scoped Context

When an application is written for the Java EE platform, it is common for there to be several different classes that handle various types of requests. Each of these classes will have its own scope in which instances of the class can exist. This scope is typically referred to as the "context" of that class. The javax.enterprise.context.RequestScoped is a scope specifically designed to provide an instance of a class for each client request.

When a client makes a request to an application, the application will create a new instance of the class associated with that request. This new instance will remain active until the request is complete. Then the instance will be discarded and released. This means that each instance of a class related to a request is effectively "scoped" for the duration of that request, thus the name "Request Scoped Context".

Example of RequestScoped Usage

The following example shows how to use the RequestScoped context. It uses a dummy web application to demonstrate the use of the javax.enterprise.context.RequestScoped annotation. 


          
           
@RequestScoped
public class MyTimeService {

	public Date getTime() {
		return new Date();
	}
}

public class MyController {

	@Inject
	private MyTimeService timeService;

	public String getTime() {
		return timeService.getTime().toString();
	}
}

In the code example above, we have a class called MyTimeService. This class is annotated with the RequestScoped annotation, which indicates that it should exist within the RequestScoped context. Then we have a class called MyController which injects an instance of MyTimeService. When a request is made, a new instance of MyTimeService will be created and injected into MyController, which will be available for the duration of that request.

Conclusion

The javax.enterprise.context.RequestScoped annotation is used to create a scope in which an instance of a class will exist for the duration of a client request. This allows an application to maintain a separate instance of a class for each request, ensuring that each client request has access to its own instance of a class.
