java.awt.GraphicsConfiguration

The java.awt.GraphicsConfiguration class describes the characteristics of GraphicsDevice objects. It provides information on various aspects such as the bounds, color capabilities, and rendering and device specific properties related to Graphics2D.

Creating a GraphicsConfiguration Object

A GraphicsConfiguration object can be obtained from an existing GraphicsDevice or GraphicsEnvironment object. For example, the following code snippet obtains the current GraphicsConfiguration from the default GraphicsDevice: 


          
           
GraphicsEnvironment ge = GraphicsEnvironment.getLocalGraphicsEnvironment();
GraphicsDevice gd = ge.getDefaultScreenDevice();
GraphicsConfiguration gc = gd.getDefaultConfiguration();

Methods of GraphicsConfiguration

The GraphicsConfiguration class provides a range of methods to access and manipulate its properties. Here are some of the most important methods: 

  • getBounds()
    - Returns the bounds of this GraphicsConfiguration in the form of a Rectangle.

isTranslucencyCapable()
    - Checks if this GraphicsConfiguration is capable of translucency effects. 
  • getDevice()

getNormalizingTransform()
    - Returns the normalizing transform used to position drawing operations relative to this GraphicsConfiguration.

    - Creates a compatible Image with this GraphicsConfiguration.

Example

The following example creates a transparent Image with the specified size. The Image is created within the bounds of the current GraphicsConfiguration:

GraphicsDevice gd = ge.getDefaultScreenDevice();
GraphicsConfiguration gc = gd.getDefaultConfiguration();
