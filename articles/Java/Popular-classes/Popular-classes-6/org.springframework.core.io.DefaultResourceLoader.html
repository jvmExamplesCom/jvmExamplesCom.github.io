org.springframework.core.io.DefaultResourceLoader

The 

          
           
DefaultResourceLoader
class is a part of the Spring framework in the package of 

          
           
org.springframework.core.io
. This class provides a simple and consistent interface for resolution of resource paths. It resolves the given path and returns a corresponding Resource instance. It supports different protocols like classpath:, file:, http:, etc. It provides support for plugging in custom Resource implementations.

Example Usage

import org.springframework.core.io.Resource;

DefaultResourceLoader resourceLoader = new DefaultResourceLoader();
Resource resource = resourceLoader.getResource("classpath:/config/application.properties");

DefaultResourceLoader
object which will be used to get a Resource object. We use the 
getResource()
method to resolve the given path and return a 
Resource

getResource() Method

The 
getResource()
method takes a path as argument and resolves it according to the underlying protocols. It then returns a corresponding 
Resource
instance. 

          
           
public Resource getResource(String location) {
    Assert.notNull(location, "Location must not be null");

    if (location.startsWith(CLASSPATH_URL_PREFIX)) {
        return new ClassPathResource(location.substring(CLASSPATH_URL_PREFIX.length()), getClassLoader());
    }
    else {
        try {
            // Try to parse the location as a URL...
            URL url = new URL(location);
            return new UrlResource(url);
        }
        catch (MalformedURLException ex) {
            // No URL -> resolve as resource path.
            return getResourceByPath(location);
        }
    }
}
In the above snippet, we can see that the 
location
is first checked for the prefix 
classpath:
. If found, a 
ClassPathResource
is returned otherwise it tries to parse the location as a 
URL
. If that also fails, it tries to resolve the location as a resource path.
