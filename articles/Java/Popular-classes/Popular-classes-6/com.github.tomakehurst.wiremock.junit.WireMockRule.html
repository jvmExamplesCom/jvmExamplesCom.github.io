What is Com.github.tomakehurst.wiremock.junit.WireMockRule

Com.github.tomakehurst.wiremock.junit.WireMockRule is a JUnit rule that provides an instance of the WireMock server configured with JUnit rules. It allows for all types of tests, from simple unit tests, to more complex integration tests, to be written and executed with ease.

Using WireMockRule with JUnit

The following code example illustrates how to use WireMockRule with JUnit:


          
           
@Rule
public WireMockRule wireMockRule = new WireMockRule(options().dynamicPort());

@Test
public void exampleTest() {
    // Your test code here

    verify(getRequestedFor(urlEqualTo("/some/thing")));
}

The above example creates an instance of WireMockRule with the default options. The test then performs a verification against the WireMock server using the 

verify
method. This can be used to check that requests have been made to the server and that the expected responses were returned.

More Advanced Usage

WireMockRule also supports more advanced usage, such as setting up stubs before executing tests. The following code example illustrates how this can be done:


          
           
@Rule
public WireMockRule wireMockRule = new WireMockRule(options().dynamicPort());

@Before
public void setupStubs() {
    stubFor(get(urlEqualTo("/some/thing"))
            .willReturn(aResponse()
            .withBody("some response")));
}

@Test

}
In this example, the 
setupStubs
method is called prior to executing the test. This method sets up a stub that returns a predefined response when a request matching the given URL is made. In this case, it will return the string “some response”.
