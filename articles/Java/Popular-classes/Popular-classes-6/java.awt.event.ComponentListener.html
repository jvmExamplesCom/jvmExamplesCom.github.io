java.awt.event.ComponentListener

The java.awt.event.ComponentListener interface is used for components to receive notification of changes to their size, position, and and visibility. ComponentListener objects are registered with an instance of the Component class using the addComponentListener() method, and removed using the removeComponentListener() method.

Example 1: Register a ComponentListener


          
           
Component frame = new Component();
 frame.addComponentListener(new ComponentListener(){
    public void componentResized(ComponentEvent e) {
        // do something
    }

    public void componentMoved(ComponentEvent e) {
        // do something
    }

    public void componentShown(ComponentEvent e) {
        // do something
    }

    public void componentHidden(ComponentEvent e) {
        // do something
    }
 });

Example 2: Unregister a ComponentListener


          
           
Component frame = new Component();
 frame.removeComponentListener(new ComponentListener(){
    public void componentResized(ComponentEvent e) {
        // do something
    }

    public void componentMoved(ComponentEvent e) {
        // do something
    }

    public void componentShown(ComponentEvent e) {
        // do something
    }

    public void componentHidden(ComponentEvent e) {
        // do something
    }
 });

Example 3: Use a Component Listener


          
           
Component frame = new Component();
frame.addComponentListener(new ComponentListener(){
    public void componentResized(ComponentEvent e) {
        System.out.println("Frame resized");
    }

    public void componentMoved(ComponentEvent e) {
        System.out.println("Frame moved");
    }

    public void componentShown(ComponentEvent e) {
        System.out.println("Frame shown");
    }

    public void componentHidden(ComponentEvent e) {
        System.out.println("Frame hidden");
    }
});

In this example, we register a ComponentListener with a frame, then when any of the four events mentioned above occur, the corresponding method will be invoked and the message will be printed out to the console.
