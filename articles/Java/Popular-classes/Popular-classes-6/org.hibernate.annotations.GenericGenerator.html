org.hibernate.annotations.GenericGenerator

The 

          
           
org.hibernate.annotations.GenericGenerator
annotation is used to specify a generator used when persisting an entity with Hibernate. This annotation provides a way to customize both the name and selection of the sequence, table or other schema-level objects used by Hibernate. It helps ensure that the identifiers generated through the annotations are unique and appropriate for the database.

Example

Below is an example of how to use the 
@GenericGenerator
annotation to assign an identifier for an entity. 

          
           
@Entity
public class Person {
  @Id
  @GeneratedValue(generator = "system-uuid")
  @GenericGenerator(name = "system-uuid", strategy = "org.hibernate.id.UUIDGenerator")
  private String id;
  
  // ...
}
In the above code example, the 
@GenericGenerator
annotation defines the 
system-uuid
generator, which is based on the 

          
           
org.hibernate.id.UUIDGenerator
. This generator will generate a unique identifier for each instance of the entity.

Attributes

The 
@GenericGenerator
annotation has two attributes: 
name
and 
strategy
. The 
name
attribute is used to define the name of the generator, and the 
strategy
attribute is used to define the type of generator that is used. 
  • name
    - This attribute is required and defines the name of the generator.
  • strategy
    - This attribute is required and defines the type of generator to be used. This must refer to a valid Hibernate generator.

Configuration

The 
@GenericGenerator
annotation can also be used to customize some of the properties of the chosen generator. The following are some of the properties that can be set: 
  • parameters
    - This is an optional attribute that can be used to pass parameters to the generator.
  • scope
    - This attribute is used to define the scope of the generator (e.g. transaction, session or application).
  • initialValue
    - This attribute is used to define the initial value that will be used by the generator.
  • incrementSize
    - This attribute is used to define the size of the increment that will be used by the generator.

          
           
@GenericGenerator(name = "custom-generator",
  strategy = "org.hibernate.id.IncrementGenerator",
  parameters = {
    @Parameter(name = "initial_value", value = "1000"),
    @Parameter(name = "increment_size", value = "7")
  },
  scope = "transaction"
)
In the above example, the 
@GenericGenerator
annotation defines a custom generator with the name 
custom-generator
. This generator uses the 

          
           
org.hibernate.id.IncrementGenerator
as its strategy and is scoped to the transaction. The 
initial_value
and 
increment_size
parameters are set to 1000 and 7 respectively.

Conclusion

The 
@GenericGenerator
annotation is used to specify the generator used by Hibernate when persisting an entity. This annotation allows for the name and selection of the sequence, table or other schema-level objects used by Hibernate to be customized. Additionally, the configuration of the generator can also be customized by setting various parameters.
