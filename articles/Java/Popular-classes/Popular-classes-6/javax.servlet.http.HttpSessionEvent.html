javax.servlet.http.HttpSessionEvent

The javax.servlet.http.HttpSessionEvent class is a subclass of javax.servlet.http.HttpEvent and provides an event that is triggered when a session is created or invalidated. When a new session is created the HttpSessionEvent will trigger the HttpSessionListener’s sessionCreated() method, and when an existing session is invalidated the HttpSessionEvent will trigger the HttpSessionListener’s sessionDestroyed() method.

Constructor

The constructor for the javax.servlet.http.HttpSessionEvent class takes a single parameter that is an instance of the HttpSession. The constructor can be used to capture the details of the created or destroyed session.


          
           
HttpSessionEvent(HttpSession session)

Methods

The javax.servlet.http.HttpSessionEvent class has two main methods: getSession() and getName(). The getSession() method returns the HttpSession instance that was passed to the constructor which can then be used to access data about the session such as its attributes and creation time. The getName() method returns the name of the HttpSessionEvent.


          
           
getSession(): HttpSession 
getName(): String

Example

This example shows how to use the javax.servlet.http.HttpSessionEvent class to capture data about a session when it is created or destroyed.


          
           
public class MyHttpSessionListener implements HttpSessionListener {

    public void sessionCreated(HttpSessionEvent event) {
        // Get the session
        HttpSession session = event.getSession();

        // Output session information
        System.out.println("Session created: " + session.getId());
    }

    public void sessionDestroyed(HttpSessionEvent event) {
        // Get the session
        HttpSession session = event.getSession();

        // Output session information
        System.out.println("Session destroyed: " + session.getId());
    }
}
