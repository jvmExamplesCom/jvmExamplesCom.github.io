android.support.v4.widget.ViewDragHelper

Android's ViewDragHelper enables a user to drag and reposition views within their layout using the touch screen. This class has been available since API level 14 (Android 4.0). It is part of the Android Support Library, so it can be used on devices running any version of Android, from the oldest versions (API level 4) up to the newest versions.

Using ViewDragHelper

To use ViewDragHelper, follow these steps:
  1. Create an instance of ViewDragHelper by calling the static create() method.
  2. Next, attach the helper to the view hierarchy by calling the attachToView() method.
  3. Override the onInterceptTouchEvent(), onTouchEvent(), and computeScroll() methods.
  4. Finally, call the processTouchEvent() method in the onTouchEvent() callback to enable dragging.

Example Code

In the following example, we will use ViewDragHelper to create a draggable view.


          
           
ViewDragHelper dragHelper = ViewDragHelper.create(view, sensitivity, new DragCallback());
dragHelper.attachToView(rootView);

@Override
public boolean onInterceptTouchEvent(MotionEvent event) {
    dragHelper.shouldInterceptTouchEvent(event);
}

@Override
public boolean onTouchEvent(MotionEvent event) {
    dragHelper.processTouchEvent(event);
}

@Override
public void computeScroll() {
    if (dragHelper.continueSettling(true)) {
        invalidate();
    }
}

In this example, we create an instance of ViewDragHelper, set up the sensitivity level, and provide a callback that will be invoked while dragging the view. The callback will receive the start, stopping, and dragging details. Then, we attach the helper instance to the view hierarchy, override the callbacks, and process the touch events. Finally, we call the computeScroll() method to notify the helper when the drag action is complete.

Conclusion

ViewDragHelper makes it easy for developers to create draggable views in their apps without having to manually implement the logic for processing touch events. It is a great tool for creating interactive user interfaces and can be easily incorporated into most existing apps.
