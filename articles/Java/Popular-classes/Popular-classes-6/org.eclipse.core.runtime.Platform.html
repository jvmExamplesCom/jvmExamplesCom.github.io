org.eclipse.core.runtime.Platform Overview

The org.eclipse.core.runtime.Platform class provides a platform-independent API for accessing system environment variables and platform configuration options. This class is mainly used to get access to the values of environment variables and other system properties, such as the current workspace location, and the Java virtual machine (JVM) version.

Getting Started

To begin using Platform, start by importing the necessary packages:


          
           
import org.eclipse.core.runtime.Platform;

Then, you can access the Platform's properties using the appropriate methods. For example, if you need the system property named "os.name", use the getProperty() method:


          
           
String osName = Platform.getProperty("os.name");

The getProperty() method takes a single parameter, which should be the name of the property you want to access. This method returns a String value that represents the requested value.

Methods

In addition to getProperty(), the Platform class provides several other useful methods. Below is a list of some of the more commonly used methods:

  • getOS()
    - Returns the current operating system.
  • getWS()
    - Returns the current windowing system.
  • getArch()
    - Returns the current processor architecture.
  • getNL()
    - Returns the current language and country code.
  • isRunning()
    - Returns true if the platform is running in an application; false otherwise.
  • getLocation()
    - Returns the current workspace location.
  • getVM()
    - Returns the JVM version.

Example Usage

Let's look at an example usage of the Platform class. The following code prints out some basic information about the current system:


          
           
System.out.println("OS Name: " + Platform.getProperty("os.name"));
System.out.println("OS Version: " + Platform.getProperty("os.version"));
System.out.println("Java Version: " + Platform.getProperty("java.version"));
System.out.println("Workspace Location: " + Platform.getLocation());
System.out.println("VM Version: " + Platform.getVM());
System.out.println("Running in Application: " + Platform.isRunning());
