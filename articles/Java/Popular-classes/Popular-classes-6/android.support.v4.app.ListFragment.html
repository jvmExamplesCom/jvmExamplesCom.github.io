Android Support V4 App ListFragment

The android.support.v4.app.ListFragment is a part of Androidâ€™s support library, which allows developers to create robust and feature-filled apps that are compatible with versions of Android prior to 4.0 (Ice Cream Sandwich). The basic purpose of ListFragment is to provide a list of items for the user to choose from; it is the same as any other ListView, but with the added advantages of the support library.

Example


          
           
public class TestFragment extends ListFragment {
    private OnFragmentInteractionListener mListener;

    public TestFragment() {
    }

    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
    }

    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container,
                             Bundle savedInstanceState) {
        // Inflate the layout for this fragment
        return inflater.inflate(R.layout.fragment_test, container, false);
    }

    @Override
    public void onAttach(Context context) {
        super.onAttach(context);
        if (context instanceof OnFragmentInteractionListener) {
            mListener = (OnFragmentInteractionListener) context;
        } else {
            throw new RuntimeException(context.toString()
                    + " must implement OnFragmentInteractionListener");
        }
    }

    @Override
    public void onDetach() {
        super.onDetach();
        mListener = null;
    }

    @Override
    public void onListItemClick(ListView l, View v, int position, long id) {
        super.onListItemClick(l, v, position, id);
        if (null != mListener) {
            // Notify the active callbacks interface (the activity, if the
            // fragment is attached to one) that an item has been selected.
            mListener.onFragmentInteraction(position);
        }
    }

    public interface OnFragmentInteractionListener {
        void onFragmentInteraction(int position);
    }
}

As you can see, using a ListFragment is quite similar to using any other type of Fragment. The only difference is that you need to add in extra code to create the list, bind it to the adapter, and handle user interaction. For example, in this example, when the user clicks on an item in the list, the 

onListItemClick
method is called, which then notifies the 

          
           
OnFragmentInteractionListener
.

  • setListAdapter()

  • getListView()
    - To get the ListView associated with the fragment.
  • setSelection()
    - To set the selection (highlighted item) in the list.
