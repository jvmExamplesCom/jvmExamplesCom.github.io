java.util.concurrent.RunnableFuture

java.util.concurrent.RunnableFuture is an interface that extends both Runnable and Future interfaces. It provides a facility to run an asynchronous task, and it offers a result upon completion. It is mainly used in multi-threaded applications, as the task can be assigned to a separate thread while other threads of the application can continue to execute.

The RunnableFuture interface has two methods: the run() method, which starts the task and the get() method, which returns the result upon completion. The run() method must be implemented by the class implementing the RunnableFuture interface.

Example 1

The following example shows how to implement the RunnableFuture interface and use it to create a simple asynchronous task.


          
           
public class MyTask implements RunnableFuture<Integer> {
    private int result;
    public void run() {
        // do some work
        result = someCalculation();
    }
    public Integer get() {
        return result;
    }
}

The MyTask class implements the RunnableFuture interface, and it contains two methods: the run() method, which does the actual work of the task (in this case, it calls a method for doing some calculations, and assigns the result to the result variable), and the get() method, which returns the result of the calculations upon completion of the task.

Example 2

The following example shows how to create a new thread, assign the RunnableFuture task to it, and wait until the task finishes.


          
           
ExecutorService executor = Executors.newFixedThreadPool(1);
MyTask myTask = new MyTask();
executor.submit(myTask);
executor.shutdown();
while (!executor.isTerminated()) {
    // wait until the task is completed
}
// get the result of the task
int result = myTask.get();

In the above example, an ExecutorService object is created with a single thread pool. A new instance of the MyTask class is then created. The task is submitted to the ExecutorService using the submit() method and the executor is shut down. Finally, the program waits until the task is finished and the result is retrieved using the get() method.

In this example, an ExecutorService object is created with five threads in its thread pool. The CompletionService is then initialized with the ExecutorService object.
