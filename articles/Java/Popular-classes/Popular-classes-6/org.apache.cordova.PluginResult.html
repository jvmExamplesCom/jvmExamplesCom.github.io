org.apache.cordova.PluginResult

The org.apache.cordova.PluginResult class is used to manage the results of a plugin's execution on a Cordova device. It offers a way to check whether the plugin was executed successfully, and also allows for messaging back from the plugin to the Cordova WebView.

Constructor


          
           
public PluginResult(final Status status, final String message)

This constructor creates a new PluginResult object, with the provided status and message. The status must be one of PluginResult.Status values.

Methods

The org.apache.cordova.PluginResult class provides the following methods:

isSuccess()


          
           
public boolean isSuccess()

This method checks whether the plugin result was successful or not. It returns true if the status was one of: Status.OK, Status.NO_RESULT, Status.INVALID_ACTION, Status.JSON_EXCEPTION, Status.ERROR.

isError()


          
           
public boolean isError()

This method checks whether the plugin result was an error. It returns true if the status was one of: Status.CLASS_NOT_FOUND_EXCEPTION, Status.ILLEGAL_ACCESS_EXCEPTION, Status.INSTANTIATION_EXCEPTION, Status.MALFORMED_URL_EXCEPTION, Status.IO_EXCEPTION, Status.INVALID_ACTION, Status.JSON_EXCEPTION, Status.ERROR.

getMessage()


          
           
public String getMessage()

This method gets a string associated with the plugin result. It might contain an error message, or some other data sent from the plugin.

Example Usage

Here is a sample plugin code showing how the PluginResult class can be used. This sample plugin echoes a message back from a Cordova app:


          
           
public class EchoPlugin extends CordovaPlugin {

    @Override
    public boolean execute(final String action, final JSONArray args, final CallbackContext callbackContext) throws JSONException {
        // Create a PluginResult to send back to the WebView
        PluginResult result;
        if (action.equals("echo")) {
            // Get the message and echo it back
            final String message = args.getString(0);
            result = new PluginResult(PluginResult.Status.OK, message);
        } else {
            // Invalid action
            result = new PluginResult(PluginResult.Status.INVALID_ACTION);
        }
        // Send the result back to the WebView
        callbackContext.sendPluginResult(result);
        return true;
    }
}

In this example, the plugin will return a successful response if it receives the "echo" action, and an error response if any other action is used.
