Introduction to javax.ws.rs.HeaderParam

javax.ws.rs.HeaderParam is a Java API for the Java Platform, Enterprise Edition (Java EE) that provides support for the development of web services and RESTful (Representational State Transfer) web services in Java. It enables developers to use the Java programming language to access the headers of HTTP requests and responses.

Example Code


          
           
@Path("/")
public class MyResource {
    @GET
    public Response get(@HeaderParam("User-Agent") String userAgent) {
        // ...
    }
}

In the above example, the javax.ws.rs.HeaderParam annotation is used to map the http request header “User-Agent” to the Java method parameter “userAgent”. It allows the developer to access the value of the “User-Agent” request header from the method parameter.

Usage

The javax.ws.rs.HeaderParam annotation can be used to inject specific request headers as parameters of a resource class or method. It can also be used for sending response headers back to the client. It is commonly used for authentication and authorization, content negotiation, and caching.

Injecting Request Headers


          
           
@Path("/")
public class MyResource {
   @GET
   @Path("users")
   public Response getUsers(@HeaderParam("userId") String userId) {
       // ...
   }
}

In the above example, the “userId” request header is injected as the parameter “userId” of the Java method via the javax.ws.rs.HeaderParam annotation. This allows the developer to access the value of the “userId” request header from the method parameter.

Injecting Response Headers


          
           
@Path("/")
public class MyResource {
   @GET
   @Path("users")
   public Response getUsers(@HeaderParam("userId") String userId) {
      return Response.ok()
                     .header("X-Response-ID", 12345)

                     .build();
