Java.net.URLStreamHandlerFactory

The 

          
           
java.net.URLStreamHandlerFactory
class is a factory interface used to provide a mechanism to create URLs and their corresponding stream handlers.

This interface allows applications to delegate the responsibility of opening and creating streams from URLs to other types of objects. It is mainly used by the Java platform to establish an application’s own protocol for handling URLS.

How does URLStreamHandlerFactory work?

A URLStreamHandlerFactory is responsible for establishing a mapping between URL protocols and their corresponding stream handler classes. The factory creates an instance of a class which implements the URLStreamHandler interface when requested by the URLStreamHandlerRegistry.

When a URL is created using the 
java.net.URL
class, it uses a URLStreamHandlerFactory to look up the appropriate URLStreamHandler for the given protocol specified in the URL. If a valid handler isn’t found, the URL is created with a null URLStreamHandler.

Example code


          
           
public class MyURLHandlerFactory implements URLStreamHandlerFactory {

    private static URLStreamHandler handler;

    public MyURLHandlerFactory() {
        handler = new MyURLHandler();
    }

    public URLStreamHandler createURLStreamHandler(String protocol) {
        if (protocol.equals("http")) {
            return handler;
        }
        return null;
}}

In this example, we have a MyURLHandlerFactory that implements the URLStreamHandlerFactory interface. This class contains a static method that returns a valid URLStreamHandler if the protocol is http. Otherwise, it returns null.

Registering the URLStreamHandlerFactory

To register our custom URLStreamHandlerFactory, we must use the URLStreamHandlerRegistry class. We can do this using the following code:


          
           
URLStreamHandlerRegistry.registerFactory(
    new MyURLHandlerFactory());

Now, the MyURLHandlerFactory will be invoked by the URLStreamHandlerRegistry whenever a new URL object is created.

Conclusion

The Java.net.URLStreamHandlerFactory class provides a mechanism for applications to create URLs and their associated stream handlers. It allows developers to register their own custom URLStreamHandlerFactory objects with the URLStreamHandlerRegistry class. By doing this, the applications can then control how the URLs are created and the type of stream handler associated with each URL.
