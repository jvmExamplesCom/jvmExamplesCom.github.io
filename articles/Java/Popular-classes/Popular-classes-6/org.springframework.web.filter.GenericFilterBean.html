org.springframework.web.filter.GenericFilterBean

The org.springframework.web.filter.GenericFilterBean class is a specialized Java filter which is part of the Spring Framework. It provides basic filter functionality and helps developers create reusable filter implementations in their web applications. This class is an abstract class that defines the contract for filtering HTTP requests and responses.

Overview

GenericFilterBean is an abstract class that provides basic filter functionality. It is designed to be extended by concrete filter classes, such as AuthenticationFilter and CORSFilter. As such, it does not contain any concrete logic. Instead, it serves as an interface for other filter classes to extend and implement their own filter logic.

GenericFilterBean provides methods to create filter chains, define filter prioritization, and some utility methods. It also provides access to the HTTP request and response objects, making it easier to handle request and response data.

Methods

Some of the most important methods of GenericFilterBean are:

  • doFilter() – this is the main method used to perform filtering. It is invoked by the servlet container each time a request passes through the filter chain. The method is passed two parameters – ServletRequest and ServletResponse objects.
  • initFilterBean() – this method is run once when the filter is initialized. It can be used to perform one-time setup tasks.
  • destroy() – this method is run once when the filter is destroyed. It can be used to perform one-time cleanup tasks.
  • getServletContext() – this returns the associated servlet context object.
  • getFilterConfig() – this returns the associated filter configuration object.
  • getFilterName() – this returns the name of the filter.

Example

Let's look at an example of how to use GenericFilterBean to create a simple filter. In this example, we'll create a filter that logs the URL of every incoming request.


          
           
public class LoggingFilter extends GenericFilterBean {
 
  @Override
  public void doFilter(ServletRequest request,
                       ServletResponse response,
                       FilterChain chain)
    throws IOException, ServletException {

    // Log the request URL
    logger.info("Request URL: {}",
                ((HttpServletRequest)request).getRequestURL());
   
    // Pass the request on
    chain.doFilter(request, response);
  }
}

In this example, we've extended GenericFilterBean and implemented the doFilter() method. The doFilter() method is called each time an incoming request passes through the filter chain.
