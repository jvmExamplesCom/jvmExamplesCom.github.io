androidx.appcompat.app.ActionBarDrawerToggle

The androidx.appcompat.app.ActionBarDrawerToggle class provides an easy way to implement the sliding hamburger drawer menu pattern in your Android app. This class is part of the AppCompat library and provides a powerful, customizable, and interactive drawer that offers users quick access to their apps' navigation controls.

Constructors

The ActionBarDrawerToggle class has two constructors:


          
           
public ActionBarDrawerToggle(
    AppCompatActivity activity,
    DrawerLayout drawerLayout,
    int openDrawerContentDescRes,
    int closeDrawerContentDescRes)

public ActionBarDrawerToggle(
    AppCompatActivity activity,
    DrawerLayout drawerLayout,
    Toolbar toolbar,
    int openDrawerContentDescRes,
    int closeDrawerContentDescRes)

The first constructor takes an AppCompatActivity, a DrawerLayout, and two resource IDs for strings that describe the action when the user opens or closes the drawer. The second constructor is identical except it also takes a Toolbar object as an argument.

Methods

The ActionBarDrawerToggle class has several methods that you can use to control the behavior of the drawer:


          
           
setDrawerIndicatorEnabled()
onOptionsItemSelected()
onConfigurationChanged()
onDrawerOpened()
onDrawerClosed()
onDrawerStateChanged()
onDrawerSlide()

The setDrawerIndicatorEnabled() method enables or disables the drawer indicator. When it is enabled, the indicator will appear when the drawer is opened and disappear when the drawer is closed. The onOptionsItemSelected() method is called when the user selects an item from the options menu. The onConfigurationChanged() method is called when the device configuration changes. The onDrawerOpened() and onDrawerClosed() methods are called when the drawer is opened or closed. The onDrawerStateChanged() method is called when the drawer state changes. And finally, the onDrawerSlide() method is called when the drawer is being slid open or closed.

Next, we'll create an Activity class that contains the ActionBarDrawerToggle and sets up the navigation menu:


          
           
public class MainActivity extends AppCompatActivity {

    private ActionBarDrawerToggle mDrawerToggle;
    private DrawerLayout mDrawerLayout;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        mDrawerLayout = (DrawerLayout) findViewById(R.id.drawer_layout);
        Toolbar toolbar = (Toolbar) findViewById(R.id.toolbar);

        mDrawerToggle = new ActionBarDrawerToggle(this, mDrawerLayout, toolbar, 
            R.string.open_drawer, R.string.close_drawer);
        mDrawerLayout.addDrawerListener(mDrawerToggle);
        mDrawerToggle.syncState();

        // set up navigation menu here
    }

    @Override
    public boolean onOptionsItemSelected(MenuItem item) {
        if (mDrawerToggle.onOptionsItemSelected(item)) {
            return true;
        }

        return super.onOptionsItemSelected(item);
    }

    @Override
    public void onConfigurationChanged(Configuration newConfig) {
        super.onConfigurationChanged(newConfig);
        mDrawerToggle.onConfigurationChanged(newConfig);
    }
}

Here, we create an instance of the ActionBarDrawerToggle and add it to the DrawerLayout. Then, we override the onOptionsItemSelected() and onConfigurationChanged() methods to call the corresponding methods on the ActionBarDrawerToggle object. Finally, we set up the navigation menu, which is outside the scope of this article.
