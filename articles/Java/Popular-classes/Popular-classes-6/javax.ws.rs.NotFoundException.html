Overview of javax.ws.rs.NotFoundException

javax.ws.rs.NotFoundException is a subclass of the javax.ws.rs.ClientErrorException. It is thrown when a requested resource is not present on the server. It is used to indicate that the resource either doesn’t exist or the user may not have the permission to access it.

Example 1: Throwing a NotFoundException

The following code example throws a NotFoundException when a requested resource is not present on the server. 


          
           
try {
    // Access resource
}
catch(NotFoundException e) {
    System.out.println("Resource not found: " + e.getMessage());
}

Example 2: Handling the NotFoundException

The following code example handles a NotFoundException by returning an appropriate message when a requested resource is not present on the server. 


          
           
try {
    // Access resource
}
catch(NotFoundException e) {
    System.out.println("Resource not found. Please try again later.");
}
