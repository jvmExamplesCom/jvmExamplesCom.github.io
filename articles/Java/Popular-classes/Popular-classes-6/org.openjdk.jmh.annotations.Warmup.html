org.openjdk.jmh.annotations.Warmup

The 

          
           
@Warmup
annotation in the Java Microbenchmark Harness (JMH) is used to define the number of warmup iterations, the number of measurements, if it should be verbose, as well as other warmup related settings. The warmup phase of a benchmark is defined as iterations that run before the measurement phase, allowing the JVM instance to warm-up and adapt to the benchmarking context.

Syntax


          
           
@Warmup(iterations = 10, time = 100, timeUnit = TimeUnit.MILLISECONDS, batchSize = 1)

Parameters

  • iterations
    - the number of warmup iterations to be performed. It must be greater than 0.
  • time
    - the amount of time allowed for all warmup iterations combined. It must be greater than 0.
  • timeUnit
