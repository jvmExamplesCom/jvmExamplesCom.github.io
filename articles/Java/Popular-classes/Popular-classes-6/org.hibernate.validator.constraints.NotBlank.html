org.hibernate.validator.constraints.NotBlank

The org.hibernate.validator.constraints.NotBlank annotation is used to indicate that a field cannot be left blank. It is part of the Hibernate Validator Framework and is used for input validation when persisting or retrieving data from a database.

Example Usage

The following code shows how to use the NotBlank annotation as part of a JavaBean to validate user input. The example assumes that User is a JavaBean class with a firstName field.


          
           
@NotBlank
public String getFirstName() {
  return firstName;
}

The above code indicates that the firstName field of the User bean must not be blank when persisting or retrieving any data associated with the bean.
