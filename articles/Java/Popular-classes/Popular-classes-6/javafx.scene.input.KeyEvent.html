An Overview of javafx.scene.input.KeyEvent

The javafx.scene.input.KeyEvent class is used to handle events related to key presses in a JavaFX application. It enables an application to respond to user input from the keyboard, performing various actions depending on the content of a key press. In this article, we will take a look at some of the properties and methods included in JavaFX's KeyEvent class, including its usage and utility in application development.

KeyEvent Properties

The KeyEvent class contains several useful fields and property accessors. These provide the means to determine the state of a key press, as well as the actual character typed. Some of the more commonly used fields are:
  • char: A character representing the key pressed.
  • code: A string representing the key code for the event key. This will be unique for each key, but may not always match the character of the key pressed.
  • shiftDown: A boolean value indicating if the shift key is pressed down.
  • altDown: A boolean value indicating if the alt key is pressed down.
  • ctrlDown: A boolean value indicating if the control key is pressed down.
  • metaDown: A boolean value indicating if the meta key is pressed down. The meta key is typically the Windows (or Command) key.
These properties are set when the KeyEvent is created and can be accessed with the corresponding property accessors.

KeyEvent Methods

The KeyEvent class also provides several useful methods, which allow the developer to respond to a key press in their application. Some of the more commonly used methods are:
  • consume(): Consumes the event so that no other handlers are invoked. Typically used to override default behavior.
  • isConsumed(): Returns true if the event has been consumed.
  • isAltDown(): Indicates if the alt key was pressed when the event occurred.
  • isControlDown(): Indicates if the control key was pressed when the event occurred.
  • isMetaDown(): Indicates if the meta key was pressed when the event occurred.
  • isShiftDown(): Indicates if the shift key was pressed when the event occurred.
These methods provide the means to customise the behaviour of an application in response to a key press. They can be used to override or create customised behaviour, depending on the context of the key press.

Example Usage

The following example illustrates how the KeyEvent class can be used to respond to a specific key press in a JavaFX application. When the 'A' key is pressed, the application will display an alert dialog box with a message. 

          
           
Scene scene = new Scene(new Group(), 300, 250);

scene.setOnKeyPressed(event -> {
    if(event.getCode() == KeyCode.A){
        Alert alert = new Alert(AlertType.INFORMATION);
        alert.setContentText("You pressed the 'A' key!");
        alert.showAndWait();
    }
});
In this example, the 

          
           
onKeyPressed()
method is used to register a handler for key press events. Whenever a key press is detected, the supplied lambda expression is evaluated. In this case, it checks if the key pressed was 'A'. If so, then an AlertDialog is displayed with a message.

The javafx.scene.input.KeyEvent class provides the ability to detect and respond to key presses in a JavaFX application.
