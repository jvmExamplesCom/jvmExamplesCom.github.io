java.nio.channels.DatagramChannel

java.nio.channels.DatagramChannel is a selectable channel that supports datagram-oriented sockets. A datagram channel is created by invoking one of the open methods of this class. It can be connected to a remote address and then used to send packets and receive packets.

A datagram channel has an associated socket address, which it uses when sending and receiving packets. A datagram channel is thread-safe, meaning multiple threads may access it concurrently.

Example 1: Creating a Datagram Channel

The following example creates a datagram channel using the 

open()
method. 

          
           
DatagramChannel channel = DatagramChannel.open();

This method opens a datagram channel that is not yet connected to a remote address. To connect the channel to a remote address, use the 

connect()
method.

Example 2: Connecting a Datagram Channel to a Remote Address

The following example connects a datagram channel to a remote address. 


          
           
InetSocketAddress remoteAddress = new InetSocketAddress("example.com", 1234);
channel.connect(remoteAddress);

After connecting to a remote address, the datagram channel can be used to send and receive packets.

Example 3: Sending Packets

The following example sends a packet using the datagram channel. 


          
           
ByteBuffer buffer = ByteBuffer.allocate(1024);
buffer.put("Hello World!".getBytes());
buffer.flip();
channel.write(buffer);
The 
write()
method of the datagram channel will send the packet to the remote address that the channel is connected to. If the channel is not connected, an exception will be thrown.

Example 4: Receiving Packets

The following example receives a packet using the datagram channel. 


          
           
ByteBuffer buffer = ByteBuffer.allocate(1024);
channel.receive(buffer);
buffer.flip();
String message = new String(buffer.array());
System.out.println("Message received: " + message);
The 
receive()
method of the datagram channel will receive a packet from the remote address that the channel is connected to. If the channel is not connected, an exception will be thrown.
