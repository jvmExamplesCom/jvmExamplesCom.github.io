java.util.concurrent.atomic.AtomicIntegerFieldUpdater

The java.util.concurrent.atomic.AtomicIntegerFieldUpdater class provides volatile variables that can be used for providing atomic operations for integer type variables. It provides the same features and guarantees of the AtomicInteger class, but it works on fields of the specified classes rather than integer values.

The AtomicIntegerFieldUpdater class can be used to create an atomic updater for a specific field. The field must be declared volatile and cannot be private or transient as this would result in an exception being thrown. All fields that are to be updated must be declared in the same class.

Example

The following is an example of how to make use of the AtomicIntegerFieldUpdater class:


          
           
public class AtomicIntegerFieldUpdaterExample {
    public static void main(String[] args) {
        AtomCounter atomCounter = new AtomCounter();

        // Create the AtomicIntegerFieldUpdater
        AtomicIntegerFieldUpdater<AtomCounter> updater = 
            AtomicIntegerFieldUpdater.newUpdater(AtomCounter.class, "value");

        // Increase the value of the atomCounter by 1
        int newValue = updater.incrementAndGet(atomCounter);

        System.out.println("AtomCounter Value: "+newValue);
    }

    private static class AtomCounter {
        volatile int value;      
    }
}

In the above example we declare a class called AtomCounter containing the volatile field 'value'. We then create an instance of the AtomicIntegerFieldUpdater using the AtomicIntegerFieldUpdater.newUpdater() method, passing in the AtomCounter class and the name of the field as parameters. The updater is then used to increment the value of the atomCounter by one.
