org.springframework.boot.ApplicationArguments

The org.springframework.boot.ApplicationArguments class is an interface that provides access to application arguments parsed from the command line or environment in a Spring Boot application. It is an important part of the Spring Boot framework and is widely used in the development of applications. This article will provide an overview of how to use ApplicationArguments, as well as some usage examples.

Overview

ApplicationArguments provides access to the arguments passed to the application on startup. These arguments can be parsed from the command line, or from the environment variables. It provides access to the argument names and values, as well as access to the raw arguments and flags.

The ApplicationArguments interface defines several methods for working with arguments, including the following:

  • 
            
             
boolean containsOption(String option)
    - returns true if the specified option exists in the arguments.
  • 
            
             
String[] getOptionValues(String option)
    - returns the value of the specified option, or an empty array if the option is not present.
  • 
            
             
String[] getNonOptionArgs()
    - returns an array of the arguments which are not associated with an option.
  • 
            
             
String[] getSourceArgs()
    - returns the raw arguments, including arguments associated with options.

In addition to the methods defined in the interface, the class also provides a number of utility methods for working with arguments. These include 


          
           
getOptionProperties(String option)
, which returns an Properties object containing the properties associated with the specified option, and

Usage examples

Here are some examples of how to use ApplicationArguments in a Spring Boot application.

Accessing parsed arguments

The following example shows how to access the arguments parsed from the command line:


          
           
@Autowired
ApplicationArguments args;

for (String arg : args.getSourceArgs()) {
    System.out.println(arg);
}

if (args.containsOption("foo")) {

method can be used to retrieve the value of an option if it is present in the supplied arguments. The following example shows how to use this method: 

          
           
String[] fooValues = args.getOptionValues("foo");

    System.out.println(value);

Retrieving non-option arguments

The 
getNonOptionArgs()
method can be used to retrieve the arguments which are not associated with an option. This can be useful when parsing arguments from the command line. The following example shows how to use this method:
