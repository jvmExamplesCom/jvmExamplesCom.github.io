Org.jetbrains.annotations.Nls

The org.jetbrains.annotations.NLS annotation is used to help with internationalization of code written in the Java programming language. This annotation can be used when writing code that should be localized, such as user-facing messages and labels. It is part of the JetBrains Annotations API, which is a small set of annotations designed to provide additional insight about code to IDEs such as IntelliJ IDEA.

What Does It Do?

The purpose of org.jetbrains.annotations.NLS is to provide a way for the IDE to recognize when code needs to be localized to different languages and provide tooling to assist the developer in doing so. The annotation itself does not actually perform any localization; it simply provides a way for the IDE to identify when localization is necessary. When the annotation is present, the IDE can provide features such as automatic string externalization and automatic message generation. Additionally, the IDE can also provide warnings when strings are not externalized or are not properly externalized.

Example Usage

To use the org.jetbrains.annotations.NLS annotation, you first need to add the JetBrains Annotations API to your project. Then, to mark a string as needing localization, you simply add an 


          
           
@NLS
annotation above it. Here is an example: 

          
           
@NLS
String message = "Hello world!";

By adding this annotation, the IDE will recognize that the string needs to be localized and can provide additional tooling to assist with this. For example, if you're using IntelliJ IDEA, you can right click on the string and choose "Externalize String..." to easily externalize the string and make it available for localization.

Additional Considerations

It's important to note that the org.jetbrains.annotations.NLS annotation only works with strings and cannot be used with other types. Additionally, since the annotation requires the JetBrains Annotations API, it will not work unless this API is present in your project.

Finally, if you are using IntelliJ IDEA and are using the org.jetbrains.annotations.Nls annotation, you should also enable the “Enable Non-English string inspection” setting in the “Inspections” settings page. This setting will allow the IDE to detect strings that are not externalized correctly.

Conclusion

The org.jetbrains.annotations.NLS annotation is a useful tool for developers looking to make their code more internationalized. It allows the IDE to recognize when code needs to be localized and provides tooling to make the process easier. Additionally, if you are using IntelliJ IDEA, enabling the “Enable Non-English string inspection” setting is also recommended.
