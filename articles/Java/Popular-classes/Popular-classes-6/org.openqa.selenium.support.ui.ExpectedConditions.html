org.openqa.selenium.support.ui.ExpectedConditions

org.openqa.selenium.support.ui.ExpectedConditions is an interface that provides a set of useful functions for waiting for certain conditions to occur. It is used in combination with WebDriverWait and WebDriver instances to wait for certain conditions to be met before taking further action.

Basic Usage

The following code shows an example of how to use ExpectedConditions. This example waits for an element to become visible and then clicks it: 

          
           
WebDriver driver;
WebElement element;

WebDriverWait wait = new WebDriverWait(driver, 10); 
wait.until(ExpectedConditions.visibilityOf(element)); 
element.click();

Common Expected Conditions

ExpectedConditions provides several methods that can be used to check for various conditions.

alertIsPresent()

Checks if an alert is present and returns true if one is found. 

          
           
boolean alertPresent = ExpectedConditions.alertIsPresent() != null;

elementToBeClickable()

Checks if an element is both visible and enabled so it can be clicked. 

          
           
wait.until(ExpectedConditions.elementToBeClickable(By.id("clickMe")));

frameToBeAvailableAndSwitchToIt()

Checks if a frame is available and switches to it if it is. 

          
           
wait.until(ExpectedConditions.frameToBeAvailableAndSwitchToIt(By.id("myFrame")));

invisibilityOfElementLocated()

Checks if an element is not visible on the page. 

          
           
wait.until(ExpectedConditions.invisibilityOfElementLocated(By.id("invisible")));

urlContains()

Checks if the current URL contains the given string. 

          
           
wait.until(ExpectedConditions.urlContains("example.com"));
