Introduction to javax.persistence.Persistence

javax.persistence.Persistence offers an interface to obtain EntityManagerFactory and EntityManager instances in order to interact with a database. It is part of the Java Persistence API (JPA) and allows developers to create, read, update and delete (CRUD) data in a relational database. This article will provide an overview of javax.persistence.Persistence and demonstrate how to use it through code examples.

What is javax.persistence.Persistence?

javax.persistence.Persistence is an interface in the Java Persistence API that provides access to an EntityManagerFactory and allows developers to interact with a database. An EntityManagerFactory is used to create EntityManagers, which are used to manage data in a relational database. The EntityManagerFactory is responsible for creating and managing EntityManagers, which then handle the data access logic.

Creating an EntityManagerFactory

The first step in using javax.persistence.Persistence is to create an EntityManagerFactory. To do this, we must first create a javax.persistence.Persistence instance using the static createEntityManagerFactory() method. This method takes two parameters: a name for the EntityManagerFactory, and a Map of properties used to configure the EntityManagerFactory. For example:


          
           
Persistence persistence = Persistence.createEntityManagerFactory("my-factory", Collections.emptyMap());

This code creates a Persistence instance and sets its name to "my-factory". The second parameter is a Map of key-value pairs used to configure the EntityManagerFactory. In this example, we are using an empty Map, but you may also specify additional properties. For example, if you were using Oracle database, you may specify a "user" and "password" property to connect to the database.

Once the Persistence instance has been created, we can obtain the EntityManagerFactory by calling the getEntityManagerFactory() method:


          
           
EntityManagerFactory emf = persistence.getEntityManagerFactory();

The EntityManagerFactory is now ready to be used to create EntityManagers.

Creating an EntityManager

Once an EntityManagerFactory has been created, we can create an EntityManager by calling the createEntityManager() method. This method takes two parameters: a Map of properties used to configure the EntityManager, and an EntityTransaction, which is used to manage the transaction state of the EntityManager. For example:


          
           
EntityManager em = emf.createEntityManager(Collections.emptyMap(), null);

This code creates an EntityManager with an empty Map of configuration properties, and no EntityTransaction. The EntityManager is now ready to be used to interact with a database.

Using the EntityManager to Access the Database

Once an EntityManager has been created, it can be used to access the database. This is done using the various methods provided by the EntityManager interface. The most commonly used methods are find() and persist(). The find() method is used to retrieve Objects from the database, while the persist() method is used to create or update Objects in the database.

We can use the find() method to retrieve an Object from the database. This method takes a Class object and an ID as parameters. For example, if we wanted to retrieve an object with an id of 1 from the database, we could use the following code:


          
           
Object obj = em.find(MyObject.class, 1);

This code would retrieve an Object of type MyObject with an id of 1. If the Object was found, it would be returned; otherwise, a null value would be returned.

We can also use the persist() method to create or update an Object in the database. This method takes an Object as a parameter and will either create a new record in the database or update an existing record depending on whether or not the Object exists in the database. For example, if we wanted to create a new record in the database, we could use the following code:


          
           
em.persist(myObject);

This code would create a new record in the database for the given myObject. If an existing record existed, the record would be updated.

Closing an EntityManager

Once you have finished interacting with the database, you should close the EntityManager. This can be done by calling the close() method. For example:


          
           
em.close();

This code will close the EntityManager and the associated resources.

Summary

In summary, javax.persistence.Persistence is used to obtain EntityManagerFactory and EntityManager instances in order to interact with a database. An EntityManagerFactory is used to create EntityManagers, which are used to manage data in a relational database. We can use the EntityManager to retrieve Objects from the database using the find() method, and to create or update Objects in the database using the persist() method. Finally, once we have finished interacting with the database, we should close the EntityManager by calling the close() method.
