javax.annotation.processing.SupportedSourceVersion

The javax.annotation.processing.SupportedSourceVersion enum type is an annotation type used to indicate the version of source code accepted by an Annotation Processor. It is typically used in conjunction with the @SupportedSourceVersion annotation.

Uses

SupportedSourceVersion is used to declare the version of source code that an AnnotationProcessor supports. This is used to prevent compilation if the source code is written in a version that the AnnotationProcessor does not understand.

Example Usage


          
           
@SupportedSourceVersion(SourceVersion.RELEASE_7) 
public class MyProcessor extends AbstractProcessor { 
    // implementation of processor
}

The example above indicates that this AnnotationProcessor supports version 7 of the Java language.

Supported Versions

The SupportedSourceVersion annotation can accept one of four static values, as illustrated in the example above:

  • RELEASE_6
  • RELEASE_7
  • RELEASE_8
  • latest()

RELEASE_6 corresponds to versions of Java up to and including Java 6. RELEASE_7 corresponds to versions of Java up to and including Java 7. RELEASE_8 corresponds to versions of Java up to and including Java 8. The latest() option corresponds to the latest version of Java available at the time of compilation.

Using latest()

The use of the latest() option is helpful in cases where code needs to be compiled against the latest version of Java available. This allows for compatibility with new language features introduced with newer versions of the language. For example, the following would compile and run on both Java 7 and 8:


          
           
@SupportedSourceVersion(SourceVersion.latest())
public class MyProcessor extends AbstractProcessor { 
    // implementation of processor
}
