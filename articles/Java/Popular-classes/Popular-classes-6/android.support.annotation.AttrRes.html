Understanding android.support.annotation.AttrRes

android.support.annotation.AttrRes is an annotation type declaration that indicates whenever a method parameter, field or variable uses a resource reference in the form of an integer. This annotation is part of the Android Support Library and is available for use on API level 4 (Android 1.6) and above. The methods and classes annotated with this type live in the android.support.annotation package. When used correctly, this annotation allows developers to indicate explicitly which integer resources their code references and provides useful metadata located in the source code. This makes the source code easier to read and maintain.

Syntax

The syntax for using the @AttrRes annotation is as follows: 

          
           
@AttrRes public int resId
This type of annotation is also called a marker annotation since it does not contain any elements.

Example Usage

Here is an example of how to use the @AttrRes annotation in an Activity class. This example shows how to declare a member variable using the annotation. 

          
           
public class MyActivity extends Activity {

    @AttrRes private int backgroundColor;

    ...
}
In this example, the variable backgroundColor is declared and annotated with the @AttrRes annotation. This tells the compiler that this variable contains an integer resource reference.

Example Code

In order for the @AttrRes annotation to be properly utilized, it must be used in conjunction with other parts of the Android platform. Here is an example of code that uses the @AttrRes annotation to set the background color of an activity. 

          
           
public void setBackground(@AttrRes int resId) {
    Resources.Theme theme = getTheme();
    TypedArray attrs = theme.obtainStyledAttributes(new int[] { resId });
    int backgroundColor = attrs.getColor(0, 0);
    attrs.recycle();

    getWindow().setBackgroundDrawable(new ColorDrawable(backgroundColor));
}
