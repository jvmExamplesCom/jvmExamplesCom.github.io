Introduction to io.dropwizard.Application

io.dropwizard.Application is a Java library that provides a convenient way to create applications from scratch, with minimal effort and configuration. The library provides a simple API that allows developers to quickly and easily create highly-functional applications using existing Java frameworks, such as Dropwizard itself, Jetty, Hibernate and others.

Getting Started with io.dropwizard.Application

Before getting started, make sure to have the latest version of io.dropwizard.Application installed. You can find the installation instructions in the project’s readme file.

To create a new io.dropwizard.Application application, create a directory for the application and create an App.java file inside it. This file will contain all the code required to create an application.


          
           
public class App
  extends Application<Configuration> {

    public static void main(String[] args) throws Exception {
        new App().run(args);
    }

    @Override
    public void initialize(Bootstrap<Configuration> bootstrap) {
    }

    @Override
    public void run(Configuration configuration,
                    Environment environment) throws Exception {
    }
}

The App.java class has two methods: initialize and run. The initialize method is called once when the application is started up, and it is responsible for setting up all the necessary dependencies for the application. The run method is called once the application is initialized and ready to go, and it is responsible for actually running the application.

Defining Resources

Once the basic application is created, you can start defining resources and routes. A resource is any class that is annotated with the @Path annotation. The @Path annotation tells io.dropwizard.Application which URI to use when accessing the resource, and what HTTP methods to use when performing a request. 


          
           
@Path("/hello")
public class HelloResource {
    @GET
    public String sayHello() {
        return "Hello, World!";
    }
}

In the example above, a resource is created with a single route that responds with “Hello, World!” when an HTTP GET request is made to the /hello path. To register this resource with the application’s environment, use its register method:


          
           
environment.jersey().register(new HelloResource());

Now, whenever an HTTP GET request is made to the /hello path, the resource’s sayHello method will be executed and will return “Hello, World!”.

Creating Views

In order to create a view, create a file with the ending .ftl in the views directory. For example, create a hello.ftl file like this:
