org.jboss.arquillian.container.test.api.Deployment

The Deployment API provides a way to define application deployments that can be used in testing using the Arquillian framework. The Deployment object is used to define the application to be deployed and the necessary configuration for it during the test. It is closely related to the deployment descriptor, and allows for more complex deployments such as multiple deployments and packaged applications. It also allows for custom configuration to be applied to the deployment.

Example 1: Basic deployment setup


          
           
@Deployment
public static WebArchive createDeployment() {
  return ShrinkWrap.create(WebArchive.class)
      .addClasses(MyBean.class, MyServlet.class)
      .addAsWebInfResource(EmptyAsset.INSTANCE, "beans.xml");
 }

The above example shows a basic deployment setup using the Deployment API. The @Deployment annotation is used to mark the method as being a deployment method, which will be run when the test is launched. The deployment is created using the ShrinkWrap library, and classes, web resources, and other artifacts are added to it. The deployment is then returned.

Example 2: Multiple Deployments


          
           
@Deployment(name="app1")
public static WebArchive getApp1Deployment() {
  return ShrinkWrap.create(WebArchive.class)
      .addClasses(AppOneServlet.class)
      .addAsWebInfResource(EmptyAsset.INSTANCE, "beans.xml");
}

@Deployment(name="app2")
public static WebArchive getApp2Deployment() {
  return ShrinkWrap.create(WebArchive.class)
      .addClasses(AppTwoServlet.class)
      .addAsWebInfResource(EmptyAsset.INSTANCE, "beans.xml");
}

This example shows how to use the Deployment API to create multiple deployments. In this case, two applications are deployed, each with its own Servlet class and beans.xml file. The @Deployment annotation is used to define a name for each deployment, which can then be used to reference the deployments when running tests.

Example 3: Deployment Configuration


          
           
@Deployment
public static WebArchive createDeployment() {
  return ShrinkWrap.create(WebArchive.class)
      .addClasses(MyBean.class, MyServlet.class)
      .addAsWebInfResource(EmptyAsset.INSTANCE, "beans.xml")
      .setContextRoot("/myapp")
      .addAsWebResource(new File("somefile.js"));
}

In this example, configuration options are applied to the deployment. The context root of the application is set, as well as an additional web resource which is added to the application. This allows for more complex deployments to be defined and configured.
