javax.persistence.NamedQuery

The javax.persistence.NamedQuery is a query that is defined in the mapping metadata and referenced by the name. It is used to define a static query in metadata, which allows the use of certain parameters to be passed into the query at runtime. A NamedQuery is defined by providing a name, which must refer to the query defined in the Java Persistence query language.

Creating a NamedQuery

The following example shows how to create a NamedQuery in the Java Persistence query language. 


          
           
@NamedQuery(
        name = "findAllCustomers", 
        query = "SELECT c FROM Customer c"
)
public class Customer {
    //...
}

This defines a query named “findAllCustomers” which returns all customers from the database.

Executing a NamedQuery

A NamedQuery can be executed using the EntityManager, as shown in the following example: 


          
           
EntityManager em = ...;
TypedQuery<customer> query = em.createNamedQuery("findAllCustomers", Customer.class);
List<customer> customers = query.getResultList();
</customer></customer>

The above code executes the “findAllCustomers” query and returns the list of customers.

Passing Parameters

NamedQueries can accept parameters, which can be passed as argument when executing the query. The following example shows how to pass a parameter to a NamedQuery. 


          
           
@NamedQuery(
        name = "findCustomerById", 
        query = "SELECT c FROM Customer c WHERE c.id = :custId"
)
public class Customer {
    //...
}

          

          
           
EntityManager em = ...;
long custId = 123;
TypedQuery<customer> query = em.createNamedQuery("findCustomerById", Customer.class);
query.setParameter("custId", custId);
Customer customer = query.getSingleResult();
</customer>

In the above example, the “findCustomerById” query is executed with the custId parameter.

Restrictions

It is important to note that NamedQueries cannot be changed at runtime. If a query needs to be modified, it must be done by modifying the query in the mapping metadata.
