android.view.Choreographer

The android.view.Choreographer class is a part of Android's view hierarchy that manages the timing of animation and user interface interactions. It is used to prevent jankiness in animations by ensuring frames are drawn correctly and at the correct time. Choreographer also allows multiple applications to share animation time effectively.

What Does Choreographer Do?

Choreographer is responsible for issuing commands to the display hardware in order to render animations and user interface elements. It is responsible for scheduling when to draw frames, which helps ensure that frames are drawn at a regular rate and eliminates any stuttering that may result from frame rates that are too fast or too slow. It also helps keep animation and user interface interactions smooth and consistent across multiple applications.

How Does Choreographer Work?

Choreographer schedules drawing frames by responding to events from the application surface. When a new event is detected, it will request the greatest common divisor (GCD) of all pending events, which it then uses to schedule when the next frame should be drawn. The GCD ensures that all the events are evenly spaced so that the frames are drawn at an even rate and the animation or user interface interactions stay fluid.

  public void run() {
    // Update the state of the application

    Choreographer.getInstance().postFrameCallback(this);
  }
};

In this example, we create a Runnable object that contains the code that needs to be executed for each frame. We then use Choreographer.getInstance() to post the Runnable to Choreographer. Choreographer will then execute the Runnable each time a new frame needs to be drawn, ensuring that the frames are drawn at a regular rate.

Conclusion

android.view.Choreographer is a powerful tool for ensuring animations and user interface interactions remain fluid and consistent. By scheduling the time when frames need to be drawn, Choreographer helps eliminate jankiness and keeps things running smoothly.
