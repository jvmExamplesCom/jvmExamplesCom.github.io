The com.tencent.bugly.crashreport.CrashReport is a utility component of the Android platform, designed to help developers debug their applications for crash reports.

Class Overview

The CrashReport class provides various methods for reporting a crash to a remote server. It enables developers to get detailed information about the state of the application before and at the time of the crash. The CrashReport class also allows developers to control the logging level and to add custom data to the report.

Methods

  • logCrash(), which logs the details of a crash to a remote server.
  • captureScreenshot(), which captures and stores a screenshot of the application at the time of the crash.
  • getLogLevel(), which returns the current log level of the application.
  • setLogLevel(), which sets the log level of the application.
  • addCustomData(), which adds custom data to the report.
  • removeCustomData(), which removes custom data from the report.
