Introduction to androidx.annotation.RestrictTo

The androidx.annotation.RestrictTo annotation is an Android Jetpack-specific library that was introduced in Android Studio 3.2. It provides us a way to mark classes and members as restricted to certain conditions and scope. Overview of androidx.annotation.RestrictTo

The RestrictTo annotation can be used to signify the scope within which a specific class or member is usable. It has three parameters that are used to specify the scope: 
  • LIBRARY_GROUP
  • SUBCLASSES
  • TESTS
The LIBRARY_GROUP parameter specifies the scope of access for a class or method. Any scope through which the marked class or method will be accessible must be defined when applying the annotation. For example, a class or method may be restricted to be accessible only by other classes in the same library group. The SUBCLASSES parameter marks a class or method as restricted to be accessible only by its subclasses. This is useful for preventing other classes from accessing certain protected methods or fields that are intended only for usage by subclasses. The TESTS parameter marks a class or method as restricted to be accessible only by tests. This is useful for testing components that are not accessible outside of a test environment.
