org.springframework.data.elasticsearch.annotations.Document

In Spring Data Elasticsearch, the annotation 
@Document
marks a domain object to be indexed in an Elasticsearch index. This annotation can be used on the root level of a class hierarchy. When applied on the root level of a class hierarchy, the field values will be extracted from all the sub-classes.

Annotation Details

The 
@Document
annotation is used to mark a class that needs to be indexed in an Elasticsearch index. It mainly has the following attributes: 
  • indexName
    : Name of the Elasticsearch index in which entity needs to be indexed.
  • type
    : Name of the Elasticsearch type in which entity needs to be indexed.
  • shards
    : Number of shards that should be created when creating the index.
  • replicas
    : Number of replicas that should be created when creating the index.
  • refreshInterval
    : Refresh interval for the index.
  • indexStoreType
    : Type of index store to use for the index.

Usage Example

To use the annotation in a class, simply add the 
@Document
annotation on the class with the appropriate attributes set. For example, if we want to index a 
Person
class in an Elasticsearch index named “people” and type “persons”, with 10 shards and 5 replicas, we could do the following: 

          
           
@Document(indexName = "people", type = "person", shards = 10, replicas = 5)
public class Person {
    ...
}

We can also define the mapping for our fields as part of the annotation. The mapping is defined using the 

mappingPath
attribute. The mapping should be a JSON file stored in the classpath. For example, the following annotation would map the Person class using the 

          
           
mappings/person_mapping.json
file: 

          
           
@Document(indexName = "people", type = "person", shards = 10, replicas = 5,
        mappingPath = "mappings/person_mapping.json")
public class Person {
    ...
}
