org.bukkit.event.player.AsyncPlayerChatEvent

The AsyncPlayerChatEvent is an event that is fired off whenever a Player sends a chat message. This event is asynchronous and is fired off before the server processes the chat message.

Methods

The AsyncPlayerChatEvent class contains several methods. Below are a few of the most commonly used:


          
           
public String getMessage() 
public void setMessage(String message)
public boolean isAsynchronous()
public void setCancelled(boolean cancelled)

The getMessage() method returns the chat message that was sent. The setMessage() method can be used to modify the chat message before it is sent to the server. The isAsynchronous() method returns true if the event is asynchronous, and the setCancelled() method can be used to set whether or not the event should be cancelled.

Example

Here is an example of how to use the AsyncPlayerChatEvent to alter a message before it is sent to the server:


          
           
@EventHandler
public void onChatMessage(AsyncPlayerChatEvent event) {
    // Get the original message
    String message = event.getMessage();

    // Change the message
    message = "Hello, World!";

    // Set the new message
    event.setMessage(message);
}

In this example, we are using the AsyncPlayerChatEvent to intercept a chat message before it is sent to the server. We first get the original message, then change it to “Hello, World!”, and finally set the new message via the setMessage() method.

Conclusion

The AsyncPlayerChatEvent is a powerful event that allows you to intercept and modify a chat message before it is sent to the server. This can be used to add custom functionality such as logging messages or altering the message in some way.
