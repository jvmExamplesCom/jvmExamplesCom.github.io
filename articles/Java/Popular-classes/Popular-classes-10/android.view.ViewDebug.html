android.view.ViewDebug for Java

The android.view.ViewDebug class provides debugging tools for debugging Android views. It contains methods for displaying view state information, for refreshing views and for capturing screens of views. This article explains how to use the ViewDebug class.

ViewDebug contains a set of methods which can be used to get information about the state of a view. The method 

          
           
dump()
can be used to print out detailed information of a view such as its size, padding, margins, etc. 

          
           
ViewDebug.dump(view);
The 

          
           
capture()
method can be used to capture the current state of a view as a bitmap: 

          
           
Bitmap capture = ViewDebug.capture(view);

refreshDrawableState()
method can be used to refresh a view's drawable states, such as the focus state or pressed state. This can be useful when developing custom views: 

          
           
ViewDebug.refreshDrawableState(view);

method can be used to capture all the view layers of a view as a bitmap. This can be useful for debugging complex view hierarchies.
