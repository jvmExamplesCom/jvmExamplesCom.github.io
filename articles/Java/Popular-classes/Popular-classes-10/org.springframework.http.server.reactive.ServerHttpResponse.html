Introduction to org.springframework.http.server.reactive.ServerHttpResponse

The org.springframework.http.server.reactive.ServerHttpResponse class provides an abstraction for server-side HTTP response operations, including status and headers. It is typically used in http server implementations to set up an HTTP response message to be sent back to the client.

Methods

This class contains a number of methods to support its use. Some of the most commonly used ones are listed below.

setStatusCode()

The setStatusCode() method is used to set the HTTP status code for the response. It takes a org.springframework.http.HttpStatus as argument. The following example shows how to set the status code to “OK”.


          
           
response.setStatusCode(HttpStatus.OK);

setHeader()

The setHeader() method is used to add an HTTP header to the response. It takes a String header name and a List of header values as arguments. The following example shows how to set a Content-Type header with a value of “text/plain”.


          
           
response.setHeader("Content-Type", Arrays.asList("text/plain"));

byte[] bytes = new byte[1024];

getCookies()

The getCookies() method is used to retrieve any cookies that were sent in the request. It returns a Map of cookie names and values. The following example shows how to retrieve the value of a cookie named “myCookie”.


          
           
Map<String, String> cookies = response.getCookies();
String myCookieValue = cookies.get("myCookie");
