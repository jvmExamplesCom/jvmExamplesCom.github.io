Introduction to org.quartz.DisallowConcurrentExecution

In Quartz, jobs are associated with triggers. A trigger specifies when a job should be executed and, optionally, how often it should be repeated. When the trigger fires, Quartz will start a new instance of the associated job.

DisallowConcurrentExecution can be used to ensure that multiple instances of a particular job are not running at the same time.
