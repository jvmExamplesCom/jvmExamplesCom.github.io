org.eclipse.swt.events.KeyAdapter

The org.eclipse.swt.events.KeyAdapter class provides basic implementation of the SWT KeyListener interface. It can be used to register a KeyListener with a specific widget and receive notifications for key events.

Constructors

The class has no public constructors and it must be used as is.

Methods

The org.eclipse.swt.events.KeyAdapter class has four methods: keyPressed, keyReleased, keyTraversed, and getKeyAdapter.

keyPressed

This method is called when a user presses a key on the keyboard while the widget has focus. It takes one parameter, a KeyEvent object, which contains information about the key press event.

Example Usage

The following example demonstrates how to use the KeyAdapter class to register a KeyListener with a Text widget and handle key press and release events. The program prints the key code and character of the keys pressed while the widget has focus.


          
           
public static void main(String[] args) {
   Display display = new Display();
   Shell shell = new Shell(display);

   Text text = new Text(shell, SWT.BORDER);
   text.setBounds(10, 10, 100, 20);
   text.addKeyListener(new KeyAdapter() {
      public void keyPressed(KeyEvent e) {
         System.out.println("Key pressed: keyCode=" + e.keyCode + ", character='" + e.character + "'");
      }

      public void keyReleased(KeyEvent e) {
         System.out.println("Key released: keyCode=" + e.keyCode + ", character='" + e.character + "'");
      }
   });

   shell.pack();
   shell.open();
   while (!shell.isDisposed()) {
      if (!display.readAndDispatch()) {
         display.sleep();
      }
   }
   display.dispose();
}
