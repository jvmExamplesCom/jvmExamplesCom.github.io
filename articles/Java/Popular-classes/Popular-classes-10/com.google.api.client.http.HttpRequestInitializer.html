com.google.api.client.http.HttpRequestInitializer

The 

          
           
HttpRequestInitializer
interface is a convenient way to initialize a request before it is executed. It is part of the Google API Client Library for Java. This article will explain how to use this interface to create a customized request and provide several code examples.

Implementing the Interface

The 

          
           
HttpRequestInitializer
interface extends from 
Object
and contains only one abstract method, 
initialize()
. To use it, implement the interface in a class and override the 
initialize()
method like this: 

          
           
public class MyHttpRequestInitializer implements HttpRequestInitializer {
  @Override
  public void initialize(HttpRequest request) throws IOException {
    // Custom initialization logic
  }
}
The 
initialize()
method takes a 
HttpRequest
as an argument. This is the request that should be initialized.

Using an HttpRequestInitializer

An instance of an 

          
           
HttpRequestInitializer
can be passed to the 
HttpRequestFactory
when creating a request. To do so, simply pass the 

          
           
HttpRequestInitializer
to the 
createRequest()
method like this: 

          
           
HttpRequestFactory requestFactory = new NetHttpTransport().createRequestFactory(myHttpRequestInitializer);
HttpRequest request = requestFactory.createRequest(requestUrl, httpMethod);
When the request is created, the 
initialize()
method of the 

          
           
HttpRequestInitializer

Setting a Timeout

To set a timeout on the request, use the

setReadTimeout()

HttpRequest

request.setConnectTimeout(1000);

Conclusion
