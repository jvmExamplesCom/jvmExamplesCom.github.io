org.testng.SkipException

org.testng.SkipException is part of the TestNG testing framework for Java, and indicates that a test case has been skipped during execution. The SkipException can be used in conjunction with a try/catch block within a test case, allowing the programmer to control when a test should be skipped - for example, if certain conditions or inputs are not present. When used in this manner, the SkipException causes the test case to exit immediately, and no further tests are run.

The SkipException can also be used within a suite of tests to indicate that all remaining tests should be aborted. In this case, the SkipException will be thrown from within the suite, and all tests that were scheduled to run beyond this point will be skipped.

Example Usage

Below is an example of using the SkipException within a test case. In this example, the test checks to see if an array of integers is empty. If the array is empty, the test case is skipped:


          
           
public void testCaseExample() {
    int[] arr = {1, 2, 3};
    try {
        if(arr.length == 0) {
            throw new SkipException("Array is empty, skipping test");
        }
        // Continue test code...
    } catch (SkipException e) {
        System.out.println(e.getMessage());
    }
}

In the above example, the try/catch block catches the SkipException and prints the corresponding error message to the console.

Skipped Test Output

When a test is skipped due to throwing a SkipException, TestNG will print out information about the skipped test. This includes the name of the test class, the name of the test method, and the message associated with the SkipException. For example:

By providing useful messages to the SkipException, it can be easier to identify which tests have been skipped when looking at the test output.

Conclusion

The org.testng.SkipException is an important part of the TestNG framework, providing a way to bypass a test case or abort a suite of tests.
