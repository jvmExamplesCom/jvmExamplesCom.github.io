Introduction to org.apache.ibatis.annotations.Update

org.apache.ibatis.annotations.Update is a Mybatis annotation that configures an SQL update statement for use with the MyBatis framework. It is used to update existing data in a database table. MyBatis is an open-source persistence framework for working with databases, mapping them to Java objects. The @Update annotation is one of several annotations available in MyBatis for mapping Java objects and databases.

Using @Update in Code

The @Update annotation is a type of mapped statement that creates an SQL update statement and executes it against a mapped database table. To use the annotation, you must specify the SQL statement as part of the @Update annotation. The following example shows a MyBatis mapper interface using the @Update annotation to map an update statement: 

          
           
@Mapper
public interface PersonMapper {
 
  @Update("update person set name = #{name} where id = #{id}")
  void updateName(@Param("id") int id, @Param("name") String name);

}
In this example, the @Update annotation is configured with an SQL update statement. The statement updates the name of a person based on the provided id. This statement will be executed when the updateName() method is invoked. The @Param annotation is used to bind parameters to the SQL statement. In this example, the parameters are provided as method arguments to the updateName() method.

Using Placeholders

MyBatis can also leverage placeholders in the SQL statement. A placeholder is a special marker in the SQL statement that is replaced with a value when the statement is executed. This makes it easy to parameterize the SQL statement with values from the Java code. For example, the following @Update annotation uses a placeholder: 

          
           
@Update("update person set name = ? where id = ?")

In this example, the SQL statement has two placeholders. When the statement is executed, the ? markers are replaced with the provided parameters for the updateName() method.

Using Expression Language

The MEL is an object-oriented expression language that provides additional features compared to standard placeholders. To use the MEL in an @Update annotation, you need to configure the annotation with an expression. For example, the following @Update annotation uses the MEL to construct an update statement:
