FMLPreInitializationEvent in Java

The FMLPreInitializationEvent is a type of event which is fired before initialization in the mod loading process. It is fired when mod classes are constructed and registry events(i.e. registering blocks, items and ores) have been handled. It allows for a bulk of setup to be performed by a mod during its loading.

Here is a simple example of how a mod can use this event to create custom blocks and items:


          
           
//inside the main mod loader class
public void preInit(FMLPreInitializationEvent event){
  ModBlocks.init();
  ModItems.init();
}

//mod blocks class
public static void init(){
  block1 = new Block1();
  block2 = new Block2();
  GameRegistry.registerBlock(block1);
  GameRegistry.registerBlock(block2);
}

//mod items class
public static void init(){
  item1 = new Item1();
  GameRegistry.registerItem(item1);
}

It is also possible to register various things with FML/Forge via the FMLPreInitializationEvent, such as a custom GUI Handler, a custom faction handler, or a custom config handler.

Registering a Custom Config Handler

One of the most common uses of the FMLPreInitializationEvent is to register a custom configuration handler. This allows a mod to load values from its configuration file and access them inside the mod.


          
           
//inside the main mod loader class
public void preInit(FMLPreInitializationEvent event){
  //register config
  ConfigurationHandler.init(event);
}

//configuration handler class
public static void init(FMLPreInitializationEvent event){
  config = new Configuration(event.getSuggestedConfigurationFile());
  try{
    config.load();
  }
  catch(Exception e){
    FMLLog.log(Level.ERROR, e, "Error Occurred trying to load configs");
  }
  finally{
    config.save();
  }
}

//inside the main mod loader class
