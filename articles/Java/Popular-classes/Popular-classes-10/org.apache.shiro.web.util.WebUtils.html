WebUtils in Java

The org.apache.shiro.web.util.WebUtils class provides several web-related utility methods commonly required by web applications and web infrastructure components such as Java Servlet containers, Spring MVC, or even web frameworks. This class is part of the Apache Shiro project which is a powerful and easy-to-use open-source security solution for Java.

Basic Utility Methods

The WebUtils class provides several basic utility methods for dealing with web-related tasks. The most useful is getPathWithinApplication(ServletRequest request) which returns the relative portion of the request URL that corresponds to the application's context path. Here's an example: 

          
           
String path = WebUtils.getPathWithinApplication(request);
The method is also overloaded to accept a HttpServletRequest if you need additional capabilities. Another useful utility provided by this class is the getCleanParam(ServletRequest request, String paramName) which will return the value of the given request parameter after trimming it of any leading and trailing whitespace. String param = WebUtils.getCleanParam(request, "paramName");

ServletContext Utility Methods

The WebUtils class provides several utility methods related to dealing with the application's ServletContext. boolean equivalent = WebUtils.isFunctionallyEquivalentContext(context1, context2);
The redirectToSavedRequest(HttpServletRequest request, HttpServletResponse response, String defaultUrl) is another very useful method which will redirect a client to the saved request currently stored in their session, or to the specified default URL if none is found. WebUtils.redirectToSavedRequest(request, response, "/default.jsp");

Session Utility Methods

The WebUtils class provides several utility methods related to dealing with a web client's HttpSession. String sessionId = WebUtils.getSessionId(request);
