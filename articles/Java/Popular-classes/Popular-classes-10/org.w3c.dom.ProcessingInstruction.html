Introduction to org.w3c.dom.ProcessingInstruction

Processing instructions (PIs) allow documents to carry processing information along with their actual content. These instructions are typically used by application programs to affect the way the documents are processed. For example, a PI may be used to indicate that a document is to be formatted using a certain style sheet language.

The org.w3c.dom.ProcessingInstruction interface represents a "processing instruction", used in XML as a way to keep processor-specific information in the text of the document. PIs are most commonly used to identify the style sheets associated with an XML document. In this case, the target is always set to "xml-stylesheet".

Example of Processing Instruction

The following is an example of a PI named "comment":


          
           
<?comment This is a comment ?>

This PI has a target of "comment" and a data value of "This is a comment".

Create Processing Instruction

The DOM Level 1 does not support the creation of PIs. However, DOM Level 2 provides a method for their creation, namely the 


          
           
document.createProcessingInstruction()
method. This method takes two parameters:
  • The target of the PI
  • The data contained in the PI

For example, the following code creates a PI named "comment" with a data value of "This is a comment":


          
           
ProcessingInstruction pi = document.createProcessingInstruction("comment","This is a comment");

Modify Processing Instructions

Once created, the target and data of a PI can be modified with the methods 

setNodeValue()
, 
setData()
and 
setTarget()
. The following code modifies the PI created in the previous example: 

          
           
pi.setNodeValue("This is a modified comment");
pi.setData("This is a modified comment");
pi.setTarget("updated-comment");

Accessing Processing Instruction Information

In addition to being able to modify a PI, it is also possible to access its information. This is done using the methods 

getNodeName()
, 
getNodeValue()
, 
getData()
and 
getTarget()
. The following code shows how to access the information for the PI created in the previous example: 

          
           
System.out.println(pi.getNodeName()); // prints "updated-comment"
System.out.println(pi.getNodeValue()); // prints "This is a modified comment"
System.out.println(pi.getData()); // prints "This is a modified comment"
System.out.println(pi.getTarget()); // prints "updated-comment"

Conclusion

The org.w3c.dom.ProcessingInstruction interface represents a "processing instruction", used in XML as a way to keep processor-specific information in the text of the document. It allows a document to carry processing information along with its actual content, and can be used to indicate that a document is to be formatted using a certain style sheet language. ProcessingInstructions can be created, modified, and accessed using the available DOM Level 2 methods.
