org.eclipse.jetty.server.handler.HandlerCollection

org.eclipse.jetty.server.handler.HandlerCollection is a Jetty-specific handler that allows multiple handlers to be registered with a single request. It can be used to register different types of request handlers such as servlets, filters, and listeners.

Writing Handlers

Writing a Handler requires extending the Handler class, overriding its handle() method, and then registering it with a Server instance. The following example shows how to create a simple handler for an HTTP GET request:


          
           
public class MyHandler extends AbstractHandler {
    @Override
    public void handle(String target, Request baseRequest, HttpServletRequest request, 
        HttpServletResponse response) throws IOException, ServletException {

        response.setStatus(HttpServletResponse.SC_OK);
        response.setContentType("text/html;charset=utf-8");
        response.getWriter().println("<h1>Hello World!</h1>");
        baseRequest.setHandled(true);
    }
}

Registering a Handler

To register a handler, you must use the 
addHandler()
method of the 
HandlerCollection
class. This method takes in an instance of the 
Handler
class. The following example shows how to register a 
MyHandler
instance with the HandlerCollection: 

          
           
public static void main(String[] args) throws Exception {
    // Create a new server instance
    Server server = new Server();

    // Create a new HandlerCollection instance
    HandlerCollection handlers = new HandlerCollection();

    // Create an instance of MyHandler
    MyHandler myHandler = new MyHandler();

    // Add the handler to the collection
    handlers.addHandler(myHandler);

    // Set the handlers on the server
    server.setHandler(handlers);

    // Start the server
    server.start();
    System.out.println("Server started");
    server.join();
}
