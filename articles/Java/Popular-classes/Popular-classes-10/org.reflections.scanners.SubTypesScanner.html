Overview of org.reflections.scanners.SubTypesScanner

The 

          
           
org.reflections.scanners.SubTypesScanner
class is an implementation of the 

          
           
org.reflections.scanners.AbstractScanner
abstract class to scan for subtypes of a given type. It can be used in combination with the 

          
           
Reflections
class for advanced scanning and reflection features. The SubTypesScanner is used to find all classes that extend or implement another class or interface, even if the relationship is transitive.

How to use SubTypesScanner

The 

          
           
SubTypesScanner
class is typically used in combination with the 

          
           
Reflections
object, which provides convenient methods to perform scanning operations. To scan for all subtypes of a given type using the Reflections object, one would use the 

          
           
Reflections.getSubTypesOf(type)
method, where type is the class or interface whose subtypes should be scanned.

Example Java Code using SubTypesScanner

Here is a simple example of using the SubTypesScanner and Reflections objects together to scan for all classes or interfaces that extend or implement a given type: 

          
           
Reflections reflections = new Reflections("myPackage");
Set<Class<? extends MyBaseType>> subTypes = reflections.getSubTypesOf(MyBaseType.class);
The 

          
           
subTypes

MyBaseType
.
