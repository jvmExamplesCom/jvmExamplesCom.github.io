org.eclipse.osgi.util.NLS

The org.eclipse.osgi.util.NLS class is an open source API from the Eclipse project that enables applications to access externalized string constants from resource bundles. It provides a simple means to access localized text for use in an application. The API provides an easy way to manage application text across different languages, allowing developers to create a single resource bundle for their application and have it be used throughout the system.

Basic Usage

The basic usage of the API is to define a constant class containing static final fields containing the text keys. These are then loaded into a resource bundle with the same name as the containing class. The resource bundle can be in any supported format such as .properties or .xml.


          
           
public class Messages {
    public static final String HELLO = "hello";
    public static final String WELCOME = "welcome";
}

The resource bundle can then be loaded using a constructor call on the org.eclipse.osgi.util.NLS class.


          
           
org.eclipse.osgi.util.NLS.getBundle(Messages.class);

Once the resource bundle is loaded, text can be accessed using the static get method.


          
           
String hello = org.eclipse.osgi.util.NLS.get(Messages.HELLO);

Formatting Strings

The NLS class provides support for formatting strings. This allows for messages that include placeholders for dynamic values. For example, a message might include a user name or a date.


          
           
public static final String GREETING = "hello {0}";

This message can be formatted using the format static method.


          
           
String greeting = org.eclipse.osgi.util.NLS.format(Messages.GREETING, "John");
// Output: hello John

Strings can also be formatted with the parameters being a list or an array.


          
           
String[] params = {"John", "Smith"};
String greeting = org.eclipse.osgi.util.NLS.format(Messages.GREETING, params);
// Output: hello John Smith
