Java AWT Data Transfer ClipboardOwner

The 

          
           
java.awt.datatransfer.ClipboardOwner
interface is used to receive notification when the contents of the system clipboard change. It is implemented by applications or applets that need to keep track of changes to the clipboard and take appropriate action.

Object Creation

To create an instance of 
ClipboardOwner
, you must define a class that implements it. The constructor will look something like this: 

          
           
public class MyClipboardOwner implements ClipboardOwner {

	public MyClipboardOwner() {
		// Constructor code here
	}

}
You can then create an instance of your 
ClipboardOwner
class like so: 

          
           
MyClipboardOwner myClipboardOwner = new MyClipboardOwner();

Methods

The 
ClipboardOwner
interface provides two methods to implement: 
  • 
            
             
lostOwnership(Clipboard clipboard, Transferable contents)
    - Called when the contents of the clipboard are no longer available.
  • 
            
             
getClipboardData(DataFlavor flavor)
    - Returns the requested data flavor from the clipboard.

Example

Let's say we have an application where a user can copy some text inside a 

JTextField
. We can use a 
ClipboardOwner
to be notified when the text in the clipboard changes, and take appropriate action. 

          
           
import java.awt.*;
import java.awt.datatransfer.*;
import javax.swing.*;

public class MyClipboardOwner implements ClipboardOwner {

	JTextField textField;

	public MyClipboardOwner(JFrame frame) {

		frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
		frame.setSize(200,100);
		textField = new JTextField();
		frame.add(textField);
		frame.setVisible(true);

	}

	@Override
	public void lostOwnership(Clipboard clipboard, Transferable contents) {
		//Called when the contents of the clipboard are no longer available.
		String data = (String) contents.getTransferData(DataFlavor.stringFlavor);
		textField.setText(data);
	}

	public static void main(String[] args) {
		SwingUtilities.invokeLater(new Runnable() {
			@Override
			public void run() {
				//Create the frame.
				JFrame frame = new JFrame("MyClipboardOwner");
				MyClipboardOwner clipOwner = new MyClipboardOwner(frame);

				//Set the clipboard owner.
				Clipboard clipboard = Toolkit.getDefaultToolkit().getSystemClipboard();
				clipboard.setContents(clipboard.getContents(null), clipOwner);

			}
		});
	}

}
In this example, the 
MyClipboardOwner
class creates a 
JFrame
with a 
JTextField
. It then sets itself as the clipboard owner. When the contents of the clipboard change, the 
lostOwnership
method is called and the new clipboard contents are set as the text of the 
JTextField
.
