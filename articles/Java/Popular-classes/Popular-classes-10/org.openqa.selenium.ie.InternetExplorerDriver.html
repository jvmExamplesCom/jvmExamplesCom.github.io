Overview of org.openqa.selenium.ie.InternetExplorerDriver

The org.openqa.selenium.ie.InternetExplorerDriver is a webdriver implementation that allows users to automate interactions with Internet Explorer. It uses Microsoft's IEDriverServer to control Internet Explorer and is compliant with the WebDriver specification. The driver communicates with a remote server, typically located on the same machine as the IE browser. This driver allows users to easily write automated tests that take advantage of the features of Internet Explorer, including Windows Integrated Authentication and automatic detection of proxy settings.

Installing & Configuring the Driver

In order to use the InternetExplorerDriver, first download the IEDriverServer from the Selenium website. Unzip the archive and place it in a location on your system path. Once the IEDriverServer is installed, you can create a new instance of the InternetExplorerDriver. To do this, you will need to provide the path to the driver executable. System.setProperty("webdriver.ie.driver", "path/to/IEDriverServer.exe");
InternetExplorerDriver driver = new InternetExplorerDriver();

InternetExplorerOptions options = new InternetExplorerOptions();

Using the Driver

Troubleshooting

Next, make sure that you are using a compatible version of the IEDriverServer for your version of Internet Explorer. Check the release notes on the Selenium website for the version of the IEDriverServer that works with your version of IE.
