AbstractWebSocketMessageBrokerConfigurer

The AbstractWebSocketMessageBrokerConfigurer class is a base class for configuring a WebSocket message broker in Spring’s MVC application context. It provides methods to set up the message broker and configure the endpoint registry. The message broker is responsible for processing messages sent by clients and sending them back after applying appropriate logic. It also works as an event-driven system, meaning it can respond to messages sent by other applications and services.

The class provides various methods for configuring the message broker, such as setting up the broker's inbound and outbound channels, as well as configuring properties related to the broker like its socket timeout or heartbeat frequency.

Example Code Usage

Below is an example of how to use the AbstractWebSocketMessageBrokerConfigurer in a Spring MVC application context.


          
           
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig extends AbstractWebSocketMessageBrokerConfigurer {

    @Override
    public void configureMessageBroker(MessageBrokerRegistry config) {
        config.enableSimpleBroker("/topic");
        config.setApplicationDestinationPrefixes("/app");
    }

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/gs-guide-websocket").withSockJS();
    }
}

In the above code, we enable the web socket message broker and set up the message broker and its inbound and outbound channels. The 


          
           
configureMessageBroker()
method configures the message broker with a simple broker protocol and the 

          
           
registerStompEndpoints()
method registers a Stomp endpoint at the URL /gs-guide-websocket with support for SockJS.

@Override
public void configureMessageBroker(MessageBrokerRegistry config) {
    config.enableSimpleBroker("/topic");
