Understanding QueueProcessingType in Java

QueueProcessingType is a class of the com.nostra13.universalimageloader.core.assist package, which is used for controlling the order of images loading and decoding as well as sorting files in disc cache. The class offers two static methods - FIFO and LIFO - to control the process using either first-in-first-out or last-in-first-out principles.

FIFO


          
           
QueueProcessingType fifo = QueueProcessingType.FIFO;
First-in-first-out (FIFO) principle means that the order in which images will be loaded and decoded is the same as the order of their appearance in a queue. The earliest image in the queue will be processed first.

LIFO


          
           
QueueProcessingType lifo = QueueProcessingType.LIFO;
Last-in-first-out (LIFO) principle means that the last image in the queue will be processed first. This is useful when processing large numbers of images, since it may speed up the loading of images by reducing the amount of time spent on processing each image.

Sorting Files in Disc Cache

QueueProcessingType is also used to sort files stored in the disc cache before they are loaded and decoded. By default, files are sorted using the FIFO principle, but this can be changed by setting the queue processing type to LIFO.
