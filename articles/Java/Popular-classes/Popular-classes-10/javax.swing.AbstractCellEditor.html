javax.swing.AbstractCellEditor

The javax.swing.AbstractCellEditor class provides a framework for editing the data contained in table cells, tree nodes and other components. It makes it easier for developers to quickly and easily implement cell editors for those components. This class provides all of the basic operations for editing cells in a user-friendly way, such as detecting changes to cells and firing appropriate events when editing is completed.

Creating a Cell Editor

To create a custom cell editor, developers need to extend the AbstractCellEditor and implement the three methods it provides, which are 

          
           
getCellEditorValue
, 

          
           
isCellEditable
and 

          
           
getTableCellEditorComponent
. 

          
           
public class CustomCellEditor extends AbstractCellEditor {

	private JTextField textField;
	
	public CustomCellEditor() {
		textField = new JTextField();
	}
	
	@Override
	public Object getCellEditorValue() {
		return textField.getText();
	}

	@Override
	public boolean isCellEditable(EventObject anEvent) {
		// we allow the cell to be edited if it is double-clicked
		if (anEvent instanceof MouseEvent) {
	    	MouseEvent mouseEvent = (MouseEvent) anEvent;
	    	if (mouseEvent.getClickCount() == 2) {
	    		return true;
	    	}
	    }
	    return false;
	}
	
	@Override
	public Component getTableCellEditorComponent(JTable table, Object value, boolean isSelected, int row, int col) {
		textField.setText((String) value);
		return textField;
	}

}

Using the Cell Editor

Once the cell editor has been created, it can then be used with any component that supports cell editing, such as a JTable or a JTree. To do this, the cell editor must first be registered with the component. 

          
           
// register the cell editor with the table
