What is javax.persistence.Embedded?

javax.persistence.Embedded is an annotation in the Java Persistence API (JPA) that indicates that a particular field should be persisted as an embedded object in a database. Embedded objects are simply fields of a class which are serialized into the database as part of the containing entity, rather than having their own unique identity and table.

Examples of Embedded Annotation

The @Embedded annotation can be used to mark a field as an embedded object, like so:


          
           
@Entity
public class Person {
    ...

    @Embedded
    private Address address;

    ...
}
In this example, the 
Person
entity has an 
Address
field which is annotated with the 
@Embedded
annotation. This means that the 
Address
object will be embedded within the 
Person
entity when it is stored in the database.

Mapping Embedded Objects

The fields of an embedded object can be mapped to the database using various annotations, such as the 

@Column
annotation: 

          
           
@Entity
public class Address {

    @Column(name = "street_address")
    private String streetAddress;

    @Column(name = "city")
    private String city;

    @Column(name = "state")
    private String state;

    @Column(name = "zip")
    private String zipCode;

    ...

}
In this example, each field in the 
Address
class is mapped to the database using the JPA 
@Column
annotation. This means that the values of these fields will be stored in the database when the 
Person
entity is persisted.
