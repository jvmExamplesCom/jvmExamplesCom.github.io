Introduction to org.apache.ibatis.mapping.BoundSql

org.apache.ibatis.mapping.BoundSql is an interface in the Apache MyBatis library for Java. It is used for binding SQL statements, parameters and settings. This interface provides access to a request’s query string, the mapped statement's parameter mappings, the map of named parameters, the SqlSource object, the timeout and other query properties.

Usage Examples

The following code example demonstrate how to use org.apache.ibatis.mapping.BoundSql: 

          
           
MappedStatement ms = configuration.getMappedStatement("com.example.SomeMapper.selectById");
BoundSql boundSql = ms.getBoundSql(parameterObject);
String sql = boundSql.getSql();
List<ParameterMapping> parameterMappings = boundSql.getParameterMappings();
Object parameterObject = boundSql.getParameterObject();
Map<?, ?> additionalParameters = (Map<?, ?>) boundSql.getAdditionalParameters();
In this example, we obtain the MappedStatement from the Configuration instance, then get its BoundSql, from which we can retrieve the SQL and parameter mappings, as well as other properties. We can also use org.apache.ibatis.mapping.BoundSql for setting parameters and timeouts for a query: 

          
           
BoundSql boundSql = ms.getBoundSql(parameterObject);
boundSql.setAdditionalParameter("timeout", 10L);
boundSql.setParameterObject(parameterObject);
In this example, we obtain the BoundSql from the MappedStatement instance and then set the timeout and parameter.
