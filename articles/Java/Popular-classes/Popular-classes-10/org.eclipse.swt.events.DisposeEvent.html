org.eclipse.swt.events.DisposeEvent

The org.eclipse.swt.events.DisposeEvent is an event generated when a widget or resources associated with it, are disposed. It is typically used to clean up any associated resources when the widget is no longer needed, such as releasing system resources.

org.eclipse.swt.events.DisposeListener

Shell shell = new Shell(display, SWT.SHELL_TRIM);
shell.addDisposeListener(new DisposeListener() {
    public void widgetDisposed(DisposeEvent e) {
        // Clean up
    }
});
The 
widgetDisposed
method of the listener is called when the 
SWT.Dispose
event is sent out. This is the last event to be sent out before the widget is destroyed, so it's a good place to clean up any remaining resources.

When handling the 
DisposeEvent
, it is important to remember that the widget associated with this event is in the process of being destroyed. All of its children have already been destroyed, and all their resources have been cleaned up. Therefore, it should not be assumed that any of the widget's state is still present.
