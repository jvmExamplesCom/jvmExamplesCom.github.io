javax.transaction.Status

The javax.transaction.Status interface provides constants that represent the various states in which a transaction can exist. In a transaction processing system, every transaction is associated with a status, which represents the stage at which the transaction is currently in. These statuses are constants and are declared as public static final ints within the Status interface.

Constants Defined by javax.transaction.Status

  • STATUS_ACTIVE
    : This constant signifies that the transaction is currently active.
  • 
            
             
STATUS_MARKED_ROLLBACK
    : This constant signifies that the transaction has been marked for rollback, perhaps due to some error that occurred during the transaction processing.
  • STATUS_PREPARED
    : A transaction always starts in the 
    STATUS_ACTIVE
    state. When the transaction is prepared, its status changes from 
    STATUS_ACTIVE
    to 
    STATUS_PREPARED
    .
  • STATUS_COMMITTED
    : This constant signifies that the transaction has been committed successfully.
  • STATUS_ROLLEDBACK
    : This constant signifies that the transaction has been rolled back.

Example

The following example shows how to use Status interface:


          
           
import java.util.concurrent.atomic.AtomicInteger;
import javax.transaction.Status;

public class Example {

    public static void main(String[] args) {

        // Create a new atomic integer with the initial value 0
        AtomicInteger txStatus = new AtomicInteger(Status.STATUS_ACTIVE);

        // Print the status of the transaction
        System.out.println("Transaction Status: " + txStatus.get());

        // Mark the transaction for rollback
        txStatus.set(Status.STATUS_MARKED_ROLLBACK);

        // Print the status of the transaction
        System.out.println("Transaction Status: " + txStatus.get());
    }
}
