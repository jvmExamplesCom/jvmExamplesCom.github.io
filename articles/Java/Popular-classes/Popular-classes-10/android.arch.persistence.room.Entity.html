Android Room Database and Entity

Android apps use Room to access information from a database. Room is an object-relational mapping (ORM) library that provides simplified access to data stored in SQLite databases. The Room library uses the @Entity annotation to define an entity class and its corresponding table in the database. This article will explain what the `@Entity` annotation is and how it can be used to store and manage data in an Android app using Room.

What is Entity?

The @Entity annotation marks a class as an entity and maps it to a table in the database. It defines the name of the table, the columns, and their types. An Entity class can also have relationships with other Entity classes, and these can be defined as well. The Entity can be used to define the layout of the database and its data.

How to use Entity?

To use the @Entity annotation, first create a class that extends the RoomDatabase class. Then, add the @Entity annotation to the class, specifying the table name, the columns and their types. 

          
           
@Entity(tableName = "Person")
public class Person extends RoomDatabase {
    
    @PrimaryKey
    public int id;
    
    public String name;
    
    public int age;
}
The above code creates a Person entity that maps to a table named “Person” in the database. The id column is the primary key for the table and the other two columns, name and age, are both strings. Next, create a Dao interface in the same class and then add the @Dao annotation. This interface will provide methods for inserting, updating, and deleting data in the Person table. For example: 

          
           
@Dao
public interface PersonDao {

    @Insert
    void insertPerson(Person person);
    
    @Update
    void updatePerson(Person person);
    
    @Delete
    void deletePerson(Person person);

}
Finally, create a method in your Activity or fragment to call the PersonDao methods. For example: 

          
           
public void savePerson(Person person) {
    PersonDao personDao = myDatabase.personDao();
    personDao.insertPerson(person);
}

Summary

The @Entity annotation can be used to map a Java class to a table in the database in Android Room. The @Entity annotation defines the table name, the columns and their types, and relationships with other entities. A Dao interface can then be used to provide methods for inserting, updating, and deleting data from the table. Finally, a method can be called from the Activity or Fragment to call the Dao methods and save the data to the database.
