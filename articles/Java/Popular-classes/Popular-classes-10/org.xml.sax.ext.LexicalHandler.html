org.xml.sax.ext.LexicalHandler

The org.xml.sax.ext.LexicalHandler is an extension to the standard SAX2 XMLReader interface, providing support for reading “lexically significant” event information on a SAX2 parser. This information includes comments, CDATA sections, entity references, and the beginning and end of a document.

Comments

Comments in XML documents are ignored by most parsers - they do not create any events. The primary purpose of comments is thus to allow authors to annotate their documents for their own reference, but also to provide meta-data information outside of the actual XML content. For example, a comment might describe the author of a document, or its version number.

The LexicalHandler provides support for these comments. It defines four methods for dealing with them: startComment(), endComment(), comment(), and commentChunk(). The two methods startComment() and endComment() indicate the beginning and end of a comment, while the methods comment() and commentChunk() provide the contents of the comment.

CDATA Sections

A CDATA section is similar to a comment, in that it is ignored by the parser but included in the source document. Its purpose is to indicate an area of text which should not be parsed. By default all text within tags is parsed by the parser, including valid XML syntax such as < and >. By including this text in a CDATA section - indicated by the tags - it can be treated as literal text, not parsed as XML.

Entity References

An entity reference is a special sequence of characters that is used to represent data outside of the XML document itself. The most common form of entity reference is an external entity reference, which is specified as follows: &name;. This will cause the parser to look for an external resource (such as a file or URL) identified by the name field, and return the contents of that resource in place of the entity reference.
