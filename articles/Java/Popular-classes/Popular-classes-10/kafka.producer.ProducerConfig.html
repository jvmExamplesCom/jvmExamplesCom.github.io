Kafka Producer Config

Kafka producer config is a Java class that enables users to configure how they produce data to a Kafka cluster. It provides a set of properties that can be used to configure the producer, including setting reliability and throughput parameters, setting batching parameters, and configuring compression and serialization settings. This class is an important component of the Kafka API, providing an interface for producers to interact with a Kafka cluster.

Basic Configuration Properties

Kafka producer config provides a number of basic configuration properties that must be set in order to produce data to a Kafka cluster. These properties include: 
  • bootstrap.servers
    - A list of host/port pairs to use for establishing the initial connection to the Kafka cluster.
  • client.id
    - An id string to pass to the server when making requests. The purpose of this is to be able to track the source of requests beyond just IP/port by allowing a logical application name to be included.
  • acks
    - The number of acknowledgments the producer requires the leader to have received before considering a request complete. This controls the durability of records that are sent.
  • retries
    - Number of retries (or attempts) to send messages until success or failure.
  • batch.size
    - The producer will attempt to batch records together into fewer requests whenever multiple records are being sent to the same partition.
  • buffer.memory
    - The total bytes of memory the producer can use to buffer records waiting to be sent to the server.
  • compression.type
    - GZIP, SNAPPY, LZ4, NONE.
  • max.request.size
    - The maximum size of a request in bytes.
  • timeout.ms
    - The maximum time the producer will wait for acknowledgments.}
