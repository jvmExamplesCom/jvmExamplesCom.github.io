Introduction to org.bukkit.event.entity.EntityDamageEvent

The 

          
           
org.bukkit.event.entity.EntityDamageEvent
is an event that is fired whenever an entity is damaged in the game. It is important to understand this event in order to properly handle damage throughout a server. This article will discuss what the event looks like, when it is fired, and also provide examples of how to use it.

Overview of EntityDamageEvent

The 

          
           
EntityDamageEvent
extends 

          
           
EntityEvent

setDamage()
method. Lastly, this event also allows a plugin to cancel any damage received.

When is EntityDamageEvent Fired?

The 

          
           
EntityDamageEvent
is fired whenever an entity takes damage. This includes damage from other entities, being in the water, and different types of exposure. For example, when a player is damaged by another player, the event is fired immediately after the health of the affected entity is reduced.

EntityDamageEvent

EntityDamageEvent
