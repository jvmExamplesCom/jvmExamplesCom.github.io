javax.ws.rs.container.PreMatching

javax.ws.rs.container.PreMatching is an API of Java API for RESTful Web Services (JAX-RS 2.0) that provides support for pre-matching of request filters and resource methods.

What It Does

In JAX-RS, each request filter is invoked for each request for the purpose of filtering it. This means that the filter is called multiple times for each request and this can have an impact on performance. Pre-matching helps to reduce the number of invocations of a filter by matching it with the incoming request before it is dispatched to the appropriate resource method.

Example Usage

The following code example shows how to create a PreMatching filter that ensures that no request is dispatched to any resource method unless it has been successfully authenticated.


          
           
@PreMatching
public class AuthFilter implements ContainerRequestFilter {

    @Override
    public void filter(ContainerRequestContext reqCtx) throws IOException {
        if (!reqCtx.getHeaders().containsKey("Authorization")) {
            throw new NotAuthorizedException("No Authorization header provided");
        }
    }
}
The AuthFilter will be invoked 
before
any resource method to ensure that the request contains an Authorization header. If the header is not present then a 

          
           
NotAuthorizedException
is thrown and the request is not dispatched to any resource method.

Conclusions

javax.ws.rs.container.PreMatching is a useful API in JAX-RS 2.0 that helps to reduce the number of invocations of filters by pre-matching them with the incoming request. This can help to improve the performance of your JAX-RS application.
