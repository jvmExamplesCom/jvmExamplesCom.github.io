Introduction to org.springframework.test.web.servlet.result.MockMvcResultMatchers

org.springframework.test.web.servlet.result.MockMvcResultMatchers is a class within the Spring Framework which provides methods for doing assertions on an HTTP response. It encapsulates the expectations of a particular endpoint and matches the content type and status code of the response with the expected value. It can be used both with JUnit and TestNG.

Methods Available

The MockMvcResultMatchers class provides the following methods: 
  • andExpect()
    - Asserts that the response has the provided status.
  • andStatus()
    - Asserts that the response has the provided content type.
  • andContentType()
    - Asserts that the response contains the expected body.
  • andJsonPath()
    - Asserts that the response contains the expected JSON path.
  • andXPath()
    - Asserts that the response contains the expected XML path.
  • andRedirectedUrl()
    - Asserts that the response is a redirect with the provided URL.
  • andFlashAttr()
    - Asserts that the response contains a flash attribute with the expected value.
  • andCookie()
    - Asserts that the response contains the expected cookie.
  • andHeader()
    - Asserts that the response contains the expected header.
  • andNoContent()
    - Asserts that the response does not have an associated body.

Examples

It is easy to use MockMvcResultMatchers to do assertions on the response of a web request. For example, to check that the response has returned a 200 (success) status code: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(status().isOk());
Or to check that the response contains the expected content type: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(contentType("application/json"));
To check that the response contains a particular body: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(content().string("[{\"id\":1,\"name\":\"John Doe\"}]"));
To check that the response contains a particular JSON path: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(jsonPath("$[0].name", is("John Doe")));
To check that the response contains a particular XML path: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(xpath("/users/user/name").string("John Doe"));
To check that the response is a redirect with the provided URL: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(redirectedUrl("http://example.com/users"));
To check that the response contains a flash attribute with the expected value: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(flash().attribute("message", "User added successfully!"));
To check that the response contains the expected cookie: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(cookie().value("JSESSIONID", "abc123"));
To check that the response contains the expected header: 

          
           
mockMvc.perform(get("/user"))
    .andExpect(header().string("Content-Type", "application/json"));
To check that the response does not have an associated body: 

          
           
mockMvc.perform(delete("/user/1"))
    .andExpect(noContent());

Conclusion

org.springframework.test.web.servlet.result.MockMvcResultMatchers makes it easy to perform assertions on HTTP responses. It provides a set of methods for checking the status, content type, body, headers and cookies of the response.
