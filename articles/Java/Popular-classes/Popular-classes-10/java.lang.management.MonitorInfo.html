Java.lang.management.MonitorInfo

Java.lang.management.MonitorInfo is an object that provides access to the runtime information of an object's monitor in the Java Virtual Machine (JVM). This functionality is useful for debugging and performance tuning applications on the JVM. It provides a way to check the state of an object at any given time, including whether it is locked or unlocked, and if the lock is held by another thread.

What is a Monitor?

A monitor, otherwise referred to as a mutex, is a mechanism for synchronizing access to a shared resource. A single thread must hold a lock on the monitor to gain exclusive access to the shared resource. Other threads that attempt to acquire the lock are blocked until the lock is released by the owning thread.

Using MonitorInfo

Java.lang.management.MonitorInfo is an interface that provides access to the monitor of an object. The basic methods of the MonitorInfo interface are:

  • 
            
             
getLockedStackDepth()
    - Gets the depth of the stack frame where the object's monitor was last acquired.
  • 
            
             
getLockedStackFrame()
    - Gets the stack frame where the object's monitor was last acquired.
  • getLockingThread()
    - Gets the thread that currently owns the object's monitor.
  • getLockOwnerName()
    - Gets the name of the thread that currently owns the object's monitor.
The MonitorInfo instance is obtained using the 

          
           
getMonitorInfo()
method of the java.lang.management.ObjectMonitor class. The 

          
           
getMonitorInfo()
method takes a reference to the object whose monitor is to be monitored as an argument.

In the following example we use the MonitorInfo class to monitor the monitor of an object.


          
           
public static void testMonitorInfo(Object obj) {
  // Get the monitor info of an object
  MonitorInfo monitorInfo = ObjectMonitor.getMonitorInfo(obj);
  // Get the locking thread
  Thread lockingThread = monitorInfo.getLockingThread();
  // Get the name of the locking thread
  String lockOwnerName = monitorInfo.getLockOwnerName();
  // Get the stack frame where the lock was acquired
  StackTraceElement lockedStackFrame = monitorInfo.getLockedStackFrame();
  // Get the stack depth where the lock was acquired
  int lockedStackDepth = monitorInfo.getLockedStackDepth();
  // Output the results
  System.out.println("Locking thread: " + lockingThread);
  System.out.println("Lock owner name: " + lockOwnerName);
  System.out.println("Locked stack frame: " + lockedStackFrame);
  System.out.println("Locked stack depth: " + lockedStackDepth);
}
