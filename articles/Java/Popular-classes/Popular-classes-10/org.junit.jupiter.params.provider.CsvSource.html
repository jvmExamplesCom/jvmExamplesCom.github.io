Introduction to org.junit.jupiter.params.provider.CsvSource

org.junit.jupiter.params.provider.CsvSource is a parameterized test method provider for JUnit 5 that provides methods with multiple arguments. It allows for tests to be run multiple times with different data sets.

Usage

The CsvSource annotation is placed on the test method and takes a comma-separated list of strings indicating the name of the data set file, or the data itself. Each row of the file or data is one test execution, and each column in the row represents an argument to the test method.

For example, given the following CSV data file: 

          
           
name, age
John, 25
Mary, 32

A parameterized test method with one string argument and one integer argument could be written as follows:


          
           
@ParameterizedTest
@CsvSource({ "John, 25", "Mary, 32" })
void testPerson(String name, int age) {
    // Test code goes here
}

Advanced Usage

There are several advanced options available when using CsvSource. For example, it is possible to indicate the delimiter being used, or skip certain rows from the source data.

The following annotations can be used to modify the behavior of the CsvSource:

  • @CsvFileSource: Indicates the name of the CSV file to use.
  • @CsvDelimiter: Indicates the character used to separate values in the CSV file. The default delimiter is a comma.
  • @CsvSkipLines: Indicates the number of lines from the top of the CSV file that should be ignored. This is useful for skipping column headers, for example.

For example, given the following CSV data file, which uses a pipe ('|') as the delimiter:


          
           
name|age
John|25
Mary|32

A parameterized test method with one string argument and one integer argument could be written as follows:


          
           
@ParameterizedTest
@CsvFileSource(resources = "/data/people.csv", delimiter = '|', skipLines=1)
void testPerson(String name, int age) {
    // Test code goes here
}

Conclusion

org.junit.jupiter.params.provider.CsvSource is a powerful way to perform parameterized tests with JUnit 5. Its simple syntax allows for multiple data sets to be used in a single test, and it can be easily customized with various annotations for more complex scenarios.
