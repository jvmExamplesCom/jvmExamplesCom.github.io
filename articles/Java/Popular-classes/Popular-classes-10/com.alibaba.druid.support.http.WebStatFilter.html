Introduction to com.alibaba.druid.support.http.WebStatFilter

com.alibaba.druid.support.http.WebStatFilter is a java class that provides web application monitoring capabilities for the Apache Tomcat servlet container. It allows developers and administrators to monitor request and response activity in their deployed applications, giving insight into application behavior and performance.

Basic Usage

To use the WebStatFilter in a web application, first add it to the web.xml configuration file:


          
           
<filter>
    <filter-name>DruidWebStatFilter</filter-name>
    <filter-class>com.alibaba.druid.support.http.WebStatFilter</filter-class>
    <init-param>
        <param-name>exclusions</param-name>
        <param-value>*.js,*.gif,*.jpg,*.bmp,*.png,*.css,*.ico,/druid/*</param-value>
    </init-param>
</filter>
<filter-mapping>
    <filter-name>DruidWebStatFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

The exclusion list is a comma separated list of patterns to ignore when tracking request/response activity. This is usually used to exclude static files such as images, javascript, and css from the monitoring. This can also be used to exclude requests to the druid jmx console from being included in the results.

Advanced Configuration

The WebStatFilter provides several additional configuration options for controlling monitoring behavior. These can be added to the web.xml configuration file as init-params for the filter:


          
           
<init-param>
    <param-name>profileEnable</param-name>
    <param-value>true</param-value>
</init-param>
<init-param>
    <param-name>sessionStatMaxCount</param-name>
    <param-value>1000</param-value>
</init-param>
<init-param>
    <param-name>sessionStatEnable</param-name>
    <param-value>true</param-value>
</init-param>

The profileEnable parameter enables/disables profiling of servlet requests. The profileThreshold parameter (default 0) can be used to specify the minimum time (in ms) a request must take in order to be included in the results. The sessionStatMaxCount parameter (default 1000) specifies the maximum number of sessions to keep track of. The sessionStatEnable parameter enables/disables collection of session state information.

Monitoring & Reporting

Once the filter has been added to the web app and enabled, it will begin to collect request/response data and maintain statistics. The data collected by the filter can be viewed using the Druid jmx console or via an html report. The html report can be accessed by visiting the URL defined by the druid.statViewServlet configuration option. By default, this URL is /druid/index.html.

The report displays basic request information including the URL, HTTP method, response status, response content type, and request duration. It also shows the number of active sessions and pending requests, the total number of requests processed, and the total number of bytes transmitted.

Conclusion

com.alibaba.druid.support.http.WebStatFilter provides a powerful and easy-to-use tool for monitoring web applications in the Apache Tomcat servlet container. With its simple configuration parameters, developers and administrators can quickly gain insight into their applications performance and behavior.
