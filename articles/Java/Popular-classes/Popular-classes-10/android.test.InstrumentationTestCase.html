android.test.InstrumentationTestCase

Android's InstrumentationTestCase is a test framework used to write user interface and functional tests for various parts of an Android application. It provides access to the various components and capabilities of an Android system, allowing for a robust suite of tests to be written against applications. This framework allows for the development of unit tests that can be integrated into any Android application.

Why Use InstrumentationTestCase?

Being able to write automated tests for an Android app brings several benefits:

  • Faster application development time - Automated tests remove the need for manually verifying each new feature, providing faster feedback on errors or defects.
  • Easier to maintain code - Automated tests are easier to maintain than manual test sets, as they can be more easily updated when changes to the codebase occur.
  • Better application quality - Automated tests can be run more frequently, leading to fewer defects being present in the final version of the application.

Using InstrumentationTestCase

InstrumentationTestCase must be subclassed in order to write a testable class. The most basic form of a testable class looks like this:


          
           
public class MyInstrumentationTest extends InstrumentationTestCase {
    public void testSomething() {
    	// Your test code here
    }
}

The testSomething() method contains all of your test logic, which is similar to other JUnit 4 test cases. The full API of InstrumentationTestCase can be found here. The API provides access to various system services of the Android device, like Android Contexts and Activities, which can be used to successfully interact with a tested application.

In addition to the API, developers should be familiar with the different Android permissions that can be enabled for a test case. Permissions can be granted to a case by using the addPermission() method, like this:


          
           
getInstrumentation().addPermission("android.permission.CAMERA");

These permissions can be used to simulate specific user actions, such as taking a picture or accessing the camera. Once the desired permission has been granted, a test can be written to ensure that the feature works as expected.

Running Instrumentation Tests

Instrumentation tests can be run easily from the command line, using the 


          
           
adb shell am instrument
command. This command takes a few parameters, including the package name of the application to be tested and the name of the test class that should be executed. 

          
           
adb shell am instrument -w com.example.myapplication/android.test.InstrumentationTestRunner

This command will run all tests in the specified test class. If you want to only execute a single test, you can do that as well:


          
           
adb shell am instrument -w com.example.myapplication/android.test.InstrumentationTestRunner -e class com.example.myapplication.MyInstrumentationTest#testSomething

This command will only execute the testSomething() test method in the MyInstrumentationTest class. To learn more about the 


          
           
adb shell am instrument
command, see here.
