org.apache.curator.framework.state.ConnectionStateListener

The org.apache.curator.framework.state.ConnectionStateListener class is a part of Apache Curator, a library of utility classes designed to help manage distributed systems. It is used to monitor and detect changes in the connection state of the Curator framework. The ConnectionStateListener is responsible for notifying the user when the connection between the client and server is lost or regained.

Examples

Below are some examples of how to use the org.apache.curator.framework.state.ConnectionStateListener class:


          
           
ConnectionStateListener listener = new ConnectionStateListener() {
    public void stateChanged(CuratorFramework client, ConnectionState newState) {
        switch (newState) {
            case LOST:
                System.out.println("Connection to server was lost");
                break;
            case CONNECTED:
                System.out.println("Connection to server was gained");
                break;
            //etc.
        }
    }
};
client.getConnectionStateListenable().addListener(listener);

In this example, a ConnectionStateListener is created and added to the client's listenable. When the state of the connection changes, the listener will be notified and can take appropriate action.


          
           
client.getConnectionStateListenable().removeListener(listener);

This example removes the specified listener from the listenable, so it will no longer be notified of changes to the connection state.
