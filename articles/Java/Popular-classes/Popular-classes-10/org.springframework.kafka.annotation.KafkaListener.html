org.springframework.kafka.annotation.KafkaListener

The org.springframework.kafka.annotation.KafkaListener annotation is used to mark a method in a class as a Kafka message listener. This annotation has several optional parameters.

Creating a KafkaListener

To create a KafkaListener, you simply need to add the @KafkaListener annotation to a method in your class with the topic name as a parameter. 

          
           
@KafkaListener(topics = "my-topic")
public void processMessage(String content) {
    // Do something with the message
}
You can also specify additional configuration options as parameters. For example, you can specify whether to commit offsets after each message or after a batch: 

          
           
@KafkaListener(topics = "my-topic", autoOffsetReset = OffsetResetStrategy.LATEST, enableOffsetCommit = true, commitOffsetsOnEachBatch = true)
public void processMessage(String content) {
    // Do something with the message
}

Message Processing

When a Kafka message is received, it will be passed into the method annotated with @KafkaListener as an argument. The type of the argument must match the type of message received. For example, if the message is a String, the method argument must be of type String. For example: 

          
           
@KafkaListener(topics = "my-topic")
public void processMessage(String content) {
    System.out.println(content);
}
This code will simply print out the contents of the message to the console.

Error Handling

You can also add error-handling logic to your KafkaListener. This is done by adding a second parameter to your method, of type ConsumerRecord. This parameter will contain any errors that were received with the message. For example:

Conclusion

The org.springframework.kafka.annotation.KafkaListener annotation is a convenient way to create a Kafka message listener. This annotation allows you to quickly configure your message listener, as well as provide custom error-handling logic.
