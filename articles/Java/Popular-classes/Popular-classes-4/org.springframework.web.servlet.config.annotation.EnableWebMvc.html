Introduction to org.springframework.web.servlet.config.annotation.EnableWebMvc

The org.springframework.web.servlet.config.annotation.EnableWebMvc annotation is part of the Spring framework, which allows developers to quickly and easily set up web application development with Java. This annotation is used to enable and configure the Spring's Web MVC (Model-View-Controller) support. In particular, it enables annotation-driven configuration of MVC components such as controllers, view resolvers and handler mappings, interceptors, and formatters.

This article provides an overview of the various features and properties of the EnableWebMvc annotation and provides examples of how to use it in web applications.

Enabling Annotation-Driven Configuration for Web MVC

To enable annotation-driven configuration for a Spring web MVC application, one only needs to add the @EnableWebMvc annotation to a configuration class:


          
           
@Configuration 
@EnableWebMvc 
public class AppConfig { 
    // ... 
}

This will automatically configure features in the same way that is done through the more verbose and manual XML configuration. However, since the annotation-based approach is more convenient, it comes with some trade-offs, such as having less control over how components are configured.

View Resolver

By default, the EnableWebMvc annotation sets up a view resolver in order to map views (HTML, JSP, etc.) to controllers. The view resolver is configured in the same way as it would be through the XML configuration:


          
           
@Configuration 
@EnableWebMvc 
public class AppConfig { 
    @Bean
    public ViewResolver viewResolver() { 
        InternalResourceViewResolver resolver = new InternalResourceViewResolver(); 
        resolver.setPrefix("/WEB-INF/views/"); 
        resolver.setSuffix(".jsp"); 
        return resolver; 
    } 
}

The view resolver can also be configured using the @ViewResolver annotation, which takes a String representing the class name of the view resolver as a parameter.

Handler Mapping

The EnableWebMvc annotation also sets up a handler mapping which is used to map request URLs to methods within the controller classes. By default, the annotation uses the RequestMappingHandlerMapping class, which can be overridden by setting the handlerMappingClass property of the annotation.

In addition to the handler mapping, the annotation also allows for the customization of URL patterns for specific resources. The urlPatterns property can be used to specify patterns for static resources such as images, CSS, or JavaScript files. 


          
           
@Configuration 
@EnableWebMvc 
public class AppConfig { 
    @Bean
    public WebMvcConfigurer webMvcConfigurer() { 
        return new WebMvcConfigurer() { 
            public void addResourceHandlers(ResourceHandlerRegistry registry) {
                registry.addResourceHandler("/static/**") 
                        .addResourceLocations("/resources/"); 
            }
        }; 
    }
}

Interceptors

The EnableWebMvc annotation also allows for the registration of custom interceptors. Interceptors can be used to pre-process requests before they are handled by controllers, as well as post-process responses before they are returned to the client. Interceptors can register themselves as beans in a configuration class:


          
           
@Configuration 
@EnableWebMvc 
public class AppConfig { 
    @Bean
    public HandlerInterceptor loggingInterceptor() { 
        return new LoggingInterceptor(); 
    } 
}

Formatters

The EnableWebMvc annotation also allows for the registration of custom formatters. Formatters are used to convert objects to and from text representations, such as when converting an object to JSON or an XML representation. Formatters can register themselves as beans in a configuration class:


          
           
@Configuration 
@EnableWebMvc 
public class AppConfig { 
    @Bean 
    public Formatter jsonFormatter() { 
        return new JsonFormatter(); 
    } 
}

Conclusion

The org.springframework.web.servlet.config.annotation.EnableWebMvc annotation is a powerful annotation for quickly and easily enabling and configuring Web MVC in a Spring application. It allows for annotation-driven configuration of various components such as view resolvers, handler mappings, interceptors, and formatters. The annotation can also be used to customize configuration settings such as URL patterns. This article provided an overview of the features of the annotation and showed examples of how to use it in web applications.
