org.springframework.context.ApplicationEventPublisher

The ApplicationEventPublisher is an interface used in Spring applications to asynchronously publish events. It provides support for publish-subscribe pattern, allowing components to subscribe to an event source and publish events that those listeners can receive.

This interface is particularly useful when an application wants to decouple its event sources from the components that want to listen to them. For example, if a library needs to expose certain events to its users, it can do so by defining an event publisher and notifying the users of its existence. Then, any user of the library can register as a listener for those events without having to know about the library's inner workings.

The ApplicationEventPublisher interface consists of the following methods:


          
           
public void publishEvent(ApplicationEvent event);
public void publishEvent(Object event);
public void publishEvent(Object event, ResolvableType eventType);
public boolean containsListener(Class<?> eventType);
public void notifyListeners(ApplicationEvent event);
public void notifyListeners(Object event, ResolvableType eventType);

publishEvent() Method

The publishEvent() method is used to asynchronously publish an event. The event is sent to all registered listeners that are subscribed to the same event type. This method can be called with either an ApplicationEvent or an Object.

Using The publishEvent() Method With An ApplicationEvent

When using the publishEvent() method with an ApplicationEvent, we must ensure that the provided event is an instance of ApplicationEvent:


          
           
if (event instanceof ApplicationEvent) {
    applicationContext.publishEvent(event);
}

applicationContext.publishEvent(event);

applicationContext.publishEvent(event, ResolvableType.forClass(MyEvent.class));

boolean hasListeners = applicationContext.containsListener(MyEvent.class);

if (event instanceof ApplicationEvent) {
    applicationContext.notifyListeners(event);

applicationContext.notifyListeners(event);
