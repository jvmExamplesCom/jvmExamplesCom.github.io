javax.ws.rs.DefaultValue

The javax.ws.rs.DefaultValue annotation is used in the Java programming language to designate a default value for a class field, method or constructor parameter.

Using DefaultValue annotation

DefaultValue can be used on field, methods and constructor parameters. The following code illustrates a field with a specified default value: 


          
           
@DefaultValue("foo")
private String myField = "bar";

The following code illustrates a method with its parameter annotated with a default value: 


          
           
@GET
public Response doGet(@DefaultValue("50") int limit) {
    // do something
}

CalledBy @DefaultValueProcessor

The DefaultValue annotation is processed by the DefaultValueProcessor, which is a sub-class of AnnotationProcessor. 


          
           
@DefaultValueProcessor
public class MyAnnotationProcessor extends AnnotationProcessor {
    // implementation details
}

Typing and conversion of default values

The DefaultValue annotation supports typed default values, such as Integer, Long, Float, etc. It also supports automatic type conversion for default values. 


          
           
@DefaultValue("12")
private Integer myField = 10; // will be converted from String to Integer

Default values in resource methods

DefaultValue annotation can be used in resource methods to provide default value for URI query parameters which are not mandatory. 


          
           
@GET
public Response doGet(@DefaultValue("50") int limit) {
    // do something
}

This allows the client to omit the value of the query parameter if they don't have it (or if they have an empty value). The server will use the default value provided in the DefaultValue annotation.
