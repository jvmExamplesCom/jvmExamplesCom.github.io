org.springframework.web.servlet.config.annotation.ViewControllerRegistry

The ViewControllerRegistry class is a helper class used to register Spring MVC view controllers. It can also be used for simple URL path mapping. This registry helps in configuring view controllers without writing explicit controller classes. View controllers enable the user to define request mappings to views without writing a controller.

Code Example – Register Single View Controllers


          
           
@EnableWebMvc
@Configuration
public class MyConfig extends WebMvcConfigurerAdapter {

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/about")
                .setViewName("about");
    }
}

Output

The above code example registers a single view controller for the 


          
           
/about
URL path. Upon requesting the 

          
           
/about
URL, the view associated with 

          
           
about
(e.g. about.jsp or about.html) will be used to render the response.

Code Example – Register Multiple View Controllers


          
           
@EnableWebMvc
@Configuration
public class MyConfig extends WebMvcConfigurerAdapter {

    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addViewController("/about")
            .setViewName("about");
        registry.addViewController("/contact")
            .setViewName("contact");
        registry.addViewController("/faq")
            .setViewName("faq");
    }
}

Output

The above code example registers multiple view controllers for the 


          
           
/about
, 

          
           
/contact
and 

          
           
/faq
URL paths. Upon requesting any of the above mentioned URL paths, the appropriate view will be used to render the response.
