androidx.fragment.app.DialogFragment

The AndroidX library's DialogFragment class is a convenient way to show an alert dialog or other embedded fragment. This class was introduced in API Level 11 (Honeycomb) to support the fragmentation of Android devices. It is based on the support library's DialogFragment but with additional features and optimizations, such as animation support and a simpler API.

Why Use DialogFragment?

DialogFragment provides a high degree of flexibility when it comes to displaying a dialog. It helps you keep your UI clean and organized by providing a single point of entry into the fragment manager. It also allows you to control which dialogs are visible and to animate them when they are shown or dismissed. With DialogFragment, you can display modal dialogs without needing to worry about the device's configuration changes or device orientation.

Creating a DialogFragment

To create a DialogFragment, you need to extend the class and override the onCreateDialog() method. This method returns a Dialog instance that you can use to customize the look and feel of the dialog. For example, you can set the title, message, and buttons for the dialog. You can also set the theme of the dialog, as well as create and add custom views.

Displaying a DialogFragment

Once you've created a DialogFragment, you can display it by calling the show() method. This method takes a FragmentManager and an optional tag as parameters. If you don't provide a tag, an automatic tag will be generated. When the show() method is called, the onCreateDialog() method is called to create the dialog, and then the dialog is displayed.

Here is an example of how to use DialogFragment to create and display an alert dialog. The following code creates a simple alert dialog with a message, an OK button, and a Cancel button. 

          
           
public class MyDialogFragment extends DialogFragment{

    @Override
    public Dialog onCreateDialog(Bundle savedInstanceState){
        AlertDialog.Builder builder = new AlertDialog.Builder(getActivity());
        builder.setMessage("This is an alert dialog")
            .setPositiveButton("OK", new DialogInterface.OnClickListener() {
                public void onClick(DialogInterface dialog, int id) {
                   //do something
                }
            })
            .setNegativeButton("Cancel", new DialogInterface.OnClickListener() {
                public void onClick(DialogInterface dialog, int id) {
                    // do something
                }
            });
        return builder.create();
    }
}

Showing And Dismissing A DialogFragment

To show a DialogFragment, you need to call the show() method. To dismiss a DialogFragment, you can either call the dismiss() method directly, or call the remove() method to remove the DialogFragment from the fragment manager. 

          
           
MyDialogFragment dialogFragment = new MyDialogFragment();
dialogFragment.show(getFragmentManager(), "MyDialogFragment");
// ....
dialogFragment.dismiss();
// or

Animating A DialogFragment

DialogFragment supports animations when it is shown or dismissed. To animate a DialogFragment, you need to create a style resource file in your project. Here is an example of a style resource file that defines a simple left-to-right animation for a dialog: 

          
           
<style name="MyDialogAnimation" parent="@android:style/Animation.Dialog">
    <item name="android:windowEnterAnimation">@anim/slide_left_in</item>
    <item name="android:windowExitAnimation">@anim/slide_right_out</item>
</style>
You can then apply the animation to a DialogFragment using the setStyle() method. The following example shows how to create a DialogFragment and apply the animation defined in the style resource file: 

          
           
DialogFragment dialogFragment = new MyDialogFragment();
dialogFragment.setStyle(DialogFragment.STYLE_NO_FRAME, R.style.MyDialogAnimation);
dialogFragment.show(getFragmentManager(), "MyDialogFragment");
