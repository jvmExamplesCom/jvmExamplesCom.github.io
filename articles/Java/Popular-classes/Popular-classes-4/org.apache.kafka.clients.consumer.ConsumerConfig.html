org.apache.kafka.clients.consumer.ConsumerConfig

org.apache.kafka.clients.consumer.ConsumerConfig is an interface that provides the configuration of a Kafka Consumer. It is used to configure a consumer's behavior when it connects to a Kafka cluster for message consumption, such as fetching messages, committing offsets and manual rebalancing. In this article, we will discuss the different components of the interface and look at some code examples.

Configuring the Consumer

The ConsumerConfig object is used to configure the consumer when connecting to the cluster. This is done by setting the appropriate configuration keys and their respective values. There are a wide range of configuration options available, which can be found in the documentation. The most important keys are:

  • bootstrap.servers:
    An array of strings that list the hosts and ports of the brokers in the Kafka cluster. If a broker fails, the consumer will automatically connect to another broker. This key is required.
  • group.id:
    The identifier of the consumer group to which the consumer belongs. All members of a consumer group share their offsets and coordinate their message consumption. If a consumer belongs to a group, then it's mandatory to set this key.
