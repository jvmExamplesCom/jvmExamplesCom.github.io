java.util.Observer

The java.util.Observer interface is a key component in the Java-based Observer design pattern. This design pattern allows an object to be notified of changes in another object.

The Observer interface defines a single method, update, which takes two parameters: an observable object and a data object. The observable is typically the object that is being observed (for example, a button being clicked), while the data object can contain arbitrary data that the observer uses to determine what action to take.

The following example shows how to use the Observer interface to respond to a button click. The MyButton class contains a field of type java.util.Observer, and when the button is clicked the observer's update() method is called. In this example, the data object is simply the String "clicked":


          
           
public class MyButton {
    private Observable observer;

    public MyButton(Observable observer) {
        this.observer = observer;
    }

    public void onClick() {
        observer.update(this, "clicked");
    }
}
We can define an observer like this: 

          
           
public class MyObserver implements Observer {
    public void update(Observable o, Object arg) {
        System.out.println("Button was clicked!");
    }
}

Finally, we can use these classes in our program like this:


          
           
MyObserver observer = new MyObserver();
MyButton button = new MyButton(observer);
button.onClick();

This will cause the MyObserver's update() method to be called, which will print out "Button was clicked!" to the console.

Summary

The java.util.Observer interface is an important part of the Observer design pattern, which allows one object to be notified of changes in another object. The Observer interface has a single update() method, which takes an observable object and a data object as parameters. The data object allows the observer to determine what action to take when the observable object changes.
