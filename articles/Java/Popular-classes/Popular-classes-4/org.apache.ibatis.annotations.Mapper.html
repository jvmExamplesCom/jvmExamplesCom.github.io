org.apache.ibatis.annotations.Mapper

org.apache.ibatis.annotations.Mapper is an annotation for the mapper interface in MyBatis programming. The annotation is used to specify the fully qualified name of the Mapper interface. It also enables automatic population of configuration information into the MapperInfo object, which is used by the MyBatis framework.

Annotation Syntax

The MyBatis Mapper annotation is declared as follows:


          
           
@Mapper
public interface MyMapper{ 
    //My Mapper methods 
}

The @Mapper annotation specifies the fully qualified name of the Mapper interface. In this example, it is "MyMapper".

Automatic Configuration Population

When the org.apache.ibatis.annotations.Mapper annotation is present, the MyBatis framework will automatically populate the configuration information into the MapperInfo object. This includes the class name and any defined parameters.

For example, if we define a parameter named "id" in the MyMapper interface:


          
           
@Mapper
public interface MyMapper{ 
   public void getRecordById(int id); 
}

Then the MapperInfo object will contain the parameter name and type. It will also contain the fully qualified name of the interface, which can be used to locate the method at runtime.

Usage Examples

Let's look at an example of how we can use the org.apache.ibatis.annotations.Mapper annotation. We will create a Mapper interface, annotate it with @Mapper, and then use MyBatis to create a SQL query that will retrieve a record from a database using the parameters defined in the Mapper interface.


          
           
// Create the Mapper Interface 
@Mapper
public interface MyMapper{ 
    public void getRecordById(int id); 
}

// Use MyBatis to create a SQL query 
String sql = session.getConfiguration() 
    .getMappedStatement("MyMapper.getRecordById") 
    .getSqlSource() 
    .getBoundSql(id).getSql();

// Execute the query 
session.selectList(sql, id);

In this example, we have used the org.apache.ibatis.annotations.Mapper annotation to specify the fully qualified name of our Mapper interface. We then used the MapperInfo object to locate the SQL query associated with our Mapper method. Finally, we used the bound sql to execute the query against the database.
