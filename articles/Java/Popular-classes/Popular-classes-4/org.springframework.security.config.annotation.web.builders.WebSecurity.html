org.springframework.security.config.annotation.web.builders.WebSecurity

The 

          
           
org.springframework.security.config.annotation.web.builders.WebSecurity
class provides access to a fluent API for configuring and creating an instance of the Spring Security web security filter chain. It allows developers to configure web security settings in code, rather than in XML files.

Example Usage

The following example demonstrates how to create an instance of the 

WebSecurity
class and use its fluent API to configure and create an instance of the Spring Security web security filter chain. 

          
           
WebSecurity web = new WebSecurity();
web
    .ignoring()
        // ignore url patterns
        .antMatchers("/resources/**")
    .and()
        // enable anonymous access
        .anonymous()
            .authorities("ROLE_USER")
    .and()
        // secure url patterns
        .authorizeRequests()
            .antMatchers("/user/**")
                .hasRole("USER")
            .anyRequest()
                .authenticated()
    .and()
        // add http basic authentication
        .httpBasic();

FilterChainProxy filterChainProxy = web.getSecurityFilterChain();

Filter Chain Creation

Once the web security configuration is complete, the filter chain proxy can be created using the 


          
           
getSecurityFilterChain()
method. This method creates a 
FilterChainProxy
instance that wraps the configured security filters. This instance can then be used as a servlet filter within an application. 

          
           
FilterChainProxy filterChainProxy = web.getSecurityFilterChain();

Configuration Options

The 
WebSecurity
class provides a number of methods for configuring the filter chain created by the 

          
           
getSecurityFilterChain()
method. These include: 
  • ignoring()
    - Specifies URL patterns to be ignored by the filter chain.
  • anonymous()
    - Configures settings for anonymous users.
  • authorizeRequests()
    - Declares URL rules that require authentication.
  • httpBasic()
    - Enables HTTP Basic authentication.
  • formLogin()
    - Enables form-based authentication.
  • x509()
    - Enables X.509 certificate-based authentication.
