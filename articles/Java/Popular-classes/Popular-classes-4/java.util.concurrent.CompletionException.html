Java.util.concurrent.CompletionException

Java.util.concurrent.CompletionException is an unchecked exception introduced in Java 8. It is used to indicate that a task execution has failed with an exception. It can be thrown from methods such as the CompletableFuture.get() and CompletionStage.join() methods.

Example 3: Application Exception Handler

In this example, we will create an application exception handler that will convert all checked exceptions into CompletionExceptions. This will ensure that all checked exceptions are handled uniformly throughout the application.


          
           
public static void handleException(Throwable ex) {
    if (ex instanceof Exception) {
        throw new CompletionException(ex);
    } else {
        throw (Error) ex;
    }
}

This exception handler will take in any Throwable object and, if it is an Exception, convert it into a CompletionException. If it is an Error, it will be thrown as is. The application can then handle the CompletionException in one unified manner.
