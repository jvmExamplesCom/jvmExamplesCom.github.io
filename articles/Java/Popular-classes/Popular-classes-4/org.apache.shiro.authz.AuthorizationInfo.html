Introduction to org.apache.shiro.authz.AuthorizationInfo

The org.apache.shiro.authz.AuthorizationInfo interface is part of the Apache Shiro security framework and is used to provide information regarding a user's authorization status. This includes access control lists (ACLs) and access control vectors (ACV). It is an essential component in Shiro’s authentication and authorization process.

Methods of AuthorizationInfo

The AuthorizationInfo interface contains several methods for providing specific authorization information about a particular user:

  • getStringPermissions(): Gets a list of string-based permissions granted to a user.
  • getRoles(): Gets a list of roles associated with a user.
  • isPermitted(): Checks if a user has permission to perform the specified action.
  • isRole(): Checks if a user is part of a specified role.

Example: Basic Usage of AuthorizationInfo

The following example demonstrates a basic usage of the AuthorizationInfo interface. We will create an application that checks a user's AuthorizationInfo and returns a boolean indicating if they have permission to do an action, such as delete a file or add a comment to a blog post. 


          
           
Subject currentUser = SecurityUtils.getSubject(); // Get the currently logged in user.

if (permitted) {
  deleteFile();

}
