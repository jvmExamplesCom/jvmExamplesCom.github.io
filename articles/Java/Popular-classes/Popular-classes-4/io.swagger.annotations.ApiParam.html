An Introduction to io.swagger.annotations.ApiParam

The io.swagger.annotations.ApiParam annotation is used to add additional meta-data to an application's RESTful API, such as additional information about what parameters are accepted and what errors may be returned. As a result, this annotation is most often used in conjunction with other RESTful annotation libraries, such as JAX-RS or Spring MVC. In this article, we will discuss the basics of using ApiParam and provide some examples to get you started.

Defining an ApiParam

To define an ApiParam, the following general syntax is used:


          
           
@ApiParam(name = "myParameter", required=true, description="This is my parameter")

Where the values supplied to the annotation correspond to the following properties:

  • name: the name of the parameter that is being described.
  • required: whether or not this parameter must be supplied in order for the request to succeed.
  • description: a description of the parameter and its purpose.

Additionally, the @ApiParam annotation has several optional properties, which can be used to further specify how a parameter should function:

  • allowableValues: a list of specific values that this parameter can accept (e.g.,[true, false, null]).
  • format: a string describing the expected format of this parameter (e.g., date-time).
  • allowEmptyValue: a boolean value indicating if empty values are allowed.
  • dataType: the type of data that this parameter is expected to contain (e.g., string, number).

Examples

Now that we have discussed the basics of ApiParam, let's look at some examples.

In the first example, we will define a parameter called "name" that is required and must contain a string value:


          
           
@ApiParam(name = "name", required = true, description = "The name of the user", dataType = "string")

In this example, we will define a parameter called "age" that is optional and must contain a number between 18 and 65:


          
           
@ApiParam(name = "age", required = false, description = "The age of the user", dataType = "number", allowableValues = "range[18,65]")

Finally, we will define a parameter called "favoriteColor" that is optional and must contain one of the following colors: red, green, or blue:


          
           
@ApiParam(name = "favoriteColor", required = false, description = "The user's favorite color", allowableValues = "[red, green, blue]")

Conclusion

As you can see, the io.swagger.annotations.ApiParam annotation allows us to add additional meta-data to our applications' RESTful APIs, providing meaningful documentation and additional information that can be used to validate requests and send appropriate error responses.

We hope this article has been helpful in introducing you to the use of this annotation and provided some useful examples to get started with.
