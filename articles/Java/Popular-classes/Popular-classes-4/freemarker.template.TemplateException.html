Using the TemplateException Class

The TemplateException class is a subclass of RunTimeException. It is used to indicate a syntax error in a template. The constructor of the TemplateException class takes a String parameter (error message) and optionally a Throwable parameter (cause).

Catching TemplateExceptions

Any Java code that uses FreeMarker templates should include a try-catch block to handle any potential TemplateExceptions that may occur. The code block should look like this: 

          
           
try {

} catch (TemplateException e) {
    // Exception handling code goes here
}
When the catch block is triggered, the error message associated with the TemplateException will be output and the cause can be accessed using the getCause() method.
