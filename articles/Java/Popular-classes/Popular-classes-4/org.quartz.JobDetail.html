org.quartz.JobDetail

The org.quartz.JobDetail class is a Java class that provide the ability to define jobs and their associated detail information. The class allows for Jobs to be written in Java, rather than relying on configuration files. This makes for a more programmatic approach to developing Job definitions. The class also has a built in mechanism for scheduling of jobs.

The core of the org.quartz.JobDetail class is the definition of the Job itself. This can be done by implementing the org.quartz.Job interface, or by extending the org.quartz.JobDetail class and overriding the execute() method.

Creating an Instance of org.quartz.JobDetail

The first step in creating a job is to create an instance of org.quartz.JobDetail in your code. This is done by calling the constructor, passing in the Class of the Job you want to execute, as well as the unique name of the Job. 


          
           
JobDetail jobDetail = new JobDetail("myJob", MyJob.class);

Setting Job Data Map

The next step is to set any data that needs to be passed to the job. This is done by creating a JobDataMap and putting values into it such as application configuration, job details, and any other data that needs to be passed to the job. 


          
           
JobDataMap jobDataMap = new JobDataMap();
jobDataMap.put("confDir", "config/");
jobDetail.setJobDataMap(jobDataMap);

Defining Job Execution

The final step is to define when and how the job will be executed. This is done by adding a Trigger to the JobDetail. You can use one of the pre-defined Triggers, or create your own. 


          
           
Trigger trigger = TriggerBuilder
    .newTrigger()
    .withIdentity("myTrigger")
    .withSchedule(CronScheduleBuilder.cronSchedule("0 0/5 * * * ?"))
    .build();

jobDetail.setTrigger(trigger);

Executing Job

Once all of the necessary steps have been completed, the job can be executed by calling the execute() method on the org.quartz.JobDetail object.
