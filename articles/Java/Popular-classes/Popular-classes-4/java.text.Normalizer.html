java.text.Normalizer

Java has a powerful utility class that is used to normalize Unicode text to the canonical decomposition form, Normalizer. It is located in the java.text package of the Java Development Kit (JDK). This class provides methods that help in dealing with Unicode text, such as converting characters to their composite forms and stripping accents.

What Is Normalization?

Normalization is the process of reducing a character or sequence of characters to a single form. This is achieved by removing combining characters or Canonical Combining Classes (CCC) from the string before performing a comparison. For example, the latin character “a” can be represented in several ways: either as a single character (LATIN SMALL LETTER A), in a composed form (LATIN SMALL LETTER A WITH ACUTE), or in a decomposed form (LATIN SMALL LETTER A followed by COMBINING ACUTE ACCENT). These representations are all equivalent, but when performing comparisons on strings, it is necessary to ensure that the same representation is used, otherwise the comparison may yield unexpected results.

Using the Normalizer Class

The Normalizer class provides methods for converting characters to their canonical decompositions and stripping accents. It also provides convenience methods for testing if two strings are equivalent, such as isEquivalent(), and for testing if a string is in its canonical decomposition, such as isNormalized().

Example 1: Converting Characters to Canonical Decompositions

In this example, we use the Normalizer.normalize() method to convert a string of characters to their canonical decompositions. Here, we convert a string containing the latin letter “a” to its decomposed form. 

          
           
String originalString = "a"; 
String normalizedString = Normalizer.normalize(originalString, Normalizer.Form.NFD);
System.out.println("Original String: " + originalString);
System.out.println("Normalized String: " + normalizedString);

Original String: a
Normalized String: á
