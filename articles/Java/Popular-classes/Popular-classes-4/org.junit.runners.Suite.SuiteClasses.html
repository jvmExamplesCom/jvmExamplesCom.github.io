org.junit.runners.Suite.SuiteClasses

The 

          
           
org.junit.runners.Suite.SuiteClasses
annotation is used to identify classes that should be run as part of a test suite. This annotation is used in conjunction with the 

          
           
@RunWith(Suite.class)
annotation in order to create a test suite.

Example Setup

A typical setup of a test suite using this annotation would look something like this:


          
           
@RunWith(Suite.class)
@SuiteClasses({
   TestClassOne.class, 
   TestClassTwo.class
})
public class TestSuite { }

Using SuiteClasses with Parameters

It is also possible to pass in parameters to the test classes that are specified with the 

@SuiteClasses
annotation. In order to do this, the parameter must adhere to the following format: 

          
           
@SuiteClasses({
    TestClassOne.class, 
    TestClassTwo.class
})
public static List<TestClassOne> getTestClassOneParameters() {
    return Arrays.asList(
        new TestClassOne("param1"),
        new TestClassOne("param2"),
        new TestClassOne("param3")
    );
};

public static List<TestClassTwo> getTestClassTwoParameters() {
    return Arrays.asList(
        new TestClassTwo("param1"),
        new TestClassTwo("param2"),
        new TestClassTwo("param3")
    );
};
The parameters can then be used as follows: 

          
           
@Parameters
public static List<Object[]> parameters() {
    return Arrays.asList(
        new Object[] {
            getTestClassOneParameters(), 
            getTestClassTwoParameters()
        }
    );
}
