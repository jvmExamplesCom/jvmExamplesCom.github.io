org.springframework.context.annotation.Lazy

The org.springframework.context.annotation.Lazy annotation is used to indicate that a bean should be lazily initialized. By default, when a Spring application context is loaded, all of the beans within it are initialized. Using Lazy, you can specify that a certain bean should not be initialized until it is specifically requested by another bean.

The Lazy annotation can be used to reduce startup time and memory consumption when the application starts. It can also be used to isolate certain components from a global application context. For example, if two components both depend on each other, initializing them at the same time can cause errors due to the order in which they are initialized. By using Lazy, one component can be initialized first, which allows the other component to be initialized in a controlled manner.

Examples of Lazy Annotation

The following code example shows how the Lazy annotation can be used to lazily initialize a bean.


          
           
@Component
@Lazy
public class MyBean {
    public void doSomething() {...}
}

In this example, the MyBean bean will not be initialized until it is specifically requested. This is useful if the MyBean bean requires other beans to be initialized before it can be used.

If both beans had the Lazy annotation, then neither would be initialized until they were requested. This could cause errors due to the order in which they are initialized.

Conclusion

The org.springframework.context.annotation.Lazy annotation can be used to control the initialization of beans within a Spring application context.
