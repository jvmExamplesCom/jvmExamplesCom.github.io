java.junit.framework.AssertionFailedError

The java.junit.framework.AssertionFailedError class is an error thrown when an assertion fails while running a JUnit test. It is a subclass of the java.lang.Error class and signals that an assertion has failed.

Creating an AssertionFailedError

The AssertionFailedError class does not define any constructors and does not have any methods apart from toString(). This means creating an instance of this class is done by simply catching the error in your finally block. 

          
           
try {
   ...
} catch (AssertionFailedError e) {
   //do something
} finally {
   throw new AssertionFailedError("Assertion Failed");
}
If no message is specified, the default message "Assertion Failed" will be printed.

Example Usage

In JUnit, assertions are used to make sure that a certain condition is true before executing a test. An AssertionFailedError is thrown if the assertion fails and signals that the test should fail. For example, assume we are trying to test if a String has a length greater than zero: 

          
           
assertTrue(string.length() > 0);
If the length of the string is not greater than zero, an AssertionFailedError will be thrown and the test will fail. 

          
           
try {
   assertTrue(string.length() > 0);
} catch (AssertionFailedError e) {
   //do something
}
In this case, the exception is caught and can be handled accordingly. In most cases, this will simply propagate the exception and cause the test to fail.
