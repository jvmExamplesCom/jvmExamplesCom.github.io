org.apache.http.config.Registry

The org.apache.http.config.Registry class provides an interface that allows users to create and manage collections of configuration objects. It is used in HttpClient to store and manage a collection of various types of related objects, such as authentication schemes and connection socket factories.

Code Examples

Here are some code examples to demonstrate how to use the Registry class in your applications.


          
           
Registry<AuthSchemeProvider> authSchemeRegistry = null;
 
authSchemeRegistry = RegistryBuilder.<AuthSchemeProvider>create()
        .register("basic", new BasicSchemeFactory())
        .register("digest", new DigestSchemeFactory())
        .register("ntlm", new NTLMSchemeFactory())
        .register("negotiate", new SPNegoSchemeFactory())
        .build();

This example above shows how to create a Registry of authentication scheme providers.


          
           
Registry<ConnectionSocketFactory> socketRegistry = null;

This code creates a Registry of connection socket factories.


          
           
RequestConfig requestConfig = RequestConfig.custom().setCookieSpec(CookieSpecs.DEFAULT).build();
HttpClientContext context = HttpClientContext.create();
context.setRequestConfig(requestConfig);
context.setAttribute("http.authscheme-registry", authSchemeRegistry);
context.setAttribute("http.socket-factory-registry", socketRegistry);

The above code sets the Registry objects as attributes on the HttpClientContext. This is used to configure the client for authentication and connection sockets.
