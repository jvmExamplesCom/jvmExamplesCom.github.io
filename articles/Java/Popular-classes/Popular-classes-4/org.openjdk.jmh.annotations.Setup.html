Introduction to org.openjdk.jmh.annotations.Setup

The org.openjdk.jmh.annotations.Setup annotation is used in Java Microbenchmark Harness (JMH) to indicate when a benchmark method should be run. It is an annotation provided with the JMH framework and must be used when writing a benchmark class. This annotation can be used on a single method or on several methods at once.

Code Examples


          
           
@Setup
public void setup() {
  // code to run before benchmark
}

The above example shows how the annotation is used, with a method named "setup" that is annotated with @Setup. This method will be called once before each benchmark.


          
           
@Setup(Level.Iteration)
public void setup() {
  // code to run before every iteration
}

In this example, the Level.Iteration option is used. This indicates that the method will be called before each iteration of the benchmark, rather than only once before the entire benchmark run.


          
           
@Setup(Level.Invocation)
public void setup() {
  // code to run before every invocation
}

In this example, the Level.Invocation option is used. This indicates that the method will be called before each invocation of the benchmark, rather than only once before the entire benchmark run.

Conclusion

The org.openjdk.jmh.annotations.Setup annotation tells the JMH framework when to run a benchmark method. It is important to understand how to use the annotation for accurate benchmark results.
