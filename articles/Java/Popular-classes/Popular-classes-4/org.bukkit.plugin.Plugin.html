Introduction to org.bukkit.plugin.Plugin

The 

          
           
org.bukkit.plugin.Plugin
class is the core component of a Bukkit Plugin. It represents the main entry point for a plugin, and contains hooks and methods used to control the plugin's execution. In this article, we'll explore the Plugin class in detail, with lots of code examples.

Constructing a Plugin Object

To construct a Plugin object you must first write your own subclass extending 

Plugin
. Here's an example: 

          
           
public class MyPlugin extends Plugin {

    public void onEnable() {
        // This method will be called when the server starts
    }

    public void onDisable() {
        // This method will be called when the server stops
    }
}

Once you've written your subclass, you can create an instance of it and register it with the server. This is done using the 


          
           
Server.getPluginManager().registerEvents(Plugin, Server)
method.

Plugin Methods

Plugin
class contains a number of useful methods. These are: 
  • getDataFolder()

  • getDescription()

  • getName()
    - Returns the name associated with this plugin.
  • getServer()
    - Returns the server associated with this plugin.

Plugin Events

Plugin
class also contains methods for receiving plugin events. The two most common are 
onEnable()

onDisable()
. These methods will be called when the plugin is enabled and disabled, respectively. Here's an example of how to use them: 

          
           
public class MyPlugin extends Plugin {

    }

        // This method will be called when the server stops
    }
}

org.bukkit.plugin.Plugin
