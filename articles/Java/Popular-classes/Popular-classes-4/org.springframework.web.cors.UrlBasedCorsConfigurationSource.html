org.springframework.web.cors.UrlBasedCorsConfigurationSource

The UrlBasedCorsConfigurationSource class is a Java library that provides a way to configure Cross-Origin Resource Sharing (CORS) using URL-based mappings to enable cross-domain requests. This class extends the CorsConfigurationSource interface, which implements the algorithm used by Spring's CORS filter.

Using the UrlBasedCorsConfigurationSource allows an application to set up CORS support for individual paths, as well as globally for the entire application, and gives more control over the specific configuration options available, such as allowed origins and headers, exposed headers, allowed request methods, and more.

Example Usage

The following example shows the basic usage of the UrlBasedCorsConfigurationSource class. The example sets up two paths with different CORS configurations: one for /foo and one for all other paths. Both paths allow only GET requests from any origin, with no additional headers or request methods specified.


          
           
UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();

CorsConfiguration config1 = new CorsConfiguration();
config1.setAllowCredentials(true);
config1.addAllowedOrigin("*");
config1.addAllowedHeader("*");
config1.addAllowedMethod("GET");
source.registerCorsConfiguration("/foo", config1);

CorsConfiguration config2 = new CorsConfiguration();
config2.setAllowCredentials(true);
config2.addAllowedOrigin("*");
config2.addAllowedHeader("*");
config2.addAllowedMethod("GET");
source.registerCorsConfiguration("/**", config2);

The UrlBasedCorsConfigurationSource can be used to configure more complex CORS configurations, such as allowing different origins, request methods, and headers for each path.
