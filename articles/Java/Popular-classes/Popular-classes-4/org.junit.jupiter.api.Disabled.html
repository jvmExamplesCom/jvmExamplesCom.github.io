The org.junit.jupiter.api.Disabled annotation is an annotation type used to disable a test, test class, or test factory for execution. It is primarily used to indicate that a particular test or tests should be skipped and not ran during execution. This annotation has no parameters.

Using Disabled Annotation

Using the disabled annotation is quite simple and straightforward. All you need to do is put the annotation before the test method or class like this:


          
           
@Disabled
public void testMethod() {
    //test code
}

This will cause the test method to be skipped during the execution process.

You can also use the disabled annotation on a test factory. To do this, the disabled annotation will need to be added after the @TestFactory annotation like this:


          
           
@TestFactory

    //test code
}

Why Use Disabled Annotation?

The Disabled annotation is useful if you have a test that is currently failing, but you don't want to delete it from the suite. By disabling the test, the test will be skipped and the test suite will execute as normal, with no errors thrown. It can also be used in conjunction with the timeout annotation so that tests that are taking too long to complete are automatically skipped.

The disabled annotation is also helpful when working in a team environment. By disabling certain tests, other team members won't be able to accidentally run them or cause them to fail in their local environment. This can help prevent unexpected errors from occurring in the finished product.
