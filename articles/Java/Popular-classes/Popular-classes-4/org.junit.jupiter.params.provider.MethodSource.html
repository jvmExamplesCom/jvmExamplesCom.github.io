Introduction to JUnit Parameterized Tests using org.junit.jupiter.params.provider.MethodSource

JUnit parameterized tests allow us to easily run the same test multiple times with different parameters. This is useful in situations where we need to make sure that our code behaves correctly in various scenarios, or when we need to identify errors in our code with data-driven testing. JUnit 5 provides a powerful API for this, in the form of the org.junit.jupiter.params.provider.MethodSource class. This class allows us to create parameterized tests that use a data source from a given method. In this article, we’ll look at how to use org.junit.jupiter.params.provider.MethodSource for creating data-driven tests in JUnit 5. We’ll also take a look at some examples of the class in action.

Parameterized Tests with org.junit.jupiter.params.provider.MethodSource

The org.junit.jupiter.params.provider.MethodSource class provides a way to create parameterized tests that use data from a given method as a data source. This allows us to easily create tests that can accept different sets of data and run them multiple times with different parameters. To use MethodSource, all we need to do is add the @MethodSource annotation to a JUnit 5 test. This annotation takes the name of a method that will provide the data for the test. The method should return either an Iterable, a Collection or a Stream of Arguments. The Arguments class is a JUnit 5 convenience class that can be used to store the parameters passed to a test. Let’s take a look at an example of using org.junit.jupiter.params.provider.MethodSource in action: 

          
           
@ParameterizedTest
@MethodSource("dataProvider")
public void testData(String data1, int data2) {
    // Test code here
}

private static Stream<Arguments> dataProvider() {
    return Stream.of(
        Arguments.of("data1", 1), // Data set 1
        Arguments.of("data2", 2)  // Data set 2
    );
}
In this example, we have created a parameterized test that uses the dataProvider method as a data source. The dataProvider method returns a Stream of Arguments, which contains the data that will be used in the test. The first argument in the Arguments object is the data1 parameter, while the second argument is the data2 parameter. When this test is executed, it will run twice – once with data1 set to “data1” and data2 set to 1, and once with data1 set to “data2” and data2 set to 2.

Using MethodSource for More Complicated Data Sources

The org.junit.jupiter.params.provider.MethodSource class also provides support for more complicated data sources. For example, we can provide an Iterable, a Collection, or even a Stream of data as a data source. 

          
           
@ParameterizedTest
@MethodSource("dataProvider")
public void testData(String data1, int data2) {
    // Test code here
}

private static Iterable<Arguments> dataProvider() {
    List<Arguments> data = new ArrayList<>();
    data.add(Arguments.of("data1", 1));  // Data set 1
    data.add(Arguments.of("data2", 2));  // Data set 2
   
    return data;
}
In this example, we have used an Iterable as a data source for our parameterized test. We have created a List of Arguments, which contains the data that will be used in the test. The first argument in the Arguments object is the data1 parameter, while the second argument is the data2 parameter. When this test is executed, it will run twice – once with data1 set to “data1” and data2 set to 1, and once with data1 set to “data2” and data2 set to 2.

Conclusion

In this article, we looked at how to use org.junit.jupiter.params.provider.MethodSource to create parameterized tests in JUnit 5. We saw how to use the @MethodSource annotation to create tests that use a data source from a given method. We also took a look at how to use more complicated data sources, such as an Iterable or a Stream. Parameterized tests are a powerful tool for creating data-driven tests in JUnit 5, and the org.junit.jupiter.params.provider.MethodSource class provides an easy way to create them.
