javax.persistence.Temporal Annotation

The javax.persistence.Temporal annotation is used to mark a field as containing a temporal value, such as a date or time. This annotation can be applied to either a field or property of type java.util.Date and java.util.Calendar.

Using the Annotations

The Temporal annotation allows the application developer to specify which parts of the java.util.Date or java.util.Calendar should be stored in the database. The datetime can be stored as either DATE, TIME, or TIMESTAMP as defined by the TemporalType enum. 

          
           
@Temporal(TemporalType.DATE)
private Date dateField;

@Temporal(TemporalType.TIME)
private Calendar timeField;

@Temporal(TemporalType.TIMESTAMP)
private Date timestampField;

Example Use Case

Let's consider an example where we have a User entity with a registration date and a last login date: 

          
           
@Entity
public class User {
	@Temporal(TemporalType.DATE)
	private Date registrationDate;

	@Temporal(TemporalType.TIMESTAMP)
	private Date lastLogin;
}
When the User entity is persisted to the database, the registrationDate field will only store the date portion of the Date object and the lastLogin field will store both the date and the time.

Conclusion

The javax.persistence.Temporal annotation is a powerful feature that allows us to control exactly how our temporal data is stored in the database. This annotation allows us to store only the piece of information we need, such as just the date or just the time, making our code more efficient and our data easier to query and manipulate.
