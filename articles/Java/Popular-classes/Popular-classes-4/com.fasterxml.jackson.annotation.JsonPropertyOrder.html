com.fasterxml.jackson.annotation.JsonPropertyOrder

The 

          
           
com.fasterxml.jackson.annotation.JsonPropertyOrder
annotation is used to define the order in which properties are serialized during a Jackson marshalling process. It provides an easy way to control ordering of properties by simply providing a list of property names as strings in order, to the annotation.

Usage Example

Consider the following Java class: 

          
           
@JsonPropertyOrder({ "name", "age", "location" })
public class Person {
	private String name;
	private int age;
	private String location;

	// Getters and setters for all fields
}

If we would then use Jackson to serialize an instance of this class, the output could look like this:


          
           
{"name":"John Doe","age":42,"location":"New York"}

That is because the output follows the order that was defined by the 

JsonPropertyOrder
annotation. The order can be easily modified if needed by simply reordering the array elements.

Nestable Annotation

The annotation itself is nestable, which means that it can be applied to the fields of the annotated class directly. This allows us to have different ordering for different fields:


          
           
@JsonPropertyOrder({ "name", "age" })
public class Person {
	@JsonPropertyOrder({ "street", "zipcode" })
	private Address address;

	// Getters and setters for all fields
}

In this example the order for the Person class would be 

name
and 
age
, but for the 
Address
class it would be 
street
and 
zipcode
.

Renaming Properties

The annotation can also be used to rename properties when they are serialized. This is done using the 

@JsonProperty
annotation in combination with the 
@JsonPropertyOrder
annotation: 

          
           
@JsonPropertyOrder({ "firstname", "lastName" })
public class Person {
	@JsonProperty("firstName")
	private String firstName;

	@JsonProperty("lastName")
	private String lastName;

	// Getters and setters for all fields
}
In this case, the 
firstName
and 
lastName
properties will be serialized as 
firstName
and 
lastName
instead of the actual field names.

Conclusion

The 
JsonPropertyOrder
annotation is a great way to easily control the order of properties when serializing objects with Jackson. The annotation also allows for nested annotations, which makes it possible to specify different orders for different classes. Finally, the annotation can also be used to rename properties for serialization.
