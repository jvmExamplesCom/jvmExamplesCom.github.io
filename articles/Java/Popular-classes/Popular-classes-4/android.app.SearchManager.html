Android.app.SearchManager

Android provides the 

          
           
android.app.SearchManager
class. It allows for system-wide information and user command input searches to be performed across different applications. This class can be used to trigger global search, register searchable activities, create and manage search suggestions, and create and manage an on-device search index database.

Getting Started

Before you can use the 

          
           
android.app.SearchManager
class, you must initialize it in your application. The following code shows how to do that: 

          
           
SearchManager searchManager = (SearchManager) getSystemService(Context.SEARCH_SERVICE);

Trigger Global Search

The 

          
           
SearchManager
class includes a 

          
           
startSearch()
method that enables you to trigger a global search from within your application. The following code shows how to use the 

          
           
startSearch()
method: 

          
           
searchManager.startSearch(query, false, componentName, null, false);
The first argument is the query string that the user should enter, the second argument is a boolean value indicating whether the search should be triggered in global mode or not. The third argument is the name of the 

          
           
ComponentName
of the activity that should receive the search intent, the fourth argument is an optional 

          
           
Bundle
. The fifth argument indicates whether the searchable activity should be launched as a new task in the back stack or as part of the activity calling 

          
           
startSearch()
.

Register Searchable Activities

You can register searchable activities in your AndroidManifest.xml file. This allows other applications to launch the searchable activity directly without having to know the 

          
           
ComponentName
. The following code shows the declaration of a searchable activity in the AndroidManifest.xml file: 

          
           
<activity android:name="SearchableActivity">
    <intent-filter>
        <action android:name="android.intent.action.SEARCH" />
    </intent-filter>
    <meta-data android:name="android.app.searchable"
               android:resource="@xml/searchable"/>
</activity>
Where 

          
           
android:resource="@xml/searchable"
refers to a searchable XML resource file that contains information about the searchable activity. This information includes the searchable configuration, the default search text, and the results we want to be returned.

Create Search Suggestions

SearchManager
class to create search suggestions for any query entered by the user. To do this, you must implement a content provider that extends from 

          
           
SearchRecentSuggestionsProvider
and override the 

          
           
onCreate()
method to setup the content provider and define the search version and authority. The following code shows how to do this: 

          
           
public void onCreate() {
    super.onCreate();
    setupSuggestions("example.search.PROVIDER", DATABASE_MODE_QUERIES);
}
Once you have created the content provider, you can use the 

          
           
SearchManager
class to query the content provider using the 

          
           
getSuggestions()
method. The following code shows how to do this: 

          
           
Cursor cursor = searchManager.getSuggestions(query, DATABASE_MODE_QUERIES);
Where the first argument is the query string, and the second argument is the search version. This will return a 

          
           
Cursor
object containing the search suggestions.

Create and Manage On-Device Search Index Database

You can use the 

          
           
SearchManager
class to create and manage an on-device search index database. By creating an index database, applications can create better search experiences for users. The 

          
           
SearchManager
class includes the 

          
           
newSearchIndexer()
method that enables you to create an on-device search index database. The following code shows how to use the 

          
           
newSearchIndexer()
method: 

          
           
SearchIndexer searchIndexer = searchManager.newSearchIndexer();
Once the index is created, you can use the 

          
           
SearchIndexer
class to update, delete, and query the index database. The following code shows how to use the 

          
           
updateIndex()
method to update the index database: 

          
           
searchIndexer.updateIndex(data, selection);
Where 

          
           
data
is an array of 

          
           
SearchIndexableData
objects and 

          
           
selection
is a boolean value indicating whether the update should be applied to all documents in the index or only to the documents that match the selection.
