Android Support Annotation: IntRange

The Android Support Annotation 

          
           
@IntRange
was introduced to the Android development system in Android 9 (API 28) and offers developers a way to create more specific annotations when they are coding. By utilizing this annotation, developers are ensured that any integer value used in the code falls within a specified range of values. This allows for better accuracy, control, and organization in the code. In this article, we will explore what IntRange is, how it works and how to apply it in your code.

What is IntRange?

The @IntRange annotation defines an inclusive range of values for an integer parameter or return value. It uses two parameters, min and max, to set the lower and higher limits of the range. The value must be greater or equal to min and less than or equal to max. If either of these parameters is absent, the corresponding limit is assumed to be unbounded. If both min and max are absent, the @IntRange is ignored and the method can take any value. For example: 

          
           
@IntRange(min=-10, max=20)  
 public int getValue(){  
   int value = 15;  
   return value;  
 }
This will ensure that the returned value is between -10 and 20. Values outside that range will trigger a compile-time error.

How does IntRange work?

The @IntRange annotation is read by the compiler, which will then check each integer parameter or return value against the limits defined in the annotation. If any integer value is outside the limits specified in the annotation, the compiler will throw a compile-time error. For example, if the following method is called: 

          
           
getValue();
And its implementation is as follows: 

          
           
@IntRange(min=-10, max=20)  
 public int getValue(){  
   int value = 25;  
   return value;  
 }
It will result in a compile-time error saying that the value is out of the range defined in the annotation.

How to use IntRange in your code

Using the @IntRange annotation is quite simple. All you need to do is add the annotation to the method signature before declaring the method. The format for adding the @IntRange annotation is as follows: 

          
           
@IntRange(min=val1, max=val2)  
 public int getValue(){  
   // method body  
 }
Where 

          
           
val1
is the lower limit of the range and 

          
           
val2
is the higher limit of the range. The annotation should be added before the return type of the method. You can also specify an optional message parameter in the annotation if you wish to include a custom message that will be shown with the compile-time error. The format for this is as follows: 

          
           
@IntRange(min=val1, max=val2, message="Your custom error message")  
 public int getValue(){  
   // method body  
 }
The @IntRange annotation is a useful tool for ensuring the accuracy and correctness of integer values in your code. By using this annotation, you can easily check that all parameters and return values are within the upper and lower bounds defined in the annotation. This offers developers more control over their code and helps prevent errors.
