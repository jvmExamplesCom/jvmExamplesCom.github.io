org.bukkit.event.EventHandler

The EventHandler is an annotation used to denote a method as a handler for events. It is used to mark methods in a class, such that when an event is fired, all methods marked with the EventHandler annotation will be called in turn.

Example Usage

The following example shows a class that has two methods annotated with the EventHandler annotation. The first one will handle all PlayerInteractEvent events, and the second will handle all PlayerJoinEvent events:


          
           
public class EventHandlerExample {
    @EventHandler
    public void onPlayerInteract(PlayerInteractEvent event) {
        // Do something when a player interacts with something
    }

    @EventHandler
    public void onPlayerJoin(PlayerJoinEvent event) {
        // Do something when a player joins the server
    }
}

To register a class so that it can be called when an event is fired, use the PluginManager API:


          
           
PluginManager pm = getServer().getPluginManager();
pm.registerEvents(new EventHandlerExample(), this);

Once the EventHandlerExample class is registered, all methods annotated with the EventHandler annotation will be called when the corresponding event is fired.

Using Event Priority

The EventHandler annotation also allows you to specify the priority of a handler. This allows plugins to specify when they want their handlers to be called relative to other plugins. The priority options are as follows:


          
           
HIGHEST
HIGH
NORMAL
LOW
LOWEST

The default priority is NORMAL, but it can be changed by passing in one of the above constants as an argument to the EventHandler annotation:


          
           
@EventHandler(priority = EventPriority.HIGH)
public void onPlayerJoin(PlayerJoinEvent event) {
    // Do something when a player joins the server
}

Conclusion

The EventHandler annotation is an easy way to organize and manage your plugin's event handlers. It allows plugins to specify what events they are interested in, and also allows them to specify the priority at which their handlers should be called. Using this annotation is a great way to make sure that your plugin's event handlers are called at the right time.
