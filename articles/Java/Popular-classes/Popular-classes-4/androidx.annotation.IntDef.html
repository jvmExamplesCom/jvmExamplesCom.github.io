androidx.annotation.IntDef

androidx.annotation.IntDef is an annotation used for annotating types that are restricted to a limited set of integral constants. It helps to prevent errors by ensuring that valid values are passed to the methods. This annotation is used to specify constants as method parameters and helps to avoid runtime errors when passing incorrect values.

Example Usage

Letâ€™s take an example of an Android application which needs to play audio from a media source. We can define constants for the possible media sources and use them in the code. 

          
           
public static final int SOURCE_MICROPHONE = 1;  
public static final int SOURCE_HEADSET = 2;  
public static final int SOURCE_SPEAKER = 3;
The constants can then be annotated with the @IntDef annotation as follows: 

          
           
@IntDef({SOURCE_MICROPHONE, SOURCE_HEADSET, SOURCE_SPEAKER})  
public @interface AudioSource {}
This approach ensures that only valid values are passed to the methods. For example, if we have a method which takes an AudioSource as parameter: 

          
           
public void playAudio(@AudioSource int source) {  
  // Play audio from the specified source  
}
If an invalid value (e.g. 4) is passed as argument to this method, the compiler will show an error as the argument violation is detected at compile time rather than runtime.

Compile-Time Constants

The @IntDef annotation also provides an additional advantage of compile-time constant usage. Since the constants are specified in the source code, the compiler can automatically replace them with the respective values at compile time. For example, if the following code is compiled with @IntDef annotation: 

          
           
public void playAudio(@AudioSource int source) {  
  if (source == SOURCE_MICROPHONE) {  
    // Do something  
  }  
}
The compiler will replace SOURCE_MICROPHONE with 1 at compile time and the resulting code will look like this: 

          
           
public void playAudio(int source) {  
  if (source == 1) {  
    // Do something  
  }  
}
This concept reduces the bytecode size significantly and improves the runtime performance as no type checking occurs at runtime.

@StringDef Annotation

The @StringDef annotation works similarly to @IntDef except that it enables compile-time checks for String-based constants. The following example shows how to use it: 

          
           
public static final String SOURCE_MICROPHONE = "microphone";   
public static final String SOURCE_HEADSET = "headset";  
public static final String SOURCE_SPEAKER = "speaker";  

@StringDef({SOURCE_MICROPHONE, SOURCE_HEADSET, SOURCE_SPEAKER})  
public @interface AudioSource {}  

public void playAudio(@AudioSource String source) {  
  // Play audio from the specified source  
}
Similar to IntDef, this annotation ensures that the passed argument satisfies the specified criteria and gives compile-time errors on violations.

Conclusion

In this article, we discussed the androidx.annotation.IntDef and androidx.annotation.StringDef annotations. These annotations help to prevent runtime errors by ensuring that valid values are passed to the methods and replace the constants with the respective values to reduce the bytecode size and improve the runtime performance.
