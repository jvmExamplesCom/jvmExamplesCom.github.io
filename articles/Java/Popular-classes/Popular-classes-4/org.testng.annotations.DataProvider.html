org.testng.annotations.DataProvider

The org.testng.annotations.DataProvider annotation is one of the many annotations in TestNG that can be used to provide data for a test method. With this annotation, we can easily pass various inputs to the same test method from a data source, such as an Excel spreadsheet or a JSON file. This allows us to automate the same test with different input values and make assertions on the expected output.

Using DataProvider Annotation

To use the DataProvider annotation, we need to provide it with the name of the data source as an argument. If our data source is an Excel spreadsheet, we can do the following: 

          
           
@DataProvider(name = "dataSource")
public Object[][] getDataFromExcel() throws Exception {
 // code to read data from the Excel spreadsheet 
}
Now, whenever we want to use this data provider to provide input to a particular test method, all we have to do is to add the annotation: 

          
           
@Test(dataProvider = "dataSource")
public void testMethod(String param1, String param2) {
  // code to execute the test
}
The parameters of the test method are mapped to the columns of the data source (the Excel spreadsheet in this case). We can then access these parameters within the test method and use them as input for the test.

Example

Let's consider a scenario where we need to test a method that takes two Strings as input and returns the concatenated value of both the strings. We can easily create a unit test for this scenario using DataProvider by passing different inputs from an Excel spreadsheet. First, let's create an Excel spreadsheet with the following data: 

          
           
A1    B1
abc   def
ghi   jkl
Next, we'll create two methods - one to read data from the Excel spreadsheet and another to test the concatenation logic: 

          
           
@DataProvider(name = "dataSource")
public Object[][] getDataFromExcel() throws Exception {
 // code to read data from the Excel spreadsheet 
}

@Test(dataProvider = "dataSource")
public void testConcatenation(String param1, String param2) {
   String result = param1 + param2; 
   Assert.assertEquals("abcdef", result);
   Assert.assertEquals("ghijkl", result);
}
In this example, the DataProvider method reads the data from the Excel spreadsheet and passes the values of both the cells to the test method. The test method then performs the concatenation and asserts that the result is as expected. By using the DataProvider annotation, we can easily pass different input values to the same test without having to write separate tests for each set of data. This makes writing automated tests more efficient and less time-consuming.
