org.springframework.web.bind.annotation.PutMapping

PutMapping is a type of annotation used in the Spring framework for web application development, to map HTTP requests to specific handler methods. It is a convenient alternative to the @RequestMapping annotation for mapping HTTP PUT requests onto Java methods. The @PutMapping annotation is a composed annotation that combines the @RequestMapping annotation and the HTTP method type PutMapping.

Example

The following is an example of using the @PutMapping annotation. Here, the method updateUser() is mapped to the URL pattern myUrl:


          
           
@PutMapping("/myUrl")
public ResponseEntity<User> updateUser(@RequestBody User user){
    // Code to update user goes here
    return new ResponseEntity<>(updatedUser, HttpStatus.OK);
}

When sending an HTTP PUT request to the specified URL, it will be handled by the updateUser() method. The code inside this method will be executed and the user object sent as part of the request will be updated accordingly.

Other Ways of Using @PutMapping

The @PutMapping annotation can be used to map multiple URLs to the same handler method. For example:


          
           
@PutMapping({"/myUrl1", "/myUrl2"})
public ResponseEntity<User> updateUser(@RequestBody User user){
    // Code to update user goes here
    return new ResponseEntity<>(updatedUser, HttpStatus.OK);
}

In this example, both /myUrl1 and /myUrl2 will be mapped to the same handler method. That means, when an HTTP PUT request is sent to either of those URLs, the code inside the updateUser() method will be executed and the user object sent as part of the request will be updated accordingly.

Multiple Request Parameters

The @PutMapping annotation can also be used to map multiple request parameters to the same handler method. For example:


          
           
@PutMapping("/myUrl")
public ResponseEntity<User> updateUser(@RequestParam(name="userId") String userId, 
                                      @RequestParam(name="name") String name, 
                                      @RequestParam(name="email") String email){
    // Code to update user goes here
    return new ResponseEntity<>(updatedUser, HttpStatus.OK);
}

In this example, the handler method updateUser() will be mapped to the URL /myUrl and it will be able to handle requests with three different parameters: userId, name, and email. If an HTTP PUT request is sent to /myUrl with these three parameters, the code inside the updateUser() method will be executed and the user object sent as part of the request will be updated accordingly.

Conclusion

PutMapping is a type of annotation used in the Spring framework for web application development, to map HTTP requests to specific handler methods. It is a convenient alternative to the @RequestMapping annotation for mapping HTTP PUT requests onto Java methods. It can be used to map multiple URLs to the same handler method and also to map multiple request parameters to the same handler method.
