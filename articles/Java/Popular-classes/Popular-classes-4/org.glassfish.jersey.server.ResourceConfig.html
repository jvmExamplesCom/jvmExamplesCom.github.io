org.glassfish.jersey.server.ResourceConfig

The org.glassfish.jersey.server.ResourceConfig class is a Java configuration class used to register components (such as resources, providers and features) to be deployed to web applications. This class provides access to the underlying Jersey runtime environment and acts as a bridge between the Java world and the web application. It also allows for customizations of the Jersey runtime for different web applications.

Instantiating ResourceConfig

The ResourceConfig class can be instantiated like any other Java class:


          
           
ResourceConfig config = new ResourceConfig();

A ResourceConfig can also be instantiated using a Builder pattern:


          
           
ResourceConfig config = ResourceConfig.builder()
        .build();

Alternatively, a ResourceConfig can be created from an existing Environment object:


          
           
ResourceConfig config = ResourceConfig.forEnvironment(env);

Registering Components with ResourceConfig

Components (such as resources, providers and features) are registered with the ResourceConfig object, which serves as a bridge to the web application. To register a component, you must first create an instance of the component and then pass it to the ResourceConfig object. For example, to register a resource class:


          
           
Resource resource = new Resource();
config.register(resource);
Similarly, to register a provider: 

          
           
Provider provider = new Provider();
config.register(provider);
To register a feature: 

          
           
Feature feature = new Feature();
config.register(feature);

Components can also be registered using their corresponding Java classes. For example, to register a resource class:

Deploying Components to the Web Application

Once components have been registered with the ResourceConfig object, they can be deployed to the web application. This can be done using the deploy() method on the ResourceConfig object:
