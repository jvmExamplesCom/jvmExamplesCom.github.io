Introduction to org.greenrobot.eventbus.Subscribe

The 

          
           
org.greenrobot.eventbus.Subscribe
annotation is a method-level annotation used for registering subscribers on the EventBus and receiving event messages from the bus. The annotation requires the method to be public, and to accept only one argument. Annotated methods are called by the EventBus when a new event of their declared type is posted to the bus.

Usage of org.greenrobot.eventbus.Subscribe

The 

          
           
org.greenrobot.eventbus.Subscribe
annotation is a convenient way of connecting a component to an EventBus. In order to use it, first a class must register itself as a subscriber by calling the 

          
           
register(Object)
method of the EventBus. We then mark a method as an event handler by annotating it with the annotation. 

          
           
MySubscriber subscriber = new MySubscriber();
EventBus bus = EventBus.getDefault();
bus.register(subscriber);
We can then annotate methods in the MySubscriber class as event handlers: 

          
           
@Subscribe
public void onEvent(MyEvent event) {
    // logic for handling the event
}
Once the event handler is registered, whenever an instance of MyEvent is posted to the bus, the EventBus will call the onEvent() method with the posted event instance as an argument.

Advanced Usage

The 

          
           
org.greenrobot.eventbus.Subscribe
annotation provides some additional features. The annotation can take arguments that allow us to specify the thread mode and individual subscribers can be prioritized.

Thread Mode

org.greenrobot.eventbus.Subscribe
annotation has an optional argument called 

          
           
threadMode
that accepts the 

          
           
ThreadMode
enum as its value. It is used to specify which thread the event handler method should be executed on. By default, the event will be delivered on the same thread that posted the event. The 

          
           
ThreadMode

POSTING,
MAIN,
MAIN_ORDERED,
BACKGROUND,
ASYNC,
DEFAULT

Priority

The 

          
           
org.greenrobot.eventbus.Subscribe
annotation also has an optional argument called 

          
           
priority
that allows us to prioritize individual subscribers. When multiple subscribers are subscribed to the same event type, the prioritized subscribers will be called before the non-prioritized ones. The argument takes an integer value as its parameter and a higher value indicates a higher priority.
