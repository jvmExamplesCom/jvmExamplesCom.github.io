Java.nio.file.attribute.PosixFilePermission

The Java.nio.file.attribute.PosixFilePermission class allows the user to set permissions on a file system object like a file or directory. It is used to manage Access Control Lists (ACLs) and set permissions that are applied to an object. This class can be used to control who may read, write, execute, or change specific files or directories.

Features

The Java.nio.file.attribute.PosixFilePermission class provides several features that allow a user to customize the access rights of a particular file or directory. These features include:

  • Setting read, write, and execute permissions for each file or directory.
  • Setting the owner and group of the object.
  • Setting the user and group mask.
  • Managing Access Control Lists (ACLs).

Example

The following code example shows how to use the Java.nio.file.attribute.PosixFilePermission class to set read, write, and execute permissions on a file.


          
           
Path path = Paths.get("/home/user/myfile.txt");
Set<PosixFilePermission> perms = PosixFilePermissions.fromString("rwxr-x---");
Files.setPosixFilePermissions(path, perms);

The permissions will be set to rwxr-x---, meaning that the user has full permissions, while the group and other users have only read (r) and execute (x) permissions.
