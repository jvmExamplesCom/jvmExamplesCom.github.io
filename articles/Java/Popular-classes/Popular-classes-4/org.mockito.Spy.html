Mockito.Spy

Mockito.Spy is a Java library used to create spies, or mocks of real objects. Spies are objects that keep track of all interactions with the real object, allowing for deep introspection into the inner workings of the application. Mockito.Spy provides an easy-to-use API for creating and managing spies, as well as providing functionality for asserting and verifying the behavior of the spied object.

Creating a Spy

To create a spy, simply use the Mockito.spy() method, passing in the class of the object to be spied upon. For example, to spy on a List:


          
           
List list = Mockito.spy(List.class);

Verifying Behavior

Once a spy has been created, it can be used to verify interactions with the spied object. To do so, Mockito.Spy provides a suite of assert* and verify* methods, which accept Hamcrest matchers or plain old arguments. For example:


          
           
verify(list).add("Foo"); //Asserts that the list was called with the argument "Foo"
assertThat(list, hasSize(1)); //Asserts that the list contains one element

Spy API

Mockito.Spy also provides a number of convenience methods for working with spies. For example:

  • reset() - Resets the state of the spy to its initial state
  • getInvocations() - Returns a list of all invocations made on the spy
  • setDefaultAnswer() - Sets a default answer to return when an unknown invocation is made on the spy

Examples

Here are a few examples of how Mockito.Spy can be used to introspect and verify the behavior of an object:


          
           
//Create a spy on a List
List list = Mockito.spy(List.class);

//Add some elements to the list
list.add("Foo");
list.add("Bar");
list.add("Baz");

//Verify the list was called with the expected elements
verify(list).add("Foo"); 
verify(list).add("Bar"); 
verify(list).add("Baz"); 

//Check that the list contains the expected elements
assertThat(list, hasSize(3));
assertThat(list, contains("Foo", "Bar", "Baz"));
