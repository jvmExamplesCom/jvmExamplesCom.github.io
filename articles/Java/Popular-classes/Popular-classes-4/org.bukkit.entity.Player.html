Overview of org.bukkit.entity.Player

The org.bukkit.entity.Player class represents a player in the game. It is used to manipulate and interact with the player, such as getting information about them or changing their state.

Methods

The org.bukkit.entity.Player class has a few methods which can be used to query and manipulate the player. The 

          
           
getName()
method returns the name of the player as a String. The 

          
           
getUniqueId()
method returns the UUID of the player as a UUID object. The 

          
           
getGameMode()
method returns the GameMode of the player as an org.bukkit.GameMode object. The 

          
           
setGameMode(GameMode mode)
method sets the GameMode of the player to the given GameMode object. The 

          
           
isOp()
method returns a boolean value indicating whether the player has operator status. The 

          
           
setOp(boolean value)
method sets the operator status of the player to the given boolean value. The 

          
           
sendMessage(String message)
method sends the given message to the player.

Example Usage

Here is an example of how to use the org.bukkit.entity.Player class to send a message to the player and then set their GameMode: 

          
           
String playerName = player.getName();
UUID uuid = player.getUniqueId();
player.sendMessage("Hello " + playerName + "! Your UUID is: " + uuid.toString());
player.setGameMode(GameMode.SURVIVAL);
This code snippet sends a message to the player with their name and UUID, and then sets their GameMode to SURVIVAL.
