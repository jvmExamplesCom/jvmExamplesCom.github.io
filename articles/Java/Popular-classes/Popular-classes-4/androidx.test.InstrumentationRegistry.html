androidx.test.InstrumentationRegistry

AndroidX Test InstrumentationRegistry provides access to the current Instrumentation and target application package. It is available since AndroidX Test version 1.1.0 and serves as the main entry point for the application under test and any test code that needs access to the application's environment.

What Is androidx.test.InstrumentationRegistry?

In short, it's an Androidx Test extension that allows for accessing two main classes: Instrumentation and PackageManager. The Instrumentation class is used in conjunction with Appium, an automation testing tool intended for mobile applications, and defines an interface for interacting with the runtime environment of the app. It provides access to resources such as test arguments and the context of the application under test. The PackageManager class is used to discover information about the application, such as the application's manifest and its declared permissions. This can be used to check permissions or APIs that are specified in the manifest file associated with the given application.

How To Use androidx.test.InstrumentationRegistry

To use the androidx.test.InstrumentationRegistry class, start by adding the dependency to your build.gradle file: 

          
           
implementation 'androidx.test:core:1.1.0'
You can then access the InstrumentationRegistry class like so: 

          
           
Instrumentation instrumentation = InstrumentationRegistry.getInstrumentation();
From here, you can use the getContext() method to get the context of the application under test, getTargetContext() to get the context of the target application, and getArguments() to access any test arguments provided. You can also access the PackageManager class like so: 

          
           
PackageManager packageManager = instrumentation.getContext().getPackageManager();
You can then use this to get any information from the application, such as its declared permissions. For example: 

          
           
String[] permissions = packageManager.getDeclaredPermissions(packageName);
The packageName should correspond to the package of the application being tested.

Conclusion

androidx.test.InstrumentationRegistry is a useful utility for Android developers looking to automate tests using Appium. It provides access to the Instrumentation and PackageManager classes, allowing developers to get data regarding the application under test.
