org.eclipse.jetty.server.Request

The org.eclipse.jetty.server.Request class is a class from the Jetty Web Server framework which helps to handle incoming HTTP requests. It's composed of methods and fields which can be used to examine the request, and respond accordingly.

Constructors

public Request(HttpChannel channel, HttpInput input)
public Request(HttpChannel channel, HttpInput input, Object attribute)
public Request(HttpChannel channel, HttpInput input, long idleTimeout)
The first constructor creates a new Request configured on the given HttpChannel, with the specified HttpInput. The second constructors also create a new Request configured on the given HttpChannel, with the specified HttpInput and an additional attribute set. The third constructor creates a Request configured on the given HttpChannel, with the specified HttpInput and an idle timeout set.

Methods

The org.eclipse.jetty.server.Request class provides several methods to process and respond to incoming requests. Some of these important methods are: 


          
           
public void dispatch() throws IOException
public void forward(String path) throws ServletException
public void forward(ServletContext context, String path) throws ServletException 
public void include(String path) throws ServletException

method is used to dispatch the request to another destination, without returning a response. The

include()
method includes another resource in the response before the response is returned to the client.

Example

Here's a simple example of how to use the org.eclipse.jetty.server.Request class to process and respond to an incoming request: 


          
           
public void handleRequest(Request request, Response response) {
  String path = request.getRequestURI();
  if (path.equals("/hello")) {
    response.setStatus(HttpServletResponse.SC_OK);
    response.setContentType("text/plain");
    response.getWriter().println("Hello World!");
  } else {
    response.sendError(HttpServletResponse.SC_NOT_FOUND);
  }
}
In this example, we examine the request's URI, and if it matches "/hello", then we respond with a "Hello World!" message, otherwise we send an error response.
