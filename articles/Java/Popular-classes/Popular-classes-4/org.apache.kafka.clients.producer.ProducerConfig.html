org.apache.kafka.clients.producer.ProducerConfig

The org.apache.kafka.clients.producer.ProducerConfig class provides configuration settings for a Kafka producer. This class contains setters and getters for the producer configuration settings such as batch size, linger time, request timeout, and so on. These settings allow you to configure how the producer sends data to the Kafka brokers.

In this article, we will look at the different ProducerConfig methods and learn how to use them with code examples.

Constructor


          
           
public ProducerConfig(Map<String, Object> configs) { 
    super(configs);
}

The ProducerConfig constructor takes a Map of configuration settings as its argument. These settings correspond to the available producer configuration options.

setBatchSize()


          
           
public ProducerConfig setBatchSize(int batchSize) { 
    this.batchSize = batchSize;
    return this;
}

The setBatchSize() method sets the maximum number of records that can be sent in a single batch. The default batch size is 16384 bytes.

setLingerMs()


          
           
public ProducerConfig setLingerMs(long lingerMs) { 
    this.lingerMs = lingerMs;
    return this;
}

The setLingerMs() method sets the time in milliseconds that the producer will wait before sending a batch of records to the broker. The default is 0. A larger linger time can result in better throughput since the producer can send more data in a single batch.

setMaxBlockMs()


          
           
public ProducerConfig setMaxBlockMs(long maxBlockMs) { 
    this.maxBlockMs = maxBlockMs;
    return this;
}

The setMaxBlockMs() method sets the maximum amount of time in milliseconds that the producer will block when sending a batch of records to the broker. If the producer does not receive an acknowledgement from the broker within this time period, it will block until the acknowledgement is received or the timeout is reached.

setMaxRequestSize()


          
           
public ProducerConfig setMaxRequestSize(int maxRequestSize) { 
    this.maxRequestSize = maxRequestSize;
    return this;
}

The setMaxRequestSize() method sets the maximum size of a request that the producer can send to the broker. This should be set to a value that is larger than the largest batch size the producer will send. The default value is 1 MB.

setRequestTimeoutMs()


          
           
public ProducerConfig setRequestTimeoutMs(int requestTimeoutMs) { 
    this.requestTimeoutMs = requestTimeoutMs;
    return this;
}

The setRequestTimeoutMs() method sets the amount of time that the producer waits for a response from the broker when sending a batch of records. If the timeout is reached before an acknowledgement is received, the request will be sent again. The default request timeout is 30 seconds.
