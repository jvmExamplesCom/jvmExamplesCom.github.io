com.android.volley.RequestQueue

The 

          
           
com.android.volley.RequestQueue
class is a core component of the Volley networking stack. It is used to manage the execution of HTTP requests, maintain an in-memory queue of pending requests, and control the priority and scheduling of those requests.

The RequestQueue has methods for adding and canceling requests, managing their priority, and requesting batches of responses.

Creating a RequestQueue Instance


          
           
// Instantiate the RequestQueue.
RequestQueue queue = new RequestQueue(new DiskBasedCache(getCacheDir()), new BasicNetwork(new HurlStack()));
The first parameter is a 
Cache
object which is used to maintain a list of responses from previous network requests. The second parameter is a 
Network
object that is responsible for performing the actual network communication. The 
HurlStack
class provided by Volley is a basic implementation which uses 
HttpURLConnection
to make its requests.

Adding Requests to the Queue

Requests are added to the RequestQueue using the 
add()
method. This method accepts a single parameter which is the request to be executed. The request will be added to the in-memory queue and can later be cancelled or removed using the 
cancel()
and 
remove()
methods.

// Add the request to the RequestQueue.

Modifying Request Priorities

By default, requests are added to the queue with a “normal” priority. The RequestQueue also supports the “low” and “high” priorities. Lower-priority requests are executed after higher-priority requests have been completed. Requests can be prioritized by calling the 

setPriority()
method on the request object before adding it to the queue. 

          
           
// Prioritize the request.
stringRequest.setPriority(Request.Priority.HIGH);

// Add the request to the RequestQueue.
queue.add(stringRequest);

Request Batches

The RequestQueue also provides a way to efficiently execute multiple requests in a single batch. Batch requests are created by passing an array of requests to the 

addBatch()
method. The requests in the batch are executed as a single unit and the result is returned when all requests have been completed. 

          
           
// Create the requests.
StringRequest request1 = new StringRequest(...);
StringRequest request2 = new StringRequest(...);
StringRequest request3 = new StringRequest(...);

// Add the requests to a batch.
Request[] batch = {request1, request2, request3};

// Execute the batch.

  }

Stopping the RequestQueue

The RequestQueue must be stopped when it is no longer needed. This can be done by calling the 

stop()
method. When the RequestQueue is stopped pending requests will not be executed and any that are currently executing will be cancelled. 

          
           
// Stop the RequestQueue.
queue.stop();
