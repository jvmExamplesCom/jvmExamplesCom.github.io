org.springframework.web.filter.OncePerRequestFilter

The org.springframework.web.filter.OncePerRequestFilter is a servlet filter that ensures a single execution per request. This can be used to avoid executing the same code more than once during a single page request, or to ensure that a certain piece of code is only executed once for each user on a particular page.

Example Usage

In the example below, we will use the OncePerRequestFilter to ensure that a certain block of code is only executed once for each user when loading a particular web page.


          
           
public class MyFilter extends OncePerRequestFilter {
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain) throws ServletException, IOException {
        // code that should only be executed once per request
        // ...
        filterChain.doFilter(request, response);
    }
}

Configuration

The OncePerRequestFilter can be configured via the web.xml file. In the example below, we configure the filter with the URL pattern "/myfilter/*" to limit requests to the filter to those URLs starting with "/myfilter/".


          
           
<filter>
    <filter-name>MyFilter</filter-name>
    <filter-class>com.example.MyFilter</filter-class>
</filter>
<filter-mapping>
    <filter-name>MyFilter</filter-name>
    <url-pattern>/myfilter/*</url-pattern>
</filter-mapping>

Output

The OncePerRequestFilter will execute its doFilterInternal() method once per request, independent of the URL matching. This means that if the same user makes multiple requests to the same URL, the filter will only be executed once for that user.
