java.nio.file.WatchEvent

Java's WatchEvent class is a powerful tool for monitoring filesystems for changes, allowing developers to create efficient programs that can react to changes made to files or folders with minimal overhead. In this article, we'll take a look at how to use the WatchEvent class for monitoring file system events.

What is a WatchEvent?

A WatchEvent is an instance of a WatchKey that is created when a program registers with the java.nio.file.WatchService. A watch key is a token that identifies the registration of a watch service with an object. It is possible to register objects in a file system to receive notification when changes occur. The WatchEvent class provides an interface to handle the events that occur on the registered object. It is an abstract class whose subclasses are specific to the filesystem event types, such as StandardWatchEventKinds.ENTRY_CREATE and STANDARD_WATCH_EVENTKINDS.ENTRY_MODIFY. The implementation for registering a watch service is provided by the operating system.

How to Use the WatchEvent Class

Using the WatchEvent class is relatively simple. First, a program must register with a watch service by calling the java.nio.file.WatchService.register() method. This method takes a Path object, which is the path to the directory or file to be monitored, as well as an array of WatchEvent.Kind objects, which are the specific types of events to be watched for. Once registered, the program can then call the WatchService.poll() method to obtain the next available WatchKey. This method will return a WatchKey instance which contains events associated with the registered object. The WatchKey class contains methods for obtaining the list of events from the key and handling each of them. For example, the WatchKey.pollEvents() method returns a List of WatchEvent\u2019s, where each event corresponds to a notification of a change in the registered object. The program should iterate through each of the events in the list and handle them appropriately. For example, if the type of an event is ENTRY_CREATE, then it could be handled by creating a new file in the same directory as the registered object.

import java.nio.file.Path;
import java.nio.file.WatchEvent;
import java.nio.file.WatchKey;
import java.nio.file.WatchService;

Register with the WatchService


          
           
// Get the path to the directory to be monitored
Path dir = FileSystems.getDefault().getPath("/path/to/dir");

// Get the watch service
WatchService watcher = FileSystems.getDefault().newWatchService();

// Register the watch service
dir.register(watcher, 
    StandardWatchEventKinds.ENTRY_CREATE, 
    StandardWatchEventKinds.ENTRY_MODIFY);

Get and Handle Events


          
           
while (true) {
    // Get the next watch key
    WatchKey watchKey = watcher.take();
    
    // Get list of events for the watch key
    for (WatchEvent<?> event : watchKey.pollEvents()) {

        // Get the type of event
        WatchEvent.Kind<?> kind = event.kind();
        
        // Handle the event 
        if (kind == StandardWatchEventKinds.ENTRY_CREATE) {
            // Create a new file
        }
        else if (kind == StandardWatchEventKinds.ENTRY_MODIFY) {
            // Modify existing file
        }
    }
    
    // Reset the watch key
    boolean valid = watchKey.reset();
    if (!valid) {
        break;
    }
}
