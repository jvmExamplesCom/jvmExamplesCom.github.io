org.apache.velocity.VelocityContext

The org.apache.velocity.VelocityContext class is a utility class used to facilitate the manipulation and access of data within the Velocity template engine. It acts as a container for key-value pairs of data, allowing objects to be accessed within the template with the use of context-specific methods. This article will provide a brief overview of the velocity context and an example of how it can be used in a template.

Overview

The VelocityContext class is the main data structure used by the Velocity template engine when it is processing a Velocity template. The aspects of this data structure, such as its methods and properties, are discussed in detail in the Velocity user guide. The purpose of the VelocityContext is to provide an interface for accessing and manipulating data within the Velocity template engine.

The VelocityContext is an implementation of the org.apache.velocity.context.Context interface, which is a set of methods that must be implemented by any object that is to act as the context for the Velocity template engine. The VelocityContext class is the most commonly used implementation of this interface, providing numerous convenience methods for manipulating data in the VelocityTemplateEngine.

Usage

The VelocityContext can be used in a template in multiple ways. The simplest way is to pass a context object into the template engine’s render() method. This ‘context’ object can simply be an instance of the VelocityContext class or a subclass of it, depending on the needs of the template. This context object will then be accessible within the template itself.

Other ways to use the VelocityContext in a template include setting the context directly in the template itself with the use of the “$context” directive, or passing the context into a macro invocation. For more information on these methods, please refer to the Velocity user guide.

Example

In this example, we will use a simple Velocity template containing two variables. The first variable, “name”, will be a string value, and the second variable, “number”, will be an integer value. We will instantiate a VelocityContext object and then use it to set the value of both variables in the template.


          
           
VelocityContext context = new VelocityContext();
context.put("name", "John Doe");
context.put("number", 42);

StringWriter out = new StringWriter();

String template = "Hello $name! Your number is $number";

Velocity.evaluate(context, out, "example", template);

System.out.println(out.toString());
