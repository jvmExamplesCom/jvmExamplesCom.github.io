org.bukkit.plugin.java.JavaPlugin

The 
JavaPlugin
class provides a way to create Plugin instances that are written in the Java programming language. It is the abstract base class for all Java-based Plugins and must be extended to create a functional Plugin.

Creating a Plugin

In order to create a Plugin, you need to extend the 

JavaPlugin
class in your Plugin's main class. Your main class should consist of the following parts: 

          
           
import org.bukkit.plugin.java.JavaPlugin;

public class MyPlugin extends JavaPlugin {
    // Code here
}
The 
JavaPlugin
class provides several methods that are important for creating and managing a Plugin. These methods include the following: 
  • onEnable():
    This method is called when the Plugin is enabled. This is where you should register any events or commands.
  • onDisable():
    This method is called when the Plugin is disabled. This is where you should unregister any events or commands.
  • getDataFolder():
    This method returns the Plugin's data folder. This is the folder where the Plugin can store any data it needs to persist between server restarts.
  • reloadConfig():
    This method reloads the Plugin's configuration file from the data folder.
  • getConfig():
    This method returns the Plugin's configuration object.
  • getLogger():
    This method returns the Plugin's logger object. This is used for logging messages to the server console.

Example Plugin

Below is an example of a Plugin that uses the 
JavaPlugin
class: 

          
           
import org.bukkit.plugin.java.JavaPlugin;

public class MyPlugin extends JavaPlugin {
    @Override
    public void onEnable() {
        getLogger().info("MyPlugin has been enabled!");
    }

    @Override
    public void onDisable() {
        getLogger().info("MyPlugin has been disabled!");
    }
}

This Plugin will log a message to the server console when it is enabled and when it is disabled.
