Introduction to org.springframework.web.bind.annotation.RestControllerAdvice

The org.springframework.web.bind.annotation.RestControllerAdvice annotation provides a convenient way to create a global exception handler for all controllers in a Spring application. This annotation can be used to define a base package to scan for all controller classes and configure a single exception handler class that will handle all exceptions thrown by the controllers.

Using RestControllerAdvice

To use @RestControllerAdvice, simply annotate your exception handler class with @RestControllerAdvice and specify the base packages to be scanned for controllers. Then, implement the handleException() method and handle any exceptions thrown by the controllers.


          
           
@RestControllerAdvice(basePackages = {"org.example.controllers"})
public class GlobalExceptionHandler {

    public ResponseEntity<ErrorResponse> handleException(Exception e) {
        // Handle exception here
    }
}

Example Usages

The following code examples show how to use the @RestControllerAdvice annotation to handle exceptions thrown by controllers.

Handling Bad Requests

This example shows how to handle a bad request exception by returning an appropriate response. 


          
           
@RestControllerAdvice(basePackages = {"org.example.controllers"})
public class GlobalExceptionHandler {

    public ResponseEntity<ErrorResponse> handleBadRequest(BadRequestException e) {
        ErrorResponse errorResponse = new ErrorResponse(e.getMessage());
        return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(errorResponse);
    }
}

Handling Not Found Exceptions

This example shows how to handle a not found exception by returning a 404 status code.


          
           
@RestControllerAdvice(basePackages = {"org.example.controllers"})
public class GlobalExceptionHandler {

    public ResponseEntity<ErrorResponse> handleNotFound(NotFoundException e) {
        ErrorResponse errorResponse = new ErrorResponse(e.getMessage());
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body(errorResponse);
    }
}

Handling Internal Server Errors

This example shows how to handle an internal server error by returning a 500 status code.


          
           
@RestControllerAdvice(basePackages = {"org.example.controllers"})
public class GlobalExceptionHandler {

    public ResponseEntity<ErrorResponse> handleInternalServerError(InternalServerErrorException e) {
        ErrorResponse errorResponse = new ErrorResponse(e.getMessage());
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(errorResponse);
    }
}

Conclusion

In conclusion, the @RestControllerAdvice annotation offers a convenient way to create a global exception handler for all controllers in a Spring application. With this annotation, you can define a base package to scan for all controller classes and configure a single exception handler class that will handle all exceptions thrown by the controllers.
