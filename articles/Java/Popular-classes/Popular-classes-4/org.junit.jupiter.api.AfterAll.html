org.junit.jupiter.api.AfterAll

The org.junit.jupiter.api.AfterAll annotation is used to signal that the annotated method should be executed after all test methods in the current test class have been executed. It helps ensure that resources required by tests (e.g. database connections, temporary files, etc.) are properly cleaned up and closed.

Simple Example

The following is a simple example of a test class with an AfterAll method:


          
           
@Test
public void testOne() {
    System.out.println("Starting test1");
    // Do something here
    System.out.println("Finished test1");
}

@Test
public void testTwo() {
    System.out.println("Starting test2");
    // Do something here
    System.out.println("Finished test2");
}

@AfterAll
public static void tearDown() {
    System.out.println("Performing tear down");
    // Clean up any resources if necessary
    System.out.println("Tear down finished");
}

Here, the tearDown method annotated with AfterAll will be executed once both testOne and testTwo have completed execution.

Output

When the above class is executed, the following output is produced:


          
           
Starting test1
Finished test1
Starting test2
Finished test2
Performing tear down
Tear down finished

Usage

While the AfterAll annotation is useful for cleaning up resources required by tests, it also has some limitations. Since the method annotated with AfterAll is static and cannot directly access instance variables, it cannot reliably depend on any test state that may have been created.

Additionally, when using AfterAll the order of test execution cannot be guaranteed. Therefore, itâ€™s important to ensure that any code that needs to be executed before the AfterAll method is properly executed before exiting the test method.
