java.lang.management.ThreadInfo

Java ThreadInfo is a class in the java.lang.management package. It provides detailed information about an active thread, such as its state, priority, name, and stack trace. It can be used to obtain information about deadlocked threads, or to gain insight into what activities a thread is performing.

Constructors

ThreadInfo has one constructor that accepts multiple arguments:


          
           
public ThreadInfo(long threadId, String threadName, long blockedTime, long waitedTime, String lockName, boolean suspended, boolean inNative, String threadState, int priority, String stackTrace)
Each argument is as follows: 
  • threadId
    – The thread’s unique identifier
  • threadName
    – The thread’s name
  • blockedTime
    – The amount of time the thread has been blocked (in milliseconds)
  • waitedTime
    – The amount of time the thread has been waiting (in milliseconds)
  • lockName
    – The name of the object on which the thread is blocked, or 
    null
    if not blocked
  • suspended
    – Whether or not the thread is suspended
  • inNative
    – Whether or not the thread is executing native code
  • threadState
    – The current state of the thread (e.g. NEW, RUNNABLE, BLOCKED, etc.)
  • priority
    – The thread’s priority
  • stackTrace
    – A stack trace for the thread, or 
    null
    if no stack trace is available

Examples

Here is a simple example of using ThreadInfo to obtain basic information about a thread:


          
           
ThreadInfo info = ManagementFactory.getThreadMXBean().getThreadInfo(threadId);
String threadName = info.getThreadName();
int priority = info.getPriority();
String threadState = info.getThreadState();
