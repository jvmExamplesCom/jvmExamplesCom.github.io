org.springframework.web.cors.CorsConfiguration

The org.springframework.web.cors.CorsConfiguration class provides an easy way to configure Cross-Origin Resource Sharing (CORS) in a Spring application. It allows fine-grained control over resources that can be accessed from other domains and provides several ways to achieve this. In this article, we'll look at how to use the CorsConfiguration class to configure CORS in a Spring application.

Configuring CORS with Spring Boot

If you're using Spring Boot, then configuring CORS is relatively straightforward. Firstly, you need to add the spring-boot-starter-web dependency to your project:


          
           
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
  <version>2.4.1</version>
</dependency>

You can then configure CORS in your application.properties file:


          
           
cors.mappings.allowed-origins=http://example.com
cors.mappings.allow-credentials=true
cors.mappings.max-age=3600

The allowed-origins property specifies which domains are allowed to access resources in this application, while the allow-credentials property indicates whether or not the application will accept cookies and other credentials sent with requests. Finally, the max-age property specifies the length of time that the configuration should be cached before it is refreshed.

Configuring CORS Manually

In addition to using Spring Boot, you can also configure CORS manually. To do this, you first need to create a CorsConfiguration object and set the necessary options:


          
           
CorsConfiguration config = new CorsConfiguration();
config.setAllowCredentials(true);
config.addAllowedOrigin("http://example.com");
config.addAllowedHeader("*");
config.setMaxAge(3600L);

You can then create a CorsFilter object and pass it the CorsConfiguration object. This filter will then be applied to all requests that match the specified mappings:


          
           
CorsFilter corsFilter = new CorsFilter(config);

Finally, you can add the filter to the Spring application. The following example adds the filter to the web application root context:


          
           
ServletContext servletContext = ...
servletContext.addFilter("corsFilter", corsFilter).addMappingForUrlPatterns(null, false, "/*");

You can also configure the CorsFilter using a property file. The following example configures the same filter as above, but uses a properties file instead:


          
           
corsfilter.mappings.allow-credentials=true
corsfilter.mappings.allowed-origins=http://example.com
corsfilter.mappings.max-age=3600

Conclusion

In this tutorial, we've seen how to use the org.springframework.web.cors.CorsConfiguration class to configure CORS in a Spring application. We looked at how to configure CORS using Spring Boot, and also manually. With this knowledge, you should be able to configure CORS in any Spring application that needs to support cross-domain requests.
