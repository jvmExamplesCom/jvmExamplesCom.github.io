org.hibernate.validator.constraints.NotEmpty

The org.hibernate.validator.constraints.NotEmpty annotation is used to make sure that the annotated string, collection, map or array is not null or empty. This annotation is a constraint-level annotation, meaning it is applied on a field/method level, and can be combined with other Bean Validation constraints.

Example 1

In the following example, we create a class called AuthDetails and add the @NotEmpty annotation to the userName field. 


          
           
public class AuthDetails {
    @NotEmpty
    private String userName;
    // ...
}

Example 2

We can also use the annotation with a collection as follows: 


          
           
public class AddressBook {
    @NotEmpty
    private List<String> contacts;  
    // ...
}

Example 3

We can use @NotEmpty with an array: 

          
           
public class OrderDetails {
    @NotEmpty
    private String[] orderItems;
    // ...
}

Example 4

We can use @NotEmpty with a map as follows: 

          
           
public class ConfigOptions {
    @NotEmpty
    private Map<String, String> options;
    // ...
}
