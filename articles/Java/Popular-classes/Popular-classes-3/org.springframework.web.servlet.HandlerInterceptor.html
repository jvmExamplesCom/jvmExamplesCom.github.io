Introduction to org.springframework.web.servlet.HandlerInterceptor

The org.springframework.web.servlet.HandlerInterceptor is an interface in the Spring Framework that provides a mechanism for pre-processing and post-processing web requests. It enables the implementation of cross-cutting concerns such as security, transaction management and request validation. The HandlerInterceptor can also be used to perform tasks such as retrieving objects from the session, setting output headers and performing logging operations.

Using org.springframework.web.servlet.HandlerInterceptor

The org.springframework.web.servlet.HandlerInterceptor can be used by implementing the interface in a class and then registering it in the WebApplicationInitializer. The interface has three methods: preHandle(), postHandle() and afterCompletion().

preHandle() Method
