Deploying the Application

An application can be deployed by using a servlet container such as Apache Tomcat, or by using an application server such as GlassFish. A servlet container will deploy the application to a web application context. An application server is typically used for enterprise applications and provides additional features such as clustering, load balancing, and management tools.

The application must be registered in the servlet or application server in order for it to be properly deployed and accessed. The registration process depends on the type of container or server being used.
