Java NIO Buffer

Java NIO Buffer is a data container that can hold fixed number of elements of the same data type. It is primarily used for communication between different components on a Java software system, such as between threads, channels and streams. A buffer holds data in its internal memory location until the data is ready to be manipulated or transferred.

Buffer Types

Java NIO Buffer comes in four types depending on the data type, namely ByteBuffer, CharBuffer, DoubleBuffer and IntBuffer.

ByteBuffer

The ByteBuffer is used when working with raw binary data. It wraps an array of bytes that can be read from and written to. The underlying array stores the data in native byte format.


          
           
ByteBuffer buffer = ByteBuffer.allocate(6);
 buffer.put((byte) 10);
 buffer.put((byte) 20);
 buffer.put((byte) 30);
 buffer.put((byte) 40);
 buffer.put((byte) 50);
 buffer.put((byte) 60);

CharBuffer

The CharBuffer is used when working with text data. It wraps an array of characters that can be read from and written to. Unlike the ByteBuffer, it typically works with Unicode characters rather than bytes.


          
           
CharBuffer buffer = CharBuffer.allocate(5);
 buffer.put('a');
 buffer.put('b');
 buffer.put('c');
 buffer.put('d');
 buffer.put('e');

DoubleBuffer

The DoubleBuffer is used when working with double-precision floating-point numbers. It wraps an array of doubles that can be read from and written to.


          
           
DoubleBuffer buffer = DoubleBuffer.allocate(3);
 buffer.put(1.1);
 buffer.put(2.2);
 buffer.put(3.3);

IntBuffer

The IntBuffer is used when working with integer numbers. It wraps an array of ints that can be read from and written to.


          
           
IntBuffer buffer = IntBuffer.allocate(4);
 buffer.put(10);
 buffer.put(20);
 buffer.put(30);
 buffer.put(40);
