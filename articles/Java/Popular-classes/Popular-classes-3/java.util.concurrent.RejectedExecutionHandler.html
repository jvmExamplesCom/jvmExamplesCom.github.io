java.util.concurrent.RejectedExecutionHandler

The java.util.concurrent.RejectedExecutionHandler interface provides a callback for failed task submissions. This is an alternative way to handle cases when a task cannot be executed due to resource constraints.

Introduction

The RejectedExecutionHandler interface defines the contract for object that can handle tasks that cannot be executed due to resource constraints in a Java Executor framework. An implementation of this interface can provide custom behavior when a task cannot be accepted for execution. The Executors class provides helper methods to create commonly used implementations of this interface.

Example

Let’s look at an example to see how to use a RejectedExecutionHandler. In this example, we will use a custom implementation of RejectedExecutionHandler to handle rejected tasks. 

          
           
public class MyRejectedExecutionHandler implements RejectedExecutionHandler {

    public void rejectedExecution(Runnable r, ThreadPoolExecutor executor) {
        System.out.println("Task " + r.toString() + " is rejected");
    }

}
We create a class named MyRejectedExecutionHandler that implements the RejectedExecutionHandler interface. This class provides a single method named rejectedExecution() that gets called when a task cannot be accepted for execution. This method simply prints out the name of the rejected task. Now let’s use this handler to create a ThreadPoolExecutor. 

          
           
ThreadPoolExecutor executor = new ThreadPoolExecutor(2, 5, 10, TimeUnit.SECONDS,
        new ArrayBlockingQueue<Runnable>(2), new MyRejectedExecutionHandler());
We create a ThreadPoolExecutor with a core pool size of 2, a maximum pool size of 5, and a keep-alive time of 10 seconds. We also provide an ArrayBlockingQueue of size 2 to store the tasks. Finally, we provide our custom implementation of the RejectedExecutionHandler - MyRejectedExecutionHandler. Now let’s submit some tasks to the executor. 

          
           
executor.submit(task1);

          

          
           
executor.submit(task2);
These tasks will be accepted for execution. Now let’s try to submit a third task. 

          
           
executor.submit(task3);
This task will be rejected because the queue is already full. The MyRejectedExecutionHandler’s rejectedExecution() method will be called and it will print out the name of the rejected task.
