org.apache.http.impl.conn.PoolingHttpClientConnectionManager

The org.apache.http.impl.conn.PoolingHttpClientConnectionManager helps in managing of HTTP connections using pooling. It is a part of the Apache HttpComponents library and is available in version 4.2 or later. This manager maintains a pool of HttpClientConnection objects that can be reused for multiple requests instead of creating and closing a connection for each request. This makes the requests more efficient and also reduces the latency due to connection establishment.

Code Examples


          
           
PoolingHttpClientConnectionManager httpConnManager = 
      new PoolingHttpClientConnectionManager();
    
    httpConnManager.setMaxTotal(100);
    httpConnManager.setDefaultMaxPerRoute(50);

The above code snippet creates a new PoolingHttpClientConnectionManager instance and sets the maximum total of connections allowed to 100 and default max per route to 50.


          
           
httpConnManager.setMaxPerRoute(new HttpRoute(host1), 10);

The above code snippet sets the maximum number of connections to host1 to 10.


          
           
// Closing the connection manager
 httpConnManager.shutdown();

The above code snippet shuts down the connection manager and releases all resources associated with it.


          
           
HttpGet httpGet = new HttpGet("http://example.com");

The above code snippet creates an HttpGet instance with the given URL.


          
           
// Closing the connection 
HttpClient httpClient= new DefaultHttpClient(httpConnManager);
HttpResponse response = httpClient.execute(httpGet);

httpClient.getConnectionManager().shutdown();

The above code snippet creates an HttpClient instance with the PoolingHttpClientConnectionManager and executes the HttpGet method. After getting the response from the server, the entity is consumed and the connection is shut down.
