Introduction to Google Ordering

Google’s ordering library helps developers sort collections of objects, allowing them to make the most of their data. With easy-to-use methods available, this library makes sorting data a breeze.

What is Google Ordering?

Google’s ordering library, com.google.common.collect.Ordering, is an implementation of the Java Collections Framework (JCF). It contains a collection of sorting algorithms and classes to use them. It is part of the Guava libraries and provides more capabilities than the JCF.

It supports a full range of sorting operations, such as sorting by field, comparing two objects, and sorting by multiple fields. Additionally, its natural ordering methods support type-aware comparisons of objects and arrays.

How Does Google Ordering Work?

Google’s ordering library implements the Java Collections Framework interface Comparator. This interface defines how objects can be compared and ordered. Using this, developers can create custom comparators that control how objects are sorted.

The library also provides a set of standard comparators which are useful for common comparison tasks. These are the NaturalOrdering, ReverseOrdering, CompoundOrdering, and PartialOrdering classes. Each of these provides different features that can help developers sort objects.

NaturalOrdering

The NaturalOrdering class compares objects based on their natural order. This means that objects will be sorted in the same way they would be in the absence of any comparator. For example, if you have a list of integers, they will be sorted from least to greatest. 


          
           
List<Integer> intList = ImmutableList.of(4, 3, 1, 2); 
Ordering<Integer> naturalOrder = NaturalOrdering.instance(); 
List<Integer> sorted = naturalOrder.sortedCopy(intList); 
System.out.println(sorted); // prints [1, 2, 3, 4]

ReverseOrdering

The ReverseOrdering class sorts objects in reverse order. This is the opposite of natural order and is useful for sorting lists in descending order. For example, a list of integers would be sorted from greatest to least. 


          
           
List<Integer> intList = ImmutableList.of(4, 3, 1, 2);
Ordering<Integer> reverseOrder = ReverseOrdering.instance(); 
List<Integer> sorted = reverseOrder.sortedCopy(intList); 
System.out.println(sorted); // prints [4, 3, 2, 1]

CompoundOrdering

The CompoundOrdering class combines multiple comparators into a single sorting strategy. This allows developers to sort objects by several criteria. For example, a list of people can be sorted by name, then by age. 


          
           
List<Person> people = ImmutableList.of( new Person("John", 32), new Person("John", 30), new Person("Alice", 32));
Comparator<Person> byName = Comparator.comparing(Person::getName);
Comparator<Person> byAge = Comparator.comparing(Person::getAge);
Ordering<Person> compoundOrder = CompoundOrdering.instance(byName, byAge);
List<Person> sorted = compoundOrder.sortedCopy(people); 
System.out.println(sorted); // prints [John, 30, John, 32, Alice, 32]

PartialOrdering

The PartialOrdering class provides a way to partially sort a collection. This can be used to sort a list of objects by a subset of criteria. For example, if a list of people needs to be sorted by age, but the age is not always known, the PartialOrdering class can be used to sort on the available values. 


          
           
List<Person> people = ImmutableList.of( new Person("John", null), new Person("Alice", 32));
Comparator<Person> byName = Comparator.comparing(Person::getName);
Comparator<Person> byAge = Comparator.comparing(Person::getAge);
Ordering<Person> partialOrder = PartialOrdering.instance(byName, byAge); 
List<Person> sorted = partialOrder.sortedCopy(people); 
System.out.println(sorted); // prints [Alice, 32, John, null]

Conclusion

Google’s ordering library makes sorting collections of objects easy and efficient. With the right combination of comparators, developers can easily and quickly get their data into the desired order.
