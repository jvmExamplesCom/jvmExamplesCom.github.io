javax.swing.event.DocumentEvent

The 

          
           
javax.swing.event.DocumentEvent
class is a Swing event used to provide notification of changes made to a document. DocumentEvent objects are generated when the document is edited and POSTed when changes occur in the document. They are dispatched to all registered DocumentListeners.

The following example creates a custom DocumentListener that prints out the details of any changes made to the document:


          
           
class MyDocumentListener implements DocumentListener {
    public void changedUpdate(DocumentEvent e) {
        System.out.println("A change was made to the document:");
        System.out.println("New text: " + e.getText());
        System.out.println("Old text: " + e.getOldText());
    }
    public void insertUpdate (DocumentEvent e) {
        System.out.println("Text was inserted into the document:");
        System.out.println("New text: " + e.getText());
    }
    public void removeUpdate (DocumentEvent e) {
        System.out.println("Text was removed from the document:");
        System.out.println("Old text: " + e.getOldText());
    }
}

The listeners can be registered in the constructor of the GUI like so:


          
           
public MyGUI() {
    // Create the GUI.
    ...
    // Register the document listener.
    document.addDocumentListener(new MyDocumentListener());
}

The DocumentListener interface also provides methods to prevent the document from being changed, such as setEditable(), setEnabled(), and isReadOnly(). These methods can be used in conjunction with the other methods to enforce restrictions on the document. The following example demonstrates how to use these methods:


          
           
// Disable document editing.
document.setEditable(false);

// Check if the document is read-only.
boolean isReadOnly = document.isReadOnly();

// Enable document editing.
document.setEnabled(true);
