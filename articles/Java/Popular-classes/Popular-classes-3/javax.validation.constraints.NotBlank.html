javax.validation.constraints.NotBlank

The NotBlank annotation can be used to validate that values are not null or an empty string. This annotation can be applied for both JavaBean properties and method parameters. This annotation is supported by the Hibernate Validator, the reference implementation of the Bean Validation API.

Examples

Below is an example of NotBlank annotation being applied for a JavaBean property.


          
           
import javax.validation.constraints.NotBlank;

public class User {

    @NotBlank
    private String username;

    // ...
}

In the following example, NotBlank annotation is used on a method parameter:


          
           
import javax.validation.constraints.NotBlank;

public class UserService {

    public void registerUser(@NotBlank String username) {
        // ...
    }
    
    // ...
}

With the following example, if the passed name is null or blank, the validation will fail.


          
           
public void registerUser(@NotBlank String name) {
    // ...
}

// calling the method
registerUser(null);

Output

When the validations fail, an exception of type ConstraintViolationException is thrown. The message contains information about why the validation failed.


          
           
javax.validation.ConstraintViolationException: must not be blank
