org.junit.jupiter.api.BeforeAll

org.junit.jupiter.api.BeforeAll is an annotation used to signal that the annotated method should be executed before all tests in a given test class. This annotation is semantically equivalent to JUnit 4’s @BeforeClass annotation. This annotation can be used on methods declared in non-static inner classes.

Code Examples


          
           
@BeforeAll
public static void setupTestSession() {
   // Initialization logic
}

The annotated method can be declared with any access modifier (public, protected, package private or private). Furthermore, it must not be declared as static. Only one method in a given test class may be annotated with @BeforeAll.


          
           
@BeforeAll

This annotation is typically used to perform time intensive activities required for all tests in a single test class, such as establishing database connections or initializing expensive resources.


          
           
@BeforeAll
static void allocateExpensiveResource() {
   // Allocation logic
}

Since this annotation is semantically equivalent to JUnit 4’s @BeforeClass annotation, methods annotated with @BeforeAll may be used as replacements for corresponding methods annotated with @BeforeClass. Furthermore, in order to run correctly, the annotated method must also be declared as public, since @BeforeClass methods were implicitly made public when running under JUnit 4.


          
           
public class MyTest {

   @BeforeAll
   public static void setupTestSession() {
      // Initialization logic
   }
}
