Java.nio.charset.CharacterCodingException

The Java.nio.charset.CharacterCodingException is an unchecked exception thrown when a coding error is detected while encoding or decoding characters. Character coding is the process of translating individual characters into a stream of bytes and vice versa. Character coding operations may fail in various ways, such as due to an invalid argument being passed in, or a malformed character sequence. Characters may be illegal for the given charset, or the input may contain invalid byte sequences.

Examples of how java.nio.charset.CharacterCodingException can be used:

Example 1


          
           
try {
    // Code that uses characters
} catch (java.nio.charset.CharacterCodingException e) {
    System.err.println(e.getMessage());
}

In this example, a try-catch block is used to catch any CharacterCodingExceptions that might be thrown. The getMessage() method is used to print the error message.

Example 2


          
           
public static void main(String[] args) throws java.nio.charset.CharacterCodingException {
    String str = "Hello World!";
    Charset cset = Charset.forName("UTF-8");
    ByteBuffer bbuf = cset.encode(str);
    System.out.println(bbuf);
}

This example shows a program that attempts to encode a string into a ByteBuffer. An appropriate Charset is provided, and then an attempt is made to encode the string. If the encoding operation fails, a CharacterCodingException will be thrown.

Example 3


          
           
String str = "Hello World!";
try {
    Charset cset = Charset.forName("UTF-8");
    ByteBuffer bbuf = cset.encode(str);
    System.out.println(bbuf);
