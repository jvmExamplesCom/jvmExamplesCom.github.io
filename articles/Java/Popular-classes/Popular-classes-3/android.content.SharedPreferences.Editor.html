Introduction to android.content.SharedPreferences.Editor

Android provides a powerful mechanism for storing and retrieving small amounts of data from within your application via the SharedPreferences object. This allows you to store and retrieve simple key-value pairs, such as settings or user preferences. In order to modify and add SharedPreferences items, you must leverage the SharedPreferences.Editor object. This article will discuss the basics of working with this object and provide several examples.

Creating the Editor

Before you can start adding and modifying preferences, you must first create an Editor instance by calling the edit() method on the desired SharedPreferences object. 

          
           
SharedPreferences prefs = getPreferences("my_prefs", Context.MODE_PRIVATE);
SharedPreferences.Editor editor = prefs.edit();
Once created, you can use the Editor to both add and modify existing SharedPreferences items.

Adding Items

Adding a new item to the SharedPreferences is a simple operation. You simply call the 

          
           
putXxx()
method that matches the type of object you are storing. For example, adding a boolean value would look something like this: 

          
           
editor.putBoolean("my_key", true);
Likewise, adding a String is done by the putString() method: 

          
           
editor.putString("my_key", "Hello World!");
You can also add other primitive types, including int, long, float, and Set. Note that when adding a Set, you must pass in a Set of type Strings, as Sets of other objects are not supported.

Modifying Items

If an item exists with the same key, calling one of the 

          
           
putXxx()
methods will simply overwrite it without any additional work needed. However, if there is no item with a matching key, then the item will not be added, and the SharedPreferences will remain unchanged. To ensure an item is added or modified, you must first check whether or not it exists using the contains() method. If it does not exist, you will need to call the associated 

          
           
putXxx()
method, otherwise call setXxx() to modify the now-existing value. For example, adding or modifying a String would look something like this: 

          
           
if (!editor.contains("my_key")) {
  editor.putString("my_key", "Hello World!");
} else {
  editor.setString("my_key", "Hello World!")
}
For all other primitive types, the pattern is the same. Simply check if the item exists with contains(), add it with 

          
           
putXxx()
if it does not exist, and then modify it with 

          
           
setXxx()
if it does.

Removing Items

To remove an item from the SharedPreferences, you must call the 

          
           
remove()
method on the Editor with the key of the item you wish to remove. 

          
           
editor.remove("my_key");

Committing Changes

None of the operations discussed so far will actually modify the SharedPreferences. To actually make the changes, you must call the 

          
           
commit()
method on the Editor. 

          
           
editor.commit();
Once committed, the changes will take effect.

Conclusion

The android.content.SharedPreferences.Editor object provides a powerful way to modify SharedPreferences objects in Android. Using the various 

          
           
putXxx()
and 

          
           
setXxx()
methods, you can easily add and modify existing items with minimal effort. Additionally, the ability to check whether or not an item exists allows you to add or modify existing items with ease. Finally, committing the changes ensures that your modifications take effect.
