Introduction to org.junit.jupiter.params.ParameterizedTest

In Java, the org.junit.jupiter.params.ParameterizedTest class allows users to execute a test multiple times, with different parameters each time. This is helpful for quickly and efficiently testing a wide variety of input scenarios and ensuring that code behaves as expected in all cases.

Syntax


          
           
@ParameterizedTest
@ValueSource(ints = { 1, 2, 3 })
void testWithValueSource(int argument) {
  // ...
}

Parameters

The parameters can be any primitive type, such as int, float, etc., as well as more complex types such as objects or arrays. It is also possible to provide a list of values that the parameters should take.

Example

To illustrate how this works, let's consider a simple example. We have a class called MyCalculator, which has a method called "add", which takes two integers as parameters and returns their sum.


          
           
public class MyCalculator {
  public int add(int x, int y) {
    return x + y;
  }
}

We want to make sure that our implementation of this method is correct, so we can use ParameterizedTest to test it with various combinations of parameters.


          
           
@ParameterizedTest
@ValueSource(ints = { 0, 1, 2, 3, 4, 5 })
void testAdd(int x, int y) {
  MyCalculator myCalculator = new MyCalculator();
  assertEquals(x + y, myCalculator.add(x, y));
}

In this example, we are providing a list of integers that the parameters should take. In this case, the test will be executed 6 times, with values 0, 1, 2, 3, 4, and 5 being passed to the parameters x and y respectively. Each time the test is executed, the assertEquals statement will verify that the result of the add method is correct.

Conclusion

ParameterizedTest is a powerful tool that can be used to quickly and efficiently test a wide range of input scenarios and ensure that the code behaves as expected. It works by allowing the user to provide a list of parameters that should be used in the test, and then executing the test with each of those parameters in turn. This approach is much faster and more efficient than manually creating separate tests for each combination of parameters.
