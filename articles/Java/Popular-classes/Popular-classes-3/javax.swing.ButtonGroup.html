javax.swing.ButtonGroup

The 

          
           
javax.swing.ButtonGroup
class is a useful tool for helping to create radio buttons and checkboxes in a user interface. It allows the application developer to easily implement a logical relationship between related buttons, in that only one button can be selected at a time. The 

          
           
ButtonGroup
class is part of the Java Swing library, which is used for creating rich user interfaces. The 

          
           
ButtonGroup
class is used to group together all the buttons related to a particular task, making sure that only one can be selected at any given time. This can be achieved either by selecting a single button from the group, or simply by deselecting all the other buttons in the group.

Creating a ButtonGroup

To create a new 

          
           
ButtonGroup
object, we must first declare it and then instantiate it using the following syntax: 

          
           
ButtonGroup myButtonGroup = new ButtonGroup();
Once created, 

          
           
ButtonGroup
objects have no visible representation. In order to use them, they must be linked to a number of 

          
           
JRadioButton
or 

          
           
JCheckBox
components on a user interface. These components are added to the 

          
           
ButtonGroup
object using the 

          
           
add()
method. For example, if we had two radio buttons, we could link them to the 

          
           
ButtonGroup
as follows: 

          
           
myButtonGroup.add(button1);
myButtonGroup.add(button2);

Using a ButtonGroup

Once the components are added to the 

          
           
ButtonGroup
, they can be manipulated as a unit. For example, it is possible to select one component at a time using the 

          
           
setSelected()
method. This method takes a single parameter, which should be an instance of the component that we want to be selected. To deselect all components, we can call the 

          
           
clearSelection()
method. It is also possible to obtain information about the currently selected component by calling the 

          
           
getSelection()
method. This method returns a 

          
           
ButtonModel
object, which contains the state and data of the currently selected component. Finally, the status of each component in the 

          
           
ButtonGroup
can be read and updated individually with the 

          
           
setEnabled()
and 

          
           
isEnabled()
methods.

Example

Here is a complete example of how to use the 

          
           
ButtonGroup
class to create a simple application with two radio buttons. The application allows the user to select either "Red" or "Blue" and displays the chosen color in a message box: 

          
           
import javax.swing.*;
import javax.swing.ButtonGroup;

public class MyApplication {

    public static void main(String[] args) {
        // Create the radio buttons
        JRadioButton redButton = new JRadioButton("Red");
        JRadioButton blueButton = new JRadioButton("Blue");
        
        // Create a ButtonGroup to group the buttons together
        ButtonGroup colorGroup = new ButtonGroup();
        colorGroup.add(redButton);
        colorGroup.add(blueButton);

        // Create a panel to hold the buttons
        JPanel panel = new JPanel();
        panel.add(redButton);
        panel.add(blueButton);
        
        // Show the panel in a window
        JFrame frame = new JFrame("My Application");
        frame.add(panel);
        frame.pack();
        frame.setVisible(true);
        
        // Check if a button has been selected
        if (colorGroup.getSelection() != null) {
            // Get the selected button
            JRadioButton selectedButton = (JRadioButton) colorGroup.getSelection();
            String chosenColor = selectedButton.getText();
            
            // Display a message box with the chosen color
            JOptionPane.showMessageDialog(null, "You chose the color: " + chosenColor);
        }
    }

}
