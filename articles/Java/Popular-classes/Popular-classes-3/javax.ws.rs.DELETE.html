javax.ws.rs.DELETE

javax.ws.rs.DELETE is an annotation of the Java API for RESTful web services (JAX-RS). This annotation is used to map HTTP DELETE requests onto specific Java methods. When a client makes a DELETE request to the URL provided by @DELETE, it can delete the resource at the corresponding URI.

Code Examples

A basic example of using the @DELETE annotation could look like this: 


          
           
@Path("/user/{id}")
public class UserResource {
    @DELETE
    @Path("/{id}")
    public void delete(@PathParam("id")  int id) {
         // Delete the user with the specified id
    }
}

In this example, the @DELETE annotation is used to map the delete() method to the corresponding HTTP DELETE request. The @PathParam annotation is used to bind the parameter "id" from the URL to the int id parameter of the method.

HTTP Responses

The @DELETE annotation often returns a response code of 204 (No Content) indicating that the request has been successfully processed and that no further action is required. In some cases, a response code of 200 (OK) is also returned. It's important to note that a 204 response will not return any message body, while a 200 response typically contains a response message.

In some cases, a @DELETE request may return a response code of 400 (Bad Request) if the request is invalid or if the server cannot process the request. In such cases, an error message is usually returned.
