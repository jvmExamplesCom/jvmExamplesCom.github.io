androidx.annotation.DrawableRes

AndroidX.annotation.DrawableRes is an annotation type in the Android Support library that allows developers to indicate that a particular parameter, field, or method return value references a drawable resource. It was introduced in API level 17 (Android 4.2) and is a useful tool for preventing accidental resource typing errors when referencing a drawable resource. This article will look at how to use the annotation, some of its attributes, and provide code examples.

What Is androidx.annotation.DrawableRes?

androidx.annotation.DrawableRes is a type of annotation that can be used on parameters, fields, and method return values. It indicates that the referenced type must be a drawable resource. It is a compile-time annotation and therefore any issues related to it will be caught during compilation. The annotation has no effect on the generated code.

Using androidx.annotation.DrawableRes

To use the annotation, simply add an @androidx.annotation.DrawableRes annotation to the parameter, field, or method return value to be validated for drawable resource use. 

          
           
@androidx.annotation.DrawableRes
int myDrawableResource;
When compiled, the annotation will ensure that only drawable resources can be assigned to the variable or method return value.

Attributes of androidx.annotation.DrawableRes

The androidx.annotation.DrawableRes annotation has two attributes:
  • value – A list of acceptable drawable resource types. Defaults to “*”, meaning all drawable resource types are acceptable.
  • resolve – Boolean attribute. When set to true, the annotation will attempt to resolve resources that are not found in the project.
The value attribute can be set to a specific type of drawable resource or a comma-separated list of acceptable types. For example: 

          
           
@androidx.annotation.DrawableRes(value = "bitmap,nine-patch")
int myDrawableResource;
This will make sure that the myDrawableResource is set to either a bitmap or nine-patch drawable resource. The resolve attribute allows the annotation to attempt to resolve resources that are not found in the project. This is useful when referencing resources with unknown identifiers.

Example Use of androidx.annotation.DrawableRes

A common use case for the annotation could be when creating a UI component that needs to use a drawable resource. As an example, let’s look at a custom ImageView: 

          
           
public class MyImageView extends ImageView {
  @androidx.annotation.DrawableRes
  int imageResource;

  public void setImageResource(@androidx.annotation.DrawableRes int imageResource) {
    this.imageResource = imageResource;
    setImageDrawable(getResources().getDrawable(imageResource));
  }
}
In this example, the setImageResource() method takes an integer argument which is annotated with @androidx.annotation.DrawableRes. This ensures that only drawable resources can be passed to the method. The annotation can also be used when inflating views from XML layouts: 

          
           
@androidx.annotation.DrawableRes
int imageResource;

imageView = findViewById(R.id.imageview);
imageView.setImageResource(imageResource);
Here, the annotation ensures that only a drawable resource can be assigned to the imageResource variable.

Conclusion

The androidx.annotation.DrawableRes annotation makes it easier for developers to prevent resource type errors when working with drawable resources on Android. The annotation is relatively straightforward to use, and can help catch any issues related to resource typing earlier on in the development process.
