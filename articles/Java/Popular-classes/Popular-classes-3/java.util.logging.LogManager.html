Introduction to java.util.logging.LogManager

The 

          
           
java.util.logging.LogManager
is a class in the Java platform that provides the fundamental operations necessary to configure and manage loggers in the Java platform. It is used by the logging framework to read configuration data, to create and manage logger instances and to operate on loggers. This class can be used directly to manipulate the logging configuration, but it is more convenient to use the Logging API.

Creating a LogManager instance

A LogManager instance is created using the 
getLogManager()
method. This method can be used to get the default LogManager instance or to create a new LogManager instance with specific configuration. 

          
           
LogManager manager = LogManager.getLogManager();

Configuring the Logger

The LogManager has methods to configure the Logger. The most commonly used methods are 

setLevel()
, 
addHandler()
and 

          
           
setUseParentHandlers()
. These methods can be used to set the log level, add log handlers, and configure whether to use the parent logic handler. 

          
           
manager.setLevel(Level.INFO); 
manager.addHandler(new ConsoleHandler()); 
manager.setUseParentHandlers(false);

Creating Loggers

The LogManager can be used to create Logger instances. A logger instance is created using the 

getLogger()
method. The name of the logger is used to identify it in the configuration. This method returns a logger instance with the configured properties. 

          
           
Logger logger = manager.getLogger("myLogger");

Logging Messages

Once the logger instance is created, messages can be logged using the 

log()
method. The log method has several overloaded versions which accept different parameters. These parameters include the log level, message, exception and objects. 

          
           
logger.log(Level.INFO, "This is an example log message");

Closing Loggers

When the logger is no longer needed, it can be closed using the 

shutdown()
method. This method closes the logger and frees any associated resources. If the logger is associated with any open file handles, those will also be closed. 

          
           
manager.shutdown();
