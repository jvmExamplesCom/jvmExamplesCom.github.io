Understanding Java Futures

Java Futures are one of the most popular concurrent utilities used to create solutions that simplify complex programming tasks. In this article, we will take a deep dive into what java futures are and look at several code examples to clarify how they are used.

What Is A Future?

In a nutshell, a future is a placeholder object for a value that will eventually become available. It represents the asynchronous task and provides methods for determining if the task is complete, which result can be retrieved, and a method to block the current thread until the result becomes available. In Java, a future can be obtained from an instance of 


          
           
java.util.concurrent.ExecutorService
or by directly instantiating the 

          
           
java.util.concurrent.FutureTask
class.

Why Use Java Futures?

Java Futures are primarily used to simplify tasks that involve code that needs to run asynchronously. For example, when calling external systems or services, it’s useful to have a way to perform the operation without blocking the current thread. This allows you to use the same thread for other tasks while waiting for the result. Additionally, by using the Java Futures API you can easily integrate calls to external systems or services into your own code, increasing performance and reducing complexity.

A Basic Code Example

Let’s walk through a basic example of creating a simple Java Future. The following code creates a 


          
           
java.util.concurrent.Callable
object and wraps it in a 

          
           
java.util.concurrent.FutureTask
. The callable object will be executed when the future task is submitted. The output of the callable can be retrieved from the future when the task completes: 

          
           
Callable<Integer> callable = () -> {
    int result = 0;
    // perform lengthy calculation here
    return result;
};

FutureTask<Integer> future = new FutureTask<>(callable);

// submit the future
ExecutorService executor = Executors.newSingleThreadExecutor();
executor.submit(future);

// block the current thread until the future completes
Integer result = future.get();

In this example, a Callable object is defined and wrapped in a FutureTask instance. The FutureTask instance is then submitted to an ExecutorService and the current thread is blocked until the future completes, at which point the result of the calculation can be retrieved.

Advanced Examples

Let’s take a look at some more advanced examples of Java Futures:

Composing Futures

You can use Java Futures to compose multiple tasks, allowing you to create more complex solutions. The following example shows how to compose two futures, where the second future is dependent on the result of the first:


          
           
Future<String> future1 = executor.submit(() -> {
    // perform lengthy operation
    return "Hello";
});

Future<String> future2 = executor.submit(() -> {
    String result = future1.get() + " World";
    return result;
});

String finalResult = future2.get();

In this example, the first future performs a lengthy operation and returns a result (in this case, the string “Hello”). The second future is dependent on the result of the first and concatenates the result with “World” before returning the result of the composition. The final result can be retrieved from the second future.

Checking For Completion

You can check to see if a future has completed by using the 

isDone()
method. If the future has not yet completed, the method will return false. Otherwise, it will return true. 

          
           
if (future.isDone()) {
	System.out.println("Task has completed");
}

Handling Errors

When writing asynchronous tasks, it is important to handle any exceptions that may arise. To handle errors, Java Futures provide the 


          
           
get(timeout, TimeUnit)
method, which allows you to specify a timeout for the task. If the task does not complete within the specified timeout period, a 
TimeoutException
will be thrown: 

          
           
try {
    future.get(10, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    System.err.println("Task did not complete in time");
}

Conclusion

In this article, we explored what Java Futures are, why they are useful, and how to use them correctly. We looked at several code examples to demonstrate how to create and compose futures, as well as how to handle errors. Understanding how to use Futures correctly is an essential skill for any software developer.
