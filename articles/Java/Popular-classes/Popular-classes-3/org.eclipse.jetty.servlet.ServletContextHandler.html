ServletContextHandler in Eclipse Jetty

Eclipse Jetty is an open source project providing an HTTP server, HTTP client, and javax.servlet container. The ServletContextHandler class is a handler that allows the configuration of javax.servlet specific features. It may be used directly or may be subclassed to configure it with a web application context.

Basic Usage

The basic usage of ServletContextHandler involves creating a new instance with a reference to the servlet context path:


          
           
ServletContextHandler context = new ServletContextHandler(ServletContextHandler.SESSIONS);
context.setContextPath("/");

You can also call setResourceBase() to set the root directory for static resources.

Configuration

The ServletContextHandler provides various configuration options, such as setting session cookies, adding listeners and filters, customising welcome files and error pages, and registering servlets.

Session Cookies

You can set the session cookie name and domain with the setSessionCookie() method:


          
           
context.setSessionCookie("my-session-cookie", "my-domain.com");

Listeners and Filters

You can add listeners and filters to the ServletContextHandler with the addEventListener() and addFilter() methods respectively. For example, to add a ServletRequestListener:


          
           
context.addEventListener(new MyServletRequestListener());

Welcome Files and Error Pages

You can set welcome files and associated error pages with the setWelcomeFiles() and setErrorPages() methods respectively. For example:


          
           
context.setWelcomeFiles(new String[]{"index.html"});
context.setErrorPages(new ErrorPageErrorHandler());

Servlets

You can register servlets with the servlet context using the addServletWithMapping() method. For example:


          
           
context.addServletWithMapping(MyServlet.class, "/*");
