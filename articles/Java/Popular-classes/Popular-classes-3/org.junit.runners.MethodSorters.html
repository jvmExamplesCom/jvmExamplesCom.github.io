org.junit.runners.MethodSorters

The org.junit.runners.MethodSorters class in JUnit provides a facility to control the order of test method execution within a test class. The sorting of test methods can be configured either at the class level or for each individual test method.

Sorting Types

MethodSorters currently provides four types of sorting:

  • MethodSorters.DEFAULT: This is the default sorting, which follows the natural order of declaration of test methods in the test class.
  • MethodSorters.NAME_ASCENDING: This sorts the test methods by their names in ascending alphabetical order.
  • MethodSorters.JVM: This sorts the test methods based on the JVM's native implementations of sorting, which may produce different results in different environments.
  • MethodSorters.RANDOM: This sorts the test methods in random order.

Configuring MethodSorter

MethodSorter can be configured either at the class level or for each individual test method. To configure at the class level, you need to annotate your test class with the 


          
           
@FixMethodOrder(MethodSorters.TYPE)
annotation: 

          
           
@RunWith(JUnit4.class)
@FixMethodOrder(MethodSorters.NAME_ASCENDING)
public class TestClass {
   ...
}
To configure for a particular method, use the 

          
           
@Test(order = N)
annotation, where N is an integer denoting the order in which the test method is to be executed. The test methods will be executed in ascending order of their corresponding integers. 

          
           
@RunWith(JUnit4.class)
public class TestClass {
    @Test(order = 3)
    public void test3() {
        ...
    }

    @Test(order = 1)
    public void test1() {
        ...
    }

    @Test(order = 2)
    public void test2() {
        ...
    }
}

Regardless of the way it is configured, MethodSorter guarantees that the test methods are executed in the same order in all the JUnit runs.
