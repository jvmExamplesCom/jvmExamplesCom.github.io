javax.tools.Diagnostic

javax.tools.Diagnostic is a Java API that provides access to the diagnostic information generated by the compiler while it processes Java programs. It enables developers to access diagnostics messages such as errors, warnings, and notes related to the compilation process. This API is part of the Java Platform Standard Edition Java Compiler API (JSR 199).

javax.tools.Diagnostic provides two interfaces namely DiagnosticListener and Diagnostic.The DiagnosticListener interface accepts the Diagnostic events from the compiler. It has a single method named report which is used to report the diagnostic event.

Example

The following example shows how to use DiagnosticListener to report compile-time errors.


          
           
import java.util.List;
import javax.tools.Diagnostic;
import javax.tools.DiagnosticListener;
import javax.tools.JavaCompiler;
import javax.tools.StandardJavaFileManager;

public class MyDiagnosticListener implements DiagnosticListener {

  @Override
  public void report(Diagnostic diagnostic) {
    System.out.println("Error on line " + 
      diagnostic.getLineNumber() + ": " + diagnostic.getMessage());
  }

  public static void main(String args[]) {
    JavaCompiler compiler = javax.tools.ToolProvider.getSystemJavaCompiler();
    StandardJavaFileManager fileManager = 
     compiler.getStandardFileManager(null, null, null);
    List<String> optionList = new ArrayList<String>();

    // set compiler's classpath to be same as the runtime's
    optionList.addAll(Arrays.asList("-classpath", 
    System.getProperty("java.class.path")));

    MyDiagnosticListener myDiagnosticListener = 
      new MyDiagnosticListener();
    Iterable<String> compilationUnits = 
      fileManager.getJavaFileObjectsFromFiles(Arrays.asList(
        new File(args[0])));
    JavaCompiler.CompilationTask task = compiler.getTask(
      null, 
      fileManager, 
      myDiagnosticListener, 
      optionList, 
      null, 
      compilationUnits);

    task.call();
    fileManager.close();
  }
}

In the preceding program, the Diagnostic instance is sent to the report method to print out the diagnostic message. The report method will be invoked by the compiler when an error or warning is encountered during the compilation process.

Conclusion

javax.tools.Diagnostic is a useful tool for debugging and understanding compile-time errors. By using this API, developers can access diagnostics messages such as errors, warnings, and notes related to the compilation process.
