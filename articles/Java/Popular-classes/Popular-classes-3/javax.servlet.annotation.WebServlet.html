javax.servlet.annotation.WebServlet

The 

          
           
javax.servlet.annotation.WebServlet
annotation is used to declare a servlet component in a web application. The servlet component is invoked by the web container upon receipt of a request that matches the configuration of the servlet component.

Declaration

The 
@WebServlet
annotation is used to declare a servlet component. The following is the basic declaration: 

          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url"})

This declaration declares a servlet component named “MyServlet” with a URL pattern of “/my-url”.

Configuration Parameters

The 
@WebServlet
annotation supports several configuration parameters for customizing the behavior and configuration of a servlet component.

Name Parameter

The “name” parameter is used to specify the name of the servlet component. This parameter is required, as it is used to identify the servlet component in the web application. 


          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url"})

URL Patterns Parameter

The “urlPatterns” parameter is used to specify the URL patterns that the servlet component should be invoked on. Multiple URL patterns can be specified, separated by commas. 


          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url","/my-other-url"})

Init Parameters Parameter

The “initParameters” parameter is used to specify initialization parameters for the servlet component. These parameters will be made available to the servlet component when it is initialized. 


          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url"}, initParameters = {"param1=value1","param2=value2"})

Load on Startup Parameter

The “loadOnStartup” parameter is used to specify the order in which the servlets are initialized. If multiple servlets are declared with this parameter, they will be initialized in the order specified. 


          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url"}, loadOnStartup = 1)

Async Supported Parameter

The “asyncSupported” parameter is used to specify whether or not the servlet should support asynchronous processing. If this parameter is set to true, the servlet will support asynchronous processing. 


          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url"}, asyncSupported = true)

Example

The following example shows a basic implementation of the 

@WebServlet
annotation: 

          
           
@WebServlet(name = "MyServlet", urlPatterns = {"/my-url"})
public class MyServlet extends HttpServlet {

  protected void doGet(HttpServletRequest request,
                       HttpServletResponse response)
      throws ServletException, IOException {

    response.getWriter().println("Hello World!");
  }
}

This example declares a servlet component named “MyServlet” with a URL pattern of “/my-url”. The servlet component will respond to requests matching the URL pattern with a “Hello World!” message.
