android.support.annotation.IdRes

android.support.annotation.IdRes is an annotation used to denote resource identifiers (such as @+id/my_id) that are safe to be used for referencing components in the Android framework.

Purpose

android.support.annotation.IdRes was introduced in API level 11 (Android 3.0 Honeycomb) as a way to prevent errors related to the use of non-existent resource identifiers. By using this annotation, developers can ensure that the resources referenced in their code will exist and be available at runtime.

Usage

The @IdRes annotation can be used to add type safety to any code which references resources in the Android framework: 

          
           
@IdRes int myResourceId = R.id.my_id;

//The View corresponding to the id is now available
View myView = findViewById(myResourceId);
This annotation can also be used to inform Android Studio that a particular identifier value is safe to use. This allows the static analysis tools to check for validity of the resource identifier and provide valuable feedback to the developer.

Example

Let’s consider the following example where we have a layout with two TextViews and we want to access them programmatically. 

          
           
<LinearLayout
    android:layout_width="match_parent"
    android:layout_height="wrap_content">
    <TextView
        android:id="@+id/tv_one"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content" />
    <TextView
        android:id="@+id/tv_two"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content" />
</LinearLayout>
We can access the two TextViews by using the @IdRes annotation: 

          
           
@IdRes int tvOneId = R.id.tv_one;
@IdRes int tvTwoId = R.id.tv_two;

TextView tvOne = findViewById(tvOneId);
TextView tvTwo = findViewById(tvTwoId);
By using the @IdRes annotation, we can ensure that the resources referenced in our code exist and are valid. This can significantly reduce the number of runtime errors encountered due to invalid resource identifiers.

Conclusion

android.support.annotation.IdRes is a powerful annotation which helps developers ensure the validity of resource identifiers being used in Android application code. It is important to remember that this annotation is only available starting in API level 11 (Android 3.0 Honeycomb) and so can not be used on earlier versions of Android.
