Android App FragmentTransaction

FragmentTransaction is an API in the Android platform that helps developers manage the interactions and behaviors between different fragments. It is used to add, remove, replace, and animate entire fragment hierarchies. This article will discuss the fundamentals of FragmentTransaction and its associated syntax and usage.

Overview

FragmentTransaction is part of the support library, which provides additional functionality for apps running on older Android versions. First, a FragmentManager object is used to obtain an instance of FragmentTransaction. Then, one or more of the following methods can be used to manipulate fragments: add(), remove(), replace(), and animate(). The add() method adds a given fragment to the calling app's activity, while the remove() method removes a given fragment from the activity. The replace() method replaces a given fragment with another one and the animate() method animates a given fragment within the activity. All these methods accept both a container view ID and a fragment instance as parameters.

Syntax


          
           
FragmentTransaction transaction = getSupportFragmentManager().beginTransaction();
transaction.add(R.id.container, newFragment);
transaction.remove(oldFragment);
transaction.replace(R.id.container, newFragment);
transaction.addToBackStack(null);
transaction.commit();
The above snippet uses the beginTransaction() method to create an instance of FragmentTransaction, and then calls the add(), remove(), replace(), and commit() methods to perform the modification. The first parameter is the ID of the container where the fragment will be hosted, and the second parameter is the fragment itself. The commit() method applies all the changes made to the fragment. The addToBackStack() method is used to make the action reversible.

Usage Example

We can use FragmentTransaction to create a simple form allowing the user to enter their first name and last name. To achieve this, we'll create two separate fragments containing EditText widgets and a button. The first fragment will contain the EditText widgets to ask for the user's first name and the second fragment will contain the EditText widgets to ask for the user's last name. Once the user has entered their details, they can click on the button to submit the form. First, we need to define the layout of each fragment. Finally, we need to add code to the activity to add the two fragments using FragmentTransaction.

transaction.add(R.id.lastNameContainer, new Fragment2());
transaction.commit();
