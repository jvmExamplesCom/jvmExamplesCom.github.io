Understanding the Javax.validation.constraints.Min Annotation

The javax.validation.constraints.Min annotation is used to specify a minimum value that can be accepted in a field or parameter. This is one of many annotations provided by the Java API for Bean Validation, also known as JSR 380.

The annotation is declared by providing a numeric type (int, double, etc) along with the minimum allowable value. It provides a simple way to validate the values in an application. This article covers the basics of the Min annotation, providing an overview and code examples for usage.

An Overview of the Min Annotation

The javax.validation.constraints.Min annotation is used to specify the minimum value that can be accepted for a field or parameter. It is used to ensure that values are within a certain range. The annotation is declared as follows:


          
           
@Min(value=<i>minValue</i>)

Where minValue is the minimum allowable value.

The annotation can be applied to fields, parameters, and constructors. When used with a field, it applies to the value of the field. Similarly, when used with a parameter, it applies to the value of the parameter being passed into a method. For constructors, any parameters that are annotated with Min will be validated before the object is constructed.

The annotation can be applied to numeric types: byte, short, int, long, float, and double. Attempts to use the annotation with a non-numeric type will result in a compile-time error.

Using the Min Annotation

To demonstrate the usage of the Min annotation, consider the following example class:


          
           
public class NumberCheck {

    private int value;
    
    public NumberCheck(@Min(10) int value) {
        this.value = value;
    }
}

This class has a single constructor, which accepts an integer argument annotated with the @Min(10) annotation. This indicates that the value must be greater than or equal to 10.

If we try to create an instance of this class using a value less than 10, we will get a javax.validation.ConstraintViolationException at runtime:


          
           
// This throws a ConstraintViolationException
NumberCheck numberCheck = new NumberCheck(9);

On the other hand, if we use a value that is greater than or equal to 10, the instance will be created successfully:


          
           
// This works
NumberCheck numberCheck = new NumberCheck(10);
