org.apache.logging.log4j.Level

Log4j Level is a class in the package org.apache.logging.log4j and provides constants for all the log levels used in Log4j logging framework. This class is usually used when customizing log messages using pattern layout.

List of Levels


          
           
OFF
FATAL
ERROR
WARN
INFO
DEBUG
TRACE
ALL
Logging process is based on the concept of level. In log4j, each log message has a corresponding log level associated with it. Log levels define the importance or severity of a log message and are defined in the hierarchy of descending priorities i.e most important log level is OFF and least important log level is ALL.

ORDER

The order of the log levels follows this structure: 

          
           
OFF > FATAL > ERROR > WARN > INFO > DEBUG > TRACE > ALL
Log4j provides various methods to set the log level. The most commonly used is the setLevel() method which allows us to set the level of the logger programmatically.

Example


          
           
Logger logger = Logger.getLogger("myLogger");
logger.setLevel(Level.INFO);
logger.info("This is an info message");
logger.debug("This is a debug message");

// Output
2020-06-04 15:30:00 INFO  myLogger - This is an info message
In the above example, the log level is set to Level.INFO so only the info log message will be printed while the debug log message will not be printed as it is lower in priority than INFO.

Conclusion

Log4j Level is an important class used in the Log4j logging framework. It defines various constants for different log levels which are then used for customizing log messages. By setting the appropriate log level, we can specify what type of logs should be printed and which should be suppressed.
