androidx.viewpager.widget.ViewPager

ViewPager is a flexible UI component that provides an easy way to display multiple screens within a single activity in an Android app. It allows the user to easily transition between different screens, while providing a smooth scrolling transition between them. It can be used to create a smooth transition between screens without needing to create complex page transitions. ViewPager supports both horizontal and vertical orientation and can handle complex layouts, such as nested ViewPagers. It also allows for custom views to be displayed, such as images, text, etc.

Code Examples


          
           
ViewPager viewPager = findViewById(R.id.viewPager);
PagerAdapter pagerAdapter = new PagerAdapter();
viewPager.setAdapter(pagerAdapter);
This example sets up a ViewPager with a PagerAdapter. The PagerAdapter will be responsible for creating the actual views that will be displayed in the ViewPager. 

          
           
viewPager.setOnPageChangeListener(new OnPageChangeListener() {
    @Override
    public void onPageSelected(int position) {
        // Do something when the page is selected
    }
});
This example sets up an OnPageChangeListener for the ViewPager. This listener will be called whenever the user scrolls to a different page. This can be used to perform an action when the user scrolls to different pages. 

          
           
viewPager.setPageTransformer(true, new PageTransformer() {
   @Override
   public void transformPage(View page, float position) {
       float normalizedPosition = Math.abs(Math.min(position, 0));
       page.setAlpha(normalizedPosition);
   }
});
This example sets up a PageTransformer for the ViewPager. The PageTransformer is an interface that can be used to define custom animations for the transition between pages. In this case, it sets the alpha of the page to a normalized value of the position. This can be used to create a fading transition between pages. 

          
           
viewPager.setOffscreenPageLimit(5);
This example sets the offscreen page limit for the ViewPager. The offscreen page limit determines the number of pages that should be kept in memory at any given time. If the offscreen page limit is set too low, the ViewPager may need to reload pages that have been scrolled away from, which can result in a poor user experience.
