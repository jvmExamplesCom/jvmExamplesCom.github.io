Introduction to org.testng.annotations.BeforeMethod

The org.testng.annotations.BeforeMethod annotation denotes methods that should be executed before each test method in the current class. This annotation is used to perform activities that are necessary before executing a test, such as instantiating an object or setting variables.

Using org.testng.annotations.BeforeMethod

To use org.testng.annotations.BeforeMethod, you must declare a public non-static method in the class where you want it to execute. This method can be annotated with @BeforeMethod and it will execute before each test method in the same class. The following example shows how to use @BeforeMethod: 

          
           
public class TestClass {
  //...

  @BeforeMethod
  public void setUp() {
    //Do setup stuff here
  }
}
Using this example, the setUp() method will be executed before each test method in TestClass.

Example Use Cases

The @BeforeMethod annotation is typically used for any setup that needs to be done before a test is run. This can include creating a test object, initializing variables, or any other activity that needs to be done before a test executes. For example, suppose you want to test a Car class. You could use the @BeforeMethod annotation to create a Car instance before each test runs: 

          
           
public class CarTest {
  private Car car;

  @BeforeMethod
  public void setUp() {
    car = new Car();
  }

  //...
}
In this example, the setUp() method is annotated with @BeforeMethod and creates a Car instance before each test in CarTest class. This ensures that each test has a valid instance of Car to work with.

Conclusion

The org.testng.annotations.BeforeMethod annotation is a useful tool for setting up an environment before a test is executed. It allows you to create objects or set variables to ensure that every test has what it needs to run successfully.
