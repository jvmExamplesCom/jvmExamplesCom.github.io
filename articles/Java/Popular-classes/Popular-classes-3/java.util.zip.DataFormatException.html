java.util.zip.DataFormatException

Introduction

Java provides a powerful API known as 

          
           
java.util.zip
that supports message-oriented data interchange and deals with data compression and decompression. The 

          
           
DataFormatException
class is specifically used when a data format related exception is thrown while working with zip files in Java. This article will explain the detail of 

          
           
java.util.zip.DataFormatException
with examples and its usage in programming.

What is java.util.zip.DataFormatException?


          
           
java.util.zip.DataFormatException
is a subclass of 

          
           
java.lang.Exception
. It is an unchecked exception that indicates a data format violation when working with zip files in Java. This exception occurs when the format of the data does not match the expected format. It mainly occurs due to the following reasons:
  • When a file is compressed using gzip and it is corrupted, this exception is thrown.
  • When a file is decompressed and the format of the data does not agree with the uncompressed form.
  • When the data being read from an InputStream is of wrong type, this exception occurs.
  • When the data contains more than one member, this exception is thrown.

is thrown when the constructor of

import java.io.File;
import java.io.FileInputStream;
import java.io.IOException;
import java.util.zip.ZipInputStream;

public class DataFormatExceptionExample {

        }
    }
}
