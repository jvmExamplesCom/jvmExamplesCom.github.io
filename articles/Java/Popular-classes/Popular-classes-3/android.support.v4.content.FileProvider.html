android.support.v4.content.FileProvider

The android.support.v4.content.FileProvider class provides a simple way to securely provide access to private files associated with an app to other apps on the device. It is part of the Android Support Library and has been since API Level 4.

Using FileProvider

To use FileProvider, a path has to be specified where private files should be exposed. This can be done by adding the path in the 

XML resource
file: 

          
           
<paths>
    <files-path name="my_files" path="Android/Data/com.example.package/files/" />
</paths>
The 
name
attribute is used to reference the path. This allows for multiple paths to be specified. For example, if there are two paths of 
data/
and 
images/
, they can be referenced respectively as 
my_data_path
and 
my_images_path
. The 
path
attribute specifies where the files are located. Note that by default the files are located in the 
data/data
directory of the app which is only accessible by the app itself. If the files need to be available to other apps, they should be stored in a different directory such as the 

          
           
Android/Data/com.example.package/files/
directory. Once the resource is defined, the 
FileProvider
is registered in the 
AndroidManifest.xml
using the following code: 

          
           
<provider
    android:name="android.support.v4.content.FileProvider"
    android:authorities="${applicationId}.fileprovider"
    android:exported="false"
    android:grantUriPermissions="true">
    <meta-data
        android:name="android.support.FILE_PROVIDER_PATHS"
        android:resource="@xml/file_paths" />
</provider>
The 
android:authorities
attribute is used to set a unique string that identifies the FileProvider. It is recommended to use the application ID as the authority string to avoid conflicts with other applications. The 
android:exported
attribute is used to indicate whether the provider is available to any application on the device, while the 

          
           
android:grantUriPermissions
attribute is used to indicate whether the provider should grant permission to apps accessing the content URI. Finally, the
tag is used to reference the resource file that defines the path where the files are stored. In this case, the 
file_paths.xml
file is referenced. Once these steps have been completed, the FileProvider can be used to share files from the app.

Sharing Files

Once the FileProvider is registered, files can be shared using the following code:


          
           
File myFile = new File("Android/Data/com.example.package/files/myfile.txt");
Uri contentUri = FileProvider.getUriForFile(context, "com.example.package.fileprovider", myFile);

Intent intent = new Intent(Intent.ACTION_SEND);
intent.setType("*/*");
intent.putExtra(Intent.EXTRA_STREAM, contentUri);
intent.addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION);
startActivity(intent);
The 

          
           
FileProvider.getUriForFile
method creates a content URI based on the fileâ€™s path and the authority string. This URI is then used in the 
Intent.EXTRA_STREAM
extra to share the file. The 

          
           
Intent.FLAG_GRANT_READ_URI_PERMISSION
flag is used to grant the receiving app permission to read the content URI.

Example

Below is an example of how to share a file using FileProvider:


          
           
File myFile = new File("Android/Data/com.example.package/files/myfile.txt");
Uri contentUri = FileProvider.getUriForFile(context, "com.example.package.fileprovider", myFile);

Intent intent = new Intent(Intent.ACTION_SEND);
intent.setType("*/*");
intent.putExtra(Intent.EXTRA_STREAM, contentUri);
intent.addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION);
startActivity(intent);

This will open a dialog where the user can choose an app that can read the file. For example, if the user selects the Gmail app, it will open the compose screen with the file already attached.
