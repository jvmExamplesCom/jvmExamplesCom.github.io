org.testng.annotations.BeforeClass

The 
@BeforeClass
annotation in TestNG is used to execute certain methods before the first test method in the current class is executed. This annotation can be used on a method that belongs to any superclass of the test class or even a method in a different class.

Syntax


          
           
@BeforeClass 
public void methodName() { 
    //some code goes here 
}

Example 1: Executing a Method Before the Test

The following example shows how the 
@BeforeClass
annotation can be used to execute a method prior to the execution of the test. 

          
           
import org.testng.annotations.*;
public class BeforeClassTest{
   @BeforeClass
   public void setUp(){
      System.out.println("This is executed before the test");
   }
   @Test
   public void test(){
      System.out.println("This is the test");
   }
}
The output of the above code will be: 

          
           
This is executed before the test
This is the test

Example 2: Executing a Method Before All Tests in the Class

The following example shows how the 
@BeforeClass
annotation can be used to execute a method that is shared by all tests in the class. 

          
           
import org.testng.annotations.*;
public class BeforeClassTest{
   static int count=0; 
   @BeforeClass
   public void setUp(){
      count++;
      System.out.println("This is executed before all tests in the class");
   }
   @Test
   public void test1(){
      System.out.println("This is the test1 - "+count);
   }
   @Test
   public void test2(){
      System.out.println("This is the test2 - "+count);
   }
}
The output of the above code will be: 

          
           
This is executed before all tests in the class
This is the test1 - 1
This is the test2 - 1
As seen from the output, the 
setUp()
method was executed only once. This is because the 
@BeforeClass
annotation in TestNG is used to execute certain methods before the first test method in the current class is executed.
