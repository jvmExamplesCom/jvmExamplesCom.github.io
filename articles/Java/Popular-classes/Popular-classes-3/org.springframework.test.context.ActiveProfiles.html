ActiveProfiles in org.springframework.test.context

The 
ActiveProfiles
annotation is part of the 

          
           
org.springframework.test.context
package and can be used to specify which profiles should be active when running a test. This is useful in situations where the application context needs to be configured differently for different tests.

Using @ActiveProfiles

The annotation can be used on a method-level or class-level. On a method-level, it will apply only to that particular test method, while on a class-level, it will apply to all test methods within that class:


          
           
@RunWith(SpringRunner.class)
@ActiveProfiles("myprofile")
public class SomeTest {

  @Test
  public void someTest() {
       //some test
  }

}

The @ActiveProfiles annotation also accepts an array of profile names, which is useful if you need to enable multiple profiles:


          
           
@RunWith(SpringRunner.class)
@ActiveProfiles({"myprofile1", "myprofile2"})
public class SomeTest {

  @Test
  public void someTest() {
       //some test
  }

}

Inheriting @ActiveProfiles

If an @ActiveProfiles annotation is present on a superclass, the profiles specified will be inherited by the subclasses. This feature can be used to enable profiles in multiple test classes without having to specify them on each individual class.


          
           
@RunWith(SpringRunner.class)
@ActiveProfiles("myprofile")
public abstract class AbstractTest {

  @Test
  public void someTest() {
       //some test
  }

}

public class InheritedTest extends AbstractTest {

  @Test
  public void anotherTest() {
       //another test
  }

}

In this example, both SomeTest and InheritedTest will be executed with the "myprofile" profile enabled.

Using @ActiveProfiles with Parameterized Tests

When using 
Parameterized
tests, the @ActiveProfiles annotation can be used to set different profiles for each test case: 

          
           
@RunWith(Parameterized.class)
@ActiveProfiles
public class SomeParameterizedTest {

   @Parameterized.Parameters
   public static List<Object[]> data() {
      //return test cases with different profiles
   }

   private String profile;

   public SomeParameterizedTest(String profile) {
      this.profile = profile;
   }

   @Before
   public void setup() {
      //set active profile
   }

   @Test
   public void someTest() {
       //some test
   }
}

In this example, the profile for each test case is set in the constructor and taken from the parameter returned by the 

data()
method. The profile can then be set in the 
setup()
method before each test is executed.
