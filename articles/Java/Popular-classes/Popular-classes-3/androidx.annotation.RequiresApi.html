androidx.annotation.RequiresApi

The androidx.annotation.RequiresApi is an annotation that indicates a requirement for certain API levels. This allows developers to provide different behaviors, such as not running certain functionality if the required API level is not available.

Usage Example

The following example shows how to use the RequiresApi annotation in your code:


          
           
@RequiresApi(api = Build.VERSION_CODES.O)
public void runAPI26Methods() {
    // Run API 26 methods here
}
In this example, the method 
runAPI26Methods()
will only be executed if the system on which it is running supports API level 26 or greater.

Annotation Properties

The RequiresApi annotation has the following properties:

  • api: The API level that must be supported for the annotated element to be used.
  • maxSdkVersion: The maximum API level that must be supported for the annotated element to be used.
  • minSdkVersion: The minimum API level that must be supported for the annotated element to be used.

Examples of Use

The following examples demonstrate how to use the RequiresApi annotation in various scenarios:

  • Checking for Support of a Specific API Level: 
    
            
             
@RequiresApi(api = Build.VERSION_CODES.M)
public void runAPI23Methods() {
    // Run API 23 methods here
}
  • Checking for Support of an API Level Range: 
    
            
             
@RequiresApi(minSdkVersion = Build.VERSION_CODES.LOLLIPOP, maxSdkVersion = Build.VERSION_CODES.N)
public void runAPI21To24Methods() {
    // Run API 21-24 methods here
}
  • Checking for Support of Multiple API Levels: 
    
            
             
@RequiresApi({Build.VERSION_CODES.JELLY_BEAN, Build.VERSION_CODES.KITKAT})
public void runAPI16and19Methods() {
    // Run API 16 and 19 methods here
}

Conclusion

The androidx.annotation.RequiresApi annotation provides a simple way to check for support of specific API levels when running code. By using this annotation, developers can ensure that code only runs if the required API level is present.
