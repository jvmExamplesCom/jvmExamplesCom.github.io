javax.xml.XMLConstants

Java XML Constants is a class in the Javax XML library that defines constants for use in an XML application. The class provides constants used in XML documents to denote common namespaces such as

XMLNS_URI,
XML_DTD,
and 
XML_SCHEMA.
The constants are useful in validating XML documents, creating XPath queries, and processing XML files with DOM, SAX, and StAX.

XMLNS_URI

The 

XMLNS_URI
defines the namespace URI of an XML document. It is used to denote the namespace of attributes and elements within the document. The namespace can be any valid URI, but it should be in the form “urn:something.” It must be unique across the web and it is used to help identify the element or attribute within the XML document. 

          
           
String xmlnsUri = "urn:example.com:namespace";
String qualifiedName = "ns:element";
Element element = doc.createElementNS(xmlnsUri, qualifiedName);

In this example, a new element is created with the XMLNS_URI of 


          
           
urn:example.com:namespace
. This namespace is then used to uniquely identify the element in the document.

XML_DTD

The

XML_DTD
contains the definition of an XML document. It is used to validate an XML document and can also be used to define elements, attributes, and more. It is typically an external file that is referenced by an XML document, but it can also be embedded within an XML document. 

          
           
<?xml version="1.0" encoding="utf-8"?>
<!DOCTYPE root [
  <!ELEMENT root (child)> 
  <!ATTLIST root version CDATA #REQUIRED>
]>
<root version="1.0">
  <child/>
</root>

root

version

XML_SCHEMA
 
        
The 

        
XML_SCHEMA
 is an alternative to the DTD and it is preferred by some developers. It uses the XML Schema language to define the structure of an XML document. It is more powerful than the DTD and can be used to define data types, attributes, and even relationships between elements in an XML document. 
         
        

          
           
<?xml version="1.0" encoding="utf-8"?>
<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="root">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="child" type="xs:string" />
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>

          
 
        
This example shows a schema that defines an XML document. It defines an element called 

        
root
 and a child element called 
        
child
, and sets the type of the child element to a string. This is defined using the XML Schema language.
