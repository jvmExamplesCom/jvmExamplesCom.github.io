Java.lang.management.RuntimeMXBean

The Java language package 

          
           
java.lang.management
provides information on the management features of the Java Virtual Machine, and one such class is the 
RuntimeMXBean
. This bean provides access to information about the JVM’s runtime, such as the name of the virtual machine and the name of the class that loaded the JVM. It also lets us find out how long the JVM has been running, and the amount of memory that is allocated to the JVM.

Using the RuntimeMXBean

To use the RuntimeMXBean class, we first need to create an instance of it:


          
           
RuntimeMXBean mxBean = ManagementFactory.getRuntimeMXBean();

Once we have the instance, we can get information about the JVM by calling methods such as 

getVmName()
: 

          
           
System.out.println("VM Name: " + mxBean.getVmName());

We can also get information about how long the JVM has been running, either in milliseconds or in nanoseconds, by calling the 

getUptime()
method: 

          
           
long uptimeMillis = mxBean.getUptime();
long uptimeNanos = mxBean.getUptimeNanos();

Finally, we can get the amount of memory that is allocated to the JVM by calling the 

getTotalMemory()
method: 

          
           
long totalMemory = mxBean.getTotalMemory();

Example

Let’s put all of this together into an example: 

          
           
import java.lang.management.ManagementFactory;
import java.lang.management.RuntimeMXBean;

public class Main {
  public static void main(String[] args) {
    // Get the RuntimeMXBean
    RuntimeMXBean mxBean = ManagementFactory.getRuntimeMXBean();

    // Print out the VM name
    System.out.println("VM Name: " + mxBean.getVmName());

    // Print out the uptime in milliseconds
    System.out.println("Uptime (ms): " + mxBean.getUptime());

    // Print out the total memory allocated to the JVM
    System.out.println("Total Memory (bytes): " + mxBean.getTotalMemory());
  }
}
