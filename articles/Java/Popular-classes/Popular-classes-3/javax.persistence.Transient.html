javax.persistence.Transient

Java Persistence API's javax.persistence.Transient annotation is used to indicate fields that should not be persisted in the database. This annotation can be used either at the field level or method level.

Field Level Usage

When used at the field level, the Transient annotation is applied to the field that should not be persisted. 

          
           
@Entity
public class Employee {

    @Id
    @GeneratedValue
    private Long id;
    
    private String name;

    @Transient
    private String tempName;

}
In the above example, the 

          
           
tempName
field will not be persisted in the database even though it is a member field of the Employee entity.

Method Level Usage

When used at the method level, the Transient annotation should be applied to the getter method of the non-persistent field. 

          
           
@Entity
public class Employee {

    @Id
    @GeneratedValue
    private Long id;
    
    private String name;

    private String tempName;

    @Transient
    public String getTempName() {
        return this.tempName;
    }

}
In the above example, the 

          
           
tempName
field will also not be persisted in the database since its getter method is annotated with 

          
           
Transient
.

Interoperability with Other JPA Providers

The 

          
           
Transient
annotation is part of the JPA specification, so it should work with any JPA provider.
