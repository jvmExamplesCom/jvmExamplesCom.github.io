javax.swing.AbstractAction

The javax.swing.AbstractAction class is a basic implementation of the Action interface that provides default implementations for the various methods. It is intended to be subclassed to provide actual action implementations. In a nutshell, it can be used to create GUI components that execute certain tasks depending on certain state conditions or external factors.

Simplest Example

Let us look at a very simple example to get started. We will create a simple Swing button that will print "Hello World!" when clicked: 

          
           
import javax.swing.*;

public class SimpleButtonAction extends AbstractAction {

    public SimpleButtonAction() {
        super("Say Hello");
    }

    public void actionPerformed(ActionEvent e) {
        System.out.println("Hello World!");
    }
}
Now, we can use this action in our main program to create a JButton which will print "Hello World!" when clicked: 

          
           
SimpleButtonAction action = new SimpleButtonAction();
JButton button = new JButton(action);

Advanced Usage

Now that we have had a simple introduction to using AbstractAction let's take a look at some more advanced features. For example, we can provide custom properties like label, icons, disabled icons, etc. Let us define a new class extending AbstractAction and adding a few of these custom properties: 

          
           
public class AdvancedAction extends AbstractAction {

    public AdvancedAction(String label, String iconPath) {
        super(label);
        ImageIcon icon = new ImageIcon(iconPath);
        putValue(SMALL_ICON, icon);
        putValue(DISABLED_ICON, icon);
    }

    public void actionPerformed(ActionEvent e) {
        // do something
    }
}
Now, we can use this class to create buttons with custom icons, labels, etc.: 

          
           
AdvancedAction action = new AdvancedAction("My Button", "/path/to/icon.jpg");
JButton button = new JButton(action);
