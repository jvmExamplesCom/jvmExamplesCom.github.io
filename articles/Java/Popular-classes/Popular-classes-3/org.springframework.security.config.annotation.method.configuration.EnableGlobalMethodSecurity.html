org.springframework.security.config.annotation.method.configuration.EnableGlobalMethodSecurity

The 

          
           
@EnableGlobalMethodSecurity
annotation enables global method security, allowing the developer to create secured methods and use method-level security annotations on the service layer. Using this annotation ensures that any method annotated with a security annotation will be evaluated against the provided configuration (i.e., an @PreAuthorize annotation).

The following code example demonstrates the use of the 


          
           
@EnableGlobalMethodSecurity
annotation: 

          
           
@Configuration
@EnableGlobalMethodSecurity(prePostEnabled=true)
public class SecurityConfiguration {
	// configuration here
}
The 

          
           
@EnableGlobalMethodSecurity
annotation takes two parameters for controlling the type of security that is enabled. The first parameter is 
securedEnabled
, which enables secure annotations 
@Secured
. The second parameter is 
prePostEnabled
which enables pre and post annotations 
@PreAuthorize
and 
@PostAuthorize
. In this case, we have enabled both parameters.

@Secured Annotation

The 
@Secured
annotation is used to define secure methods and protect them from unauthorized access. It takes a list of strings that represent roles that are granted access. For example, the following code defines a method that can only be accessed by users with the role "ADMIN". 

          
           
@Secured("ROLE_ADMIN")
public void foo() {
	// code here
}

@PreAuthorize Annotation

The 
@PreAuthorize
annotation is used to define methods that are protected by an expression. This expression can contain any logic involving the security context, including user roles and other properties. For example, the following expression only permits the method to be called if the user has the "ADMIN" role or the "MANAGER" role. 

          
           
@PreAuthorize("hasRole('ADMIN') or hasRole('MANAGER')")
public void bar() {
	// code here
}

@PostAuthorize Annotation

The 
@PostAuthorize
annotation is similar to the 
@PreAuthorize
annotation, but it is evaluated after the method is executed. This allows the expression to take into account the return value of the method. For example, the following expression only permits the method to be called if the user has the "ADMIN" role and the method returns true. 

          
           
@PostAuthorize("hasRole('ADMIN') and returnObject")
public boolean baz() {
	// code here
