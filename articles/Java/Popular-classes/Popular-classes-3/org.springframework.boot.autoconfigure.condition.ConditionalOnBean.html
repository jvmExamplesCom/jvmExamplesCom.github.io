Introduction to org.springframework.boot.autoconfigure.condition.ConditionalOnBean

The 

          
           
org.springframework.boot.autoconfigure.condition.ConditionalOnBean
annotation allows the application to specify certain bean classes or types that must be present in order for a particular configuration to be applied. This allows developers to create more reusable and pluggable components, as well as components that are easier to troubleshoot.

Annotating with ConditionalOnBean

To use this annotation, developers must specify the classes and/or types of beans that must be present in the application context. These classes must have already been statically defined in the application context, either explicitly or through annotations. For example, an application annotated with @Configuration can define a bean class like this:


          
           
@Configuration
class AppConfiguration {
    @Bean
    public MyService myService() {
        return new MyService();
    }
}

To ensure that this bean is present before a particular configuration is applied, the annotation should be used like this:


          
           
@Configuration
@ConditionalOnBean(MyService.class) //MyService must be present
class OtherConfiguration { 
    // ... 
}
In this case, the 
OtherConfiguration
class will only be applied if the 
AppConfiguration
class is present and contains the 
MyService
bean.

Negating the Condition

In some cases, it may be useful to conditionally apply a configuration only if a certain bean is not present. This can be accomplished by using the 


          
           
@ConditionalOnBean(value=MyService.class, negated=true)
annotation attribute. 

          
           
@Configuration
@ConditionalOnBean(value=MyService.class, negated=true) //MyService must not be present
class YetAnotherConfiguration { 
    // ... 
}
In this case, the 

          
           
YetAnotherConfiguration
class will only be applied if the 
MyService
bean is not present in the application context.

Conclusion

The 
@ConditionalOnBean
annotation makes it easy to conditionally apply configuration in Spring Boot applications.
