What is a BiMap?

A BiMap (short for Bidirectional Map) is a type of Map that associates two unique keys with each other in both directions. This means that if you have two keys A and B, then A maps to B, and B maps to A. It is important to note that each key can only map to one other key, so if you inserted multiple items with the same key they would all overwrite each other. Unlike a traditional Map, a BiMap also allows you to retrieve the keys associated with a value. This means that if you have an object with a value of 'foo', you can lookup the key associated with the object.

How Does a BiMap Work?

In order to understand how a BiMap works, we must first look at the data structure itself. A BiMap contains two maps, a forward map and a reverse map, which both point to each other. The forward map associates keys with values, while the reverse map associates values with keys. The biMap will use both of these maps whenever it needs to perform a lookup. For example, if we were performing a lookup for the key ‘foo’, the biMap would first check the forward map for a mapping from ‘foo’ to its associated value. If a mapping was found, then the key would be returned. If no mapping was found in the forward map, then the biMap would then check the reverse map for a mapping from the value to its associated key. If a mapping was found, then the key would be returned. If no mapping was found in either the forward or reverse map, then the biMap would return null.

When Should I Use a BiMap?

BiMaps are a powerful tool that can be used when you need bidirectional mapping between objects. This is especially useful in situations where you need to look up an object by either its key or its value. For example, let's say you have a list of objects and each object has a unique ‘name’ property. If you wanted to quickly find an object by its name, you could create a BiMap that maps the object's name to its reference.
