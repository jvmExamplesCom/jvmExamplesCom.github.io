javax.ws.rs.core.Response.Status

The javax.ws.rs.core.Response.Status class defines a set of constants that are used to map from the version of the HTTP protocol used in a response to the appropriate Response.Status value.

Response.Status provides a convenient way for applications to get a Response based on the version of HTTP being used. This makes it easier for developers to handle different versions of HTTP responses, as they only need to define a single set of status codes.

Examples

Let's look at an example of how to use Response.Status.


          
           
Response response = Response.status(Response.Status.OK)
    .entity("This is the response body").build();

In this example, the response is created using the Response.Status.OK constant. This means that the the response will return with the HTTP status code 200 (OK).

We can also create a Response object with a specific HTTP status code. For example:


          
           
Response response = Response.status(Response.Status.NOT_FOUND)
    .entity("The requested resource was not found").build();

This example creates a Response object with the HTTP status code 404 (NOT FOUND).

We can also create a Response object with a custom status code and message.


          
           
Response response = Response.status(418)
    .entity("I'm a teapot").build();

In this example, the response will return with the HTTP status code 418 (I’m a teapot).

Conclusion

The Response.Status class makes it easy for applications to handle different versions of HTTP responses. By mapping from the version of the HTTP protocol used in a response to the appropriate Response.Status value, developers can save time and reduce code complexity.
