Java.time.OffsetDateTime

Introduction

Java 8 introduced the concept of a new API for working with dates and times, namely 

          
           
java.time
. 

          
           
OffsetDateTime
is one of the classes from the new java.time API that stores an instant on the timeline in terms of a local date-time and an offset from UTC/Greenwich. It provides unmodifiable objects that represent a date-time, with various methods for manipulation and queries.

Creating an OffsetDateTime Object

We can create an instance of 

          
           
OffsetDateTime
easily using the static factory method 

          
           
now()
which returns a 

          
           
OffsetDateTime
object initialized to the current system time. Additionally, there are methods like 

          
           
now(clock)
and 

          
           
now(ZoneId)
, to get the date-time based on the provided clock or time zone respectively. Let's look at a few examples. The following code snippet will create an 

          
           
OffsetDateTime
object initialized to the current system time: 

          
           
OffsetDateTime offsetDT = OffsetDateTime.now();
The following snippet will create an 

          
           
OffsetDateTime
object using a pre-defined 

          
           
Clock
object: 

          
           
Clock clock = Clock.systemUTC();
OffsetDateTime offsetDT = OffsetDateTime.now(clock);

OffsetDateTime
by providing a 

          
           
ZoneId

ZoneId zoneId = ZoneId.of("Europe/Paris");
OffsetDateTime offsetDT = OffsetDateTime.now(zoneId);

class provides several methods for manipulating a date-time value. These manipulation methods allow us to add or subtract from an existing 

          
           
OffsetDateTime
