javax.persistence.PersistenceContext

The javax.persistence.PersistenceContext annotation is used to specify the persistence context in a Java EE environment. The persistence context is an object-relational mapping service, which allows Java objects to be mapped to persistent storage and connection pooling. It is used to define the mapping between Java classes and database tables, as well as to specify the data source that will provide access to the database.

The PersistenceContext annotation is often used with an EntityManagerFactory, which provides access to the database. The EntityManagerFactory is responsible for creating EntityManagers, which are used to query the database and store Java objects in the database.

Examples

To use the PersistenceContext annotation, it must be added to the class by using the @PersistenceContext annotation. For example:


          
           
@PersistenceContext
private EntityManager entityManager;

This example will create an EntityManager that is able to query the database and store Java objects. The EntityManager can then be used to retrieve and store data. The following example shows a simple query:


          
           
String query = "SELECT * FROM EMPLOYEE";
Query q = entityManager.createQuery(query);
List resultList = q.getResultList();

This example will retrieve all records from the employee table and store them in a list. The resultList can then be iterated over to process each record.

Another common use of the PersistenceContext annotation is to define the data source that will be used to access the database. This is done by setting the “dataSource” attribute of the annotation. For example:


          
           
@PersistenceContext(dataSource="jdbc/MyDataSource")
private EntityManager entityManager;

This example will create an EntityManager that will use the data source “jdbc/MyDataSource” to access the database. It is important to note that this data source must be configured in the application server before it can be used.
