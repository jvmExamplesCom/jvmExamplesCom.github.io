javax.servlet.DispatcherType

The javax.servlet.DispatcherType enum class is a Java type used to identify the various types of request dispatching in the Servlet API. It contains constants for REQUEST, FORWARD, INCLUDE, and ASYNC.

The REQUEST constant indicates the initial request type of the servlet, or another request caused by a client action, such as clicking a link or submitting a form.


          
           
// Example of using the DispatcherType enum	
if (request.getDispatcherType() == DispatcherType.REQUEST) {
	// Handle the initial request
}

The FORWARD constant indicates the request was made due to a servlet forwarding the request. This could happen due to errors, or an action in which a request was forwarded to a different resource.


          
           
// Example of using the DispatcherType enum
if (request.getDispatcherType() == DispatcherType.FORWARD) {
	// Handle the forwarded request
}

The INCLUDE constant indicates that the request was made due to a request for a servlet include.


          
           
// Example of using the DispatcherType enum
if (request.getDispatcherType() == DispatcherType.INCLUDE) {
	// Handle the included request
}

ASYNC DispatcherType

The ASYNC constant indicates that the asynchronous processing mode of a servlet was used. This allows a long-running operation to occur without blocking other requests.


          
           
// Example of using the DispatcherType enum
if (request.getDispatcherType() == DispatcherType.ASYNC) {
	// Handle the async request
}

You can also check the dispatcher type in a web application context by calling 


          
           
servletRequest.getDispatcherType()
. This will return an instance of DispatcherType, with one of the above constants.
