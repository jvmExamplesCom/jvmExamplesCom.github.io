KafkaProducer

KafkaProducer is a Java library used for sending records to Apache Kafka. This client acts as a producer, which able to publish streams of records to topics in the Kafka cluster. The following example creates a new Producer instance and sends a message to a specific topic: 


          
           
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("key.serializer", StringSerializer.class.getName());
props.put("value.serializer", StringSerializer.class.getName());
 
KafkaProducer<String, String> producer = new KafkaProducer<>(props);
 
ProducerRecord<String, String> record = new ProducerRecord<>("my-topic", key, value);
producer.send(record);

The ProducerRecord is the primary object for producing messages into the Kafka cluster. It contains the topic, the optional key and the message payload.

To customize the behavior of the producer, you can set the following configuration properties:

  • acks: The number of acknowledgments the producer requires from the Kafka broker before considering a produce request successful.
  • retries: The maximum number of times the producer will retry a failed request before giving up and failing the request.
  • batch.size: The maximum size of a batch of messages sent to the Kafka broker.
  • linger.ms: The amount of time the producer will wait before sending a batch of messages.

Sending Messages Synchronously

Producers can send messages synchronously or asynchronously. Sending messages synchronously blocks the calling thread until the Kafka broker acknowledges the message. To send messages synchronously, use the send() method of the Producer instance. The following example shows how to send a single message synchronously:


          
           
Future<RecordMetadata> future = producer.send(record);
RecordMetadata metadata = future.get();

The send() returns a Future, which can be used to retrieve the result of the send operation. The get() method of the Future blocks the calling thread until the operation is complete and returns a RecordMetadata instance.

Sending Messages Asynchronously

Producers can also send messages asynchronously, by providing a callback function that is triggered when a message is successfully sent or an error occurs. To send a message asynchronously, use the send() method with the optional callback parameter. The following example shows how to send a single message asynchronously:


          
           
producer.send(record, (metadata, exception) -> {
    if (exception == null) {
        // Message was successfully sent
    }
    else {
        // Handle error
    }
});

The send() method triggers the provided callback with either a RecordMetadata instance or an exception. The callback is invoked in the background thread.

When you are finished using the producer, you should close it to release resources and cleanly disconnect from the Kafka cluster. The following example shows how to close the producer:
