Mockito.Matchers

Mockito's Matchers class provides a library of matcher objects that can be used to create Mockito-based unit tests. Matchers are used to match specific scenarios, such as verifying that a method call is invoked with the proper arguments or that a field or method is set to a specific value.

Examples of Matchers

One of the most common Matchers is the any() Matcher. This Matcher will match any argument passed to a mocked method call:


          
           
// Verify method was invoked with any argument
verify(mockObject).someMethod(any());

The anyInt() Matcher allows you to verify that a method was invoked with an integer argument:


          
           
// Verify method was invoked with an int argument
verify(mockObject).someMethod(anyInt());

The anyString() Matcher verifies that a method was called with a String argument:


          
           
// Verify method was invoked with a String argument
verify(mockObject).someMethod(anyString());

The isA() Matcher verifies that a method was called with an instance of the specified class:


          
           
// Verify method was invoked with an ArrayList
verify(mockObject).someMethod(isA(ArrayList.class));

The eq() Matcher verifies that a method was called with the exact same object reference:


          
           
ArrayList obj = new ArrayList(); 
// Verify method was invoked with the same object
verify(mockObject).someMethod(eq(obj));

The argThat() Matcher allows you to define your own custom argument matcher. This is useful if your test requires more complex matching logic than the default matchers provide:


          
           
// Custom argument matcher
verify(mockObject).someMethod(argThat(arg -> arg.length() == 10));

Conclusion

Mockito's Matchers class provides powerful tools for creating flexible, expressive tests. By leveraging the various Matchers available, you can verify that your code is behaving as expected without relying on complicated mock objects.
