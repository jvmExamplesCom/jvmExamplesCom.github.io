javax.naming.Context

The javax.naming.Context interface is used to represent a naming context in Java. It provides methods for looking up names and binding objects to the context. Each naming context contains a single path of names, referred to as a naming context. The Context interface also provides methods for creating new subcontexts, searching for objects within a specific context, and for removing a named object from its context.

The following code example shows how to create a new Context object and use it to lookup a name:


          
           
Context ctx = new InitialContext();
Object obj = ctx.lookup("name");

The above code example creates a new Context object and uses it to lookup a name. The lookup operation returns an Object that can be cast to the expected type.

Binding Objects

The javax.naming.Context interface provides methods for binding objects to the context. The following code example shows how to bind an object to a name:


          
           
Context ctx = new InitialContext();
ctx.bind("name", obj);

The above code example binds an object to a name in the context. The object must be Serializable in order to be bound.

The javax.naming.Context interface provides methods for creating subcontexts. The following code example shows how to create a new subcontext:


          
           
Context ctx = new InitialContext();

NamingEnumeration results = ctx.search("name", "*");
