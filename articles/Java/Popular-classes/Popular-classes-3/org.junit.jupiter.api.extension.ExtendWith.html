org.junit.jupiter.api.extension.ExtendWith

The 
ExtendWith
annotation is used to register a custom JUnit Jupiter extension for the annotated test class or test method. This annotation can be used as an alternative to the @ExtendWith annotation when using JUnit 4-style tests (classes annotated with @RunWith(JUnitPlatform.class)) on the JUnit Platform.

Example

The following example shows how to extend a JUnit Jupiter test class with a custom extension.


          
           
@ExtendWith(MyExtension.class)
public class MyTest {

    @Test
    public void testCase() {
        // ...
    }
}

The following example shows how to extend a JUnit Jupiter test method with a custom extension.


          
           
public class MyTest {

    @ExtendWith(MyOtherExtension.class)
    @Test
    public void testCase() {
        // ...
    }
}

Using ExtensionContext to Access Test State

When writing extensions, you may need to access the state of individual tests. To do this, you must use the ExtensionContext parameter of the extension methods (e.g. beforeEach(), afterEach(), etc.). The ExtensionContext provides access to the underlying test instance, test method, and associated test class. It also provides access to the current Store, which can be used to share state between extension methods.


          
           
public class ExampleExtension implements BeforeEachCallback, AfterEachCallback {

    public void beforeEach(ExtensionContext context) {
        System.out.println("Before each test: " + context.getDisplayName());
    }

    public void afterEach(ExtensionContext context) {
        System.out.println("After each test: " + context.getDisplayName());
    }
}

For more information on the various methods available in ExtensionContext, please refer to the Javadoc.

Writing your Own Extension

To write your own extensions, you must create a class that implements the Extension interface. This interface has several methods that are called by the JUnit Platform when a test is being executed. For example, the beforeAll() and afterAll() methods are called before and after the entire test suite is run respectively.


          
           
public class MyExtension implements Extension {

    public void beforeAll(ExtensionContext context) {
        System.out.println("Before all tests");
    }

    public void afterAll(ExtensionContext context) {
        System.out.println("After all tests");
    }

}

You can also implement the BeforeEachCallback and AfterEachCallback interfaces to provide logic that should be executed before and after each test respectively.


          
           
public class MyExtension implements BeforeEachCallback, AfterEachCallback {

    public void beforeEach(ExtensionContext context) {
        System.out.println("Before each test: " + context.getDisplayName());
    }

    public void afterEach(ExtensionContext context) {
        System.out.println("After each test: " + context.getDisplayName());
    }

}

The Extension interface also provides methods for overriding the test execution order. You can implement the getExecutionMode() and the getExecutionOrder() methods to control the order in which tests are executed. 


          
           
public class MyExtension implements Extension {

    public ExecutionMode getExecutionMode() {
        return ExecutionMode.CONCURRENT;
    }

    public int getExecutionOrder() {
        return 30;
    }

}
