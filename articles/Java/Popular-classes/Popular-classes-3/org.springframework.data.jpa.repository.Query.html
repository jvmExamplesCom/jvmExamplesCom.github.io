What is JPA?

Java Persistence API (JPA) is a Java application programming interface that defines how to access and persist data between Java objects/classes and relational databases. It's used to manage the persistence of data in applications such as web applications, desktop applications, mobile applications, and enterprise applications.

org.springframework.data.jpa.repository.Query

org.springframework.data.jpa.repository.Query is an interface used to create and execute JPA queries. The Query interface allows developers to express database queries with a fluent API. It also provides access to query creation methods such as createNamedQuery() and createNativeQuery().

Usage

To use org.springframework.data.jpa.repository.Query you must first import the interface: 

          
           
import org.springframework.data.jpa.repository.Query;
Once imported, you can declare a method in your JPA repository and annotate it with the @Query annotation, passing in the query string as a parameter: 

          
           
@Query("SELECT u FROM User u WHERE u.name like ?1")
List<User> findAllByName(String name);
The above query will return all User entities with a matching name. You can also use native SQL queries with org.springframework.data.jpa.repository.Query by annotating the method with the @Query annotation and passing in the native query as a parameter: 

          
           
@Query(value="SELECT * FROM User WHERE name=?1", nativeQuery = true)
List<User> findAllByName(String name);
The above query will return all User entities with a matching name.

Parameters

The @Query annotation supports parameters which allow you to pass method arguments into your query for better control over the results. For example, if you want to return a list of users based on the age parameter, you can use the following query: 

          
           
@Query("SELECT u FROM User u WHERE u.age > ?1")
List<User> findAllByName(Integer age);
The above query will return all users who are older than the given age parameter.

Named Parameters

In addition to positional parameters, you can also use named parameters in your queries. Named parameters are defined using a colon followed by the parameter name. For example, the following query uses a named parameter to filter results based on the user's name: 

          
           
@Query("SELECT u FROM User u WHERE u.name = :name")
List<User> findAllByName(@Param("name") String name);
The above query will return all users with a matching name.

Limiting Results

You can also limit the number of results returned in a query by using the @MaxResults and @FirstResult annotations. For example, the following query will return a list of users limited to three results: 

          
           
@Query("SELECT u FROM User u WHERE u.age > ?1")
@MaxResults(3)
@FirstResult(0)
List<User> findAllByName(Integer age);
The above query will return the first 3 users who are older than the given age parameter.

Conclusion

org.springframework.data.jpa.repository.Query is a powerful tool for creating and executing JPA queries. It allows developers to easily create and execute queries using a fluent API, as well as add parameters and limiting the results returned.
