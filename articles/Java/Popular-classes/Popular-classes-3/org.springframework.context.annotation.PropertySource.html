Introduction to org.springframework.context.annotation.PropertySource

The 

          
           
org.springframework.context.annotation.PropertySource
class allows Spring applications to externalize their configuration from code. This enables quick configuration changes and proper modularization in an application. It also allows for controlling the configuration of an application with an externalized source such as a properties file or an environment variable. In this article, we will explore how to use this class and how it can be useful in different scenarios.

Using org.springframework.context.annotation.PropertySource

The 

          
           
org.springframework.context.annotation.PropertySource
annotation is used to load external configuration for a Spring application. It is typically used in conjunction with the @Configuration annotation as shown in the example below: 

          
           
@Configuration
@PropertySource("classpath:application.properties")
public class AppConfig {
   // other configuration
}
The 

          
           
@PropertySource
annotation takes a single argument which is the location of the external configuration to use for the application. The location can be a classpath resource, an absolute path, a file path or an URL. The 

          
           
@PropertySource
annotation can also take optional parameters. For example, it can be used with the 

          
           
ignoreResourceNotFound
parameter which can be set to 

          
           
true
if the resource is not found, to ignore the missing resource. 

          
           
@Configuration
@PropertySource(value="classpath:application.properties", ignoreResourceNotFound=true)
public class AppConfig {
   // other configuration
}
The 

          
           
@PropertySource
annotation can also be used in conjunction with the @Profile annotation to specify which profiles the external configuration should be loaded for. For example, the following configuration will only load the external configuration when the profile “dev” is activated: 

          
           
@Configuration
@Profile("dev")
@PropertySource("classpath:application.properties")
public class AppConfig {
   // other configuration
}

Retrieving Values From External Configurations

Once the external configuration is loaded with the 

          
           
@PropertySource
annotation, the values from that configuration can be retrieved using the Environment abstraction available in Spring. The 

          
           
Environment
object can be injected using the @Autowired annotation. For example, the following code will inject an Environment object into the class and retrieve values from the external configuration that was loaded: 

          
           
@Component
public class MyBean {

    @Autowired
    private Environment env;

    public void doSomething() {
        String value = env.getProperty("someProperty");
        System.out.println(value);
    }
}
