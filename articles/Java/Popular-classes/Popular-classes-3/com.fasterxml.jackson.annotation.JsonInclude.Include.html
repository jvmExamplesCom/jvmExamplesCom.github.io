JsonInclude.Include

The JsonInclude.Include enumeration is a set of constant values that indicate how properties or fields that have null-values are handled by Jackson during serialization and deserialization.

The simplest way to use the enumeration is to annotate classes and fields with the appropriate constants. This can be done with the @JsonInclude annotation.

Example 1: Setting Default Serialization Behavior 


          
           
@JsonInclude(JsonInclude.Include.ALWAYS) 
public class MyClass { ... }

The above example sets the default serialization behavior for all members of the MyClass class. With this configuration, all members of the class will be serialized, even if their corresponding fields or properties have null-values.

Example 2: Annotating Fields Directly 


          
           
public class MyClass { 
   @JsonInclude(JsonInclude.Include.NON_NULL) 
   private String field1;
   private String field2;
   ...
}

This example shows how fields can be annotated directly in order to override their default serialization behavior. In this case, the field named field1 will only be serialized if its value is not null. The field named field2 will be serialized regardless of its value, since it has no @JsonInclude annotation.

Example 3: Nested Annotation 


          
           
@JsonInclude(value=JsonInclude.Include.NON_EMPTY, content=JsonInclude.Include.NON_NULL) 
public class MyClass { ... }

The nested annotation shown in this example allows for more granular control over serialization behavior. In this case, the outer annotation applies to the entire class, and indicates that any member whose value is an empty collection (e.g., list or map) will not be serialized. The inner annotation overrides this behavior in the case of members whose values are not collections. In those cases, the inner annotation dictates that the member will only be serialized if its value is not null.
