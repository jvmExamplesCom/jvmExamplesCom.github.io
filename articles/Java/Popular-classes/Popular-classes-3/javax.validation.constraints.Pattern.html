Javax.Validation.Constraints.Pattern

Javax.Validation.Constraints.Pattern is an annotation type used for dependency injection to define a set of constraints on a given field or property when it comes to validation. It uses regular expressions for the purpose of verification. It allows for easy and efficient implementation of validators to validate input data. It is accomplished by simply annotating a field or method with particular annotation defined in the javax.validation.constraints package.

Example of usage

Let's assume that we have a Java class called MyClass which contains one field called "name" of type String.


          
           
public class MyClass {
    private String name;
}

We want to make sure that the value of this field is valid. For this, we can use the Javax.Validation.Constraints.Pattern annotation as follows:


          
           
@Pattern(regexp = "^[a-zA-Z]+$")
public class MyClass {
    private String name;
}

This annotation will ensure that the value of the field is a valid string, i.e., it should only contain letters from the alphabet. Any other characters and/or numbers will throw an exception at runtime.

How does it work?

The Javax.Validation.Constraints.Pattern annotation takes two arguments - a regular expression and an error message. The regular expression is used to verify if the data is valid or not and the error message provides information about what went wrong.

When a particular field is annotated with a Pattern annotation, the validate() method of the Validator class is invoked. This method verifies the data using the supplied regular expression and throws an exception if the data does not match the pattern.


          
           
@Pattern(regexp="^[a-zA-Z]+$", message="value must consist of only letters")
public class MyClass {
    private String name;
}

In the above example, if the data provided does not match the regular expression ^[a-zA-Z]+$, an exception will be thrown and the user will see the error message “value must consist of only letters”.

Conclusion

Javax.Validation.Constraints.Pattern is a useful annotation type that helps developers quickly and efficiently implement validators to validate input data. It is particularly useful when dealing with strings and is easy to understand and use.
