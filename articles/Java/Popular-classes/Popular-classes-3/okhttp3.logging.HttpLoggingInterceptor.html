Introduction to okhttp3.logging.HttpLoggingInterceptor

okhttp3.logging.HttpLoggingInterceptor is a powerful logging interceptor for the 

OkHttp
library. It allows you to easily log all HTTP requests and responses from your app so you can debug it quickly.

Installation

This library is available in Maven Central and can be easily added to your project:


          
           
compile 'com.squareup.okhttp3:logging-interceptor:3.X.X'

Usage

Once you have added the library to your project, you can then use it in your code like so:


          
           
HttpLoggingInterceptor logging = new HttpLoggingInterceptor();
logging.setLevel(Level.BASIC);
OkHttpClient client = new OkHttpClient.Builder()
    .addInterceptor(logging)
    .build();
// ...
client.newCall(request).execute(); // logs request and response
The 

          
           
HttpLoggingInterceptor
accepts an 
enum Level
, where 
Level.BASIC
is the lowest verbosity level and 
Level.BODY
is the highest.

If you have a need for more granular control over the log output, you can also provide a custom 

Logger
implementation. 

          
           
HttpLoggingInterceptor logging = new HttpLoggingInterceptor(new Logger() {
    @Override public void log(String message) {
        // Insert custom logging behavior here
    }
});
// ...

Examples

Here are some examples of how you can use the 

          
           
HttpLoggingInterceptor
to log different parts of a request or response.

Request Headers

You can log a request's headers with the 
HEADERS
level: 

          
           
HttpLoggingInterceptor logging = new HttpLoggingInterceptor();
logging.setLevel(Level.HEADERS);
OkHttpClient client = new OkHttpClient.Builder()
    .addInterceptor(logging)
    .build();
// ...
client.newCall(request).execute(); // logs request headers

Response Headers

You can log a response's headers with the 
HEADERS
level: 

          
           
HttpLoggingInterceptor logging = new HttpLoggingInterceptor();
logging.setLevel(Level.HEADERS);
OkHttpClient client = new OkHttpClient.Builder()
    .addInterceptor(logging)
    .build();
// ...
client.newCall(request).execute(); // logs response headers

Request and Response Body

You can log both a request and response body with the 

BODY
level: 

          
           
HttpLoggingInterceptor logging = new HttpLoggingInterceptor();
logging.setLevel(Level.BODY);
OkHttpClient client = new OkHttpClient.Builder()
    .addInterceptor(logging)
    .build();
// ...
client.newCall(request).execute(); // logs request &amp; response body

Conclusion

The 

          
           
okhttp3.logging.HttpLoggingInterceptor
is a powerful and flexible way to log requests and responses on the 
OkHttp
library. With its various log levels, you can easily debug any issue you run into while developing your app.
