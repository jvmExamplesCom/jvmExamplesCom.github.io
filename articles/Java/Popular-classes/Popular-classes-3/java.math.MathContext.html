Java.math.MathContext

Java.math.MathContext enables precision math operations on arbitrary-precision numbers by encapsulating a given precision setting. It includes a set of constants that define certain common MathContexts, such as DECIMAL32, DECIMAL64, DECIMAL128, and so forth. It also lets you create arbitrary MathContexts with whatever settings you provide.

Overview

A Java MathContext object consists of the following parameters:
  • Precision - The number of digits used for the result of an operation.
  • Rounding Mode - The rounding algorithm used for operations that produce fractional results.
  • Form - The form in which arithmetic results are expressed.
The MathContext class is immutable. Once created, the properties cannot be modified.

Examples

To create a MathContext object with precision=10, rounding mode=HALF_EVEN, and form=UNNECESSARY, use the following code: 

          
           
MathContext mc = new MathContext(10, RoundingMode.HALF_EVEN, Form.UNNECESSARY);
To create a predefined MathContext object, use one of the constants provided by the MathContext class, such as: 

          
           
MathContext mc2 = MathContext.DECIMAL32;
To use the MathContext object in a calculation, simply pass it as an argument to the appropriate method. For example: 

          
           
BigDecimal d1 = new BigDecimal("123.4567");

The Java.math.MathContext class provides a flexible framework for performing precision math operations on arbitrarily-precision numbers. By encapsulating a precision setting in an object, it makes it easy to pass this setting around in your code, and also to modify it if needed.
