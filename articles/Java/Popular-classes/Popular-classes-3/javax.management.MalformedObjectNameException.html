javax.management.MalformedObjectNameException

javax.management.MalformedObjectNameException is a Java exception that is thrown when an invalid ObjectName is passed to the JMX server. The object name must be formatted according to the rules set out in the JMX specification in order to be accepted by the server. If an invalid object name is sent, then this exception will be thrown.

try {
    ObjectName objName = new ObjectName(objectNameVal);
} catch (MalformedObjectNameException e) {
   ...
}

Example

In the example below, we create an invalid object name, which causes the MalformedObjectNameException to be thrown.


          
           
try {
    ObjectName objName = new ObjectName("name:*");
} catch (MalformedObjectNameException e) {
    System.out.println("Error: " + e.getMessage());
}

Cause

The cause of this exception is usually caused by an invalid ObjectName that has been passed to the JMX server. As per the JMX specification, an ObjectName must be in the form of 


          
           
domain:property=value
. If the ObjectName contains any characters that are not valid, then this exception will be thrown.

Fixing the Error

In order to fix the error, you need to ensure that the ObjectName that is being passed to the JMX server is valid according to the JMX specification. This means that all characters should either be alphanumeric, or one of the following special characters: 

:.-_*?
