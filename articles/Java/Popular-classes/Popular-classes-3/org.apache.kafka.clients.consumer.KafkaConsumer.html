Introduction to org.apache.kafka.clients.consumer.KafkaConsumer

KafkaConsumer is an API provided by Apache Kafka for Java developers to integrate their applications with the Apache Kafka messaging system. This API allows developers to read, write, and process data from Kafka topics. It provides key features such as a transactional API, support for various message formats, and robust error handling. In this article, we will explore the KafkaConsumer API in detail and see how it can be used for applications.

Overview of KafkaConsumer API

The KafkaConsumer API is a set of classes and methods that allow developers to integrate their applications with the Apache Kafka messaging system. It provides functions for creating and configuring the consumer object, subscribing to topics, consuming messages from topics, and handling errors. It also has support for batch processing and transactions. The API is designed to be extensible and can be used for other messaging systems such as RabbitMQ and ActiveMQ.

Creating a KafkaConsumer instance

To create a KafkaConsumer instance, you need to provide a set of configuration settings. The configuration settings control the behavior of the consumer and must be specified before the consumer can be used. The following code example shows how to create a KafkaConsumer instance: 

          
           
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "test");
props.put("enable.auto.commit", "true");
props.put("auto.commit.interval.ms", "1000");
props.put("session.timeout.ms", "30000");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
In this example, we are creating a KafkaConsumer instance with the bootstrap.servers property set to localhost:9092, the group.id set to test, and the key and value deserializers set to org.apache.kafka.common.serialization.StringDeserializer. Additionally, we are setting enable.auto.commit to true, auto.commit.interval.ms to 1000, and session.timeout.ms to 30000.

Subscribing to Topics

Once the consumer is created, the next step is to subscribe to the desired topics. This can be done using the consumer's subscribe() method. The following code example shows how to subscribe to a single topic: 

          
           
consumer.subscribe(Collections.singletonList("topic-name"));
In this example, we are subscribing the consumer to the topic-name topic. This method can also be used to subscribe to multiple topics, in which case the topics should be specified as a list of strings.

Consuming Messages

Once the consumer is subscribed to the desired topics, the next step is to consume messages from the topics. This can be done using the consumer's poll() method. The poll() method returns records for all the subscribed topics. Each record contains the topic, partition, offset, and key-value pair of the message. The following code example shows how to consume messages from the subscribed topics: 

          
           
ConsumerRecords<String, String> records = consumer.poll(100);
for (ConsumerRecord<String, String> record : records) {
    System.out.println("topic = " + record.topic() 
            + ", partition = " + record.partition()
            + ", offset = " + record.offset()
            + ", key = " + record.key()
            + ", value = " + record.value());
}
In this example, we are using the poll() method to retrieve records from the subscribed topics. We then iterate over the records and print out the topic, partition, offset, key, and value for each record.

Handling Errors

KafkaConsumer provides support for handling errors in the form of callbacks. These callbacks are triggered when an error occurs such as when the consumer fails to commit offsets or when the consumer fails to process a message. The following code example shows how to define a callback to handle errors: 

          
           
consumer.setErrorHandler((throwable, data) -> {
    // Handle the error here...
});
In this example, we are setting up an error handler to handle any errors that might occur while consuming messages. The error handler is invoked with the Throwable object and the ConsumerRecord that caused the error.

Conclusion

The KafkaConsumer API is a powerful tool for Java developers to easily integrate their applications with Apache Kafka. It provides features for creating consumers, subscribing to topics, consuming messages, and handling errors. With this API, developers can create robust applications that can process messages from Kafka topics in an efficient and reliable manner.
