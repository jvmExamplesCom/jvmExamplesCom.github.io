io.jsonwebtoken.Jwts

io.jsonwebtoken.JWTs is a class in the Java library designed to provide an easy way to create and validate JWT tokens. These tokens are used to securely store information and identity claims, and can be used to transmit sensitive data over HTTP or JSON. In this article, we will look at the core class of the library and explore some of its features.

Creating JWTs

The simplest way to create a JWT token is to call the static 

Jwts.builder()
method: 

          
           
String jwt = Jwts.builder()
    .setSubject("user")
    .signWith(SignatureAlgorithm.HS256, "secret")
    .compact();

This will create a JWT token for the given subject, signed with the provided secret. It is also possible to set additional claims and headers when creating a token:


          
           
String jwt = Jwts.builder()
    .setSubject("user")
    .setClaim("role", "admin")
    .setHeaderParam("typ", "JWT")
    .signWith(SignatureAlgorithm.HS256, "secret")
    .compact();

Here, an additional claim and header have been added to the token. This can be useful for storing additional information about the user, such as roles or permissions, or for providing additional claims that can be used to verify the integrity of the token.

Validating JWTs

Once a JWT token has been created, it must be validated before it can be used. The simplest way to validate a token is to call the static 

Jwts.parser()
method: 

          
           
Jwts.parser().setSignatureKey("secret").parse(jwt);

This will parse the given JWT token and validate it against the provided secret key. It is also possible to validate the token against a PublicKey or SecretKey instead of a simple String:


          
           
Jwts.parser().setSignatureKey(publicKey).parse(jwt);

Jwts.parser().setSignatureKey(secretKey).parse(jwt);

In addition to signature validation, it is also possible to validate additional claims. For example, to validate the 

role
claim: 

          
           
Jwts.parser().setSignatureKey("secret")
    .requireClaim("role").equals("admin")
    .parse(jwt);
This will validate that the token has the 
role
claim equal to 
admin
before allowing the token to be used.

Conclusion

In this article, we have looked at the io.jsonwebtoken.JWTs class and explored some of its features. We have seen how to use it to create and validate JWT tokens, and how to set additional claims and headers. For more information, see the Javadocs or the library's Github page.
