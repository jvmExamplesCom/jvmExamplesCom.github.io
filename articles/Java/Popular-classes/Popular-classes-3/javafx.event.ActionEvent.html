Introduction to javafx.event.ActionEvent

JavaFX ActionEvent is a class used to handle events that require some type of user interaction such as a mouse click, key press, or button click. ActionEvents are fired by nodes when an action takes place and can be intercepted by a listener, which can then take the appropriate action. The ActionEvent class contains several properties that allow developers to customize their event handling.

Example Code and Output

The following code example shows how to create and use an ActionEvent. It creates a simple button and uses a handler to listen for when it is clicked, and then prints out a message.


          
           
Button btn = new Button("Click me");
btn.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        System.out.println("You clicked me!");
    }
});

The following code example shows how to get the source of the ActionEvent when fired and print it out.


          
           
Button btn = new Button("Click me");
btn.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        Object source = event.getSource();
        System.out.println("Source of the event: " + source);
    }
});

The following code example shows how to consume an event so that it does not propagate further. This is useful if you want to ensure that no other handlers down the chain receive the event.


          
           
Button btn = new Button("Click me");
btn.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        event.consume();
    }
});

Conclusion

javafx.event.ActionEvent provides a way for developers to respond to user interactions in their application. With the various API methods available, developers can customize their event handling to meet their needs.
