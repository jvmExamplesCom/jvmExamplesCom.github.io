Introduction to org.apache.maven.plugins.annotations.Mojo

org.apache.maven.plugins.annotations.Mojo is a Java annotation used to create custom Maven plugins. It allows developers to easily and quickly define a Mojo class, its parameters, required dependencies and other information related to the plugin. This annotation can also be used to generate an entire project structure in one go.

In this article, we will look at the details of how to use the annotation and how it works in the background. We'll also discuss some code examples and provide insights into how to make the most out of it.

What is org.apache.maven.plugins.annotations.Mojo?

The Mojo annotation was introduced with Maven 3 and is part of the Maven API. It provides a convenient way of defining the components needed to make a custom plugin.

It allows you to provide some essential information about your plugin such as:

  • A goal identifier
  • A description of what the goal does
  • A list of required dependencies
  • A list of optional dependencies
  • A list of parameter names and types

The annotation also provides options to generate and/or retrieve various derived objects such as executors, descriptors, and so on.

How to Use org.apache.maven.plugins.annotations.Mojo?

Using the Mojo annotation is fairly straightforward. All you need to do is to add the annotation to your class and then provide the needed details. Here's an example of how to define a simple Mojo:


          
           
@Mojo(name="my-mojo", 
      description="My custom Maven Mojo", 
      requiresDependencyResolution=ResolutionScope.TEST)
public class MyMojo implements Mojo {
   //...
}

In the above example, we define a Mojo class called 

MyMojo
, and provide a name and a description. We also specify that this Mojo requires dependency resolution of 
test
scope.

We can also configure parameters for our Mojo. In the following example, we define a string parameter called 

inputFile
, a boolean parameter called 
verbose
and an integer parameter called 
maxIterations
: 

          
           
@Mojo(name="my-mojo",
      parameters={
        @Parameter(name="inputFile", type=String.class),
        @Parameter(name="verbose", type=Boolean.class, defaultValue="false"),
        @Parameter(name="maxIterations", type=Integer.class, defaultValue="50")
      })
public class MyMojo implements Mojo {
    //...
}
Note that the 
defaultValue
attribute can be used to provide default values for the parameters. These default values will be used when the parameter is not specified in the command line.

The Mojo annotation allows us to specify various Plugin Realization elements too. In the example below, we define three plugin executions that will run when the 

install
phase is executed. The first execution will run if the 
propertyName
property is set to 
true
, the second will run if the property has any value, and the third will always run: 

          
           
@Mojo(name="my-mojo",
      executions={
        Execution(id="exec-1", phase="install", goal="myGoal", properties={ "propertyName"="true" } ),
        Execution(id="exec-2", phase="install", goal="myGoal", properties={ "propertyName"="*" } ),
        Execution(id="exec-3", phase="install", goal="myGoal")
      })
public class MyMojo implements Mojo {
    //...
}

Finally, we can also provide information about required dependencies, as well as optional dependencies, by using the 

dependencies
and 

          
           
optionalDependencies
attributes respectively. For example, the following code defines two required dependencies and one optional dependency: 

          
           
@Mojo(name="my-mojo",
      dependencies={
        Dependency(groupId="org.apache.maven.plugins", artifactId="maven-plugin-api", version="2.0.4"),
        Dependency(groupId="org.apache.maven.plugins", artifactId="maven-compiler-plugin", version="2.0.4")
      },
      optionalDependencies={
        Dependency(groupId="org.apache.maven.plugins", artifactId="maven-surefire-plugin", version="2.0.4")
      })
public class MyMojo implements Mojo {
    //...

Conclusion
