Introduction to ReactContextBaseJavaModule

ReactContextBaseJavaModule is a core class of the React Native bridge system, and it provides a basic set of functionality that allows Java code to interface with JavaScript.

Functionality Provided by ReactContextBaseJavaModule

With ReactContextBaseJavaModule, developers can:
  • register native modules with the bridge
  • expose constants to the JavaScript context
  • invoke callbacks from Java in the JavaScript context
  • get a handle to the bridge instance

Using ReactContextBaseJavaModule

Using ReactContextBaseJavaModule is quite simple. Below is an example of a basic class that uses ReactContextBaseJavaModule.


          
           
public class BasicModule extends ReactContextBaseJavaModule {
  public BasicModule(ReactApplicationContext reactContext) {
    super(reactContext);
  }

  @Override
  public String getName() {
    return "BasicModule";
  }
}

The ReactApplicationContext is a subclass of the Context class, so it contains all the same methods and behavior. When instantiating a ReactContextBaseJavaModule, you should always pass in your ReactApplicationContext.

Registering Native Modules

Once your module is setup, you can register the native modules you want to expose to the JavaScript context.


          
           
@ReactMethod
public void registerModules() {
  mNativeModules.add(new MyNativeModule(getReactApplicationContext()));

In this example, two native modules are registered. The first is MyNativeModule, which takes a ReactApplicationContext as a parameter and the second is AnotherNativeModule which takes ReactApplicationContext and some other parameter.
