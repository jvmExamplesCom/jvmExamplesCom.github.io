Introduction to android.app.ActionBar

The android.app.ActionBar class is a powerful tool that allows developers to create sleek, modern interfaces for their applications. It provides a consistent user interface across all Android devices and is based on the material design guidelines. With it, developers can create action items and navigation tabs, present contextual menus, and more. This article will explain how to use the ActionBar class and illustrate it with example code.

Getting Started

Using the ActionBar class requires an understanding of the Activity class and its lifecycle. This article assumes a basic knowledge of the Activity lifecycle in Android. To get started using the ActionBar class, first create an instance of the class in your Activity class. This can be done in either the onCreate() or onCreateOptionsMenu() methods: 

          
           
ActionBar actionBar = getActionBar();
Once you have obtained an instance of the ActionBar class, you can customize its appearance and behavior.

Setting ActionBar Properties

The ActionBar class provides several methods for customizing the appearance and behavior of the action bar. These include setting the title, display mode, navigation mode, home button, and more. The following example code sets the title and displays an Up button in the action bar: 

          
           
actionBar.setTitle("My App");
actionBar.setDisplayHomeAsUpEnabled(true);
The Up button is used for navigating up one level in the application hierarchy.

Adding Action Items

Action items are buttons that appear in the ActionBar and allow users to perform actions. To add an action item to the ActionBar, create an XML menu resource file. This file contains the list of items that will be displayed in the ActionBar. For example: 

          
           
<menu xmlns:android="http://schemas.android.com/apk/res/android" >

    <item android:id="@+id/action_search"
        android:title="@string/action_search"
        android:icon="@drawable/ic_search"
        android:showAsAction="ifRoom"  />
    
     <item android:id="@+id/action_settings"
        android:title="@string/action_settings"
        android:showAsAction="never"  />

</menu>
The first item specifies that the action should be displayed as an icon if there is room in the ActionBar. The second item specifies that the action should never be displayed in the ActionBar. Once the menu resource is created, it can be loaded in the Activity class. This can be done with the onCreateOptionsMenu() method: 

          
           
@Override
public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.main_menu, menu);
    return true;
}
The menu items can then be handled by the onOptionsItemSelected() method: 

          
           
@Override
public boolean onOptionsItemSelected(MenuItem item) {
    switch (item.getItemId()) {
        case R.id.action_search:
            // Handle action search 
            return true;
        case R.id.action_settings:
            // Handle action settings
            return true;
        default:
            return super.onOptionsItemSelected(item);
    }
}

Adding Tabs

Tabs are used to present items in different categories. To add tabs to the ActionBar, first set the navigation mode to Tab mode: 

          
           
actionBar.setNavigationMode(ActionBar.NAVIGATION_MODE_TABS);
Then, create a TabListener object to handle tab selection and deselection events. This class must implement the OnTabChangeListener interface and the onTabSelected(), onTabUnselected(), and onTabReselected() methods. 

          
           
// Create a TabListener object
MyTabListener tabListener = new MyTabListener();

// Add the TabListener to the ActionBar
actionBar.addTab(actionBar.newTab()
        .setText("Tab 1")
        .setTabListener(tabListener));
actionBar.addTab(actionBar.newTab()
        .setText("Tab 2")
        .setTabListener(tabListener));
Finally, handle tab selection and deselection events in the onTabSelected() and onTabUnselected() methods in the TabListener class: 

          
           
public class MyTabListener implements ActionBar.TabListener {

    // Handle tab selection and deselection events
    @Override
    public void onTabSelected(Tab tab, FragmentTransaction ft) {
        // Handle tab selection
    }

    @Override
    public void onTabUnselected(Tab tab, FragmentTransaction ft) {
        // Handle tab deselection
    }

    @Override
    public void onTabReselected(Tab tab, FragmentTransaction ft) {
        // Handle tab reselection
    }
}

Conclusion

The android.app.ActionBar class is a powerful tool for creating modern apps. This article has explained how to use the ActionBar class to customize the appearance and behavior of the action bar, add action items, and add tabs.
