Introduction to org.xml.sax.ErrorHandler

org.xml.sax.ErrorHandler is an interface used for reporting errors during the process of parsing XML documents. This interface is implemented by applications that use the SAX parser, and when an error is encountered, it will be reported through the implementation of this interface. In this article, we will look at how to use this interface and the methods it contains.

Using ErrorHandler Interface

To use ErrorHandler interface, all we need to do is create a class implementing this interface. Let's demonstrate this by creating a class called MyErrorHandler which implements the ErrorHandler interface: 

          
           
public class MyErrorHandler implements ErrorHandler {
   // Implementation of ErrorHandler interface goes here     
}
Once we have created our custom ErrorHandler class, we can register it with the parser so that when a parsing error occurs, our custom ErrorHandler will be used to handle the error. The following code shows how to do this: 

          
           
SAXParserFactory parserFactory = SAXParserFactory.newInstance();
SAXParser saxParser = parserFactory.newSAXParser();
MyErrorHandler handler = new MyErrorHandler();
saxParser.setErrorHandler(handler);

Error Handler Methods

The ErrorHandler interface contains three methods: 

          
           
warning(SAXParseException e)
error(SAXParseException e)
fatalError(SAXParseException e)
The 

          
           
warning()
method is used to report warnings during parsing, such as not recognizing the document’s character encoding. The 

          
           
error()
method is used to report errors encountered while parsing, such as not being able to find the end tag of an element. Finally, the 

          
           
fatalError()
method is used to report fatal errors encountered while parsing, such as not being able to open the XML document due to it being corrupted. When one of these methods is called, a 

          
           
SAXParseException
object is passed as an argument. This exception contains information about the location of the error in the source document and the error message. For example, the 

          
           
getMessage()
method of the exception can be used to get the error message reported by the parser.

Example Implementation

Now let’s look at an example implementation of the ErrorHandler interface. The following code shows a simple implementation which just prints out the error message: 

          
           
public class MyErrorHandler implements ErrorHandler {

   public void warning(SAXParseException e) {
      System.out.println("Warning: " + e.getMessage());
   }

   public void error(SAXParseException e) {
      System.out.println("Error: " + e.getMessage());
   }

   public void fatalError(SAXParseException e) {
      System.out.println("Fatal Error: " + e.getMessage());
   }
}
This implementation will simply print out the error messages when an error occurs. Of course, in a real-world application you would likely want to take a different action depending on the type of error, such as displaying an error message or writing the error to a log file.
