javax.persistence.FetchType

The 

          
           
javax.persistence.FetchType
enum provides the type of fetching that can be used in the EJB query language. These include eager loading and lazy loading of related entities.

Code Examples

Here are some examples of using 

          
           
javax.persistence.FetchType
: 

          
           
@Entity
public class Employee {
    @ManyToOne(fetch = FetchType.EAGER)
    @JoinColumn(name="department_id")
    private Department department;
}
In this example, the 
Department
entity will be eagerly fetched when the 
Employee
entity is loaded. 

          
           
@Entity
public class Department {
    @ManyToMany(fetch = FetchType.LAZY)
    @JoinTable(name="employees_in_department",
        joinColumns=@JoinColumn(name="department_id"),
        inverseJoinColumns=@JoinColumn(name="employee_id"))
    private List<Employee> employees;
}
In this example, the list of 
Employee
entities will be lazily fetched when the 
Department
entity is loaded.

Output

With eager loading, the related entity is fetched automatically when the main entity is loaded. With lazy loading, the related entities are only fetched when they are accessed. This allows for better performance if the related entities are not needed right away.
