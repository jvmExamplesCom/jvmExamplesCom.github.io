org.testng.annotations.AfterTest

The org.testng.annotations.AfterTest annotation is part of the TestNG testing framework, and is used to indicate a method that should be executed after a test has completed. It can be used to perform any necessary clean up or other operations that must be done after a test has been run. This annotation can be applied to any public void method that takes no parameters.

The following code shows a basic example of using the AfterTest annotation:


          
           
public class MyTestClass
{
    @AfterTest
    public void afterTest()
    {
        // Perform cleanup here
    }
}

In this example, the afterTest() method will be executed after each test in the class has completed. This is useful for performing any operations that must be done after each test, such as cleaning up resources or restoring state that may have been changed during the test.

{
    // Executed only for tests in group1 and group2
}

    // Always executed regardless of whether tests are skipped
