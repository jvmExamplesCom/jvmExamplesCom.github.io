io.netty.handler.codec.http.websocketx.TextWebSocketFrame

io.netty.handler.codec.http.websocketx.TextWebSocketFrame is a subclass of io.netty.handler.codec.websocket.WebSocketFrame which represents a text WebSocket frame. It allows users to send text data over the web using WebSocket connections.

Creating TextWebSocketFrame

Creating a TextWebSocketFrame instance is simple and can be done with the following code snippet:


          
           
TextWebSocketFrame frame = new TextWebSocketFrame("message text");

In this example, we are creating a TextWebSocketFrame with the text “message text”. You can also create a TextWebSocketFrame from a ByteBuf.


          
           
ByteBuf buf = Unpooled.copiedBuffer("message text", StandardCharsets.UTF_8);
TextWebSocketFrame frame = new TextWebSocketFrame(buf);

Sending Data

Sending a TextWebSocketFrame is similar to sending other WebSocket Frames. Here is an example of how you could send a TextWebSocketFrame using the Netty API:


          
           
ctx.channel().writeAndFlush(frame);

This code snippet uses the ctx object which is an instance of the ChannelHandlerContext class. This object is responsible for managing the context of a particular channel. The writeAndFlush() method is used to write data to the channel and then flush it, ensuring that the data is sent.

Reading Data

Reading data from a TextWebSocketFrame is similarly easy and can be done using code like the following:


          
           
TextWebSocketFrame frame = (TextWebSocketFrame) msg;
String messageText = frame.text();

The first line in this code snippet casts the received message into a TextWebSocketFrame. We can then use the text() method to get the text from the frame.
