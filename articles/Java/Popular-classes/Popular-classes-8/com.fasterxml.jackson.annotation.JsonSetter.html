com.fasterxml.jackson.annotation.JsonSetter

The JsonSetter annotation is an annotation introduced in Jackson 2.0 that can be used to mark a method as a setter for deserialization of JSON properties to Java objects. The annotated method should have one parameter, the value of the property being deserialized. The name of the method will be used to map the JSON property name to the method and the parameter type gives the type of the property value.

Code Example 1


          
           
public class Person {
  private String name;
  
  @JsonSetter("name")
  public void setName(String name) {
    this.name = name;
  }
}

In this example, the JsonSetter annotation is used to mark the "setName" method as a setter for the "name" property from the JSON object. This means that when Jackson deserializes the JSON object, it will look for the "name" property and call the "setName" method with the value of the property as the parameter.

Code Example 2


          
           
public class Address {
  private String street;
  private String city;
  
  @JsonSetter("street_address")
  public void setStreet(String street) {
    this.street = street;
  }
  
  @JsonSetter("city")
  public void setCity(String city) {
    this.city = city;
  }
}

In this example, two setters are annotated with the JsonSetter annotation. The first one is marked with "street_address" which means Jackson will try to find a property with that name in the JSON object and pass its value to this setter. The second one is marked with "city" so Jackson will try to find a property with that name in the JSON object and pass its value to this setter.

Code Example 3


          
           
public class Person {
  private int age;
  
  @JsonSetter(value="age", nulls=Nulls.SKIP)
  public void setAge(int age) {
    this.age = age;
  }
}

This example is similar to the first one, but with one additional parameter. The "nulls" parameter is used to specify how Jackson should handle null values for this property. In this case, if the "age" property in the JSON object is null, Jackson will not call the setter and the "age" field in the Java object will not be set.
