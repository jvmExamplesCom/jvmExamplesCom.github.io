org.xml.sax.SAXNotSupportedException

The org.xml.sax.SAXNotSupportedException is an exception that is thrown when a feature or property is not supported by the SAX implementation. This typically happens when the requested feature or property is not supported by any of the underlying XML processors involved in the parsing. The SAXNotSupportedException is a subclass of SAXException, which is the parent class for all types of SAX-related exceptions.

Example 1: Parse XML Document With Unsupported Feature

In this example we'll try to parse an XML document that uses an unsupported feature. This will result in an org.xml.sax.SAXNotSupportedException being thrown.


          
           
import java.io.File;

import javax.xml.parsers.SAXParser;
import javax.xml.parsers.SAXParserFactory;

import org.xml.sax.InputSource;

public class SaxNotSupportedExceptionExample {

        SAXParserFactory factory = SAXParserFactory.newInstance();

        SAXParser parser = factory.newSAXParser();

        InputSource inputSource = new InputSource(xmlFile.toURI().toString());

    }

}

In order to avoid the org.xml.sax.SAXNotSupportedException, we can use an implementation of the XmlReader interface instead of the SAXParser. The XmlReader is more flexible than the SAXParser and supports more features. import javax.xml.parsers.SAXParserFactory;

public class XmlReaderExample {

    public static void main(String[] args) throws Exception {
        XMLInputFactory xmlInputFactory = XMLInputFactory.newInstance();

}
