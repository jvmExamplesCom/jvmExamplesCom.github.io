org.springframework.web.reactive.function.client.WebClient

Spring WebFlux provides a reactive and fully asynchronous non-blocking web framework for the server side with support for the WebClient – a component in the spring-webflux module that helps to make requests and process response from a server. It is built using the reactive programming model – meaning that it can handle large number of concurrent connections and scale better than the traditional blocking approach. The WebClient is the Spring equivalent of the RestTemplate with the same purpose – to make remote calls transparently and synchronously.

Creating a WebClient

Creating a new WebClient is fairly simple and straightforward. All you need to do is use the static factory methods provided by the WebClient class. For example, to create a new WebClient instance:


          
           
WebClient webClient = WebClient.create();

This will create an instance of a WebClient that can be used to send requests to a server. We can further customize the instance by passing in additional parameters to the create() method call:


          
           
WebClient webClient = WebClient.create("http://example.com");

This will create a new WebClient that is configured to send requests to the given URL. This can be used for convenience if all requests should be sent to the same host. Another set of options is to configure a custom ClientHttpRequestFactory. This can be done with the following code:


          
           
ClientHttpRequestFactory requestFactory = ...
WebClient webClient = WebClient.create(requestFactory);

This will create a new WebClient instance that is configured with the given request factory. This can be used to customize various aspects of the underlying HTTP connection such as authentication, connection pooling, etc.

Making Requests with the WebClient

Once the WebClient instance is created, the next step is to make requests. This can be done with the WebClient’s exchange() method. This method accepts a RequestEntity which holds all the necessary information about the request such as the HTTP method, URL, headers, body, etc. For example, to make a GET request to a resource:


          
           
RequestEntity<Void> request = RequestEntity.get(URI.create("http://example.com/resource"))
    .accept(MediaType.APPLICATION_JSON)
    .build();

ResponseEntity<String> response = webClient.exchange(request, String.class);

System.out.println(response.getBody());

This will send a GET request to the given URL and print out the response body. We can also make POST requests with the exchange() method. For example, to make a POST request to the same URL with some data:


          
           
RequestEntity<String> request = RequestEntity.post(URI.create("http://example.com/resource"))
    .contentType(MediaType.APPLICATION_JSON)
    .body("{\"foo\": \"bar\"}");

ResponseEntity<String> response = webClient.exchange(request, String.class);

System.out.println(response.getBody());

This will send a POST request with the given body to the given URL and print out the response body. We can further customize the request by setting custom HTTP headers, cookies, etc.

The WebClient is a powerful and fully asynchronous non-blocking HTTP client that is ideal for high performance web applications. With support for all popular HTTP methods, custom headers, cookies, and more, it’s the perfect tool for building modern web applications.
