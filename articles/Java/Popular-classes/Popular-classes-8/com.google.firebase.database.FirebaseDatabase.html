Introduction to Firebase Database

Firebase is a real-time mobile development platform developed by Google. It helps mobile and web developers quickly build high quality applications without having to manage servers or write server-side code. Firebase includes many features, namely Firebase Real-Time Database (RTD) and Firebase Authentication. In this article, we'll take a deep-dive into Firebase Database, exploring the different capabilities and use-cases for the RTD in modern application development.

Overview of Firebase Database

The Firebase Database is an NoSQL cloud-hosted database service that stores and sync data among users in real-time. It's a great choice when it comes to developing applications with high scalability, real-time updates, and cost efficient hosting. Data is stored in JSON format and is accessible from any internet-connected device. Firebase provides SDKs for Android, iOS, JavaScript, and Unity, and there are many third-party libraries available for other languages.

Getting Started with Firebase Database

Before getting started, you'll need to create a Firebase Project by signing up for Firebase and creating a project on the Firebase console. Once your project is created, you can add Firebase to your app by following the instructions provided. To start working with the Firebase Database, add the Firebase Database library to your project.

Populating the Firebase Database

Once you have the Firebase Database library added to your project, you can populate it with data. You can insert data using the 

          
           
setValue()
method and use the 

          
           
push()
method to assign a unique key to each piece of data. For example, if you wanted to store a list of books in the database, you could do so by using the following code: 

          
           
DatabaseReference ref = FirebaseDatabase.getInstance().getReference("books");
 
Book book1 = new Book("Harry Potter", "J.K. Rowling");
String book1Id = ref.push().getKey();
ref.child(book1Id).setValue(book1);
 
Book book2 = new Book("The Hobbit", "J.R.R. Tolkien");
String book2Id = ref.push().getKey();
ref.child(book2Id).setValue(book2);
The code above will create two entries in the "books" node of the database with the book titles and authors.

Retrieving Data from Firebase Database

You can retrieve data from the Firebase Database by using the 

          
           
addValueEventListener()
method. This method takes a ValueEventListener object as a parameter which is then called when the data is changed in the database. For example, if you wanted to retrieve all of the books from the database, you could do so by using the following code: 

          
           
DatabaseReference ref = FirebaseDatabase.getInstance().getReference("books");
 
ref.addValueEventListener(new ValueEventListener() {
    @Override
    public void onDataChange(DataSnapshot dataSnapshot) {
        for(DataSnapshot ds : dataSnapshot.getChildren()) {
            Book book = ds.getValue(Book.class);
            Log.d("TAG", "Book title: " + book.getTitle());
            Log.d("TAG", "Book author: " + book.getAuthor());
        }
    }
 
    @Override
    public void onCancelled(DatabaseError databaseError) {
        Log.e("TAG", "Failed to read value.", databaseError.toException());
    }
});
The code above will loop through each book in the "books" node and log out the title and authors of those books.

Updating Data in Firebase Database

If you need to update existing data in the Firebase Database, you can use the 

          
           
updateChildren()
method. The 

          
           
updateChildren()
method takes a Map of key/value pairs which can be used to update data in the database. For example, if we wanted to update the author of the first book in the list, we could do so with the following code: 

          
           
DatabaseReference ref = FirebaseDatabase.getInstance().getReference("books");
 
Map<String, Object> updates = new HashMap<>();
updates.put("author", "J.K. Rowling");
ref.child("book1Id").updateChildren(updates);
The code above will update the "author" field of the book identified by "book1Id" to "J.K. Rowling".

Deleting Data from Firebase Database

Finally, you can delete data from the Firebase Database by using the 

          
           
removeValue()
method. This method takes no parameters and will delete the data at the reference that the method is called on. For example, if we wanted to delete the first book in the list, we could do so with the following code: 

          
           
DatabaseReference ref = FirebaseDatabase.getInstance().getReference("books");
 
ref.child("book1Id").removeValue();
The code above will delete the data located at the "book1Id" reference.

Conclusion

In this article, we took a deep dive into Firebase Database, looking at the different capabilities and use-cases for the RTD in modern application development. We discussed how to get started with Firebase Database, how to populate the database with data, how to retrieve data from the database, how to update existing data, and how to delete data from the database. With Firebase Database, it's easier than ever to develop scalable, real-time applications.
