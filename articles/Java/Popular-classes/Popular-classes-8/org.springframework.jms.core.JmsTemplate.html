Introduction to org.springframework.jms.core.JmsTemplate

The 

          
           
JmsTemplate
class is the central class for interacting with middleware-managed messaging services such as Java Message Service (JMS). Located in the 

          
           
org.springframework.jms.core
package, it provides a simple way to send and receive messages, declaratively managing the JMS infrastructure objects needed, such as connection factory, destination, and message producer and consumer. The 

          
           
JmsTemplate
is designed to help you easily create applications that use JMS asynchronously or synchronously. By using the 

          
           
JmsTemplate
, you enable your applications to quickly perform common JMS operations without having to explicitly manage the underlying objects.

Using JmsTemplate

Using the 

          
           
JmsTemplate
requires the following steps:
  1. Create the JmsTemplate object, and inject the ConnectionFactory, Destination, and any other optional configuration into it.
  2. Use the appropriate method of the JmsTemplate class to execute one of the send, receive, convertAndSend, or receiveAndConvert methods.
  3. Optionally configure message headers and properties as needed by the send or publish methods.

Example Code

JmsTemplate
to send a message: 

          
           
JmsTemplate jmsTemplate = new JmsTemplate();
jmsTemplate.setConnectionFactory(connectionFactory);
jmsTemplate.setDefaultDestinationName("queue.example");

jmsTemplate.convertAndSend("Hello World!");

JmsTemplate jmsTemplate = new JmsTemplate();
