Java: hudson.model.Run

The hudson.model.Run class is a core Java class for running Hudson builds. It handles the task of running a build and reporting back results to the user. The Run class is part of the Hudson job scheduling plugin, which provides an easy-to-use way for developers and build engineers to schedule and execute builds.

Run Class Overview

The Run class provides several methods for running a build. The most basic of these is the start() method, which kicks off the build process. It then calls the build() method to execute the build, and finally the postBuild() method to report the results. The Run class also provides methods for getting various pieces of information about the build. For example, the getStatus() method can be used to retrieve the status of a build, and the getLog() method can be used to get the console log of the build. The Run class also provides methods for accessing build data such as build parameters, artifacts, and workspace files. These methods can be used to get the values of various settings in the build, or to retrieve files generated during the build process.

Example Code

To demonstrate how the Run class works, we’ll create a simple example that simply prints out the build status, console log, and workspace files. 

          
           
public class Example {
    public static void main(String[] args) {
        Hudson hudson = new Hudson();

        Run run = hudson.getBuild("example");

        // print the build status
        System.out.println("Build Status: " + run.getStatus());

        // print the console log
        System.out.println(run.getLog());

        // print the workspace files
        for (File file : run.getWorkspaceFiles()) {
            System.out.println(file);
        }
    }
}

Then we call the getBuild() method with our example build name, and get a reference to the Run instance.
