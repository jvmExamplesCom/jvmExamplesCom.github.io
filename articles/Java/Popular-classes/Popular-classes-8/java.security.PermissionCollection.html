java.security.PermissionCollection

The java.security.PermissionCollection class is used to represent a collection of Permission objects. It provides an access control architecture for enforcing security policies for Java code running within the Java environment.

A PermissionCollection is an object that contains a collection of Permission objects. The Collection can be used to optimize the aggregate Permission checks made against SecurityManager and AccessControlContexts. A PermissionCollection should be used when a collection of homogenous Permissions need to be stored and accessed. For example, an application might want to store its entire set of permissions in a single PermissionCollection.

The PermissionCollection implements the Serializable interface so that it can be stored as part of a serialized stream. It also implements the Enumeration interface so that it can be enumerated. By implementing Enumeration, a PermissionCollection can be used as a standard input to other collection types.

Methods

The following methods are available on a PermissionCollection:

  • add(Permission permission) - Adds the specified Permission object to the Collection, if it is not already present.
  • implies(Permission permission) - Returns true if the specified Permission object is implied by any permission in the collection.
  • elements() - Returns an enumeration of all the Permission objects in the Collection.
  • isReadOnly() - Returns true if the Collection is marked as read-only.
  • setReadOnly() - Marks the Collection as read-only.
