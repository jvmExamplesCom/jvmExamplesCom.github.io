org.eclipse.ui.IWorkbenchPage

The org.eclipse.ui.IWorkbenchPage interface provides access to individual workbench pages. A workbench page contains a hierarchy of views and an optional editor area for editing files. Each page also has its own set of actions to enable basic navigation and manipulation of the workbench components within the page.

Overview

This interface is not intended to be implemented or extended by clients. An instance of this is obtained via the 

getActivePage()
method from the workbench.

Basic Operations

The most common operations on the interface are:
  • Creating and managing views
  • Creating and managing editors
  • Navigating between view and editor areas
  • Accessing contributions such as action sets, perspectives and more

Creating and Managing Views

To create a new view, you can use the 

          
           
showView(String viewId)
method. The viewId is the id of the view extension as specified in the corresponding plugin.xml file. This method returns an 
IViewPart
instance representing the newly created view. You can also open an existing view with the 

          
           
findView(String viewId)
method, which returns an 
IViewPart
instance if the view already exists, or 
null
if it does not exist.

Both of these methods support the concept of multiple instances of a view. In this case, you must pass an additional parameter of type 

String secondaryId
to each method. This id specifies the unique identifier for the particular view instance. It can be used later to find the view with the 

          
           
findView(String viewId, String secondaryId)
method. You can also close a view with the 

          
           
hideView(IViewPart part)
method. This method takes the 
IViewPart
instance of the view to be closed as a parameter.

Creating and Managing Editors

The 

          
           
openEditor(IEditorInput input, String editorId, boolean activate)
method is used to create and open an editor. The 
input
parameter specifies the editor's input (e.g. a file or a document), and the 
editorId
specifies the editor extension as specified in the corresponding plugin.xml file. The 
activate
parameter specifies whether the editor should be activated or not. If this parameter is set to 
true
, then the editor will be given focus. This method returns an 
IEditorPart
instance representing the newly opened editor. You can also open an existing editor using the 

          
           
findEditor(IEditorInput input)
method, which returns an 
IEditorPart
instance if the editor already exists, or 
null
if it does not exist. You can close an editor using the 

          
           
closeEditor(IEditorPart editor, boolean save)
method. The 
editor
parameter is the 
IEditorPart
instance of the editor being closed, and the 
save
parameter specifies whether the changes in the editor should be saved or not.

Navigating Between View and Editor Areas

The 

          
           
activate(IWorkbenchPart part)
method is used to activate either a view or an editor. This method takes an 
IWorkbenchPart
instance as a parameter and gives it focus. The 
getActiveEditor()
and 
getActiveView()
methods are used to return the active editor and view, respectively. These methods return an 
IEditorPart
and 
IViewPart
instance, respectively. The 
getEditors()
and 
getViews()
methods are used to return all of the editors and views in the page. These methods return arrays of 
IEditorPart
and 
IViewPart
instances, respectively.

Accessing Contributions

The 

          
           
getActiveActionBars()
method is used to get an array of active action bars in the page. Action bars provide access to contributions such as menus, toolbars and more. The 
getPerspective()
method is used to get the active perspective in the page. Perspectives are used to define the layout and initial content of a workbench page. Each perspective can contain its own set of views, editors and actions. Finally, the 
getActionSets()
method is used to get an array of active action sets in the page. Action sets are used to contribute toolbar and menu actions to a workbench page.

Example Usage


          
           
IWorkbenchPage page = PlatformUI.getWorkbench().getActiveWorkbenchWindow().getActivePage();

// Create a new view
IViewPart view = page.showView("org.eclipse.sample.view");

// Find an existing view
IViewPart existingView = page.findView("org.eclipse.sample.view");

// Close a view
page.hideView(view);

// Open an editor
IEditorPart editor = page.openEditor(new FileEditorInput("myfile.txt"), "org.eclipse.sample.editor", true);

// Find an existing editor
IEditorPart existingEditor = page.findEditor(new FileEditorInput("myfile.txt"));

// Close an editor
page.closeEditor(editor, true);

// Get active editor and view
IEditorPart activeEditor = page.getActiveEditor();
IViewPart activeView = page.getActiveView();

// Get all editors and views
IEditorPart[] editors = page.getEditors();
IViewPart[] views = page.getViews();

// Activate a view or editor
page.activate(activeEditor);
page.activate(activeView);

// Get active action sets
IActionSetDescriptor[] actionSets = page.getActionSets();

// Get active perspective
IPerspectiveDescriptor perspective = page.getPerspective();

// Get active action bars
IActionBarDescriptor[] actionBars = page.getActiveActionBars();
