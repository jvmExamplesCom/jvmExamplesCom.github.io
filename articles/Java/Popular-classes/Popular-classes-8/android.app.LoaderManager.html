Android.app.LoaderManager

Android.app.LoaderManager is an abstract class that provides an interface for managing a Loader. The LoaderManager handles communication with the Loader and ensures that the Loader is properly initialized, started, paused and stopped.

Underlying Principles

It is important to understand the underlying principles of the LoaderManager. The LoaderManager is responsible for managing the Loader and coordinating the communication between the Loader and the application. It also helps to ensure that the Loader is always in the correct state.

The LoaderManager has a lifecycle that is driven by the application's lifecycle. When the application is resumed, the LoaderManager must ensure that the Loaders are resumed. Similarly, when the application is paused, it must ensure that the Loaders are also paused.

Methods

The LoaderManager provides a number of methods for managing Loaders. The following methods are the most commonly used: 

  • initLoader()
    : This is used to initialize a new Loader.
  • restartLoader()
    : This is used to restart an existing Loader. 
  • destroyLoader()
    : This is used to destroy an existing Loader. 
  • getLoader()
    : This is used to get an existing Loader.

Examples


          
           
// Initialize a Loader
getSupportLoaderManager().initLoader(LOADER_ID, null, this);

// Restart a Loader
getSupportLoaderManager().restartLoader(LOADER_ID, null, this);

// Destroy a Loader
getSupportLoaderManager().destroyLoader(LOADER_ID);

// Get a Loader
