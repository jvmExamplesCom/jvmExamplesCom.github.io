HTTP Multipart Mode in Java

Introduction HttpMultipartMode is an interface in the org.apache.http.entity.mime package of Apache HttpClient library. It is used to determine the behavior and structure of a multipart request body. In this article, we will discuss how to use HttpMultipartMode in Java.

Overview of HttpMultipartMode

HttpMultipartMode is an Enum that represents the different modes available for creating multipart requests. The available modes are STRICT, BROWSER_COMPATIBLE and RFC7578. 

          
           
public enum HttpMultipartMode {
	STRICT,
	BROWSER_COMPATIBLE,
	RFC7578;
}
The different modes affect the way the request body is structured, and ultimately determines how the server should interpret it.

Using HttpMultipartMode

Using HttpMultipartMode is easy. All that needs to be done is to create an instance of the MultipartEntityBuilder class, set the mode, and build the entity. 

          
           
MultipartEntityBuilder builder = MultipartEntityBuilder.create();
builder.setMode(HttpMultipartMode.STRICT);
HttpEntity entity = builder.build();
The entity can then be passed to an HttpPost request as the request body. 

          
           
HttpPost post = new HttpPost("http://example.com/upload");
post.setEntity(entity);

HttpMultipartMode Use Cases

When creating a multipart request, it is important to select the correct mode. The mode selected depends on the server that is receiving the request, and the data being sent.

STRICT Mode

STRICT mode is the most widely used mode. It is recommended for requests submitting non-form data, such as images or files. 

          
           
MultipartEntityBuilder builder = MultipartEntityBuilder.create();
builder.setMode(HttpMultipartMode.STRICT);
HttpEntity entity = builder.build();

BROWSER_COMPATIBLE Mode

BROWSER_COMPATIBLE mode is used for requests submitting form data. It is used for compatibility with web browsers. 

          
           
MultipartEntityBuilder builder = MultipartEntityBuilder.create();
builder.setMode(HttpMultipartMode.BROWSER_COMPATIBLE);
HttpEntity entity = builder.build();

RFC7578 Mode

RFC7578 mode is used for compatibility with OData protocol. 

          
           
MultipartEntityBuilder builder = MultipartEntityBuilder.create();
builder.setMode(HttpMultipartMode.RFC7578);
HttpEntity entity = builder.build();
