Java com.google.inject.Scopes

In Java, the com.google.inject.Scopes class provides support for defining scopes for objects in a Guice-based application. Scopes define the lifetime of objects, allowing developers to control how often Guice creates new instances of objects and when it reuses existing instances.

Types of Scopes

The com.google.inject.Scopes class offers several types of scopes:

  • Singleton (default): Objects are created only once and reused whenever they are required by the application. This is the default scope.
  • NoScope: A NoScope instance is created every time an object is requested.
  • Prototype: Objects are created every time they are requested.

How to Use com.google.inject.Scopes

The com.google.inject.Scopes class provides several static methods that allow developers to bind objects to a specific scope:

  • bindScoped(): Binds the specified object to a given scope.
  • scope(): Returns the Scope for the specified type.
  • inScope(): Executes the specified operation within the specified scope.

The following code example shows how to use the bindScoped() method to bind an object to the Singleton scope:


          
           
bindScoped(MyObject.class).to(MyObjectImpl.class).in(Singleton.class);

Once an object has been bound to a particular scope, it can be retrieved by calling the injector.getInstance() method:


          
           
MyObject myObject = injector.getInstance(MyObject.class);

The injected object will be an instance of the specified type, and it will be bound to the specified scope.
