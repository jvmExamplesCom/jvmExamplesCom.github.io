What is java.lang.Override?

Java Override is an annotation type used to indicate that a subclass is overriding a method of one of its superclass. When a method is marked with @Override annotation, the compiler checks whether the method actually exists in its superclass or not. Using this annotation makes the code more readable and maintainable.

How do we use java.lang.Override?

To use it correctly we have to use the annotation before the method declaration within a class that is extending another class. We should also note that the method signature has to be exactly the same as the method declared in the superclass in order for the @Override annotation to take effect. Let us consider this pseudo code example below as an illustration: 

          
           
public class SpecialCalculator extends Calculator {

    @Override
    public int doCalculation(int number1, int number2) {
        //method body
    }

}
In this example, we are overriding the method doCalculation() of the superclass Calculator which means that if our overridden method in SpecialCalculator class returns something different than what the superclass Calculator method returns, it will take precedence over the original method.

Advantages of using java.lang.Override

One of the advantages of using @Override annotation is that it warns us about bugs in the code. If the method signature does not match the method in the superclass then it throws a compile time error. This helps developers to avoid potential errors caused by incorrect overrides. Another advantage of using @Override annotation is that it makes the code easier to read and code maintenance becomes easier. It gives clear indication to the readers that this method is overriding the method from the superclass.

Disadvantages of using java.lang.Override

The biggest drawback of using @Override annotation is that it is checked only at the compile time. That means if the method signature matches with the method in superclass, but the implementation is wrong then it will still not throw an error at compile time. Also, it can lead to confusion in situations where multiple classes with same method names exist. In such cases, we cannot override a method which comes from a third party library. This might result in unexpected results while executing the code.

Conclusion

Java.lang.Override is an annotation type used to indicate that a subclass is overriding a method of its superclass. It makes the code more readable and maintainable, but it must be used carefully as it can lead to potential errors.
