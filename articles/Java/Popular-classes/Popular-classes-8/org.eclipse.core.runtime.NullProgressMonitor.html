org.eclipse.core.runtime.NullProgressMonitor

The org.eclipse.core.runtime.NullProgressMonitor class is a utility class in the Eclipse platform which can be used to indicate that no progress reporting is required. This class can be used for operations that are not expected to take a long time and therefore do not require a user-visible progress indication.

Methods

NullProgressMonitor includes two methods:
  • beginTask(): This method is used to begin a task. It doesn't have any parameters and returns void.
  • done(): This is used to indicate that the task is complete and no further work will be happening. It also doesn't have any parameters and returns void.

Code Examples

Here is an example of using NullProgressMonitor to begin and end a task:


          
           
NullProgressMonitor nullProgressMonitor = new NullProgressMonitor()

nullProgressMonitor.beginTask();

// Some operation

nullProgressMonitor.done();

Here is an example of using NullProgressMonitor to track a simple task:


          
           
NullProgressMonitor nullProgressMonitor = new NullProgressMonitor()

nullProgressMonitor.beginTask("Task Name", 10);

for(int i = 0; i < 10; i++){
    // Do work
    nullProgressMonitor.worked(1);
}

nullProgressMonitor.done();
