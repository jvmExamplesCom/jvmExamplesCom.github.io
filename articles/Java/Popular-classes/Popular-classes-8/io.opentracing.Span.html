io.opentracing.Span

io.opentracing.Span is a Java API, part of the OpenTracing project, which provides an abstraction layer for distributed tracing. It allows developers to instrument their code and trace application activity across a distributed environment. The API supports a wide range of features, such as distributed context propagation, local root span creation, and span API chaining.

Creating Span with io.opentracing.Tracer

A Span instance can be created using the 

          
           
io.opentracing.Tracer
API by providing the details regarding the span. Below is a sample snippetCode for creating a span: 

          
           
Tracer tracer = Tracer.getInstance("My-Tracer");
Map<String, String> tags = Collections.singletonMap("host", "localhost");
Span span = tracer.buildSpan("span-name").withTag("span-type", "operation")
                                       .withStartTimestamp(System.currentTimeMillis())
                                       .withTags(tags)
                                       .start();

Adding Logs to Span

Logs are used to document a span’s progress and add additional context. Logs can be added to a span using the 

log()
method. The below example shows can demonstrate how to log a key/value pair: 

          
           
span.log("event", "test-event");
span.log("userId", 12345);

Closing a Span

When a span’s operation is complete, it should be closed using the 

finish()
method. This will update the span with end time information and send the span data to the tracing system. The below example demonstrates how to use the 
finish()
method: 

          
           
span.finish();

Other Span API Methods

The 
Span
API also provides various other methods that allow developers to further control the behavior of the span. These include methods to set tag values, set operation name, set start and finish timestamps, get the trace id, etc. Some of these methods are listed below: 

          
           
setTag(): sets the specified tag with given value.
setOperationName(): sets the name of the operation associated with the span.
getTraceId(): returns the trace id of current span.
setStartTimestamp(): sets the start timestamp of the span.
setFinishTimestamp(): sets the finish timestamp of the span.

Related Classes

In addition to the 
Span
API, the OpenTracing project also provides two related helper classes — 
Scope
and 
SpanContext
. The Scope class is used to manage the life cycle of a span while the 
SpanContext
class is used to propagate span context across distributed systems. For more information on these classes, please consult the official OpenTracing documentation.
