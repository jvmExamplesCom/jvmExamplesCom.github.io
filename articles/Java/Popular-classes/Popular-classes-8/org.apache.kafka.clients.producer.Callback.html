Understanding org.apache.kafka.clients.producer.Callback

org.apache.kafka.clients.producer.Callback is an Apache Kafka interface that helps an application to be notified of the successful or failed delivery of a record. It is an asynchronous API and uses callbacks instead of blocking the caller with a synchronous API. This allows an application to continue while the sent message is being processed. The Callback interface can be used to perform various tasks during the completion of specific operations.

Creating a Callback Function

In order to create a callback function, the class will have to implement the Callback interface. Below is a code example demonstrating this:


          
           
public class MyCallback implements Callback {
  public void onCompletion(RecordMetadata metadata, Exception exception){
     if(exception == null) {
        // Message was sent successfully
     } else {
        // Message was not sent
     }
  }
}

Once this is done, the application can pass this class when sending records over the network. This enables it to be notified after the record is sent.

Using the Callback Function

The following is an example on how to use the callback function that was created earlier:


          
           
ProducerRecord<String, String> record = new ProducerRecord<>("topic", "key", "value");
MyCallback callback = new MyCallback();

//Send record using the callback
Producer.send(record, callback);

By doing this, the application will be notified when the record is sent successfully or an exception is thrown. The record metadata such as the partition and the offset can also be obtained by the callback.

Conclusion

The Callback interface allows an application to use an asynchronous API for delivering Kafka messages. This allows the application to continue to respond to requests while the messages are being sent. Applications can also be notified when the sent messages are delivered successfully or if an exception is thrown. Using the Apache Kafka Callback interface is an effective way for applications to build reliable messaging systems.
