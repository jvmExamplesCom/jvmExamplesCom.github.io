androidx.constraintlayout.widget.ConstraintLayout

ConstraintLayout is a powerful and flexible Android layout library, which is used from Android 2.3 and higher versions. It allows developers to create complex layouts with flat views hierarchy and can improve the performance of their applications. ConstraintLayout allows developers to easily implement the Android Design Support Library.

Features:

  • Flexible layout and widgets
  • Constrained view hierarchies
  • Optimal performance
  • Android Design Support Library integration

How to Use ConstraintLayout

The first step in creating a ConstraintLayout is to add the line 

          
           
implementation 'androidx.constraintlayout:constraintlayout:2.1.0'
to the Gradle files. Then, use the 'File -> New -> Layout Resource File' option to create a new layout file. After that, the layout should be opened in the Android Studio layout editor. Now, the designer can start adding views to the ConstraintLayout. An example of a simple layout with two TextViews is shown below: 

          
           
<androidx.constraintlayout.widget.ConstraintLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent">

    <TextView
        android:id="@+id/textView1"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="TextView 1"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

    <TextView
        android:id="@+id/textView2"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="TextView 2"
        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>
This example shows a basic usage of ConstraintLayout. The first TextView is constrained on its start edge to the start edge of the parent, and the second one is constrained on its end edge to the end edge of the parent. The two views are also constrained on their top edge to the top edge of the parent, but any other constraint can be specified as well.
