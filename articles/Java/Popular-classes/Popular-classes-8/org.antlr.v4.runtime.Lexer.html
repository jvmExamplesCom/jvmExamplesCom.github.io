org.antlr.v4.runtime.Lexer

Overview

org.antlr.v4.runtime.Lexer is an abstract class providing a default implementation of the org.antlr.v4.runtime.Lexer inteface. It is the primary class for implementing lexers using ANTLR 4. It is used to create tokens (symbolic names) while parsing the input text. It provides various methods to manipulate the tokens and their values.

Example Code

In this example, we will create a lexer for a simple expression language that consists of identifiers, operators, and numbers: 

          
           
import org.antlr.v4.runtime.*;

public class SimpleLexer extends Lexer {  

    public SimpleLexer(CharStream input) {
        super(input);
    }

    @Override
    public Token nextToken() {
        Token token = super.nextToken();

        if (token.getType() == Token.EOF) {
            return token;
        }

        // Identifier 
        if (Character.isAlphabetic(token.getText().charAt(0))) {
            return new CommonToken(SimpleLexer.ID, token.getText());
        }

        // Operator 
        if (token.getText().equals("+") || token.getText().equals("-") 
            || token.getText().equals("*") || token.getText().equals("/")) {
            return new CommonToken(SimpleLexer.OP, token.getText());
        }

        // Number 
        if (Character.isDigit(token.getText().charAt(0))) {
            return new CommonToken(SimpleLexer.NUM, token.getText());
        }

        return token;
    }

    public static final int 
        ID = 1, 
        OP = 2, 
        NUM = 3;
}
