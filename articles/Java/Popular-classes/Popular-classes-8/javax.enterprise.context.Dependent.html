javax.enterprise.context.Dependent

The javax.enterprise.context.Dependent annotation is a component of the Java EE platform and is used to indicate that an annotated class has a dependent scope.

A dependent scoped bean is one that is bound to the lifecycle of its owning bean. When the owning bean is destroyed, so is the dependent scoped bean.

The Dependent annotation can be applied to a field, constructor parameter or method in a class. It can also be applied to a bean class itself.

Example

In the following example we have a class called ShoppingCartBean which has a dependency on the CartItem class.


          
           
@Dependent 
public class ShoppingCartBean { 

    private CartItem item; 

    @Inject 
    public ShoppingCartBean(CartItem item) { 
        this.item = item; 
    } 

    public void doSomething() { 
        ... 
    } 
}

Here the Dependent annotation is applied to the ShoppingCartBean class, indicating that it has a dependent scope, meaning that its lifecycle is bound to the lifecycle of its owning bean.

Output

When code using the Dependent annotation is executed, the output will be a bean in the application context with a dependent scope.


          
           
Bean: ShoppingCartBean scope: dependent
