javax.persistence.NamedQueries

A 
NamedQuery
is a predefined query stored in the mapping metadata. It can be used to simplify lookups of persistent objects, making them more efficient and easier to use. javax.persistence.NamedQueries provides a fluent API for constructing named queries.

Example 1 - Simple Named Query

We can create a simple named query like this: 

          
           
@NamedQuery(
   name = "findAllEmployees",
   query = "select e from Employee e"
)
public class Employee { ... }

In the above code, we have defined a named query named 

findAllEmployees
that selects all 
Employee
objects from the database. We can then use the named query like this: 

          
           
TypedQuery<Employee> query = em.createNamedQuery("findAllEmployees", Employee.class);
List<Employee> employees = query.getResultList();

The above code will execute the named query and return a list of 

Employee
objects.

Example 2 - Named Query With Parameters

We can also define a named query with parameters: 

          
           
@NamedQuery(
   name = "findEmployeesByName",
   query = "select e from Employee e where e.name = :name"
)
public class Employee { ... }

In the above code, we have defined a named query named 

findEmployeesByName
that selects all 
Employee
objects from the database whose 
name
attribute matches the given parameter. We can then use the named query like this: 

          
           
TypedQuery<Employee> query = em.createNamedQuery("findEmployeesByName", Employee.class);
query.setParameter("name", "John");
List<Employee> employees = query.getResultList();

The above code will execute the named query and return a list of 

Employee
objects whose 
name
attribute is 
John
.

Example 3 - Named Native Query

We can also define named native queries, which are queries written in the native SQL dialect of the underlying database:


          
           
@NamedNativeQuery(
   name = "findEmployeesByCountry",
   query = "select * from employee e where e.country = :country",
   resultClass = Employee.class
)
public class Employee { ... }

In the above code, we have defined a named query named 


          
           
findEmployeesByCountry
. This query is written in the native SQL dialect of the underlying database, and it selects all 
Employee
objects from the database whose 
country
attribute matches the given parameter. We can then use the named query like this: 

          
           
TypedQuery<Employee> query = em.createNamedQuery("findEmployeesByCountry", Employee.class);
query.setParameter("country", "USA");
List<Employee> employees = query.getResultList();

The above code will execute the named query and return a list of 

Employee
objects whose 
country
attribute is 
USA
.

Conclusion

javax.persistence.NamedQueries provides a convenient way to define and execute named queries. It simplifies lookups of persistent objects, making them more efficient and easier to use.
