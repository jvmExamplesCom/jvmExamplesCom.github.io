Introduction to io.undertow.Undertow

io.undertow.Undertow is a lightweight web server and a Java servlet container. It's designed to be fast, flexible, secure, and easy to use. Undertow can be used for a wide variety of tasks ranging from traditional web applications to RESTful web services, microservices, and much more. It also provides support for many programming languages and technologies, such as Java EE, JAX-RS, JAX-WS, and many others.

Setting Up an Undertow Server

Setting up an Undertow server is relatively straightforward. The first step is to add the Undertow dependency to your project. This can be done in Maven by adding the following snippet to your pom.xml file:


          
           
<dependency>
    <groupId>org.wildfly.undertow</groupId>
    <artifactId>undertow-servlet</artifactId>
    <version>2.0.14.Final</version>
</dependency>

To start the server, add the following code snippet to your application:


          
           
Undertow server = Undertow.builder()
        .addHttpListener(8080, "localhost")
        .setHandler(exchange -> {
            exchange.getResponseHeaders().put(Headers.CONTENT_TYPE, "text/plain");
            exchange.getResponseSender().send("Hello World!");
        })
        .build();
server.start();

This code will start an Undertow server on port 8080 and serve a simple “Hello World!” response. You can now access the server at http://localhost:8080.

Writing an Undertow Handler

Undertow uses a handler-based architecture. To write a custom handler, you will need to implement the 

HttpHandler
interface. For example, the following handler logs all incoming requests: 

          
           
public class LoggingHandler implements HttpHandler {

    @Override
    public void handleRequest(HttpServerExchange exchange) throws Exception {
        System.out.println("Received request from " + exchange.getSourceAddress());
        exchange.getResponseHeaders().put(Headers.CONTENT_TYPE, "text/plain");
        exchange.getResponseSender().send("This is the response!");
    }
}

Once the handler is defined, it can be added to the server using the following code:


          
           
Undertow server = Undertow.builder()
        .addHttpListener(8080, "localhost")
        .setHandler(new LoggingHandler())
        .build();
server.start();

Now every time someone makes a request to the server, it will be logged to the console.

Servlet Support

Undertow also supports the Java Servlet API, which makes it easy to deploy existing web applications or create new ones using the standard servlet APIs. To enable servlet support, add the undertow-servlet dependency to the project:


          
           
<dependency>
    <groupId>org.wildfly.undertow</groupId>
    <artifactId>undertow-servlet</artifactId>
    <version>2.0.14.Final</version>
</dependency>

This allows you to deploy servlets using the standard deployment descriptor (

web.xml
) and register servlet mappings. For example, this web.xml file registers a Hello World servlet: 

          
           
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee
         http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd"
         version="3.1">
    <servlet>
        <servlet-name>HelloWorldServlet</servlet-name>
        <servlet-class>com.example.HelloWorldServlet</servlet-class>
    </servlet>
    <servlet-mapping>
        <servlet-name>HelloWorldServlet</servlet-name>
        <url-pattern>/hello</url-pattern>
    </servlet-mapping>
</web-app>

This servlet can then be deployed using the following code:


          
           
DeploymentInfo servletBuilder = deployment()
        .setClassLoader(MyApplication.class.getClassLoader())
        .setContextPath("/")
        .setDeploymentName("myapp.war")
        .addServlets(
                servlet("HelloWorldServlet", com.example.HelloWorldServlet.class)
                        .addMapping("/hello"));

DeploymentManager manager = defaultContainer()
        .addDeployment(servletBuilder);
manager.deploy();

HttpHandler servletHandler = manager.start();

Undertow server = Undertow.builder()
        .addHttpListener(8080, "localhost")
        .setHandler(servletHandler)
        .build();
server.start();

Now when a request is made to http://localhost:8080/hello, the request will be routed to the 

HelloWorldServlet
.

Conclusion

Undertow is a powerful, lightweight web server and servlet container that is easy to use and highly configurable. It provides support for many different programming languages and technologies and makes it easy to deploy existing web applications. With its handler-based architecture and servlet support, Undertow is a great choice for any web application.
