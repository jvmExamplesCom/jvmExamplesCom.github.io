Java.rmi.registry.Registry

The Java.rmi.registry.Registry class provides a way for remote objects to be registered and looked up in a particular Java Virtual Machine (JVM). It is used when creating an RMI server application. The Registry is a distributed technology that allows application components running on separate JVMs to find each other and communicate with each other through remote procedure calls (RPCs).

Overview of Java.rmi.registry.Registry

Java.rmi.registry.Registry is an interface between the client and the server in the Remote Method Invocation (RMI) framework. It is responsible for maintaining a list of objects that can be accessed remotely by other applications. When an application wants to use a remote object, it looks up the name of the object in the registry and then invokes a method on the object. The registry can also be used to bind an object to a well-known name, allowing clients to look up the object and invoke its methods.

Creating an RMI Registry

A registry can be created in the server application by calling the static method 


          
           
createRegistry(int port)
. This creates a registry on the specified port, or if the port is not specified, it will create a registry on random port. The registry allows clients to lookup for remote objects that are bound to this registry.

Binding Objects to the Registry

Once the registry is created, objects can be bound to it using the 


          
           
bind(String name, Remote obj)
method. This method binds the remote object to the specified name. The client applications can then use the name to lookup for the object.

Looking up Objects from the Registry

Clients can look up objects in the registry by using the 


          
           
lookup(String name)
method. This method returns the remote object associated with the specified name. The client can then invoke methods on the remote object.

Unbinding Objects from the Registry

Objects can be unbound from the registry using the 


          
           
unbind(String name)
method. This method unbinds the object associated with the specified name from the registry.

Example

The following is an example of creating an RMI registry and binding an object to it:


          
           
Registry registry = LocateRegistry.createRegistry(1099);
MyRemoteObject obj = new MyRemoteObjectImpl();
registry.bind("MyRemoteObjectName", obj);

In the above code, the registry is created on port 1099 and an object is bound to the registry. The client applications can now lookup the object using the name "MyRemoteObjectName".
