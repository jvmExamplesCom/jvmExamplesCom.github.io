java.awt.image.IndexColorModel

The java.awt.image.IndexColorModel class is a color model used for indexed images. It allows you to specify an array of red, green and blue values that represent the colors in your image. This can be used to generate palettes that contain only certain colors as well as creating an indexed-color representation of an image.

Constructors

The IndexColorModel class has one constructor: 

          
           
public IndexColorModel (int bits, int size, byte[ ] cmap, int start, boolean hasAlpha)
The constructor requires the following parameters:
  • bits - The number of bits each pixel occupies.
  • size - The size of the color component arrays.
  • cmap - An array of color components.
  • start - The starting offset of the cmap array.
  • hasAlpha - A boolean value indicating whether the cmap contains alpha components.

Examples

Here is an example of how to use the IndexColorModel class: 

          
           
// Create an indexed color model
IndexColorModel icm = new IndexColorModel(8, 256, new byte[] { 0, (byte)255, (byte)127 }, 0, false);

// Set the red component of the first color
icm.setReds(new int[] { 255 });

// Get the RGB value of the first color
int rgb = icm.getRGB(0);
System.out.println("RGB value of color 0: " + rgb);
In this example, we create an indexed color model with 8 bits per pixel and 256 colors in the color map. We then set the red component of the first color to 255 and retrieve the RGB value of the color. The output of this program is:
