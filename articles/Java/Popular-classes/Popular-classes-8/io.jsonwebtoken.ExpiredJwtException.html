io.jsonwebtoken.ExpiredJwtException

The io.jsonwebtoken.ExpiredJwtException is an exception thrown when a JSON Web Token (JWT) has expired and can not be used for authentication or authorization purposes.

Cause of the Exception

The exception is thrown when the JWT has expired as determined by the "exp" claim in the JSON payload. The exp claim must be represented as the number of seconds from 1970-01-01T0:0:0Z UTC until the specified date/time. When the JWT is validated, the value of the "exp" claim is compared against the current date/time. If the "exp" claim is before the current date/time, then the JWT is considered expired and the exception is thrown.

Handling the Exception

When this exception is thrown, it indicates that the JWT being used to authenticate or authorize a request is invalid. The client should receive an appropriate error response and the server should have a means of monitoring for this type of failure. In some cases, the client may be able to request a new JWT before retrying the request.
