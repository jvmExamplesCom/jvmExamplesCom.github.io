android.content.DialogInterface.OnCancelListener

An onCancelListener is a listener that helps respond to cancel events when the user interacts with a dialog. It is implemented in the android.content.DialogInterface class and is used to handle events that occur when a dialog is cancelled.

Implementation

An OnCancelListener listener is registered to a dialog using the DialogInterface.setOnCancelListener() method. Once registered, the onCancel() method of the OnCancelListener will be invoked when the user cancels out of the dialog. 

          
           
DialogInterface.setOnCancelListener(new DialogInterface.OnCancelListener() {
        @Override
        public void onCancel(DialogInterface dialog) {
            // Perform some task when the dialog is cancelled
        }
    });
The code sample above sets an OnCancelListener listener to a DialogInterface dialog. Whenever the user cancels out of the dialog, the onCancel() method will be invoked and can be used to perform some task.

Examples

Example 1

The following example shows how to create a dialog and set an OnCancelListener listener to it. When the user cancels out of the dialog, an alert dialog is shown to the user. 

          
           
AlertDialog dialog = new AlertDialog.Builder(context)
    .setMessage("Do you want to cancel this operation?")
    .setPositiveButton("Yes", new DialogInterface.OnClickListener() {
        @Override
        public void onClick(DialogInterface dialogInterface, int i) {
            // Perform some task 
        }
    })
    .setNegativeButton("No", null)
    .create();

dialog.setOnCancelListener(new DialogInterface.OnCancelListener() {
    @Override
    public void onCancel(DialogInterface dialogInterface) {
        showAlertDialog("Operation cancelled");
    }
});

dialog.show();

In this example, we create an activity that has an EditText view and an okButton. When the user clicks on the okButton, a dialog appears asking for confirmation. If the user clicks on “Yes”, then the text in the EditText is saved. If the user clicks on “No”, then the text in the EditText is not saved. 

          
           
public class MyActivity extends AppCompatActivity {
    private EditText editText;
    private Button okButton; 

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_my);

        editText = findViewById(R.id.edit_text);
        okButton = findViewById(R.id.ok_button);

        okButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View view) {
                showConfirmationDialog();
            }
        });
    }

    private void showConfirmationDialog() {
        AlertDialog dialog = new AlertDialog.Builder(this)
                .setMessage("Do you want to save the changes?")
                .setPositiveButton("Yes", new DialogInterface.OnClickListener() {
                    @Override
                    public void onClick(DialogInterface dialogInterface, int i) {
                        saveChanges();
                    }
                })
                .setNegativeButton("No", null)
                .create();

        dialog.setOnCancelListener(new DialogInterface.OnCancelListener() {
            @Override
            public void onCancel(DialogInterface dialogInterface) {
                discardChanges();
            }
        });

        dialog.show();
    }

    private void saveChanges() {
        String text = editText.getText().toString();
        // Perform some task with the text
    }

    private void discardChanges() {
        editText.setText("");
    }
}
In the example above, an OnCancelListener listener is registered to the dialog. When the user cancels out of the dialog, the onCancel() method of the OnCancelListener is invoked and the text in the EditText is discarded.
