android.webkit.WebResourceError

android.webkit.WebResourceError is an error class that describes errors generated in the network layer when loading resources such as web pages. It contains information about the error code and error description associated with the error. This class is used in conjunction with android.webkit.WebViewClient to provide error info during asynchronous page loading.

Example usage

The following example shows how to use android.webkit.WebResourceError to detect and handle errors in a WebView application.


          
           
webView.setWebViewClient(new WebViewClient() {
    public void onReceivedError(WebView view, WebResourceRequest request, WebResourceError error) {
        if (error != null) {
            int errorCode= error.getErrorCode();
            String description = error.getDescription();
            // handle error according to error code
        } 
     }
});

In this example, we set a WebView object’s WebViewClient. Inside the onReceivedError() method, the WebResourceError is accessed via the third parameter. The error code and description are both retrieved from the WebResourceError object.

Error codes

The following error codes are supported:
  • ERROR_AUTHENTICATION
  • ERROR_BAD_URL
  • ERROR_CONNECT
  • ERROR_FAILED_SSL_HANDSHAKE
  • ERROR_FILE
  • ERROR_FILE_NOT_FOUND
  • ERROR_HOST_LOOKUP
  • ERROR_IO
  • ERROR_PROXY_AUTHENTICATION
  • ERROR_REDIRECT_LOOP
  • ERROR_TIMEOUT
  • ERROR_TOO_MANY_REQUESTS
  • ERROR_UNKNOWN
  • ERROR_UNSUPPORTED_AUTH_SCHEME
  • ERROR_UNSUPPORTED_SCHEME

Example of handling authentication error

The following example demonstrates how to handle authentication errors:


          
           
if (error.getErrorCode() == WebResourceError.ERROR_AUTHENTICATION) {
    // handle authentication error
}

In this example, we check for the ERROR_AUTHENTICATION error code and in case of a match, handle the authentication error appropriately.
