org.springframework.dao.DuplicateKeyException

The org.springframework.dao.DuplicateKeyException is an unchecked exception thrown when a record could not be inserted into a database due to a duplicate primary key. This exception is thrown from the Spring Data Access framework and indicates a potential problem with the application's data integrity.

Example

Consider the following example which inserts a new user record into a database.


          
           
public void insertUser(User user) throws DuplicateKeyException {
    jdbcTemplate.update(
        "insert into users (name, email) values (?, ?)",
        user.getName(), user.getEmail();
);}

If there is already a record in the database with the same email address as that provided by the User object, then the statement will fail and the framework will throw a DuplicateKeyException.

Handling the Exception

The best way to handle a DuplicateKeyException is to catch it and provide specific feedback to the user explaining what has happened. 


          
           
try {
    insertUser(user);
} catch (DuplicateKeyException e) {
    System.out.println("A user with that email address already exists.");
}

It is also worth noting that this exception can be thrown from multiple parts of the database layer, so it is important to anticipate where it could be thrown and handle it appropriately.
