java.nio.file.AccessDeniedException

java.nio.file.AccessDeniedException is an exception thrown when an attempt is made to access a file or directory that the user does not have permission to access. This exception is thrown when a call to one of the following methods fails with a SecurityException: createFile(), createDirectory(), delete(), open(), readAttributes(), setAttribute().

Example 1 - CreateFile() Method


          
           
public static void main(String[] args) {
      Path path = Paths.get("/test");

      try {
         Files.createFile(path);
      } catch (AccessDeniedException e) {
         System.out.println("Access denied - cannot create file");
      } catch (IOException e) {
         System.out.println("IOException");
      }
   }

In the above example, a Path object for the /test directory is created and then used in a call to the createFile() method. If the user does not have permission to create files in that directory, an AccessDeniedException will be thrown and the appropriate message will be printed.
