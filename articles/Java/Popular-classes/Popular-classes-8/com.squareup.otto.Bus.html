com.squareup.otto.Bus

The com.squareup.otto.Bus is a powerful tool for simplifying communication between components in any Android application. It is an open source solution for cross-component communication in applications, providing a mechanism for publish-subscribe messaging between components which can be used to create modular and more maintainable architectures. This article will explore how to use the Bus to send messages between components in an Android application.

Creating the Bus

The first step when using the Bus is creating an instance of it, which can be done by simply calling the constructor:


          
           
Bus bus = new Bus();

Registering and Unregistering Listeners

Next, any component which needs to be able to send or receive messages must register itself as a listener on the Bus. For example, a Fragment might need to register when it is created and unregister it when it is destroyed. To do this, the component should implement the BusListener interface, which provides two methods: 

          
           
onRegister()
and 

          
           
onUnregister()
. Once the BusListener has been implemented, registering and unregistering the component as a listener on the bus is done with the following calls: 

          
           
bus.register(listener);
bus.unregister(listener);

Publishing and Receiving Messages

After registering and unregistering listeners, the next step is to actually send and receive messages. To publish a message, simply call the Bus’s post() method, passing in the message object: 

          
           
bus.post(message);
Any component that is registered as a listener on the Bus will then receive the message by implementing the onMessageReceived() method, which provides an argument of the same type as the message object.
