javax.naming.ldap.LdapName

The javax.naming.ldap.LdapName class extends javax.naming.Name and represents a LDAP distinguished name as specified in RFC 2253. It is used to represent LDAP names for various purposes like search and parsing.

Creating a LdapName Object

A LdapName can be created from a string that contains its components:


          
           
String dn = "cn=John Doe,ou=people,dc=example,dc=com";
LdapName ln = new LdapName(dn);

LdapName Methods

The following are some of the useful methods of the LdapName class:


          
           
getRdns() - Returns the list of Relative Distinguished Names (RDNs) of this name as an unmodifiable List of LdapRdn objects in order.
add(LdapRdn rdn) - Appends the given RDN to the end of the name.
isEmpty() - Returns true if the name is empty.
remove(int pos) - Removes the RDN at the given position.

Example

The following example shows how to use the LdapName class:


          
           
String dn = "cn=John Doe,ou=people,dc=example,dc=com";
LdapName ln = new LdapName(dn);

System.out.println("Name components: " + ln.getRdns());
System.out.println("Is the name empty? " + ln.isEmpty());

ln.add(new LdapRdn("emailAddress=john.doe@example.com"));

System.out.println("New name components: " + ln.getRdns());

ln.remove(2);
System.out.println("Name components after removal: " + ln.getRdns());
This example will output: 

          
           
Name components: [cn=John Doe, ou=people, dc=example, dc=com]
Is the name empty? false
New name components: [cn=John Doe, ou=people, dc=example, dc=com, emailAddress=john.doe@example.com]
Name components after removal: [cn=John Doe, ou=people, dc=example]
