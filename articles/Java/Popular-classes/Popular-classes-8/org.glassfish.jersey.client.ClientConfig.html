What is org.glassfish.jersey.client.ClientConfig?

org.glassfish.jersey.client.ClientConfig is an interface that provides a way to configure the behaviour of Jersey-based client applications. It defines the necessary configuration information to register custom components such as filters, connectors, and other services.

Using ClientConfig

The following code example shows how to create an instance of ClientConfig and add it to a javax.ws.rs.client.WebTarget instance: 

          
           
ClientConfig config = new ClientConfig(); 
WebTarget target = ClientBuilder.newClient(config).target(...)
You can also use the ClientConfig to customize components such as connectors, filters, and other services: 

          
           
// Set a custom filter 
config.register(MyFilter.class); 

// Set a custom connector 
config.connectorProvider(new MyConnectorProvider()); 

// Set custom properties 
config.property("myProperty", "myValue");
You can then use the configured WebTarget instance to invoke requests:
