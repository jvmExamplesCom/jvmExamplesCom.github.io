org.springframework.data.jpa.repository.config.EnableJpaAuditing

Summary

org.springframework.data.jpa.repository.config.EnableJpaAuditing is an annotation used to enable JPA auditing in the Spring Data JPA application. It allows the application to capture changes made to the entities and persist them in the database.

Example Usage

To enable JPA auditing in the application you can simple add the @EnableJpaAuditing annotation to your @Configuration class: 

          
           
@Configuration
@EnableJpaAuditing
public class MyConfiguration {
	// ...
}
By adding this annotation to the configuration class, Spring Data JPA will automatically enable auditing for all entities which are managed by it.

Using Audit Specific Configuration

You can also use the configuration properties provided by @EnableJpaAuditing to apply audit-specific configurations like the auditor provider to use or the date time provider. These properties can be set in the application.properties file: 

          
           
spring.jpa.properties.javax.persistence.schema-generation.database.action=create

          

          
           
spring.jpa.auditing.date-time-provider=org.springframework.data.jpa.domain.support.AuditingDateTimeProvider

          

          
           
spring.jpa.auditing.auditor-aware-ref=myAuditorAware

Conclusion

The annotation org.springframework.data.jpa.repository.config.EnableJpaAuditing provides a convenient way to enable JPA auditing in the application. It can be used to capture changes made to the entities and persist them in the database. You can also use the configuration properties provided by this annotation to apply audit-specific configurations.
