What is org.springframework.security.web.access.AccessDeniedHandler?

The org.springframework.security.web.access.AccessDeniedHandler is an interface used for handling access denied exceptions in Spring Security. It is called when an authenticated user tries to access a resource that they're not authorized to use. It also provides a way to respond to the user, such as redirecting them to a custom error page or showing an error message.

How Does AccessDeniedHandler Work?

The AccessDeniedHandler will be invoked whenever an unauthenticated user attempts to access a restricted resource. When this happens, the AccessDeniedHandler will examine the request and make a decision on whether or not the user should be allowed access. If the user is not authorized, then the AccessDeniedHandler will execute its handle() method. This method takes two parameters - request and response. The request parameter contains information about the current request, while the response parameter contains an object that can be used to construct an appropriate response. In some cases, the AccessDeniedHandler may simply send an HTTP error code (e.g. 403 Forbidden) back to the user. In other cases, it may redirect the user to a custom error page.

Example Code

Below is an example of how to implement the AccessDeniedHandler interface in a Spring application. 

          
           
@Component
public class MyAccessDeniedHandler implements AccessDeniedHandler {
    @Override
    public void handle(HttpServletRequest request,
                       HttpServletResponse response, 
                       AccessDeniedException exception) throws IOException {
        response.sendRedirect("/error-page");
    }
}
The code above defines an implementation of the AccessDeniedHandler interface which will send an HTTP redirect to the /error-page page if an authentication fails.
