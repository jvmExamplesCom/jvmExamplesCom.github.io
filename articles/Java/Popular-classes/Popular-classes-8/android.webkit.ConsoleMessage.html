Android Webkit ConsoleMessage

This article looks at the Android Webkit ConsoleMessage, which is an object used to capture all instances of JavaScript errors and warnings emitted by a web page. It helps developers identify and resolve errors quickly, as well as provide useful information about the state and performance of the page’s code. ConsoleMessage objects are sent to webViewClient's onConsoleMessage() callback, which can then be handled as appropriate for the application. The onConsoleMessage() callback signature is: 

          
           
public boolean onConsoleMessage(ConsoleMessage cm);
The ConsoleMessage object passed in as the argument contains three important pieces of information; the message itself (as a String), the line number the message occurred on and the source ID.

ConsoleMessage methods

The ConsoleMessage object contains two public methods: 

          
           
getMessage()
Returns the message as a String. This will be the actual error or warning message as generated by the web page’s JavaScript. 

          
           
getLineNumber()
Returns the line number of the script or expression where the message originated. 

          
           
getSourceId()
Returns the sourceID of the message, which is typically the URL of the page that produced the message.

Example

Here is an example of how you could use the ConsoleMessage object: 

          
           
public class ConsoleLogger extends WebViewClient {

@Override
public boolean onConsoleMessage(ConsoleMessage cm) {

    //Get the error/warning message
    String message = cm.getMessage();

    //Get the line number where the error occurred
    int lineNumber = cm.getLineNumber();

    //Get the source ID (typically the URL of the page that caused the problem)
    String sourceId = cm.getSourceId();

    //Print the console message to LogCat
    Log.d("WebView", message + " -- From line " + Integer.toString(lineNumber) + " of " + sourceId );

    //Return true to indicate that we handled it
    return true;
}
}
In this example, whenever an error or warning message is emitted by the JavaScript code running in the WebView, the ConsoleMessage object will be sent to the onConsoleMessage() callback. We can then use its getMessage(), getLineNumber(), and getSourceId() methods to get more information about the message. In this example, the log message is printed to the LogCat.

Conclusion

In conclusion, the Android Webkit ConsoleMessage object is a useful tool for capturing JavaScript errors and warnings from a web page and displaying them in the application’s log. It also allows developers to get more detailed information about the exact cause of the problem, such as the line number and source ID.
