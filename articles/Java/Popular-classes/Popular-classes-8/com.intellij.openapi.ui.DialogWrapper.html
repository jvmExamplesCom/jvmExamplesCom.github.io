Overview of DialogWrapper

doOKAction()

Creating a Dialog with Dialog Wrapper

Creating a dialog using the DialogWrapper class requires extending the class and overriding its two abstract methods, 


          
           
getPrefferedFocusedComponent()
and 
doOKAction()
. The former method is used to define the component initially focused when the dialog box appears on the screen. The latter method, on the other hand, is used to define the action to be performed when the user presses the OK button.

The following code example illustrates how a DialogWrapper may be extended and used in a simple Java application. In this example, the DialogWrapper is extended and its two abstract methods, 


          
           
getPrefferedFocusedComponent()
and 
doOKAction()
, are overridden. The 

          
           
getPrefferedFocusedComponent()
method returns a text field and the 
doOKAction()
method prints a message to the console. 

          
           
public class MyDialog extends DialogWrapper {
    private JTextField nameField;

    public MyDialog(@Nullable Project project) {
        super(project);
        init();
        setTitle("My Dialog");
    }

    @Nullable
    @Override
    protected JComponent createCenterPanel() {
        JPanel panel = new JPanel(new BorderLayout());
        nameField = new JTextField();
        panel.add(nameField, BorderLayout.NORTH);
        return panel;
    }

    @Nullable
    @Override
    public JComponent getPrefferedFocusedComponent() {
        return nameField;
    }

    @Override
    protected void doOKAction() {
        String name = nameField.getText();
        System.out.println("Hello, " + name);
        super.doOKAction();
    }
}
