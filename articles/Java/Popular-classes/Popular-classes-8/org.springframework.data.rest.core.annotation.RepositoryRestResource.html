org.springframework.data.rest.core.annotation.RepositoryRestResource

org.springframework.data.rest.core.annotation.RepositoryRestResource is an annotation used to expose Spring Data Repositories as RESTful resources. This annotation can be used to control the details of how the repository is exposed over HTTP and how the responses will look like.

Using the Annotation

The org.springframework.data.rest.core.annotation.RepositoryRestResource annotation is used to configure a repository for exposing it as a REST resource. It can be used on an interface or a class.


          
           
@RepositoryRestResource
public interface AccountRepository extends CrudRepository<Account, Long> {
   // Methods
}

In the above example, the AccountRepository is annotated with the @RepositoryRestResource annotation. This means that all methods declared in the interface will be exposed as endpoints. This includes all the CRUD (create, read, update, delete) methods.

Attributes of the Annotation

The @RepositoryRestResource annotation has several attributes which can be specified as part of the annotation. These attributes can be used to configure the endpoint.


          
           
@RepositoryRestResource(path = "accounts", collectionResourceRel = "accounts")
public interface AccountRepository extends CrudRepository<Account, Long> {
   //Methods
}

In the above example, the path attribute is set to "accounts" which means that the endpoints will be exposed under the path "/accounts". The collectionResourceRel attribute is set to "accounts" which means that the response that is sent back will contain a link with the rel "accounts".

Other Attributes

The @RepositoryRestResource annotation also supports other attributes like itemResourceRel, exported etc. For example, the itemResourceRel attribute can be used to configure the relation to be used when the response contains a single resource instead of a collection.


          
           
@RepositoryRestResource(path = "accounts", collectionResourceRel = "accounts", itemResourceRel = "account")
public interface AccountRepository extends CrudRepository<Account, Long> {
   //Methods
}

In the above example, the itemResourceRel is set to "account" which means that the response containing a single resource will contain a link with the rel "account".

The exported attribute can be used to configure whether the endpoint should be exposed over HTTP or not. By default, the exported attribute is set to true which means that the endpoint will be exposed over HTTP. The exported attribute can be set to false if the endpoint should not be exposed over HTTP.


          
           
@RepositoryRestResource(path = "accounts", collectionResourceRel = "accounts", exported = false)
public interface AccountRepository extends CrudRepository<Account, Long> {
   //Methods
}

In the above example, the exported attribute is set to false which means that the endpoint will not be exposed over HTTP.
