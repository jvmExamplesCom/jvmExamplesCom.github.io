javax.ws.rs.core.FeatureContext

The javax.ws.rs.core.FeatureContext interface provides a mechanism for the application to configure and query provider objects configured in the application or container. Each feature represents a provider, such as a filter, interceptor, servlet or JAX-RS extension, which can be dynamically enabled or disabled by the application (as opposed to static registration in the deployment descriptor).

Methods

The FeatureContext interface contains the following methods:

  • getConfiguration() - Returns the Configuration associated with this FeatureContext instance.
  • getFeatures() - Returns a set of Feature instances that are enabled in this FeatureContext instance.
  • getContext() - Returns the Context associated with this FeatureContext instance.
  • getProperties() - Returns a set of properties set in this FeatureContext instance.
  • setProperty(String name, Object value) - Sets the property identified by name to the specified value.
  • removeProperty(String name) - Removes the property with the given name from the FeatureContext instance.

Example

Let's look at an example how to use FeatureContext. We will create a Filter class called MyFilter which extends javax.ws.rs.container.ContainerRequestFilter and implements FeatureContext interface.


          
           
public class MyFilter implements ContainerRequestFilter, FeatureContext {

    @Context
    private Configuration config;

    @Override
    public void filter(ContainerRequestContext requestContext) {
        System.out.println("MyFilter is invoked.");
    }

    @Override
    public Configuration getConfiguration() {
        return config;
    }

    @Override
    public Set<Feature> getFeatures() {
        return null;
    }

    @Override
    public Context getContext() {
        return null;
    }

    @Override
    public Map<String, Object> getProperties() {
        return null;
    }

    @Override
    public Object setProperty(String name, Object value) {
        return null;
    }

    @Override
    public Object removeProperty(String name) {
        return null;
    }

}

We have now implemented the required methods declared by FeatureContext interface. The filter method is called whenever a request is sent to the server. The other methods are only relevant when using this filter with a FeatureContext instance.

The getConfiguration() method returns the Configuration object associated with this FeatureContext instance. It is used to access information about the application or container.

The getFeatures() method returns a set of Feature objects that are enabled in this FeatureContext instance. This is useful for accessing optional features that may be enabled by the application. An example would be an authentication filter which is only enabled if certain conditions are met.

The getContext() method returns the Context object associated with this FeatureContext instance. It is used to access contextual information about the application or container.

The getProperties() method returns a set of properties that have been set on this FeatureContext instance. These properties can be used to store arbitrary information related to the current request.

The setProperty and removeProperty methods allow properties to be set and removed from this FeatureContext instance. These methods are useful for setting and removing contextual information related to the current request.
