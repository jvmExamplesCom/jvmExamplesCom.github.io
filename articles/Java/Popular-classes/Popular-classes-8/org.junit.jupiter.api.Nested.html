What is org.junit.jupiter.api.Nested?

The org.junit.jupiter.api.Nested annotation is used in JUnit 5 to define nested tests. It allows developers to group related tests into classes or methods and run them as part of the same test suite in order to minimize redundancy and maximize code coverage. By nesting tests, developers can also better organize test cases and reduce noise in their test results.

Example of org.junit.jupiter.api.Nested Usage

Let's look at an example of how the org.junit.jupiter.api.Nested annotation can be used. Consider a class containing a number of methods that will be tested:


          
           
public class MyClass {
    public int add(int a, int b) {
        return a + b;
    }

    public int subtract(int a, int b) {
        return a - b;
    }

    public int multiply(int a, int b) {
        return a * b;
    }
}

Now we can write some tests for these methods, but we can group related tests together using the Nested annotation:


          
           
@Nested
class MyClassTests {
    private final MyClass myClass = new MyClass();

    @Test
    void testAdd() {
        assertEquals(4, myClass.add(2, 2));
    }

    @Test
    void testSubtract() {
        assertEquals(0, myClass.subtract(2, 2));
    }

    @Nested
    class MultiplyTests {
        @Test
        void testSmallValues() {
            assertEquals(4, myClass.multiply(2, 2));
        }

        @Test
        void testLargeValues() {
            assertEquals(1_000_000, myClass.multiply(1_000, 1_000));
        }
    }
}

Here, we have grouped all of our related tests into the MyClassTests class and then further nested the multiply tests into the MultiplyTests inner class. This makes it easy to organize our tests in an efficient and maintainable way while still minimizing redundancy.

Conclusion

In summary, the org.junit.jupiter.api.Nested annotation is a powerful tool in JUnit 5 for organizing tests and reducing redundancy. By using it, developers can better organize their test suites and ensure code coverage without having to repeat tests.
