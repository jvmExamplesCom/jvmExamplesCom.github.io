Introduction to org.eclipse.jetty.server.handler.ResourceHandler

org.eclipse.jetty.server.handler.ResourceHandler is a resource handler which serves static content from the filesystem, classpath, or URL contexts. This handler can be configured with welcome files, aliases, and directory listings, making it ideal for serving static resources such as images, documents, and HTML pages.

Creating an instance of ResourceHandler

The simplest use of ResourceHandler is to create an instance of it and provide it with the location of the static resources that need to be served. 

          
           
ResourceHandler resourceHandler = new ResourceHandler();
resourceHandler.setResourceBase("/path/to/static/files");
This will create a resource handler that will serve resources found in the "/path/to/static/files" folder.

Configuring Resources

The ResourceHandler can be configured to serve specific files when a request is made for a directory. This is done by setting the "welcome files" for the handler. This is a list of filenames that will be checked in each directory. If one of these is found then it will be served. 

          
           
resourceHandler.setWelcomeFiles(new String[] { "index.html", "default.html" });

resourceHandler.setAliases(new HashMap<String, String>() {{
    put("/images", "/static/images");

resourceHandler.setShowDirListing(true);

Serving Resources

Once the ResourceHandler has been configured, it can be used to serve static resources. The handler can be added to an existing Jetty server, or it can be used in conjunction with a context handler to serve static resources from the classpath or a URL. 

          
           
ContextHandler contextHandler = new ContextHandler("/");
contextHandler.setResourceBase("classpath:/public/");
