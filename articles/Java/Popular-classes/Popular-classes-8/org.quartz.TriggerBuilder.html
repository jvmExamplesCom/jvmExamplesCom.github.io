What is org.quartz.TriggerBuilder

org.quartz.TriggerBuilder is a class used to create triggers in the Quartz Scheduler. It provides several methods to define start and end time, schedule rules, misfire instructions, and other trigger-related attributes. This article provides an overview of the various methods available to create triggers with TriggerBuilder and includes code examples of each.

Creating Triggers

TriggerBuilder allows for the creation of a new trigger using the 


          
           
newTrigger
method, which takes a string as an argument representing the name of the trigger. After calling newTrigger, the created Trigger can be further configured with a set of methods. Alternatively, existing triggers can be cloned and modified using the 

          
           
copyOf
method.

Scheduling Rules

The 

          
           
startAt
and 

          
           
endAt
methods are used to define the start and end times of a trigger. Both methods take a Date argument and must be called before a trigger can be used. The 

          
           
withSchedule
method is used to define scheduling rules for a trigger. This method takes a ScheduleBuilder argument, which can be used to create cron expressions, simple schedules, or calendars.

Misfire Instructions

In cases where trigger firings are missed due to system downtime or other issues, misfire instructions can be defined using the 


          
           
withMisfireHandlingInstruction
method. This method takes a MisfireInstruction argument, which has several possible values to determine how misfires should be handled.

Other Attributes

Other attributes such as the job key, trigger description, and trigger priority can be set using the 


          
           
withIdentity
, 

          
           
withDescription
, and 

          
           
withPriority
methods. Once all desired attributes have been set, the trigger can be built using the 

          
           
build
method.

Code Examples

Below are a few code examples illustrating how to create triggers using the TriggerBuilder class.

Simple trigger firing every 5 seconds: 

          
           
Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .withSchedule(SimpleScheduleBuilder.repeatSecondlyForever(5))
    .build();
Cron trigger firing every Friday at noon: 

          
           
Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .withSchedule(CronScheduleBuilder.cronSchedule("0 0 12 * * FRI"))
    .build();

Trigger that runs from 10am to 4pm on Wednesdays starting tomorrow:


          
           
Date startTime = DateBuilder.todayAt(10, 0, 0);
  Date endTime = DateBuilder.tomorrowAt(16, 0, 0);
  
  Trigger trigger = TriggerBuilder.newTrigger()
    .withIdentity("myTrigger", "group1")
    .startAt(startTime)
    .endAt(endTime)
    .withSchedule(CronScheduleBuilder.weeklyOnDayAndHourAndMinute(Calendar.WEDNESDAY, 10, 0))
    .build();

Conclusion

The TriggerBuilder class provides a convenient way to create triggers with the Quartz Scheduler. With its variety of methods, it can be used to configure start and end times, schedule rules, misfire instructions, and other trigger-related attributes. The code examples provided above illustrate the use of these methods and demonstrate how easy it is to create triggers with TriggerBuilder.
