Cucumber.api.CucumberOptions

Cucumber.api.CucumberOptions is a Java-based library that allows users to write automated tests for behavior-driven development. Basic Structure

The core concept of Cucumber.api.CucumberOptions is that each script consists of a feature, followed by defined scenarios. Associated scenarios are a set of steps necessary to carry out the feature, such as “Given I'm on the shop page, When I select an item, Then it should appear in my shopping cart”.

@When("I select an item")

    // code to execute when 'then' is called
}

The @Given, @When and @Then annotations are used to bind the natural language description of each step to the actual code that should be executed. After setting up the scenario, the script can be run with the command-line program 


          
           
cucumber.api.CucumberOptions.run()
, which will execute the functions in order to complete the scenario and generate a report.

  • Hooks: Hooks allow code to be executed before or after a scenario is executed, allowing users to perform setup/teardown operations or check/assert results.
