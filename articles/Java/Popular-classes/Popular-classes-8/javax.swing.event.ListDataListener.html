javax.swing.event.ListDataListener

The 

          
           
javax.swing.event.ListDataListener
interface is a part of the Java Swing library and is used to provide notification of changes to lists. It is implemented by Classes that need to track changes in a list data model.

The ListDataListener interface contains 3 methods; 


          
           
intervalAdded(ListDataEvent e)
, 

          
           
intervalRemoved(ListDataEvent e)
and 

          
           
contentsChanged(ListDataEvent e)
. These methods will be called on any modification of the underlying list data model.

IntervalAdded method

The IntervalAdded method will be called whenever new elements are added to the list data model. It will take a ListDataEvent as the parameter which contains details about the range of indices that have changed and the type of change (add or remove).


          
           
public void intervalAdded(ListDataEvent e) {
    int startIndex = e.getIndex0();
    int endIndex = e.getIndex1();
    System.out.println("Elements were added from index "+startIndex+" to index "+endIndex);
}

IntervalRemoved method

The IntervalRemoved method will be called whenever elements are removed from the list data model. It will also take a ListDataEvent as the parameter and contains details about the range of indices that have changed and the type of change (add or remove). 


          
           
public void intervalRemoved(ListDataEvent e) {
    int startIndex = e.getIndex0();

    System.out.println("Elements were removed from index "+startIndex+" to index "+endIndex);
}

ContentsChanged method

The ContentsChanged method can be called whenever there is a significant change in the list data model. This could involve an element being added, removed or updated. It will take a ListDataEvent as the parameter which contains details about the range of indices that have changed and the type of change (add or remove).
