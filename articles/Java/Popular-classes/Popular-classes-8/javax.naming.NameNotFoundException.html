java.naming.NameNotFoundException is a subclass of javax.naming.NamingException.

What is javax.naming.NameNotFoundException?

This exception is thrown if the name being resolved does not exist in the context, or does not identify a binding. It may also be thrown if the name is syntactically invalid. For example, java.naming.NameNotFoundException is thrown when attempting to look up a non-existent entry using the Naming class's lookup() method.

Example 1

In this example we will try to look up a non-existent entry in the Context:


          
           
try {
  Naming.lookup("nonExistentEntry");
} catch (javax.naming.NameNotFoundException e) {
  System.out.println("NameNotFoundException: " + e);
}
The output will be: 

          
           
NameNotFoundException: javax.naming.NameNotFoundException: nonExistentEntry not found

Example 2

In this example, we will try to lookup an entry with an invalid syntax:


          
           
try {
  Naming.lookup(">badSyntax");
} catch (javax.naming.NameNotFoundException e) {
  System.out.println("NameNotFoundException: " + e);
}
The output will be: 

          
           
NameNotFoundException: javax.naming.NameNotFoundException: Invalid syntax

What can be done to resolve this exception?

The most common cause of this exception is an incorrect name, so it is important to check that the names being used are syntactically valid. If the name is valid, then it is possible that the binding does not exist, so we should check that the appropriate entries have been created.
