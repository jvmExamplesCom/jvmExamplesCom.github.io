org.eclipse.paho.client.mqttv3.MqttClient

The org.eclipse.paho.client.mqttv3.MqttClient class provides a client for the MQTT protocol version 3.1 and 3.1.1. It can be used to connect with and to publish data to and receive data from an MQTT server. The class is easy to use and provides a variety of options for configuring the connection.

Creating an MQTT Client Instance

An MQTT client instance can be created using the following code:


          
           
String brokerUrl = "tcp://localhost:1883";
String clientId = "MyClientId";
MqttClient mqttClient = new MqttClient(brokerUrl, clientId);
The 
brokerUrl
specifies where the MQTT server should be accessed, i.e., the URL of the MQTT server. The 
clientId
specifies the ID of the client. It must be unique. This ID will be used by the server to identify the client.

Connecting to the MQTT Server

Once an instance of MqttClient is created, it can be used to connect to the server using the following code:


          
           
MqttConnectOptions connOpts = new MqttConnectOptions();
connOpts.setCleanSession(true);
mqttClient.connect(connOpts);
The 
MqttConnectOptions
class is used to configure the connection parameters. The 
setCleanSession
method sets the clean session flag that indicates whether the server should discard any previously stored session information. If the flag is set to true, the session is started anew and any existing session information will be discarded by the server.

Subscribing to a Topic

Once the connection is established, the client can subscribe to one or more topics using the following code:


          
           
String topic   = "MyTopic";
int qos        = 0;
mqttClient.subscribe(topic, qos);
The 
topic
variable specifies the topic to which the client will subscribe, and the 
qos
variable specifies the Quality of Service level of the subscription. The QoS level is an integer value, ranging from 0 to 2, indicating the reliability of the message delivery. A QoS level of 0 indicates that messages are delivered at least once, while a QoS level of 1 indicates that messages are delivered exactly once, and a QoS level of 2 indicates that messages are delivered exactly once and with acknowledgment.

Publishing a Message

The client can publish a message to a topic using the following code:


          
           
String topic   = "MyTopic";
int qos        = 0;
byte[] payload = "Hello World!".getBytes();
mqttClient.publish(topic, payload, qos, false);
The 
topic
variable specifies the topic to which the message should be published. The 
payload
variable contains the message to be published and the 
qos
variable specifies the Quality of Service level of the message. The last parameter is a 
boolean
flag indicating whether the message should be retained by the server.

Disconnecting from the Server

The client can disconnect from the server using the following code:


          
           
mqttClient.disconnect();

This method will close the connection to the server and stop all communication. Any pending messages will be delivered before the connection is closed.
