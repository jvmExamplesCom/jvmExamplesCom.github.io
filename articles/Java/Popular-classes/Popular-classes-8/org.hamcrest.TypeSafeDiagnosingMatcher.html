The org.hamcrest.TypeSafeDiagnosingMatcher Class

The 

          
           
org.hamcrest.TypeSafeDiagnosingMatcher
class allows developers to create custom matchers for unit testing in Java. Matchers are objects that determine whether an actual value meets certain criteria, checking for conditions such as adherence to a given type or equality with an expected value. The 

          
           
TypeSafeDiagnosingMatcher
class is a specialized and more versatile version of the matcher concept. This guide provides an introduction to this class and explores its uses through several code examples.

General Features

The 

          
           
TypeSafeDiagnosingMatcher
class derives from the 

          
           
BaseMatcher
class, which provides several methods for determining the truth or falsity of a test. It takes a generic type parameter that indicates the type of value being matched against. Its main features include:
  • Type safety. The type of an expected value must match the type of the actual value.
  • Diagnostic failure messages. Returns a detailed message when a match fails.
  • Decomposability. Checks for conditions through multiple tests within a single match method.

Simple Example

The following example shows a basic usage of this class to check if the length of a string matches a given integer: 

          
           
public static Matcher<String> hasLength(int length) {
    return new TypeSafeDiagnosingMatcher<String>() {
      @override
      public void describeTo(Description description) {
        description.appendText("string with length " + length);
      } 

      @override
      protected boolean matchesSafely(String str, Description mismatchDescription) {
        int actual = str.length();
        if (actual != length) {
          mismatchDescription.appendText("had length " + actual);
          return false;
        }
        return true;
      }
    };
  }
This code defines a 

          
           
hasLength()
factory method that takes an integer as a parameter and returns a matcher. This matcher can then be passed to the 

          
           
assertThat()
method, like so: 

          
           
assertThat("12345", hasLength(5));
If the length of the given string does not match the provided integer, the matcher will return a diagnostic failure message stating that the string had a length other than that expected.
