javax.script.ScriptEngineFactory

Introduction

The javax.script.ScriptEngineFactory is a part of Java Scripting API, which can be used to get information about available scripting language implementations. It also provides access to the corresponding 

ScriptEngine
instance. This allows one to create a custom scripting engine in Java, which provides support for any scripting language.

Methods

The javax.script.ScriptEngineFactory interface provides the following methods:

  • getEngineName()
    : returns the name of the scripting engine.
  • getEngineVersion()
    : returns the version of the scripting engine.
  • getLanguageName()
    : returns the language supported by the scripting engine.
  • 
            
             
getLanguageVersion()
    : returns the version of the language supported by the scripting engine.
  • getNames()
    : returns an array of strings representing the short names for the scripting engine.
  • 
            
             
getParameter(String key)
    : returns the value of the specified parameter.
  • getScriptEngine()
    : returns the ScriptEngine instance associated with the ScriptEngineFactory.

Example Usage

The following example illustrates how to use the 
ScriptEngineFactory
interface to determine which scripting languages are available on the system: 

          
           
ScriptEngineManager manager = new ScriptEngineManager();

for (ScriptEngineFactory factory : manager.getEngineFactories()) {
  String engineName = factory.getEngineName();
  String languageName = factory.getLanguageName();
  String languageVersion = factory.getLanguageVersion();

  System.out.println("Engine name: " + engineName);
  System.out.println("Language name: " + languageName);
  System.out.println("Language version: " + languageVersion);
}
