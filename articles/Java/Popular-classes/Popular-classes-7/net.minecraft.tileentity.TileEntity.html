net.minecraft.tileentity.TileEntity

TileEntity is an abstract class within the net.minecraft.tileentity package of the Minecraft Java Edition game. It is used for handling block entities, such as Chests, Furnaces and Brewing Stands, as well as more complex blocks, such as Dispensers, Jukeboxes and even Command Blocks.

The Basics

TileEntities are created when a block is placed in the world. There are two ways to create a TileEntities instance. The first way is to call the 

createTileEntity()
method on a Block instance: 

          
           
Block block = Blocks.CHEST;
TileEntity te = block.createTileEntity();

The second way is to use the TileEntity type's constructor, like this:


          
           
TileEntity te = new TileEntityChest();

TileEntity Data

TileEntities can store data about the block it represents. This data is stored in an NBTTagCompound, which is a collection of key-value pairs. These key-value pairs can be used to store and retrieve any information you wish. Here is an example of setting and getting data from a TileEntity:


          
           
TileEntityChest chestTE = (TileEntityChest) te;
NBTTagCompound nbt = new NBTTagCompound();
nbt.setInteger("item_count", 5);
chestTE.writeToNBT(nbt);
int itemCount = nbt.getInteger("item_count");

You can also read and write NBT data directly to and from a TileEntity, without creating an NBTTagCompound:


          
           
chestTE.setInteger("item_count", 5);
int itemCount = chestTE.getInteger("item_count");

TileEntities also keep track of their block's position in the world. This information can be retrieved using the 

getPos()
method, like this: 

          
           
BlockPos pos = te.getPos();

Synchronizing TileEntity Data with the Client

The data stored by a TileEntity is not automatically synchronized with clients in the game. In order for a client to receive updates from the server about the TileEntity's data, the server must call the 

syncTileEntity()
method on the TileEntity. This will send an update packet to the client with all of the TileEntity's NBT data. Here is an example of syncing a TileEntity with the client: 

          
           
te.syncTileEntity();

Conclusion

TileEntity is an integral part of modding in Minecraft Java Edition. It is used to store and synchronize block-specific data with clients in the game. Understanding how to create, read and write TileEntity data is essential for making your mods work correctly.
