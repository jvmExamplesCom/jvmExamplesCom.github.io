com.rabbitmq.client.Connection

Java is one of the most popular programming languages in the world. It is widely used for many applications, from web development and mobile apps to enterprise applications. The RabbitMQ client library makes it easy to connect to a RabbitMQ server using the Java programming language.

The com.rabbitmq.client.Connection class provides the functionality necessary to establish and manage a connection to a RabbitMQ server. Once connected, users can send messages, receive messages, and monitor the state of the connection.

Creating a Connection

To create a connection, use the static factory method ConnectionFactory.newConnection(). This method accepts two arguments: a connection configuration object, and an IFrameHandler object.


          
           
Connection connection = ConnectionFactory.newConnection(
    new ConnectionConfig(), 
    new IFrameHandler()
);

The ConnectionConfig object contains configuration settings for the connection, such as the hostname and port of the RabbitMQ server.


          
           
ConnectionConfig config = new ConnectionConfig();
config.host("192.168.0.1");
config.port(5672);

The IFrameHandler object is used to handle incoming frames. Incoming frames must be handled in order for messages to be received. For more information on working with frames, see the javadocs.

Sending Messages

Once connected, the Connection object can be used to send messages. To send a message, first create a Channel object with the connection's createChannel() method.


          
           
Channel channel = connection.createChannel()

Then use the channel's basicPublish() method to send a message.


          
           
channel.basicPublish(
    "exchangeName", 
    "routingKey", 
    MessageProperties.BASIC,
    "Hello World!".getBytes()
);

The arguments to the basicPublish() method specify the exchange the message should be routed to, the routing key or topic to use, the message properties, and the message body. For more information on sending messages, see the javadocs.

Receiving Messages

To receive messages, use the channel's basicConsume() method to consume messages from a queue.

    "queueName", 
    true, 
    new DefaultConsumer(channel)
);

The arguments to the basicConsume() method specify the queue to consume from, whether or not auto-acknowledge is enabled, and a consumer that will handle the incoming messages. For more information on receiving messages, see the javadocs.

Closing a Connection

Finally, when a connection is no longer needed, it can be closed with the connection's close() method.

This method closes all open channels and terminates the connection. For more information on closing connections, see the javadocs.
