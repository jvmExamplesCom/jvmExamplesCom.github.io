javax.security.auth.login.Configuration

javax.security.auth.login.Configuration provides a static Configuration object, which is used by the LoginContext class to find and instantiate LoginModules. It contains methods used to specify the set of LoginModules used by a particular application, as well as the shared state between them.

Configuration Methods

The Configuration class contains several methods that allow users to specify the LoginModules and shared state associated with the application. The methods include:

  • 
            
             
getAppConfigurationEntry()
    – returns an array of Configuration.AppConfigurationEntry objects, representing the LoginModules configured for this application.
  • refresh()
    – refreshes the Configuration object with the latest information from the underlying security provider.
  • setConfiguration()
    – allows the user to specify an alternative Configuration instance for the application.

Example

In this example, we will use the Configuration class to configure two LoginModules (MyLoginModule1 and MyLoginModule2) for our application.


          
           
Configuration.setConfiguration(new Configuration() {
public AppConfigurationEntry[] getAppConfigurationEntry() {
    Map<String, String> options = new HashMap<>();
    options.put("mode", "required");
    return new AppConfigurationEntry[] {
        new AppConfigurationEntry("com.example.MyLoginModule1",
            AppConfigurationEntry.LoginModuleControlFlag.REQUIRED, options),
        new AppConfigurationEntry("com.example.MyLoginModule2",
            AppConfigurationEntry.LoginModuleControlFlag.REQUIRED, options),
    };
}});

This will configure the MyLoginModule1 and MyLoginModule2 LoginModules to be required to successfully authenticate the application. Note that if either of the LoginModules fails, then the authentication process will fail.
