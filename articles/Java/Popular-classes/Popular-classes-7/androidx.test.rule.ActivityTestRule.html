androidx.test.rule.ActivityTestRule

The androidx.test.rule.ActivityTestRule is a JUnit rule which provides functional testing of a single activity. It is designed to launch activities in isolation and gives the test writer access to the underlying activity instance. This allows tests to control the state of the activity and to interact with its user interface.

Usage

To use the ActivityTestRule, you must declare an instance of it as a public field in your test class: 

          
           
@Rule
public ActivityTestRule<MyActivity> activityRule = 
    new ActivityTestRule<>(MyActivity.class);
Then, before running each test, the activity will be launched. Alternatively, the launchActivity method can be used to manually launch the activity: 

          
           
MyActivity activity = activityRule.launchActivity(null);
The launchActivity method takes an Intent as an argument, which can be used to launch the activity in a specific state. For example, to start the activity with some data: 

          
           
Intent intent = new Intent();
intent.putExtra("some_data", "hello");
MyActivity activity = activityRule.launchActivity(intent);
Once the activity is launched, the test writer has access to the underlying activity instance. This can be used to check the current state of the activity, or to interact with its user interface. For example, to check the content of a TextView: 

          
           
TextView textView = (TextView) activity.findViewById(R.id.my_text_view);
assertThat(textView.getText(), is("hello"));
When the test is finished, the activity will be automatically destroyed.
