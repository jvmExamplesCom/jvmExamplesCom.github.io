Introduction to org.hibernate.Transaction

The 

          
           
org.hibernate.Transaction
class is part of the Hibernate framework and is a Java object used for managing transactions in a database. It is responsible for controlling the behavior of a transaction, such as committing and rolling back the changes made. The 

          
           
org.hibernate.Transaction
class may be configured differently depending on the type of database being used and how the transaction is implemented.

Creating a Transaction Object

A 
Session
object must be created before a 
Transaction
object can be instantiated. The 
Session
object uses the Database Connection object to create the 
Transaction
. The code below creates a 
Session
object and a 
Transaction
object: 

          
           
Session session = SessionFactoryUtils.getSession();
    Transaction tx = session.beginTransaction();

Committing a Transaction

Once a 
Transaction
has been instantiated, it can be committed using the 
commit()
method. This method signals the database to save all changes that have been made since the 
Transaction
was started. It is important to note that the 
commit()
method can only be called once per 
Transaction
, otherwise an exception will be thrown. The code below demonstrates how to commit a 
Transaction
: 

          
           
tx.commit();

Rolling Back a Transaction

In the event that something goes wrong during the course of a 

Transaction
, it can be rolled back using the 
rollback()
method. This will undo any changes that have been made since the transaction began. The code below demonstrates how to roll back a 
Transaction
: 

          
           
tx.rollback();

Closing a Transaction

Once a 
Transaction
has been committed or rolled back, it should be closed by calling the 
close()
method. This method tells the database that the 
Transaction
is complete, and any pending changes will no longer be applied. The code below demonstrates how to close a 
Transaction
: 

          
           
tx.close();

Example of a Full Transaction

The following example demonstrates a full transaction cycle using the 


          
           
org.hibernate.Transaction
class. This example creates a 
Session
, begins a 
Transaction
, executes some SQL statements, commits the changes, closes the 
Transaction
and finally closes the 
Session
: 

          
           
// Create a Session
    Session session = SessionFactoryUtils.getSession();
    
    // Begin a Transaction
    Transaction tx = session.beginTransaction();
    
    // Execute some SQL statements
    session.executeQuery("UPDATE table SET column = 'value'");
    
    // Commit the changes
    tx.commit();
    
    // Close the Transaction
    tx.close();
    
    // Close the Session
    session.close();
