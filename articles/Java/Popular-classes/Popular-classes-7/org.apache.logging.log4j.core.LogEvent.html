org.apache.logging.log4j.core.LogEvent

The org.apache.logging.log4j.core.LogEvent class is an implementation of the Logger interface provided by the Apache Logging Framework. It represents an event resulting from a logging operation which can include a message, exception, or stack trace. It provides an easy to use mechanism for customizing and formatting log messages, as well as allowing applications to send log events to different destinations, such as files, databases, or other systems.

Creating and Using Log Events

Log events are created using the LoggerFactory.getLogEvent() method, passing in the appropriate log event type. The static LogEvent enum includes the available log event types, such as DEBUG, ERROR, or FATAL:


          
           
LogEvent logEvent = LoggerFactory.getLogEvent(LogEvent.Type.DEBUG);

The LogEvent instance is then used to set any log event properties, such as the message or exception:


          
           
logEvent.setMessage("This is a debug message");
logEvent.setException(new Exception("This is an exception"));

The log event can then be logged using the Logger.log() method:


          
           
Logger logger = LoggerFactory.getRootLogger();
logger.log(logEvent);

Formatting Log Events

Log events may also be formatted using layout formatting patterns. The default pattern includes information such as the date and time, the log level, and the message. The following code example shows how to set the pattern for a log event:


          
           
LayoutFormatter<logevent> formatter = LayoutFormatter.newBuilder()
    .setPattern("%d %p %m%n")
    .build();

logEvent.setFormatter(formatter);
</logevent>

The pattern string consists of characters that represent various pieces of information about the log event, such as the date, the log level, and the message. The following are some of the available pattern strings:

  • %d
    Date and time.
  • %p
    Log level (e.g. DEBUG, INFO, WARN, ERROR, FATAL).
  • %m
    Message.
  • %n
    New line.

Customizing Loggers

The Logger class allows for customizing the log output via Appenders, filters, and layouts. Appenders are responsible for sending log events to a target destination, such as a file, database, or other system. Filters allow for filtering log events based on a certain criteria, such as the log level or message content. Layouts allow for formatting log events into specific formats, such as XML or JSON. The following example shows how to configure a logger with an appender, filter, and layout:


          
           
Logger logger = LoggerFactory.getRootLogger();

// Create a file appender
Appender<logevent> appender = FileAppender.createAppender("/var/log/mylog.log", "false");

// Create a log level filter
AppenderFilter filter = LogLevelFilter.createFilter(LogLevel.DEBUG, LogLevel.ERROR);

// Create a JSON layout
LayoutFormatter<logevent> layout = LayoutFormatter.newBuilder()
    .setType(LayoutType.JSON)
    .build();

// Add the appender, filter, and layout to the logger
logger.addAppender(appender, filter, layout);
</logevent></logevent>

Once the logger has been configured, the log events sent to the logger will be sent to the configured appender, filtered by the filter, and formatted by the layout.
