org.springframework.security.oauth2.config.annotation.web.configurers.ResourceServerSecurityConfigurer

org.springframework.security.oauth2.config.annotation.web.configurers.ResourceServerSecurityConfigurer provides a mechanism for configuring the security for an OAuth 2.0 resource server. It allows applications to secure their resources without having to implement the OAuth 2.0 specification themselves. The ResourceServerSecurityConfigurer is used to specify the security configuration for the resource server. This includes setting the authentication manager, authentication entry point and access denied page. It also configures the request matchers that regulate access to resources.

Example

Below is an example of a configuration of the ResourceServerSecurityConfigurer: 

          
           
@EnableWebSecurity
public class OAuth2ResourceServerSecurityConfig extends WebSecurityConfigurerAdapter {

  @Override
  protected void configure(HttpSecurity http) throws Exception {
      http
        .authorizeRequests()
        .anyRequest().authenticated()
        .and()
        .oauth2ResourceServer()
          .resourceId("resource-server-1")
          .tokenServices(tokenService())
        .and()
          .authenticationEntryPoint(new CustomAuthenticationEntryPoint());
  }

  @Bean
  public ResourceServerTokenServices tokenService() {
     return new RemoteTokenServices();
  }

}
In the example, the apply() method is used to set the authentication manager (which defaults to the global authentication manager if none is specified), authentication entry point and access denied page. The authentication entry point is used when authentication fails as part of the OAuth2 protocol, while the access denied page is invoked when access to a resource is denied. The resourceId() method can be used to set the id of the resource server. This is necessary in order to ensure that authorization requests are routed to the correct resource server. The tokenServices() method can be used to configure a ResourceServerTokenServices instance. This is necessary in order to access the token information required to validate authorization requests. Finally, the requestMatchers() method can be used to specify which requests should be processed by the resource server. This allows for more granular control over which requests should be validated against the OAuth 2.0 protocol.

Conclusion

The ResourceServerSecurityConfigurer provides a convenient way to configure the security for an OAuth 2.0 resource server.
