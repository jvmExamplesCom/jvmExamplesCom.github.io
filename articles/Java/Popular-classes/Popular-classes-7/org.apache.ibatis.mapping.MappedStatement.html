org.apache.ibatis.mapping.MappedStatement

The MappedStatement interface is a core of MyBatis and its main purpose is to hold the SQL that was parsed from the mapper XML files or annotation methods. It also holds additional information such as the type of statement, whether it is anupdate, select etc; the parameters needed to invoke the statement; the cache used to store the statement; the timeout settings and so forth.

Creating a MappedStatement Instance

The simplest way to create a MappedStatement instance is to use the SqlSessionFactoryBuilder.build method. This method takes an argument which is an InputStream that contains the XML mapper configuration. Example:


          
           
InputStream inputStream = Resources.getResourceAsStream("mybatis-config.xml");
SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

Once the SqlSessionFactory is created, you can obtain a MappedStatement instance by calling the getMappedStatement() method. In this example, we are obtaining the MappedStatement for the statement named "select * from employees":


          
           
MappedStatement statement = sqlSessionFactory.getMappedStatement("select * from employees");

Accessing the MappedStatement Properties

Once a MappedStatement instance is obtained, you can access its properties using the getters available in the MappedStatement class. For example, to obtain the type of the statement, you can use the getStatementType() method. The returned value will be of type StatementType, which is an enumeration with the following values:


          
           
STATEMENT

CALLABLE

To obtain the parameters required to invoke the statement, you can use the getParameterMap() method. This method returns a Map object where the key is of type String and the value is of type ParameterMapping. The ParameterMapping object contains information about the parameter such as its type, name and so forth.

To obtain the timeout setting for the statement, you can use the getTimeout() method. This method returns an int value representing the number of seconds to wait for the statement to execute before throwing a timeout exception.

Executing the MappedStatement

Once you have obtained the MappedStatement instance, you can execute it using the SqlSession.execute() method. Example:


          
           
SqlSession sqlSession = sqlSessionFactory.openSession()
Object result = sqlSession.execute(statement, parameterMap);

The first argument to the execute() method is the MappedStatement instance. The second argument is the parameter map which contains the parameters required to invoke the statement. The return type of this method depends on the type of statement being executed.

Caching the MappedStatement

MyBatis supports caching of MappedStatement instances. This means that if there multiple calls to the same statement, instead of parsing the statement from the XML every time, it will be retrieved from the cache. To enable caching, you can call the setCache() method on the MappedStatement object. Example:


          
           
statement.setCache(true);

This will enable caching for this statement and the cached version will be used when the statement is invoked multiple times.

Conclusion

In this article, we looked at the MappedStatement interface, how to create and access its properties, and how to execute the statement and enable caching.
