What is Java.util.concurrent.CompletionService?

Java.util.concurrent.CompletionService is an interface in Java that allows a concurrent applications to track the progress and results of asynchronous tasks, as well as provide a mechanism for retrieving such results if they are available already. This interface provides the ability to submit Callable objects or Runnable objects, and to receive a Future object representing the completion status of the task.

Creating an instance of CompletionService

In order to use an implementation of the CompletionService interface, you must first create an instance of it. This is done by calling the create() method of a class implementing the CompletionService interface, for example Executors.newFixedThreadPool():


          
           
ExecutorService executorService = Executors.newFixedThreadPool(2);
CompletionService<String> completionService = new ExecutorCompletionService<>(executorService);

Writing Tasks to be Executed by CompletionService

Once an instance of the CompletionService is created, tasks can be written to be executed. The CompletionService can execute both Callable and Runnable tasks. To submit a Callable task, the submit()method of the CompletionService is used:


          
           
Future<String> future = completionService.submit(() -> {
    Thread.sleep(1000);
    return "Done";
});

To submit a Runnable task, the same submit() method of the CompletionService is used, but the call() method of the Runnableobject itself must return a value, for example:

    Thread.sleep(1000);
}, "Done");

Getting Results from CompletionService

Once tasks have been submitted to the CompletionService, the results can be obtained. The take()method of the CompletionService is used to retrieve a Future object representing the result of the first task that has completed:


          
           
Future<String> future = completionService.take();

The Future object obtained from the CompletionService can then be used to retrieve the result of the task using the get()method:
