org.eclipse.ui.IEditorPart

The 

          
           
org.eclipse.ui.IEditorPart
interface provides the foundational elements for all Eclipse platform editors. It is an abstraction for the Eclipse workbench that allows developers to create, open, and interact with any kind of file within the Eclipse platform. An editor part is the base class for all graphical editor implementations in Eclipse. All these editor parts are displayed within an “Editor” page in the workbench's window, which consists of an editor area and three main toolbars (the Button, Tab, and Toolbar).

Methods

The 

          
           
org.eclipse.ui.IEditorPart
interface provides a number of methods that are used to perform various tasks related to editing files. Some of these methods are listed below: 
  • createPartControl()
    - This method creates the user interface for the editor. It is called after the editor part is instantiated, but before it is attached to the workbench window.
  • getEditorInput()
    - This method returns an object representing the input for the editor. It is used to obtain information about the file that is being edited.
  • getTitle()
    - This method returns the title of the editor part. It is used to set the tab title of the page in the workbench window.
  • 
            
             
doSave(IProgressMonitor monitor)
    - This method saves the contents of the editor to the file. It is called by the workbench when the user selects the “Save” action.
  • setFocus()
    - This method sets the focus on the editor part. It is used by the workbench to make sure that the editor part has the focus when the window is activated.
