org.eclipse.jetty.websocket.api.Session

The org.eclipse.jetty.websocket.api.Session class provides a web socket session that can be used to send and receive data on a web socket.

Creating a Session

The Session object is created using the WebSocketSessionFactory and passing in a URI object which contains the address you wish to connect to:


          
           
URI uri = new URI("ws://localhost:8080/myapp");
WebSocketSessionFactory factory = new WebSocketSessionFactory();
Session session = factory.newSession(uri);
session.connect();

The Session object is then used to send and receive data from the server. It is recommended that you use the asynchronous API provided by the session for sending and receiving data. The examples below show how to use the asynchronous API for both sending and receiving data.

Sending Data Asynchronously

The sendText() method is used to send text data asynchronously. The method takes a string as input. The method is called with a completion handler as an argument. The completion handler is called when the data has been sent successfully:


          
           
String data = "This is some data";
CompletionHandler<Void> handler = new CompletionHandler<Void>() {
    public void onComplete(Void result) {
        System.out.println("Data sent!");
    }
};
session.sendText(data, handler);

The same mechanism can be used to send binary data. The sendBinary() method is used to send binary data asynchronously:


          
           
byte[] data = getData();
CompletionHandler<Void> handler = new CompletionHandler<Void>() {
    public void onComplete(Void result) {
        System.out.println("Data sent!");
    }
};
session.sendBinary(data, handler);

Receiving Data Asynchronously

The receiveText() and receiveBinary() methods are used to receive data asynchronously. The methods take a completion handler as an argument. The completion handler is invoked when the data is received. The data received is passed as an argument to the completion handler:


          
           
CompletionHandler<String> textHandler = new CompletionHandler<String>() {
    public void onComplete(String result) {
        System.out.println("Received: " + result);
    }
};
session.receiveText(textHandler);

CompletionHandler<byte[]> binaryHandler = new CompletionHandler<byte[]>() {
    public void onComplete(byte[] result) {
        System.out.println("Received binary data of length " + result.length);
    }
};
session.receiveBinary(binaryHandler);

It is important to note that the completion handlers are invoked once for each message received. If multiple messages are sent from the server, the completion handlers will be invoked for each message separately.

Closing the Session

The session can be closed at any time by calling the close() method on the session object. The close() method takes a closure code and a message as parameters:


          
           
session.close(1000, "Bye!");

The closure code and message are sent to the server indicating why the session is being closed. The server may also send a closure code and message when it closes the session.
