com.netflix.hystrix.contrib.javanica.annotation.HystrixCommand

The 
HystrixCommand
annotation is a way to wrap code in a Hystrix command, making it easier to use with Hystrix and enabling the application to more easily handle errors, timeouts, and thread pool rejection. Using this annotation can help developers take advantage of Hystrix's features, such as circuit-breakers, thread-pools, and fallback methods.

Annotation Definition

The 
HystrixCommand
annotation is defined as follows: 

          
           
@Target({ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@Documented
public @interface HystrixCommand {
    String groupKey() default "";
    String commandKey() default "";
    String threadPoolKey() default "";
    int threadPoolProperties() default -1;
    int executionTimeoutInMilliseconds() default -1;
    boolean requestCacheEnabled() default true;
    boolean ignoreExceptions() default false;
}

Usage

The 
HystrixCommand
annotation can be used to wrap code in a Hystrix command to make it easier to use with Hystrix, like so: 

          
           
@HystrixCommand(groupKey = "myGroup", commandKey = "myCommand")
public void myMethod() {
    // My method code
}

Attributes

The following attributes can be used to customize the command annotation:

  • groupKey
    – Sets the Hystrix command group key for the command.
  • commandKey
    – Sets the Hystrix command key for the command. This is used to identify the command, so that it can be monitored or configured in Hystrix.
  • threadPoolKey
    – Sets the Hystrix thread pool key for the command, which is used to map the command to a specific thread pool.
  • 
            
             
threadPoolProperties
    – Sets the Hystrix thread pool properties for the command. This can be used to control the size of the thread pool, the maximum queue size, and other parameters.
  • 
            
             
executionTimeoutInMilliseconds
    – Sets the amount of time, in milliseconds, that Hystrix will wait for the command to execute before timing out.
  • requestCacheEnabled
    – Determines whether Hystrix should cache successful requests or not.
  • ignoreExceptions
    – Determines whether Hystrix should ignore certain exceptions from the command.

HystrixCommand
