org.gradle.api.DefaultTask

The org.gradle.api.DefaultTask class is the default task implementation in Gradle. It provides an easy way to write tasks— the user only needs to override the execute() method. This class implements the Task interface, which is a marker interface for Gradle tasks.

Example Usage:


          
           
public class MyTask extends DefaultTask {

    @TaskAction
    public void doStuff() {
        // do something
    }

}

In this example, we create a custom task, called “MyTask” by extending the 

DefaultTask
class. We then define a 
@TaskAction
annotation to indicate the method that will be executed when the task is run. This is the equivalent of overriding the 
execute()
method in the 
DefaultTask
class.

The 
configure
closure is used to configure the 
DefaultTask
. It allows you to set various properties on the task, such as its name and description, and any additional configuration options you need.

    // set task properties
    name = 'myTask'
    description = 'This is myTask'

    // set additional configuration options

    }
}
