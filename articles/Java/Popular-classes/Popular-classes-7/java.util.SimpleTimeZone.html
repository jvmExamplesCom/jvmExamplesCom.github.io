java.util.SimpleTimeZone

Java provides the SimpleTimeZone class which helps to represent a time zone offset and rules for daylight savings time. A SimpleTimeZone object is created with its constructors by passing an integer offset value from GMT and optionally a daylight savings time parameters.

Constructors


          
           
//Constructs <br>
SimpleTimeZone(int rawOffset, String ID) 

// Constructs with specific rules  <br>
SimpleTimeZone(int rawOffset, String ID, 
 int startMonth, int startDay, int startDayOfWeek,
 int startTime, int endMonth, int endDay,
 int endDayOfWeek, int endTime)

// Constructs with specific rules and custom DST savings <br>
SimpleTimeZone(int rawOffset, String ID, 
int startMonth, int startDay, int startDayOfWeek,
int startTime, int endMonth, int endDay, int endDayOfWeek, 
int endTime, int dstSavings)

Example

Let’s create a SimpleTimeZone object for EST as per Indian Standard Time (IST). 

          
           
SimpleTimeZone est = new SimpleTimeZone(+5 * 60 * 60 * 1000, "EST");
The raw offset is +5 hours from GMT while the optional ID is “EST” here.

Methods

The SimpleTimeZone class provides some useful methods related to the time zone created. 

          
           
boolean hasSameRules(TimeZone other) 

int getDSTSavings() 

int getOffset(long date) 

int getRawOffset()

Example

Let’s use some of the methods from the SimpleTimeZone class. 

          
           
// let's get the offset from GMT at given time 
int offset = est.getOffset(System.currentTimeMillis());

// print the output 
System.out.println("Offset from GMT is: " + offset);

Conclusion

In conclusion, the java.util.SimpleTimeZone class provides a simple way to represent a time zone offset and rules for daylight savings time in Java programs. It creates a SimpleTimeZone object with one of its constructors and offers useful methods like getOffset(), getDSTSavings(), etc.
