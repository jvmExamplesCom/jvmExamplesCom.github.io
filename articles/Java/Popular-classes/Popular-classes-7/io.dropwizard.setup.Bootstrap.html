Java io.dropwizard.setup.Bootstrap: An In-Depth Guide

Dropwizard is a Java framework for developing RESTful web services that enables rapid development and deployment of high-performance web applications. Its core features include an embedded Jetty web server, a configuration system, and various utilities for data access, health checks, and logging. It also provides a set of libraries called "Dropwizard Bundles" that make it easy to add new features, such as authentication and validation. The io.dropwizard.setup.Bootstrap class is the main entry point into Dropwizard. It is responsible for initializing the application, setting up the environment, and loading configuration files. In this tutorial, we'll take a look at how to use Bootstrap and how to handle configuration files.

Creating an Application

Before we can use Bootstrap, we must first create an application. We do this by writing a subclass of the abstract class 

          
           
io.dropwizard.Application
. In our subclass, we define the following: 

          
           
public class MyApplication extends Application<MyConfiguration> {

    @Override
    public void run(MyConfiguration configuration, Environment environment) {
        // set up resources, health checks, etc.
    }
}
The 

          
           
run()
method of our subclass is the entry point into our application. Here, we configure our resources, health checks, and other features. For example, we might register a servlet with the environment, or set up a database connection.

Configuring the Application

To configure the application, we need to create a configuration file. This file can be written in either YAML or JSON format, and typically contains all the settings necessary to run the application. For example, our configuration file might look like this: 

          
           
server:
    port: 8080
    adminPort: 8082

logging:
    level: INFO
The configuration file is loaded by Bootstrap in the 

          
           
run()
method of our application subclass. Inside the 

          
           
run()
method, we create an instance of Bootstrap and call its 

          
           
configure()
method: 

          
           
public class MyApplication extends Application<MyConfiguration> {
    @Override
    public void run(MyConfiguration configuration, Environment environment) {
        final Bootstrap<MyConfiguration> bootstrap = new Bootstrap<>(this);
        bootstrap.configure(configuration);
        // set up resources, health checks, etc.
    }
}
The 

          
           
configure()
method takes the configuration file as a parameter and parses it using the Jackson library. Any errors encountered in the configuration file will result in an Exception being thrown. Once the configuration file has been successfully parsed, Bootstrap uses it to configure the application. For example, it sets up the embedded Jetty server, configures logging, and sets up other components, such as health checks and Jersey resources.

Adding Commands

Dropwizard also allows us to add commands to our application. Commands are executable tasks that can be run from the command line. For example, we could add a command to print out the application's version number, or to migrate the database. To add commands, we call the 

          
           
addCommand()

io.dropwizard.cli.Command

VersionCommand
class.
