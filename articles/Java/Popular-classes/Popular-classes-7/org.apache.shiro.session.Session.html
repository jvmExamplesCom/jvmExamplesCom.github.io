org.apache.shiro.session.Session

The org.apache.shiro.session.Session class provides a mechanism through which Shiro applications can access and interact with the underlying user session. This class provides the capabilities necessary to represent and manage the user's current session state, as well as provide security via authentication, authorization, and encryption.

In order to use the Session class, a user must first create an instance of it via one of the available constructors. Depending on how the user defines the constructor parameters, different levels of security can be established for the associated session.

Methods

The Session class contains several methods that allow the user to access or manipulate its properties. Here is an overview of the most commonly used methods:

  • setAttribute() - Sets the value of an attribute in the session.
  • getAttribute() - Retrieves the value of an attribute from the session.
  • removeAttribute() - Removes an attribute from the session.
  • isAuthenticated() - Returns whether the user is currently authenticated.
  • authorize() - Performs an authorization check against a given permission.
  • encrypt() - Encrypts a string using the provided key.
  • decrypt() - Decrypts a string using the provided key.

Example Usage

String attributeValue = "bar";
session.setAttribute(attributeName, attributeValue);

//Retrieve the attribute from the session
