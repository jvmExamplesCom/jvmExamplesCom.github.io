Introduction to org.openjdk.jmh.annotations.Level

The org.openjdk.jmh.annotations.Level annotation is used in the JMH project, an open-source micro-benchmarking tool for Java applications. The Level annotation is used to determine how deeply the benchmark should execute: a shallow execution will execute the code only once, while a deep execution will trace through the code several times to get more accurate results. In this article, we will explore the various Levels available in JMH and how they can be used to get the best results from your micro-benchmarks.

Levels Overview

The Level annotation is used to denote the depth to which the benchmark should be executed. The following Levels are available in JMH: - Level.INVOKE: This level will only execute the benchmarked unit once. - Level.ITERATION: This level will execute the benchmarked unit multiple times, up to the specified iteration count. - Level.SURFACE: This level will execute the benchmarked unit multiple times, including deeper calls and branches in the code. - Level.BENCHMARK: This level will execute the benchmarked unit multiple times, including multiple instances of the same code.
