org.springframework.scheduling.quartz.SchedulerFactoryBean

SchedulerFactoryBean is an important class used in the task scheduling framework of the Spring framework. It is used to create and configure an instance of Quartz Scheduler, which is responsible for running jobs at specified times. In this article, we will discuss how SchedulerFactoryBean is used to schedule jobs in a Spring-based application.

How to Use SchedulerFactoryBean

To use SchedulerFactoryBean, add it to your application context like so:


          
           
<bean id="schedulerFactory"
      class="org.springframework.scheduling.quartz.SchedulerFactoryBean">
   ...
</bean>

You can then use the SchedulerFactoryBean to specify the jobs that should be scheduled:


          
           
schedulerFactory.setJobs(myJobs);

The myJobs list is a list of JobDetail objects, each representing a job to be executed by Quartz. You can create a JobDetail using the JobBuilder method:


          
           
JobDetail jobDetail = JobBuilder.newJob(MyJob.class).build();

Where MyJob is the class representing the job you want to execute. This class must implement the org.quartz.Job interface.

You also need to specify a trigger for each job. A trigger defines when the job should be executed. For example, you can create a simple cron trigger like this:


          
           
Trigger cronTrigger = TriggerBuilder
    .newTrigger()
    .withIdentity("myTrigger")
    .withSchedule(
        CronScheduleBuilder.cronSchedule("0 0/5 * * * ?")
    )
    .build();

This trigger will execute the job every five minutes. You can then add the trigger to the SchedulerFactoryBean:


          
           
schedulerFactory.setTriggers(cronTrigger);
Lastly, you need to start the scheduler: 

          
           
schedulerFactory.start();

That's all there is to it! As long as the scheduler is running, jobs will be executed according to their triggers.

Using Annotations

You can also use annotations to specify jobs and triggers. This can help make your code more maintainable and less error-prone. To use annotations, define a class and annotate the methods with @Scheduled :


          
           
@Component
public class MyScheduler {

    private final Logger logger = LoggerFactory.getLogger(this.getClass());

    @Scheduled(
        cron="0 0/5 * * * ?",
        zone="America/New_York"
    )
    public void runTask() {
        logger.info("Running scheduled task!");
    }
}

The @Scheduled annotation takes a cron expression and a time zone qualifier. In this case, the task will run every five minutes in the Eastern Time Zone.

You can also use the @Schedules annotation to specify multiple triggers for a single job. For example:


          
           
@Schedules({
    @Scheduled(
        cron="0 0/5 * * * ?",
        zone="America/New_York"
    ),
    @Scheduled(
        cron="0 0/15 * * * ?",
        zone="America/Chicago"
    )
})
public void runTask() {
    logger.info("Running scheduled task!");
}

In this case, the task will run every five minutes in the Eastern Time Zone and every fifteen minutes in the Central Time Zone.

Conclusion

In this article, we discussed the SchedulerFactoryBean class from the Spring framework and how it can be used to schedule jobs in a Spring-based application. We looked at both the programmatic and annotation-based approaches for scheduling jobs, as well as how to create triggers for those jobs. Understanding how to use SchedulerFactoryBean is essential for any Spring developer looking to incorporate task scheduling into their applications.
