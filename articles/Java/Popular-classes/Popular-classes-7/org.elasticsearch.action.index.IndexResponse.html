Introduction to org.elasticsearch.action.index.IndexResponse

The org.elasticsearch.action.index.IndexResponse class provides an API for creating, updating and deleting documents in an Elasticsearch index. It is part of the org.elasticsearch package and it is used to index documents in an elasticsearch cluster.

Creating IndexDocuments

Creating a document in an elasticsearch index is done with the IndexRequest class. An IndexRequest takes three parameters: the index name, the document ID, and the document content.

The following example indexes a document in the "myindex" index with an ID of 1, and the contents of the document in the form of a JSON object.


          
           
IndexRequest request = new IndexRequest("myindex", "1")
    .source(jsonBuilder()
        .startObject()
            .field("name", "John Doe")
            .field("age", 35)
        .endObject());

The IndexRequest can also be configured to specify whether or not it should overwrite an existing document if one exists with the same ID. This can be set with the setIfExists() method:


          
           
request.setIfExists( <b>true</b> );

The IndexRequest can then be submitted to the Elasticsearch cluster with the IndexResponse.execute() method.


          
           
IndexResponse response = client.index(request).get();

Updating IndexDocuments

Updating a document in an elasticsearch index is done with the UpdateRequest class. An UpdateRequest takes three parameters: the index name, the document ID, and the document content.

UpdateRequest request = new UpdateRequest("myindex", "1")

request.setIfNotExists( <b>true</b> );

UpdateResponse response = client.update(request).get();

Deleting IndexDocuments

Deleting a document from an elasticsearch index is done with the DeleteRequest class. A DeleteRequest takes two parameters: the index name and the document ID.
