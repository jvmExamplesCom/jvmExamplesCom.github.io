javax.websocket.OnOpen

The javax.websocket.OnOpen annotation enables endpoints to register a Java method to be called when a new web socket connection is accepted and opened by the server.

Example Code


          
           
@ServerEndpoint("/endpoint")
public class MySocket {

    @OnOpen
    public void onOpen() {
        System.out.println("WebSocket opened");
    }

}

In the above example, the endpoint is registered with the ServerEndpoint annotation and specifies a specific URI pattern. The onOpen() annotated method is executed every time a connection is opened.

Necessary Objects

The javax.websocket.OnOpen annotation only requires one parameter, which is the reference of the Session object associated with the opened connection.


          
           
@OnOpen
public void onOpen(Session session) {...}

Methods

The OnOpen annotation can be used to invoke any method that is defined in your WebSocket endpoint. It is important to note that the parameters and return type of the method must match the specified guidelines.

Parameter Constraints

The parameter of the method must be of type javax.websocket.Session.


          
           
@OnOpen 
public void onOpen(Session session) {...}

Return Type Constraints

The return type of the method must be void or an instance of BoxedPrimitives or java.lang.String.


          
           
@OnOpen
public void onOpen() {...}

@OnOpen
public String onOpen() {...}

@OnOpen
public Integer onOpen() {...}

Throws Constraint

The method may throw any un-checked exception but must not throw any checked exceptions.


          
           
@OnOpen
public void onOpen() throws IOException {...} // Not allowed
