org.openqa.selenium.TakesScreenshot

org.openqa.selenium.TakesScreenshot is an interface provided by Selenium WebDriver API to capture screenshot of the current web page. It is a very useful interface that helps users to take screenshots while testing the application.

Creating an Instance of TakesScreenshot

TakesScreenshot interface can be instantiated by using the following code:


          
           
WebDriver driver = new FirefoxDriver();
TakesScreenshot ts = (TakesScreenshot) driver;

Capture Screenshot

The 
getScreenshotAs()
method of TakesScreenshot class can be used to capture the screenshot. It takes two parameters, the target type and the file path. The target type can be one of the 
OutputType
enum values. It supports the 
FILE
, 
BASE64
and 
BYTES
output types.

The following code will capture the screenshot in the specified file path:


          
           
File srcFile = ts.getScreenshotAs(OutputType.FILE); 
FileUtils.copyFile(srcFile, new File("C://screenshots//screenshot.jpg"));

Capture Screenshot as Base64 String

The 
getScreenshotAs()
method can also be used to get the screenshot in the form of a Base64 string. The following code will return the screenshot as a Base64 string: 

          
           
String screenshotBase64 = ts.getScreenshotAs(OutputType.BASE64);

Capture Screenshot as Bytes Array

The 
getScreenshotAs()
method can also be used to get the screenshot in the form of a bytes array. The following code will return the screenshot as a bytes array: 

          
           
byte[] screenshotBytes = ts.getScreenshotAs(OutputType.BYTES);
