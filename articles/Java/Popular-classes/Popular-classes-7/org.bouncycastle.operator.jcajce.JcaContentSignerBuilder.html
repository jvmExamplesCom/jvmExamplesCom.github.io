JcaContentSignerBuilder Overview

The org.bouncycastle.operator.jcajce.JcaContentSignerBuilder is an API from the Bouncy Castle Java Cryptography API (JCA) that helps developers to create content signers based on the JCA / JCE specifications (Java Cryptography Extension). Content signers are used to sign arbitrary data and provide a secure way to verify the authenticity of the data contents. This API provides methods which can be used to create signer objects and also provides a variety of other functions related to content signing.

Creating Signer Objects

A signer object is created by passing the parameters to the 


          
           
JcaContentSignerBuilder.build(parameters)
method. The parameters are passed as a JcaSignerInfoGeneratorParameters object which contains the signer's asymmetric key, digest algorithm name, and encryption algorithm name (optional). The code example below demonstrates how to create a signer object with the RSA algorithm and SHA-256 digest algorithm: 

          
           
JcaSignerInfoGeneratorParameters parameters = 
    new JcaSignerInfoGeneratorParameters.Builder(new JcaDigestCalculatorProviderBuilder().build())
    .setKey(publicKey)
    .setAlgorithmName("SHA-256withRSA")
    .build();
JcaContentSignerBuilder builder = new JcaContentSignerBuilder("SHA-256withRSA");
ContentSigner signer = builder.build(parameters);

Signing Content

Once the signer object is created, it can be used to sign arbitrary content. The 


          
           
signer.getOutputStream()

ByteArrayOutputStream baos = new ByteArrayOutputStream();

os.close();
byte[] signature = baos.toByteArray();

Verifying Signed Content

The signature created in the previous step can be used to verify the authenticity of the data contents. The 


          
           
verifier.verify(signature)
