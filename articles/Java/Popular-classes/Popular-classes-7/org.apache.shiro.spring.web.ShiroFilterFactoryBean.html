org.apache.shiro.spring.web.ShiroFilterFactoryBean

ShiroFilterFactoryBean is an utility class provided by Apache Shiro, a Java security framework. It allows applications to quickly and easily integrate Shiro security into their existing web applications. In this tutorial we will explore how to use ShiroFilterFactoryBean to integrate Shiro security with a Spring MVC application.

Using ShiroFilterFactoryBean

Using ShiroFilterFactoryBean is quite straightforward. First, you need to define the bean in your application context file like this:


          
           
<bean id="shiroFilter" class="org.apache.shiro.spring.web.ShiroFilterFactoryBean">
    <property name="securityManager" ref="securityManager"/>
    <property name="filterChainDefinitions">
        <value>
            /admin/** = authc
            /user/** = authc
            /** = anon
        </value>
    </property>
</bean>

The above configuration defines a filter chain which will intercept all incoming requests starting with “/admin” and “/user” and check for authentication (authc) and for all other requests anonymous access is allowed (anon).

Second, you need to add a reference to the bean in your Spring MVC configuration:


          
           
<bean class="org.springframework.web.servlet.handler.SimpleUrlHandlerMapping">
    <property name="mappings">
        <props>
            <prop key="/*">shiroFilter</prop>
        </props>
    </property>
</bean>

The above configuration tells Spring MVC to use the shiroFilter bean as the handler for all incoming requests.
