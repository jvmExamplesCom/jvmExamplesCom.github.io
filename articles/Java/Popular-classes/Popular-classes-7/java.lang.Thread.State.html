Java.lang.Thread.State: Overview

The Thread.State class in Java is a set of enumerated constants that allow access to the state of a thread in an application. It also provides information about whether or not the thread is blocked or waiting for something to happen. This article will explore the different states available in the Thread.State class and provide examples of code that demonstrate how each state works.

NEW

The state of a thread that has been newly created but has not started executing yet is NEW. The following code shows an example of a thread in this state.


          
           
Thread t = new Thread();
System.out.println(t.getState()); // prints NEW

Once the thread has been started, it enters the

state. In this state, the thread is actively running its code. The following code snippet shows an example of a thread in this state. 

          
           
Thread t = new Thread();
t.start();
System.out.println(t.getState()); // prints RUNNABLE

If the thread is waiting for something to happen, such as when it's waiting for a notification from another thread, it enters the WAITING state. The following code shows an example of a thread that is waiting for a notification.


          
           
Thread t = new Thread();

TIMED_WAITING

If the thread is waiting with a timeout, it enters the TIMED_WAITING state. This state is similar to WAITING, but it will return to the RUNNABLE state after the specified timeout has expired. The following code shows an example of a thread that is waiting with a timeout.


          
           
Thread t = new Thread();
t.start();
synchronized(obj) {
    obj.wait(1000); // wait for 1 second
