android.support.annotation.StringDef

android.support.annotation.StringDef is an annotation type which can be used to designate a field, a parameter or a return value to that accepts or returns only one or few specific String constants. This prevents us from passing any other type of values to the API resulting in better type safety and improved code readability.

Usage

To use the annotation type, we need to specify the valid string constants using the syntax @StringDef({CONST1, CONST2, …}).

Example 1

Below we will look at an example where we are defining a StringDef to accept either “Hello” or “Goodbye” as valid parameters. 

          
           
@StringDef({
        HELLO, GOODBYE
})
@Retention(RetentionPolicy.SOURCE)
public @interface Greeting {

}

public class MyClass {
    public static final String HELLO = "hello";
    public static final String GOODBYE = "goodbye";

    public void greeting(@Greeting String str1) {
        // do something
    }
}
In few words, by using the @StringDef we are ensuring that when calling the function “greeting()” we must pass it only one of the strings specified in the @Greeting annotation type. So, calling the function with any other string as argument results in a compile time error.

Example 2

Here, we define a StringDef to set the gender of a person to either “male” or “female”. 

          
           
@Retention(RetentionPolicy.SOURCE)
@StringDef({MALE, FEMALE})
public @interface Gender {

}

public class Person {
    public static final String MALE = "male";
    public static final String FEMALE = "female";

    private @Gender String gender;

    public Person(@Gender String gender) {
        this.gender = gender;
    }

    public void setGender(@Gender String gender) {
        this.gender = gender;
    }
}
Again, this helps us to ensure that the field “gender” of the class “Person” accepts only the values “male” or “female” through parameters or method calls.

Conclusion

The annotation type android.support.annotation.StringDef allows us to restrict the parameters of a certain API to set of valid values by finding out invalid ones during compilation phase. This not only improves code readability but also makes type checking easier.
