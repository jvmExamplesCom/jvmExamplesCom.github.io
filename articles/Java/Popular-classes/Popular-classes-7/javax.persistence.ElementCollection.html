Introduction to javax.persistence.ElementCollection

javax.persistence.ElementCollection is a Java Persistence API (JPA) annotation used to denote a collection of basic type or embeddable elements in an entity. Using this annotation, we can map a java.util.Collection of basic or embeddable type field in the class hierarchy.

Annotation Details

The ElementCollection annotation has the following parameters:

  • targetClass: specifies the target class of the collection, if it is an interface or an abstract class. This parameter is not required if it is a concrete class.
  • fetch: specifies the fetching strategy to be used when loading the data from the database.
  • mappedBy: specifies the relationship field that is used to map the collection. This parameter is used only when the collection is part of a one-to-many or many-to-many relationship.
  • orphanRemoval: specifies whether or not the associated entities should be removed when the parent entity is removed.

Example

Let's take a look at an example of how the ElementCollection annotation can be used. Consider the following class hierarchy:


          
           
@Entity
public class Customer {
  @Id
  private Long id;
  private String name;

  @ElementCollection
  private Set<String> tags;
  // getters &amp; setters not shown
}

In this example, we have an Entity class named Customer which contains a Set field named tags. The tags field is annotated with the @ElementCollection annotation, which tells the JPA provider that the tags field should be mapped as a collection of basic type values.
