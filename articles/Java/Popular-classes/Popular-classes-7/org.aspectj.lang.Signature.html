org.aspectj.lang.Signature

The org.aspectj.lang.Signature class represents a method signature in the AspectJ programming language. It is used to represent a method inherited from a superclass or implemented in an interface. It is often used as an argument to abstract methods that are applicable to any type of method, such as advice execution or pointcut matching.

Overview

A Signature object holds useful information about a method, including its declaring type, its name, its modifiers, its parameter types, its return type, and its exception types. This can be used for various purposes, such as logging and debugging methods, matching join points to a pointcut, or providing method information in message displays. It is also used internally by the AspectJ compiler to provide source line mapping information.

Constructors

The class has two constructors, one for a constructor, and one for a method: 

          
           
Signature(Constructor constructor)
Signature(Method method)
The constructor takes an instance of either the java.lang.reflect.Constructor or a java.lang.reflect.Method class and creates a signature object with the appropriate information.

Methods

The class provides several utility methods to help work with signatures: 

          
           
getDeclaringType(): Returns the fully-qualified name of the type which declared this signature
getName(): Returns the name of the method
getModifiers(): Returns the modifiers of the method
getParameterTypes(): Returns an array of the parameter types
getReturnType(): Returns the return type of the method
getExceptionTypes(): Returns the array of exception types thrown by the method
getDeclaringTypeName(): Returns the fully-qualified name of the declaring type
It also provides several methods for matching join points. These include: 

          
           
matches(String pattern): Checks to see if the signature matches the specified pattern
matches(Signature other): Checks to see if this signature matches another signature
The pattern is a regular expression which is used to match the signature. The other signature must have the same declaring type and name as this one.

Example

The following example shows how to use the org.aspectj.lang.Signature class to get information about a method and match it to a particular pointcut. 

          
           
import org.aspectj.lang.Signature;

public class SignatureExample {
    public void someMethod() {}

    public static void main(String[] args) {
        Signature s = new Signature(SignatureExample.class.getDeclaredMethod("someMethod"));
        System.out.println("Declaring Type: " + s.getDeclaringType());
        System.out.println("Name: " + s.getName());
        System.out.println("Modifiers: " + s.getModifiers());
        System.out.println("Parameter Types: " + s.getParameterTypes());
        System.out.println("Return Type: " + s.getReturnType());
        System.out.println("Exception Types: " + s.getExceptionTypes());

        if (s.matches("*.someMethod()")) {
            System.out.println("Signature matched pattern");
        }
    }
}
