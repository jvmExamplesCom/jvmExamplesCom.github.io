com.intellij.psi.PsiManager Overview

The 

          
           
com.intellij.psi.PsiManager
class provides access to the PSI (Program Structure Interface) for the IntelliJ IDEA platform. The PsiManager is used to manage and query the PSI structure and provides access to elements such as classes, variables, methods, and other PSI related information. It is used by the IntelliJ IDEA IDE to provide code completion, refactoring, and other functionality. The PsiManager is a singleton and can be accessed from anywhere in your code.

Creating a PsiManager Instance

To create a new instance of the 
PsiManager
, use the following Java code: 

          
           
PsiManager psiManager = PsiManager.getInstance(myProject);
In the above example, we are passing the 
myProject
variable to the getInstance method. This variable should be an instance of the 
Project
class. You can obtain a reference to the current project using the following code: 

          
           
Project myProject = ProjectManager.getInstance().getDefaultProject();

Methods

The 
PsiManager
provides many useful methods for querying and managing the PSI structure. Some of the more commonly used methods are listed below: 
  • findClass()
    : This method is used to find a class in the current project based on its fully qualified name.
  • findFilesByName()
    : This method is used to find files by their name. It can accept wildcards as well.
  • findPackage()
    : This method is used to find a package by its name.
  • 
            
             
getModificationTracker()
    : This method is used to track changes in the PSI structure.

For a complete list of methods and their descriptions, refer to the official IntelliJ documentation.

Examples

Let's take a look at some code examples to see how the 

PsiManager
class can be used. The first example shows how to find a class in the current project: 

          
           
PsiClass aClass = psiManager.findClass("com.example.MyClass", GlobalSearchScope.allScope(project));

In this example, we pass the fully qualified class name and the scope of the search to the 

findClass
method. The scope can be either a global search scope or a specific one. The above example will search through the entire project. The next example shows how to use the 
findFilesByName
method to find all files that match a given name pattern: 

          
           
PsiFile[] files = psiManager.findFilesByName("*.txt", GlobalSearchScope.allScope(project));

This example searches the entire project for all files with the 

.txt
extension. We can also restrict the search scope to a specific directory or package if needed. Finally, let's take a look at how to use the 

          
           
getModificationTracker
method. This method returns a 

          
           
PsiModificationTracker
object which can be used to track modifications in the PSI structure.: 

          
           
PsiModificationTracker tracker = psiManager.getModificationTracker();
The 

          
           
PsiModificationTracker
object provides methods to check if the PSI structure has been modified since a certain point in time. For example, we can use the 

          
           
hasOutOfCodeBlockModifications()
method to check if the PSI structure has been modified since a certain time: 

          
           
boolean modified = tracker.hasOutOfCodeBlockModifications(System.currentTimeMillis());

If the PSI structure has been modified since the given timestamp, the 

modified
variable will be set to 
true
; otherwise it will be set to 
false
.
