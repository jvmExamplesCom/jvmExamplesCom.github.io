Introduction to org.openjdk.jmh.annotations.Param

org.openjdk.jmh.annotations.Param is a Java annotation used to control the execution of parameterised JMH (Java Microbenchmark Harness) benchmarks. A parameterised benchmark is a benchmark that is invoked multiple times with different parameters and then their performance is compared across the different execution scenarios. Each invocation of the parameterised benchmark will have one set of the parameters that are defined in the @Param annotation.

Annotation Overview

The org.openjdk.jmh.annotations.Param annotation takes two arguments, both strings. The first argument is the name of the parameter and the second is the value of the parameter. For example,


          
           
@Param(name="numIterations", value="1000")

This annotation will set the value of the parameter "numIterations" to 1000.

Using Multiple Parameters

Multiple parameters can be used in a parameterised JMH benchmark by using multiple @Param annotations and each parameter will have a different value for each iteration of the benchmark. For example,


          
           
@Param(name="numIterations1", value="1000")
@Param(name="numIterations2", value="2000")

This will run the parameterised benchmark twice, once with numIterations1 set to 1000 and numIterations2 set to 2000 and again with numIterations1 set to 2000 and numIterations2 set to 1000.

Limitations of the Parameter List Form
