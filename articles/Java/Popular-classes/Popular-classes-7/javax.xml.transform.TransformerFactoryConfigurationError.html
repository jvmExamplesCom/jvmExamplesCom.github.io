javax.xml.transform.TransformerFactoryConfigurationError

Introduction

The 

          
           
javax.xml.transform.TransformerFactoryConfigurationError
is thrown when an 

          
           
Error
occurs during a configuration operation on an instance of the 

          
           
TransformerFactory
. The 

          
           
TransformerFactory
class provides a reusable factory for obtaining Transformer objects, based on an underlying implementation. This error indicates that the underlying implementation cannot be instantiated.

Example

Consider the following code example: 

          
           
TransformerFactory tf = TransformerFactory.newInstance();
Transformer transformer = tf.newTransformer();
When this code is executed, it will check the system to see if an implementation of the 

          
           
TransformerFactory
interface is available. If an implementation is not available, a 

          
           
TransformerFactoryConfigurationError
is thrown. This error can be caught and handled as follows: 

          
           
try {
    TransformerFactory tf = TransformerFactory.newInstance();
    Transformer transformer = tf.newTransformer();
} catch (TransformerFactoryConfigurationError e) {
    // Handle the error here
}

Solution

In order to solve this problem, it is necessary to provide a valid implementation of the 

          
           
TransformerFactory
interface in the classpath. JAXP (the Java API for XML Processing) contains an implementation of the 

          
           
TransformerFactory
interface and should be included in the classpath for applications making use of it. Alternatively, an implementation can be provided as part of a specific JDK version, or included within the application itself. Once an implementation is available, the 

          
           
TransformerFactoryConfigurationError
will no longer be thrown, and the application should operate as expected.
