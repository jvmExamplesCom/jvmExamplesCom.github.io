org.springframework.web.servlet.LocaleResolver

The LocaleResolver interface is used by the Spring framework to define a locale for the current request. It provides a way for applications to handle locale changes in a uniform fashion and allows for different strategies to be configured. The class org.springframework.web.servlet.i18n.AcceptHeaderLocaleResolver is a common implementation that simply sets the locale using the Accept-Language header from the HTTP request.

Code Examples

Below are examples of how to use the LocaleResolver interface for various purposes.

Retrieve Current Locale


          
           
Locale locale = LocaleContextHolder.getLocale();

Set Default Locale


          
           
LocaleResolver localeResolver = new AcceptHeaderLocaleResolver();
localeResolver.setDefaultLocale(Locale.US);

Change Locale


          
           
localeResolver.setLocale(request, response, new Locale("es"));

Retrieve User Locale from Cookie


          
           
CookieLocaleResolver cookieLocaleResolver = new CookieLocaleResolver();
cookieLocaleResolver.setCookieName("user-locale");
locale = cookieLocaleResolver.resolveLocale(request);

Get List of Supported Locales


          
           
List<Locale> supportedLocales = new ArrayList<>();
supportedLocales.add(Locale.CANADA);
supportedLocales.add(Locale.US);
supportedLocales.add(new Locale("fr"));
localeResolver.setSupportedLocales(supportedLocales);
