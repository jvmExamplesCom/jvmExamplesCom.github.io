org.apache.http.client.config.CookieSpecs

The org.apache.http.client.config.CookieSpecs class provides static constants used to define cookie policies of the HTTP transport layer of the Apache HttpComponents library. The CookieSpecs class defines several standard cookie policy settings that can be used to configure the cookie store in an HTTP request.

Overview of org.apache.http.client.config.CookieSpecs

The org.apache.http.client.config.CookieSpecs class provides static constants used to define the cookie policy of an HTTP request. The available cookie policies are:
  • BEST_MATCH: A policy that selects the cookie policy that best matches the given request.
  • BROWSER_COMPATIBILITY: A policy that selects the most compatible cookie policy that ensures consideration of cookies sent from all compliant user agents.
  • IGNORE_COOKIES: A policy that ignores all cookies.
  • NETSCAPE_DRAFT: A policy compliant with the original version of the Netscape draft specification.
  • RFC_2109: A policy compliant with RFC 2109.
  • RFC_2965: A policy compliant with RFC 2965.

Using CookieSpecs in Requests

The cookie policy of a request is set using the org.apache.http.client.config.RequestConfig class. The RequestConfig class provides a setCookieSpec method which sets the cookie policy to one of the pre-defined policies provided by the CookieSpecs class, such as BEST_MATCH or RFC_2109. The following example code shows how to set the cookie policy to BEST_MATCH when making an HTTP request: 

          
           
HttpClient httpClient = HttpClientBuilder.create().build();
String url = "http://example.com";

HttpClientRequest request = HttpClientRequest.get(url);

RequestConfig config = RequestConfig.custom()
                .setCookieSpec(CookieSpecs.BEST_MATCH)
                .build();

request.setConfig(config);

HttpResponse response = httpClient.execute(request);
