java.util.concurrent.DelayQueue

What is it?

Java.util.concurrent.DelayQueue is a special type of queue used in Java to store objects which are supposed to be processed after a certain period of time has passed. It is an unbounded queue and its main purpose is to provide thread safety. This queue is quite useful when you need to execute a task after a given amount of time, such as in the case of scheduling jobs.

How do you use it?

To use the DelayQueue you must first create a queue and add your objects to it. These objects must implement the interface java.util.concurrent.Delayed, which has one method, getDelay() that returns a long value representing the remaining delay time in milliseconds: 

          
           
DelayQueue<MyDelayedObject> queue = new DelayQueue<>();

queue.add(object);
Once the objects have been added to the queue, you can retrieve the next element using the take() method. 

          
           
MyDelayedObject object = queue.take();
The take() method will wait until an object has been available to be processed and will then return it.
