Introduction to org.quartz.CronTrigger

CronTrigger is a part of the Quartz Scheduler library, which helps run jobs and processes in Java applications according to a defined schedule. It maps cron expressions to fire times and is represented by an instance of the org.quartz. CronTrigger class. By parsing cron expressions, CronTrigger can determine the time when a job is scheduled to be executed.

Understanding Cron Expressions

Cron expressions are strings used to specify the frequency at which a given job should execute. They are composed of six or seven sub-expressions that are separated by whitespace. For example: 


          
           
0 0 10 * * ? *

This cron expression runs on the 10th of every month at midnight. The sub-expressions are as follows: 


          
           
Seconds: 0
Minutes : 0
Hours : 10
Day-of-Month : *
Month : *
Day-of-Week : ?
Year : *

The asterisk symbol (*) instructs CronTrigger to select all available options for corresponding sub-expressions. The question mark (?) symbol stands for "no specific value", and is used to indicate days of the week or month depending on context. For example, in the above cron expression, the Day-of-Week is set to "no specific value" meaning it will run on any day of the week.

Configuring a CronTrigger

To create a CronTrigger instance, you need to provide a name and group for the trigger, and provide a valid cron expression. The following example creates a CronTrigger with a name and group of 'MyTrigger' and a cron expression of '0 0 10 * * ? *': 


          
           
CronTrigger trigger = new CronTrigger("MyTrigger", 
                                     "MyGroup",
                                     "0 0 10 * * ? *");

Once the trigger is created, you can set its start time, end time, priority, misfire instructions and other settings. You can also add a list of job listeners and set the trigger's job data map. The following example sets the start and end times, priority, and misfire instructions: 


          
           
trigger.setStartTime(new Date());
trigger.setEndTime(new Date(System.currentTimeMillis() + 1000*60*60));
trigger.setPriority(10);
trigger.setMisfireInstruction(CronTrigger.MISFIRE_INSTRUCTION_FIRE_ONCE_NOW);

Finally, when the trigger is configured, you can add it to a Scheduler and start executing your jobs using it. The following example creates a Scheduler and adds a CronTrigger to it:


          
           
Scheduler scheduler = StdSchedulerFactory.getDefaultScheduler();
scheduler.start();
scheduler.scheduleJob(job, trigger);

That's all there is to configuring and using a CronTrigger instance. With this powerful feature, you can easily schedule job executions in a very precise and repeatable way.
