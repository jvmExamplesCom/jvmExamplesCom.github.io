org.slf4j.bridge.SLF4JBridgeHandler

The org.slf4j.bridge.SLF4JBridgeHandler is a Java package that allows users to redirect or bridge logging calls from java.util.logging (JUL) to the Simple Logging Facade for Java (SLF4J). The intention is to allow JUL users to benefit from the logging features and adapters of SLF4J.

Usage Example

In order to use it, first you have to add the SLF4J JAR files to your classpath. Then, you must install the bridge handler which will redirect your JUL logs to SLF4J.


          
           
SLF4JBridgeHandler.install();

This command should be called before any calls to java.util.logging classes.

Configuration Example

In addition, you can use the SLF4JBridgeHandler to configure your JUL environment by setting logging levels and adding handler classes.


          
           
SLF4JBridgeHandler.removeHandlersForRootLogger(); 
SLF4JBridgeHandler.install(); 

Logger rootLogger = Logger.getLogger(""); 
rootLogger.setLevel(Level.INFO); 

Handler consoleHandler = new ConsoleHandler(); 
consoleHandler.setLevel(Level.FINE); 
rootLogger.addHandler(consoleHandler);

The code above will ensure that all logs of INFO level or higher are logged to the console. It will also ensure that only messages of FINE level or higher are printed.

Conclusion

The org.slf4j.bridge.SLF4JBridgeHandler is a useful way to bridge logging calls from JUL to SLF4J. By using this bridge, users of JUL can benefit from the features of SLF4J while still using the same logging framework.
