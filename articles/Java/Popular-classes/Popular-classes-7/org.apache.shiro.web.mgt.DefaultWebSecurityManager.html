Introduction to Apache Shiro DefaultWebSecurityManager

Apache Shiro is an open source framework for Java web application development. It provides authentication, authorization, and roles management with a strong emphasis on security. The DefaultWebSecurityManager is a subclass of the DefaultSecurityManager which is used for web applications. It provides methods and classes for managing users, roles, and permissions in a web application.

Creating a SecurityManager and Initializing it with a Realm

The first step when using the DefaultWebSecurityManager is to create a SecurityManager and initialize it with the realm that contains the user, role, and permission data. The following example creates a IniRealm instance and sets it as the realm for the DefaultWebSecurityManager: 

          
           
IniRealm realm = new IniRealm("classpath:users.ini");
DefaultWebSecurityManager securityManager = new DefaultWebSecurityManager(realm);
The IniRealm takes as a parameter a resource location of a “.ini” file which contains user, role, permission declarations. In the above example, the file “users.ini” must exist in the classpath of the application.

Using the SecurityManager with the Web Environment

The next step is to register the SecurityManager with the web environment. This is done with the “setSecurityManager” method of the WebUtils class. The following example shows how to set the SecurityManager as the current security manager in a web application: 

          
           
WebUtils.setSecurityManager(securityManager);

Authenticating and Authorizing Users

The DefaultWebSecurityManager provides methods for authenticating and authorizing users. The “authenticate” method takes a username and password and returns an authenticated subject if the credentials match a user in the realm. The following example shows how to use the “authenticate” method to authenticate a user: 

          
           
Subject subject = securityManager.authenticate(new UsernamePasswordToken(username, password));
The “isPermitted” method can then be used to check if the authenticated user has access to a certain resource or action. The following example shows how to use the “isPermitted” method to check if a user has permission to view a certain page: 

          
           
boolean permitted = subject.isPermitted("page:view");

Conclusion

The Apache Shiro DefaultWebSecurityManager provides powerful authentication, authorization, and roles management for web applications. It allows developers to easily add secure authentication and authorization to their web applications with a few simple lines of code.
