javax.persistence.UniqueConstraint

The javax.persistence.UniqueConstraint annotation is used to define a unique constraint for an entity in Java Persistence API (JPA). It defines a set of columns which should be unique with respect to each other. This annotation can be applied to a persistent class or property to create a unique constraint for a table or for a column.

Usage

UniqueConstraint works by mapping a single entity field to a single UniqueConstraint annotation. The annotation takes two parameters, the name and the columns, which are optional. The name allows us to reference a unique constraint that was created in the database while the columns parameter defines the columns that should be unique with respect to each other.

Examples

We have an entity called Product which has an id, name and description fields:


          
           
@Entity
public class Product {
  @Id
  private Long id;
  private String name;
  private String description;
// getter and setter methods
}

If we want to ensure the id field is unique in the database, we can apply the UniqueConstraint annotation with the following syntax:


          
           
@Entity
@UniqueConstraint(name = "product_id_unique", columns = "id")
public class Product {
  @Id
  private Long id;
  private String name;
  private String description;
// getter and setter methods
}

Now the database will enforce a unique constraint on the column named “id” in the table associated with the Product entity. We can also specify multiple columns if we want to create a multi-column unique constraint. For example, if we want to ensure that both the name and description fields are unique together in the database, we can use the following syntax:


          
           
@Entity
@UniqueConstraint(name = "product_unique", columns = {"name","description"})
public class Product {
  @Id
  private Long id;
  private String name;
  private String description;
// getter and setter methods
}

The above example will create a unique constraint on the columns named “name” and “description”. It is important to note that a UniqueConstraint annotation will only apply to the table associated with the entity to which it is applied.

It is also possible to apply the UniqueConstraint annotation to individual properties. This can be useful if we want to create a unique constraint on a specific property instead of on the table as a whole. For example, if we want to ensure the name field is unique in the database, we can use the following syntax:


          
           
@Entity
public class Product {
  @Id
  private Long id;
  @UniqueConstraint(name = "product_name_unique", columns = "name")
  private String name;
  private String description;
// getter and setter methods
}

Now the database will enforce a unique constraint on the column named “name” in the table associated with the Product entity. It is important to note that when using the UniqueConstraint annotation on an individual property, the name of the constraint must be unique within the scope of the entity.

Conclusion

In summary, the javax.persistence.UniqueConstraint annotation is used to define a unique constraint for an entity in Java Persistence API (JPA). It can be applied to a persistent class or property to create a unique constraint for a table or for a column. This annotation takes two parameters, the name and the columns, which are optional. The name allows us to reference a unique constraint that was created in the database while the columns parameter defines the columns that should be unique with respect to each other.
