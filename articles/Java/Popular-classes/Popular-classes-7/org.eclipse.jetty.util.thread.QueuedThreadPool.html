org.eclipse.jetty.util.thread.QueuedThreadPool

The org.eclipse.jetty.util.thread.QueuedThreadPool is an implementation of the ThreadPool interface that utilizes a queue for request handling. It is part of the Jetty open source server project. The QueuedThreadPool class provides a way to configure the size of the thread pool, set the maximum number of requests that can be handled concurrently and also set the timeout period for how long requests can remain unserviced.

Basic Usage

A QueuedThreadPool can be created with a simple constructor call that takes the maximum pool size and thread timeout parameters.


          
           
QueuedThreadPool qtp = new QueuedThreadPool(5, 2);

This will create a thread pool with a maximum of 5 threads and a timeout of 2 seconds. After this point the thread pool will start to automatically manage the threads to accommodate incoming requests.

Queueing Requests

Requests can be queued using the 
execute()
method: 

          
           
qtp.execute(new Runnable() {
    public void run() {
        // do something
    }
});

The thread pool will handle the queuing of requests internally, so there is no need to explicitly manage a queue.

Getting Statistics

The QueuedThreadPool class provides methods to retrieve recent statistics about request handling. For example, the 

getIdleThreads()
method will return the number of currently idle threads: 

          
           
int idleThreads = qtp.getIdleThreads();
The 
getActiveThreads()
method will return the number of actively processing requests: 

          
           
int activeThreads = qtp.getActiveThreads();

Shutting Down

When the thread pool is no longer needed it should be shutdown to ensure all resources are released. This can be done by calling the 

shutdown()
method: 

          
           
qtp.shutdown();

Once the shutdown method has been called no more requests will be accepted and any existing requests in the queue will be discarded.

Conclusion

The org.eclipse.jetty.util.thread.QueuedThreadPool provides an easy to use API for creating threads and queuing requests.
