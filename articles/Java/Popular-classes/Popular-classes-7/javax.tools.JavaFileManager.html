javax.tools.JavaFileManager

The 

          
           
javax.tools.JavaFileManager
class is a standard interface for managing the compilation of Java programs within a particular environment. It allows the user to control which files the Java compiler accesses during the compilation process and how the compiled code is stored.

File Locations

The 
JavaFileManager
provides functionality for managing file locations. It defines three broad, distinct categories of file locations that a 
JavaFileManager
instance can work with:
  1. System locations, which correspond to the standard locations where system classes are usually stored (such as the Java runtime library).
  2. Classpath locations, which correspond to user-specified locations (such as the current working directory).
  3. Output locations, which define where the compiled code should be stored.

Compiling a File

The 
JavaFileManager
class provides several useful methods for compiling a single file. These include: 
  • 
            
             
getJavaFileObjectsFromStrings(String ... names)
    , which takes a list of strings representing the names of source files to be compiled and returns a set of 
    JavaFileObject
    s corresponding to them.
  • 
            
             
getFileForInput(Location location, String packageName, String relativeName)
    , which returns a 
    FileObject
    representing the specified file in the given location and package.
  • 
            
             
getJavaFileForInput(Location location, String className, JavaFileObject.Kind kind)
    , which returns a 
    JavaFileObject
    representing the specified file in the given location with the specified class name and file kind.
  • 
            
             
getFileForOutput(Location location, String packageName, String relativeName, FileObject sibling)
    , which creates and returns a 
    FileObject
    representing the specified output file in the given location and package.
  • 
            
             
getJavaFileForOutput(Location location, String className, JavaFileObject.Kind kind, FileObject sibling)
    , which creates and returns a 
    JavaFileObject
    representing the specified output file in the given location with the specified class name and file kind.

Example Usage

The following example shows how to use the 
JavaFileManager
to compile a hello world program: 

          
           
// Create a StandardJavaFileManager instance
StandardJavaFileManager fileManager = ToolProvider.getSystemJavaCompiler().getStandardFileManager(null, null, null);

// Get the list of source files to be compiled
Iterable<? extends JavaFileObject> sources = fileManager.getJavaFileObjectsFromStrings(Arrays.asList("HelloWorld.java"));

// Create a compilation task
CompilationTask task = ToolProvider.getSystemJavaCompiler().getTask(null, fileManager, null, null, null, sources);

// Execute the compilation task
task.call();

// Close the file manager
fileManager.close();
This example will compile the file 
HelloWorld.java
in the current working directory and store the result in the default output location.
