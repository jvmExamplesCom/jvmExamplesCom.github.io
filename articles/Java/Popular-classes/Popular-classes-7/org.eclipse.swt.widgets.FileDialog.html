org.eclipse.swt.widgets.FileDialog

The org.eclipse.swt.widgets.FileDialog class provides a user interface for browsing and selecting files from the local file system. It supports both open (to read from existing files) and save (to write a new file) operations. This tutorial will provide a quick overview of how to create and use a FileDialog instance.

Creating a FileDialog Instance

Creating a FileDialog instance is straightforward—all that is required is to give the dialog a parent Shell, an SWT style and an initial directory or file path:


          
           
FileDialog dialog = new FileDialog(parentShell, SWT.OPEN);
dialog.setFilterPath("C:/");

Showing a FileDialog

Once the FileDialog is initialized, it can be opened by calling its open() method. When the user closes the dialog, the method returns the chosen file path:


          
           
String filePath = dialog.open();
System.out.println(filePath);

Setting a Filter

It is possible to limit the files that the user can select from when using the FileDialog. This can be done with the setFilter() method, which takes an array of Strings representing the filter patterns. For example, if only text files should be selectable, the following filter can be used:


          
           
dialog.setFilterExtensions(new String[] { "*.txt" });

Setting a Default File Name

If the FileDialog is being used in a save operation, a default file name can be supplied with the setFileName() method:


          
           
dialog.setFileName("my_file.txt");

Conclusion

The FileDialog is a useful feature for allowing users to browse and select files from the local file system. With its setFilter() and setFileName() methods, the dialog can be customized to match the application's needs.
