Understanding org.openqa.selenium.OutputType

The 

          
           
org.openqa.selenium.OutputType
interface is a fundamental part of the Selenium WebDriver API, providing an enumeration of different types of output files that a test may produce. It allows users to specify the type of output file for WebDriver methods such as 
getScreenshotAs()
, 
savePageSource()
, and 
getPageSource()
. Through this interface, users can select from various output types such as PNG, PDF, HTML, TEXT, BINARY, and FILE.

Overview of Output Types

PNG and PDF are the two most commonly used output types, as they are both image based file types. PNG is a lossless image format which is smaller in size than PDF and is best suited for screenshots, while PDF is larger in size and can be used for storing text-like documents. HTML and TEXT formats are suitable for web pages and texts, respectively. BINARY stores generic binary data, while FILE is used to save a file in its native format.

Code Examples

Here is a code example showing how to capture a screenshot of a web page as a PNG file:


          
           
File screenshot = ((TakesScreenshot)driver).getScreenshotAs(OutputType.FILE);
FileUtils.copyFile(screenshot, new File("/tmp/screenshot.png"));

The following code snippet saves a web page's HTML source code as an HTML file:


          
           
File htmlSource =
    ((TakesScreenshot)driver).getScreenshotAs(OutputType.HTML);

To save the web page's source code as a text file, use the following code:


          
           
File txtSource =

FileUtils.copyFile(txtSource, new File("/tmp/source.txt"));

And to save a web page's source code as a binary file, use this code snippet:


          
           
File binSource =
    ((TakesScreenshot)driver).getScreenshotAs(OutputType.BINARY);
FileUtils.copyFile(binSource, new File("/tmp/source.bin"));

Finally, to save a web page's source code in its native format, use the following code:


          
           
File nativeSource =
    ((TakesScreenshot)driver).getScreenshotAs(OutputType.FILE);
