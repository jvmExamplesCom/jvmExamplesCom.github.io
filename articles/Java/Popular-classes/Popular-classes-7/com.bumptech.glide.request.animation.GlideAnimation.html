com.bumptech.glide.request.animation.GlideAnimation

GlideAnimation is an abstract class from Glide library and part of android image loading library which can be used to provide the animation for an image loading request. This class has several subclasses like DrawableCrossFadeFactory, NoAnimation and ViewAnimation. In this article, we will learn about these classes and their example codes.

DrawableCrossFadeFactory

DrawableCrossFadeFactory is a factory class that creates a new instance of DrawableCrossFadeAnimation. DrawableCrossFadeAnimation class implements GlideAnimation interface and provides the animation while loading images. While creating an animation with this class, we need to specify the initial frame duration as well as the frame duration for any following frames.

Example code for creating an instance of DrawableCrossFadeFactory class.


          
           
DrawableCrossFadeFactory drawableCrossFadeFactory = 
                    new DrawableCrossFadeFactory.Builder().setCrossFadeEnabled(true).build();

NoAnimation

NoAnimation class is a subclass of DrawableCrossFadeFactory class and provides no animation while loading an image. This class is useful when we want to load an image without any animation. To create an instance of this class we need to call newInstance() method of this class.

Example code for creating an instance of NoAnimation class:


          
           
NoAnimation noAnimation = NoAnimation.getInstance();

ViewAnimation

ViewAnimation class is a subclass of GlideAnimation and provides the animation for an image loading request. This class takes the View object as a parameter and performs the animation on the given view. To create an instance of this class we need to call newInstance() method of this class.

Example code for creating an instance of ViewAnimation class:


          
           
ViewAnimation viewAnimation = ViewAnimation.newInstance();
