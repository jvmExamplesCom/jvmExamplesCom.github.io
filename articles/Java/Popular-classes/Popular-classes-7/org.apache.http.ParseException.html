org.apache.http.ParseException

The org.apache.http.ParseException is thrown when attempting to parse an HTTP message which is invalid in some way. The underlying cause of the ParseException is usually a malformed HTTP message, meaning that it does not conform to the rules governing the format of HTTP messages.

The most common symptom of a ParseException is a "malformed request" error, which suggests that the underlying problem is with the content of the HTTP message. In some cases, this can be caused by a bug in the client application.

Example 1: Throwing a ParseException

In this example, we will create a ParseException by passing an invalid HTTP message to the HttpRequest.parse() method:


          
           
HttpRequest req = new HttpRequest("GET / HTTP/1.1\r\nHost:example.com\r\n\r\n");
try {
    req.parse();
} catch (ParseException e) {
    System.err.println("Failed to parse request: " + e);
}

Example 2: Handling a Malformed Request

Once a ParseException has been thrown, it is possible to determine where and why the request failed. This is because the ParseException holds a reference to the HttpRequest which was attempted to be parsed. In this example, we will attempt to parse an invalid request and handle the ParseException accordingly:


          
           
HttpRequest req = new HttpRequest("GET / HTTP/1.1\r\nHost:example.com\r\n\r\nInvalid Header");
try {
    req.parse();
} catch (ParseException e) {
    HttpRequest invalidReq = e.getRequest();
    System.err.println("Failed to parse request at line: " + invalidReq.getLineNumber());

    System.err.println("Failed to parse request error message: " + e.getMessage());
}

Example 3: Catching a ParseException

It is also possible to catch a ParseException before it affects the entire request. In this example, we will attempt to read the headers from an invalid request and catch the ParseException if it is thrown:
