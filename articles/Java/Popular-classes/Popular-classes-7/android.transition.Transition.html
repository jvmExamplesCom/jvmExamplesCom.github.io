android.transition.Transition

Android API-level 21 and higher offer developers the ability to easily create animations and scene transitions with the android.transition.Transition class. It simplifies the process of creating visually appealing transitions between different scenes in an Android app, allowing developers to create stunning visual effects for their users. In this article, we will explore how to use the Transition API to create various animation effects.

Creating Transitions

The android.transition package contains several classes and interfaces which provide developers with the ability to create custom transitions between screens. To get started, developers can either choose from the various classes in the android.transition package or create their own custom transition. The most basic use of Transition involves creating a new Transition object and assigning it to a Scene object. The Scene represents a state of the UI hierarchy in an activity while the Transition is responsible for animating changes that occur when switching between two scenes. 

          
           
// Create a new Scene object
Scene scene1 = Scene.getSceneForLayout(rootView, R.layout.scene_1, this);

// Create a Transition object
Transition transition = new Fade();

// Apply the transition to the Scene
transition.setDuration(1000);
transition.setInterpolator(new LinearInterpolator());
scene1.setEnterTransition(transition);
When the transition is set, it will be triggered whenever the current scene is replaced by the Scene being transitioned to. It will animate any changes that occur during the transition, such as views entering and leaving the scene, animations, and changes in layout, color, and more.

Types of Transitions

The android.transition package provides a variety of pre-built Transition classes. These classes include:
  • Fade - fades out one scene while fading in another.
  • Slide - slides the current view away while sliding in the next view.
  • Explode - explodes the current view while simultaneously imploding the next view.
  • ChangeBounds - animates changes in the size and position of views.
  • ChangeClipBounds - animates changes in the clip bounds of views.
  • ChangeTransform - animates changes in the position, rotation, and scale of views.
  • ChangeImageTransform - animates changes in the image transform of views.
Each of these classes has useful methods and properties that can be used to customize the transition. For example, the setDuration() method can be used to change the duration of the transition, and the setInterpolator() can be used to change the type of interpolation used to blend the transitions together.

Customization

In addition to the pre-built Transition classes, developers can also create their own custom Transition classes. For example, they might want to create a transition that zooms in on the next view while zooming out from the current view. To do this, developers can subclass the Transition class and override the onAppear() and onDisappear() methods. These methods are responsible for animating the transition and the developer can use them to create custom animations. 

          
           
public class ZoomTransition extends Transition {

    @Override
    public void onAppear(ViewGroup sceneRoot, View view, TransitionValues startValues,
                         TransitionValues endValues) {
        view.animate().scaleX(1.2f).scaleY(1.2f).setDuration(500).start();
    }
    
    @Override
    public void onDisappear(ViewGroup sceneRoot, View view, TransitionValues startValues,
                            TransitionValues endValues) {
        view.animate().scaleX(0.8f).scaleY(0.8f).setDuration(500).start();
    }
}
Once the custom Transition class has been created, it can then be assigned to a Scene via the setEnterTransition() or setExitTransition() methods.

Conclusion

The android.transition.Transition class provides developers with an easy way to create beautiful animations and scene transitions in their Android apps. By taking advantage of the various types of transitions and customization options provided by the class, developers can create visually appealing transitions that give their apps a truly professional look.
