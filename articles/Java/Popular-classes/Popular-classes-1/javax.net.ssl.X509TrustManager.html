X509TrustManager: SSL and Java

The 

          
           
javax.net.ssl.X509TrustManager
is a vital part of Java Security, providing authentication for server-client connections established using Secure Socket Layer (SSL). A Trust Manager manages certificates that are used to authenticate the server and the client which are communicating over a secure connection.

An X509TrustManager is created by implementing the 

X509TrustManager
interface, which provides a set of methods to manage and evaluate SSL certificates from the server and the client. X509TrustManager allows an application to determine if it should allow a secure connection with a remote entity, based on information about the type of certificate and the certificate authority.

X509TrustManager

X509TrustManager

:

Creating an SSL Context

Once an X509TrustManager has been implemented, you can then create an SSL context. An SSL context is responsible for managing the secure connection between two peers. To create the SSL Context use the

Conclusion

javax.net.ssl.X509TrustManager
is an important part of SSL and secure communication in Java applications. An X509TrustManager verifies certificates from the client and the server and determines if they should be allowed to communicate over a secure connection. It is implemented by creating a class which implements the X509TrustManager interface, then creating an SSL context and initializing it with the trust managers.
