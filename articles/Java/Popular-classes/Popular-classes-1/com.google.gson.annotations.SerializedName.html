The SerializedName annotation marks a field in a class to be written as part of a JSONObject when serializing the object. It can also be used when deserializing a JSONObject into an object to identify the name of the field to set in the object.

@SerializedName("name") Example

The @SerializedName("name") annotation can be used to specify the name of a field in the JSONObject that represents the object. For example, consider a class that represents a person:


          
           
public class Person {  
  private String firstName;  
  private String lastName;  
  
  public Person(String firstName, String lastName) {  
    this.firstName = firstName;  
    this.lastName = lastName;  
  }  
}

To serialize an instance of the Person object, we need to specify the name of the fields in the JSONObject that correspond to the fields of the object. We can do this by adding the @SerializedName annotation to the fields of the class, like so:


          
           
public class Person {  
  private String firstName;  
  private String lastName;  
  
  @SerializedName("first_name")  
  public String getFirstName() {  
    return firstName;  
  }  
  
  @SerializedName("last_name")  
  public String getLastName() {  
    return lastName;  
  }  
  
  public Person(String firstName, String lastName) {  
    this.firstName = firstName;  
    this.lastName = lastName;  
  }  
}

Now, when the object is serialized, the JSONObject will have the fields "first_name" and "last_name" corresponding to the object's firstName and lastName fields, respectively.

@SerializedName can also be used to specify multiple values that should be serialized as a single field. For example, let's say we want to serialize an Address object that has two fields, city and state, which should be serialized as a single field, called "location":


          
           
public class Address {

We can use the @SerializedName annotation to specify that the two fields should be combined into one, like so:


          
           
public class Address {  
  private String city;  
  private String state;

  public Address(String city, String state) {

Now, when the Address object is serialized, the JSONObject will have a single field, "location", which contains the city and state separated by a comma.

Conclusion

The @SerializedName annotation is a powerful tool for controlling how objects are serialized and deserialized into and out of JSONObjects. It can be used to control the names of fields in the JSONObject, as well as to combine multiple fields into a single field.
