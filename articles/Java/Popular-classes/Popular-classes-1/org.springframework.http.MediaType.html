org.springframework.http.MediaType

The org.springframework.http.MediaType is a class in the Spring Framework which provides a framework for setting media type parameters which can be used by various components, such as sending content-type headers in response to HTTP requests. This is essential for working with content types and encoding schemes like JSON, XML, and more.

The class provides some useful static constant methods for setting common media types, as well as a method for creating custom media types.

Creating MediaType Objects

The static factory methods in the MediaType class can be used to create a MediaType object with a given media type.


          
           
MediaType mediaType = MediaType.APPLICATION_JSON;

The MediaType class includes static methods for common media type values such as APPLICATION_JSON, APPLICATION_XML, TEXT_PLAIN, TEXT_HTML, IMAGE_JPEG, and more.

If you need to create a custom media type, you can use the create(String type, String subtype) method:


          
           
MediaType customMediaType = MediaType.create("application", "vnd.example.custom+json");

Getting Media Type Parameters

The MediaType class also provides methods for getting media type parameters such as charset, language, and more. The getParameters() method returns a map of the media type parameters:


          
           
Map<String, String> parameters = mediaType.getParameters();

You can access the value of a specific parameter using the getParameter(String name) method:


          
           
String charset = mediaType.getParameter("charset");

Comparing Media Types

The MediaType class also provides methods for checking whether two media types are equal, or whether one media type is compatible with another one. This can be useful when you need to check if a content-type header supports a certain media type.

To check if two media types are equal, you can use the equals(Object obj) method:


          
           
boolean isEqual = mediaType.equals(MediaType.APPLICATION_JSON);

To check if one media type is compatible with another one, you can use the includes(MediaType other) method:


          
           
boolean isCompatible = mediaType.includes(MediaType.APPLICATION_JSON);
