java.io.Serializable

Java.io.Serializable is an interface that marks a class as being capable of being serialized. Serialization is the process of taking an object's state information and converting it into a stream of bytes. This allows objects to be stored in a persistent form, transferred between applications, or stored on disk.

The Serializable Interface

The Serializable interface itself has no methods. It simply acts as a marker that can be used by the Java runtime when serializing and deserializing objects. To use the Serializable interface, a class must simply implement it:


          
           
public class MyClass implements Serializable {
    // ...
}

Once a class is marked as Serializable, its state can be written to a stream of bytes via the ObjectOutputStream and read back in via the ObjectInputStream. Each instance variable of the class is written to the stream. Note that not all types of variables are supported; for example, static variables cannot be serialized as they are shared between different instances of the same class.

SerialVersionUID

If a class implements the Serializable interface, then it should also declare a static final long field named serialVersionUID. This field ensures that different versions of the same class can still be deserialized, even if some of the fields have changed. When an object is deserialized, the serialVersionUID of the class is compared against the serialVersionUID of the object being deserialized. If the two numbers do not match, then a 


          
           
InvalidClassException
is thrown. 

          
           
public class MyClass implements Serializable {

    private static final long serialVersionUID = 1234567890L;
    
    // ...
}

The value of the serialVersionUID field can be generated automatically by the IDE or command-line tool. Usually, the value is just some random number, but it can also be based on the class name, methods, and fields. It is best practice to always declare a serialVersionUID when implementing the Serializable interface.

Serializing Objects

To serialize an object, create a new 
ObjectOutputStream
and write the object to it. The ObjectOutputStream will convert the object into a stream of bytes and write them to the underlying output stream. 

          
           
// Create an output stream
FileOutputStream fos = new FileOutputStream("myfile.dat");
ObjectOutputStream oos = new ObjectOutputStream(fos);

// Write an object
MyClass obj = new MyClass();
oos.writeObject(obj);

// Close streams
oos.close();
fos.close();

The above code snippet will write the state of the 

MyClass
object to the file 
myfile.dat
. Note that this file is not readable by humans; it is a binary file that contains the state information of the object.

Deserializing Objects

To deserialize an object, create a new 
ObjectInputStream
and read the object from it. The ObjectInputStream will convert the binary data back into an object instance. 

          
           
// Create an input stream
FileInputStream fis = new FileInputStream("myfile.dat");
ObjectInputStream ois = new ObjectInputStream(fis);

// Read an object
MyClass obj = (MyClass) ois.readObject();

// Close streams
ois.close();
fis.close();
The above code snippet will read the state of the 
MyClass
object from the file 
myfile.dat
. Note that if the class has changed since the file was written, then a 

          
           
InvalidClassException
may be thrown. This can be prevented by declaring the 
serialVersionUID
field.

Conclusion

The Serializable interface is a powerful tool for persisting and transferring objects between applications. By simply implementing the interface and declaring a serialVersionUID field, classes can be made serializable. The ObjectOutputStream and ObjectInputStream classes provide the API for serializing and deserializing objects.
