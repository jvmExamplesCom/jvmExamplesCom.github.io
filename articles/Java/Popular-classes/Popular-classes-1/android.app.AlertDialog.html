android.app.AlertDialog

Android includes a variety of pre-built UI components such as buttons, text fields, and 

          
           
android.app.AlertDialog
which allow developers to create rich and interactive user interfaces for applications. AlertDialog is one of the most commonly used components in Android application development, and it is used to alert the user about an important event or action that needs to be taken. AlertDialog can be configured to display different types of messages such as warnings, errors, confirmations and others. AlertDialog consists of three parts - title, message, and buttons. The title is displayed at the top of the dialog in bold and serves as a short description of the message. The message is the main content which is usually displayed in a scrollable format. Finally, buttons are used to give the user an option to respond to the alert (e.g. OK or Cancel). In order to create an instance of the AlertDialog class, you need to use the AlertDialog.Builder class. The Builder class allows you to specify parameters such as the title and message, as well as the type of buttons to be used. For example, the following code snippet creates an AlertDialog with an Error message, a title, and two buttons: “OK” and “Cancel”. 

          
           
AlertDialog.Builder builder = new AlertDialog.Builder(this);
builder.setMessage("Error Occurred!");
builder.setTitle("Error");
builder.setPositiveButton("OK", new DialogInterface.OnClickListener() {
    @Override
    public void onClick(DialogInterface dialog, int which) {
        // handle “OK” click
    }
});
builder.setNegativeButton("Cancel", new DialogInterface.OnClickListener() {
    @Override
    public void onClick(DialogInterface dialog, int which) {
        // handle “Cancel” Click
    }
});

AlertDialog dialog = builder.create();
dialog.show();
Once the dialog has been created and shown, the user will be presented with the title, message and buttons defined by the developer. When the user clicks on either the “OK” or “Cancel” button, the respective listener methods will be invoked. Alternatively, you can also create an AlertDialog without any buttons. This might be useful if you want to display a simple message to the user and don’t want them to take any action afterwards. To do this, you need to set the “setCanceledOnTouchOutside” property of the builder to “true”. 

          
           
AlertDialog.Builder builder = new AlertDialog.Builder(this);
builder.setMessage("Error Occurred!");
builder.setTitle("Error");

// Set “cancel on touch outside” to true
builder.setCanceledOnTouchOutside(true);

AlertDialog dialog = builder.create();
dialog.show();
This will create an AlertDialog without any buttons, and the user will be able to close the dialog by clicking outside the dialog window. Finally, you can also customize the look and feel of the AlertDialog by setting a custom layout. You can do this by using the “setView” method of the builder and passing in a layout resource. 

          
           
AlertDialog.Builder builder = new AlertDialog.Builder(this);
builder.setMessage("Error Occurred!");
builder.setTitle("Error");

// Set custom view to AlertDialog
builder.setView(R.layout.custom_alert_dialog);

AlertDialog dialog = builder.create();
dialog.show();
This will set the custom layout to the AlertDialog and the user will be presented with the contents of the layout instead of the default message and buttons. In conclusion, AlertDialog is a powerful component which can be used to create a variety of different types of dialogs. It can be configured to display a message, title, and buttons, and it also allows developers to customize its look and feel by using custom layouts.
