android.support.v4.app.FragmentTransaction

Introduction

android.support.v4.app.FragmentTransaction is an Android class used to manage operations that add, remove, replace, and/or animate Fragments. This class provides a set of methods that simplifies the adding and removing of Fragments into a container, as well as control how they are animated, if at all.

Basics to Fragment Transactions

Before using android.support.v4.app.FragmentTransaction, it is important to understand the fundamentals of Fragments first. A Fragment, much like an Activity, is a container for View elements. When creating a Fragment, an implementation of a Layout (e.g. XML for Android) must be provided. This allows for a range of different user interfaces contained within a single Activity. A FragmentTransaction enables developers to add, remove, replace, and animate Views contained within a set of Fragments. The following operations are supported by this class:
  • add() – Add a Fragment to the top of the stack.
  • remove() – Remove a Fragment from the stack.
  • replace() – Replace an existing Fragment with a new one.
  • hide() – Hide a Fragment.
  • show() – Show a previously hidden Fragment.
  • detach() – Remove a Fragment from the Activity (but keep it in memory).
  • attach() – Re- attach a detach Fragment back to the Activity.
  • setCustomAnimations() – Set up custom animations for a Fragment transition.
