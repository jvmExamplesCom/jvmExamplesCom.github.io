android.widget.BaseAdapter

The android.widget.BaseAdapter class is a base class which enables data binding and adapter view recycling. It is an abstract class that must be subclassed to be used. When feasible, it is recommended that the class CursorAdapter be used to avoid creating multiple unnecessary instances of BaseAdapter.

Methods

The android.widget.BaseAdapter class contains a number of methods which can be used to bind data to views, recycle views, register content observers and notify about data set changes.

The methods are as follows: 
  • 
            
             
public void bindView(View view, Context context, Cursor cursor)
    - this method binds the Cursor to the view provided.
  • 
            
             
public View newView(Context context, Cursor cursor, ViewGroup parent)
    - this method creates a new view for the specified Cursor.
  • 
            
             
public void changeCursor(Cursor cursor)
    - this method swaps the current Cursor with a new one.
  • 
            
             
public void unregisterDataSetObserver(DataSetObserver observer)
    - this method removes a previously registered DataSetObserver.
  • 
            
             
public void registerDataSetObserver(DataSetObserver observer)
    - this method registers an observer to respond to data set changes.
  • 
            
             
public void notifyDataSetChanged()
    - this method notifies the attached observers that the underlying data has been changed and any View reflecting the data set should refresh itself.

Implementation

The following code example demonstrates how to use the android.widget.BaseAdapter class for a simple ListView. The adapter binds the data to the view using a custom layout.


          
           
public class MyAdapter extends BaseAdapter {

    private ArrayList<String> mData;
    private LayoutInflater mInflater;

    public MyAdapter(Context context, ArrayList<String> data) {
        mData = data;
        mInflater = LayoutInflater.from(context);
    }

    @Override
    public int getCount() {
        return mData.size();
    }

    @Override
    public Object getItem(int position) {
        return mData.get(position);
    }

    @Override
    public long getItemId(int position) {
        return position;
    }

    @Override
    public View getView(int position, View convertView, ViewGroup parent) {

        ViewHolder holder;

        if(convertView == null) {
            convertView = mInflater.inflate(R.layout.list_item, null);
            holder = new ViewHolder();

            holder.name = (TextView) convertView.findViewById(R.id.name);

            convertView.setTag(holder);
        } else {
            holder = (ViewHolder) convertView.getTag();
        }

        holder.name.setText(mData.get(position));

        return convertView;
    }

    static class ViewHolder {
        TextView name;
    }
}

In the code above, the adapter is created and passed in a 

Context
and 

          
           
ArrayList<String>
which contains the data to be bound to the views. The 
getView()
method uses the LayoutInflater to create the views for the ListView. The custom layout, which is inflated by the LayoutInflater, contains a TextView which is populated with the data from the 

          
           
ArrayList<String>
.

The code snippet below shows how the adapter will be used in an Activity.


          
           
MyAdapter adapter = new MyAdapter(this, data);
listView.setAdapter(adapter);

The above code sample creates an instance of the custom adapter and passes in the data. The adapter is then set as the adapter for the ListView.

Conclusion

The android.widget.BaseAdapter class provides a base class for creating adapters. It can be used to bind data to views and register content observers. This class should ideally be replaced with the CursorAdapter class when feasible.
