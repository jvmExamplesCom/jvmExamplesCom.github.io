Android widget AdapterView

Android widget AdapterView is an interface that allows you to bind data to a view. It can be used to build complex views with lists of items and subviews. AdapterView is a subclass of ViewGroup and provides a few simple methods that allow the adapter to provide data to the user interface.

Creating an Adapter

The first step in using AdapterView is to create an Adapter. An adapter is responsible for providing data to the View from the data source. To create an adapter, you must implement the Adapter interface, which requires you to define the getCount, getItem, and getView methods. Here is an example implementation: 

          
           
public class MyAdapter extends BaseAdapter {
   private List<String> data;

   public MyAdapter(List<String> data) {
      this.data = data;
   }

   @Override
   public int getCount() {
      return data.size();
   }

   @Override
   public Object getItem(int position) {
      return data.get(position);
   }

   @Override
   public long getItemId(int position) {
      return position;
   }

   @Override
   public View getView(int position, View convertView, ViewGroup parent) {
      // Create view for item
   }
}
Once you have implemented the Adapter interface, you must instantiate it with the data you want to display in the View.

Attaching the Adapter to the View

Once the Adapter is created, you can attach it to the View by using the setAdapter method. This method takes an Adapter object as a parameter. 

          
           
AdapterView view = (AdapterView) findViewById(R.id.listView);
MyAdapter myAdapter = new MyAdapter(data);
view.setAdapter(myAdapter);

Displaying Data in the View

Once the Adapter is attached to the View, it will start to populate the View with the data from the adapter. Each item in the data list will be rendered according to the getView method. If the data is changed, the View will be updated accordingly.

Customizing the View

The getView method of the Adapter can be overridden to customize the way the data is displayed in the View. This method takes three parameters: the position of the item in the list, a convertView object which is used for recycling views, and the parent ViewGroup. The getView method must return a View object which is used to render the item. 

          
           
@Override
public View getView(int position, View convertView, ViewGroup parent) {
   // Check if an existing view is being reused, otherwise inflate the view
   if (convertView == null) {
       convertView = LayoutInflater.from(context).inflate(R.layout.list_item, parent, false);
   }

   // Lookup view for data population
   TextView tvName = (TextView) convertView.findViewById(R.id.tvName);

   // Populate the data into the template view
   tvName.setText(data.get(position));

   // Return the completed view to render on screen
   return convertView;
}
The above code example shows how to use the convertView parameter to recycle views. If the convertView is null, then a new view should be created, otherwise the existing view is reused.

Conclusion

AdapterView allows you to bind data to a View in an efficient way. It also allows you to customize the way the data is displayed in the View by implementing the getView method of the Adapter.
