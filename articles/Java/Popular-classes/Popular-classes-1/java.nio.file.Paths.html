java.nio.file.Paths

Introduction

Java.nio.file.Paths is a Java class that provides access to the underlying file system for working with files and directories. It provides utility methods for constructing Path objects from strings and other Path objects, as well as methods for operating on paths such as getting the file name, parent directory, etc. It also manages symbolic links. The Paths class is a part of the Java NIO (New I/O) package. It provides many useful methods to support applications that need to interact with the filesystem.

Constructing a Path Object

The Paths class provides two utility methods for constructing Path objects: 

          
           
get()
and 

          
           
get(URI)
. Both methods return an instance of a Path object. The 

          
           
get()
method accepts a String argument representing a pathname in the underlying platform's default file system. 

          
           
Path path = Paths.get("C:\\MyFile.txt");
The 

          
           
get(URI)
method accepts an argument of type URI, which represents the Uniform Resource Identifier (URI) of the file. 

          
           
Path path = Paths.get(URI.create("file:///C:/MyFile.txt"));

Operating on a Path Object

Once you have constructed a Path object, you can use it to interact with the underlying file system. The Path class provides several useful methods for interacting with the file system, such as getting the file name, parent directory, and checking if the given file is readable, writable, or executable. Some of the useful methods are listed below: 
  • getFileName()
    - Returns the file name of the path object.
  • getParent()
    - Returns the parent directory of the path object.
  • toAbsolutePath()
    - Returns the absolute path.
  • 
            
             
toRealPath(LinkOption... options)
    - Returns the real path.
  • isReadable()
    - Returns true if the file is readable.
  • isWritable()
    - Returns true if the file is writable.
  • isExecutable()
    - Returns true if the file is executable.
