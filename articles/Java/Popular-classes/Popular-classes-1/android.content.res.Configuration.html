Introduction to android.content.res.Configuration

The 

          
           
android.content.res.Configuration
class provides information about the current configuration of the device, including locale, the type of hard keyboard available and more. This class is useful for localizing an application, managing resources and more. It is also used to set up the user interface layout with the current configuration, such as changing between portrait and landscape layouts.

Features

The 

          
           
android.content.res.Configuration
class has a few key features that make it useful for working with localized resources:
  • Locale: The Configuration object has a locale field which is a java.util.Locale object. This indicates the current language, country, variant and script.
  • Layout Directions: The Configuration object also has a field for layout direction. This indicates whether the user interface should be laid out in a left-to-right or right-to-left orientation.
  • Hard Keyboard Configuration: The Configuration object also provides information about the presence and type of any hard keyboard attached to the device.

Methods

The 

          
           
android.content.res.Configuration
class has several useful methods, such as: 
  • 
            
             
getLayoutDirection()
    : Returns the current layout direction.
  • getLocale()
    : Returns the current locale.
  • 
            
             
getLayoutDirectionFromLocale(Locale loc)
    : Returns a layout direction for the given Locale.
  • hasHardKeyboard()
    : Returns true if there is a hardware keyboard present.
  • getKeyboardType()
    : Returns an integer indicating the type of hardware keyboard that is present.

Code Examples

The following code examples will demonstrate how to use the 

          
           
android.content.res.Configuration
class.

Example 1: Checking for Hard Keyboard

To check whether a hardware keyboard is present, you can use the 

          
           
hasHardKeyboard()
method. This example will print out either "Hard keyboard present" or "No hard keyboard": 

          
           
if (config.hasHardKeyboard()) {
    System.out.println("Hard keyboard present");
} else {
    System.out.println("No hard keyboard");
}

Example 2: Getting Keyboard Type

To get an integer denoting the type of hardware keyboard present, you can use the 

          
           
getKeyboardType()
method. This example will print out one of the following strings based on the returned keyboard type: 

          
           
int type = config.getKeyboardType();
String typeString;

switch (type) {
    case Configuration.KEYBOARD_QWERTY:
        typeString = "QWERTY";
        break;

    case Configuration.KEYBOARD_12KEY:
        typeString = "12-Key";
        break;

    case Configuration.KEYBOARD_NOKEYS:
        typeString = "No keys";
        break;

    default:
        typeString = "Unknown";
        break;
}

System.out.println("Keyboard type: " + typeString);
