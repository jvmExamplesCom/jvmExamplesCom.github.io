Introduction to android.support.test.InstrumentationRegistry Class

The android.support.test.InstrumentationRegistry class is an essential part of the Android instrumentation testing framework. It provides access to application-specific resources and classes, as well as methods for manipulating Android’s activity lifecycle. By using this class, developers can build tests to effectively check the correctness of their application’s code.

Methods Available in android.support.test.InstrumentationRegistry

The android.support.test.InstrumentationRegistry class has a number of helpful methods available:
  • getContext(): This method returns a 
    Context
    object to be used for performing operations related to the current test environment.
  • getArguments(): This method returns a 
    Bundle
    containing any arguments that were passed to the instrumentation.
  • getTargetContext(): This method returns the 
    Context

  • registerInstance(): This method registers an 
    Instrumentation

  • runOnMainSync(): This method runs the given runnable on the UI thread of the application.
  • waitForIdleSync(): This method waits for the UI thread to become idle.
