android.annotation.TargetApi

The android.annotation.TargetApi annotation is a part of Android SDK Platform. It is used for indicating that a certain piece of code should be run on devices running Android version higher than or equal to the given version. This can be useful for optimizing code with platform-specific features, or for restricting its use to newer platforms.

Usage

The annotation should be placed at the beginning of a class, method, or field declaration and specify a minimum version of the platform:


          
           
@TargetApi(Build.VERSION_CODES.KITKAT)
public class MyClass { ... }

When used on a class, all methods, fields, and inner classes will run in the specified environment. If a method or field uses a newer platform feature, it will generate an error unless the annotation is specified.

When used on a method or field, only that specific item will run in the specified environment.

Example

Let's imagine you want to use the new NotificationCompat.Builder.setGroupSummary() method, which was introduced in API Level 19. To use it, you would need to annotate the class or method containing the code with a 

@TargetApi
annotation, specifying a minimum API level of 19: 

          
           
@TargetApi(Build.VERSION_CODES.KITKAT)
public NotificationCompat.Builder setupNotification(){
  NotificationCompat.Builder builder = new NotificationCompat.Builder(getContext());
  builder.setGroupSummary(true);
  return builder;
}

The same annotation can be used for restricting the use of legacy code on newer platforms. For example, if you have some SDK calls that are no longer supported, you can annotate the method with a 

@TargetApi
annotation specifying the oldest version of Android that the code still works on: 

          
           
@TargetApi(Build.VERSION_CODES.HONEYCOMB)
public void doLegacyWork() {
  // Legacy SDK calls
}
