android.support.v4.app.FragmentActivity

android.support.v4.app.FragmentActivity is a base class to use fragments in an activity. It is included in the Android Support Library v4. It provides various methods to interact with fragment, like add/replace fragment, find fragment by given tag/id etc. It also provide backward compatibility of fragments with older versions of android. In this article we will look at some examples and features of Fragment Activity.

Adding Fragment

Adding fragment is quite simple. We need to create an instance of fragment (e.g., MyFragment) firstly. Then use the method 


          
           
getSupportFragmentManager().beginTransaction().add(R.id.container, MyFragment).commit();
to add the fragment to the specified container. In the following example, we add a fragment to the view with id "container". 

          
           
MyFragment myFragment = new MyFragment();
getSupportFragmentManager().beginTransaction().add(R.id.container, myFragment).commit();

Replacing Fragment

We can replace a fragment with another one using the similar method mentioned in the previous section. The only difference is we use 


          
           
replace
instead of 

          
           
add
. Here's a example: 

          
           
MyFragment myFragment = new MyFragment();
getSupportFragmentManager().beginTransaction().replace(R.id.container, myFragment).commit();

Finding Fragment By Tag or Id

In some cases we may need to find a fragment by its tag or id. FragmentActivity provides the following methods to do so: 


          
           
findFragmentById
and 

          
           
findFragmentByTag
. Here is an example: 

          
           
MyFragment myFragment = (MyFragment) getSupportFragmentManager().findFragmentByTag("my_tag");
MyFragment myFragment = (MyFragment) getSupportFragmentManager().findFragmentById(R.id.my_id);

Backward Compatibility

FragmentActivity also provides backward compatibility of fragments with older versions of Android. This means we can use fragments on devices running 2.x or earlier. To support the backward compatibility we need to use the 


          
           
FragmentManager
from 

          
           
getSupportFragmentManager()
instead of 

          
           
getFragmentManager()
.

Conclusion

In this article we looked at some examples and features of android.support.v4.app.FragmentActivity. We learned how to add/replace/find a fragment and use FragmentActivity to provide backward compatibility of fragments with older versions of Android.
