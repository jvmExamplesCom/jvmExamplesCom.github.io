java.util.function.Function

Java.util.function.Function is an interface in the Java API that enables users to represent a function that accepts one argument and produces a result. It’s a parameterized interface that is widely used in Java 8 lambda expressions and it can be used along with some of the other functional interfaces like Predicate, Consumer, or Supplier. This article provides an overview of the Function interface and how to use it.

Overview of Function

The Function interface has two main Abstract methods: apply(Object) and compose(Function). Additionally there are also default methods, a few of which are – andThen(Function), identity(), and applyOrElse(). The identity() method returns a Function object that always returns its input unchanged regardless of what the return type is.

The Function interface is useful when you need to define a transformation that takes one argument and produces a single result. This can be used in scenarios such as mapping a List to a Set or converting a String to an Integer. One of the powerful things that can be done with this interface, is to compose multiple functions together. This is useful when you need to combine numerous operations on an object before returning the final result.

Syntax

The syntax of the Function interface is as follows:


          
           
public interface Function<T,R> {
    R apply(T t); 
    default <V> Function<V,R> compose(Function<? super V,? extends T> before);
    default <V> Function<T,V> andThen(Function<? super R,? extends V> after);
    static <T> Function<T,T> identity();
}

Example Usage

Function<String, Integer> parseInt = Integer::parseInt;
