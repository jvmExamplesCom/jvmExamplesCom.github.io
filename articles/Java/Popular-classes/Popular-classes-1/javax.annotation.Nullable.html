javax.annotation.Nullable

The javax.annotation.Nullable is a package used to indicate whether a variable, field, or parameter is allowed to be a null reference. It can be used by the compiler and tools to avoid potential Null Pointer Exceptions.

How to Use Nullable

We can use the @Nullable annotation in two ways: instance-level and type-level declarations.

Instance-Level Annotations

Instance-level annotations are used on the declared variables. We can do this with a single statement:


          
           
@Nullable String str = null;

We can also use it when declaring method parameters:


          
           
public void myMethod(@Nullable String str) { }

Type-Level Annotations

Type-level annotations are used when declaring class variables and methods, like so:


          
           
public class ExampleClass {
    @Nullable
    private String str;

    @Nullable
    public String getStr() { return str; }
}

External Tools Using Nullable

There are several tools that can use the Nullable annotation, such as FindBugs which can detect potential Null Pointer Exceptions in code. Other tools include IntelliJ IDEA, Eclipse, SonarQube, and more.

Conclusion

The javax.annotation.Nullable package is a powerful tool for helping developers avoid potential Null Pointer Exceptions. By using the instance-level and type-level annotations, the compiler and tools can check for potential issues that can be caught early on.
