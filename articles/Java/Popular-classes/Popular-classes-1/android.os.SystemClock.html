Android.os.SystemClock

The android.os.SystemClock class provides access to several features of the device’s system clock, including the system elapsed time, the current time and date, and other methods for converting between time formats. This class is useful for applications that need to measure elapsed time, or synchronize their activities with respect to the system clock.

Methods

elapsedRealtime()

This method returns the time since the system was booted, in milliseconds. It is counted in realtime and will count even when the system enters sleep mode. 

          
           
long elapsedTime = SystemClock.elapsedRealtime();

elapsedRealtimeNanos()

This method works similarly to elapsedRealTime(), but it returns the time in nanoseconds instead of milliseconds. 

          
           
long elapsedTimeNano = SystemClock.elapsedRealtimeNanos();

setCurrentTimeMillis(long millisecond)

This method sets the current time of the device in milliseconds since January 1, 1970 00:00:00 GMT. It requires a long value representing the number of milliseconds since this date. 

          
           
long millis = System.currentTimeMillis();
SystemClock.setCurrentTimeMillis(millis);

uptimeMillis()

This method returns the time since the system was booted, in milliseconds. It is slightly different from elapsedRealtime(), as it will not count sleep time in its calculation. 

          
           
long uptimeMillis = SystemClock.uptimeMillis();

Conclusion

The android.os.SystemClock class provides access to several features of the device’s system clock, which can be used for applications that require accurate time measurements or synchronization activities with respect to the system clock. With this class, users can measure elapsed time, set the current time, and get the system uptime in both milliseconds and nanoseconds.
