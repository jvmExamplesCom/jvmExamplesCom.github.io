java.util.logging.Logger

Java.util.logging.Logger is a class used to log messages for a specific system or application component. Loggers are normally named, using a hierarchical dot-separated namespace. Logger names can be arbitrary strings, but they should normally be based on the package name or class name of the logged component.

The Logger class provides methods such as log() and severe() to log messages at different levels of severity. It also provides access to handlers which can be used to write log messages to one or more destinations such as the console, a file, or a database.

The following code example illustrates how to use the Logger class to log messages:


          
           
import java.util.logging.Logger;
 
public class LoggingExample {
    private static final Logger logger = Logger.getLogger(LoggingExample.class.getName());
 
    public static void main(String[] args) {
        logger.severe("This is a severe message");
        logger.warning("This is a warning message");
        logger.info("This is an info message");
        logger.config("This is a config message");
        logger.fine("This is a fine message");
    }
}
The output of the above code is shown below: 

          
           
SEVERE: This is a severe message
WARNING: This is a warning message
INFO: This is an info message
CONFIG: This is a config message
FINE: This is a fine message

In addition to logging messages, Loggers can also be used to filter messages based on the log level. For example, the following code snippet shows how to configure a Logger to only log messages that have a severity of SEVERE or higher:


          
           
import java.util.logging.Level;
 
public class LoggingExample {
    private static final Logger logger = Logger.getLogger(LoggingExample.class.getName());
 
    public static void main(String[] args) {
        logger.setLevel(Level.SEVERE);
        logger.severe("This is a severe message");
        logger.warning("This is a warning message");
    }
}
The output of the above code is shown below: 

          
           
SEVERE: This is a severe message

The logger object also provides methods to add, remove, and list handlers. Handlers are used to write log messages to one or more destinations such as the console, a file, or a database. The following code example shows how to add a FileHandler to a logger:


          
           
import java.io.IOException;
import java.util.logging.FileHandler;
import java.util.logging.Logger;
 
public class LoggingExample {
    private static final Logger logger = Logger.getLogger(LoggingExample.class.getName());
 
    public static void main(String[] args) {
        try {
            FileHandler handler = new FileHandler("logfile.txt");
            logger.addHandler(handler);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

In addition to adding handlers, the Logger class also provides methods to filter messages based on the logger name, log record level, filter, formatter, and log manager. The following code example shows how to filter log messages with a custom filter class:


          
           
import java.util.logging.Filter;
import java.util.logging.LogRecord;
import java.util.logging.Logger;
 
public class LoggingExample {
    public static class MyFilter implements Filter {
        @Override
        public boolean isLoggable(LogRecord record) {
            return record.getLevel().intValue() >= Level.WARNING.intValue();
        }
    }
 
    private static final Logger logger = Logger.getLogger(LoggingExample.class.getName());
    private static final MyFilter myFilter = new MyFilter();
 
    public static void main(String[] args) {
        logger.setFilter(myFilter);
        logger.severe("This is a severe message");
        logger.warning("This is a warning message");
    }
}
The output of the above code is shown below: 

          
           
WARNING: This is a warning message

As you can see, the Logger class makes it easy to log messages and filter them based on severity or custom criteria. You can also use handlers to write log messages to various destinations such as the console, a file, or a database.
