android.content.pm.PackageInfo

android.content.pm.PackageInfo is an Android class used to retrieve meta-data from an application package. It can be used to get information such as the version name, code path, and signatures of an installed application.

Creating a PackageInfo Instance

To create a PackageInfo instance, use the 

          
           
getPackageInfo()
method of the PackageManager class. This method takes two arguments: the application package name and the flag specifying which meta-data should be retrieved. 

          
           
PackageManager pm = getPackageManager();
String packageName = "com.example.app";
int flags = PackageManager.GET_SIGNATURES;
PackageInfo packageInfo = pm.getPackageInfo(packageName, flags);

Getting Version Information

Version information such as the version name and version code can be retrieved using the 


          
           
versionName
and 

          
           
versionCode
fields of the PackageInfo instance. 

          
           
int versionCode = packageInfo.versionCode;
String versionName = packageInfo.versionName;

Getting Code Paths

The code paths of the application's source and native libraries can be retrieved using the 


          
           
applicationInfo
field of the PackageInfo instance. This field contains an ApplicationInfo instance which contains the fields 

          
           
sourceDir
and 

          
           
nativeLibraryDir
. 

          
           
ApplicationInfo appInfo = packageInfo.applicationInfo;
String sourceDir = appInfo.sourceDir;
String nativeDir = appInfo.nativeLibraryDir;

Getting Signatures

The signatures of an installed application can be retrieved using the 


          
           
signatures
field of the PackageInfo instance. This field contains an array of Signature instances which contain the byte array of the application's signature. 

          
           
Signature[] signatures = packageInfo.signatures;
for (Signature signature : signatures) {
    byte[] signatureBytes = signature.toByteArray();
}
