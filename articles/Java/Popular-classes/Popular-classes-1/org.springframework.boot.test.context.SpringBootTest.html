Introduction to Spring Boot Test

The 

          
           
org.springframework.boot.test.context.SpringBootTest
annotation is a powerful tool used in testing applications built with the Spring Framework. It allows developers to test their application’s behaviors without having to actually run a complete instance of it. This makes testing faster and more efficient, while still providing the same level of confidence as a full-fledged integration test.

Basic Use of SpringBootTest

Using the SpringBootTest annotation is simple. All one needs to do is add the annotation to the test class and the necessary features will be loaded into the TestContext:


          
           
@SpringBootTest
public class MyTest {
// Test code here
}

The annotation also supports parameters that can be used to customize the behavior of the TestContext. These parameters include:

  • webEnvironment
    : This can be used to specify whether or not the application should be started with an embedded web server (such as Tomcat). It has three possible values: 
    LIVE_SERVER
    , 
    RANDOM_PORT
    and 
    NONE
    .
  • properties
    : This parameter allows you to pass in additional properties that will be set in the environment when the TestContext is created.
  • classes
    : This parameter allows you to specify which classes should be used to configure the TestContext.

Example: Testing a Web Application with SpringBootTest

Let’s say we have a web application called 
MyApp
that is written with the Spring Framework. To test the application, we can use the following code: 

          
           
@SpringBootTest(webEnvironment = RANDOM_PORT)
public class MyAppTest {
    @LocalServerPort
    private int port;

    @Autowired
    private TestRestTemplate restTemplate;

    @Test
    public void getHello() {
        String response = restTemplate.getForObject("http://localhost:" + port + "/hello", String.class);
        assertEquals("Hello World!", response);
    }
}
In this example, we are using the 
@SpringBootTest
annotation to start the application with a web server on a random port. We are then using the 
@LocalServerPort
annotation to get the port that was used so that we can make requests to the application. Finally, we are using the 
TestRestTemplate
to make a request to the application, and asserting that the response is what we expect it to be.

Conclusion

The 

          
           
org.springframework.boot.test.context.SpringBootTest
annotation is a powerful tool for testing applications written with the Spring Framework. It allows you to quickly and easily test your application without having to spin up a full instance of it. This makes tests faster and easier to write, while still providing the same level of confidence as a full-fledged integration test.
