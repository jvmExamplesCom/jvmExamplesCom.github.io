Javax.persistence.Id

Javax.persistence.Id is an annotation used in Java to facilitate the mapping of a primary key field to a feature of an entity class. It is part of the Java Persistence API and is used to mark a field, or property, as the one representing the id for the class. It can also be used in conjunction with the @GeneratedValue annotation to control the generation of the primary keys. This annotation is typically used when implementing the Entity component of a JPA application.

Annotation Syntax

The syntax for Javax.persistence.Id is as follows: 

          
           
@Id
private Long id;
The first line is the annotation itself. The second line defines a private Long type field which holds the generated identity key for the entity.

Usage and Benefits

To use Javax.persistence.Id with the @GeneratedValue annotation, the relevant code should look like this: 

          
           
@Id
@GeneratedValue
private Long id;
The @GeneratedValue annotation ensures that the Id field will be automatically generated, saving time and effort when creating an entity.

Example Usage

A simple example of Javax.persistence.Id in action is as follows: 

          
           
@Entity
public class Book {

@Id
@GeneratedValue
private Long id;

private String title;

public Long getId() {
   return id;
}

public void setId(Long id) {
   this.id = id;
}

public String getTitle() {
   return title;
}

public void setTitle(String title) {
   this.title = title;
}
}
This example creates an entity object called “Book”, with fields for the book’s id and title. The id field is marked with the @Id and @GeneratedValue annotations. This ensures that the id will be generated automatically once the entity is persisted.

Conclusion

Javax.persistence.Id is an annotation used to mark a field as the primary key for an entity class.
