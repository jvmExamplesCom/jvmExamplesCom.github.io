About javax.annotation.Nonnull

javax.annotation.Nonnull is an important part of the Java programming language that is used to enforce non-nullability when dealing with collections of objects. It makes sure that developers do not accidentally pass null values into a method or operation. This is done by throwing a NullPointerException when a non-null value is expected, as opposed to a normal warning message.

Code Examples


          
           
@Nonnull
String getName() {
    return this.name;
}
The above code example shows how you can use the @Nonnull annotation to make sure a method always returns a non-null value. It will throw a NullPointerException if the method returns a null value instead of the expected non-null one. 

          
           
int add(@Nonnull int a, int b) {
    return a + b;
}
The above code example shows how you can use the @Nonnull annotation to make sure an argument to a method is never null. It will throw a NullPointerException if the argument passed in is null instead of the expected non-null one. 

          
           
List<Integer> list = Arrays.asList(1, 2, null);
list.stream().map(Objects::nonNull).forEach(System.out::println);
The above code example shows how you can use the Objects class to filter out null values from a collection. In this case it will print out 'true' for each element of the list that is not null, and 'false' for each element of the list that is null.
