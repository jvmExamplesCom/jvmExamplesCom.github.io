javax.servlet.http.HttpSession

The 

          
           
javax.servlet.http.HttpSession
class provides an interface between the servlet container and the web application components. It is used to store and retrieve objects from the session context, associated with a particular user's session. The objects stored in the session are accessible for the entire duration of the session.

Creating HttpSession Object

The 

          
           
javax.servlet.http.HttpSession
object can be created by calling the 
getSession()
method on the 
HttpServletRequest
object, as shown below: 

          
           
HttpServletRequest request = (HttpServletRequest) req; 
HttpSession session = request.getSession();
The 

          
           
HttpServletRequest.getSession()
method creates a new session if the request doesn't already have a valid session associated with it. If the request has an active session, then it returns the same.

Storing Objects in Session

The 
setAttribute()
method is used to store an object in the session. It takes two parameters as arguments. The first argument is the name of the session object and the second argument is the actual object to be stored: 

          
           
session.setAttribute("userName", "John Doe");

Here we have stored the username "John Doe" in the session object.

Retrieving Objects from Session

The 
getAttribute()
method is used to retrieve an object from the session. It takes one argument as an argument, which is the name of the session object to be retrieved: 

          
           
String userName = (String) session.getAttribute("userName");

This will return the value of the session attribute "userName", which is "John Doe".

Invalidating Sessions

The 
invalidate()
method is used to invalidate a session and remove all objects stored in it. It takes no arguments: 

          
           
session.invalidate();

This will remove all objects stored in the session. It also removes any authentication information associated with the session.
