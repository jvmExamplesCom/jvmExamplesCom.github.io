java.lang.reflect.Method

The 

          
           
java.lang.reflect.Method
class provides a mechanism to access and modify the behavior of methods in the Java programming language. It is part of the Java Reflection API which allows programs to analyze the structure of Java code. The Method class allows you to access information about a method, such as the declaring class, its modifiers, its return type, its name, its parameters, and its exceptions.

Creating Method Instances

To create an instance of the Method class, an application must first obtain a reference to the Class object for the class or interface that declares the method. Then, the Class object's getDeclaredMethods() or getMethods() method can be called to obtain an array of all the declared methods or all the public methods of the class, respectively.


          
           
// Obtain Method instance for method "toString"
Class c = SomeClass.class; 
Method m = c.getDeclaredMethod("toString");

Invoking a Method

Once you have a Method object for an accessible method, you can invoke it by calling the invoke method of the Method class. This method takes two parameters: the object on which the method is to be invoked (or null for static methods), and an array of objects to be passed as arguments to the method. The number of elements in this array must correspond to the number of parameters in the method.


          
           
// Invoke method "toString" on object "obj"
Object obj = new SomeClass();
Object result = m.invoke(obj, null);

Accessing Method Information

The Class object also provides a number of methods that can be used to access information about a method, such as its modifiers, its return type, its name, its parameters, and its exceptions. The getModifiers(), getReturnType(), getName(), getParameterTypes(), and getExceptionTypes() methods provide this information, respectively.
