Java Security MessageDigest

The Java Security MessageDigest classes allow for creating a digital signature for a message, verifying the digital signature of a message, and creating a hash value for a message. MessageDigest is an abstract class that provides the functionality of a cryptographic hash function. A MessageDigest object starts out initialized. Overview of MessageDigest

A MessageDigest object is an instance of a cryptographic hash function that takes in data and produces a fixed size digest (also known as a message digest or fingerprint) that uniquely identifies the data. This digest is used to verify the integrity of the data, meaning that it has not been changed or tampered with. The MessageDigest class is part of the Java Security package and includes several algorithms for hashing data. These algorithms include SHA-1, SHA-256, SHA-384, SHA-512, MD5, and more. Each algorithm produces a different size digest and are used for different levels of security.

Creating a MessageDigest Instance

Before a MessageDigest object can be used, it must be initialized with an algorithm via the getInstance() static method from the MessageDigest class. Updating the MessageDigest Object

Once a MessageDigest object is initialized, the data to be hashed can be passed to it through the update() method. The update() method accepts a byte array, a byte, a ByteBuffer, and an InputStream as parameters. All of the data passed in through these methods are added to the data that is already in the MessageDigest object, and the data is then hashed into a digest.
