com.bumptech.glide.Glide

Introduction

Glide is an image loading and caching library for Android focused on smooth scrolling. Glide offers an easy to use API, a performant and extensible resource decoding pipeline and automatic resource pooling. Glide's primary focus is on making scrolling any kind of a list of images as smooth and fast as possible, but Glide is also effective for almost any case where you need to fetch, resize, and display a remote image.

Getting Started

To add Glide to your project, include the following in your app/build.gradle file:


          
           
implementation 'com.github.bumptech.glide:glide:4.8.0'

Once the dependency has been downloaded, you can start using Glide in your app. The simplest way to load an image is to directly pass in the URL as a string:


          
           
// Load an image from a URL 
String url = "http://example.com/image.png";
Glide.with(context).load(url).into(imageView);

You can also use the load() method of the RequestBuilder class to load a resource from its Uri or File object:


          
           
// Load an image from a Uri 
Uri uri = ...;
Glide.with(context).load(uri).into(imageView);

// Load an image from a File
File file = ...;
Glide.with(context).load(file).into(imageView);

Glide also has a number of other features, including the ability to pre-fetch images and automatically resize them to fit the size of the ImageView. We will look at some of these features in more detail below.

Pre-Fetching Images

Glide allows you to pre-fetch images for future use, which can help improve the user experience if you know that the user is likely to need certain images later on. The pre-fetch functionality is enabled by the fetch() method of the RequestBuilder class:


          
           
// Pre-fetch an image from a URL 
String url = "http://example.com/image.png";
Glide.with(context).load(url).fetch();

You can also pre-fetch multiple images in a single request:


          
           
// Pre-fetch multiple images 
String[] urls = { "http://example.com/image1.png", "http://example.com/image2.png" };
Glide.with(context).load(urls).fetch();

When pre-fetching an image, Glide will save a copy of the image to the device's internal storage, allowing it to be reused later without needing to re-download the image. This can help save bandwidth and improve the performance of your app.

Resizing Images

Glide also makes it easy to resize images to fit the size of the ImageView. The resize() method of the RequestBuilder class takes two parameters—a width and a height—and will resize the image to the specified dimensions. You can also use the optional third argument to specify whether the resulting image should be a square or a rectangle:


          
           
// Resize an image to a square 
int width = 200;
int height = 200;
boolean isSquare = true;
Glide.with(context).load(url).resize(width, height, isSquare).into(imageView);

// Resize an image to a rectangle
int width = 300;
int height = 200;
boolean isSquare = false;
Glide.with(context).load(url).resize(width, height, isSquare).into(imageView);

Resizing the image can help reduce memory usage, as the image is stored in memory at the size of the ImageView instead of loaded from the network at its full size.

Conclusion

Glide is a powerful library for loading and displaying images in Android applications. It offers an easy to use API, automatic resource pooling, and a performant resource decoding pipeline. It also has the ability to pre-fetch images and automatically resize them to fit the size of the ImageView. Glide is the perfect library for applications that rely heavily on images, as it can help improve performance and save bandwidth.
