android.support.v4.app.FragmentManager

The android.support.v4.app.FragmentManager class provides the framework to manage fragments in an Android application. Fragments are portions of an Activity that can display on screen and receive input from the user. They are usually used to divide a single Activity into multiple self-contained modules.

The FragmentManager class provides several methods for managing fragments, such as adding, removing, and replacing them. It can also manage the back stack of an Activity, allowing users to navigate between different screens. Additionally, it can be used to save and restore fragment state across configuration changes.

Adding Fragments

Fragments can be added to an Activity using the 
add()
method of the FragmentManager. This method takes two parameters: the container ViewGroup that the fragment will be added to, and the fragment instance to be added. 

          
           
FragmentManager fm = getSupportFragmentManager();
Fragment fragment = ExampleFragment.newInstance();
fm.beginTransaction().add(R.id.container, fragment).commit();

Once the fragment has been added, it will be visible on screen. To remove a fragment, use the 

remove()
method of the FragmentManager. 

          
           
fm.beginTransaction().remove(fragment).commit();

Replacing Fragments

Sometimes it is necessary to replace one fragment with another. This can be done using the 

replace()
method of the FragmentManager. This method takes two parameters: the container ViewGroup that the fragment will be added to, and the fragment instance to replace the old one. 

          
           
Fragment newFragment = NewFragment.newInstance();
fm.beginTransaction().replace(R.id.container, newFragment).commit();

Managing the Back Stack

The FragmentManager can also be used to manage the back stack of an Activity. This allows users to navigate between different screens, similar to how web browsers work. The 

addToBackStack()
method allows fragments to be added to the back stack, and 
popBackStack()
can be used to remove them. 

          
           
// Add fragment to the back stack
fm.beginTransaction().replace(R.id.container, newFragment).addToBackStack(null).commit();

// Pop the fragment off the back stack
fm.popBackStack();

Saving and Restoring State

The FragmentManager can also be used to save and restore fragment state when the configuration of an Activity changes. This is done using the 


          
           
onSaveInstanceState()
and 

          
           
onRestoreInstanceState()
methods. 

          
           
@Override
public void onSaveInstanceState(Bundle outState) {
    super.onSaveInstanceState(outState);
    fm.putFragment(outState, "example_fragment", exampleFragment);
}

@Override
public void onRestoreInstanceState(Bundle savedInstanceState) {
    super.onRestoreInstanceState(savedInstanceState);
    exampleFragment = fm.getFragment(savedInstanceState, "example_fragment");
}

Using these methods, it is possible to save and restore the state of a fragment across configuration changes, allowing the user experience to be preserved.
