android.content.BroadcastReceiver

The android.content.BroadcastReceiver class is a system-provided class that allows developers to register for and receive notifications from an application execution context. It can be used as part of an Intent filter to receive broadcasts from the system or from another application.

BroadcastReceiver objects are registered using IntentFilter objects and then receive broadcasts when they match the IntentFilter criteria. These broadcast events can be used to trigger further action on the receiving side, such as launching an Activity, making a network request, or even performing long running tasks.

Syntax


          
           
public abstract class BroadcastReceiver extends Object { 
    public void onReceive (Context context, Intent intent); 
}

The BroadcastReceiver class takes a Context object and an Intent object as parameters. The Context object can be used to access resources for the app, while the Intent object contains additional information about the broadcast event, such as the action and data associated with it.

Examples

Here’s a simple example of a BroadcastReceiver that will display a toast message when the device's battery level has changed:


          
           
public class BatteryChangedReceiver extends BroadcastReceiver { 
    public void onReceive (Context context, Intent intent) { 
        String action = intent.getAction(); 
        if (action.equals(Intent.ACTION_BATTERY_CHANGED)) { 
            int level = intent.getIntExtra(BatteryManager.EXTRA_LEVEL, -1); 
            Toast.makeText(context, "Battery level changed: " + level, Toast.LENGTH_SHORT).show(); 
        } 
    } 
}

In order for this BroadcastReceiver to receive notifications about the battery level, it must first be registered with the system. This can be done in the app's AndroidManifest.xml file as follows:


          
           
<receiver android:name=”.BatteryChangedReceiver” > 
    <intent-filter> 
        <action android:name=”android.intent.action.BATTERY_CHANGED” /> 
    </intent-filter> 
</receiver>

Now the BatteryChangedReceiver will be invoked every time the battery level changes. This example can be extended to handle other system broadcasts as well, such as 


          
           
Intent.ACTION_CONFIGURATION_CHANGED
or 

          
           
Intent.ACTION_PACKAGE_ADDED
.

Conclusion

The android.content.BroadcastReceiver class provides a powerful way to receive system broadcasts and trigger further action in response. It can be used to monitor system events such as the battery level, configuration changes, or package installations, as well as to launch Activities, make network requests, or perform long running tasks.
