android.net.Uri in Java

The URI class in Java provides a uniform way to address a resource. The android.net.Uri class is an extension of this class to cover Android-specifics, like content Uri and intent Uri. This article will discuss the features of android.net.Uri and provide code samples and output examples.

Constructors

The Uri class provides several different constructors:


          
           
Uri uri = new Uri (String uriString);
Uri uri = Uri.parse(String uriString);
Uri uri = Uri.fromParts(String scheme, String ssp, String fragment);

The first constructor takes a String as an argument and creates a Uri object from it.


          
           
Uri uri = new Uri ("https://www.example.com/path?query#fragment");
System.out.println(uri.toString()); //prints "https://www.example.com/path?query#fragment"

The second constructor is similar to the first, except that it calls 


          
           
parse()
method internally. This is a convenient way to quickly create simple URIs. 

          
           
Uri uri = Uri.parse("https://www.example.com/path?query#fragment");
System.out.println(uri.toString()); //prints "https://www.example.com/path?query#fragment"

The third constructor is used when more control over the construction of the URI is needed. This constructor takes three strings as arguments: 


          
           
scheme
, 

          
           
ssp
and 

          
           
fragment
. The 

          
           
scheme
specifies the protocol used by the URI. The 

          
           
ssp
stands for Scheme-Specific Part, which contains the rest of the URI excluding the protocol and fragment. Finally, the 

          
           
fragment
is the optional part at the end of the URI that is separated from the rest of the URI by a ‘#’ symbol. 

          
           
Uri uri = Uri.fromParts("http", "www.example.com", "/path?query#fragment");
System.out.println(uri.toString()); //prints "http://www.example.com/path?query#fragment"

URI Components

The Uri class provides methods to retrieve the components of a URI string. The following methods are examples of some useful components:


          
           
getScheme(): returns the protocol of the URI as a String.
getHost(): returns the hostname of the URI as a String.
getPath(): returns the path of the URI as a String.
getQuery(): returns the query parameters as a String.
getFragment(): returns the fragment of the URI as a String.

These methods are useful when parsing a URL and extracting its components. For example, consider the following URI:


          
           
Uri uri = Uri.parse("http://www.example.com/path/to/resource?arg1=val1&amp;arg2=val2#fragment");
System.out.println(uri.getScheme()); //prints "http"
System.out.println(uri.getHost()); //prints "www.example.com"
System.out.println(uri.getPath()); //prints "/path/to/resource"
System.out.println(uri.getQuery()); //prints "arg1=val1&amp;arg2=val2"
System.out.println(uri.getFragment()); //prints "fragment"

Building a URI

The Uri class also provides methods to build a URI string from its components. These methods are useful when creating a URI from scratch or modifying an existing one. The following methods are examples of some useful components:


          
           
buildUpon(): returns a builder object with which components can be added or removed.
withAppendedPath(): adds a path to the existing path of the URI.
withPath(): sets the path of the URI.
withScheme(): sets the scheme of the URI.
 appendQueryParameter(): adds a query parameter to the existing list.
For example, consider the following URI: 

          
           
Uri originalUri = Uri.parse("http://www.example.com/path/to/resource?arg=val#fragment");
Uri.Builder builder = originalUri.buildUpon();
builder.appendQueryParameter("newArg", "newVal");
Uri modifiedUri = builder.build();
System.out.println(modifiedUri.toString()); //prints "http://www.example.com/path/to/resource?arg=val&amp;newArg=newVal#fragment"

Conclusion

In conclusion, the android.net.Uri class in Java provides a uniform way to address resources on an Android device. It has several constructors to create URIs, and provides methods to retrieve and modify its components. This makes it a powerful and convenient tool to use when dealing with URIs.
