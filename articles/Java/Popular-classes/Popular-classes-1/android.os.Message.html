Introduction to android.os.Message

android.os.Message is a class used for sending messages between different parts of an application. It enables communication between different threads and even across processes on different devices. It can be used for anything from sending notifications to passing data between activities, services, and other components. It can also be used for inter-process communication (IPC). In this article, we'll explore the basics of android.os.Message, and take a look at some code examples.

Creating and Sending Messages

Creating a message is easy. Simply call android.os.Message.obtain() with an optional Handler object, a message id, and the data you want to send. You can then send the message via a Handler. For example:


          
           
Handler handler = new Handler();
Message msg = Message.obtain(handler, 1, "Hello World!");
handler.sendMessage(msg);

You can also create messages without a Handler by calling android.os.Message.obtain(null, id, data) instead. You'll need to set the Message's target Handler manually before sending it:


          
           
Message msg = Message.obtain(null, 2, "Hello World!");
msg.setTarget(handler);
handler.sendMessage(msg);

Handling Messages

When a message is sent, its receiver will receive it in its handleMessage() method. To listen for messages, simply override the handleMessage() method of your activity or service, and then set it as the target of the message using setTarget() or via the obtain() method. 


          
           
MyActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) { 
        super.onCreate(savedInstanceState);

        // ...

        Handler handler = new Handler() {
            @Override
            public void handleMessage(Message msg) {
                String data = msg.obj;
                Log.d("MyActivity", data);
            }
        };

        msg.setTarget(handler);
        handler.sendMessage(msg);

        // ...
    }
}

Getting Data From Messages

To get data out of a message, use its member variable obj. This variable can be set to any type of Object, so you can send custom objects or serializable classes as well. You can also use the what field to pass an int value - this is often used as a message identifier.


          
           
Message msg = Message.obtain(null, 3, myCustomObject);
myCustomObject obj = (myCustomObject) msg.obj;
int id = msg.what;

Conclusion

android.os.Message is a powerful tool for inter-thread and inter-process communication. It can be used for sending notifications, passing data, and even for exchanging objects.
