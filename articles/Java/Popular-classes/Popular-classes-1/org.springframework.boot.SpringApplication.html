org.springframework.boot.SpringApplication

The 
SpringApplication
class provides a convenient way to bootstrap a Spring application that is started from a Java main method. The primary purpose of the 
SpringApplication
class is to create an application context from one or more configuration sources, and then run the applications from the same application context.

Creating an Application Context

The 
SpringApplication
class can be used to create an application context from sources such as a Java class, XML configuration, or an annotation-driven configuration. The following example uses a Java configuration class to create an application context: 

          
           
public class MyConfiguration {
        @Bean
        public HelloWorld helloWorld() {
            return new HelloWorld();
        }
    }

public class MyApplication {
        public static void main(String[] args) {
            SpringApplication.run(MyConfiguration.class, args);
        }
    }
The 

          
           
SpringApplication.run
method creates an application context from the given configuration class, and then starts the application in the context. It is important to note that a single application context can only be started once.

SpringApplication
class has several convenience methods for dealing with command line arguments. The following example shows how to parse command line arguments: 

          
           
public class MyApplication {
        public static void main(String[] args) {
            Options options = new Options();

            CommandLineParser parser = new DefaultParser();

    }

run
method. This allows you to add command line options to the application without needing to parse them manually: 

          
           
public class MyApplication {

By default, the 
SpringApplication
class launches the application in development mode. This means that certain features, such as auto-configuration, are enabled. To launch the application in production mode, you can use the

SpringApplication.run(MyApplication.class, args, ProductionMode.ON);

This will disable certain features, such as auto-configuration, that are not necessary in production. It is recommended that you always launch your application in production mode when deploying to a production environment.
