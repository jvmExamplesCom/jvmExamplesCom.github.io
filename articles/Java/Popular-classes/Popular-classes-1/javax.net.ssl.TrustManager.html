javax.net.ssl.TrustManager

The javax.net.ssl.TrustManager class is a core part of the Java Secure Socket Extension (JSSE). It provides a set of classes and methods for verifying the authenticity of an incoming communication. The TrustManager allows Java components to accept or reject particular incoming connections based on their trustworthiness.

Overview

A TrustManager is an entity that can validate an encrypted connection. It makes sure that data travelling over the connection is both secure and authentic. It is responsible for handling the certificate exchange process between two parties, also known as the handshake process.

When two parties establish a secure connection, they exchange certificates. The TrustManager verifies that the other party’s certificate is valid and trustworthy. If it is not, the TrustManager will not allow the connection to be established.

TrustManager API

The javax.net.ssl.TrustManager API defines a set of methods for interacting with server certificates. These methods can be used to validate the authenticity of an incoming connection.

The API includes three main methods: checkClientTrusted(), checkServerTrusted(), and getAcceptedIssuers(). The first two methods are used to verify the validity of the server certificate while the third returns a list of certificates that are trusted by the TrustManager instance.
