Java: android.view.Display

Android provides a powerful framework for building applications, designed to easily access the device's hardware and software. One of these components is the android.view.Display class. This class represents an abstract representation of a display, such as that of a television or computer monitor. The android.view.Display class is important because it provides access to a variety of information about the connected display. This includes the width, height, maximum refresh rate, pixel format, and orientation of the display. It also allows you to query the features of a display, such as if it is built-in or not, if it supports virtual reality, and if it supports HDR.

Using android.view.Display

To use the android.view.Display class, you must first obtain an instance of it through the DisplayManager class. The DisplayManager class can be obtained from the Android context like so: 

          
           
DisplayManager displayManager = (DisplayManager) context.getSystemService(Context.DISPLAY_SERVICE);
From there, you can call the getDisplays() method to get a list of displays currently connected to the device. For example: 

          
           
Display[] displays = displayManager.getDisplays();
You can then iterate through the list of displays and find the one you are looking for. Once you have the desired display instance, you can call various methods on it to get information about it, such as its width and height: 

          
           
int width = display.getWidth();
