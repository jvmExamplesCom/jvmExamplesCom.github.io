Introduction to javax.servlet.ServletResponse

The javax.servlet.ServletResponse interface is a Java interface which allows web applications to communicate back and forth with clients. It is part of the Java Servlet API, which provides a way for Java EE web applications to respond to requests. This interface provides methods to write data, set cookies, and set headers, among other things.

Methods

The javax.servlet.ServletResponse interface provides several methods to enable communication between a web application and a client. These include:

  • 
            
             
void addCookie(Cookie cookie)
    – adds a cookie to the response.
  • 
            
             
void addHeader(String name, String value)
    – adds a header with given name and value.
  • 
            
             
String encodeURL(String url)
    – encodes the URL if necessary.
  • 
            
             
void sendError(int sc, String msg)
    – sends an error message with given status code and message.
  • 
            
             
void sendRedirect(String location)
    – sends a redirect to the given location.
  • 
            
             
void setCharacterEncoding(String charset)
    – sets the character encoding of the response.
  • 
            
             
void setContentType(String contentType)
    – sets the content type of the response.

Example

The following example shows how to use the javax.servlet.ServletResponse interface. In this example, we create a servlet that sets the content type, character encoding, and then writes a simple “Hello World” response.


          
           
@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
    public void doGet(HttpServletRequest request, 
                      HttpServletResponse response)
        throws IOException, ServletException {

        response.setContentType("text/plain");
        response.setCharacterEncoding("UTF-8");

        PrintWriter writer = response.getWriter();
        writer.write("Hello World!");
    }
}
In this example, we use the 
setContentType
and 

          
           
setCharacterEncoding
methods to set the content type and character encoding, respectively. We then get the 
PrintWriter
from the response and write our “Hello World” message.

Conclusion

The javax.servlet.ServletResponse interface provides a way for a web application to communicate back and forth with clients. This interface provides methods to set headers, cookies, and to write data.
