org.springframework.http.HttpStatus

The org.springframework.http.HttpStatus class provides a set of enumerated constants that map to server response codes. These constants are used to specify the HTTP status code when constructing an HttpServletResponse object. It can be used to create a response header or to add information to an existing header.

Code Examples

Below is an example of using the HttpStatus class to set the HTTP status code for a response.


          
           
HttpStatus status = HttpStatus.OK;
response.setStatus(status.value());

In the case of an error, the same approach can be used to send back an appropriate error message.


          
           
HttpStatus status = HttpStatus.BAD_REQUEST;
response.setStatus(status.value());

Using the HttpStatus Enum

The HttpStatus enum provides a simple way to use the HTTP status codes. Instead of using the raw numeric values, you can use the constants defined in this class.

For example: 

          
           
// Return "OK"
response.setStatus(HttpStatus.OK.value());

// Return "Unauthorized"
response.setStatus(HttpStatus.UNAUTHORIZED.value());

This makes it easier to keep track of the codes and reduces the possibility of mistakes. Furthermore, since it is an enum, using the compiler-time checks will help ensure that the right value is being used.

Additional Features

if (HttpStatus.isInformational(status.value())) {
    // Process an informational response
}

    // Process a successful response
}

    // Redirect the user
}
