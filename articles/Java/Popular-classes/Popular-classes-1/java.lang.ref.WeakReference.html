Introduction to WeakReferences in Java

Java provides a way of allowing objects to be garbage-collected even though theyâ€™re still referenced by other objects. This type of reference is called a WeakReference. A weak reference points to an object, but doesn't prevent it from being garbage-collected. Weak references are useful for objects that are used in caches.

Creating a WeakReference in Java

Weak References are created using the java.lang.ref.WeakReference class. The following example creates a WeakReference object to a String object: 

          
           
String str = "This is a string";
WeakReference<String> weakString = new WeakReference<String>(str);
The WeakReference object stores a reference to the String object and will return a valid reference as long as the object hasn't been garbage-collected.

Checking if a WeakReference is Valid

    System.out.println("Object is still valid");

    System.out.println("Object has been collected");

WeakReferences and Garbage Collection

WeakReferences are only cleared from memory when the garbage collector runs. In most cases, this happens automatically, but it is possible to force the garbage collector to run by calling System.gc().

Example of a Caching System Using WeakReferences

WeakReferences can be used to implement a caching system. The following example demonstrates how a cache could be implemented using WeakReferences: 

          
           
// Create a map to store the cached objects
Map<String, WeakReference<Object>> cache = new HashMap<>();

cache.put("key", new WeakReference<>(obj));

Conclusion

WeakReferences provide a way of referencing objects without preventing them from being garbage-collected. This can be useful when implementing caching systems, where objects may need to be recreated if they've been garbage-collected.
