Introduction to org.junit.BeforeClass

org.junit.BeforeClass is a JUnit annotation used to run a set of code only once, before the start of all tests. The method annotated with this annotation must be static and must be public. When writing unit tests, it is often desired to execute a certain piece of code only once before all tests are executed. This is the purpose of the BeforeClass annotation. In the following sections we will look at an example of how this annotation can be used.

Example of org.junit.BeforeClass

In this example, we will declare a static List instance and use the BeforeClass annotation to initialize it. Let's start by looking at the imports:


          
           
import org.junit.BeforeClass;
import org.junit.Test;
import java.util.List;
import java.util.ArrayList;

As you can see, we import the BeforeClass annotation from org.junit, the Test annotation from org.junit, a List from java.util and an ArrayList from java.util.

Next, we need to declare our List instance: 

          
           
private static List<string> list;</string>

Now, let's add the BeforeClass annotation to a method that will initialize the List:


          
           
@BeforeClass
public static void setUp() {
    list = new ArrayList<>();
    list.add("Item 1");
    list.add("Item 2");
}

As you can see, the method annotated with the BeforeClass annotation is static and it initializes our List with two Strings. Now let's take a look at a test method:


          
           
@Test
public void testAdd() {
    list.add("Item 3");
    assertEquals(3, list.size());
}

This test method adds a String to the list and asserts that the size of the list is now 3.
