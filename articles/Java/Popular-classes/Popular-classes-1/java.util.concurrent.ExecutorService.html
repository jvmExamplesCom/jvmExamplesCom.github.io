java.util.concurrent.ExecutorService

The java.util.concurrent.ExecutorService is a powerful component of the Java Platform, providing the ability to execute tasks in parallel and on separate threads. It allows developers to maximize the use of system resources, offloading work from one thread or processor to another. ExecutorService provides facilities for representing both asynchronous and synchronous tasks, as well as methods for managing their execution states and control their completion.

Creating an ExecutorService

An ExecutorService instance can be created using the static factory methods of the class java.util.concurrent.Executors:


          
           
// Create an ExecutorService and define it's size
ExecutorService executor = Executors.newFixedThreadPool(5);

Submitting Tasks for Execution

Tasks are submitted for execution using the method submit of the ExecutorService. This method takes a Runnable or a Callable object as its argument. A Runnable is a command object that executes a task in the background, while a Callable is a more sophisticated type of task that returns a result after completion:


          
           
// Create and submit a Runnable
Runnable runnableTask = () -> { System.out.println("Executing Runnable Task"); };
Future<?> future = executor.submit(runnableTask);
 
// Create and submit a Callable
Callable<String> callableTask = () -> { System.out.println("Executing Callable Task"); return "Task Executed"; };
Future<String> future = executor.submit(callableTask);

Awaiting Completion of Tasks

The Future object returned from the submit method provides methods for awaiting the completion of the task, for example:


          
           
// Wait for the completion of the Runnable
try {
    future.get();
} catch (InterruptedException e) {
    // Handle interruption
} catch (ExecutionException e) {
    // Handle exception
}

// Wait for the completion of the Callable with a timeout
try {
    String result = future.get(5, TimeUnit.SECONDS);
} catch (InterruptedException e) {
    // Handle interruption
} catch (ExecutionException e) {
    // Handle exception
} catch (TimeoutException e) {
    // Handle timeout
}

Shutting down an ExecutorService

When the ExecutorService is no longer needed, it should be shut down by invoking its shutdown method. This will stop the service from accepting new tasks, and wait for the completion of all running tasks before shutting down:


          
           
executor.shutdown();
