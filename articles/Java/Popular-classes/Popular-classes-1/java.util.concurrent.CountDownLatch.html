Java.util.concurrent.CountDownLatch

The Java.util.concurrent.CountDownLatch class is a synchronization aid that allows one or more threads to wait until a set of operations being performed in other threads has completed. It is mainly used to make sure that all participating threads start and finish at the same time, and also to prevent any thread from starting its task before others have finished theirs. A CountDownLatch requires an initial count, which defines the number of threads that must call await before it is considered to be done. Once all threads have counted down the latch, all waiting threads are released and can continue with their work.

Example Usage

In this example, we have three threads that need to execute in parallel. The main thread creates a CountDownLatch and sets its initial count to 3. Then, each of the threads calls await on the latch, which causes them to wait until the other threads are done. When all threads have completed their tasks, the main thread's latch is released, allowing the main thread to continue with its execution. 

          
           
// Create a CountDownLatch with initial count of 3
CountDownLatch latch = new CountDownLatch(3);

// Start the first thread
Thread thread1 = new Thread(() -> {
    // Do some work
    // ...
    // Notify that the thread is done
    latch.countDown();
});
thread1.start();

// Start the second thread
Thread thread2 = new Thread(() -> {
    // Do some work
    // ...
    // Notify that the thread is done
    latch.countDown();
});
thread2.start();

// Start the third thread
Thread thread3 = new Thread(() -> {
    // Do some work
    // ...
    // Notify that the thread is done
    latch.countDown();
});
thread3.start();

// Wait for all threads to finish
latch.await();

// All threads are finished, continue with main thread

Methods

The Java.util.concurrent.CountDownLatch class provides several methods for working with the latch. 

          
           
getCount()
This method returns the current count of the latch, i.e. the number of threads that still need to call await on the latch before it can be considered done. 

          
           
await()
This method causes the current thread to wait until the latch is released. If the latch has not yet been released (i.e. its current count is greater than 0), then the thread will block until all participating threads have called await. 

          
           
countDown()
This method decrements the latch's count by one. If the current count is 0, then the latch is released and all waiting threads are allowed to proceed.

With the help of the CountDownLatch, we can ensure that all participating threads start and finish their tasks at the same time.
