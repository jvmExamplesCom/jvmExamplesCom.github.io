org.apache.log4j.Logger

The 

          
           
org.apache.log4j.Logger
class is part of the Apache log4j open source logging framework for Java. It provides powerful, flexible, and efficient methods for logging to adapt to different logging needs. It also allows applications to log messages at runtime using predefined levels, such as INFO, DEBUG, WARNING, and ERROR. This class is intended to provide a basic interface for working with and configuring loggers, in order to enable developers to efficiently and quickly develop applications with proper logging capabilities.

Creating a Logger Instance

A Logger instance can be created with the following statement:


          
           
org.apache.log4j.Logger logger = org.apache.log4j.Logger.getLogger("MyClass");

The above statement returns a Logger instance named "MyClass". It is important to note that the log name is case sensitive, so it should be used consistently throughout the application.

Configuring the Logger

Once a Logger instance is created, its configuration can be fine tuned. This can be done by specifying the log level, setting the logging output target (e.g. file, console), and creating custom filters.

The log level can be set by calling the methods 
setLevel(int level)
, 

          
           
setLevel(String levelStr)
, or 

          
           
setLevel(Level level)
on the created Logger instance. All of these methods take an integer, String, or Level respectively, which will be the log level of this Logger. The log levels are: 
  • TRACE
    – the lowest level, typically used for detailed debugging information
  • DEBUG
    – used for event-level detailed debugging information
  • INFO
    – used to indicate general operational events
  • WARN
    – used to indicate potential problems that could arise
  • ERROR
    – used to indicate actual errors
  • FATAL
    – used to indicate critical errors.

The logging output target can be set by calling the methods 


          
           
setTarget(Object target)
or 

          
           
setTarget(String targetStr)
on the created Logger instance. All of these methods take an Object or String respectively, which will be the output target for this Logger's log messages. The accepted output targets are: 
  • CONSOLE
    – output to the console
  • FILE
    – output to a file
  • NETWORK
    – output to a network socket
  • MEMORY
    – output to memory buffer
Custom filters can be created by implementing the 

          
           
org.apache.log4j.Filter
interface. Once implemented, the filter can be registered with the Logger instance by calling the 

          
           
addFilter(Filter filter)
method on the created Logger instance.

Logging Messages

Once the Logger is configured, messages can be logged by calling the 


          
           
log(int level, Object message)
method on the created Logger instance. This method takes an integer, which indicates the log level, and an Object, which is the message to be logged. The log levels mentioned earlier in this article can be used as the integer value, and the Object can be any type of object.

The following examples illustrate how to log messages at different log levels:


          
           
logger.log(Level.TRACE, "This is a trace log message.");
logger.log(Level.DEBUG, "This is a debug log message.");
logger.log(Level.INFO, "This is an info log message.");
logger.log(Level.WARN, "This is a warning log message.");
logger.log(Level.ERROR, "This is an error log message.");
logger.log(Level.FATAL, "This is a fatal log message.");

Log messages can also be logged with formatting by using the 


          
           
log(int level, String format, Object... arguments)
method on the created Logger instance. This method takes an integer, which indicates the log level, a String which serves as the format of the message, and multiple optional parameters which will be replaced into the provided format String.

The following example illustrates how to log a formatted message:


          
           
logger.log(
    Level.INFO, 
    "This message includes both a %s and a %d.", 
    "string", 
    1);
Which would log the following message: 

          
           
This message includes both a string and a 1.

The 

          
           
org.apache.log4j.Logger
class provides a simple yet powerful logging interface for Java applications. It allows applications to easily configure their loggers and log messages at any log level.
