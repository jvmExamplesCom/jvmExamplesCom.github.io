java.util.concurrent.Executor

The java.util.concurrent.Executor class is designed to handle the execution of tasks by using pool threads. It provides an interface for executing tasks and managing the tasks. The Executor interface has two core methods: execute() and submit().

Executor Interface

The execute() method of the Executor interface is used to execute a Runnable object without returning any value. It is useful for running asynchronous tasks. The syntax of the execute() method is as follows: 

          
           
public void execute(Runnable command)
The submit() method of the Executor interface is used to execute a Callable or Runnable object along with returning its result. It can be used for synchronous tasks. The syntax of the submit() method is as follows: 

          
           
public Future<?> submit(Runnable task)

Executor Examples

Let us take a look at a simple example to understand the Executor interface: 

          
           
import java.util.concurrent.Executor; 
  
class MyTask implements Runnable { 
  
    public void run() 
    { 
        System.out.println("Task is executing."); 
    } 
} 
  
public class MyExample { 
    public static void main(String[] args) 
    { 
        Executor myExecutor = new Executor() { 
            public void execute(Runnable command) 
            { 
                new Thread(command).start(); 
            } 
        }; 
  
        myExecutor.execute(new MyTask()); 
    } 
}
In the above program, an implementation of the Executor interface is created which uses a thread to execute the given tasks. Then, an object of our custom executor is created and it is used to execute a Runnable object. When the MyTask object is passed to the execute() method, it is wrapped in a thread by the executor and the run() method of the Runnable object is executed. The output of the above program is: 

          
           
Task is executing.
