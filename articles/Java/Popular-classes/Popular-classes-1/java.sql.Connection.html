java.sql.Connection

The java.sql.Connection interface provides methods for interacting with a database connection. It allows you to establish a connection to a database, execute SQL statements, commit and rollback transactions, retrieve metadata, and more.

To use the Connection interface, you first need to create a Connection object. This can be done using the DriverManager class, which provides various static methods for creating and managing database connections.

Creating a Connection Object

To create a Connection object, you can use the static method 


          
           
DriverManager.getConnection()
. This method takes a JDBC URL as its argument. Here is an example of creating a Connection object that connects to a MySQL database: 

          
           
String dbURL = "jdbc:mysql://hostname/databasename"; 
String username = "username";
String password = "password";
Connection con = DriverManager.getConnection(dbURL, username, password);

Once the Connection object has been created, it can be used to execute SQL statements.

Executing SQL Statements

The Connection interface has a 
createStatement()
method for creating Statement objects, which can be used to execute SQL statements. 

          
           
Connection con = ...;
Statement statement = con.createStatement();

Once you have created the Statement object, you can use it to execute a SQL query or an update statement. Here is an example of executing a SQL query:


          
           
ResultSet resultSet = statement.executeQuery("SELECT * FROM table_name");

The executeQuery() method returns a ResultSet object, which contains the results of the query. You can then use the ResultSet methods to iterate over the query results.

You can also use the Statement class to execute update statements, such as INSERT, UPDATE and DELETE. Here is an example of executing an INSERT statement:


          
           
int affectedRows = statement.executeUpdate("INSERT INTO table_name VALUES (1, 'value1')");

The executeUpdate() method returns the number of affected rows. In this example, it will return 1 if the update was successful.

Committing and Rolling Back Transactions

With the Connection interface, you can also manage transactions. This means that you can commit some changes to the database and then roll back those changes if something goes wrong.

To begin a transaction, you can use the 
setAutoCommit()
method, which disables the auto-commit mode. By default, the auto-commit mode is enabled, which means that each statement is automatically committed to the database. Here is an example of disabling the auto-commit mode: 

          
           
con.setAutoCommit(false);

Once the auto-commit mode has been disabled, all statements will be part of the same transaction. To commit the changes to the database, you can use the 

commit()
method. Here is an example of committing a transaction: 

          
           
con.commit();
And to roll back a transaction, you can use the 
rollback()
method. Here is an example of rolling back a transaction: 

          
           
con.rollback();

Once the transaction has been committed or rolled back, the auto-commit mode is enabled again.
