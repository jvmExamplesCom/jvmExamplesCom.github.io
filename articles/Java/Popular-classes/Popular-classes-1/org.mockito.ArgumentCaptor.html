org.mockito.ArgumentCaptor in Java

Mockito is a popular open source Java testing framework commonly used for unit testing of Java applications. Mockito provides various tools to effectively test the code written for applications. One such tool is the org.mockito.ArgumentCaptor. It is an utility class designed to capture the arguments passed during method invocations.

What is ArgumentCaptor?

ArgumentCaptor is an object used to capture arguments passed while calling a method. It is a facility provided by mockito as part of its verification API. The object itself is created using the factory method 


          
           
ArgumentCaptor.forClass(Class)
. ArgumentCaptor can be used to capture arguments of type Integer, String and so on.

Example of ArgumentCaptor

Consider the following example: We have a service class that we want to verify if its methods are called correctly throughout the execution of our application. We can use ArgumentCaptor to capture the arguments passed to the service's methods.


          
           
public class MyService {
    public void methodToBeVerified(Integer param1, Long param2) {
        //do something
    }
}

Letâ€™s assume that somewhere in our application we call the methodToBeVerified on an instance of MyService class.


          
           
// create instance of MyService
MyService myService = new MyService();

// call methodToBeVerified with some argument
myService.methodToBeVerified(1, 2L);

We can capture the arguments passed with the help of ArgumentCaptor.


          
           
// define argument captor
ArgumentCaptor<Integer> intCaptor = ArgumentCaptor.forClass(Integer.class);
ArgumentCaptor<Long> longCaptor = ArgumentCaptor.forClass(Long.class);

// capture the arguments
verify(myService).methodToBeVerified(intCaptor.capture(), longCaptor.capture());

// get the value of captured arguments
Integer capturedInt = intCaptor.getValue();
Long capturedLong = longCaptor.getValue();

// verify that the arguments are correct
assertEquals(1, capturedInt.intValue());
assertEquals(2L, capturedLong.longValue());

In the above code snippet, we defined two ArgumentCaptor objects to capture the arguments of the right type passed to the methodToBeVerified. Then, we captured the arguments with the capture() method. Finally, we verified that the values are correct.

Conclusion

org.mockito.ArgumentCaptor is a handy utility class provided by Mockito to capture arguments passed while calling a method. It can be used in combination with the verify() method to check whether the method was called with the proper arguments.
