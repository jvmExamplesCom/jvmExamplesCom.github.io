java.util.Optional

Java.util.Optional is a class included in Java 8 and later versions that offers functionality for working with potentially null objects. It helps developers to avoid null pointer exceptions when dealing with potentially absent values. The optional class allows for the representation of the presence or absence of a value within a single object. It allows a developer to check if a value is present, determine the value, or provide a default value if it is not present.

Example

Let’s explore an example where Optional class can be used. Let’s assume we have a list of objects of type Person and they have a String property called “name”. For each person object, we want to extract their name. The name might be null, so we need to handle that situation.


          
           
List<Person> peopleList = new ArrayList<>();

for(Person p : peopleList) {
    //check if name is present
    if (p.getName() != null) {
        //output name
        System.out.println("Name: " + p.getName());
    }
}

The above code is vulnerable to a “NullPointerException”, if the name value is null. To address this issue, we can use the Optional class. We will update the above example to demonstrate its usage.


          
           
List<Person> peopleList = new ArrayList<>();

for(Person p : peopleList) {
    //convert name to Optional
    Optional<String> nameOpt = Optional.ofNullable(p.getName());

    //check if name is present
    if (nameOpt.isPresent()){
        //output name
        System.out.println("Name: " + nameOpt.get());
    }
}

In this example, we are using the “ofNullable” method to convert the potential null value into an Optional. Then, we can use the “isPresent()” method to check if the name value is present. And, if it is present we can retrieve the value by calling the “get()” method on the Optional.

Additional Features

In addition to being able to handle the presence or absence of values, the optional class also provides a few other useful features. The following example illustrates how the “orElse” method can be used to provide a default value in the event that the value is not present.


          
           
Optional<String> nameOpt = Optional.ofNullable(p.getName());
System.out.println("Name: " + nameOpt.orElse("[unknown]"));

In this example, if the name value is not present, then “[unknown]” will be printed instead. Additionally, the “map” and “flatMap” methods can be used to apply transformation functions to the Optional, or to combine multiple Optionals together.


          
           
//combine 2 Optionals
Optional<String> firstNameOpt = Optional.ofNullable(p.getFirstName());
Optional<String> lastNameOpt = Optional.ofNullable(p.getLastName());
Optional<String> fullNameOpt = firstNameOpt.flatMap(fn -> lastNameOpt.map(ln -> fn+" "+ln));
System.out.println("Full Name: " + fullNameOpt.orElse("[unknown]"));

In this example, if both first and last names are present, then we can use the “flatMap” and “map” operations to create a new Optional containing the full name. Otherwise, if either value is missing, then we will get the default value of “[unknown]”.

The Optional class is a powerful tool that can help developers to deal with potentially blank values in a reliable way. By avoiding NullPointerExceptions, it can improve the overall stability of an application.
