javax.net.ssl.SSLContext

The javax.net.ssl.SSLContext class provides a secure socket protocol implementation which acts as an entry point for secure communication and creates secure sockets using the TLS, SSL and DTLS protocols. SSLContext encapsulates all the necessary components required to use secure communication over the network and requires no external configurations. It can be used to establish encrypted connections and create secure server-side and client-side sockets.

Creating an SSLContext Instance

To create an instance of SSLContext, we need to pass a parameter to its constructor specifying the protocol that needs to be used. For example, to use TLSv1.2 protocol, we can create object of SSLContext as follows: 


          
           
SSLContext sslContext = SSLContext.getInstance("TLSv1.2");

We can also set the trust manager, key manager and list of supported protocols while creating a new instance of SSLContext. The example below shows creating an SSLContext object and setting its trust manager, key manager and list of protocols:

tmf.init(ks);

kmf.init(ks, "changeit".toCharArray());

List<String> enabledProtocols = Arrays.asList("TLSv1.2","TLSv1.1"); 

SSLContext sslContext = SSLContext.getInstance("TLS");

sslContext.setEnabledProtocols(enabledProtocols.toArray(new String[0]));

The getSocketFactory() method of the SSLContext class is used to get an instance of an SSLSocketFactory. We can use this socket factory to create secure socket objects which can be used in communication between a client and a server.


          
           
SSLSocketFactory socketFactory = sslContext.getSocketFactory();

Setting Supported Protocols

The setEnabledProtocols() method of the SSLContext class is used to set the list of protocols that are supported by secure sockets created with this context. The example below sets the list of protocols to TLSv1.2 and TLSv1.1:

sslContext.setEnabledProtocols(enabledProtocols.toArray(new String[0]));
