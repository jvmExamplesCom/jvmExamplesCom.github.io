Java.lang.annotation.RetentionPolicy is an important concept of the Java language that helps to control the visibility of annotations throughout the program life cycle. What is Java.lang.annotation.RetentionPolicy?

Java.lang.annotation.RetentionPolicy is an annotation type meta-annotated with the @Retention annotation that tells the compiler when and how the annotations should be retained. It can be used to control the scope of the annotations within the program, as well as determine when the annotations should be discarded and no longer available. There are three retention levels available: Source (SOURCE), Class (CLASS) and Runtime (RUNTIME).

SOURCE Retention Policy

The Source retention policy is the most restrictive of the three and specifies that the annotation is available only within the source code. This means that once the program is compiled, the annotations are no longer visible and the compilation process will not take them into consideration. Annotations with this policy are useful for providing hints to the compiler rather than being used at runtime. For example, the @SuppressWarnings annotation has Source retention and can be used to instruct the compiler to ignore certain warnings during compilation.

CLASS Retention Policy

The Class retention policy is less restrictive than the Source retention policy and specifies that the annotations are retained after compilation and made available to the class loader and virtual machines. This means that the annotations survive the compilation process and remain available for inspection at runtime. Annotations with this policy are useful for runtime introspection and reflective programming practices.     // code
}

RUNTIME Retention Policy

The Runtime retention policy is the least restrictive of the three and specifies that the annotations are retained not only after compilation, but also during program execution.
