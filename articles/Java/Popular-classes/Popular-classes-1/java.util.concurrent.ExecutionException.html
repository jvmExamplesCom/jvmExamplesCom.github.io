java.util.concurrent.ExecutionException

The java.util.concurrent.ExecutionException is an exception that is thrown when an error occurs while trying to execute a task using the java.util.concurrent package. This type of exception is very important in dealing with concurrency and parallelism because it provides details on why a task failed and how to fix it. In this article, we will discuss the basics of the ExecutionException and how to use it.

What is the ExecutionException?

The ExecutionException is a checked exception which is thrown when an exception is thrown while executing a task. The exceptions thrown by the tasks are wrapped in an ExecutionException so that the details of the underlying exception can be obtained. Once the exception is created, it can be handled by the calling code

How to Use the ExecutionException?

An ExecutionException is used to provide detailed information about what went wrong while executing a task. To use the ExecutionException, you can either wrap your task in a Callable or runnable and catch the exception, or you can use the Executors class to create a thread pool and handle the exception.

Examples of ExecutionException

Let's look at some examples of the ExecutionException to get a better understanding of how it works.

Example 1: Wrapping a Task in a Callable


          
           
Callable<Integer> task = () -> {
    int result = 1/0;
    return result;
};
  
try {
    Integer result = task.call();
} catch (ExecutionException e) {
    System.out.println(e);
}

In this example, we wrapped a task in a Callable and tried to execute it. Since the task was invalid, an ExecutionException was thrown and caught. The output of the program is:


          
           
java.util.concurrent.ExecutionException: java.lang.ArithmeticException: / by zero

Example 2: Using Executors Class


          
           
Runnable task = () -> {
    int result = 1/0;
    System.out.println(result);
};
  
ExecutorService executorService = Executors.newSingleThreadExecutor();

try {
    executorService.submit(task);
    executorService.shutdown();
} catch (ExecutionException e) {
    System.out.println(e);
}

In this example, we created a thread pool using the ExecutorService and submitted our task to it. Since the task was invalid, an ExecutionException was thrown and caught. The output of the program is:


          
           
java.util.concurrent.ExecutionException: java.lang.ArithmeticException: / by zero
