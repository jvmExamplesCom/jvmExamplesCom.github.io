Java.lang.annotation.Retention

Java.lang.annotation.Retention is an annotation type that indicates how long annotations with the annotated type are to be retained, by default. It can be used to indicate whether the annotation is to be stored in the class file, made available at runtime through reflection, or ignored completely.

Usage and Examples

The Retention annotation must be applied to a type (class, interface, enum or annotation type). The annotation includes a policy element that must have one of the following values:

  • RetentionPolicy.SOURCE - The annotation is to be discarded by the compiler.
  • RetentionPolicy.CLASS - The annotation is to be recorded in the class file by the compiler but is not to be retained at runtime.
  • RetentionPolicy.RUNTIME - The annotation is to be recorded in the class file by the compiler and is to be retained at runtime, so it can be retrieved using reflection.

Here is an example of how to use the @Retention annotation and specify the policy:


          
           
@Retention(RetentionPolicy.RUNTIME)
public @interface Bookmark {
    String description();
}

In this example, the @Bookmark annotation can be used on other types and it would be retained at runtime, so it can be retrieved using reflection.

Limitations

It is important to note that the @Retention annotation does not apply recursively; therefore annotations applying to a given type will not apply to any of the members of that type.

Additionally, the @Retention annotation cannot be used to instruct the compiler to keep the annotation associated with the class during compilation but discard it before the class is loaded. This is because the RetentionPolicy.CLASS value only indicates that the annotation should be kept in the class file, not necessarily in the class instance.
