javax.ws.rs.core.Response

javax.ws.rs.core.Response is Java's standard Uniform Resource Identifier (URI) Response object. It provides a way for a client to respond to an HTTP request. The Response object can contain metadata, such as headers, response code, and other information related to the response.

Constructors

The Response class has three main constructors: 

          
           
Response.ok()
Response.status(int statusCode)
Response.status(int statusCode, Object entity)
The 
ok()
constructor returns a 
200 OK
response with an empty body. The 
status()
constructor allows the caller to specify a response code of their choice. The third constructor allows the caller to specify a response code and an entity object, which contains the body of the response.

Methods

The Response class includes several methods that are useful for building and manipulating responses.


          
           
Response.header(String key, String value)
Response.entity(Object entity)
Response.build()
Response.close()
The 
header()
method allows the caller to set HTTP headers for the response. The 
entity()
method sets the body of the response. The 
build()
method finalizes the response and prepares it to be sent back to the client. Finally, the 
close()
method closes the response and releases any associated resources.

Example Usage

The following example shows how to use the Response class to create a response with a custom response code and body.


          
           
Response response = Response.status(404)
                           .header("Content-Type", "application/json")
                           .entity("{\"error\": \"Not Found\"}")
                           .build();

// Send the response
response.close();

In this example, the response code is set to 404 and the body is set to a JSON string. The Content-Type header is also set so that the client knows how to interpret the response body.

Conclusion

javax.ws.rs.core.Response is a powerful and flexible way to create HTTP responses. It allows developers to easily set headers, response codes, and bodies. This makes it ideal for creating RESTful APIs.
