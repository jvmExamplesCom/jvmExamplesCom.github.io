Java.util.concurrent.atomic.AtomicReference

The Java.util.concurrent.atomic.AtomicReference class provides a way to use atomic operations on reference variables. Atomic operations are thread-safe, meaning they can be used in multiple threads running simultaneously. This means that all threads will see the same result and no one will be able to modify the state of the variable without the other threads noticing.

Example 1: Creating an AtomicReference


          
           
AtomicReference<String> ref = new AtomicReference<>("initial");

This creates an AtomicReference of type String with an initial value of "initial".

Example 2: Updating An AtomicReference


          
           
ref.set("updating");

The set method sets the value of the AtomicReference to "updating", making it visible to all threads.

Example 3: Comparing And Swapping An AtomicReference


          
           
boolean success = ref.compareAndSet("initial", "updated");

The compareAndSet method compares the current value of the AtomicReference to the argument passed in. If it matches, the AtomicReference is updated to the new value. In this example, if the current value of ref is "initial", it will be updated to "updated" and the boolean value returned is true.

Example 4: Getting An AtomicReference Value


          
           
String value = ref.get();

The get method returns the current value of the AtomicReference.
