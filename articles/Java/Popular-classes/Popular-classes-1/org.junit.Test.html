Introduction to JUnit Testing using org.junit.Test

JUnit testing is an important part of software development. It is used to test the functionality of code, and detect any bugs or errors, before the code is put into production. The 

          
           
org.junit.Test
annotation is part of the JUnit 5 library, and allows you to create individual unit tests. In this article, we'll explore how to use this annotation, and cover some code examples.

Running a Single Unit Test with the org.junit.Test Annotation

The syntax for running a test with the org.junit.Test annotation is as follows: 

          
           
@Test
public void testMethodName() {}
To run the test, you can call the 

          
           
testMethodName()
method from within the same class. For example, 

          
           
public class TestClass {
	@Test
	public void testMethodName() {
		//test code here
	}
	public static void main(String[] args) {
		TestClass tc = new TestClass();
		tc.testMethodName();
	}
}
In this example, we are running the 

          
           
testMethodName()
method, which has been annotated with the 

          
           
@Test
annotation.

Writing Your Own Unit Tests with the org.junit.Test Annotation

The simplest way to write a unit test is to use the assertEquals() method from the org.junit.Assert class. This method takes two parameters - expected output, and the actual output. If the two parameters match, then the test passes. If not, then the test fails. For example, 

          
           
@Test
public void testMethodName() {
	int expected = 10;
	int actual = 15;
	assertEquals(expected, actual);
}
In this example, the 

          
           
expected
value is 

          
           
10
, while the 

          
           
actual
value is 

          
           
15
. Therefore, the test will fail, since the two values do not match.

Using Parameters with the org.junit.Test Annotation

If you want to run multiple tests with different input data, you can pass parameters to the unit test method. For example, 

          
           
@Test 
public void testMethodName(int num1, int num2) {
	int expected = num1 + num2;
	int actual = 20;
	assertEquals(expected, actual);
}
In this example, we have passed two parameters to the 

          
           
testMethodName()
method. These parameters will be used to calculate the expected output. We then compare the expected output to the 

          
           
actual
output, and if the two match, the test passes.

Conclusion

In this article, we have explored how to use the 

          
           
org.junit.Test
annotation to create and run unit tests. We have also gone through code examples to show how to use parameters with the annotation. With this information, you should now be able to write and run your own unit tests using 

          
           
org.junit.Test
.
