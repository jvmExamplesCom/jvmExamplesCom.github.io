java.net.URISyntaxException

The java.net.URISyntaxException class is an unchecked exception that indicates that a string could not be parsed as a URI reference. This exception is thrown when a string is passed to the URI class constructor or when the URI class methods are invoked and the given string does not conform to the expected syntax for a URI.

Example

The following code snippet creates a URI from a string. If the string is not a valid URI reference, this code will throw a URISyntaxException:


          
           
try {
    URI uri = new URI("https://www.example.com/");
} catch (URISyntaxException e) {
    System.out.println("Invalid URI");
    e.printStackTrace();
}

Constructors

java.net.URISyntaxException has two constructors :

  • 
            
             
URISyntaxException(String input, String reason)
    This constructor takes two strings as parameters : an input string , an error message
  • 
            
             
URISyntaxException(String input, String reason, int index)
    This constructor takes three parameters : an input string, an error message, an index at which error occurred.

Methods

java.net.URISyntaxException has some methods which can be used for understood the cause of exception:

  • String getInput()
    This method returns the input which caused the error.
  • String getReason()
    This method returns the detail of why the input was rejected.
  • int getIndex()
    This method returns the index at which the error occurred.
