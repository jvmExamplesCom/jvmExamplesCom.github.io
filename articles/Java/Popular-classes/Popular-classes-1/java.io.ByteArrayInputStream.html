java.io.ByteArrayInputStream

java.io.ByteArrayInputStream class implements an input stream that uses a byte array as the source of data. It is mostly used to read data from a byte array or to write data to a byte array.

Creating a ByteArrayInputStream

A ByteArrayInputStream can be created using the following constructor:


          
           
public ByteArrayInputStream(byte[] buf)

where buf is the byte array which will be used as the source of input. Here is an example:


          
           
byte[] b = {1, 2, 3, 4, 5};
ByteArrayInputStream bais = new ByteArrayInputStream(b);

Reading Data from a ByteArrayInputStream

A ByteArrayInputStream can be read using the read() method. The read() method returns one byte at a time from the underlying byte array. Here is an example that reads three bytes from the ByteArrayInputStream:


          
           
int read1 = bais.read();
int read2 = bais.read();
int read3 = bais.read();

The read() method can also be used to read a subset of the underlying byte array into a specified byte array:


          
           
byte[] b = new byte[5];
bais.read(b, 0, 5);

The above example will read five bytes from the underlying array and store them in the byte array b.

Writing Data to a ByteArrayInputStream

Data can be written to a ByteArrayInputStream using the write() method. This method takes one byte at a time and adds it to the end of the underlying byte array. Here is an example that adds two bytes to the underlying array:


          
           
bais.write(10);
bais.write(20);

The write() method can also be used to write a subset of the underlying byte array. Here is an example that writes a five-byte array to the end of the underlying array:


          
           
byte[] b = {1, 2, 3, 4, 5};
bais.write(b, 0, 5);

Closing a ByteArrayInputStream

A ByteArrayInputStream should be closed when it is no longer needed. It can be closed using the close() method:


          
           
bais.close();

Closing a ByteArrayInputStream will not close the underlying byte array, so it is important to make sure that any references to it are removed when the ByteArrayInputStream is no longer needed.
