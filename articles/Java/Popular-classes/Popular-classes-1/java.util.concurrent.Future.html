java.util.concurrent.Future

The java.util.concurrent.Future interface provides a result-bearing facility for asynchronous computation. This interface is implemented by classes that perform computations that return a result. A Future object serves as a handle to a future result, and can be used by multiple threads to monitor the progress of a computation. It also serves as a means of cancelling a running computation.

A Future object may be in one of three states:
  • Waiting: The computation has not yet started, or is not complete.
  • Completed: The computation has finished. The Future object now contains the result of the computation.
  • Cancelled: The computation was cancelled before it could complete.

Example

Let's say we want to submit a task for execution and store the resulting value in a Future object. We can do this with the following code:


          
           
// Create a Future to store the task result
Future<Integer> result = executor.submit(() -> { 
    // Do some work here
    return 42; 
});

In this example, the executor is an instance of an ExecutorService, and the task produces an Integer result. This task will then be submitted to the executor, which will assign it to a thread for execution. When the task has finished executing, the result will be stored in the Future object.

We can check the state of the result by calling the isDone() method:


          
           
boolean isDone = result.isDone();

This will return true if the task has finished and false otherwise. We can also retrieve the result of the computation by calling the get() method:


          
           
int resultValue = result.get();

If the task has not yet finished executing, the get() method will block until the result is available. Otherwise, the result of the computation will be returned immediately.

We can also cancel a task that has not yet started by calling the cancel() method:


          
           
boolean cancelled = result.cancel(true);

This will attempt to cancel the task if it has not yet started. If it is successful, the cancelled flag will be set to true, otherwise false.

Finally, we can wait for the result to become available by calling the get() method with a timeout:


          
           
int resultValue = result.get(10, TimeUnit.SECONDS);

This will block until the result becomes available, waiting at most 10 seconds. If the result is not available within the timeout, a TimeoutException will be thrown.
