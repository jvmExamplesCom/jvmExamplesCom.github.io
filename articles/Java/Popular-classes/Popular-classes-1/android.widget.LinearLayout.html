Android.widget.LinearLayout

Android.widget.LinearLayout is a ViewGroup subclass that arranges its children in a single column or row. It allows you to align elements horizontally or vertically in a linear fashion. LinearLayout is one of the most used layout classes in Android.

Creating a Layout Using XML

A LinearLayout can be created using an XML layout resource file: 

          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:orientation="vertical" >
    <TextView 
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!" />
</LinearLayout>
The layout_width and layout_height attributes of the 

          
           
<LinearLayout>
element specify that the LinearLayout should stretch to take up all available space. The orientation attribute allows you to specify which direction the elements contained within the LinearLayout should be arranged in. In this example, we have set the orientation to vertical, meaning that the elements contained in the LinearLayout will be arranged vertically.

Adding Elements to a LinearLayout

Elements can be added to a LinearLayout using either XML or programmatically. To add elements using XML, simply add any desired View elements as children of the LinearLayout element. For example, this XML will create two buttons side-by-side within a LinearLayout: 

          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:orientation="horizontal" >

    <Button android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Button 1"/>

    <Button android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Button 2"/>

</LinearLayout>
Alternatively, you can use the 

          
           
addView()
method to add elements to a LinearLayout programmatically: 

          
           
LinearLayout linearLayout = (LinearLayout) findViewById(R.id.layout);

Button btn1 = new Button(this);
btn1.setText("Button 1");
linearLayout.addView(btn1);

Button btn2 = new Button(this);
btn2.setText("Button 2");
linearLayout.addView(btn2);

Layout Attributes

LinearLayout provides several attributes which can be used to control how views are laid out. For example, the 

          
           
android:layout_weight
attribute specifies the relative weight of a view. This can be used to set the relative sizes of elements in the layout. By default, each view will be given the same weight. By setting the layout_weight of one view to be greater than the others, it will occupy more space in the layout. 

          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:orientation="horizontal" >

    <Button android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Button 1"
            android:layout_weight="1"/>

    <Button android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="Button 2"
            android:layout_weight="2"/>

</LinearLayout>
In this example, Button 1 and Button 2 will be displayed side-by-side, with Button 1 occupying less space and Button 2 occupying more space.

Conclusion

Android.widget.LinearLayout is a powerful and versatile layout class which can be used to create simple, complex, and interactive user interfaces. By using XML attributes and the 

          
           
addView()
method, you can arrange your UI elements in a linear fashion. You can also use the 

          
           
android:layout_weight
attribute to assign each view a relative weight, allowing you to create dynamic layouts which respond to different screen sizes or device orientations.
