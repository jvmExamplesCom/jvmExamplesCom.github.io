Java.util.ResourceBundle

Java.util.ResourceBundle is a type of a Java class that is used to store localized text and other resources in the Java application. It provides an API for loading and managing resource bundles that are stored in different files. It also offers access to resources using localized keys for better internationalization support. ResourceBundle is designed to be used by various languages, including Arabic, big5, Chinese, Cyrillic, Greek, etc. This class enables the developers to make their applications more portable across different languages with ease.

Using ResourceBundle

The general usage of Resource Bundle is to provide internationalized message support to the application. To do this, we need to create our own resource bundle file which includes all the localized strings that we would like to display in our application. All these strings must have localized keys associated with them.

We can create the .properties file manually or use any IDE to create it with the help of a wizard or autogenerated code. To load the resource bundle, we must first create a new instance of ResourceBundle class and pass it the fully qualified name of the resource bundle file we want to use:


          
           
ResourceBundle bundle = ResourceBundle.getBundle("com.mypackage.myresources");

Once the ResourceBundle instance is created, we can then use it to read our localized strings by providing the corresponding key. For example, if we have the following entry in our properties file:


          
           
title=Welcome
We can then read it using the following code: 

          
           
String welcomeMessage = bundle.getString("title");

With this in place, we can now display our localized string on any page, by simply providing the localized key we are interested in.

Internationalization

ResourceBundle class also offers support for internationalization. Internationalization is a process of adapting applications for multiple language support. It is important to understand that internationalization is a process and not only a single action. To successfully internationalize our applications, we must consider several aspects such as language, date and time formats, currency, etc.

Resource Bundle class helps us to internationalize our application with ease. We can use ResourceBundle to store localized strings for each language, as well as other resources such as images, audio and video in different files. We can then access these resources using their localized keys in our application.
