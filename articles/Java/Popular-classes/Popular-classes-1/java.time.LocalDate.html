java.time.LocalDate

The java.time.LocalDate class is a date-time class that represents a date without time in the ISO-8601 calendar system, such as 2018-04-24. It provides an API for manipulating and querying dates, and is used by many applications.

Creating a LocalDate

A LocalDate can be created by specifying a year, month, and day. It can also be created from the current date or from a Calendar object.


          
           
LocalDate today = LocalDate.now();

LocalDate aprilFoolsDay = new LocalDate(2018, Month.APRIL, 1);

Calendar calendar = Calendar.getInstance();
calendar.set(2017, 1, 3);
LocalDate thirdFeb2017 = LocalDate.fromCalendarFields(calendar);

Querying Fields from a LocalDate

LocalDate provides getter methods for retrieving the year, month, and day. It also provides utility methods for accessing other fields, such as the week of year, day of week, and day of month.


          
           
LocalDate aprilFoolsDay = new LocalDate(2018, Month.APRIL, 1);

int year = aprilFoolsDay.getYear();                    // 2018
Month month = aprilFoolsDay.getMonth();                 // APRIL
int day = aprilFoolsDay.getDayOfMonth();                // 1
DayOfWeek dayOfWeek = aprilFoolsDay.getDayOfWeek();     // MONDAY
int week = aprilFoolsDay.getWeekOfWeekyear();           // 14

Manipulating a LocalDate

LocalDate provides methods for adding and subtracting years, months, and days. These methods return a new LocalDate with the modified date. LocalDate also provides methods for comparing two dates, such as isBefore(), isAfter(), and isEqual().


          
           
LocalDate aprilFoolsDay = new LocalDate(2018, Month.APRIL, 1);

Formatting a LocalDate

The toString() method of LocalDate returns a ISO 8601 compliant date string. This string can be parsed back into a LocalDate using the parse() method.


          
           
LocalDate aprilFoolsDay = new LocalDate(2018, Month.APRIL, 1);

LocalDate parsedDate = LocalDate.parse(dateString);       // 2018-04-01
