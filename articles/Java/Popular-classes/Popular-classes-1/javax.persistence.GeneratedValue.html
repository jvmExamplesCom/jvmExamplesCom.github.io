javax.persistence.GeneratedValue

The javax.persistence.GeneratedValue annotation is used to define the primary key generation strategy for an entity. It provides an additional layer of abstraction over the standard Java language mechanism for generating primary keys. By configuring the annotation, you can control how the primary key is generated in both supported and unsupported database systems.

Usage

To use the javax.persistence.GeneratedValue annotation, you must first identify the field in the entity that is to be a primary key. Then, you must add the annotation to the field, specifying the generation strategy.


          
           
@Entity
public class MyEntity {

    @Id
    @GeneratedValue(strategy= GenerationType.AUTO)
    private Long id;

}

In this example, the Entity “MyEntity” has a field named “id” which is annotated as the primary key and configured to be generated using the AUTO strategy.

Generation Type

The javax.persistence.GeneratedValue annotation supports several strategies for generating primary keys. These strategies are represented by constants in the 

GenerationType
enum and include: 
  • 
            
             
GenerationType.IDENTITY
    : Used to denote that the primary key is generated by the underlying database system.
  • 
            
             
GenerationType.SEQUENCE
    : Used to denote that the primary key is generated using a database sequence.
  • 
            
             
GenerationType.TABLE
    : Used to denote that the primary key is generated using a database table.
  • GenerationType.AUTO
    : Used to denote that the primary key is generated automatically by the persistence provider. The provider will choose an appropriate strategy based on the underlying database system.
The 
GenerationType.AUTO
type is the default and is recommended for most applications. However, if you need to explicitly set the strategy, you can use one of the other types.

Sequences

The 

          
           
GenerationType.SEQUENCE
type is used to denote that the primary key is generated using a database sequence. To configure a sequence, you must provide the name of the sequence via the 
@SequenceGenerator

@GeneratedValue

@Entity
public class MyEntity {

    @Id
    @GeneratedValue(strategy= GenerationType.SEQUENCE, generator="my_seq")
    @SequenceGenerator(name="my_seq", sequenceName="my_sequence")

}

In this example, the Entity “MyEntity” has a field named “id” which is annotated as the primary key and configured to be generated using the SEQUENCE strategy. The sequence name is “my_sequence” and should be created by the database administrator before the application is deployed.

Summary

The javax.persistence.GeneratedValue annotation provides an additional layer of abstraction over the standard Java language mechanism for generating primary keys. It can be used to configure a database sequence, database table, or database identity to generate primary keys. In most cases, it is recommended to use the
