java.util.NoSuchElementException

A 

          
           
java.util.NoSuchElementException
is a RunTimeException that is thrown when an attempt is made to access or retrieve an element from a collection which does not exist. Typically, it is encountered when the collection is iterated, and the next element in the iteration cannot be found.

Example 1

In the example below, an instance of 

          
           
ArrayList<String>
is created, and then an element which does not exist is tried to be accessed by using the 
get()
method. This results in an instance of 

          
           
NoSuchElementException
being thrown: 

          
           
ArrayList<String> list = new ArrayList<>();
list.add('Apple');
list.add('Banana');
list.add('Orange');

// This will throw a NoSuchElementException as 5 is not an existing index
String fifthElement = list.get(5);

Example 2

The next example shows how no such element exception is thrown when an attempt is made to access the next element in an iterator which does not exist. The 

forEachRemaining()
method is used, but because there are no more elements to iterate over, an exception is thrown: 

          
           
ArrayList<String> list = new ArrayList<>();
list.add('Apple');
list.add('Banana');
list.add('Orange');

Iterator<String> iter = list.iterator();
while (iter.hasNext()) {
    System.out.println(iter.next());
}

// This will throw NoSuchElementException as there are no more elements
iter.forEachRemaining(e -> System.out.println(e));

Finally, here is an example that uses the 
Stream#findFirst
method to access the first element of a stream. As this method returns an Optional containing the element, it is necessary to specify what action should be taken if no element is found, which can be done using an 
orElseThrow()
method. If no element is found, a 

          
           
NoSuchElementException
is thrown: 

          
           
List<Integer> list = List.of(2, 4, 6, 8, 10);
