Introduction to org.springframework.boot.context.properties.ConfigurationProperties

The org.springframework.boot.context.properties.ConfigurationProperties annotation is used to bind external application configuration data from property sources such as .properties or .yml files into a Java object. This annotation helps to simplify the development of applications by allowing developers to easily configure external properties in a consistent and organized way, without having to manually read and parse the configuration data.

Using ConfigurationProperties

Using ConfigurationProperties is quite simple. First, add the @ConfigurationProperties annotation to the configuration class or Bean that you want to bind external application configuration data to. Set the prefix attribute to the value of the properties key prefix that you would like to be bound to the configured Bean. Then, simply add the configured fields to the class.


          
           
@ConfigurationProperties(prefix="app")
public class AppConfig {
    private String language;
    private String country;
    
    public String getLanguage() {
        return language;
    }
    public void setLanguage(String language) {
        this.language = language;
    }
    public String getCountry() {
        return country;
    }
    public void setCountry(String country) {
        this.country = country;
    }
}

Finally, add the external configuration data to the property sources. In this example, the external configuration data will be placed in a .properties file with the following content:


          
           
app.language=en
app.country=US

Now, all you need to do is inject the AppConfig bean into your application and you can access the properties in it.

Reading Properties Using ConfigurationProperties

Reading properties using the ConfigurationProperties annotation is quite straightforward. You just need to inject the configured bean into the application and you can access its properties. For example, you can inject and use the AppConfig bean above like this:


          
           
@Autowired
private AppConfig appConfig;

public void printProperties(){
    System.out.println("Language: " + appConfig.getLanguage());
    System.out.println("Country: " + appConfig.getCountry());
}
The output of the above code is: 

          
           
Language: en
Country: US

Accessing Other Property Sources

By default, the ConfigurationProperties annotation will bind properties from the application.properties file. However, you can specify other property sources to bind from by specifying the locations attribute of the annotation. For example, if you have the following yml file:


          
           
app:
  language: es
  country: MX

You can bind properties from that yml file by setting the locations attribute of the ConfigurationProperties annotation like this:


          
           
@ConfigurationProperties(prefix="app", locations="classpath:app.yml")
public class AppConfig {
    private String language;
    private String country;

    public String getLanguage() {
        return language;
    }
    public void setLanguage(String language) {
        this.language = language;
    }
    public String getCountry() {
        return country;
    }
    public void setCountry(String country) {
        this.country = country;
    }
}

Now, the values of the properties will be taken from the yml file instead of the application.properties file.

Conclusion

The ConfigurationProperties annotation makes it easy to bind external configuration data from multiple property sources into Java objects, simplifying the development of applications. By using this annotation, you can easily configure external properties in a consistent and organized way, without having to manually read and parse the configuration data.
