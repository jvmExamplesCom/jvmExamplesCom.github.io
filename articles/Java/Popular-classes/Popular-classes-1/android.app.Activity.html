android.app.Activity

The android.app.Activity class is a standard Android class used to create, manage, and interact with activities. An activity is a single unit of user interaction within an Android application. This class is the primary method of interacting with the user interface and receiving an app's input. It acts as the conduit between the app's View objects and the operating system, allowing developers to create a functioning app with a consistent and predictable behavior.

Creating an Activity

To create an activity, you will need to extend the Activity class and override certain methods. Here is a basic example of how to create an Activity: 

          
           
public class ExampleActivity extends Activity {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.example_activity);
  }

}
In this example, we are extending the Activity class and overriding the onCreate() method. The onCreate() method is called when an Activity is first created, and it is responsible for initializing the Activity's layout and initializing any variables or objects the Activity needs.

Lifecycle of an Activity

The Activity class has a lifecycle associated with it that defines the different states of an activity throughout its lifetime. This lifecycle can be broken down into several distinct states.
  • onCreate(): This method is called when an activity is first created. It is responsible for initializing the activity's layout and variables.
  • onStart(): This method is called when an activity becomes visible to the user.
  • onResume(): This method is called when an activity is brought to the foreground and the user can interact with it.
  • onPause(): This method is called when an activity loses focus and the user cannot interact with it any longer.
  • onStop(): This method is called when an activity is no longer visible to the user.
  • onDestroy(): This method is called when an activity is being destroyed.
It is important to understand the lifecycle of an activity so you can ensure your app behaves in a predictable and consistent way.

Working with Intents

Intents are objects used to communicate with other components in the Android operating system. Activities can create and send intents, as well as receive and respond to intents sent from other components of the operating system. An Intent can be used to start another activity, such as launching a new Activity from within the current one. They can also be used to broadcast messages within the app or even to other apps. Here is an example of how you can create and send an intent from within an Activity: 

          
           
Intent intent = new Intent(this, TargetActivity.class);
intent.putExtra("key", "value");
startActivity(intent);
In this example, we are creating an intent and passing it to the startActivity() method. The startActivity() method is responsible for starting the target Activity and passing it any data included in the Intent.

Conclusion

The android.app.Activity class is a powerful tool for creating a functioning Android application. It allows developers to create activities and interact with the user, as well as create and manage intents and broadcast messages. By understanding the lifecycle of an Activity and how to utilize Intents, developers can create robust and consistent apps that provide a great experience for users.
