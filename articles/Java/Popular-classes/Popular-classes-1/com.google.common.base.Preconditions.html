Introduction to com.google.common.base.Preconditions

The Preconditions class provides a set of methods used to check whether certain conditions are met before proceeding. These preconditions help to ensure that code behaves as expected, and ensures data is valid before being passed in or processed by methods. This article will discuss the use of Preconditions and include some simple code examples demonstrating how to use them.

What is the Preconditions Class?

The Preconditions class is part of Googleâ€™s Guava library and provides a set of static methods for checking if a condition is met prior to running a method, or passing any data or parameters into a method. The main purpose of the Preconditions class is for basic error-checking in the early stages of a program. This can save time and resources in the long-run, as the Preconditions checks only serve to stop bad data from being processed or inadvertently sent to the user.

Using Preconditions

Using Preconditions is simple and straightforward. Simply add a check at the beginning of a method, before executing any data processing or other business logic. For example, the following snippet checks that an Integer is not null before attempting to add it to a total: 

          
           
public int processData(Integer value){
    Preconditions.checkNotNull(value);
    int total = 0;
    total += value;
    return total;
}
In this example, if the value of the Integer comes in as null, then the code will throw an illegal argument exception, and prevent any further processing from occurring.

Testing Conditions with Preconditions

The Preconditions class can also be used for more complex testing. The 

          
           
checkArgument()
method allows for a boolean statement to be tested, such as whether a value is within a certain range. For example: 

          
           
public int processData(Integer value){
    Preconditions.checkArgument(value > 0 &amp;&amp; value < 100);
    int total = 0;
    total += value;
    return total;
}
In this example, the value cannot be less than 0 or greater than 100, otherwise an illegal argument exception will be thrown.

Checking Multiple Conditions

The Preconditions class also provides the ability to check multiple conditions at once. The 

          
           
checkAll()
method checks a sequence of boolean statements and throws an illegal argument exception if any of them fail. This can be useful for quickly and easily confirming that all of a set of conditions are met. For example: 

          
           
public int processData(Integer value1, Integer value2){
    Preconditions.checkAll(
        Preconditions.checkNotNull(value1),
        Preconditions.checkArgument(value2 > 0 &amp;&amp; value2 < 100)
    );
    int total = 0;
    total += value1 + value2;
    return total;
}
In this example, both values must not be null and value2 must be between 0 and 100 for the code to execute correctly. If either of these conditions are not met, an illegal argument exception will be thrown.

Conclusion

The Preconditions class is a powerful tool for performing basic error checks on data before it is processed and can be a useful addition to any programming project. Although it is not a catch-all solution and should only be used to validate simple conditions, it can greatly reduce the amount of time and effort expended on debugging and development in the long run.
