org.springframework.context.annotation.Bean

The org.springframework.context.annotation.Bean annotation is a powerful tool for defining Java classes as beans for use within the Spring Framework. By annotating a class with @Bean, it can be configured as part of the Spring application context, allowing objects to be easily created from such beans. With the help of @Bean annotation, application developers can avoid coding boilerplate, while still being able to use the same Spring configuration.

Usage

In order to use @Bean annotation in an application, the class that is being annotated must be registered in the application context. This can be done by adding the following line of code to the application configuration file: 


          
           
@Configuration
public class MyConfig {
  ...
}
Once the configuration file has been configured, the @Bean annotation can then be used to configure the class: 

          
           
@Bean
public MyClass myBean() {
  return new MyClass();
}
This will define a bean that can be accessed within the application context and can then be referenced in other parts of the application. The name of the bean is determined by the method name, but it can also be explicitly set via the name attribute.

The @Bean annotation can also be used to set properties on the bean instance. For example, to set a string property on the bean, the following syntax can be used:


          
           
@Bean
public MyClass myBean(String prop) {
  MyClass instance = new MyClass();
  instance.setProperty(prop);
  return instance;
}

The @Bean annotation also allows a bean to be defined with an instance of another bean. For example, to define a bean with a dependency on another bean, the following syntax can be used:


          
           
@Bean
public MyClass myBean(OtherClass otherBean) {
  MyClass instance = new MyClass();
  instance.setOtherBean(otherBean);
  return instance;
}

Example

As an example, consider a simple Java class that needs to be defined as a bean within a Spring application context:


          
           
public class MyService {

  private String someProperty;

  public String getSomeProperty() {
    return someProperty;
  }

  public void setSomeProperty(String someProperty) {
    this.someProperty = someProperty;
  }

}

Using @Bean annotation, this class can be defined as a bean in the application context and configured with a value for the someProperty:


          
           
@Configuration
public class MyConfig {

  @Bean
  public MyService myBean(String someProperty) {
    MyService instance = new MyService();
    instance.setSomeProperty(someProperty);
    return instance;
  }

}

Now, when an instance of the MyService class is needed, it can be looked up from the application context:


          
           
ApplicationContext ctx = ... 
MyService service = ctx.getBean(MyService.class);

The @Bean annotation is a powerful tool for defining Java classes as beans for use within the Spring Framework. It can be used to set properties on the bean instance and even reference other beans in the application context. Using the @Bean annotation makes it easy to create beans without having to write any boilerplate code.
