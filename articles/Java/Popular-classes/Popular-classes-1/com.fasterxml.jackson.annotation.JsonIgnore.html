What is com.fasterxml.jackson.annotation.JsonIgnore?

com.fasterxml.jackson.annotation.JsonIgnore is a Jackson annotation used to indicate that a field or property of type should be ignored during serialization and deserialization. It provides a way for developers to control the serialization process when working with objects.

How to Use com.fasterxml.jackson.annotation.JsonIgnore

To use JsonIgnore, you simply add it to the field or property of the Java object. When a field or property is annotated with JsonIgnore, the corresponding value will be omitted from the resulting JSON when serializing the object.

Example

The following example shows how to use the JsonIgnore annotation. We have a simple Person class that contains some basic information about a person. We want to ignore the age field when serializing to JSON, so we add the @JsonIgnore annotation to the field.


          
           
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @JsonIgnore
    public int getAge() {
        return age;
    }
}

Now when we serialize an instance of this Person class to a JSON string, the age field will be omitted:


          
           
Person person = new Person("John Doe", 42);
String json = new ObjectMapper().writeValueAsString(person);
// json will be "{\"name\":\"John Doe\"}"

Excluding Fields in Subclasses

JsonIgnore can also be used to mark fields in subclasses that should be excluded from the JSON representation. This is useful when dealing with inheritance hierarchies, where some fields may not make sense in the JSON representation of a subclass.


          
           
public class Employee extends Person {
    private long id;

    public Employee(String name, int age, long id) {
        super(name, age);
        this.id = id;
    }

    @JsonIgnore
    public long getId() {
        return id;
    }
}

In this example, we have an Employee class which extends Person. The Employee class contains an additional field, id. We want to exclude this field when serializing the Employee to JSON, so we annotate the getId() accessor method with @JsonIgnore. When we serialize an instance of Employee to JSON, the id field will be omitted:


          
           
Employee employee = new Employee("Jane Doe", 21, 1234);
String json = new ObjectMapper().writeValueAsString(employee);
// json will be "{\"name\":\"Jane Doe\"}"

Excluding Fields from All Classes

The JsonIgnore annotation can also be used to mark fields that should be excluded from all classes. This is useful for fields that are common to many classes but should not be included in the JSON representation of any of them. To do this, the annotation must be placed on the field itself, rather than on an accessor method.


          
           
public class Person {
    private String name;
    @JsonIgnore
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public int getAge() {
        return age;
    }
}

public class Employee extends Person {
    private long id;

    public Employee(String name, int age, long id) {
        super(name, age);
        this.id = id;
    }

    public long getId() {
        return id;
    }
}

In this example, we have annotated the age field with @JsonIgnore. This means that the age field will be omitted from the JSON representation of both the Person and Employee classes:


          
           
Person person = new Person("John Doe", 42);
String json = new ObjectMapper().writeValueAsString(person);
// json will be "{\"name\":\"John Doe\"}"

Employee employee = new Employee("Jane Doe", 21, 1234);
json = new ObjectMapper().writeValueAsString(employee);
// json will be "{\"name\":\"Jane Doe\",\"id\":1234}"
