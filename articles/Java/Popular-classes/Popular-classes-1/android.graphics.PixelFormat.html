android.graphics.PixelFormat

The android.graphics.PixelFormat class represents the format of a pixel in an image. It is used to determine which colors should be used for which pixels, and also how many bits are needed for a specific color. In this article, we will discuss the different types of PixelFormats and their uses.

Types of PixelFormats

There are several different types of PixelFormats that can be used with the android.graphics.PixelFormat class:
  • RGB_565: This type of PixelFormat uses 5 bits for red and blue channels and 6 bits for green channel. This is one of the most commonly used formats due to its compact size.
  • ARGB_8888: This type of PixelFormat uses 8 bits for each of the red, green, and blue channels, making it the most versatile of the PixelFormats. It is ideal for creating vivid colors and sharp images.
  • RGBA_F16: This type of PixelFormat is used for high dynamic range (HDR) images. It uses 16 bits for each color channel, allowing for more accurate colors and better representation of color details.
