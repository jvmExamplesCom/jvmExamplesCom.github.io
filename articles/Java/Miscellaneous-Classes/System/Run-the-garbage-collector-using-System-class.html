Using System Class to Run the Garbage Collector in Java

The garbage collector is an important part of Java which is responsible for memory management. The System class provides a static method which can be used to run the garbage collector in Java programs.

Syntax for Running the Garbage Collector Using System Class


          
           
System.gc()

The System.gc() method can be used to call the garbage collector explicitly.

Example of Running the Garbage Collector Using System Class


          
           
public class GCDemo {
    public static void main(String[] args) {
        Runtime rt = Runtime.getRuntime();
        long mem1, mem2;
        Integer someints[] = new Integer[1000];

        System.out.println("Total: " + rt.totalMemory() + " Free: " + rt.freeMemory());
        mem1 = rt.freeMemory();
        System.gc();
        mem2 = rt.freeMemory();
        System.out.println("Refreshed free memory: " + (mem1 - mem2));
    }
}

The above example demonstrates how to run the garbage collector using the System class. The program begins by creating an array of integers. Then, it prints out the total and free memory available to the program. It then calls the System.gc() method to run the garbage collector, and then prints out the difference in the amount of free memory before and after running the garbage collector.

Output


          
           
Total: 6991232 Free: 4870296
Refreshed free memory: 207008

The output of the above program shows that calling the System.gc() method results in an increase in the amount of free memory available for the program.

Conclusion

In conclusion, the System.gc() method can be used to explicitly call the garbage collector in a Java program. This can be used to clean up memory and free up resources which are not being used by the program.
