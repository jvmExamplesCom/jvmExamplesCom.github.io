Performing Binary Search on Java Char Array Example

Java is a powerful programming language that allows developers to create robust applications. One of its many useful features is the ability to perform binary searches on an array of characters. Binary searches are a popular algorithm used to locate an item in an array without having to look through each element. In this tutorial, we will demonstrate how to use the binary search algorithm on a Java char array using an example.

Example of Binary Search On Java Char Array

First, let's look at an example of how to use the binary search algorithm on a Java char array. We will be using a char array of five characters and searching for the character 'c' in the array.


          
           
char[] arr = {'a','b','c','d','e'};
int result = binarySearch(arr, 'c');

System.out.println("Position of element: " + result);

We can call the binarySearch() method as shown above and it will search the array for the specified character. The function returns an integer which indicates the position of the character in the array or -1 if the character is not found.

Implementation of Binary Search On Java Char Array

Now let's look at the implementation of the binarySearch() method. The code below shows the binarySearch() method which takes a char array and the target character as its parameters.


          
           
public static int binarySearch(char[] arr, char x) 
{ 
    int l = 0, r = arr.length - 1; 
    while (l <= r) { 
        int m = l + (r - l) / 2; 
  
        // Check if x is present at mid 
        if (arr[m] == x) 
            return m; 
  
        // If x greater, ignore left half 
        if (arr[m] < x) 
            l = m + 1; 
  
        // If x is smaller, ignore right half 
        else
            r = m - 1; 
    } 
  
    // if we reach here, then element was 
    // not present 
    return -1; 
}

The method uses a divide and conquer approach, starting from the midpoint of the array, to locate the specified character. At each step, the array is halved and the element being searched for is compared to the elements in the middle. If the element is less than the middle element, then the search continues on the left side of the array. Likewise, if the element is greater, then the search continues on the right side of the array. This process of halving the array continues until either the element is found or no more elements are left to search.

Once the search is complete, either the position of the element is returned or -1 if the element is not found in the array. The time complexity of this algorithm is O(log n), making it suitable for large arrays.
