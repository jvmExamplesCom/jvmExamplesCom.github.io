Java HashMap Keys Example: Get, Set View of Keys from a Java HashMap

The HashMap class in Java is an implementation of the Map interface. It is used to store key-value pairs and contains numerous methods for manipulating these pairs, including those that return a "view" of the key set. In this tutorial, we will look at the get, set, and view methods of HashMap that enable us to get and manipulate keys.

Creating a HashMap with Keys

Before we can use the get, set, and view methods of HashMap, we must first create the map. We do so by instantiating a HashMap object and filling it with key-value pairs. Here is an example of how to do this: 

          
           
HashMap<String, String> map = new HashMap<String, String>();
map.put("key1", "value1");
map.put("key2", "value2");
map.put("key3", "value3");
Next, we can use the methods of HashMap to manipulate the keys inside the map.

Get Method of HashMap

The get method of HashMap is used to retrieve the value associated with a given key in the map. The method takes in a key as an argument and returns the associated value, if present. If the key is not present in the map, then the method will return null. Let's look at an example of using the get method: 

          
           
String value1 = map.get("key1"); // value1 = "value1"
String value2 = map.get("key2"); // value2 = "value2"
String value3 = map.get("key3"); // value3 = "value3"
String value4 = map.get("key4"); // value4 = null
As you can see, the get method enables us to retrieve the value associated with a given key in the map.

Set Method of HashMap

The set method of HashMap is used to modify the value associated with a given key in the map. The method takes in two arguments: the key and the new value to associate with the key. If the key is present in the map, then the method will replace the existing value with the new one, and if the key is not present in the map, then the method will add a new key-value pair to the map. Let's look at an example of using the set method: 

          
           
map.set("key1", "newValue1");  // map now contains {"key1": "newValue1", "key2": "value2", "key3": "value3"}
As you can see, the set method enables us to modify the value associated with a given key in the map.

View Methods of HashMap

In addition to the get and set methods, HashMap also provides several methods that enable us to get a "view" of the keys contained in the map. These methods return a collection view of the keys, which can be used to iterate over the keys or manipulate them in any other way. The collection view of the keys returned by these methods are backed by the map, meaning that any changes made to the collection view will be reflected in the map, and vice versa. The view methods of HashMap include the keySet, entrySet, values, and keySet methods. Let's look at an example of each of these methods: 

          
           
Set<String> keys = map.keySet();      // keys = {"key1", "key2", "key3"}
Set<Map.Entry<String, String>> entries = map.entrySet();  // entries = {{"key1", "newValue1"}, {"key2", "value2"}, {"key3", "value3"}}
Collection<String> values = map.values(); // values = {"newValue1", "value2", "value3"}
Set<String> keys2 = map.keySet();     // keys2 = {"key1", "key2", "key3"}
As you can see, the view methods of HashMap provide a convenient way to get a collection view of the keys contained in the map, which can then be used to iterate over or manipulate the keys in any other way.

Conclusion

In this tutorial, we looked at the get, set, and view methods of HashMap that enable us to get and manipulate keys. We saw how to create a HashMap with keys and values, how to get a value associated with a key, how to set a new value for a key, and how to get a collection view of the keys contained in the map. Using these methods, we can easily get and manipulate keys in a HashMap.
