How to Maintain Insertion Order in HashMap

HashMap is a popular Java data structure which provides efficient and fast access to look up values based on keys. However, one of its drawbacks is that it does not maintain the insertion order of its elements. This can be a hindrance when it comes to debugging and tracing code that rely on the entries being in a particular order. In this article, we will explore ways to maintain insertion order in a HashMap.

Using LinkedHashMap

One of the simplest ways to maintain insertion order in a HashMap is to use a special implementation of HashMap, called LinkedHashMap. LinkedHashMap maintains the insertion order of its elements by using doubly linked lists to keep track of the order in which elements are added. Let's look at an example of how to use LinkedHashMap to maintain insertion order: 

          
           
Map<String, String> map = new LinkedHashMap<>();
map.put("A", "Apple");
map.put("B", "Banana");
map.put("C", "Cherry");
System.out.println(map);
The output of the above code will be: 

          
           
{A=Apple, B=Banana, C=Cherry}
As you can see, the output is in the same order as the elements were inserted into the map.

Using Comparator

Another way to maintain insertion order in a HashMap is to use an implementation of Comparator. Comparator is an interface which provides methods for comparing two objects. We can use Comparator to sort our HashMap elements in any order we want. Let's look at an example of how to use Comparator to maintain insertion order: 

          
           
Map<String, String> map = new HashMap<>();
map.put("A", "Apple");
map.put("B", "Banana");
map.put("C", "Cherry");

Comparator<String> comparator = new Comparator<String>() {
  @Override
  public int compare(String s1, String s2) {
    return s2.compareTo(s1);
  }
};

Map<String, String> sortedMap = new TreeMap<>(comparator);
sortedMap.putAll(map);
System.out.println(sortedMap);
The output of the above code will be: 

          
           
{C=Cherry, B=Banana, A=Apple}
As you can see, the output is in the reverse order of the elements that were inserted into the map.

Using Stream API

The last way in which we can maintain insertion order in a HashMap is to use the Stream API. The Stream API provides a powerful set of methods for transforming and manipulating data collections. Let's look at an example of how to use the Stream API to maintain insertion order in a HashMap: 

          
           
Map<String, String> map = new HashMap<>();
map.put("A", "Apple");
map.put("B", "Banana");
map.put("C", "Cherry");

Map<String, String> sortedMap = map.entrySet()
  .stream()
  .sorted(Map.Entry.comparingByKey())
  .collect(Collectors.toMap(Map.Entry::getKey, Map.Entry::getValue));

System.out.println(sortedMap);
The output of the above code will be: 

          
           
{A=Apple, B=Banana, C=Cherry}
As you can see, the output is in the same order as the elements were inserted into the map.

Conclusion

In this article, we explored three different ways to maintain insertion order in a HashMap. We looked at using LinkedHashMap, Comparator, and Stream API, and saw how each can be used to keep the order in which elements were inserted.
