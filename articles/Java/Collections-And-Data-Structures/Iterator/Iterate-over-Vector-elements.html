Iterating Over Vector Elements in Java

Vector is a widely-used class in Java and provides an array-like data structure with dynamic length. It is part of the Java Collections Framework and provides a variety of methods for manipulating elements, including iterating over its contents. In this article, we'll explore different ways to iterate over the elements of a Vector in Java.

Using the Enhanced For Loop

The simplest and most straightforward way to iterate over a Vector is to use the enhanced for loop:


          
           
Vector<String> v = new Vector<String>();
v.add("a");
v.add("b");
v.add("c");

for (String s : v) {
    System.out.println(s);
}

This code will print out the three elements of the Vector (in order), one per line:


          
           
a
b
c

Using Iterators

Another way to iterate over the elements of a Vector is by using an Iterator. An Iterator is an object that provides a way to retrieve and remove elements from a Collection. In this case, you can use an Iterator to iterate over the elements of a Vector:


          
           
Vector<String> v = new Vector<String>();
v.add("a");
v.add("b");
v.add("c");

Iterator<String> it = v.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}

This code will produce the same result as the previous example:


          
           
a
b
c

Using an Iterable Interface

Since Vector implements the Iterable interface, you can also use the forEach() method to iterate over the elements of a Vector:


          
           
Vector<String> v = new Vector<String>();
v.add("a");
v.add("b");
v.add("c");

v.forEach(s -> System.out.println(s));

This code will also print out the three elements of the Vector (in order), one per line:


          
           
a
b
c

Conclusion

In this article, we've explored how to iterate over the elements of a Vector in Java. We've seen how the enhanced for loop, Iterators, and the forEach() method can all be used for this purpose. Each method has its own advantages and disadvantages, so it's important to consider which approach is best for your particular application.
