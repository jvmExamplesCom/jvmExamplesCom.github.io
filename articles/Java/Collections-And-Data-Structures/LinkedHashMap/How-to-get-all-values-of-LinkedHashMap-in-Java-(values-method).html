How to get all values of LinkedHashMap in Java

LinkedHashMap is a popular implementation of the Map interface in Java that stores data in the form of key-value mappings, with entries being ordered according to insertion order. It is important to note that when using LinkedHashMap, the iteration order of the values is based on the order in which elements were inserted into the map. In this article, we will look at how to get all the values stored in a LinkedHashMap using the values() method. We will also learn how to iterate over the values retrieved from the LinkedHashMap and how to handle potential exceptions.

Java LinkedHashMap Values

The 

          
           
values()
method of the 

          
           
LinkedHashMap
class returns a collection view of the values contained in the LinkedHashMap. The returned values are sorted in the same order they were inserted in the map. The signature of the 

          
           
values()
method is as follows: 

          
           
public Collection<V> values()
The 

          
           
values()
method does not take any argument and does not throw any checked exception.

Java LinkedHashMap Values example

Let's understand the above method with the help of an example program. In this example program, we will create a LinkedHashMap and then get the values from it using the 

          
           
values()
method. Finally, we will iterate over the returned collection and print the values. 

          
           
import java.util.Collection;
import java.util.LinkedHashMap;

public class LinkedHashMapValuesExample {

    public static void main(String[] args) {

        // Create a LinkedHashMap
        LinkedHashMap<String, Integer> lhm = 
                            new LinkedHashMap<>();

        // Add elements to the LinkedHashMap
        lhm.put("A", 1);
        lhm.put("B", 2);
        lhm.put("C", 3);
        lhm.put("D", 4);
        lhm.put("E", 5);

        // Get the values from the LinkedHashMap
        Collection<Integer> values = lhm.values();

        // Iterate over the values and print them
        for(Integer value : values){
           System.out.println(value);
        }
    }
}
The above program prints the following output: 

          
           
1
2
3
4
5
As seen in the output, the values are printed in the same order they were added to the LinkedHashMap.

Handling Exceptions

When handling the values returned by the 

          
           
values()
method of the LinkedHashMap class, it is important to remember to catch potential exceptions. The 

          
           
values()
method throws an 

          
           
UnsupportedOperationException
if the LinkedHashMap has been modified after the iterator was created. This means that if the LinkedHashMap is modified after getting the values from it, the iterator might throw this exception. Therefore, it is important to handle this exception whenever using the 

          
           
values()
method. The code snippet below shows how this can be done: 

          
           
import java.util.Collection;
import java.util.LinkedHashMap;

public class LinkedHashMapValuesExample {

    public static void main(String[] args) {

        try {
            // Create a LinkedHashMap
            LinkedHashMap<String, Integer> lhm = 
                                    new LinkedHashMap<>();

            // Add elements to the LinkedHashMap
            lhm.put("A", 1);
            lhm.put("B", 2);
            lhm.put("C", 3);
            lhm.put("D", 4);
            lhm.put("E", 5);

            // Get the values from the LinkedHashMap
            Collection<Integer> values = lhm.values();

            // Iterate over the values and print them
            for(Integer value : values){
               System.out.println(value);
            }
        } catch(UnsupportedOperationException e) {
            System.err.println(e.getMessage());
        }

    }
}
The above code snippet handles the 

          
           
UnsupportedOperationException
which is thrown when the LinkedHashMap is modified after the iterator was created.
