Maintaining Insertion Order of Elements in HashMap

Javaâ€™s 
HashMap
class uses a hash table to store keys and values, which cannot guarantee the order of the elements in which they are inserted. If you want to maintain the insertion order of the values and keys, you can use the 
LinkedHashMap
class.

Using LinkedHashMap Class

The 
LinkedHashMap
class is a specific type of hash map. It stores ("maps") values and keys like a regular hash map, but additionally also preserves the order of the key-value pairings, making it similar to a linked list. This class behaves like a regular hash map when iterating through the map.

Example

Here is an example that uses the LinkedHashMap class to maintain the insertion order of key-value pairs:


          
           
// import the LinkedHashMap class 
import java.util.LinkedHashMap; 

public class Example { 
	
	// create a LinkedHashMap instance 
	private static LinkedHashMap<String, Integer> map = new LinkedHashMap<>(); 

	public static void main(String[] args) { 
		// add some elements to the map 
		map.put("One", 1); 
		map.put("Two", 2); 
		map.put("Three", 3); 

		// print all elements of the map 
		for (String key : map.keySet()) { 
			System.out.println("Key: " + key + ", Value: " + map.get(key)); 
		} 
	} 
}
This code will output the following: 

          
           
Key: One, Value: 1
Key: Two, Value: 2
Key: Three, Value: 3

As you can see, the elements were stored in the same order in which they were inserted.

Conclusion

The 
LinkedHashMap
class can be used to maintain the insertion order of elements in a hash map. This can be useful if you need to iterate over the elements in the same order in which they were added. As you can see from the example above, this is very easy to do.
