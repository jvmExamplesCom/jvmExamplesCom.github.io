Sort LinkedHashMap by Keys in Java

Java LinkedHashMap is an ordered version of HashMap that remembers the order in which keys were inserted into the map. In some cases, you may need to sort LinkedHashMap by its keys. This can be achieved by using a TreeMap with the same key-value pairs or by using a custom Comparator to sort the LinkedHashMap by its keys in natural order or custom order.

Using TreeMap to Sort LinkedHashMap by Keys

The easiest way to sort LinkedHashMap by keys is to create a TreeMap with the same set of key-value pairs. TreeMap is a Red-Black tree based implementation of the NavigableMap interface. It is sorted according to the natural ordering of its keys. 

          
           
// Create a LinkedHashMap 
LinkedHashMap<string, integer> lHashMap = new LinkedHashMap<>(); 

// Add key-value to LinkedHashMap 
lHashMap.put("Geeks", 10); 
lHashMap.put("for", 20); 
lHashMap.put("Geeks", 30); 
lHashMap.put("Welcome", 40); 
  
// Print the LinkedHashMap 
System.out.println("LinkedHashMap: " + lHashMap); 

// Create a TreeMap with the same set of key-value pairs 
TreeMap<string, integer> tMap = new TreeMap<>(lHashMap); 

// Print the TreeMap 
System.out.println("TreeMap: " + tMap);

Using Comparator to Sort LinkedHashMap by Keys

You can also sort LinkedHashMap by its keys using a custom Comparator. The Comparator can be used to sort the keys in either natural order or custom order. 

          
           
import java.util.*; 
  
// A comparator to sort keys in natural order. 
class NaturalOrderComparator implements Comparator<string> { 
    @Override
    public int compare(String o1, String o2) { 
        return o1.compareTo(o2); 
    } 
} 
  
// A comparator to sort keys in reverse order. 
class ReverseOrderComparator implements Comparator<string> { 
    @Override
    public int compare(String o1, String o2) { 
        return o2.compareTo(o1); 
    } 
}

    public static void main(String[] args)
