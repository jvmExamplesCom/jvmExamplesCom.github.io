Checking LinkedHashMap Size in Java

In java, the LinkedHashMap class is used to store objects which are associated with a key and that also maintains the order of insertion. It is important to be able to check the size of the LinkedHashMap so that it can be determined whether it is empty or whether it has enough elements.

Using the size() Method

The most basic way of finding out the size of a LinkedHashMap is to use the 

          
           
size()
method. This method will return the number of elements in the LinkedHashMap: 

          
           
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();

map.put("one", 1);
map.put("two", 2);
map.put("three", 3);

int size = map.size(); // size will be 3
System.out.println("Size of LinkedHashMap is: "+size);

Using the isEmpty() Method

Another way of finding out the size of a LinkedHashMap is to use the 

          
           
isEmpty()
method. This method will return true if the LinkedHashMap is empty and false if it is not: 

          
           
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();

boolean isEmpty = map.isEmpty(); // isEmpty will be true
System.out.println("LinkedHashMap is empty: "+isEmpty);

Using the entrySet() Method

A third way of finding out the size of a LinkedHashMap is to use the 

          
           
entrySet()
method. This method will return a Set of Map.Entry objects which represent the entries in the LinkedHashMap. This set can then be used to get the size of the LinkedHashMap: 

          
           
LinkedHashMap<String, Integer> map = new LinkedHashMap<>();

map.put("one", 1);
map.put("two", 2);
map.put("three", 3);

Set<Map.Entry<String, Integer>> entries = map.entrySet();
int size = entries.size(); // size will be 3
System.out.println("Size of LinkedHashMap is: "+size);

Conclusion

In this article, we have seen three different ways of checking the size of a LinkedHashMap in Java. Each of the methods has its own advantages and disadvantages and so the best method to use depends on the specific situation.
