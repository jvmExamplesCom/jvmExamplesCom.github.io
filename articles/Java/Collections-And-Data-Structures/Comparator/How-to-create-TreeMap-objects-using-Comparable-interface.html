Using Comparable Interface to Create TreeMap Objects in Java

Java provides the 

          
           
TreeMap
class for data structure implementations. This class implements the 

          
           
Map
interface, and provides a way to store key-value pairs of data. It stores information in an ordered fashion by maintaining the keys in sorted order. The keys are sorted according to the compareTo( ) method (or the compare( ) method for older versions) of the objects used as keys. In order to make use of this sorting feature, the object used as a key must implement the 

          
           
Comparable
interface. In this article, we will discuss how to use the 

          
           
Comparable
interface to create TreeMap objects.

Creating TreeMaps by Implementing Comparable

The 

          
           
Comparable
interface provides the compareTo( ) method, which allows objects to be sorted using their instance variables as criteria. The signature of this method looks like this: 

          
           
public int compareTo(Object o)
This method returns an integer based on the comparison of two objects; if the current object is larger than the object passed in, it returns a positive integer; if smaller, a negative integer; if equal,0. The implementation of this method then looks something like this: 

          
           
public int compareTo(Object o) { 
    SomeObject so = (SomeObject)o; 
    if(this.myNum < so.myNum) 
        return -1; 
    else if(this.myNum == so.myNum) 
        return 0; 
    else 
        return 1; 
}
In this example, the comparison is done using the myNum member variable of the SomeObject class. This could have been any other type of data, or multiple variables combined together. Once the 

          
           
Comparable
interface is implemented, the objects can now be sorted within the TreeMap. To learn how to use a TreeMap, let’s look at an example. Consider the following class: 

          
           
public class SomeObject implements Comparable { 
    private int myNum; 
    public SomeObject(int num) { 
        myNum = num; 
    } 
    public int compareTo(Object o) { 
        SomeObject so = (SomeObject)o; 
        if(this.myNum < so.myNum) 
            return -1; 
        else if(this.myNum == so.myNum) 
            return 0; 
        else 
            return 1; 
    } 
}
Using this class, let’s create a TreeMap and populate it with some data. 

          
           
TreeMap<SomeObject, String> treeMap = new TreeMap<SomeObject, String>();
treeMap.put(new SomeObject(5), "Five");
treeMap.put(new SomeObject(7), "Seven");
treeMap.put(new SomeObject(9), "Nine");
treeMap.put(new SomeObject(3), "Three");
Now, to print out the contents of the TreeMap, we can iterate over each element and print out the key-value pair like this: 

          
           
for (Map.Entry<SomeObject, String> entry : treeMap.entrySet()) {
    System.out.println("Key: " + entry.getKey() + " Value: " + entry.getValue());
}
Notice that the output is sorted based on the value of the myNum variable used in the 

          
           
compareTo()
method: 

          
           
Key: Three Value: Three
Key: Five Value: Five
Key: Seven Value: Seven
Key: Nine Value: Nine
This demonstrates how to sort a TreeMap based on an object's 

          
           
compareTo()
method.

Conclusion

In this article, we discussed how to use the 

          
           
Comparable
interface to create TreeMap objects. We saw an example of how to create a TreeMap, sort it according to the value of an object's instance variable, and print out the contents of the TreeMap. With this knowledge, developers should be able to take full advantage of the 

          
           
TreeMap
class, and use it to efficiently store and retrieve data.
