Sorting a LinkedHashMap by Values Using Comparator Interface

In Java, the LinkedHashMap is a collection class that is used to store key/value pairs. It acts like a HashMap, but it maintains the order in which elements were inserted. This can be useful for when you want to maintain the order of entries in the map.

Sometimes, however, you may need to sort the LinkedHashMap by values instead of keys. This can be done using the Comparator interface. A Comparator is a functional interface that defines how two objects of a given type are compared. By implementing this interface, we can create a custom comparator for the LinkedHashMap that will sort its values. In this article, we'll look at how to do this.

Implementing the Comparator Interface

The first step to sort a LinkedHashMap by values is to implement the Comparator interface. This involves implementing the abstract compare() method which takes two arguments—the first being the first object to compare and the second being the second object to compare. Within this method, we can define our own comparison logic between the two objects. This can then be used to sort the LinkedHashMap.


          
           
public class MapValueComparator implements Comparator<Integer, String> {

    public int compare(Integer key1, String value1, Integer key2, String value2) {
        // comparison logic
    }
}

In the example above, we have created a custom Comparator for a LinkedHashMap that stores Integer keys and String values. The compare() method will be called with two objects, each containing the key and value of two different entries in the LinkedHashMap. We can then use these objects to perform our own comparison logic.

For example, if we want to sort the LinkedHashMap based on the length of the strings, our compare() method might look something like this:


          
           
public class MapValueComparator implements Comparator<Integer, String> {

    public int compare(Integer key1, String value1, Integer key2, String value2) {
        int value1Length = value1.length();
        int value2Length = value2.length();

        if (value1Length < value2Length) {
            return -1;
        } else if (value1Length > value2Length) {
            return 1;
        } else {
            return 0;
        }
    }
}

In this case, the compare() method will return -1 if value1 is shorter than value2, 1 if value1 is longer than value2, and 0 if they are the same length.

Sorting the LinkedHashMap

Once we have implemented our Comparator, we can use it to sort the LinkedHashMap. To do this, we can use the Collections.sort() method which takes two arguments—the first being the LinkedHashMap to sort and the second being the Comparator. This will sort the map according to the comparison logic defined in our Comparator.


          
           
LinkedHashMap<Integer, String> map = new LinkedHashMap<>();
// populate the map

MapValueComparator comparator = new MapValueComparator();

Collections.sort(map, comparator);

After running this code, the LinkedHashMap will have been sorted according to the comparison logic defined in our Comparator. For example, if we used the comparison logic defined above, the map will now be sorted by the length of the strings in the values.

Conclusion

In this article, we looked at how to sort a LinkedHashMap by values using the Comparator interface. We implemented the Comparator interface and created a custom comparison logic. We then used this Comparator to sort the LinkedHashMap using the Collections.sort() method.
