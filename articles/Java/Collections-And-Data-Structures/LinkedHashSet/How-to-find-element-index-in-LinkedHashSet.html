How to Find Element Index in LinkedHashSet

LinkedHashSet is a special type of collection in Java that maintains the insertion order of the elements. It extends the HashSet class and implements the Set interface. This makes it easy to store and retrieve data from the collection in the insertion order of elements. In this article, we will discuss how to find an element index in the LinkedHashSet in Java. We will also discuss some basic methods to search for an element in the set and the complexities associated with each method.

Using the LinkedHashSet.indexOf() Method

The LinkedHashSet class provides a built-in method, indexOf(), which can be used to search for an element and return its index. This method takes an element as an argument and returns the index of the element if it is present in the collection. Here is an example of how to use the indexOf() method: 

          
           
LinkedHashSet<Integer> set = new LinkedHashSet<>(); 
set.add(2); 
set.add(3); 
set.add(1); 
int index = set.indexOf(3);
System.out.println(index); // Outputs 1
In this example, we first create a LinkedHashSet and add three elements (2, 3, and 1) to it. We then use the indexOf() method to get the index of the element 3, which is 1. The complexity of the indexOf() method is O(n), where n is the number of elements in the set. This means that it will take linear time to search for an element in the set. Therefore, if the set contains a large number of elements, it may take some time to find the element.

Using Iteration to Find Element Index in LinkedHashSet

We can also use a loop to search for an element in the LinkedHashSet and return its index. This approach requires iterating over the elements of the set and keeping track of the index of the element being searched. Let’s take a look at an example of using a loop to search for an element in the LinkedHashSet: 

          
           
LinkedHashSet<Integer> set = new LinkedHashSet<>(); 
set.add(2); 
set.add(3); 
set.add(1); 

int index = -1; 
Iterator<Integer> iterator = set.iterator(); 
while (iterator.hasNext()) { 
    int element = iterator.next(); 
    index++; 
    if (element == 3) { 
        break; 
    } 
} 
System.out.println(index); // Outputs 1
In this example, we first create a LinkedHashSet and add three elements (2, 3, and 1) to it. We then use a loop to iterate over the elements of the set and keep track of the index of the element being searched. When the element is found, the loop is broken and the index is printed. The complexity of this approach is also O(n), where n is the number of elements in the set. Like the indexOf() method, this approach also has linear time complexity and will take some time if the set has a large number of elements.

Conclusion

In this article, we discussed how to find an element index in the LinkedHashSet in Java. We looked at two approaches – using the LinkedHashSet.indexOf() method and using a loop to iterate over the elements of the set. Both of these approaches have linear time complexity, which means they will take some time if the set contains a large number of elements.
