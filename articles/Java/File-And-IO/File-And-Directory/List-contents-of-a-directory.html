The Java List Directory File System

Java offers an easy way to list the contents of a directory, enabling developers to quickly and easily create file systems. In this article, we will explore the code used to list directory contents in Java and provide several examples to demonstrate how it works.

Listing Directories in Java

To list the contents of a directory in Java, you can use the Files.list() method from the java.nio.file package. This method takes a Path object for the directory you want to list, and returns a Stream of Path objects for the contents of that directory.

For example, to list the contents of the “/tmp” directory, you could do something like this:


          
           
Path dir = Paths.get("/tmp");
try (Stream<Path> listing = Files.list(dir)) {
  listing.forEach(System.out::println);
}
catch (IOException e) {
  System.err.println("Error while listing files: " + e.getMessage());
}

This would print out the names of all the files and subdirectories in the “/tmp” directory to the console. Note that you need to run this code with try-with-resources in order to properly close the Stream afterwards.

Filtering Directories in Java

If you only want to list certain types of files or directories, you can use the Files.list() method with a filter. The filter is specified as a Predicate object, and it can be used to match on attributes such as whether the file is a directory, or its size, or any other attribute of the Path object.

For example, you could list all the files in the “/tmp” directory that end with the “.txt” extension, like this:


          
           
Path dir = Paths.get("/tmp");
Predicate<Path> txtFilesOnly = path -> path.toString().endsWith(".txt");
try (Stream<Path> listing = Files.list(dir).filter(txtFilesOnly)) {
  listing.forEach(System.out::println);
}
catch (IOException e) {
  System.err.println("Error while listing files: " + e.getMessage());
}

The stream that’s returned by Files.list() will only contain files that match the filter predicate.

Listing Subdirectories in Java

The Files.list() method only lists the contents of a single directory. If you want to recursively list the contents of a directory and its subdirectories, you need to use the Files.walk() method. This method takes a Path object and a depth limit, and returns a Stream of all the Path objects in that directory and its subdirectories, up to the specified depth limit.

For example, to list all the files in the “/tmp” directory, including all its subdirectories, you could do this:


          
           
Path dir = Paths.get("/tmp");
try (Stream<Path> listing = Files.walk(dir, Integer.MAX_VALUE)) {
  listing.forEach(System.out::println);
}
catch (IOException e) {
  System.err.println("Error while listing files: " + e.getMessage());
}

The Integer.MAX_VALUE parameter indicates that the walk should traverse all subdirectories.

Filtering Subdirectories in Java

You can also use the Files.walk() method with a filter, in order to restrict the files that are listed. This works the same way as with the Files.list() method: you specify a Predicate and the returned Stream will only contain files that match the filter.

For example, to list all the files in the “/tmp” directory and its subdirectories, that are bigger than 100 bytes, you could do something like this:


          
           
Path dir = Paths.get("/tmp");
Predicate<Path> sizeBetterThan100Bytes = 
  path -> {
    try {
      return Files.size(path) > 100;
    } catch (IOException e) {
      return false;
    }
  };

try (Stream<Path> listing = Files.walk(dir, Integer.MAX_VALUE).filter(sizeBetterThan100Bytes)) {
  listing.forEach(System.out::println);
}
catch (IOException e) {
  System.err.println("Error while listing files: " + e.getMessage());
}

Here, the filter calls the Files.size() method to get the size of each file, and if it’s bigger than 100 bytes, the file is included in the returned stream.

Conclusion

In this article, we saw how to list the contents of a directory in Java, using the Files.list() and Files.walk() methods. We also looked at how to filter the results, using Predicate objects, and showed several examples of how to do this.
