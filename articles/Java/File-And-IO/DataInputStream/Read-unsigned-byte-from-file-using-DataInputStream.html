Reading Unsigned Bytes from a File using DataInputStream

Java has the 

          
           
DataInputStream
class, which allows you to read in primitive data types (such as bytes) from an InputStream. This can be useful when reading in binary data from a file, as the 

          
           
DataInputStream
class provides methods to easily read in data as primitive data types without having to manually convert it to the desired type. In this article, we will discuss how to use the 

          
           
DataInputStream
class to read in unsigned bytes from a file.

Creating a DataInputStream

In order to read from a file using a 

          
           
DataInputStream
, we must first create a 

          
           
DataInputStream
object. To do this, we can use the following code: 

          
           
FileInputStream fis = new FileInputStream("myfile.bin");
DataInputStream dis = new DataInputStream(fis);
This creates a 

          
           
DataInputStream
object that is connected to the 

          
           
FileInputStream
object, which is connected to the file specified in the constructor call. We can now use the 

          
           
DataInputStream
to read in bytes from the file.

Reading Unsigned Bytes

The 

          
           
DataInputStream
class provides two methods for reading in bytes from the stream: 

          
           
readByte()
and 

          
           
readUnsignedByte()
. The first method, 

          
           
readByte()
, reads in a signed byte from the stream. This means that the byte can be either positive or negative. The second method, 

          
           
readUnsignedByte()
, reads in an unsigned byte from the stream. This means that the byte is always positive, with a range of 0 to 255. For example, if we wanted to read in the first byte of our file, we would use the following code: 

          
           
int b = dis.readUnsignedByte();
This reads in the first byte of the file and stores it in the 

          
           
b
variable as an integer. Note that the byte is read in as an integer, not a byte, as the range of an unsigned byte is 0 to 255, which is too large for a byte to represent. Once we have read in the byte, we can then process it as we see fit. For example, if the byte was an ASCII character, we could then print out the corresponding character.

Closing the Streams

Once we are done reading from the file, it is important to close both the 

          
           
FileInputStream
and the 

          
           
DataInputStream
so that resources are released properly. To do this, we can use the following code: 

          
           
dis.close();
fis.close();
This closes both streams and releases any system resources they were using.
