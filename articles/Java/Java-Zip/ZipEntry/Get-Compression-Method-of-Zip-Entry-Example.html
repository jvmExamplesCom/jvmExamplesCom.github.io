Compression Method of Zip Entry Example with Java

Compressing data is an important task for applications that need to store and transfer information efficiently. Java provides an API, the java.util.zip package, to enable developers to incorporate robust file compression and decompression directly into their applications. This article will demonstrate the use of the java.util.zip API to compress data within a Java application.

ZipEntry API

The first step in using the java.util.zip API is to create a ZipEntry instance. This is simply a wrapper class for the data that is stored in the zip file. The constructor for the ZipEntry class takes a String which is the name of the zip file, and an optional int that sets the compression method.


          
           
//create a new zip entry
ZipEntry entry = new ZipEntry("myZipFile.zip", ZipEntry.DEFLATED);

The above example creates a new ZipEntry object called “myZipFile.zip” with the default compression method of DEFLATED. There are three possible methods for compressing the data within a ZipEntry: DEFLATED, STORED, and BZIP2.

Deflate Compression Method

DEFLATED is the most common method of compressing data and the one used in this example. It is a combination of the LZ77 algorithm and Huffman coding. The DEFLATED algorithm reduces redundancy in the data and stores it more efficiently, resulting in a smaller file size.


          
           
//set compression method to DEFLATED
entry.setMethod(ZipEntry.DEFLATED);

Stored Compression Method

The STORED compression method does not actually compress the data at all. Instead, it just stores the data in the ZipEntry as-is. This method is useful when compressing small files where the loss of data due to compression is greater than the benefit of reducing file size.


          
           
//set compression method to STORED
entry.setMethod(ZipEntry.STORED);

Bzip2 Compression Method

The BZIP2 compression method is based on the Burrows-Wheeler transform, and is more effective than DEFLATED for compressing large files. BZIP2 is slower than DEFLATED, however, so it may not be suitable for some applications where speed is a priority.


          
           
//set compression method to BZIP2
entry.setMethod(ZipEntry.BZIP2);

Once the ZipEntry object has been created and configured, it can be added to a ZipOutputStream, which will then write the compressed data to the zip file.

Example Code

The following example code demonstrates how to use the java.util.zip API to create a zip file containing a single text file. The text file is compressed using the DEFLATED compression method.


          
           
try {
  //create output stream
  FileOutputStream fos = new FileOutputStream("myZipFile.zip");
  ZipOutputStream zos = new ZipOutputStream(fos);

  //create new zip entry
  ZipEntry entry = new ZipEntry("myTextFile.txt", ZipEntry.DEFLATED);
  zos.putNextEntry(entry);

  //write content to entry
  Writer writer = new BufferedWriter(
                    new OutputStreamWriter(zos));
  writer.write("This is the content of myTextFile.txt");
  writer.flush();
  zos.close();
} catch (IOException e) {
  System.err.println(e);
}

This example code creates a new ZipOutputStream and a ZipEntry instance with the compression method set to DEFLATED. The ZipOutputStream is then used to write the content of the text file to the zip file.

Conclusion

The java.util.zip package provides a robust set of tools for developers to easily incorporate compression and decompression of data into their applications. This article demonstrated how to create a ZipEntry instance and how to write compressed data to a zip file using the three different compression methods available in the java.util.zip API.
